ObjectPurge&()

Syntax

ObjectPurge&(
    Object@)

 

Description

Purges the specified object from the Process Engine server cache using the OID.

Parameter

Parameter

Description

Object@

OID of the object to purge.

 

Returns

1 if successful; otherwise, an exception is raised.

Remarks

This function performs the same task as ObjectPurgeByKey&() but uses the object OID rather than the class and object keys. Purging an entire dynamic class impacts performance and is deprecated. A class purge request for dynamic and keyless classes is ignored. Entire static classes can be purged.

Examples

To purge all objects of the Issue_Type static class:

{
try {
    ObjectPurge&(Class:Issue_Type@);
    print('ObjectPurge succeeded');
    } except {
    print('ObjectPurge failed');
    }
}


To purge the Enquiry object in the Issue_Type class:


{
try {
     ObjectPurge&(StaticObj@('Issue_Type', 'Enquiry'));
     print('ObjectPurge succeeded');
     } except {
     print('ObjectPurge failed');
     }
}


Note:
When functions are called from Convergent Billing, Process Engine constants are not available (for example, Class:Issue_Type&).  Hard-coded constants are used instead (for example, 170).
