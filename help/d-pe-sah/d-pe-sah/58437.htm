Online Help for Administration Tool for Singleview Process Engine 9.1

Administration Tool Overview

The Process Engine server acts as an interface between users and Singleview, optimising and relaying requests for data and operations from user to database, and relaying requested data from database to user.

The Administration Tool allows the performance of Process Engine to be monitored by examining the server's activity. The reload configuration operation can be initialised from the Administration Tool when the server is in restricted mode.

To access the Administration Tool, an administrative user requires a profile that has the appropriate form role associated with it (see the Security and Auditing Configuration Guide for Singleview).

The tool has the following tabs:

  • System
  • Locking
  • Messages
  • Users
  • Reports
  • Debug Options
  • Debug Trace.

Options in the System menu also allow:

  • Separate specification for each tab (except System and Reports) of the frequency with which its details are to be updated (from 30 to 999 seconds).
  • Data displayed by the tab in use to be cleared.
  • Administrators to exit the tool.

Before Use

For information to be displayed in the Administration Tool, the appropriate check boxes must be selected. The first time the Administration Tool is used, perform the following:

  1. Select the System Options… option. The Options window is displayed.

     

  2. Select the following check boxes:
    • Get Lock Information in the Locking tab
    • Poll for Messages in the Messages tab
    • Get User Information in the Users tab.
  3. Click the OK button. The Administration Tool now displays the relevant information in each tab.
