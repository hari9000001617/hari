biReferenceCodeUpdate&()

Syntax

biReferenceCodeUpdate&(
    ReferenceTypeId&,
    var LastModified~,
    const OpCodes&[],
    const Codes&[],
    const FieldNames$[],
    const FieldValues?[])

 

Description

Inserts, updates, and deletes reference code details.

Attributes

Function Type

Remote Synchronous

Service

biFnEvaluateRW

Application Environment

Any

Context

Any

 

Parameter

Description

ReferenceTypeId&

In: ID of the reference type for the reference codes to be updated.

LastModified~

In: Last modified date-time of the reference types to be updated. Or, if using extended security, the last modified date-time of the reference code table of the associated reference type to be updated.

Out: Last modified date-time of the updated reference type. Or, if using extended security, the last modified date-time of the reference code table of the associated reference type.

OpCodes&[]

In: Operations to perform on each reference code specified in Codes&[].

For each record specified in Codes&[], the OpCodes&[] value at the same array position specifies the operation to perform on that record. For example, if OpCodes&[] = [2, 2, 3] and Codes&[] = [4, 5, 6], records 4 and 5 are updated and record 6 is deleted.

Codes&[]

In: Number used to identify a given reference code defined within the specified reference type to insert, update, or delete.

The number can be a reference code or ID, depending on the configuration of the reference type.

If the reference type supports extended features and does not have ALLOW_REFERENCE_CODE_IND_CODE set, and OpCodes&[] contains only insert operations, it can be an empty array.

FieldNames$[]

In: Names of the fields to insert or update for each record specified in Codes&[].

When deleting records, FieldNames&[] can be an empty array.

The REFERENCE_CODE_TRE_V view contains the field names that can be specified.

FieldValues?[]

In: Array of arrays containing the field values to insert or update for each record specified in Codes&[].

The field values in each nested array corresponds to the fields listed in FieldNames$[].

Using the example given above in the OpCodes&[] parameter, if FieldNames&[] = ['CODE_LABEL', 'DESCRIPTION'], then FieldValues?[] = [ ['Code Label 4', 'Code Description 4'], ['Code Label 5', 'Code Description 5'], [ ] ]. The last nested array is empty because the last record is being deleted.

When deleting records, FieldValues?[] can be an empty array.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

A reference code can only be modified if the user has sufficient privileges to update the:

  • Associated reference type, for non-extended reference types
  • Reference codes, for extended reference types.

When a reference code is updated, the LAST_MODIFIED value is also updated on the associated reference:

  • Type, for non-extended reference types
  • Code table, for extended reference types.

If the reference type is tenanted (the TENANT_IND_CODE is set to 1 and the implementation is tenanted):

  • A user can only insert, update, or delete reference codes associated with its tenant.
  • Each tenant has its own reference code table. A table is created for each tenant when the first reference code associated with the tenant is inserted.
