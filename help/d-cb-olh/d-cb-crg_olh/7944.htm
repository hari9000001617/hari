CONTRACT_HISTORY_SEARCH_V

The CONTRACT_HISTORY_SEARCH_V view defines the set of columns available for constraining contract searches, and for entity validation overrides.

The view is used internally by the system and system configurers.

Column Definitions

Column Name

Properties

Description

CONTRACT_NAME

VARCHAR (40)

Name of the contract, used as the external ID for the contract.

It is usually automatically assigned on contract creation by the contract types entity validation.

CUSTOMER_REFERENCE

VARCHAR (40)

Customer reference for the contract that allows the customer to identify the contract within their own accounts payable (or equivalent) system.

Convergent Billing does not enforce uniqueness on this column.

EFFECTIVE_DATE

DATE

Date-time of the search.

NODE_NAME

VARCHAR (200)

Name of the customer node associated with this contract.

Only root customer nodes can be associated with a contract.

PRODUCT_ID

NUMBER (8)

Product that can be associated with this contract.

SERVICE_TYPE_ID

NUMBER (8)

Type of service that can be associated with this contract.

CONTRACT_TYPE_ID

NUMBER (8)

Type of contract that entered into.

ACCOUNT_MANAGER_OPERATOR_ID

NUMBER (8)

Account manager for this contract.

Account managers must be defined as users of Convergent Billing.

It defaults from the contract type.

ASSIGNED_CSR_OPERATOR_ID

NUMBER (8)

User who nominally maintains this contract.

It defaults to the user who first created the contract.

CONTRACT_STATUS_CODE

NUMBER (8)

Status of this contract. Valid values are:

  • 1 (In Negotiation)
  • 2 (Pending)
  • 3 (Active)
  • 8 (In Grace)
  • 9 (Cancelled).

Of these values, only the statuses of Active and Cancelled have significance in the system.

Values are client configurable. It defaults from the CONTRACT_TYPE.

If a contract is active, all companion product instances associated with this contract must also be active for the active date range of the contract if their base product instance is active; otherwise, they must have a status of Provisioned, or the same status as their base product instance. It is verified by the DVP.

Users are not able to manually modify the status of companion products associated with a contract.

If a contract is cancelled, all companion product instances associated with this contract must also have a status of Cancelled. It is verified by the DVP.

Note:

No relationship exists between contract statuses and the status of base product instances or services associated with the contract.

FROM_EXPIRY_DATE

DATE

Start of the date-time range for contracts that have expired or are expected to expire.

TO_EXPIRY_DATE

DATE

End of the date-time range for contracts that have expired or are expected to expire.

FROM_RENEGOTIATE_DATE

DATE

Start of the date-time range for contracts due to be renegotiated.

TO_RENEGOTIATE_DATE

DATE

End of the date-time range for contracts due to be renegotiated.

FROM_EXPIRY_WARNING_DATE

DATE

Start of the date-time range for contracts whose expiry warning notification is due.

TO_EXPIRY_WARNING_DATE

DATE

End of the date-time range for contracts whose expiry warning notification is due.
