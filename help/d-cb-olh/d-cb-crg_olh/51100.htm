biOperatorUpdate&()

Syntax

biOperatorUpdate&(
    OperatorId&,
    var LastModified~,
    FieldNames$[],
    FieldValues?[])

 

Description

Updates the details, based on OPERATOR_TRE_V view, for the user specified by OperatorId& and maintains an associated Oracle user account, if necessary.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

OperatorId&

In: ID of the user to be updated.

LastModified~

In: Last modified date-time of the user to be updated.

Out: Last modified date-time of the updated user.

FieldNames$[]

In: Names of fields from OPERATOR_TRE_V view to be updated.

Read-only columns cannot be specified.

FieldValues?[]

In: Field values, in the same order that the field names are specified in FieldNames$[].

The PASSWORD field is used to update the user's login password.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

For information about validation requirements for user name (OPERATOR_LOGIN_NAME) and password (PASSWORD) fields, refer to the Security and Auditing Configuration Guide for Singleview.

The user calling the function must have appropriate privileges before proceeding with the requested changes. They must have:

  • Update privilege on the security group associated with the user record unless the caller is changing only the password of the caller's account.
  • Profile assignment privilege on the old and new profiles if they are changing the user profile.
  • Security Group update ('Update records associated with this security group') privilege on the old and new security groups for the user if they are changing the user's security group.
  • Superuser privilege (SUPERUSER_IND_CODE set to TRUE (1)) to enable superuser privilege for the user. Security Group update privilege for the user record is sufficient to disable superuser privilege.

If the PASSWORD field is specified, the user must have sufficient privileges to change the password; otherwise, an error is raised. Any necessary changes for the underlying Oracle user for the user, including enabling, locking, or expiring the account, are performed. The password is set, and the appropriate Oracle role is granted to the user.

If the only parameter passed to the function is to change a password, the last modified date is not affected. This preserves the cache integrity during Process Engine operations run through Process Engine web client.

An AUDIT_USER_UPDATE audit message is logged with the date of the change and name of the user who was updated successfully; otherwise, AUDIT_USER_UPDATE_ERROR is logged.

Field Processing Restrictions

Restrictions for field processing are:

  • OPERATOR_LOGIN_NAME is checked to ensure that it does not contain any quote characters, otherwise an error is raised.
  • OPERATOR_LOGIN_NAME can be modified only if the user account is disabled (ENABLED_IND_CODE is False (0)).
  • If ENABLED_IND_CODE is NULL and is specified but PASSWORD is not specified, an error is raised.
  • If EXPIRED_IND_CODE is NULL and is specified, the EXPIRY_DATE field is set to the current date; otherwise, EXPIRY_DATE is set to NULL.
  • If LOCKED_IND_CODE is NULL and is specified, the LOCK_DATE field is set to the current date; otherwise, LOCK_DATE is set to NULL.
