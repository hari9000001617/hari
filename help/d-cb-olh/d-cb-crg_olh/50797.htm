INV_INVOICE_V

The INV_INVOICE_V view defines all details of invoice or statement images for the IGP. The BGP generates invoices (with a 0 amount due) for nodes that have None specified for their report level.

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

ID of a customer node invoice.

INVOICE_SEQNR

NUMBER (3)

Not null

Sequence number of an invoice image for this invoice.

INVOICE_FORMAT_ID

NUMBER (8)

Not null

ID of the invoice format associated with this invoice image.

INVOICE_FORMAT_NAME

VARCHAR (30)

Not null

Name for the invoice format associated with this invoice image.

INVOICE_FORMAT_DISPLAY_
NAME

VARCHAR (60)

Not null

Name of the invoice format displayed.

It defaults to INVOICE_FORMAT_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

LAST_MODIFIED

DATE

Not null

Date-time when the invoice was last modified.

CUSTOMER_INVOICE_STR

VARCHAR (30)

Not null

ID generated for this invoice.

The format depends on the specific implementation, but is usually derived from INVOICE_ID.

INVOICE_TYPE_ID

NUMBER (8)

Not null

Redundant

Type of this invoice, derived from the account type of the invoice's account.

BILL_RUN_ID

NUMBER (15)

Not null

ID of the bill run that generated this invoice.

RUNNING_IND_CODE

NUMBER (1)

Set to 1 (True) while the billing engine is in the process of calculating the value for this invoice or statement.

It can be used to quickly identify incomplete invoices associated with a failed bill run that need to be revoked.

QA_IND_CODE

NUMBER (1)

Set to 1 (True) if this is an invoice or statement generated for a Quality Assurance or Statement only bill run.

It can be derived from the bill run when the invoice or statement is originally created. If it is not 1, it must be null.

Records with this set must not be applied to accounts.

SUPPRESS_IND_CODE

NUMBER (1)

Set to 1 (True) if this invoice is marked for suppression.

It is set by the invoice type expressions, and is not applicable and cannot be set for statements or Quality Assurance invoices.

If all invoices in a customer hierarchy have this flag set for a bill run, the invoices and statements are immediately revoked, including any rental charges and adjustments run for this customer hierarchy and bill run. It is as though the bill run was not run for this hierarchy.

The CUSTOMER_NODE_BILL_RUN table indicates that invoice generation for the bill run was suppressed for this hierarchy.

Revoking of suppressed invoices is performed by the bill run operation.

If only some of the invoices in a customer hierarchy have this flag set for a bill run, it has no effect. However, the flag remains set for those invoices in the hierarchy that qualified for suppression.

ACCOUNT_ID

NUMBER (15)

Not null

ID of the account to which this invoice was applied.

ACCOUNT_CURRENCY_ID

NUMBER (8)

Not null

Currency of the account, and in which the invoice is issued.

ACCOUNT_TYPE_ID

NUMBER (8)

Not null

Type of the account.

ACCOUNT_NAME

VARCHAR (30)

Not null

Name of the account.

ACCOUNT_TYPE_NAME

VARCHAR (30)

Not null

Name for the type of account.

ACCOUNT_TYPE_DISPLAY_NAME

VARCHAR (60)

Not null

Name of type of account displayed. Defaults to ACCOUNT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

ACCOUNT_CLASS_CODE

NUMBER (2)

Not null

Class that an account belongs to. Valid values are:

  • 1 (Asset)
  • 2 (Liability).

ACCOUNT_CATEGORY_CODE

NUMBER (2)

Not null

Category that an account belongs to. Valid values are:

  • 1 (Receivable)
  • 2 (Payable)
  • 3 (Deposit)
  • 4 (Credit)
  • 5 (Bank (cash)).

CUSTOMER_ACCOUNT_INVOICE_
STR

VARCHAR (30)

Optional invoice ID generated for this invoice.

If defined, it is only unique within the account. Its use is implementation-dependent.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

Derived

Customer node that received this invoice, derivable from the ACCOUNT_ID.

IMAGE_GENERATED_IND_CODE

NUMBER (1)

Set to True (1) when all images for the invoice have been successfully processed by the IGP, including images that have been suppressed. It is set by the IGP.

APPLIED_IND_CODE

NUMBER (1)

Set to 1 (True) when this invoice is applied to the account.

It is set by the apply_invoices server utility, as appropriate. After an invoice is applied, it cannot be modified in any way (except for flagging it for reprint).

Any account corrections must be made using adjustments displayed on the next invoice run for the customer.

EFFECTIVE_DATE

DATE

Not null

Effective date of the bill run that generated this invoice.

ISSUE_DATE

DATE

Not null

Date on which this invoice was issued.

ORIGINAL_PAYMENT_DUE_DATE

DATE

Not null

Date on which this invoice was originally due to be paid.

PAYMENT_DUE_DATE

DATE

Not null

Date on which this invoice is due to be paid, usually the same as ORIGINAL_PAYMENT_DUE_DATE, but it can be updated by privileged users.

EARLY_PAYMENT_DATE

DATE

Not null

If the customer pays before this date, they receive an additional discount.

The discount will have been incorporated into the INVOICE_TOTAL.

INVOICE_AMOUNT

NUMBER (19,3)

Not null

Total amount of charges associated with this invoice, excluding the balance forward.

If handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.

If handling of liability accounts is not enabled, a positive value is a debit for an asset account and a credit for a liability account.

Values stored in this column are at the limits of double precision.

STATEMENT_AMOUNT

NUMBER (19,3)

Not null

Total amount of charges associated with this statement, excluding the balance forward.

For secondary accounts, if handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.

BALANCE_FORWARD

NUMBER (19,3)

Not null

Balance on the account that was present before application of this invoice.

ACCOUNT_BALANCE

NUMBER (19,3)

Not null

Derived

Account balance after this invoice was generated.

ACCOUNT_INITIAL_DUE

NUMBER (19,3)

Not null

Initial amount due to be paid on this account.

It is usually equal to ACCOUNT_BALANCE, but can be different because of cents rounding or the presence of payment plans (not supported) for the account.

CURRENT_DUE

NUMBER (19,3)

Not null

Derived

Amount left to be paid on this invoice.

TOTAL_PAYMENTS

NUMBER (19,3)

Not null

Total payments made on this account between the issuing of the previous invoice and this invoice.

The payments are not made against this invoice, and the amount is usually displayed on this invoice.

It is calculated by the billing engine.

TOTAL_ADJUSTMENTS

NUMBER (19,3)

Not null

Total adjustments made on this account between the issuing of the previous invoice and this invoice.

The adjustments are not made against this invoice, and the amount is usually displayed on this invoice.

It is calculated by the billing engine.

EARLY_PAYMENT_DISCOUNT

NUMBER (19,3)

Not null

Derived

Early payment discount (if any) applied for this invoice.

It is derived by the billing engine, and can be re-derived, based on details in the CHARGE table.

PAYMENT_DUE_DISCOUNT

NUMBER (19,3)

Not null

Derived

Pay on time discount (if any) that was applied for this invoice.

It is derived by the billing engine, and can be re-derived, based on details in the CHARGE table.

EARLY_PAYMENT_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the EARLY_PAYMENT_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set this column if necessary to record the account balance at the early payment date. For example, this could be set by a collections or treatment module.

PAYMENT_DUE_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the PAYMENT_DUE_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set it if necessary to record the account balance at the date the invoice is due to be paid. For example, it could be set by a collections or treatment module.

PENDING_CONSOLIDATION_IND_
CODE

NUMBER (1)

Set to 1 (True) if the statement was an invoice flagged by the BGP to be suppressed for the current bill cycle.

If set to 1:

  • STATEMENT_AMOUNT must be specified.
  • Statement image is not generated.

If not 1, it must be null.

CONSOLIDATION_INVOICE_ID

NUMBER (15)

ID of the invoice created to consolidate the statement and other statements for the same account into a single invoice.

If specified:

  • STATEMENT_AMOUNT must be specified.
  • PENDING_CONSOLIDATION_IND_CODE must be null.

CONSOLIDATED_INVOICE_IND_
CODE

NUMBER (1)

If set to 1 (True):

  • The record is a consolidated invoice for one or more statements on this account.
  • STATEMENT_AMOUNT must be null.
  • There must be one or more rows in the INVOICE table that reference this INVOICE_ID as their CONSOLIDATION_INVOICE_ID, have the same ACCOUNT_ID as this invoice, and a STATEMENT_AMOUNT sum that equals this invoice's INVOICE_AMOUNT.

If not set to 1, it must be null.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10.

Their use and meaning is controlled by entity validation on the invoice type.

They are populated as required by invoice type expressions.

INVOICE_FORMAT_GENERAL_1 -
INVOICE_FORMAT_GENERAL_10

VARCHAR (255)

General fields 1 - 10 associated with this invoice image for this customer node.

Their meaning is dependent on the invoice format.
