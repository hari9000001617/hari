NORM_ERROR_TYPE_V

The NORM_ERROR_TYPE_V view is a join of the NORMALISED_EVENT_ERROR, NORMALISED_EVENT_FILE, and NORMALISED_EVENT_TYPE tables. It is used to generate the CALL_DATA_ERROR_CV view.

Note:

The join must be defined before CALL_DATA_ERROR_CV.

Column Definitions

Column Name

Properties

Description

NORMALISED_EVENT_ID

NUMBER (15)

Not null

ID of a normalised event.

NORMALISED_EVENT_TYPE_ID

NUMBER (8)

Not null

ID of a normalised event type.

NORMALISED_EVENT_TYPE

VARCHAR (30)

Not null

Name for this normalised event type, as known by users.

NE_TYPE_DISPLAY_NAME

VARCHAR (30)

Not null

Name of normalised event type displayed.

It defaults to NORMALISED_EVENT_TYPE.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

NORMALISED_EVENT_FILE

VARCHAR (255)

Not null

ID of the file from which this normalised event record was obtained.

All events are associated with a file, even if the event was not extracted from a file on the system.

ERROR_MESSAGE

VARCHAR (4000)

Not null

Error message text explaining why this event failed to be processed.

It should include details on the field in error.

A_PARTY_ID

VARCHAR (40)

Full national number of the calling party (A party).

A_PARTY_NAME

VARCHAR (40)

Name of the A party, if known.

For example, this can be provided by the switch for user-assisted calls.

A_PARTY_CARRIER_CODE

NUMBER (8)

Carrier from which the call originated.

A_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the origin of the call to the finest granularity supported by the system.

Higher forms of location such as country and continent can be derived from this value.

B_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for the B party service.

B_PARTY_NAME

VARCHAR (40)

Name of the B party, if known.

For example, this can be provided by the switch for user-assisted calls.

B_PARTY_CARRIER_CODE

NUMBER (8)

Carrier to which the call is destined (may not be known).

B_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the destination of the call to the finest granularity supported by the system.

Higher forms of location such as country and continent can be derived from this value.

C_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for a chargeable party service.

It is usually equal to A_PARTY_ID, but is equal to B_PARTY_ID party for reverse charge and 008 calls, and some other third party for conference calls.

C_PARTY_NAME

VARCHAR (40)

Name of the C party, if known.

For example, this can be provided by the switch for user-assisted calls.

C_PARTY_CARRIER_CODE

NUMBER (8)

Carrier associated with the chargeable party.

C_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the destination of the chargeable party to the finest granularity supported by the system.

Higher forms of location such as country and continent can be derived from this value.

CHARGE_START_DATE

DATE

Date-time for charging purposes, usually derived from the switch date.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

DURATION

NUMBER

Duration of call.

The units depend on event type.

For rentals, the duration can be a decimal (a fraction of the standard rental period).

CHARGE

NUMBER (22, 6)

Charge as it was received from the switch.

The units are dependent on the switch.

CURRENCY_ID

NUMBER (8)

Currency for the above charge.
