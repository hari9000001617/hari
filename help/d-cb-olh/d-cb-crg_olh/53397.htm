PROFILE_TRE_V

The PROFILE_TRE_V view provides information about supported security profiles.

Column Definitions

Column Name

Properties

Description

PROFILE_ID

NUMBER (8)

Not null

ID of a security profile.

PROFILE_NAME

VARCHAR (60)

Not null

Full name of a profile.

If the profile is linked to an Oracle profile, the profile name must conform to Oracle's naming standard for profiles:

  • The name must be specified in all uppercase characters.
  • The first character must belong to the set [A-Z].
  • Subsequent characters must belong to the set [A-Z, _, 0-9].

DESCRIPTION

VARCHAR (255)

General description of the profile.

LAST_MODIFIED

DATE

Not null

Date-time at which the record was last modified.

OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified the record.

GROUP_ID

NUMBER (8)

Not null

Security group associated with, and control of, the record.

PRIMARY_GROUP_ID

NUMBER (8)

Not null

Security group associated with data created by users using this profile.

CONFIGURATION_GROUP_ID

NUMBER (8)

Not null

Security group associated with configuration data created by users using this profile.

If not specified, it defaults to PRIMARY_GROUP_ID.

MAX_ADJUSTMENT_AMOUNT

NUMBER (19,3)

Not null

Maximum value of an adjustment that can be performed by users using the profile.

ADJUSTMENT_CURRENCY_ID

NUMBER (8)

Not null

Currency of the maximum adjustment amount.

ACCESS_LEVEL

NUMBER (8)

Not null

Bit mask defining access options permitted for users associated with the profile.

Options operate independently of each other.

MAX_CONCURRENT_LOGINS

NUMBER (3)

Maximum number of concurrent logins by each user.

If not specified, there is no limit.

If specified, the value must be between 1 and 999 inclusive.

When attempting to log in after the maximum number of logins is reached, and if TRE_LOGIN_OVERRIDE or TRE_PE_OVERRIDE have been passed as an option to treConnectFlags(), one of the existing connections is terminated and the login attempt succeeds.

PASSWORD_FUNCTION_DEFN_
ID

NUMBER (10)

Name of an EPM function to validate a password.

The function must be defined as a local function and have an interface of:

<Function>&(const OldPasswd$, const NewPasswd$)

The function must return TRUE (1) if the new password passes validation. An error must be raised if the password does not pass validation. The error message must be defined in the database, and must not contain any parameters in the message text. An undefined value is passed for OldPasswd$ if the user did not have a password specified.

The SVPasswordVerify&() default password verification function, is supplied.

A password validation function can be specified only if ACCESS_LEVEL has Internal Authentication set.

ORACLE_PROFILE_ENABLE_
IND_CODE

NUMBER (1)

Oracle profile flag.

If 1 (TRUE), an Oracle profile is maintained in conjunction with the user profile.

The Oracle user account, for each user assigned to the profile, is assigned to the corresponding Oracle profile.

If not set to 1 (TRUE), it must be NULL.

 

Oracle Profile Password Attributes

Column Name

Properties

Description

FAILED_LOGIN_ATTEMPTS

NUMBER (3)

Number of consecutive failed login attempts allowed before locking the user's account.

It can contain a value only if ORACLE_PROFILE_ENABLE_IND_CODE is set.

PASSWORD_LOCK_TIME

NUMBER (3)

Number of days an account remains locked after FAILED_LOGIN_ATTEMPTS+1 consecutive failed login attempts.

It can be specified only if FAILED_LOGIN_ATTEMPTS has a value.

PASSWORD_LIFE_TIME

NUMBER (3)

Number of days the current password can be used for authentication.

The password expires if it is not changed within the specified period.

It can contain a value only if ORACLE_PROFILE_ENABLE_IND_CODE is set.

PASSWORD_GRACE_TIME

NUMBER (3)

Number of additional days after the first attempt to log in after a password has expired that logins are still accepted.

If the password is not changed during the grace period, subsequent logins are denied.

It can contain a value only if PASSWORD_LIFE_TIME has a value.

ENABLE_PASSWORD_REUSE

NUMBER (1)

Password reuse flag.

Password reuse means that a password can be reused even though the same password was used previously.

If NULL and Oracle profiles are in use, the Oracle profile does not permit password reuse.

If TRUE (1), password reuse is enabled for users with the profile.

It can contain a value only if ORACLE_PROFILE_ENABLE_IND_CODE is set.

PASSWORD_REUSE_MAX

NUMBER (3)

Number of times a password must be changed before a previously used value can be specified.

A password must satisfy both the PASSWORD_REUSE_MAX and PASSWORD_REUSE_TIME constraints before it can be reused.

It can be specified only if ENABLE_PASSWORD_REUSE is set. Both PASSWORD_REUSE_MAX and PASSWORD_REUSE_TIME must be NULL or both must have a value.

PASSWORD_REUSE_TIME

NUMBER (3)

Number of days before a password can be reused.

A password must satisfy both the PASSWORD_REUSE_MAX and PASSWORD_REUSE_TIME constraints before it can be reused.

It can be specified only if ENABLE_PASSWORD_REUSE is set. Both PASSWORD_REUSE_MAX and PASSWORD_REUSE_TIME must be NULL or both must have a value.

Column Name

Properties

Description

CPU_PER_CALL

NUMBER (4)

Number of seconds of CPU that can be used in processing a single SQL statement.

It can be specified only if ORACLE_PROFILE_ENABLE_IND_CODE is set and either of the access options Database Direct Read Only or Database Direct Full Access of ACCESS_LEVEL is set.

It can be specified only if ORACLE_PROFILE_ENABLE_IND_CODE is set and either of the access options Database Direct Read Only or Database Direct Full Access of ACCESS_LEVEL is set.

ACCESS_LEVEL Options

The following table lists PROFILE_TRE_V access-level options.

 

ACCESS_LEVEL Options

Bit Position (value)

Option Name

Description

0 (1)

Login Enabled

Must be set to allow users using the profile to log in to Singleview.

The option is independent of the database direct access options, and clearing the flag does not disable the user from accessing the database.

1 (2)

Internal Authentication

Enables log in using a user name and password.

Authentication is implemented using an Oracle user account with the same name as the user.

The Oracle account is created for validation purposes and does not permit direct access to data in the database.

2 (4)

External Authentication

Enables log in by external authentication using treConnectExternal&().

3 (8)

CB Client Access

Enables connection through the TRE API using Convergent Billing connection and transaction semantics.

The bit must be set for all profiles used by users of the Convergent Billing client.

4 (16)

PE Client Access

Enables connection through the TRE API using Process Engine connection and transaction semantics.

The bit must be set for all profiles for users who need to use the Process Engine client.

5 (32)

Database Direct Read Only

Permits direct connection to the database through another application with read-only access to all Singleview tables and views.

It can be set only if:

  • Internal Authentication is selected.
  • TENANT_ID is NULL.

6 (64)

Database Direct Full Access

Permits direct connection to the database through another application, with read and write access to all Singleview tables and views.

It can be set only if:

  • Internal Authentication is selected.
  • TENANT_ID is NULL.

7 (128)

Self-service Password Reset

Enables self-service password reset.

It can be set only if Internal Authentication is selected.
