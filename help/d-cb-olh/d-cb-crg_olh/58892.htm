biFileRegisterUpdateSepTrans&()

Syntax

biFileRegisterUpdateSepTrans&(
    FileRegisterId&,
    var LastModified~, 
    const FieldNames$[], 
    FieldValues?[])

 

Description

Updates a file register record.

This function is executed as a separate transaction, and is a wrapper function around the FileRegisterUpdate&() function.

Caution:

To avoid deadlock issues, this function cannot be called from a transactional server (for example, from within another remote EPM function call).

 

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

FileRegisterId&

In: ID of the record to be updated.

LastModified~

In: Date-time the record was last modified before this update.

Out: Date-time when the record was updated.

FieldNames$[]

In: Names of the fields from the FILE_REGISTER_TRE_V view whose values are to be updated.

FieldValues?[]

Out: Field values in the same order in which the field names are specified in the FieldNames$[].

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

This function is designed to allow an external process to update the state and the statistics of the file during processing.

It maintains the physical file, triggering an automatic move, rename, or deletion based on the values set in PHYSICAL_FILENAME and PHYSICAL_DIRECTORY. To prevent this function from updating the physical file system and only update the table record, REGISTER_ONLY_IND_CODE must be set to 1(True).
