ProductOfferFetch?[]()

Syntax

Interface 1

 

Interface 2

 
ProductOfferFetch?[](
    const ProductOfferId&,
    const FieldNames$[])
 
 
 
ProductOfferFetch?[](
    const ProductOfferName$,
    const FieldNames$[])

 

Description

Retrieves details of a product offer from the product offer cache. If the product offer does not exist in the cache, it is retrieved from the database and stored in the cache.

Attributes

Function Type

Built-in

Service

(Local)

Application Environment

Any

Context

Any

 

Parameter

Description

ProductOfferId&

In: ID of product offer to retrieve.

ProductOfferName$

In: Name of product offer to retrieve.

FieldNames$[]

In: List of field names to retrieve. Names are not case-sensitive.

If a requested field has no value for the retrieved product offer, an undefined value is returned for that field.

Names of valid fields are the same as the names of the columns in the PRODUCT_OFFER table, except that the following columns cannot be specified in FieldNames$[]:

  • ATLANTA_GROUP_ID
  • ATLANTA_OPERATOR_ID
  • CATEGORIES
  • DESCRIPTION
  • LAST_MODIFIED
  • VERSION_STR.

The table below lists field names that are also valid.

 

Returns

Array of values corresponding to, and in the same order as, the fields specified in FieldNames$[]. If an invalid field name is requested, or if the specified product offer does not exist, an error is raised.

Remarks

When fetching by:

  • ProductOfferId&, a product offer is retrieved if security groups have appropriate permissions.
  • ProductOfferName$, a product offer is retrieved based on the product offer name and tenant ID belonging to effective user. Where no tenant is assigned, the product offer name without a tenant ID is retrieved.

The following table lists valid fields for FieldNames$[].

 

Valid Field Names

Field Name

Description

CHILD_PRODUCT_OFFER

Array of details of child product offers for this product offer.

Each element in the array is a hash with the keys listed in the table below.

Its value is undefined if this product offer has no child product offers.

COMPANION_PRODUCT

Array of details of companion products contained in this product offer.

Each element in the array is a hash with the keys listed in second table below.

Its value is undefined if this product offer contains no companion products.

PRODUCT_OFFER_NODE_ID

Array of product offer node IDs that reference this product offer.

It contains one ID if the product offer is a root product offer; otherwise, it contains one or more IDs.

SERVICE_TYPE_ID

Array of IDs of all service types with which this product offer is compatible.

It contains at least one ID if the product offer is service-specific; otherwise, it is undefined.
