InstanceFunctions

The InstanceFunctions derived attribute table identifies functions that require instance-specific versions to be created for use in a multi-instance environment.

The table is used primarily by the biClusterConfigure&() function.

The table is applicable only for multi-instance configurations.

Column Definitions

Column Name

Properties

Description

Function Name (Index 1)

String (67)

Not null

Name of a function that needs instance-specific versions created for use in a multi-instance environment.

Interface Nr (Index 2)

Integer

Not null

Interface number of the function requiring duplication.

It is necessary to identify overloaded functions.

Remote Service Name (Result 1)

String (14)

Not null

Remote service name of the function identified by the first two columns.

It is provided to make biClusterConfigure&() easier to use by not having to retrieve the remote service name from the database.

Modify Text? (Result 2)

Integer

Not null

If set to:

  • 1 (TRUE), the function's definition text needs to be modified for the instance-specific versions of the function.
  • 0 (FALSE), the function definition text for the instance-specific versions of the function is identical to the base version.

    Note:

    It uses the BOOLEAN attribute type to provide TRUE and FALSE options.

For more information, refer to the biClusterConfigure&() function in the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2).

Auto Include Parameters? (Result 3)

Integer

Not null

If set to:

  • 1 (TRUE), all the function's parameters are to be included in any instance-specific versions of the function, regardless of the presence or absence of #<INSTANCE> and #</INSTANCE> tags in the parameter block.
  • 0 (FALSE), only parameters between the #<INSTANCE> and #</INSTANCE> tags are included in an instance specific version of the function.

The value is irrelevant if Modify Text? is set to 0 (FALSE).

Note:

It uses the BOOLEAN attribute type to provide TRUE and FALSE options.

Adjust Queuespace? (Result 4)

Integer

Not null

If set to:

  • 1 (TRUE), the queuespace name of instance-specific functions derived from the function are changed using the format <Queuespace Name><INSTANCE NAME>.

    The remote service name of the instance specific functions is not changed.

  • 0 (FALSE), no changes are made to the queuespace names of the instance-specific functions and the remote service name of the instance-specific functions are changed as described in the function syntax for biClusterConfigure&().

For more information, refer to the biClusterConfigure&() function.

Note:

It uses the BOOLEAN attribute type to provide TRUE and FALSE options.
