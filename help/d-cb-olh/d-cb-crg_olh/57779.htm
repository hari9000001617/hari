BatchOutbound

This interface type is used to configure file extraction logic.

Note:

Users of this interface type must have insert and update security group access level for:

  • ED_AUTO, the security group associated with automatically-generated external definition functions.
  • IFD_AUTO, the security group associated with automatically-generated interface definition functions.

 

Interface Type Parameters

Tab

Parameter

Type

Description

File Process Manager

Processing Directory

[Text]

Path of the directory where the processed file is saved.

File Group

[Text]

UNIX group associated with the file.

If it is specified and Local Server File Secure is specified, the directories and files generated have the specified UNIX group. The group is set only on creation of the directory or the file.

For more information, refer to the System Configuration Guide for Singleview Convergent Billing.

Action on Error

[List]

Action to perform on the processed file if an error occurs during processing. Valid values are:

  • Delete File

    File is deleted.

  • Rename File

    File is renamed to the name specified by Error File Pattern.

  • Rename and Move File

File is renamed to the name specified by Error File Pattern and moved to the directory specified by Error Directory.

  • Move File

    File is moved to the directory specified by Error Directory.

  • None

    File is left unchanged.

It is based on the BIMFPM_EXTRACT_ERROR_ACTION_TYPE reference type.

Error Directory

[Text]

Directory into which the processed file is moved if the batch extraction terminates due to errors.

It is mandatory if Action on Error is set to one of the following:

  • Move File
  • Rename and Move File.

Error File Pattern

[Text]

Name to which the processed file is renamed, if the batch extraction terminates due to errors.

By default, it is set to <task>.<filename>.

It is mandatory if Action on Error is set to one of the following:

  • Rename File
  • Rename and Move File.

Error File Pattern can contain the following tags:

  • <basename>: File name of the processed file excluding the extension.
  • <filename>: File name of the processed file including the extension.
  • <task>: ID of the task that processed the file.
  • <date>: Current date in the format yyyymmdd.
  • <datetime>: Current date-time in the format yyyymmddhhnnss.

Batch Extractor

Filter Key

[Text]

Key used to tag the record for later identification and filtering.

Records per Transaction

[Integer]

Number of processed records to be committed per transaction. Valid values are:

  • 0 or blank: All records are committed in a single transaction.
  • <Number>: Number of records that are committed per transaction. By default, this value is set to 1000.

File Process Manager Static

Filename Expression

Text

EPM expression that returns a string with the name of the processed file, not including the file path.

File Type

List

File data group classification (for example, Payments).

File Completeness Method

List

Method used to confirm that the generated file is ready to be used.

Temporary File Pattern

[Text]

File name to use while the file is being processed.

Only applicable if File Completeness Method is set to Filename.

At the completion of file generation, the file is renamed to Filename.

The pattern can contain the following tags:

  • <basename>: File name of the processed file excluding the extension.
  • <filename>: File name of the processed file including the extension.

Control File Pattern

[Text]

Name of the control file.

Only applicable if File Completeness Method is set to Control File.

At the completion of file generation, the control file and the processed file are created.

The pattern can contain the following tags:

  • <basename>: File name of the processed file excluding the extension.
  • <filename>: File name of the processed file including the extension.

Checksum

[List]

Checksum algorithm added to the control file.

If not selected, the presence of the control file is used to indicate completeness.

If selected, both the presence and the content of the control file indicate completeness. The content is set to the hash of the processed file contents using the selected algorithm.

Local Server File Secure

Yes/No

Specifies if UNIX directory and file permissions are to be set for files on the local server file system.

The directory's permissions are set to allow only Singleview and users in the UNIX group specified by File Group to access the directory and its files.

The directory's permissions are set only on creation. On completion of the batch extraction, the file's permissions are set to read-only (user and group)

For more information, refer to the System Configuration Guide for Singleview Convergent Billing.

Batch Extractor Static

Delayed Extract

Yes/No

Indicates if the processing is a delayed batch extraction.

If set to Yes, the:

  • Delayed Batch Extractor and the Delayed External Definition Static tabs are displayed.
  • External Definition Static tab is not displayed.

Audit

Yes/No

Indicates if the batch extraction has auditing enabled.

Header

Yes/No

Indicates whether a header record is to be created as the first record of the processed file.

File Format

List

Format of the processed file.

Custom Format Function

[Text]

Name of the custom-formatting function used to generate the output text.

It is only used if File Format is set to Custom.

The custom-formatting function must match the syntax of the <CustomExtractFormatFuncName>&()>&() function.

XML Root Element Name

[Text]

Name given to the XML root element.

If it is not specified, it is set to the value of External Definition suffixed with the word List (that is, <External_Definition_Name>List).

It is applicable only if File Format is set to XML.

It is ignored if the specified external definition is an abstract external definition.

XML Data Element Name

[Text]

Name given to the parent element of each extracted record.

If not specified, it is set to the value of External Definition.

It is applicable only if File Format is set to XML.

It is ignored if the specified external definition is an abstract external definition.

Character Encoding

List

Character set of the processed file.

Encoding rules apply.

It is based on the BIMBP_CHARACTER_ENCODING_TYPE reference type.

External Definition Static

Note:

It is only displayed if the Delayed Extract field is set to No in the Batch Extractor Static tab.

Namespace

[List]

Namespace prefixes or names.

External Definition

[List]

External definitions that belong to the specified namespace.

Name

[Text]

External definition element.

Only elements with an assigned XML format type are included in the drop-down list.

Expression

[Text]

EPM expression that returns a value expected for the selected external definition element.

Delayed External Definition Static

Note:

It is only displayed if the Delayed Extract field is set to Yes in the Batch Extractor Static tab.

Namespace

[List]

Namespace prefixes or names.

External Definition

[List]

External definitions that belong to the specified namespace.
