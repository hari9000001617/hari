CALL_DETAIL_V

The CALL_DETAIL_V view displays details of normalised events and their associated charges.

The view is used by the Call Details form and its associated report.

Any query on the view should include a CONVERSION_CURRENCY_ID equality constraint; otherwise, multiple rows may be returned for each charge.

Column Definitions

Column Name

Properties

Description

CHARGE_ID

NUMBER (15)

Not null

Charge record associated with this normalised event.

SERVICE_ID

NUMBER (15)

Service associated with this normalised event, if any.

ACCOUNT_ID

NUMBER (15)

Not null

Account to which this charge was allocated.

All charges (billable and non-billable) must be associated with an account.

The customer node is identified for the aggregation of the charges by the billing engine using the account.

INVOICE_ID

NUMBER (15)

Invoice associated with this charge record, if any.

If this is null, the charge is not invoiced.

UNINVOICED_IND_CODE

NUMBER (1)

Set to 1 (TRUE) if this charge is not assigned to an invoice.

All records entered by processes other than the BGP have this column set to 1.

The BGP clears this column after it has processed the charge and assigned it to an invoice.

NORMALISED_EVENT_ID

NUMBER (15)

Not null

ID of the normalised event.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

Properties

Chargeable (C) Party Details

Column Name

Properties

Description

C_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for a chargeable party service.

It is usually equal to A_PARTY_ID, but is equal to B_PARTY_ID for reverse charge, 008 calls, and some other third party for conference calls.

C_PARTY_NETWORK_ID

VARCHAR (40)

Network ID of the chargeable party (C party), if any.

For a GSM service, this corresponds to the IMSI number.

At least one of the party details (C_PARTY_ID or C_PARTY_NETWORK_ID) must be specified, although this is not enforced for performance reasons.

C_PARTY_NAME

VARCHAR (40)

Name of the C party, if known.

For example, this can be provided by the switch for user-assisted calls.

C_PARTY_INTERNAL_ID

NUMBER (15)

ID of the chargeable party.

It is used by the Rental Generation Process (RGP) to guarantee identification of the service involved.

If provided, it is used instead of the C_PARTY_ID or C_PARTY_NETWORK_ID to perform the service lookup.

Note:

External data feeds must not populate this column. If C_PARTY_INTERNAL_ID is set and the EVENT_CLASS_CODE is not Usage (1), the billing engine includes any charges associated with this event in the current bill cycle, even if they have a CHARGE_START_DATE after the bill cycle effective date.

C_PARTY_TON_CODE

NUMBER (8)

Charge party type of number.

C_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the origin of the call to the finest granularity supported by the system.

 

Miscellaneous Details

Column Name

Properties

Description

FULL_PATH

VARCHAR (40)

Available details relating to the route that a call used (for example, X.25 DNIC sequence for call).

It can alternatively be used to store the direction or type of call (for example, incoming, outgoing, internal, transit).

CHARGE_START_DATE

DATE

Date-time for charging purposes.

It is usually derived from the switch date.

PERIOD_START_DATE

DATE

Date-time of the start of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

PERIOD_END_DATE

DATE

Date-time of the end of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

EVENT_CLASS_CODE

NUMBER (2)

General classification of the event for reporting purposes.

Valid values are:

  • 1 (Usage event): The event is in relation to the usage of a service
  • 2 (Recurring event): The event represents a rental period, typically generated by the RGP.
  • 3 (One-off or ad hoc event): These include automatically generated one-off events (generated by the RGP), and ad hoc events generated by users or other sources.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

DURATION

NUMBER

Duration of call.

The units depend on event type.

VOLUME

NUMBER

Volume of data associated with call.

The units depend on the call type (for example, bytes or segments).

If populated by events generated by the RGP, its contents should be specified as volume per unit duration.

PULSES

NUMBER (8)

Number of meter pulses associated with call.

If populated by events generated by the RGP, its contents should be specified as pulses per unit duration.

RATE_BAND

VARCHAR (40)

Time or rate band that was calculated for this call.

 

Charge Details

Column Name

Properties

Description

ORIGINAL_CHARGE

NUMBER (22,6)

Not null

Value of this charge in its ORIGINAL_CURRENCY_ID.

PROPORTION

NUMBER (22,6)

Proportion of the total charge that this charge corresponds to.

It only contains a value (usually 1) if the charge is associated with a tariff.

ORIGINAL_CURRENCY_ID

NUMBER (8)

ID of the currency in which this charge was made.

It is mandatory for all charges, but is not required for subtotals that are aggregations of non-currency amounts.

CONVERSION_CURRENCY_ID

NUMBER (8)

ID of the currency to which the charge was converted.

CONVERSION_RATE

NUMBER (22,12)

Value for converting from FROM_CURRENCY_ID to TO_CURRENCY_ID for the period defined by the start and end dates.

CHARGE

NUMBER (15,6)

Value of this charge in the currency specified by CONVERSION_CURRENCY_ID.

TARIFF_ID

NUMBER (8)

Tariff, discount, or tax that generated this charge, if any.

TARIFF_NAME

VARCHAR (30)

Name of the tariff, discount, or tax that generated this charge, if any.

TARIFF_TYPE_CODE

NUMBER (8)

Numeric code that defines the type of the tariff.

INVOICE_SECTION_NR

NUMBER (8)

Invoice section number associated with this charge.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number associated with this charge.

INVOICE_TXT

VARCHAR (255)

Optional invoice text that can be displayed in association with this charge.

KEY_VALUE

VARCHAR (255)

Hash key value or array index value associated with this charge for this tariff or subtotal.

It is only populated for charges generated by hash or array tariffs and subtotals. For:

  • Hash based tariffs or subtotals, it stores the string hash key associated with this charge.
  • Array tariffs or subtotals, it stores the integer array index associated with this charge.

 

Event General Columns

Column Name

Properties

Description

GENERAL_1 - GENERAL_10

VARCHAR (4000)

General fields 1 - 10 containing normalised event data.

Their use is controlled by the normalised event type.

Typical values are for the derivation of derived normalised event attributes that need to be stored for reporting, or decoding of location codes.

 

Charge General Columns

Column Name

Properties

Description

CHARGE_GENERAL_1 -
CHARGE_GENERAL_20

VARCHAR (255)

General fields 1 - 20 containing charge data.

They can be used to store intermediate calculations and lookups performed during rating (for example, distance band, time band, multi-time band flag).

The information is accessible by the Bill Generation Process (BGP) and the Invoice Generation Process (IGP).
