FileUploadInitialise&()

Syntax

FileUploadInitialise&(
    const RelativePath$,
    FileSize&)

 

Description

Notifies the server of a pending file upload to the location specified by RelativePath$, and how much data to expect.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

RelativePath$

In: Relative path to the file on the server to be uploaded.

For secure file transfer, it is mapped to a physical directory using the UploadDirectories derived attribute table.

FileSize&

In: Size in bytes of the file to upload.

The value cannot be negative.

 

Returns

TRUE when the server is ready for the upload; otherwise, an error is raised.

Remarks

Creates an empty file to receive the file data. Call this function before calling FileUpload&().

An error is raised if:

  • The user does not have security privileges to write the file to the relative path.
  • FileSize& is invalid.
  • The file exists on the server.

    Users can upload files to the server, and overwrite files they have uploaded. They cannot overwrite files with the same file name uploaded by other users. The File Already Exists error occurs when a user attempts to upload and overwrite a file uploaded by another user.

The use of this function is described in the UPLOAD_AUDIT table (see the Security and Auditing Configuration Guide for Singleview).
