ClientDetailForm&()

Syntax

ClientDetailForm&(
    EntityName$,
    EntityId&,
    EffectiveDate~,
    ParamNames$[],
    ParamValues?[])

 

Description

Displays the detail form for an instance of the specified entity.

Attributes

Application Environment

Data Entry

Context

Any

Method

Local

 

Parameter

Description

EntityName$

In: Name of the entity.

For a list of valid entity names, refer to the table below.

EntityId&

In: ID of the entity to be displayed.

For example, if EntityName$=CUSTOMER_NODE, EntityId& corresponds to the CUSTOMER_NODE_ID of the customer record being retrieved.

EffectiveDate~

In: Effective date used on the form.

ParamNames$[]

In: Array of the parameters to display.

ParamValues?[]

In: Array of the values corresponding to the parameters.

 

Returns

Status code where:

  • 0 (OK): Automation call was successful.
  • 1 (General Error): Unknown general failure.
  • 2 (No Form): Specified entity name does not correspond to a form.
  • 3 (No View): User has inadequate security to view the specified form or record.
  • 4 (No Update): User has inadequate security to update the specified form or record.
  • 5 (No Insert): User has inadequate security to insert the specified form or record.
  • 6 (Validation): An invalid field was specified.

Remarks

For some forms, the values for ParamNames$[] and ParamValues?[] are meaningless. For example, the Person Detail form displayed in View Only mode does not have any fields available for control.

A complete list of entity names to be used in the EnityName$ parameter can be obtained using the ClientSupportedEntityNames&() function.
