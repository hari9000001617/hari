Description

Inserts a row in a global derived attribute array.

Context

The following groups of variables are available throughout all stages of processing a global derived attribute insert request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value GDAASVR.
    • evLastRowInd& contains the value 0 during all stages of processing, except the evaluation of the global derived attribute pre-commit expressions for the last row processed.
    • evRowStatus$ contains the value Insert for the global derived attribute processing stages.
  • Derived attribute.

Parameters

Parameter

Description

StartDate~

Date when the new row becomes effective.

If not defined, the current date is used.

LastModified~

Initial value must match the value in the record for the derived attribute ID identified by FieldValues?{}.

If not defined, the last modified value of the derived attribute table for the specified StartDate~ is used.

On successful return, the value is the date-time of this insertion.

FieldValues?{}

Data to be inserted for the global derived attribute array.

The data can be entered according to the corresponding column name in the DERIVED_ATTRIBUTE_EVS_V view. When keyed according to column name, the data passed must match the data type and value stored in the database.

If attribute validation is enabled, the data can be keyed by the interface or display label in the appropriate entity validation or in core defaults. In this case, the data must match the type and value, defined in the appropriate attribute type. Denormalised fields must pass the denormalised string values. Other fields must pass data of the type defined in the attribute type.

It specifies the values for the primary identifiers of the global derived attribute array. For standard derived attribute tables, if a sequence number is not specified, a new row is added to the end of the derived attribute table.

Note:

Extended derived attribute tables do not use table row sequencing.

If an effective end date is not specified, the end of time value is used.

IdentifierKeys?{}

Values of the identifier fields of the DERIVED_ATTRIBUTE_ARRAY validation table if the function successfully returns.

 

Processing

Use FieldValues?{} to determine derived attribute ID, and effective end date, if specified.

Continues processing as if fEV_GlobalDAInsertRows& was called for row.

Returns

On success, a non-zero value; otherwise, an appropriate error is raised.
