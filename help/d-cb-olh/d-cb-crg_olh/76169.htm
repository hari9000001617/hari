fCMN_TREProductInstanceUpdateByOffer_R&()

Syntax

fCMN_TREProductInstanceUpdateByOffer_R&(
    l_ProductInstanceId&,
    l_EffectiveDate~,
    l_LastModified~,
    l_OfferStruct?{},
    l_ProductInstance?{})

 

Description

Updates the details of a specified base product instance, and its associated companion products and services.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_ProductInstanceId&

In: ID of the base product instance to update.

l_EffectiveDate~

In: Effective date of the record to be updated. If not specified, it defaults to current date.

l_LastModified~

In: Last modified date of the base product instance record.

It must match the last modified date for the product instance record being updated; otherwise, an exception is raised.

Out: Date-time the record was last modified.

l_OfferStruct?{}

In: Hash containing the hierarchical product offer structure.

 

For more information about the parameter hash keys, refer to the table below.

l_ProductInstance?{}

In: Hash containing the values for all mandatory and optional fields for the product instance and related entities.

For more information about the parameter hash keys, refer to Remarks below.

 

Returns

Last modified date, if successful; otherwise, an error is raised.

Remarks

Updating Multiple Offers of the Same Type

If multiple instances of child product offers are required, the following rules apply:

  • If Insert (1) is specified:
    • All mandatory child offers must be specified in l_OfferStruct?{}.
    • For service-specific offer instances of the same type, SERVICE_NAME must be specified on the highest level service-specific product offer.
    • SERVICE_NAME must be specified for each service-specific offer on the highest level in l_OfferStruct?{} as required.
  • If Update (2) or Delete (3) is specified, the ID for any non-cancelled product offer instance must be specified in l_OfferStruct?{}.

Updating Top Level Product Offers

When assigning a new top level product offer to the base product instance, only Insert (1) is supported for OPERATION_IND_CODE or OPERATION_IND. An error is raised if a different operation is specified to be called.

Note:

OPERATION_IND_CODE is optional for any new top level product being assigned to the base product instance.

The biProductInstanceOfferUpgrade&() BASE_INSTALL function is called to reallocate existing product instances to the new product offer hierarchy where applicable. Product instances that are not compatible or required are cancelled.

If Insert (1) is specified, the following rules apply:

  • The existing product offer hierarchy is replaced with the offer hierarchy specified in l_OfferStruct?{}.
  • All instances of a child offer required must be specified in l_OfferStruct?{}.
  • Mandatory companion products are always inserted regardless of whether they are specified or not in l_OfferStruct?{}.
  • Any linked services no longer required for a service-specific product offer are automatically cancelled.

Note:

Reusable product instances are updated with new PRODUCT_OFFER_INSTANCE_ID and PARENT_PROD_OFFER_INSTANCE_ID when they are reallocated to the new offer hierarchy. If {'PROD_INST_OFFER_UPGRADE'} is specified in l_OfferStruct?{}, product instances are reused depending on the order specified before calling the biProductInstanceOfferUpgrade&() function.

Companion products for the product instance.

The companion products must be defined as compatible products in the product definition.

[<ROW_NUMBER>] is an array (0..n) of companion products.
