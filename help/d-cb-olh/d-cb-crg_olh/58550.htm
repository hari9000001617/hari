Description

Calculates and returns charge information based on the event parameters set in the FieldNames$[] and FieldValues$[] arrays.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

FieldNames$[]

In: Field names from NORMALISED_EVENT_V view of the details of the event to be rated.

FieldValues?[]

In: Field values, in the same order that their names are specified in FieldNames$[].

ChargeFieldNames$[]

In: Charge fields to be returned for each charge calculation.

ChargeFieldValues?[]

Out: Charge values, corresponding to ChargeFieldNames$[].

Each row in the array corresponds to the requested charge fields.

DebugLevel&

In: Level of rating debug to turn on for rating the event.

If set to 0, no debug tracing is activated.

For information about available debug levels, refer to the Event Normalisation (ENM) process in the System Configuration Guide for Singleview Convergent Billing.

TraceFile$

Out: Path to the file containing the debug information (if DebugLevel& is set).

The file name is based on current date-time.

The path to the file is contained in the TRACE_DIR attribute of the TRERATE_TRAN configuration item for the process rating the event.

ErrorId&

Out: Error ID if an error occurred during rating.

ErrorMessage$

Out: Error message if an error occurred during rating.

 

Returns

Number of charges if successful; otherwise, an error is raised.

Remarks

The number of values in each data type array is returned in that type's return parameter.

The function can used for providing a quote for service usage. For example, to answer the question "How much will a 10 minute call to Pakistan at 9:45pm tonight cost?".

During calls to the function, subtotal values accessed include any current reservations in their returned values (except for the current reservation being processed).

If this function is called to determine the amount to reserve for an account and either AccountReservationInsert$() or AccountReservationUpdate&()executed (to create or update the account reservation), a call should be made to SubtotalEventCommit&() or EntitlementEventCommit&() to make reservations against rating subtotals and entitlements.

Note:

Reservations are available only as part of the separate Commerce Engine licence option.

The function only implicitly populates the direct variables from the NORMALISED_EVENT_V view that have been passed into it.
