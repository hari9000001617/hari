Entity Validation Variables

This section lists the direct variables provided for entity validation, known as entity validation variables. The variables are arranged in sections, one for each validation table and associated view that can be used to configure a form (or form tab in some cases). The information provided is the variable name, data type, and a brief description of the variable.

Many of these entity validation items are associated with two variables, one holding the value entered by the user, and the other holding the value before data entry. The naming convention makes it clear which is which; the previous value is held in a variable whose name begins with evOld. Entity validation variables with names that begin with evOld are configured for OLD_<viewname> views where <viewname> is the name of the view for which the regular entity validation variables are configured.

Note:

  • OLD_<viewname> views are not real views, but are a naming convention used by the Entity Validation Server.
  • In this section, references to 'view' mean 'table' if the view name is not suffixed with _V, _CV, or _TRE_V.

The application environment for all entity validation variables is Data Entry, and the context is Any.

For each Convergent Billing form that can be configured using entity validation variables, the table below lists:

  • Form name (and tab, if applicable) that can be configured.
  • Validation table to select using the Entity Validation Definition form to configure the form.
  • Validation table's view name that can be used to list the variables associated with the view (see the Configuration Fundamentals Guide for Singleview Convergent Billing) and validation table using the Variable Definition Search form.
  • Form name and field where the entity validation is specified to configure the form.

A number of common validation variables are available for use in all forms.

Note:

Not all entity validation variables can be used to configure a form but can be used in other contexts (for example, within expressions). These variables' validation tables are not listed in the table below.
