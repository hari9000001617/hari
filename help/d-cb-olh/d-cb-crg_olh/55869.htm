SOAPOutbound

The SOAPOutbound interface type provides support for generating configuration settings for outbound interfaces. The interface type is used to generate an outbound SOAP interface to be used by the Web Services Adapter, and contains additional parameters for specifying external definitions.

Type

Static SOAP

Config File Template

Text

Used for interface types where the configuration is based on a template file.

The default path is $ATA_REL_SERVER_CONFIG/tresoap/tresoap.xml.

Note:

This file can be customised by implementation teams. If so, it should be renamed and copied to an implementation-specific directory. For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

SOAP Version

[List]

Version of SOAP used by the adapter. Valid values are:

  • 1.1
  • 1.2.

Transfer Encoding

[List]

Enables or disables chunked transfer.

Default is None.

Tuxedo Service Name

[Text]

Name of the Tuxedo service used to connect to outbound interfaces.

It is used in generation of REMOTE_SERVICE configuration item.

Connection Timeout (s)

[Integer]

Length of time in seconds to wait for a connection before timing out.

Request Timeout (s)

[Integer]

Length of time in seconds to wait for a request before timing out.

Idle Timeout (s)

[Integer]

Length of time in seconds a connection remains idle before it is closed.

If set, the option HttpIdleOnBorrowEnabled of the FunctionNameToOperationService is set to True (see the Installation and Integration Guide for Singleview Web Services Adapter).

Disable Attachments?

List

Enables or disables SOAP with Attachments (SwA) in the SOAP message.

Default is Yes.

If set to:

  • Yes, SwA is disabled
  • No, SwA is enabled
  • Default, SwA is defined by the enableSwA parameter in the axis2.xml configuration file.

Operational SOAP

[List]

Transport protocol used. Valid values are:

  • HTTP
  • HTTPS.

THP Instance

[List]

THP instance used to connect to Singleview.

If Cluster Managed is set to True, this field must not be set. Otherwise, it is mandatory.

SV Instance

[List]

Singleview instance on which the interface is run.

It is used in generation of REMOTE_SERVICE configuration item.

Note:

Ignored if Cluster Managed is True.

TRESOAP Sequence Number

[Integer]

TRESOAP configuration item sequence number for the instance.

Default is the next available sequence number.

TREPROXY Server Group

[Text]

Tuxedo group associated with the treproxy server in the Tuxedo configuration file.

It is used in generation of REMOTE_SERVICE configuration item.

[Integer]

Max Host Connections

[Integer]

Maximum number of concurrent connections to the remote end point.

TREPROXY Port

[Text]

Port used to connect to the Web Services Adapter.

It is specified as a port number or service name as defined in /etc/services.

Listen Port

[Text]

Port used to listen for incoming HTTP connections.

Cluster Managed

List

Indicates if this instance is managed by a high availability configuration, and determines the number of TRESOAP and REMOTE_SERVICE configuration items generated.

Default is False.

If set to:

  • False, only one of each configuration item is created.
  • True, one of each configuration item is created for each instance in the InstanceStatus derived attribute table.

THP Group

[List]

THP group to associate with the SOAP interface configuration in a clustered environment. It is used to determine the THP to use on each instance when the SOAP configuration is generated.

If Cluster Managed is set to True, this field is mandatory. Otherwise, it must not be set.

Operational HTTP
Security

HTTP Authentication Scheme

List

HTTP authentication scheme to use. Valid values are:

  • None
  • Basic
  • Digest
  • NTLM
  • Any (attempts NTLM, Digest, then Basic).

HTTP Authentication Realm

Text

Authentication realm used with Basic, Digest and NTLM authentication schemes.

If the authentication scheme is not NTLM or None, this parameter must be set.

NTLM Authentication Domain

Text

Authentication domain used with the NTLM authentication scheme.

If the authentication scheme is NTLM, this parameter must be set.

HTTP Authentication Username

Text

Username used for Basic, Digest and NTLM authentication schemes.

If the authentication scheme is not None, this parameter must be set.

HTTP Authentication Password

Text

Password used with the user name for Basic, Digest and NTLM authentication schemes.

If the authentication scheme is not None, this parameter must be set.

SSL

Keystore Location

[Text]

File path to the keystore that contains the identification certificate.

Note:

For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

The keystore can be created using the keytool utility that is included in the Java standard edition SDK (see the Security and Auditing Configuration Guide for Singleview).

This parameter can be specified as an environment variable.

Keystore Password

[Text]

Password used to access the keystore.

Key Password

Password used to access the key in the keystore.

Truststore Location

[Text]

File path to the truststore that contains the trusted certificates.

Note:

For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

This parameter can be specified as an environment variable.

The truststore can be created using the keytool utility that is included in the Java standard edition SDK (see the Security and Auditing Configuration Guide for Singleview).

Truststore Password

[Text]

Password used to access the truststore.

Require Trust?

Yes/No

If Yes is specified, client trust must be established using the truststore when a connection is established.

[Text]

Note:

If Yes is specified, the external definitions specified in the External Definition field use default qualified namespaces, that is, no prefix.

[Text]

Name of external definition associated with the service group.

The external definition must support only one of Insert, Update, Delete, Fetch, or Search operations.

Response Element

Root element name for the XML response message.

SOAP Action

SOAP outbound remote Web Service Adapter function.

Function Prefix

[Text]

Prefix used to map interface functions to the service.

For each service name, only one Function Prefix can be specified.

Target Namespace

[Text]

Namespace for the service.

It is used as the namespace for the RequestElement of the SOAP outbound request messages.

Note:

For each service name, the same Target Namespace value must be specified.

If it is:

  • Set to DEFAULT, the root element namespace is set to the external definition namespace associated with the root element. The RequestElement is prefixed with the associated namespace prefix. The TargetNamespace property of com.intecbilling.tresoap.soap.
    outbound.FunctionNameToOperationService is not modified.
  • Specified but is not DEFAULT, the TargetNamespace property of com.intecbilling.tresoap.soap.outbound.
    FunctionNameToOperationService is inserted in the outbound service definition for the specified service name. This results in a RequestElement without a namespace prefix but declared in the namespace specified by the TargetNamespace property.
  • Not specified or empty, the TargetNamespace property for the service is omitted from com.intecbilling.tresoap.soap.outbound.
    FunctionNameToOperationService.
