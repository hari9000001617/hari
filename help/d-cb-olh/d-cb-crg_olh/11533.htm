Description

Interface 1 causes the collection of statistics on cache activity from various caches. Statistics are collected on the local instance, or on each instance if part of a cluster, and sent to the tremon server of the instance.

Interface 2 allows the specification of the instance name, and an ID to be associated with the collected statistics.

Interface 3 provides the same functionality as interface 2, except that the LogToFile& parameter of biMonitorLog&() can be explicitly specified.

Attributes

Parameter

Description

InstanceName$

In: Name of the Convergent Billing instance on which the function executes.

Id&

In: ID for the statistics.

It is passed to biMonitorLog&() as the parameter PrimaryId&.

LogToFile&

In: Flag passed to biMonitorLog&() as the parameter of the same name.

It enables the file logging option of tremon to be overridden on a per call basis.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised. For statistical result information, refer to the return values of the individual statistical function.

Remarks

Statistics are collected by CacheMonitor&() for the following caches:

  • Account
  • Customer node
  • Rating subtotal
  • Service
  • Global shared derived attribute tables.

All interfaces use CacheMonitor&() to collect and send the statistics to the tremon server on an instance.

For interface 1, if CacheMonitor&() is called in the context of a cluster implementation, the instance name and the instance's position in the set of active instances are specified as the InstanceName$ and Id& parameters, respectively. Otherwise, the local instance name is specified for InstanceName$ and '0' is specified for Id&. The LogToFile& parameter of biMonitorLog&() defaults to 1.

For interface 2, the LogToFile& parameter of biMonitorLog&() defaults to 1.
