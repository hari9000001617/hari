fEVS_ValidateDefaultKeys&()

Syntax

fEVS_ValidateDefaultKeys&(
    const Entity$,
    const KeyNames$[],
    var ImplicitEffectiveDate&)

 

Description

Checks if the supplied key names are a valid identifying key combination for the specified entity type.

Context

Not applicable.

Parameters

Parameter

Description

Entity$

Name of the entity to validate the keys for.

KeyNames$[]

Names of the keys to validate.

ImplicitEffectiveDate&

Function output.

Indicates if an effective date key is implicit in the key combination.

If True, indicates the entity is date ranged, but no effective date or date range key was supplied.

If False, indicates that the entity is either of the following:

  • Date-ranged, and an effective date range key was supplied.
  • Not date-ranged.

 

Processing

The function has no configurable processing steps.

Remarks

For date-ranged entities, an effective date key is optional in a valid key combination. A lookup of a date-ranged entity can be performed with an implicit effective date range key.

The following table lists valid key combinations for available entities. Attempting to validate an unknown entity results in error.
