CUSTOMER_NODE_HISTORY_SEARCH_V

The CUSTOMER_NODE_HISTORY_SEARCH_V view displays customer details. It is used to perform searches.

Column Definitions

Column Name

Properties

Description

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node associated with this instance.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal toEFFECTIVE_START_DATE.

CUSTOMER_NODE_TYPE_ID

NUMBER (8)

Not null

Type of this customer node instance.

PRIMARY_IDENTIFIER

VARCHAR (80)

Optional primary ID for the customer node.

It can be set to be the same as another column in this table.

The type of value in this field can be dependent on the customer node type or values in other columns.

PRIMARY_IDENTIFIER2

VARCHAR (80)

Optional secondary primary ID for the customer node.

It can be set to be the same as another column in this table.

The type of value in this field may be dependent on the customer node type or values in other columns.

Both primary IDs are usually not specified, but both can be, if required.

NODE_NAME

VARCHAR (200)

Not null

Name associated with this node.

NODE_NAME_UPPERCASE

VARCHAR (200)

Not null

Derived

Node name converted to upper case.

It is equal to UPPERCASE(NODE_NAME).

The value is enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

Duplicate names for customer nodes with the same parent may be permitted depending on the value of the SYSTEM_CLIENT configuration attribute DUPLICATE_CHILD_NODE_NAME.

Root customer nodes (nodes with a PARENT_CUSTOMER_NODE_ID value of NULL) can have duplicate node names (see the System Configuration Guide for Singleview Convergent Billing).

NODE_NAME_SOUNDEX

VARCHAR (10)

Not null

Derived

Soundex value for NODE_NAME, derived using a customised version of the Soundex algorithm.

It is used for soundex searches based on NODE_NAME.

ATLANTA_GROUP_ID

NUMBER (8)

Not null

Security group with access to this customer node.

Only users with access to this security group can view the details associated with this customer node.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified this record.

GL_CODE_ID

NUMBER (8)

Not null

GL code associated with all charges for this customer node.

PARENT_CUSTOMER_NODE_
ID

NUMBER (15)

Parent customer node for this node.

Only customer node types that support hierarchies can have a value in this column.

HIERARCHY_LEVEL

NUMBER (3)

Not null

Level in the hierarchy of this node.

The root node of a hierarchy has a level of 1. Therefore, it must have a value of 1 if PARENT_CUSTOMER_NODE_ID is null; otherwise, it must have a value greater than 1.

The values are enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

For values other than 1, the value must be one greater than the value assigned to the PARENT_CUSTOMER_NODE_ID. It is verified by the DVP.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Derived

If this instance of a node has a PARENT_CUSTOMER_NODE_ID, this is the ID of the customer node at the root of the hierarchy.

If PARENT_CUSTOMER_NODE_ID is null, it is also null.

CUSTOMER_NODE_STATUS_
CODE

NUMBER (4)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-listed).

CREATED_DATE

DATE

Not null

Date-time that the customer was first added to the system.

Its finest granularity is one second.

ACTIVE_DATE

DATE

Date-time that the customer first became an active subscriber to the network (that is, when their status first changed to Active).

Its finest granularity is one second.

 

Primary Contact Details

Column Name

Properties

Description

PERSON_ID

NUMBER (15)

Not null

Primary contact person for this customer node.

Any correspondence that relates to this customer node is addressed to this person.

Other contacts are referred to in the CUSTOMER_CONTACT secondary contacts list table that displays the contact person identification details, and the reason for the association with the customer node.

 

Financial Details

Column Name

Properties

Description

PRIME_ACCOUNT_ID

NUMBER (15)

Not null

Primary account associated with this customer node.

Nodes with only Statement or None requirements must have a primary account.

REPORT_LEVEL_CODE

NUMBER (2)

Not null

Level of financial reporting required for this customer node. Valid values are:

  • 1 (Invoice)
  • 2 (Statement)
  • 3 (No Reporting)
  • 4 (Transferred Statement).

POSTAL_ADDRESS_ID

NUMBER (15)

Reference to the address details for all postal correspondence regarding this customer node, including invoices and statements.

Address details are not required if REPORT_LEVEL_CODE is None.

SITE_ADDRESS_ID

NUMBER (15)

Reference to the physical location of this customer (or division of the customer).

It is used if it is necessary to courier any details directly to the customer.

CURRENCY_ID

NUMBER (8)

Currency for all invoice and statement generation.

No currency is required if REPORT_LEVEL_CODE is None.

SCHEDULE_ID

NUMBER (15)

Not null

Standard invoice cycle that the customer node is assigned to.

It defaults to the invoice cycle of the parent node.

 

Credit Status Details

Column Name

Properties

Description

TURNOVER

NUMBER (19,3)

Revenue for the last financial year for this customer node.

It is only required if REPORT_LEVEL indicates that invoices are to be produced.

TURNOVER_CURRENCY_ID

NUMBER (8)

Currency for the above turnover value.

CREDIT_LIMIT

NUMBER (19,3)

Amount of credit that a customer node can have at any one time.

It is required only if REPORT_LEVEL indicates that invoices are to be produced.

CREDIT_LIMIT_CURRENCY_
ID

NUMBER (8)

Currency for the above credit limit value.

CREDIT_RATING_CODE

NUMBER (4)

Indicator for how a customer is treated regarding financial accounting practices including invoicing and payment collection.

VIP customers are given the highest rating of 1. The credit rating is defaulted by the CUSTOMER_NODE_TYPE.

CREDIT_COMMENTS

VARCHAR (255)

Comments relating to a customer's credit rating and the limit assigned, and provides reasons and justifications for the assigned values.

TAX_CLASS_CODE

NUMBER (4)

Tax class with which this customer node is associated.

PAYMENT_METHOD_CODE

NUMBER (4)

Method of payment for the customer node; only required if REPORT_LEVEL indicates that invoices must be produced.

PAYMENT_LOCATION_CODE

NUMBER (4)

Location where payments are to be made.

Examples are postal delivery, third party institution such as a bank or post office, or in person at a specific or general customer service centre.

BANK_CODE

NUMBER (3)

Code of the bank or financial institution that a customer uses for their account which relates to invoice payments (that is, for drawing cheques, and automatic debit).

BRANCH_CODE

NUMBER (5)

Code or number of the branch of the defined financial institution that the customer node uses.

BANK_ACCOUNT_NUMBER

VARCHAR (60)

Customer's account number for automatic payments.

BANK_ACCOUNT_REF

VARCHAR (15)

Account reference number that the customer must quote with their payment at banking institutions and other third party payment collection points.

It allows payments to be associated with a single bank account.

Payment entries to this account are allocated the required user access reference number.

CARD_TYPE_CODE

NUMBER (4)

Type of credit card used for payments, and is based on CARD_TYPE reference code.

It is used as default information for payment.

CARD_NUMBER

VARCHAR (60)

Main or default credit card number for making payments.

CARD_EXPIRY_DATE

DATE

Date on which the main or default credit card expires.

ASSIGNED_OPERATOR_ID

NUMBER (8)

User assigned to liaise with this customer node in all matters.

SALES_CHANNEL_CODE

NUMBER (4)

Sales channel for this customer node.

COMPANY_NUMBER

VARCHAR (15)

Official company or business registration number (for example, A.C.N).

 

Classification-related Fields

Column Name

Properties

Description

INDUSTRY_CODE

NUMBER (4)

Code of the type of industry or activity the customer node is involved in (for example, banking, manufacturing, government).

REGION_CODE

NUMBER (8)

Code or map grid reference of the geographic region where the customer node is located.

The region can be based on the physical address details such as suburb.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be defined in the type definition table.

The constraints on each column are defined by the details of its CUSTOMER_NODE_TYPE.

SEARCH_IND_CODE

NUMBER

Code of the mode of the search form.

A value of 0 indicates a search operation; a value of 1 indicates a new operation.
