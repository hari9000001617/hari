BillingConfiguration

The BillingConfiguration derived attribute table defines the TRE function to call to implement a specific billing operation or revoke operation for a specific billing configuration (see the System Configuration Guide for Singleview Convergent Billing).

The table is accessed by the biBillRunExecute&() function and is pre-populated in Convergent Billing with data for implementing all supported standard bill run operations. Entries can be modified and new entries added.

During a bill run, if:

  • An operation or revoke operation is defined for the specified billing configuration, the associated function is used.
  • An operation or revoke operation is not defined for the specified billing configuration, the function associated with the default billing configuration is used.
  • Any operations defined in the specified billing configuration have no bill run operation function or corresponding revoke operation defined, the bill run fails.

The table has an evaluation type of Table Lookup By Multiple (1-10) Keys.

Not null

Name of the TRE function to call to implement the specified bill run operation for the billing configuration.
