AdjustmentInvoiceUpdate&()

Syntax

AdjustmentInvoiceUpdate&(
    AdjustmentId&,
    var LastModified~,
    const Opcode&[],
    const AccountId&[],
    const Seqnr&[],
    const FieldNames$[],
    const FieldValues?[])

 

Description

Maintains the ADJUSTMENT_INVOICE table for accounts associated with each adjustment.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

AdjustmentId&

In: ID of the adjustment.

LastModified~

In: Last modified date-time of the adjustment to be updated.

Out: Last modified date-time of the updated adjustment.

Opcode&[]

Operations to perform on the row.

AccountId&[]

In: ID of each account.

Seqnr&[]

In: Sequence number for this combination of adjustment and account.

FieldNames$[]

In: Set of fields from ADJUSTMENT_INVOICE_TRE_V being inserted or updated.

The same fields must be specified for all rows.

FieldValues?[]

In: Two-dimensional array of field values in [Row][Field] order being inserted, updated, or deleted.

An empty row can be specified for records that are being deleted and have NULL INVOICE_ID, ALLOCATED_ADJUSTMENT_ID, and PAYMENT_ID.

 

Returns

Number of rows successfully processed (should equal the size of Opcode&[]); otherwise, an error is raised.

Remarks

The results of changes to the adjustment invoice records must ensure that the sum of the adjustment invoice amounts for the:

  • To account equals the TO_TOTAL_AMOUNT for the adjustment.
  • From account equals the FROM_TOTAL_AMOUNT for the adjustment.

If the adjustment was applied:

  • Net changes to receivable type allocations are reflected by entries in the CHARGE table, with the ACCOUNT_HISTORY being updated to indicate changes have occurred to the adjustment, but have not been uploaded to the General Ledger.
  • If the GL subsystem is enabled, the net changes to allocations are reflected by new GL guidance records.

If an allocation is associated with:

  • An invoice, CURRENT_DUE for the invoice is updated to reflect the allocation.
  • A receivable type, CURRENT_DUE for the invoice receivable type is updated to reflect the allocation.

If the allocation is to a liability account and account handling functionality is:

  • Not enabled, CURRENT_DUE and INVOICE_HISTORY are increased by the amount of the allocation.
  • Enabled, CURRENT_DUE and INVOICE_HISTORY are decreased by the amount of the allocation.

A last modified check and date ranged update is automatically performed on the ADJUSTMENT record associated with this change, with the updated ADJUSTMENT last modified date-time being returned in LastModified~.

The update operations that can be performed are 1 (Insert), 2 (Update), and 3 (Delete), and are represented by the value placed in the Opcode&[] array. If multiple updates are to be performed on this adjustment, the number of parameters in the Opcode&[] array must equal the number of parameters in the AccountId&[], Seqnr&[], FieldNames$[], and FieldValues?[] arrays.

The field names to be updated can be placed in any order in the FieldNames$[] array, but must be valid fields from the ADJUSTMENT_INVOICE_TRE_V view. The field values corresponding to these field names must follow the [Row][Field] convention to allow the correct fields to be updated. These rows must also match to the correct sequence numbers, accounts, and operation codes.

The effective user must have update security group access level for the customer associated with the record manipulated by this function call.

GROUP_ACCESS_TRE_V view lists user group access level values.
