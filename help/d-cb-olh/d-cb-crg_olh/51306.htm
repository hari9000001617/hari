REPROCESSED_FILE_TRE_V

The REPROCESSED_FILE_TRE_V view defines the set of fields that can be passed to the biReprocessedFile* functions.

Column Definitions

Column Name

Properties

Description

REPROCESSED_FILE_ID

NUMBER (15)

Not null

Read-only

ID of a reprocessed file.

LAST_MODIFIED

DATE

Not null

Date-time when this record was last modified.

FILENAME

VARCHAR (255)

Not null

Full path name of the file.

It must be specified on insert.

STATUS_CODE

NUMBER (2)

Not null

Read-only

Status of the reprocessing file.

If not specified on insert, it defaults to 1.

It is based on the REPROCESSED_FILE reference type.

ATLANTA_OPERATOR_ID

NUMBER (10)

Not null

Read-only

User who initiated the reprocessing of the file.

TASK_QUEUE_ID

NUMBER (15)

Read-only

ID of the task that generated this file.

PROCESS_START_DATE

DATE

Read-only

Date-time when events from this file began processing.

EVENT_SOURCE

VARCHAR (40)

Not null

ID of the source from which this group of normalised events originated.

It must be specified on insert.

INPUT_EVENT_COUNT

NUMBER (10)

Not null

Derived

Read-only

Number of error events for reprocessing originally associated with this file.

If not specified on insert, it defaults to 0.

NORMALISED_EVENT_FILE_ID

NUMBER (15)

Derived

Read-only

ID of the normalised event file.

It is set during rating when the file has started to be processed.
