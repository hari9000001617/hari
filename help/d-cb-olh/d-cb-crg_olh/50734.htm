INV_CUSTOMER_NODE_HISTORY_V

The INV_CUSTOMER_NODE_HISTORY_V view displays customer node details for use by the IGP.

Column Definitions

Column Name

Properties

Description

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node associated with this instance.

BILL_RUN_ID

NUMBER (15)

Not null

ID of the bill run for which these customer details have been retrieved.

EFFECTIVE_DATE

DATE

Not null

Effective date of this bill run.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

CUSTOMER_NODE_TYPE_NAME

VARCHAR (30)

Not null

Type of this customer node instance.

CUSTOMER_NODE_TYPE_DISP_
NAME

VARCHAR (60)

Not null

Name of the customer node type displayed.

It defaults to CUSTOMER_NODE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

CUSTOMER_NODE_TYPE_ID

NUMBER (8)

Not null

Type of this customer node instance.

PRIMARY_IDENTIFIER

VARCHAR (80)

Optional primary ID for the customer node.

It can be the same as another column in this table.

The type of value can be dependent on the customer node type or values in other columns.

PRIMARY_IDENTIFIER2

VARCHAR (80)

Optional secondary primary ID for the customer node.

It can be the same as another column in this table.

The type of value can be dependent on the customer node type or values in other columns.

Both primary IDs are usually not specified, but both can be, if required.

NODE_NAME

VARCHAR (200)

Not null

Name associated with this node.

PARENT_CUSTOMER_NODE_ID

NUMBER (15)

Parent customer node for this node.

Only customer node types that support hierarchies can have a value in this column.

HIERARCHY_LEVEL

NUMBER (3)

Not null

Level in the hierarchy of this customer node.

The root node of a hierarchy has a level of 1.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Derived

If this instance of a node has a PARENT_CUSTOMER_NODE_ID, this is the ID of the customer node at the root of the hierarchy.

If PARENT_CUSTOMER_NODE_ID is null, it is also null.

CUSTOMER_NODE_STATUS_
ABBREV

VARCHAR (60)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-Listed).

CUSTOMER_NODE_STATUS_
CODE

NUMBER (4)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-Listed).

CREATED_DATE

DATE

Not null

Date that the customer was first added to the system, with a status of Prospect (or any other status).

ACTIVE_DATE

DATE

Date that the customer first became an active subscriber to the network; that is, when their status first changed to Active.

INVOICE_SEQNR

NUMBER (3)

Not null

Sequence number of a specific invoice image for this customer node.

Note:

It controls the primary contact and address details returned for the customer node.

  • 4 (Subscriber & Business Contact)

BIRTH_DATE

DISABILITY_TYPE_CODE

VARCHAR (30)

Standard telephone voice and fax network dial numbers.

EMAIL_ADDRESS

PROFESSION_CODE

EMPLOYER

NATIONALITY_CODE

PASSPORT

WRITTEN_LANGUAGE_CODE

SPOKEN_LANGUAGE_CODE

SOCIAL_SECURITY_NUMBER

VARCHAR (40)

Social security number.

DRIVERS_LICENCE

VARCHAR (40)

Driver's licence of the primary contact.

Credit Status Details

Column Name

Properties

Description

TURNOVER

NUMBER (19,3)

Revenue for the last financial year for this customer node.

It is required only if REPORT_LEVEL indicates that invoices are to be produced.

TURNOVER_CURRENCY_ID

NUMBER (8)

Currency for the above turnover value.

CREDIT_LIMIT

NUMBER (19,3)

Amount of credit that a customer node can have at any one time.

It is required only if REPORT_LEVEL indicates that invoices are to be produced.

CREDIT_LIMIT_CURRENCY_
ID

NUMBER (8)

Currency of the above credit limit value.

CREDIT_RATING_CODE

NUMBER (8)

How a customer is treated regarding financial accounting practices including invoicing and payment collection.

VIP customers are given the highest rating of 1.

The credit rating is defaulted by the CUSTOMER_NODE_TYPE.

CREDIT_COMMENTS

VARCHAR (255)

Comments relating to a customer's credit rating and the limit assigned.

It provides reasons and justifications for the assigned values.

TAX_CLASS_CODE

NUMBER (8)

Tax class with which this customer node is associated.

 

Payment Method Details

Column Name

Properties

Description

PAYMENT_METHOD_CODE

NUMBER (8)

Code of method of payment for the customer node.

It is only required if REPORT_LEVEL indicates that invoices must be produced.

PAYMENT_LOCATION_CODE

NUMBER (8)

Code of the location to which payments are to be made.

Examples are postal delivery, third party institution such as a bank or post office, or in person at a specific or general customer service centre.

BANK_CODE

NUMBER (8)

Code of the bank or financial institution that a customer uses for their account that relates to invoice payments (that is, for drawing of cheques and automatic debit).

BRANCH_CODE

NUMBER (8)

Code or number of the branch of the defined financial institution that the customer node uses.

BANK_ACCOUNT_NUMBER

VARCHAR (60)

Customer's account number for automatic payments.

BANK_ACCOUNT_REF

VARCHAR (15)

Account reference number that the customer must quote with their payment at banking institutions and other third party payment collection points.

It allows payments to be associated with a single bank account.

Payment entries to this account are allocated the required user access reference number.

CARD_TYPE_CODE

NUMBER (8)

Type of credit card used for payments, and based on CARD_TYPE reference code.

It is used as default information for payment.

CARD_NUMBER

VARCHAR (60)

Main or default credit card number for making payments.

CARD_EXPIRY_DATE

DATE

Date that the main or default credit card expires.

ASSIGNED_OPERATOR_ID

NUMBER (8)

User assigned to liaise with this customer node in all matters.

SALES_CHANNEL_CODE

NUMBER (8)

Sales channel for this customer node.

COMPANY_NUMBER

VARCHAR (15)

Official company or business registration number (for example, A.C.N).

 

Classification Related Details

Column Name

Properties

Description

INDUSTRY_CODE
