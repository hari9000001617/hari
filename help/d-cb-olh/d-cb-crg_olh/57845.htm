PRODUCT_OFFER_OFFER_TRE_V

The PRODUCT_OFFER_OFFER_TRE_V view provides details about child product offers of a single product offer.

Column Definitions

Column Name

Properties

Description

PRODUCT_OFFER_ID

NUMBER (8)

Not null

ID of the product offer.

CHILD_PRODUCT_OFFER_ID

NUMBER (8)

Not null

ID of a product offer that is contained in this product offer (that is, it is a child product offer of this product offer).

It cannot have the same value as PRODUCT_OFFER_ID.

A product offer with a not null TENANT_ID can only be made a child offer of another product offer with the same TENANT_ID.

SEQNR

NUMBER (4)

Not null

Sequence number that controls the display order of the child product offer for the current offer.

DESCRIPTION

VARCHAR (255)

Description of the compatibility between the parent and child product offers.

MANDATORY_IND_CODE

NUMBER (1)

If set to 1 (True), the child product offer is a mandatory component of this product offer during the period it is available.

If not NULL, it must have a value of 1.

AVAILABLE_START_DATE

DATE

Not null

Date-time at which this product offer becomes available.

If this product offer has no parent, it is identical to the start date-time of the availability period of this product offer.

If the node has a parent node, the available start date is set to the latest of the available start date of:

  • This product offer association, if specified.
  • This product offer.
  • The parent product offer node.

AVAILABLE_END_DATE

DATE

Not null

Date-time after which the product offer is no longer available.

If this product offer has no parent node, it is identical to the end date-time of the availability period of the product offer.

If the node has a parent node, the available start date is set to the earliest the available end date of:

  • This product offer association, if specified.
  • This product offer.
  • The parent product offer node.

The value of this column may be earlier than AVAILABLE_START_DATE. If so, this product cannot be selected as part of its parent product offer.

OFFER_MIN

NUMBER (4)

Not null Derived

Minimum number of times the same offer can be sold under each instance of a product offer.

If OFFER_MIN is:

  • Specified, the value must be greater than or equal to 0.
  • Not specified, the value is derived from MANDATORY_IND_CODE. If MANDATORY_IND_CODE is:
    • NULL, OFFER_MIN defaults to 0.
    • Not NULL, OFFER_MIN defaults to 1.

OFFER_MAX

NUMBER (4)

Maximum number of times the same offer can be sold under each instance of a product offer.

If OFFER_MAX is:

  • Specified, its value must be greater than or equal to OFFER_MIN, or NULL to indicate no maximum.
  • Not specified, its value defaults to 1 when OFFER_MIN is 0; otherwise, the value of OFFER_MIN.

DEFAULT_IND_CODE

NUMBER (1)

If set to 1 (TRUE), the child product offer is automatically selected by default when ordering the parent product offer. The child product offer can be deselected during the ordering process.

If it is not NULL, it must be 1 (TRUE).

If MANDATORY_IND_CODE is 1, it must be NULL.
