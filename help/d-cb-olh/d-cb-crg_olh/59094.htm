PRODUCT_INSTANCE_TRE_V

The PRODUCT_INSTANCE_TRE_V view provides details about product instances.

Column Definitions

Column Name

Properties

Description

PRODUCT_INSTANCE_ID

NUMBER (15)

Not null

Read-only

ID of a product instance.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that this row was last modified.

OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified the record.

The value can be overridden, but the specified user must be the same as the TRE-connected user.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

If this is a companion product instance, the date must correspond to a date for the base product instance associated with the product instance. This is checked by the DVP.

For inserts, it is set to the StartDate~ parameter.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

If this is a companion product instance, the date must correspond to a date for the base product instance associated with the product instance. This is checked by the DVP.

For inserts, it defaults to the end of time if it is not specified.

PRODUCT_NAME

VARCHAR (60)

Not null

Read-only

PRODUCT_DISPLAY_NAME

VARCHAR (60)

Not null

Read-only

Display name of the product associated with the product instance.

The product must exist for at least this date range.

PRODUCT_ID

NUMBER (8)

Not null

Read-only

ID of the product associated with the product instance.

It must exist for at least this date range.

PRODUCT_INSTANCE_
STATUS_CODE

NUMBER (8)

Not null

Current status of this product, mirroring service statuses. Valid values include:

  • 0 (Product Creation): Temporary status during product creation. When product creation is complete, this status is updated.
  • 1 (Provision Required): Provisioning must be performed.
  • 2 (Provisioned)
  • 3 (Active)
  • 4 (Suspended)
  • 5 (Temporary Disconnect Required): Deactivation must be performed.
  • 6 (Temporarily Disconnected)
  • 7 (Reactivation Required): Reactivation must be performed.
  • 8 (Cancellation Required): Deactivation must be performed.
  • 9 (Cancelled)
  • 10 (Moved)
  • 11 (Upgraded).

For more information, refer to Remarks below.

Not null

ID of the base product instance.

CONTRACT_ID

NUMBER (15)

Read-only

Contract that caused the creation of this companion product instance.

FROM_PRODUCT_INSTANCE_
ID

NUMBER (15)

Read-only

If this product instance was created as a replacement for a product instance, this is the ID of the product instance that was replaced.

When PRODUCT_INSTANCE_ID has a status of Active, it must have a status of Moved or Upgraded, and eventually is deleted. For this reason, this value may not reference a valid product instance.

If the product instance exists, the following values are checked:

  • The referenced product instance has a TO_PRODUCT_INSTANCE_ID. Since a single product instance could have been split to create this product instance and others, the referenced product instance's TO_PRODUCT_INSTANCE_ID may not be equal to PRODUCT_INSTANCE_ID.
  • If this product instance has a status of Active, to see whether the other product instance has a status of Moved or Upgraded.

For a companion product instance, the value is NULL.

FROM_PRODUCT_NAME

VARCHAR (60)

Read-only

Product ID associated with the FROM_PRODUCT_INSTANCE_ID.

It is not redundant, because after a replacement, FROM_PRODUCT_INSTANCE_ID is archived from the system.

For a companion product instance, the value is NULL.

FROM_PRODUCT_ID

NUMBER (8)

Read-only

Product ID associated with the FROM_PRODUCT_INSTANCE_ID.

It is not redundant, because after a replacement, FROM_PRODUCT_INSTANCE_ID is archived from the system.

For a companion product instance, the value is NULL.

TO_PRODUCT_INSTANCE_ID

NUMBER (15)

If this product instance was replaced, this is the ID of the product instance that replaced it.

When it has a status of Active, this product instance must have a status of Cancelled and is eventually deleted.

For a companion product instance, the value is NULL.

TO_PRODUCT_NAME

VARCHAR (60)

Read-only

Product ID associated with TO_PRODUCT_INSTANCE_ID.

It is obtained from TO_PRODUCT_INSTANCE_ID.

For a companion product instance, the value is NULL.

TO_PRODUCT_ID

NUMBER (8)

Read-only

Redundant

Product ID associated with TO_PRODUCT_INSTANCE_ID.

It is obtained from TO_PRODUCT_INSTANCE_ID.

For a companion product instance, the value is NULL.

UNCOMPLETED_IND_ODE

NUMBER (1)

Read-only

Redundant

For base products only.

If set to 1 (TRUE), the product instance was not completely installed, or is in the process of being completed; otherwise, the value is NULL.

If set to 1 (TRUE), it is necessary for either of the following to be in place:

  • The product instance to have a status of Product Creation.
  • One or more rows to exist in the PRODUCT_INSTANCE_SERVICE or PRODUCT_INSTANCE_EQUIP tables for this product instance and a date range with an UNCOMPLETED_IND_CODE of 1 (TRUE).

MULTIPLE_IND_CODE

NUMBER (1)

Read-only

For companion products only.

If set to 1 (TRUE), multiple instances of the companion product can be active on a single base product instance.

If set to NULL, only one instance of the companion product can be active on a single base product instance.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields.

DATE

Read-only

Date-time when the status of the product instance was first changed to Active.

INDUSTRY_GENERAL_10

General fields for common attributes for a product, irrespective of individual client customisation.

SERVICE_NAME

Not searchable

Name of the service with which this companion product instance is associated.

  • A companion product
  • Configured to be service-specific.

DO_NOT_PURGE_IND_CODE

NUMBER (1)

Pseudo-column

Used for updates only

If set to 1 (TRUE), no notification is sent to the service cache of any updates to the product instance.

It can be used for bulk updates of product instances before a restart of the service cache.

If not specified, the service cache is purged after the update is committed.

DO_NOT_UPDATE_STATUS_
IND_CODE

NUMBER (1)

Pseudo-column

Used for updates only

If set to 1 (TRUE), updates to a base product instance's status are not automatically applied to its:

  • Companion product instances
  • Services
  • Facility group instances.

IGNORE_OFFER_END_DATE_
IND_CODE

NUMBER (1)

Pseudo-column

Used for inserts and updates only

If set to 1 (TRUE), the end of the availability period of the associated product offer is ignored.

The insert or update succeeds provided that the start of the availability period of the product offer is no later than the EFFECTIVE_START_DATE of the product instance.

Remarks

The following rules apply to the statuses of base product instances and their related entities:

  • Changing the status of a product instance to Cancelled, Moved or Upgraded will change the status of its services, companion product instances and facility group instances to Cancelled, Moved or Upgraded respectively, irrespective of their current status and irrespective of the value of the DO_NOT_UPDATE_STATUS_IND_CODE field supplied to biProductInstanceUpdate&().
  • Changing the status of a product instance to Active will change the status of all its services, companion product instances and facility group instances to a status of Active if they have either the same status as the product instance, or a status of Provisioned, and if the biProductInstanceUpdate&() function is called with no value specified for the DO_NOT_UPDATE_STATUS_IND_CODE field. Otherwise their statuses will not be changed.
  • Changing the status of a product instance to any status other than Cancelled will change the status of all its services, companion product instances and facility group instances that had the same status as the product instance to the new status if the biProductInstanceUpdate&() function is called with no value specified for the DO_NOT_UPDATE_STATUS_IND_CODE field. Otherwise their statuses will not be changed.
  • A service, companion product instance or facility group instance cannot have a status of Active unless their base product instance has a status of Active.
  • A service, companion product instance or facility group instance must have a status of Cancelled, Moved or Upgraded if their base product instance has a status of Cancelled, Moved or Upgraded respectively.
  • A service must have a status of Product Creation if its base product instance has a status of Product Creation.
  • A base product instance with a status of Product Creation cannot have any companion product instances or facility group instances.
