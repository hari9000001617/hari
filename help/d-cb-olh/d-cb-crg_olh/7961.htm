INV_ADJUSTMENT_V

The INV_ADJUSTMENT_V view defines all invoice adjustments known to the system. It is used by the IGP.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_ID

NUMBER (15)

Not null

ID of any adjustment.

ACCOUNT_ID

NUMBER (15)

Not null

Account against which this adjustment was made.

INVOICE_ID

NUMBER (15)

Not null

Invoice on which this adjustment is displayed.

This is not the invoice against which the adjustment was made.

LAST_MODIFIED

DATE

Not null

Date-time when this record was last modified.

ADJUSTMENT_NR

VARCHAR (30)

Not null

ID for this adjustment that can be referenced by users.

It is automatically generated, based on the ADJUSTMENT_FORMAT_EXPR of the adjustment type and ADJUSTMENT_ID.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Standard category for the adjustment entered.

If the adjustment is associated with a batch, it defaults to the adjustment type of the batch.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

Standard category for the adjustment entered.

If the adjustment is associated with a batch, it defaults to the adjustment type of the batch.

DESCRIPTION

VARCHAR (255)

Not null

Description of the adjustment.

It is not displayed on the invoice, but entered by the user, and can contain the justification for the adjustment.

If the adjustment is associated with a batch, it defaults to the description of the batch.

ADJUSTMENT_TEXT

VARCHAR (80)

Not null

Text displayed on the customer invoice in the summary section with the details of this invoice adjustment.

It defaults to the text associated with the adjustment type, but it can be modified by the user.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who entered the adjustment.

AMOUNT

NUMBER (19,3)

Not null

Adjustment amount in the currency of the account.

A positive adjustment indicates a credit to the customer that must be included on the invoice; for example, a customer has difficulties with their service during the last invoice period so the service rental charge for that invoice is being refunded.

A negative amount indicates a debit.

ADJUSTMENT_AMOUNT

NUMBER (19,3)

Not null

Adjustment amount using the currency of the adjustment.

It is equal to AMOUNT, unless the currency is different.

ADJUSTMENT_CURRENCY_ID

NUMBER (8)

Not null

ID of the currency in which the adjustment is made.

ADJUSTMENT_DATE

DATE

Not null

Date-time when the adjustment was made.

ADJUSTMENT_RECEIVED_DATE

DATE

Not null

Date-time when the adjustment was applied to the account, allowing adjustments to be backdated.

ADJUSTMENT_DUE_DATE

DATE

Date-time at which the adjustment was due.

It is usually only set for adjustments representing debit notes.

ADJUSTMENT_INVOICE_CODE

NUMBER (2)

Not null

Summary of the details in the ADJUSTMENT_INVOICE table for this adjustment.

Valid values are:

  • Invoice (0): The adjustment is against a single invoice (distributed across zero or more receivable types).
  • Split (1): The adjustment is against one or more invoices, adjustments, payments, and/or a direct credit or debit to the account.
  • Unallocated (2): The adjustment is a direct credit to the account (distributed across zero or more receivable types).
  • Adjustment (3): The adjustment is against another adjustment (for example, where a debit note is allocated to a credit note).
  • Payment (4): The adjustment is against a payment (for example, where a debit note is allocated to a payment).

ADJUSTMENT_INVOICE_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is Invoice (0), this is the ID of the single invoice in the ADJUSTMENT_INVOICE table for this adjustment.

ALLOCATED_ADJUSTMENT_ID

NUMBER (15)

Derived

If the ADJUSTMENT_INVOICE_CODE is Adjustment (3), it is the ID of the single adjustment in the ADJUSTMENT_INVOICE table.

It is derived and cannot be specified for Insert or Update operations.

PAYMENT_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is Payment (4), it is the ID of the single payment in the ADJUSTMENT_INVOICE table for this adjustment.

CUSTOMER_INVOICE_STR

VARCHAR (30)

ID for the invoice for which this adjustment was made.

It only has a value if ADJUSTMENT_INVOICE_ID is not null.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be defined in the type definition table.

The constraints of each column are defined by the entity validation associated with its ADJUSTMENT_TYPE.

 

Charge Partition Details

Column Name

Properties

Description

PARTITION_NR

NUMBER (2)

Not null

Customer partition number associated with the charge generated by the adjustment.

Because all charges associated with an invoice are associated with the same account (and therefore customer), when customer partitioning is implemented, all charges for an invoice have the same PARTITION_NR.

It can be used as an additional selection criterion to improve access to the CHARGE table by eliminating irrelevant CHARGE table partitions.
