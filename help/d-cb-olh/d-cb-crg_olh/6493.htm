Individual Adjustment Detail Form

The following table lists automation parameters for the Individual Adjustment Detail form, data type, and supported functions.

The description column is provided to supplement the description in the ADJUSTMENT_TRE_V view, or to provide information about parameters not contained within the database view.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

ADJUSTMENT_TYPE_ID

Integer

Insert

Update

Type of adjustment.

Note:

It is recommended that this field be passed in first. Failure to do so might cause other fields to be ignored. An error occurs if TO_ACCOUNT_ID is passed in before ADJUSTMENT_TYPE_ID.

DESCRIPTION

String

Insert

Update

Description of the adjustment.

DEBIT_ACCOUNT_ID

(FROM_ACCOUNT_ID)

Integer

Insert

Update

Account from which the adjustment is deducted.

FROM_ACCOUNT_ID is deprecated and provided only for backward compatibility.

CREDIT_GL_CODE_ID

(FROM_GL_CODE_ID)

Integer

Insert

Update

GL code to credit with the adjustment amount.

FROM_GL_CODE_ID is deprecated and provided only for backward compatibility.

ADJUSTMENT_TEXT

String

Insert

Update

Text to be displayed on the customer's invoice.

CREDIT_ACCOUNT_ID

(TO_ACCOUNT_ID)

Integer

Insert

Account to which the adjustment is to apply.

It can only be specified if the ADJUSTMENT_TYPE_ID has been supplied.

TO_ACCOUNT_ID is deprecated and provided only for backward compatibility.

An error occurs if CREDIT_ACCOUNT_ID is passed in before ADJUSTMENT_TYPE_ID.

DEBIT_GL_CODE_ID

(TO_GL_CODE_ID)

Integer

Insert

Update

GL code to debit with the adjustment amount.

It can only be updated if the adjustment is not applied.

PERSON_ID

Integer

Insert

Update

Customer associated with the adjustment.

AMOUNT

Real

Insert

Update

Adjustment amount.

CURRENCY_ID

Integer

Insert

Update

Adjustment currency.

It can only be updated if the adjustment is not applied.

CURRENCY_CONVERSION_DATE

Date

Insert

Update

Effective date-time of the conversion rate to use when converting amounts.

It can only be updated if the adjustment is not applied.

ADJUSTMENT_DATE

Date

Insert

Date-time the adjustment was made.

ADJUSTMENT_RECEIVED_DATE

Date

Insert

Update

Date-time the adjustment is applied to the account.

CUSTOMER_QUERY_ID

Integer

Insert

Update

Reference to the customer query that caused the adjustment.

GENERAL_1 - GENERAL_10

String

Insert

Update

General fields 1 - 10 for the adjustment.

INVOICE_ID

Integer

Insert

If the ADJUSTMENT_INVOICE_CODE is 0 (Invoice), this is the ID of the single invoice in the ADJUSTMENT_INVOICE table for the adjustment.

ALLOCATED_ADJUSTMENT_ID

Integer

Insert

ID of the single adjustment in the ADJUSTMENT_INVOICE table.

INVOICE_FORMAT_ID

Integer

Insert

Update

ID of the adjustment note invoice format.

ENABLE_AGING_IND_CODE

Integer

Insert

Update

If set to 1 (True), unallocated amounts associated with an adjustment are aged according to the account type aging rules of TO_ACCOUNT_ID.

Aging cannot be enabled for transfer adjustments.

It can only be set to 1 if FROM_ACCOUNT_ID is NULL.

If not set to 1, it must be set to NULL.

PENDING_GENERATION_IND_CODE

Integer

Insert

Update

If set to 1 (True), the adjustment is waiting for adjustment image generation (or regeneration).

IMAGE_GENERATION_IND_CODE

Integer

Insert

Update

If set to 1 (True), an image is generated for the adjustment.

It cannot be specified as part of an Insert or Update operation.

It is populated automatically when the adjustment image is generated or regenerated.

PENDING_PRINT_IND_CODE

Integer

Insert

Update

If set to 1 (True), the image for the adjustment is awaiting printing (or reprinting).

It can only be set to 1 if an adjustment image was or is waiting to be generated.

PRINTED_IND_CODE

Integer

Insert

Update

If set to 1 (True), the image for the adjustment was printed or emailed.

When the image is printed (or reprinted), it is automatically set to 1.

STORED_IMAGE_TYPE_CODE

Integer

Insert
Update

Type of image stored in ADJUSTMENT_IMAGE, derived from the value stored in INVOICE_FORMAT_ID at image creation.

It is set by biAdjustmentImageGenerate&().

PAYMENT_ID

Integer

Insert

ID of the single payment in the ADJUSTMENT_INVOICE table.
