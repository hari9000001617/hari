DIRECT_VARIABLE_V

The DIRECT_VARIABLE_V view displays the details of all direct variables and their view details.

Note:

A direct variable can be referenced by multiple views. Therefore, this view may return multiple rows for a given variable name and date.

Column Definitions

Column Name

Properties

Description

VARIABLE_DEFN_ID

NUMBER (8)

Not null

ID of a variable.

VARIABLE_DEFN_NAME

VARCHAR (30)

Not null

Name of this variable.

In general, variable names cannot be changed. The name of the variable must conform to the following rules:

  • The first character must be in the set (A-Z, a-z, _).
  • Subsequent characters except the last must be in the set (A-Z, a-z, 0-9 or _).
  • The last character must be one of &, #, $ or ~ (depending on the data type).

Naming rules are enforced by the C_VARIABLE_DEFN_NAME check constraint.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date when this instance of the variable became effective.

EFFECTIVE_END_DATE

DATE

Not null

Date when this instance of the variable is no longer effective.

DATA_TYPE_CODE

NUMBER (2)

Not null

Type of the variable (integer, real, string, or date-time).

VARIABLE_TYPE_CODE

NUMBER (2)

Not null

Type of the variable. Valid values are:

  • 0 (Direct): The variable is directly defined from a database table and column.
  • 1 (Derived Attribute): The variable is a derived attribute (refer to DERIVED_ATTRIBUTE table)
  • 2 (Tariff): The variable is a type of tariff or discount.
  • 3 (Subtotal): The variable is a subtotal (simple or conditional accumulation of one or more other variables).
  • 4 (Function): The variable is a function.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which this variable operates. Valid values are:

  • 0 (Any)
  • 1 (Normalised Event)
  • 2 (Service)
  • 3 (Customer Node)
  • 4 (Customer).

For direct variables, this effectively defines the table and query conditions that apply (except that Customer Node context also covers queries on the CUSTOMER_NODE_HISTORY, ACCOUNT, and INVOICE tables).

The CONTEXT_CODE of Any is used for when the VARIABLE_TYPE_CODE is Function and the function does not reference any variables.

APPLICATION_ENV_CODE

NUMBER (2)

Not null

Application execution environment in which this variable can be used. Valid values are:

  • 0 (Any)
  • 1 (Data Entry)
  • 2 (Rating)
  • 3 (Billing)
  • 4 (Invoicing)
  • 7 (PE)
  • 8 (GL Guidance)
  • 9 (EVS).

Variables in the environment PE must have a CONTEXT_CODE of Any and a VARIABLE_TYPE_CODE of Function or Direct.

Variables in the environments GL Guidance and EVS must have a CONTEXT_CODE of Any and a VARIABLE_TYPE_CODE of Function.

VIEW_NAME

VARCHAR (30)

Not null

Name of the database view where the value of the variable can be obtained.

COLUMN_NAME

VARCHAR (30)

Not null

Name of the database column of this view where the value of the variable can be obtained.

COLUMN_NR

NUMBER (3)

Not null

Sequence number for ordering the column in the specified view.

It defaults based on the order of the column in the view (if the view exists).

Note:

The COLUMN_NR value for a direct variable does not have to match the column number of the corresponding column in the view. However, each COLUMN_NR value for each direct variable in a view must be unique.

DESCRIPTION

VARCHAR (255)

Not null

General description for the use of this variable in conjunction with this view.
