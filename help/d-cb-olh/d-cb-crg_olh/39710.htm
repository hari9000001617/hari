biCustomerNodeReencrypt&()

Syntax

biCustomerNodeReencrypt&(
    CustomerNodeId&,
    EffectiveDate~,
    var LastModified~,
    Cascade&)

 

Description

Re-encrypts data belonging to a specific customer, including related records.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

CustomerNodeId&

In: ID of the customer node to re-encrypt.

EffectiveDate~

In: Effective date-time of the customer node data to re-encrypt.

LastModified~

In: Date-time the customer node was last modified.

Out: Date-time the customer node was updated by this function.

Cascade&

In: Scope of data to be re-encrypted.

If set to:

  • 0 (FALSE), only the customer node record and immediate customer dependencies are re-encrypted.
  • 1 (TRUE), all dependent records that belong to the customer node's security group are re-encrypted.

For details of the scope of data re-encrypted, refer to Remarks below.

 

Returns

1 if one or more fields are re-encrypted; otherwise, 0.

Remarks

This function should be called whenever the security group (GROUP_ID) is updated for a customer node. Only encrypted fields are updated.

The function scans the records specified by CustomerNodeId& and Cascade& for encrypted fields, and re-encrypts the data in those fields using the new security group's encryption key.

If Cascade& is set to 0 (FALSE), the following records are re-encrypted:

  • Customer node record, and associated postal and site addresses
  • Customer invoice format records
  • Customer derived attribute array records
  • Customer query records.

If Cascade& is set to 1 (TRUE), all records dependent on the customer security group are scanned for re-encryption. This includes:

  • All of the records listed above
  • Account and account reservation records
  • Invoice records
  • Payment records
  • Adjustment records
  • Product instance records, including:
    • Base products, companion products, and services
    • Derived attribute tables associated with the services.
