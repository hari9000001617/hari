effective_date()

Syntax

Interface 1

 

Interface 2

 
effective_date()
 
 
 
effective_date(
    date)

Description

Returns the effective date-time that is used to evaluate the current expression.

Interface 2 can be used to set the effective date-time for the remainder of the current expression or function.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

date

Effective date-time used for the remainder of the current expression.

 

Returns

The following table displays the possible return values and the condition under which the function is called.

 

Return Value

Condition

Original value of the effective date-time.

Called from an expression with date-time set by an earlier call to effective_date(date).

Effective date-time of the current bill run.

Called in an expression evaluated during billing (for example, in an IGP or BGP process).

Called in an expression evaluated in the RGP or RAP and the tariff is in Bill Cycle Issue Date (BIDS) mode.

Start date of the event.

Called in an expression evaluated in the RGP or RAP and the tariff is in Rental Event Period Start date (REPS) mode.

Called in a rating expression within the ERT or TRERATE process, or from a file decoding expression in the ENM process.

Date-time that the ENM process was started.

Called in an expression evaluated as part of a DIL specification in an ENM process.

EffectiveDate~ specified by the calling process that made the remote TRE call.

Called from a function being remotely evaluated in a Tuxedo server.

 

Remarks

The value of the effective date-time varies according to the process and context in which the expression is being evaluated.

The effective date is typically set to the current date-time at which the call was made. However, the calling process can specify any EffectiveDate~. 

For more information, refer to the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 1).
