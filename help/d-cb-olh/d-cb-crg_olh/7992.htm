PRODUCT_ORDER_V

The PRODUCT_ORDER_V view defines the order in which services and equipment are displayed in the list of tasks displayed when creating a product instance of a product.

The view is based on the PRODUCT_SERVICE_TYPE and PRODUCT_SERVICE_TYPE_EQUIP tables.

Column Definitions

Column Name

Properties

Description

PRODUCT_ID

NUMBER (8)

Not null

ID of a product.

PRODUCT_SERVICE_TYPE_ID

NUMBER (8)

Not null

ID of a type of service used for this product.

EFFECTIVE_START_DATE

DATE

Not null

Date-time when this record becomes effective for the product.

It must match an effective start date for the product in the PRODUCT_HISTORY table.

It is checked by the DVP.

EFFECTIVE_END_DATE

DATE

Not null

End date-time when this record becomes effective for the product.

It must match an effective end date for the product in the PRODUCT_HISTORY table.

It is checked by the DVP.

SERVICE_ITEM_SEQNR

NUMBER (4)

Not null

ID of a service type within the product.

SERVICE_RANGE_MIN

NUMBER (4)

Minimum number of service instances with the specified SERVICE_ITEM_SEQNR that must be created on an instance of this product.

The following constraints apply:

  • It is mandatory for base products, and always null for companion products. It is checked by the DVP.
  • The minimum value is 1 if SERVICE_ITEM_SEQNR = 1; otherwise, 0. There must be at least a single instance of the first service on a product. It is enforced by C_PRODUCT_SERVICE_TYPE check constraint.

SERVICE_RANGE_MAX

NUMBER (4)

Maximum number of service instances with the specified SERVICE_ITEM_SEQNR that can be created on an instance of this product.

The following constraints apply:

  • It must contain a value if SERVICE_RANGE_MIN contains a value, and must be null if SERVICE_RANGE_MIN is null. It is enforced by C_PRODUCT_SERVICE_TYPE check constraint.
  • If it contains a value, it must be greater than or equal to SERVICE_RANGE_MIN. It is enforced by C_PRODUCT_SERVICE_TYPE check constraint.

SERVICE_TYPE_ID

NUMBER (8)

Not null

ID of the type of service associated with this SERVICE_ITEM_SEQNR.

EQUIPMENT_ITEM_SEQNR

NUMBER (4)

ID of the equipment type within a service type of a product.

EQUIPMENT_RANGE_MIN

NUMBER (4)

Minimum number of equipment items with the specified EQUIPMENT_ITEM_SEQNR that must be allocated to the corresponding service instance of this product.

The following constraints apply:

  • It is mandatory for base products, and always null for companion products. It is checked by the DVP.
  • The value must be greater than or equal to 0. It is enforced by C_PRODUCT_SERVICE_TYPE_EQUIP check constraint.

EQUIPMENT_RANGE_MAX

NUMBER (4)

Maximum number of equipment items with the specified EQUIPMENT_ITEM_SEQNR that must be allocated to the corresponding service instance of this product.

The following constraints apply:

  • It must contain a value if EQUIPMENT_RANGE_MIN contains a value, and must be null if EQUIPMENT_RANGE_MIN is null It is enforced by C_PRODUCT_SERVICE_TYPE_EQUIP check constraint.
  • If it contains a value, it must be greater than or equal to EQUIPMENT_RANGE_MIN, enforced by C_PRODUCT_SERVICE_TYPE_EQUIP check constraint.

EQUIPMENT_TYPE_ID

NUMBER (8)

ID of the equipment type associated with this EQUIPMENT_ITEM_SEQNR for this service.

SEARCH_ENTITY_
VALIDATION_ID

NUMBER (8)

Optional set of labels, hint text and defaults for the search fields to be used on the Equipment Allocation form for the allocation of this equipment to an instance of this product.

Unlike other forms, the order of the attribute types associated with this entity validation specifies the order in which the attribute types are displayed on the Equipment Allocation form.

MAX_ROWS_DISPLAYED

NUMBER (3)

Optional maximum number of rows of equipment of Idle status to display on the Equipment Allocation form for the allocation of this equipment to an instance of this product.

CREATION_ORDER

NUMBER (4)

Not null

Order defaults to all equipment required for a service, then the service.

For example, service type 1 requires equipment type 1 and equipment type 2, and service type 2 requires equipment type 3 and equipment type 4. Therefore, creation order is:

  1. Equipment type 1
  2. Equipment type 2
  3. Service type 1
  4. Equipment type 3
  5. Equipment type 4
  6. Service type 2.

SERVICE_NAME_EQUIPMENT_
CO

NUMBER (4)

Creation order of the item of equipment where the service name is allocated for this service.

If not specified, the service is not allocated a unique name automatically.

The default is based on the service type.

It is optional for base products and always null for companion products. It is checked by the DVP.

The value must correspond to a row in the PRODUCT_SERVICE_TYPE_EQUIP table for the product and date range.

It is checked by the DVP.

SERVICE_NAME_SERIALNR_
IND_CODE

NUMBER (1)

Only contains a non-null value if SERVICE_NAME_EQUIPMENT_CO is not null, enforced by C_PRODUCT_SERVICE_TYPE check constraint.

In this case, if it is set to 1 (TRUE), the serial number is used to define the service name; otherwise, the network address is used.

The default is based on the service type.

NETWORK_NAME_EQUIPMENT_
CO

NUMBER (4)

Creation order of the item of equipment where the network name for this service is allocated.

If not specified, the service is not allocated a network name automatically by the system.

It is optional for base products and always null for companion products.

It is checked by the DVP.

It must correspond to a row in the PRODUCT_SERVICE_TYPE_EQUIP table for the product and date range.

NETWORK_NAME_SERIALNR_
IND_CODE

NUMBER (1)

Only contains a non-null value if NETWORK_NAME_EQUIPMENT_CO is not null, enforced by C_PRODUCT_SERVICE_TYPE check constraint.

In this case, if set to 1 (TRUE), the serial number is used to define the network name; otherwise, the network address is used.

The default is based on the service type.
