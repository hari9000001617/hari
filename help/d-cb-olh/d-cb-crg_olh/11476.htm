biBillRunImmediate&()

Syntax

biBillRunImmediate&(
    CustomerNodeId&,
    FieldNames$[],
    FieldValues?[],
    FromOperation&,
    ToOperation&)

 

Description

Creates an immediate bill run and performs the specified range of operations for a customer hierarchy. Standard and quality assurance bill runs can be performed.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

CustomerNodeId&

In: Internal ID of the root customer node for which this bill run is being performed.

FieldNames$[]

In: Names of the fields to set for the bill run.

Valid field names can be found in the BILL_RUN_TRE_V view.

FieldValues?[]

In: Field values to insert.

They correspond to the fields listed in FieldNames$[]).

FromOperation&

In: First operation of the range of calling programs to process

It must be less than 128 or greater than 128.

Valid operational codes can be found in the BILL_RUN_OPERATION_TRE_V view for the BILL_RUN_OPERATION_CODE field.

ToOperation&

In: Last operation of the range of calling programs to process.

It must be less than 128 or greater than 128.

Valid operational codes can be found in the BILL_RUN_OPERATION_TRE_V view for the BILL_RUN_OPERATION_CODE field.

 

Returns

ID of the immediate bill run created, if successful; otherwise, an error is raised indicating the reason for the bill run failure.

Remarks

If FieldNames$[] and FieldValues?[] are empty, an immediate bill run with an effective date equal to the current date-time is performed.

The following table lists fields in BILL_RUN_TRE_V that have a default.

 

BILL_RUN_TRE_V Default Values

Field

Default Value

BILL_RUN_TYPE_ID

3 (Immediate Bill Run)

EFFECTIVE_DATE

Current date

BILLING_SCHEDULE_ID

Undefined if CREATION_TASK_ID is defined, but BILLING_SCHEDULE_ID is not defined.

CUSTOMER_COUNT

1

STATUS_CODE

Value returned by ReferenceCodeByLabel&('TASK_STATUS','PENDING')

SUMMARY_STATUS_CODE

Value returned by ReferenceCodeByLabel&('TASK_STATUS','PENDING')
