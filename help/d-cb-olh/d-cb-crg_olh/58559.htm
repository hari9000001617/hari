Interface 13

 

 

 
EntitlementConsume&(
    Context&,
    EntityId&,
    EntitlementId&,
    AllocationId&,
    EffectiveDate~, 
    ReservedAmount#,
    ConsumedAmount#)
 
 
 
 
 

 

Description

Consumes the available amount of an allocation.

Attributes

Function Type

Local

Service

(Local)

Application Environment

Any

Context

Any

 

Parameter

Description

Context&

In: Context of the entity being consumed. Valid values are:

  • 2 (Service).
  • 3 (CustomerNode).
  • 4 (Customer).

EntityId&

In: ID of the entity to which the entitlement being consumed belongs. It depends on the context of the entitlement.

For a context of:

  • Service, it corresponds to the service ID.
  • CustomerNode and Customer, it corresponds to the customer node ID.

Category&

In: Reference code of the entitlement category being consumed.

It must match a reference code in the ENTITLEMENT_CATEGORY reference type.

SubCategory$

In: Sub-category of the entitlement being consumed.

It may be passed as an empty string.

EntitlementId&

In: ID of the entitlement.

AllocationId&

In: ID of the allocation.

EffectiveDate~

In: Effective date of the consumption.

Amount#

In: Amount being consumed.

It must be positive; otherwise, an error is raised.

Note:

An error is raised if the requested amount is greater than the available amount for the entitlement.

ReservedAmount#

In: Amount reserved for the reservation. This amount excludes the consumed amount. Only applies in rate and return mode.

The total amount of the reservation is the sum of this reserved amount and the consumed amount. This total must be positive; otherwise, an error is raised.

ConsumedAmount#

In: Amount consumed from the reservation. Only applies in rate and return mode.

Note:

Consumed amounts are unavailable to concurrent usage sessions.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

The allocations are consumed in order of priority and expiry date, as follows:

  • Highest priority, which is the lowest numerical value.
  • Earliest expiry date.

Multiple categories can be specified using interfaces 4, 5, or 6. In this case, allocations are consumed based first on the order of priority, and then expiry date.

If an interface is called with no EffectiveDate~ parameter, the effective date of the expression is used. In a Rating context, EffectiveDate~ is the charge start date.

In rate and return mode, the reserved and consumed amount of the reservation can be explicitly specified using interfaces 11-13. The other interfaces specify a single amount which is automatically split into the reserved and consumed components on commit, according to the mode of reservation consumption specified in EntitlementEventCommit&(). Both the manual and automatic EntitlementConsume&() interfaces may be called within the same session. The manual amounts are added to the automatically split components when the changes are committed.
