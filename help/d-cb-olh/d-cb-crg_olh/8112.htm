CALL_DATA_ERROR_CV

The CALL_DATA_ERROR_CV view stores details on normalised events (call data) that could not be processed.

Column Definitions

Column Name

Properties

Description

NORMALISED_EVENT_ID

NUMBER (15)

ID of the normalised event.

NORMALISED_EVENT_TYPE

VARCHAR (30)

Type of this normalised event.

NORMALISED_EVENT_FILE

VARCHAR (255)

File from which this normalised event record was obtained.

ERROR_MESSAGE

VARCHAR (4000)

Error message test explaining why this event failed to be processed.

It should include details on data item in error.

A_PARTY_ID

VARCHAR (40)

Full national number of the calling party (A party).

A_PARTY_NAME

VARCHAR (40)

Name of the A party.

A_PARTY_CARRIER

VARCHAR (60)

Carrier from which call arrived.

A_PARTY_LOCATION

VARCHAR (60)

Geographic location that specifies the origin of the call.

B_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for the B party service.

B_PARTY_NAME

VARCHAR (40)

Name of the B party, if known.

B_PARTY_CARRIER

VARCHAR (60)

Carrier to which call is destined (may not be known).

B_PARTY_LOCATION

VARCHAR (60)

Geographic location that specifies the destination of the call.

C_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for a chargeable party service.

C_PARTY_NAME

VARCHAR (40)

Name of the C party.

C_PARTY_CARRIER

VARCHAR (60)

Carrier associated with the chargeable party.

C_PARTY_LOCATION

VARCHAR (60)

Geographic location that specifies the destination of the chargeable party.

CHARGE_DATE

DATE

Date-time used for charging purposes.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

DURATION

NUMBER

Duration of call in seconds.

CHARGE

NUMBER (22,6)

Charge as it was received from the switch. The units are dependent on the switch.

Note:

CHARGE is not generated within Convergent Billing.

CURRENCY_ID

NUMBER (8)

Currency for the above charge.
