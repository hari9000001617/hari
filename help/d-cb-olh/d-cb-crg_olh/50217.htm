ProcessSpecificStats?{}()

Syntax

ProcessSpecificStats?{}

 

Description

Provides statistics specific to the current process.

Attributes

Function Type

Local Built-in

Service

(Local)

Application Environment

Any

Context

Any

 

Parameter

Description

None.

 

 

Returns

Hash containing process performance statistics for the STD, TLM or SFM process.

Remarks

This function returns statistics for the Schedule Task Dispatcher (STD), Task Launch Monitor (TLM), and the Scheduled Function Monitor (SFM) processes, depending on where it is called. The biTREServerMonitor&() function calls this function to collect the statistics from these processes for logging to the tremon server.

The following table lists ThreadsPool hash keys.

 

ThreadsPool Hash Keys

Key

Type

Description

Size

Long

Size of the threads pool.

It is the highest number of threads that have been active at the same time (Running + Stopping).

Running

Long

Number of threads currently in use, excluding any threads that are stopping.

It is expected to equal should be equal to the value of MaxEvalThreads + 3.

Stopping

Long

Number of threads that are waiting to complete their execution before they are stopped.

It is only populated when the thread count is being scaled down or the SFM is stopping.

 

The following table lists Functions array hash keys.

 

Functions Array Hash Keys

Key

Type

Description

Category

String

Category of the functions.

The value is the category code abbreviation defined by the SCHEDULED_FUNC_CATEGORY reference type.

If the category code does not exist in the reference type, the category code is used as the Category value.

Success

Long

Total number of scheduled functions in Category that have been evaluated successfully.

Error

Long

Total number of scheduled functions that have completed evaluation with a status of Failure or Failure (Retries Pending).

Failures

Long

Number of functions that have failed evaluation and are updated to a status of Failure after evaluation.

The functions are included in the Error count.

ElapsedTimeSuccess

Real

Total elapsed time (in seconds) for scheduled functions in Category that completed with a status of Success.

ElapsedTimeError

Real

Total elapsed time (in seconds) for scheduled functions with this Category that completed with a status of Failure or Failure (Retries Pending).

DelayTime

Real

Total delay time (in seconds) for scheduled functions in Category.

The delay time is the time difference between the SCHEDULED_START_DATE value of a scheduled function and the date-time at which it was evaluated.

DelayTimeMax

Real

Largest delay time (in seconds) between the SCHEDULED_START_DATE of a scheduled function in Category and the date-time at which it was evaluated.

QueueSize

Long

Number of functions in Category currently in the queue of pending scheduled functions.

QueueSizeMax

Long

Largest number of functions in Category that were in the queue of pending scheduled functions.

ThreadsLimit

Long

Maximum number of threads that can evaluate scheduled functions in Category.

If undefined, no limit has been set for this category.

ThreadsCurrent

Long

Number of threads currently evaluating scheduled functions in Category.

ThreadsMax

Long

Highest number of threads that have evaluated scheduled functions in Category at the same time.

 

The following table lists BatchFunctions array hash keys.

 

BatchFunctions Array Hash Keys

Key

Type

Description

Category

String

Category of the functions processed as a batch.

The value is the category code abbreviation defined by the SCHEDULED_FUNC_CATEGORY reference type.

If the category code does not exist in the reference type, the category code is used as the Category value.

BatchSuccess

Long

Total number of batches completed successfully in Category.

Note:

Individual scheduled functions in the batch may still have failed with an error. If all scheduled functions in a batched fail with an error but the batch is not rolled back, it is considered a batch success.

BatchError

Long

Total number of batches that failed in Category, where the batch function itself returned an error and it failed to process any of the scheduled functions in the batch.

Success

Long

Total number of individual functions processed in batches that completed successfully. This count is also included in the Functions category Success count.

Error

Long

Total number of individual functions processed in batches that failed individually or as part of a BatchError. This count is also included in the Functions array Error count.

ElapsedTimeBatchSuccess

Real

Total elapsed time (in seconds) for all successfully completed batches in Category. This time is included in the Functions array ElapsedTimeSuccess count.

ElapsedTimeBatchError

Real

Total elapsed time (in seconds) for all batches in Category that that failed with an error. This time is included in the Functions array ElapsedTimeError count.
