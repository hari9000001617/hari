EquipmentAllocateRange&()

Syntax

EquipmentAllocateRange&(
    ProductInstanceId&,
    EffectiveDate~,
    var LastModified~,
    EquipmentId&,
    var EquipmentLastModified~,
    CreationOrder&,
    ServiceRangeSeqNr&,
    EquipRangeSeqNr&)

 

Description

Allocates the specified equipment item to the specified product instance from EffectiveDate~ to the end of time.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

ProductInstanceId&

In: ID of the product instance to which the equipment is to be allocated.

EffectiveDate~

In: Date-time from which the equipment is to be allocated to the product instance.

LastModified~

In: Last modified date-time of the product instance to which the equipment is to be allocated.

Out: Last modified date-time of the product instance to which the equipment was allocated.

EquipmentId&

In: ID of the equipment being allocated.

EquipmentLastModified~

In: Last modified date-time of the equipment instance to be allocated.

Out: Last modified date-time of the allocated equipment instance.

CreationOrder&

In: Identifies the equipment item required by the product instance.

ServiceRangeSeqNr&

In: Range sequence number of the service within the product.

EquipRangeSeqNr&

In: Range sequence number of the equipment within the product.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The unique combination of CreationOrder&, ServiceRangeSeqNr&, and EquipRangeSeqNr& indicates the item required by the product instance that is being allocated.

Any equipment that is allocated to the ProductInstanceId& for the specified creation order and range sequence numbers over the allocation date-time range is de-allocated by having its status re-derived with EquipmentDeriveStatus&.

The equipment specified must be of the equipment type required by the product instance for this creation order/range sequence number combination.

The status of the equipment is updated to its specified initial status for any date-time range where its status was Idle.

To enforce consistency between service name/network name and equipment serial number/network address, all services associated with any equipment items de-allocated because of the allocation are updated for the date-time range of the deallocation. This in turn may perform date-time range updates against each service's base product instance to keep them aligned with the service.

The LastModified~ value passed must be the last modified date-time of the product instance as at EffectiveDate~. The value of LastModified~ passed back is the new last modified date-time for the product instance after the allocation.

The EquipmentLastModified~ value passed must be the last modified date-time of the equipment item being allocated as at EffectiveDate~. The value of EquipmentLastModified~ passed back is the new last modified date-time for the equipment item after the allocation. The effective user must have update security group access level for the group associated with the customer and the group associated with the equipment record manipulated by this function call. If the effective date of the update is less than the current date (00:00:00 time stamp), direct update security group access level is required.

GROUP_ACCESS_TRE_V view lists user group access level values.
