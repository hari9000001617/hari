LicenceStatus&()

Syntax

LicenceStatus&(
    Entity$,
    var CurrentValue$,
    var Limit$,
    var WarningPercent&,
    var MaxPercent&,
    var ChallengeString$)

 

Description

Returns the current status (the last call to LicenceRecalc&() or server startup) of Entity$.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

Entity$

In: Entity for which the status is required. Valid values are:

  • SERVICE
  • CUSTOMER_NODE
  • CUSTOMER
  • OPERATOR
  • CEXP
  • INVD
  • CE
  • PE
  • ENC.

CurrentValue$

Out: Current entity count for the specified entity.

Limit$

Out: Licence limit for the current entity.

WarningPercent&

Out: Percentage that determines if 0 or 1 is being returned.

MaxPercent&

Out: Percentage that determines if 2 or 3 is being returned.

ChallengeString$

In/Out: If specified, the value is used to generate a returned ChallengeString$.

The calling process can perform validation on the returned value to assert that the request was handled by an authorised process.

Use of this parameter is limited to internal Singleview processes only, because no external interface exists to reconstruct the parameter.

The parameter is not displayed in any trace output.

 

Returns

The following is returned for the SERVICE, CUSTOMER_NODE, CUSTOMER, and OPERATOR entities:

  • 0 if the entity count is less than the licence limit and warning percentage
  • 1 if the entity count is less than the licence limit but within the warning percentage of limit
  • 2 if the entity count is greater than the licence limit but within the maximum percentage over limit
  • 3 if the entity count is greater than the licence limit and greater than the maximum percentage over limit.

The following is returned for the CEXP, INVD, CE, PE, or ENC entities:

  • 0 if the entity is licensed for use
  • 3 if the entity is not licensed for use.

Each time 1, 2, or 3 is returned, a corresponding TRE event (biLicenceWarningLT(), biLicenceWarningGT(), and biLicenceError() respectively) is posted.

If the function is not successful, an error is raised.

Remarks

If the licence is valid and no entity limit is separately specified for the CUSTOMER, CUSTOMER_NODE, or SERVICE entities, the call for the entity returns CurrentValue$, Limit$, WarningPercent&, and MaxPercent& as -1.
