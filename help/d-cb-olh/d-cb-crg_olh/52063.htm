Remarks

External definitions require interface and validation functions to operate.

Interface functions are used to update database entities or objects linked with external definitions.

Validation functions are used to:

  • Validate input from external systems to Singleview interfaces.
  • Validate output from Singleview to external systems.
  • Choose the element or structure to process in a choice content model.

The biEDFunctionsGenerate&() (see the Configurer's Reference Guide for Singleview Convergent Billing) function generates interface and validation functions for an external definition. The specified external definition is treated as the root of a hierarchy of external definitions for the generation process. Functions for the root external definition and all external definitions associated with it are created. Functions can only be generated if all external definitions in the structure are validated.

To generate interface and validation functions, the root external definition must:

  • Be validated.
  • Be external.
  • Not be a simple choice content model.
  • Not use a choice element.
  • Be configured to do one or more of the following operations:
    • Insert
    • Update
    • Delete
    • Search
    • Fetch.

The interface functions generated are dependent on the operations configured for the root external definition. For example, if the root external definition for the generation is an insert definition, insert interface functions are generated for the structure.

Calls to the structure's interface functions from the root external definition's interface are based on the structure's sequence number and hierarchy depth.

In a nested hierarchy of external definitions with different entities, entities with an interface sequence number defined (API_PROCESSING_SEQNR) always take precedence over entities without a sequence number. For example, if only one external definition has API_PROCESSING_SEQNR defined at the deepest hierarchy depth, this external definition is the first entity called in the interface processing. If API_PROCESSING_SEQNR is defined and conflicts with nested entities, the priority is based on element sequence order and then hierarchy depth.

For nested external definition structures under an element that is the start of a two-hierarchy depth of one-to-many occurrences, to preserve the logical grouping of nested entities under one-to-many types and keep array of arrays in logical order, biEDFunctionsGenerate&() ignores API_PROCESSING_SEQNR after the first nested structure, and processes nested structures by hierarchy depth.

An element that is the start of a two-hierarchy depth of one-to-many occurrences can be defined as an element with elements with MIN_OCCURS > 1 for two or more hierarchy levels deep.

Single-occurrence elements of nested structure entities can be ordered by API_PROCESSING_SEQNR under a one-to-many element.

The values of elements with SENSITIVE_IND_CODE enabled are hidden in all output.

 

By default, the format of generated functions is:

<NS_PREFIX>:<ED_NAME>.<OPERATION>

where:

  • <NS_PREFIX> is the namespace prefix of the external definition.
  • <ED_NAME> is the external definition name.
  • <OPERATION> can be Insert, Update, Delete, Search, or Fetch.

If the function name specified by <NS_PREFIX>:<ED_NAME> is greater than 45 characters, the following applies:

  1. If <NS_PREFIX> is greater than 15 characters, <NS_PREFIX> becomes:

    (first seven characters of <NS_PREFIX>) + (external definition namespace ID)

  2. The resulting function name is recalculated.
  3. If the recalculated function name is still greater than 45 characters, <ED_NAME> becomes:

    (first 21 characters of <ED_NAME>) + (external definition ID)

If the external definition name contains characters not allowed in function names (for example, spaces), these characters are replaced with an underscore '_', and a check done to verify that the external definition name does not exist in the namespace. If the name exists, an error is raised.

External definitions that have been configured to do search operations require a linked fetch external definition.
