to_date()

Syntax

Interface 1

 

Interface 2

 
to_date(
    value?)
 
 
 
to_date(
    value$,
    format_string$)

Interface 3

 
 
 
 
 
to_date(
    value&,
    SECOND|MINUTE|HOUR|DAY)
 
 
 
 

 

Description

Interface 1 converts the supplied value to a date value. The value passed must be a date value or a runtime error is raised.

Interface 2 converts a string value in the specified format to a date value. If the string does not conform to the specified format, an error is raised.

Interface 3 returns a date duration that can be used to perform date arithmetic with other date values. For example, the following sets the due date to 14 days after the issue date:

InvDuedate~ := InvIssuedate~ + to_date(14, DAY)

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

value

In: Value to be cast or converted to a date.

SECOND|MINUTE|HOUR|DAY

In: Integer keywords (use exactly as displayed).

format_string$

In: Date format picture as follows.

 

Returns

A date.

Remarks

Note:

System locale settings are taken from the platform on which the expression is evaluated (see the Internationalisation Configuration Guide for Singleview).

The format_string$ parameter may include the format specifiers listed in the following table.

 

Format Specifiers

Specifier

Description

c

Interprets the string as a representation of the date 'short date' format, followed by the time in the long time format.

The short date and long time formats are specified in the system locale settings.

d

Interprets the string as a numerical representation of a day of the month without a leading 0 (for example, 1 or 31).

dd

Interprets the string as a two-digit number representing of a day of the month with a leading 0 if required (for example, 01 or 31).

ddd

Interprets the string as a case-sensitive abbreviation of a day of the week using the abbreviation format specified in the system locale settings.

dddd

Interprets the string as a case-sensitive day of the week using the full name format specified in the system locale settings.

ddddd

Interprets the string as a representation of a date using the short date format specified in the system locale settings.

dddddd

Interprets the string as a representation of a date using the long date format specified in the system locale settings.

m

Interprets the string as a numerical representation of a month without a leading 0 (for example, 1 or 12).

mm

Interprets the string as a two-digit number representing a month with a leading 0 if required (for example, 01 or 12).

mmm

Interprets the string as a case-sensitive abbreviation of a month using the abbreviation format specified in the system locale settings.

mmmm

Interprets the string as a case-sensitive month using the full name format specified in the system locale settings.

yy

Interprets the string as a two-digit number representing a year.

yyyy

Interprets the string as a four-digit number representing a year.

h

Interprets the string as a numerical representation of the hour without a leading 0 (for example, 1 or 12).

hh

Interprets the string as a two-digit number representing the hour with a leading 0 if required (for example, 01 or 12).

n

Interprets the string as a numerical representation of the minute without a leading 0 (for example, 1 or 59).

nn

Interprets the string as a two-digit number representing the minute with a leading 0 if required (for example, 01 or 59).

s

Interprets the string as a numerical representation of the seconds without a leading 0 (for example, 1 or 59).

ss

Interprets the string as a two-digit number representing the seconds with a leading 0 if required (for example, 01 or 59).

t

Interprets the string as a representation of the time using the short time format specified in the system locale settings.

tt

Interprets the string as a representation of the time using the long time format specified in the system locale settings.

am/pm

A string matching am indicates a time before noon. A string matching pm indicates a time after noon. Matches are not case-sensitive.

a/p

As for am/pm. Also accepts a for am and p for pm.

ampm

A string matching the 'am' indicator specified in the system locale settings indicates a time before noon.

A string matching the 'pm' indicator specified in the system locale settings indicates a time after noon.

'xx'/"xx"

Characters enclosed in single or double quotes are output as they are, and do not affect formatting.
