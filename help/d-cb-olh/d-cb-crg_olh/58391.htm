Description

Progresses the specified query to a work group.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

QueryId&

In: ID of the query to be progressed.

LastModified~

In: Last modified date-time of the query to be progressed.

Out: Last modified date-time of the progressed query.

Workforce$

In: Name of the work group the query is to be progressed to.

WorkforceId&

In: ID of the work group the query is to be progressed to.

 

Returns

1 (TRUE) if successful. If the status of the specified customer query is closed, an error is raised.

Remarks

The following processing is used by the function:

  • If the status of the specified customer query is Closed, an error is raised.
  • If the PROGRESS_SEQNR of the CUSTOMER_QUERY record is not null, the latest CUSTOMER_QUERY_PROGRESS record is updated with the following values:
    • LAST_MODIFIED is set to the current date-time.
    • OPEN_QUERY_IND_CODE is set to Null.
    • ACTION_DATE is set to the value of LAST_MODIFIED.
    • If RESOLUTION_TEXT is Null, RESOLUTION_TEXT is set to the value of ACTION_TEXT.
  • The CUSTOMER_QUERY record is updated with the following values:
    • LAST_MODIFIED is set to the current date-time.
    • If PROGRESS_SEQNR is Null, PROGRESS_SEQNR is set to 1; otherwise, PROGRESS_SEQNR is incremented.
    • If NEXT_CALLBACK_DATE is earlier than the current date-time, NEXT_CALLBACK_DATE is set to the current date-time plus the DEFAULT_RESPONSE_TIME for this query type (a Null value for DEFAULT_RESPONSE_TIME is treated as zero seconds); otherwise, DEFAULT_RESPONSE_TIME for this query type is added to NEXT_CALLBACK_DATE.
    • QUERY_WORK_FORCE_ID is set based on a lookup on Workforce$ or WordforceId&.
  • A CUSTOMER_QUERY_PROGRESS record is created. The values of the fields in the CUSTOMER_QUERY_PROGRESS record are set based on the corresponding fields in the CUSTOMER_QUERY record:

The following table lists criteria for setting CUSTOMER_QUERY_PROGRESS fields.
