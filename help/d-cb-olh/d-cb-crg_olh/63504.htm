biGLAccountingPeriodAllocate?[]()

Syntax

biGLAccountingPeriodAllocate?[](
    const EffectiveDate~, 
    const Transactions?[], 
    const AccountingPeriods?[], 
    const AccountingPeriodCloseFlg&)

 

Description

Called by the gl_sales_journal_post server utility to allocate transaction amounts to accounting periods.

Attributes

Application Environment

GL Guidance

Context

Any

Method

Remote

 

Parameter

Description

EffectiveDate~

In: Effective date-time of a Sales Journal posting task.

Transactions?[]

In: Details of the financial transactions to be allocated to accounting periods.

Each element in the array is a hash representing a transaction.

The table below lists the hash keys.

AccountingPeriods?[]

In: Array of details of accounting periods, ordered by START_DATE.

The transaction amounts are allocated to accounting periods that exist in the array. Each element in the array is a hash representing an accounting period.

The table below lists the hash keys.

CloseFlg&

In: Indicates whether the task is part of an operation to close an accounting period.

It is True if the Sales Journal posting task is part of the operation to close an accounting period, and False otherwise.

 

Returns

An array of hashes, with each hash representing the allocation of a transaction amount to a number of accounting periods. The keys of the hash are text representations of ACCOUNTING_PERIOD_ID values.

For example, a transaction amount of 97.14 is to be allocated to the accounting periods with IDs 1001, 1002, and 1003; the amounts to be allocated to these accounting periods are (respectively) 23.89, 56.23, and 17.02. This allocation is represented by the following hash:

{ '1001' => 23.89, '1002' => 56.23, '1003' => 17.02 }

The nth element in the returned array represents the allocations for the nth transaction in Transactions?[].

The number of hashes in the returned array must match the number of elements in the Transactions?[] parameter. A hash can be empty, indicating that none of the transaction's amount is to be allocated to any accounting period.

Remarks

This function can be used to return details of the previous allocations for the transactions, in order to calculate a delta posting per accounting period. The following must be specified:

  • POSTED_AMOUNT parameter.
  • Previous value of AMOUNT from the normalised event file GL guidance record in the AMOUNT parameter.
  • Second Transaction?[] element with:
    • POSTED_AMOUNT unspecified
    • AMOUNT equal to the new AMOUNT value from the normalised event file GL guidance record.

This function can be customised, usually for modifying the allocation logic.

A default implementation is provided for this function. It iterates over the Transactions?[] array, passing each element to the zGLAccountingPeriodAllocateDefault#{}() function, and returning the aggregate of the results from each call.

The following table lists the Transactions?[] hash keys.

 

Transactions?[] Hash Keys

Key

Description

AMOUNT

Amount to be allocated over accounting periods.

When the function is being used to return details of the previous allocations for the transaction, it is the previous value of AMOUNT from the normalised event file GL guidance record or invoice GL guidance record.

CURRENCY_ID

ID of the currency of the amount.

It can be used for rounding allocation amounts to an appropriate number of decimal places.

START_DATE

Start date-time of period of allocation.

END_DATE

End date-time of period of allocation.

EARLIEST_END_DATE

No amount can be apportioned to an accounting period that ends before this date-time.

It is usually identical to START_DATE.

POSTED_AMOUNT

Amount of the transaction that has been posted to date.

It exists only if the transaction has been partially posted during an earlier Sales Journal posting task.

For a normalised event file with a recognition period, it is the POSTED_AMOUNT from an normalised event file GL guidance record.

For an invoice with a deferral amount, it is the BE_POSTED_AMOUNT from an invoice GL guidance record.

FLAGS

Bitmask controlling how this amount is to be allocated to accounting periods. Valid values are:

  • 0

    The function raises an error if a required accounting period is absent from the AccountingPeriods?[] parameter.

  • 1

    The function does not raise an error if a required accounting period is absent.

    Note:

    It is recommended that customisations keep this functionality.

 

The following table lists the AccountingPeriods?[] hash keys.

 

AccountingPeriods?[] Hash Keys

Key

Description

ACCOUNTING_PERIOD_ID

ID of accounting period.

START_DATE

Start date-time of accounting period.

END_DATE

End date-time of accounting period.
