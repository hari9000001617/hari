EntitlementAllocGen&()

Syntax

EntitlementAllocGen&(
    const EntitlementId&,
    const AllocStartDate~)

 

Description

Inserts an allocation to an entitlement.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

EntitlementId&

In: ID of the entitlement to which an allocation is applied.

AllocStartDate~

In: Start date-time of the allocation.

 

Returns

ID of the newly-created entitlement allocation; otherwise, an error is raised.

Remarks

If an allocation generate function is not specified,the biEntitlementInsert&() function calls this function by default to generate an allocation for a new entitlement.

An error is raised if:

  • AllocStartDate~ does not match the value of the NEXT_ALLOCATION_DATE of the entitlement.
  • NEXT_ALLOCATION_DATE of the entitlement is NULL.

If the entitlement is recurring and INITIAL_ALLOCATION_AMOUNT is not defined or the allocation is not the initial allocation for the entitlement, the allocation amount is determined by RECURRING_PERIOD_AMOUNT; otherwise, the allocation amount is determined by INITIAL_ALLOCATION_AMOUNT.

The allocation amount is prorated if all of the following conditions are met:

  • PRORATE_IND_CODE of the entitlement is set to 1.
  • VALIDITY_PERIOD_UNIT_CODE or RECURRING_PERIOD_UNIT_CODE of the entitlement is equal to Bill Cycle.
  • START_DATE of the allocation does not match the start date time of the bill cycle.

If prorating is required, PRORATE_FUNCTION_DEFN_ID of the entitlement specifies the function called to return the prorated portion of the allocation amount. The default function is EntitlementAllocProrate#().

The expiry date time of the allocation is specified by VALIDITY_PERIOD of the entitlement. If VALIDITY_PERIOD is not specified, the expiry date time of an allocation for a:

  • Recurring entitlement is specified by RECURRING_PERIOD.
  • One-off entitlement is the expiry date time of the entitlement.
