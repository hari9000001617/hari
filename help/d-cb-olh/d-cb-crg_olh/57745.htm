Encoding Rules

The following table lists encoding rules for selected encodings.

 

Encoding Rules

Character Encoding

Description

UTF-8

Encodes as UTF-8 without a byte-order marker (BOM) at the beginning of the file.

For some applications, the encoding type may need to be explicitly defined during import.

UTF-8-SIG

Encodes as UTF-8 with a BOM at the beginning of the file.

A BOM is not required for UTF-8, but some Microsoft Windows applications require it for decoding the file without intervention. Some applications may not be compatible with the BOM.

UTF-16LE

Encodes as UTF-16 little-endian with a BOM.

Little-endian is used by Microsoft Windows.

UTF-16BE

Encodes as UTF-16 big-endian with a BOM.

Note:

Big-endian encoded files are not supported by all applications.

US-ASCII

Encodes files using the ASCII character range.

If Character Encoding is set to XML, invalid characters are written using XML numeric character references; that is, XML characters are encoded but are represented differently.

If File Format is set to CSV, TSV, or Custom, invalid characters are converted to a ? character.

All other character encoding types

Encodes files to the selected character encoding.

Invalid characters are converted to a ? character.
