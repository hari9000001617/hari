Description

Retrieves function details.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

FunctionName$

In: Name of the function to fetch details from.

EffectiveDate~

In: Effective date-time to perform the lookup.

AliasLookup&

In: Determines whether function aliasing is taken into account when returning the details of the function.

A non-zero value (TRUE) indicates that aliasing is considered.

InterfaceNr&[]

Out: EPM interface numbers associated with each returned function, used to identify overloaded functions.

Interface numbers start at 1.

FunctionDefn$
FunctionDefn$[]

Out: Returned EPM function header declaration(s) for the function, which is interpreted by the C TRE API to determine the function's return type and parameter details.

StartDate~
StartDate~[]

Out: Start date-time for valid function declaration(s).

The returned StartDate~ must be less than or equal to EffectiveDate~.

EndDate~
EndDate~[]

Out: End date-time for the valid function declaration(s).

The returned EndDate~ must be greater than or equal to EffectiveDate~.

AliasFunctionName$
AliasFunctionName$[]

Out: Name of the function to call on the server when the current client calls FunctionName$.

AliasFunctionName$ only contains a value if AliasLookup& is a non-zero value (TRUE).

If the server does not support function aliasing, AliasFunctionName$ is set to FunctionName$ if an AliasLookup& is requested.

ServiceName$
ServiceName$[]

Out: Name of the Tuxedo service that implements FunctionName$ (or AliasFunctionName$ if an AliasLookup& is performed).

The TRE API directs all calls to FunctionName$ to the Tuxedo service specified in ServiceName$.

Flags&
Flags&[]

Out: TRE flags associated with the specified function.

Flags& is a bit-wise combination of the following constants:

  • TRE_ASYNC (call function asynchronously)
  • TRE_QUEUE (place request to call function on Tuxedo transactional queue)
  • TRE_NOTRANS (call function as part of a separate transaction)
  • TRE_NOTIME (disable blocking timeouts while calling the function)
  • TRE_ALIASCALLS (direct remote functions through aliased lookups)
  • TRE_DETERMINISTIC (result of the function is deterministic).

QueueSpaceName$
QueueSpaceName%[]

Out: Name of the Tuxedo Queue Space to be used when enqueuing requests to the function.

It only contains values when TRE_QUEUE is set in Flags&.

ReturnType&
ReturnType&[]

Out: Return type of the function. It is called by the Java TRE API.

ParameterNames$[]
ParameterNames?[]

Out: Name of each parameter passed to the function.

If InterfaceNr& is being used, it is a two-dimensional array of strings.

ParameterTypes&[]
ParameterTypes?[]

Out: TRE data type of each parameter passed to the function.

ParameterTypes?[] is a two-dimensional array of integers.

 

Returns

Number of returned interfaces for the specified function; 0 (FALSE) if the specified function does not exist; -1 if an AliasLookup& was requested and the user does not have access to any of the interfaces for the function.

Remarks

Using this function with FunctionDefn$ is ineffective, and returns only the declaration for the first function interface.

When a lookup is performed on a function, all defined and accessible interfaces are returned in order of InterfaceNr&[]. If an AliasLookup& is performed, only those interfaces accessible to the user are returned.
