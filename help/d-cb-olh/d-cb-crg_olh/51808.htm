Description

Creates a default external definition from a specified source. The source can be specified as follows:

  • Entity validation (Interface 1)
  • Source external definition (Interface 2)
  • Process Engine class (Interface 3).

ExternalDefnNamespaceId&

In: ID of the external definition namespace that the new external definition is generated from.

ExternalDefnName$

In: Name of the generated external definition.

ViewName$

In: Name of the generated external definition view.

EntityValidationId&

In: ID of the entity validation that the external definition is generated from.

In: ID of the source that the external definition is generated from.

ParentExternalDefnId&

In: ID of the parent external definition that the external definition is generated from.

Note:

InterfacePrefix$

In: Optional. Prefix to append to the INTERFACE_LABELS of the generated elements in the new definition.

ClassId&

In: ID of the Process Engine class that the external definition is generated from.

ObjectTypeId&

In: Optional. ID of the object type that the external definition is created from.

Note:

The generated external definition includes facts:

  • In the class associated with the object type
  • Not associated with any object type.

 

Returns

ID of the external definition that has been generated; otherwise, an error containing the error details.

Remarks

Interface 1

The generated external definition is a sequence content model with a status of Incomplete. The elements are generated using ExternalDefnElementGenerate&().

Note:

Depending on the entity validation used, the generated external definition may need corrections made to its elements before the status can be successfully updated to Validated.

Interface 2

The generated external definition has the same structure as the specified source external definition. However, the generated external definition is associated with the specified parent definition, and has the same namespace as the parent.

If the source external definition is:

  • Abstract (that is, no Convergent Billing entity or Process Engine class is associated with it) the generated external definition has the Convergent Billing entity or Process Engine class of the parent definition associated with it.
  • Not abstract, the generated definition has the same Convergent Billing entity or Process Engine class as the source definition.

This interface can be used to construct instances of an abstract class. For example, if an abstract address external definition is defined, the interface can be used to generate separate home, postal, and work addresses for a person external definition from this abstract address definition.

Note:

Interfaces or views cannot be created from abstract external definitions. External definitions must have an entity type to be used to create interfaces or views.

Interface 3

The generated external definition is a sequence content model with a status of Incomplete. One external definition element is created for each row in the ED_FACT_CONFIG_V view with the specified ClassId& and optional ObjectTypeId&.
