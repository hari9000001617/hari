EQUIPMENT_HISTORY_SEARCH_V

The EQUIPMENT_HISTORY_SEARCH_V view is used by the Equipment Search form to perform searches on equipment.

Column Definitions

Column Name

Properties

Description

EQUIPMENT_ID

NUMBER (15)

Not null

ID of an item of equipment.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective. The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

EQUIPMENT_TYPE_ID

NUMBER (8)

Not null

ID of the type of equipment.

EQUIPMENT_STATUS_CODE

NUMBER (2)

Not null

Status of the equipment item (for example, In Store, At Retail Outlet, Lost, Stolen, In Test, and In Use).

The available items are dependent on the equipment type.

REASON_CODE

NUMBER (8)

Optional code for defining possible reasons for the current status of the equipment.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified this entry.

ATLANTA_GROUP_ID

NUMBER (8)

Not null

Security group with access to this entry.

EQUIPMENT_SUBTYPE

VARCHAR (40)

ID for referencing the equipment subtype category to which this service belongs (for example, alpha-numeric and numeric display within equipment type pager).

EQUIPMENT_ORIGIN_CODE

NUMBER (4)

ID for indicating where the equipment originally came from, or who supplied it (for example, from store or customer).

MAKE

VARCHAR (40)

Maker of the equipment.

MODEL

VARCHAR (40)

Model of the equipment.

NR_ITEMS

NUMBER (8)

Not null

Number of items of equipment associated with this record (usually 1).

SERIAL_NUMBER

VARCHAR (64)

Not null

Serial number or ID for the item of equipment.

If NR_ITEMS is greater than 1, this defines the start of a contiguous range of serial numbers of this equipment.

DECODED_SERIAL_NUMBER

VARCHAR (64)

If the equipment type has a decode expression defined, stores the numeric value of the decoded serial number.

The DVP checks that this column is not null if the equipment type has a SERIAL_NUMBER_DECODE_EXPR, but it cannot verify if the value is correct.

TO_SERIAL_NUMBER

VARCHAR (64)

Inclusive end of a contiguous range of serial numbers for this equipment.

The equipment type's SERIAL_NUMBER_FORMAT_CODE defines the format of this serial number and the meaning of a range.

DECODED_TO_SERIAL_
NUMBER

VARCHAR (64)

If the equipment type has a decode expression defined, it stores the numeric value of the decoded serial number.

The DVP checks that it is not null if TO_SERIAL_NUMBER is not null and the equipment type has a SERIAL_NUMBER_DECODE_EXPR, but it cannot verify if the value is correct.

DECODED_TO_NETWORK_
ADDRESS

VARCHAR (64)

If the equipment type has a TO_NETWORK_ADDRESS specified, it stores the numeric value of the decoded to network address.

It is checked by the DVP. The DVP cannot verify that the value is correct.

RESERVED_CUSTOMER_NODE_
ID

NUMBER (15)

If this equipment was reserved for a customer, the customer node associated with the reservation.

Note:

The reservation of equipment for customers is independent of the equipment's status.

The customer must exist (not necessarily as a single record) for the entirety of this record's date range, and is checked by the DVP.

Equipment reserved for a customer node can only be allocated to that customer node or any customer node in the same hierarchy.

Equipment reserved for a customer node cannot be allocated to a customer node not in the same hierarchy for any period of the reservation. It is checked by the DVP.

NETWORK_ADDRESS

VARCHAR (64)

Network address programmed into the item of equipment where this value does not change dynamically or cannot be reassigned while it is in use by a customer (for example, Mobile Data Network LLI that can only be assigned by the network provider).

If a range of serial numbers is specified (NR_ITEMS > 1), it is usually null, unless network addresses are associated with the equipment before use.

DECODED_NETWORK_ADDRESS

VARCHAR (64)

If the equipment type has a network decode expression defined and NETWORK_ADDRESS is not null, it stores the numeric value of the decoded serial number.

It is checked by the DVP. The DVP cannot verify that the value is correct.

TO_NETWORK_ADDRESS

VARCHAR (64)

Inclusive end of a contiguous range of network addresses for this equipment.

It is only specified if NR_ITEMS > 1 and network addresses are associated with the equipment before use.

The equipment type's NETWORK_ADDRESS_FORMAT_CODE defines the format of this network address, and the meaning of a range.

ADDRESS_ID

NUMBER (15)

Reference to the address details for the current location of the equipment item (for example, store address, retailer address, and customer service address).

The address must exist as a single record for the entirety of this record's date range.

SENDER_PERSON_ID

NUMBER (15)

ID of the contact who sent this item of equipment.

It cannot be directly set, but is set to the person associated with the user who updated the equipment to an In-Transit status by the biEquipmentUpdate&() function.

Users who do not have a corresponding contact record cannot set equipment to an In-Transit status.

It must contain a value if the equipment has a status of In-Transit, enforced by the C_EQUIPMENT_HISTORY check constraint.

The person must exist for at least the date range of this record. It is checked by the DVP.

RECIPIENT_PERSON_ID

NUMBER (15)

ID of the contact or group who is to receive this item of equipment.

It can only be set when equipment is being updated to, or in an In-Transit status. The group of the equipment is updated to the group of the assigned receiver.

It must contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

The person must exist for at least the date range of this record. It is checked by the DVP.

TRANSIT_ORGANISATION

VARCHAR (80)

Name of the transit organisation for sending the equipment to the recipient.

It can only contain a value if the equipment has a status of In-Transit, and is enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_CONTACT_NUMBER

VARCHAR (30)

Contact phone number for the transit organisation.

TRANSIT_CONTACT_NUMBER can only contain a value if the equipment has a status of In-Transit, and is enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_REFERENCE_
NUMBER

VARCHAR (30)

Reference or receipt number given by the transit organisation for the transportation of this equipment.

TRANSIT_REFERENCE_NUMBER can only contain a value if the equipment has a status of In-Transit, and is enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_DATE

DATE

Date-time when the equipment's status was last updated to a status of In-Transit.

It can only contain a value if the equipment has a status of In-Transit, and is enforced by C_EQUIPMENT_HISTORY check constraint.

GENERAL_1 - GENERAL_3

VARCHAR (255)

General fields 1 - 3 associated with this equipment item.

They are included so they can be used as search criteria.

SEARCH_IND_CODE

NUMBER

Code of the mode of the search form.

A value of 0 indicates a search operation; a value of 1 indicates a new operation.
