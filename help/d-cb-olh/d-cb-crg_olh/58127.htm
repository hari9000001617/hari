biBimFPMFilesReadyList&()

Syntax

biBimFPMFilesReadyList&(
    const FilePattern$,
    const FileProcessingOrder&, 
    const FileAvailabilityMethod&, 
    const FileAvailabilityDate~, 
    const ControlFilePattern$, 
    var FilesReadyList$[])

 

Description

Retrieves a list of files ready for processing by the batch loader.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

FilePattern$

In: UNIX shell file name pattern, including the path, of the files to be retrieved.

FileProcessingOrder&

In: Order in which the files are processed. Valid values are:

  • 1 (Last Modified): Files are processed in ascending date order (oldest to most current)
  • 2 (Name): Files are processed in ascending alphabetical order.

FileAvailabilityMethod&

In: Method used to select the files as ready for processing. Valid values are:

  • 1 (Last Modified): Files that match FilePattern$ and have a last modified date earlier than FileAvailabilityDate~ are selected for processing.
  • 2 (Control File): Files that match FilePattern$ and have an existing control file that matches ControlFilePattern$ are selected for processing.
  • 3 (Filename): Files that match FilePattern$ are selected for processing.

FileAvailabilityDate~

In: Date-time used to compare with the file's last modified date.

Files that have a last modified date earlier than FileAvailabilityDate~ are selected for processing.

Note:

It is only applicable if FileAvailabilityMethod& is set to 1 (Last Modified).

ControlFilePattern$

In: Control file name pattern, including the path, used to define the control file name from the specified file name.

The control file name pattern can contain the following tags:

  • <basename>: file name of the processed file excluding the extension.
  • <filename>: file name of the processed file including the extension.

    Note:

    It is only applicable if FileAvailabilityMethod& is set to 2 (Control File).

FilesReadyList$[]

Out: List of files ready for processing.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.
