Description

Validates the EPM function definition stored in the database for FunctionName$ at the specified date-time.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

FunctionName$

In: Name of function to be validated.

EffectiveDate~

In: Date-time used to validate the function definition.

InterfaceNr&

In: Function interface number.

ReturnType&

Out: Function return type (TRE_DATATYPE).

ParamNames$[]

Out: Names of function parameters.

ParamTypes&[]

Out: Types of function parameters (TRE_DATATYPE).

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

Any functions referenced in the expression are checked to ensure that they are being called correctly.

This function does not perform any checks relating to application environment or context; FunctionValidateFull&() should be used if this is required.

Interface 1 checks all EPM function definitions stored in the database with the name FunctionName$.

Interface 2 only checks the function definitions of functions with the name FunctionName$ and the specified interface number.

Interface 3 does the same check as the second and also sets the ReturnType&, ParamNames$[], and ParamTypes&[] output parameters (refer to Parameters above).
