Description

Inserts an invoice format row for a specified customer.

Context

The following groups of variables are available during attribute type normalisation and data resolution:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value INVFMTSVR.
    • evLastRowInd& contains the value 0 for all stages of processing except for the evaluation of customer invoice format pre-commit expressions for the last row processed.
    • evRowStatus$ contains the value Insert for all stages of processing except for the customer post commit processing, where it contains the value Update.
  • Customer
  • Customer invoice format.

    The customer invoice format variables may only be available during the customer invoice format processing stages.

Parameters

Parameter

Description

StartDate~

Date on which the insertion first becomes effective.

If not specified, the current date is used.

LastModified~

Initial value must match the value in the customer record identified by FieldValues?{}.

It can be undefined. If not defined, it is automatically derived.

On successful return, the value is the date-time of this insertion.

FieldValues?{}

Customer invoice format data to be inserted.

The sequence number of the invoice format entry to be inserted must be included.

It is expected to specify values for the primary IDs of the customer invoice format.

If the customer cannot be identified by FieldValues?{}, an error is raised. If an effective end date is not specified, the end of time is used.

The data can be keyed according to the corresponding column name in the CUSTOMER_NODE_INV_FORMAT_TRE_V view. When keyed according to database column name, the data passed must match in both type and value the data stored in the database.

If attribute validation is enabled, data can also be keyed by the interface or display label specified in the appropriate entity validation, or in Convergent Billing defaults.

When keyed by interface or display label, the data passed must match the type and value expected as defined in the appropriate attribute type. Denormalised fields must pass the denormalised string values. Other fields must pass data of the type defined in the attribute type.

IdentifierKeys?{}

Populated on successful return with the values of the CUSTOMER_NODE_INV_FORMAT identifier fields.

 

Processing

StartDate~ identifies the historical customer record to be updated. The customer node ID, sequence number, and effective dates are determined form FieldValues{}?.

Processing continues as if fEV_CustomerInvInsertRows&() was called for one row.

Returns

A non-zero value on success; otherwise, an error is raised.
