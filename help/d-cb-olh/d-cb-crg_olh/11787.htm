Description

Maintains derived attribute tables that have a storage context of Global or Global-Shared.

Note:

  • Interfaces 1 and 2 are remote wrapper functions for the DerivedAttributeArrayUpdate&() function.
  • Interface 6 is a remote wrapper function for interface 2 of the DerivedAttributeArrayUpdate&() function, but does not require the derived attribute table's last modified date to be passed because the function determines it before passing the other parameters to interface 2 of DerivedAttributeArrayUpdate&().

Attributes

Function Type

Remote Synchronous

Service

biDerivedAttr

Application Environment

Any

Context

Any

 

Parameter

Description

DerivedAttributeId&

In: ID of the derived attribute to be updated.

OldEffectiveDate~

In: Date-time used to retrieve the derived attribute table.

LastModified~

In: Last modified date-time of the derived attribute table effective as at OldEffectiveDate~.

Out: Last modified date-time of the updated derived attribute table as at EffectiveStartDate~.

EffectiveStartDate~

In: Start of the date-time range for which changes are to be applied.

A value must be specified.

EffectiveEndDate~

In: End of the date-time range for which changes are to be applied.

A value greater than EffectiveStartDate~ must be specified.

Opcode&[]

Operations to perform on the row.

Seqnr&[]

In: If a derived attribute is using standard features, sequence number of the row or rows being inserted, updated, or deleted.

The following applies when specifying sequence numbers for derived attributes using standard features:

  • Sequence numbers are processed (and therefore interpreted) in the order in which they are specified. Therefore, deleted rows must be specified in descending sequence order (for example, 6,3,2), and inserted rows must be specified in ascending sequence order (for example, 2,3,6).
  • The starting number for the first row is 1 (not 0). For example, to delete the first three rows from the table, specify 3,2,1 (not 2,1,0).
  • The contiguity of sequence numbers is maintained by this function.

For derived attributes using extended features, it contains extended derived attribute row IDs instead of normal sequence numbers. When inserting rows, it must be specified with a placeholder value (for example, null&) because the row IDs are allocated automatically, and cannot be specified.

Rows for derived attribute tables using extended features are processed in the following order:

  1. Insert
  2. Update
  3. Delete.

FieldNames$[]

In: Names of fields for the rows being inserted or updated.

Possible field names are defined in the DERIVED_ATTRIBUTE_ARRAY_TRE_V view.

Note:

Only some fields from the view can be specified.

FieldValues?[]

In: Two-dimensional array of field values (in [Row][Field] order) inserted, updated, or deleted.

An empty row can be specified for records being deleted.

DoNotPurgeIndCode&

If set to 0 (FALSE), on successful function completion, and the derived attribute table has a storage context of:

  • Global, the table is purged from the global process cache.
  • Global-Shared, the same row operations are applied to the global-shared derived attribute table cache.

Otherwise, the derived attribute table is not purged.

 

Returns

Returns 1 (TRUE) if successful.

For all function interfaces, if the function is not successful or if either EffectiveStartDate~ or EffectiveEndDate~ are outside the derived attribute record's date-time range, an error is raised.

Remarks

Interface 3

This interface is used to add, update, or delete rows.

The following applies to this interface:

  • The operation (insert, update, or delete) is specified explicitly using Opcode&[].
  • No entity validation is performed on derived attribute table rows.

    The table's field names are explicitly defined using FieldNames$[], rather than implicitly using the entity validation definitions.

    Values for fields from the DERIVED_ATTRIBUTE_ARRAY_TRE_V view, other than the following fields, are derived by the function and ignored:

    • Index value and result value fields (for example, INDEX1_VALUE and RESULT1_VALUE).
    • DESCRIPTION
    • ATLANTA_GROUP_ID, VERSION_STR, and CATEGORIES, which are available only for derived attributes using extended features.
  • For insert and update operations, all columns configured for the derived attribute should be specified.

    Field values that are not specified are set to NULL.

  • If DoNotPurgeIndCode& is FALSE, biDerivedTablePurge&() is called to purge the derived attribute table from all server processes or the Global-Shared derived attribute table cache.
  • Multi-range updates are not supported.
  • On successful completion, if the updated derived attribute is of storage context:
    • Global, it is purged from the process caches.
    • Global-Shared, it is incrementally updated from the shared cache.
  • Other functionality and restrictions that apply to interface 1 also apply to this interface.

Interface 4

This interface provides the same functionality as interface 3. However, it uses pessimistic locking and therefore does not use a LastModified~ parameter.

Interface 5

This interface provides the same functionality as interface 3. However, on successful completion, the derived attribute table is purged.
