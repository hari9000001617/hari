OPERATOR_V

The OPERATOR_V view returns the list of details associated with a user, including details associated with their profile. This view is queried by the client immediately after a user logs in. The details are retained for the duration of the session, and can be used in any expressions evaluated on the client.

Column Definitions

Column Name

Properties

Description

OPERATOR_ID

NUMBER (8)

Not null

ID of the user.

OPERATOR_FULL_NAME

VARCHAR (80)

Not null

Full name of the user.

OPERATOR_LOGIN_NAME

VARCHAR (30)

Not null

Oracle user name of the user.

It should correspond to their UNIX user name.

PROFILE_ID

NUMBER (8)

Not null

Profile to be used by this user.

EMAIL_ADDRESS

VARCHAR (255)

Not null

Email address for the user.

WORK_PHONE_NR

VARCHAR (30)

Work phone number for the user.

HOME_PHONE_NR

VARCHAR (30)

Home phone number for the user.

FAX_NR

VARCHAR (30)

Facsimile number for this user.

PERSON_ID

NUMBER (15)

If the user is a customer or member of a customer, it references the full details for the user.

It can also be used to specify additional details about the user.

SUPERUSER_IND_CODE

NUMBER (1)

Not null

If set to 1 (TRUE), this user has the capability of performing TRE operations using the treOperatorSetId&() TRE API call.

It can be used in conjunction with treOperatorSetId&() for interfaces to connections from trusted third party applications operating in another Tuxedo domain (server to server connection), or from applications connecting as a Tuxedo client to multiplex multiple client requests over a single connection and maintaining user information for auditing purposes.

OPERATOR_GENERAL_1 - OPERATOR_GENERAL_10

VARCHAR (255)

General fields 1 - 10 for entering user details.

The contents and validation of the fields is controlled by an ENTITY_VALIDATION associated with the SYSTEM_CLIENT configuration item.

PROFILE_NAME

VARCHAR (60)

Not null

Full name of the user security profile.

PRIMARY_GROUP_ID

NUMBER (8)

Not null

Security group associated with new data created by any users using this profile.

MAX_ADJUSTMENT_AMOUNT

NUMBER (19,3)

Not null

Maximum value of an adjustment that can be performed by users using this profile.

ADJUSTMENT_CURRENCY_ID

NUMBER (8)

Not null

Currency of the maximum adjustment amount.

PROFILE_GENERAL_1 - PROFILE_GENERAL_10

VARCHAR (255)

General fields 1 - 10 for entering profile details.

The contents and validation of these fields is controlled by an ENTITY_VALIDATION associated with the SYSTEM_CLIENT configuration item.

TENANT_ID

NUMBER (8)

ID of the tenant associated with the user.
