biDateRangedSQLRangePrepare&()

Syntax

biDateRangedSQLRangePrepare&(
    const EntInfo?{},
    Id&,
    const DesiredDates~[])

 

Description

Prepares a date ranged instance of an entity for further updates by range updating the instance so that its set of dates is a subset of the final set of dates, with the first start date and the last end date also matching.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

EntInfo?{}

In: Entity information for the entity to be operated on.

Id&

In: ID of entity record to be operated on.

DesiredDates~[]

In: Required date ranges.

 

Returns

ID of the new entity; otherwise, an error is raised.

Remarks

The function is used by the bi<entity name>RangeUpdate&() functions.

The EntInfo?{} hash contains information used by the function to perform updates. The hash always contains the TableName, IsDateRanged, NameColName, and IdColName keys set to the correct values for the current entity. The hash can also include the key 'Children' that contains details for dependant entities of this entity.

All records for the entities history table are retrieved. Each record in the retrieved set that falls completely outside the range of required dates, or partially falls after the end of the range is marked for deletion. An array of records to be deleted and an array of the remaining records is constructed.

The set of remaining records is traversed in preparation for a range update that will move the end date of any record that does not correspond with an appropriate end date to the end date of the next original record that does correspond with an appropriate end date, or at worst, the last required end date. An array of records to be range updated (including updated values) is constructed.

The records specified in the array of records to be deleted are removed and the changes specified in the array of records to be range updated are applied.
