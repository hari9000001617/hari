PRODUCT_TRE_V

The PRODUCT_TRE_V view provides details of product definitions.

A product consists of a combination of service types, facility groups, tariffs, and validation rules associated with them.

Column Definitions

Column Name

Properties

Description

PRODUCT_ID

NUMBER (8)

Not null

ID of the product.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

PRODUCT_NAME

VARCHAR (60)

Not null

Name of the product.

PRODUCT_DISPLAY_NAME

VARCHAR (120)

Not null

Name of the product displayed.

It defaults to PRODUCT_NAME.

It can be translated.

DESCRIPTION

VARCHAR (4000)

Not null

Description of the product.

LAST_MODIFIED

DATE

Not null

Date-time at which the record was last modified.

OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified the record.

GROUP_ID

NUMBER (8)

Not null

Security group associated with the creation and modification of the record.

The security group ID controls who can view or modify the record.

VERSION_STR

VARCHAR (255)

Version ID used to tag this entity.

It is provided for use by third party configuration tools, and is not used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this entity.

Refer to the CATEGORY_V view for the list of available categories.

ICON_ID

NUMBER (8)

Not null

Icon associated with this product.

COMPANION_IND_CODE

NUMBER (1)

Not null

If set to:

  • 1 (TRUE), this product is a companion product.
  • 0 (FALSE), this is a base product.

Only base products appear on the customer hierarchy form, and only base products can be associated with a customer node to create a new product instance.

INITIAL_STATUS_CODE

NUMBER (4)

Not null

Initial status of an instance of this product.

It is based on the PRODUCT_INSTANCE_STATUS reference type.

DEFAULT_WORK_FORCE_ID

NUMBER (8)

Default work group associated with this product.

CONTRACT_TYPE_ID

NUMBER (8)

Type of contract associated with this product.

PAYMENT_ACCOUNT_TYPE_ID

NUMBER (8)

Account type associated with the initial payment required by this product.

PRODUCT_ENTITY_
VALIDATION_ID

NUMBER (8)

Validation rules, label, and hint text overrides for product instance details displayed for a product of this type.

For a base product, the specified entity validation must have VALIDATION_TABLE_NAME set to PRODUCT_INSTANCE_HISTORY.

For a companion product, if specified, the specified entity validation must have VALIDATION_TABLE_NAME set to the COMP_PROD_INSTANCE_HISTORY_V view.

PROD_SVC_ENTITY_
VALIDATION_ID

NUMBER (8)

Validation rules for the details of each service associated with this product instance.

It can only be set for base products, and the specified entity validation must have VALIDATION_TABLE_NAME set to PRODUCT_INSTANCE_HISTORY.

PROD_EQP_ENTITY_
VALIDATION_ID

NUMBER (8)

Validation rules for the details of each service associated with this product instance.

It can only be set for base products, and the specified entity validation must have VALIDATION_TABLE_NAME set to PRODUCT_INSTANCE_HISTORY.

FAC_GRP_ENTITY_
VALIDATION_ID

NUMBER (8)

Validation rules for details of each facility group associated with this product instance.

It can only be set for base products, and the specified entity validation must have VALIDATION_TABLE_NAME set to PRODUCT_INSTANCE_HISTORY.

MULTIPLE_IND_CODE

NUMBER (1)

If set to 1 (True), no more than one instance of this companion product can be active on a base product instance.

If null, only one instance of this companion product can be active on a base product instance.

It can only be set for companion products.

PRODUCT_OFFER_IND_CODE

NUMBER (1)

If set to 1 (True), an instance of this base product must have a product offer assigned to it before product completion.

Its value can only be NULL or 1.

It can only be set to 1 if COMPANION_IND_CODE is 0.

SERVICE_SPECIFIC_IND_
CODE

NUMBER (1)

If set to 1 (True), this product is a service specific companion product.

It can only be NULL or 1.

It can only be set to 1 if COMPANION_IND_CODE is 1.

CANCEL_STATUS_CODE

NUMBER (8)

Status of the product instance that is to be considered cancelled when product offers are being validated.

If NULL, the status used is 9 (Cancelled). Where external interfaces need notification of cancellation, it is recommended that the status be set to some other value such as 8 (Cancellation Required).

A product instance is not considered part of the current product offer hierarchy when product offers are being validated if its status is:

  • 9 (Cancelled).
  • The value of CANCEL_STATUS_CODE.
  • Listed in the PRODUCT_INSTANCE_CANCELLED reference type.

STS_UPDATE_PROD_OFFER_
IND_CODE

NUMBER (1)

If set to 1 (True), status updates to an instance of this product are applied throughout the product offer associated with the product instance.

The status codes of product instances or services are also updated.

It can only be set to 1 if COMPANION_IND_CODE is 1; otherwise, it must be null.
