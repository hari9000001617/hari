Variables Based on CUSTOMER_V

All direct variables based on the CUSTOMER_V view have an application environment of Billing and a context of Customer.

Variable Definitions

Variable

Type

Description

CustAccountName$

String (30)

Name of the customer's primary account.

CustActive~

Date

Date the customer became active.

CustBankCode&

Integer

Customer's bank code.

CustBranchCode&

Integer

Branch code for the customer's bank.

CustCardType&

Integer

Credit card type used by the customer.

CustCreated~

Date

Date the customer is created.

CustCreditLimit#

Real

Customer's credit limit.

CustCreditRating&

Integer

Customer's credit rating.

CustCurrency$

String (5)

Currency used by the customer.

CustGeneral1$ -
CustGeneral10$

String (255)

General fields 1 - 10.

CustIndustry&

Integer

Code for the industry the customer belongs to.

CustName$

String (80)

Customer node name.

CustNodeId&

Integer

ID of the customer node.

CustPaymentLocation&

Integer

Code for the preferred location where payments are made.

CustRegion&

Integer

Region in which the customer is located.

CustReportLevel$

String (40)

Reporting level text for the customer.

CustReportLevel&

Integer

Reporting level code for the customer. Valid values are:

  • 1 (Invoice)
  • 2 (Statement)
  • 3 (No reporting).

CustSalesChannel&

Integer

Sales channel for the customer.

CustScheduleId&

Integer

Schedule ID for the bill cycle to which the customer is assigned.

CustStatus$

String (40)

Status text for the customer.

CustStatus&

Integer

Status code for the customer. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-list).

CustTaxClass&

Integer

Taxation class for the customer.

CustTurnover#

Real

Customer's annual turnover.

CustTypeId&

Integer

ID of the customer type.

CustTypeName$

String (30)

Name of the customer type.

CustTypeDisplayName$

String (60)

Name of the customer type displayed.

It defaults to CustTypeName$.

Caution:

The value of this column can be translated into a locale-specific language and therefore differ between implementations.
