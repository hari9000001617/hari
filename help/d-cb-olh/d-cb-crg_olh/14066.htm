treClientNotify&()

Syntax

treClientNotify&(
    ClientId$,
    NotificationName$,
    FieldValues?[])

 

Description

Sends an asynchronous notification, with the name NotificationName$, to the TRE client identified by ClientId&.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

ClientId&

In: ID of the client to which notification is sent.

It is returned by treClientGetId$() or biStatusClient&().

NotificationName$

In: Type of notification to send.

It is received as the event name.

FieldValues?[]

In: Parameters to send to the client.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The details that are sent in the notification message are specified in the FieldValues?[] parameter.

Asynchronous notifications are detected by a client in the same manner as asynchronous events are detected (that is, by setting up an event handler and making calls to treEventCheck()). NotificationName$ is used as the event name in the event handler.

The function returns immediately. No indication is given of whether the client received the notification. Use the function treClientNotifyWait&() to get this information.

A TRE server process can obtain the ClientId& of the current client making the call by using the treClientGetId$() call. The biStatusClient&() TRE function can also be used to obtain the ClientId& of TRE clients that are connected.

Setting Focus in Process Engine Web Client

To use this function to set the focus in Process Engine web client, set NotficationName$ to FOCUS.

The FieldValues?[] parameter must contain the elements listed in the following table.

 

FieldValues?[] Elements for Setting Focus

Array Position

Parameter

Description

0

ClassKey&

Class key of the focus object, FactKey&, and ObjectKey& parameters.

ClassKey& is set to the class key of the fact specified by FactKey& if:

  • ClassKey& is not supplied.
  • FactKey& is valid.

1

FactKey&

Fact key of the focus object.

2

ObjectKey&

Object key of the focus object.

Note:

ObjectKey& is ignored when using treClientNotify&() for advanced searches because there are no objects visible on the page.

3

Section$

Section to focus; either DETAIL or TAB.

Only groups in the specified section are examined when finding the field to focus.

4

TabName$

Tab to focus.

Only groups in the specified tab are examined when finding the field to focus. For unopened tabs, the tab is opened before focusing on a field in the tab.

 

Caution:

If any FieldValues?[] elements are not required, they must be included in the array with the appropriate null$ or null& value.
