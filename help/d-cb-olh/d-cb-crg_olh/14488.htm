CustomerNodeDelete&()

Syntax

CustomerNodeDelete&(
    CustomerNodeId&,
    StartDate~,
    EndDate~)

 

Description

Deletes the specified customer node for the specified date range.

Note:

Because an entry in the CustomerPartitionException derived attribute table has no impact for a deleted entity, the entry is not deleted from the table.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

CustomerNodeId&

In: ID of the customer node being deleted.

StartDate~

In: Start of the date-time range for the customer node being deleted.

EndDate~

In: End of the date-time range for the customer node being deleted.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

After a successful delete:

  • biCustomerNodePurge&() function is called to clear data for the deleted customer node from the customer node cache.
  • biAccountPurgeLocal&() function is called to purge the deleted secondary accounts. The purge triggers the replication of the customer node and the account data to any separate rating instances.

The effective user must have delete security group access level for the group associated with the record manipulated by this function call. If the effective date of the delete is less than the current date (00:00:00 time stamp), direct delete security group access level is required.

GROUP_ACCESS_TRE_V view lists security group access level values.
