Description

Deletes a row of data from a service derived attribute array identified by the supplied keys.

Context

The following groups of variables are available throughout all stages of processing a service derived attribute delete row request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value SERVSVR.
    • evLastRowInd& contains the value 0 during all stages of processing, except the evaluation of the service pre-commit expressions and the service derived attribute pre-commit expressions for the last row processed.
    • evRowStatus$ contains the value Delete for the service derived attribute processing stages and the value Update for the service processing stages.
  • Service

    The service variables may only be available during the service processing stages and the evaluation of service derived attribute pre- and post commit expressions.

  • Derived attribute

    The derived attribute variable may only be available during the service derived attribute processing stages.

Parameters

Parameter

Description

Keys?{}

Keys used to identify the service derived attribute row being deleted.

The expected keys are service ID, derived attribute ID, and sequence number.

LastModified~

Initial value must match the value in the service record identified by Keys?{}.

If Keys?{} is specified, it can be undefined. If not defined, it is automatically derived from SERVICE_HISTORY_V.

On successful return, the value is the date-time of these deletions.

StartDate~

Date on which the deletion first becomes effective.

If the value is not defined, the current date is used.

EndDate~

Date on which the deletion ceases to be effective.

If the value is not defined, the end of time value is used.

IdentifierKeys?{}

Values of the service derived attribute row identifier fields derived from Keys?{}.

 

Processing

If Keys?{} is specified, calls the service derived attribute key lookup function to retrieve, for the row being deleted, service ID, derived attribute ID, sequence number, and effective date. If the lookup function does not find a unique row, an error is raised.

Continue processing as in fEV_ServiceDADeleteRows&() for one row when a sequence number is specified.

Returns

A non-zero value on success; otherwise, an appropriate error is raised.
