fCMN_TRECustomerNodeSearchLike_R?[]()

Syntax

fCMN_TRECustomerNodeSearchLike_R?[](
    EffectiveDate~,
    l_SearchParams?{})

 

Description

Searches for customer nodes matching specified search parameters, returning the customer node IDs and last modified dates.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

EffectiveDate~

In: Optional. If not specified, use current date.

l_SearchParams?{}

In: Hash containing the values for the search criteria.

Each name key and element pair in l_SearchParams?{} form a name value pair to be assembled into a WHERE clause (using 'and' between each pair). For example, the following parameter values map to a WHERE clause:

  • l_SearchParams?{'NODE_NAME'} := 'Big Company';
  • l_SearchParams?{'CUSTOMER_NODE_STATUS_CODE'} := 3;

The WHERE clause is:

NODE_NAME LIKE '%Big Company%' AND CUSTOMER_NODE_STATUS_CODE LIKE '%3%'

The parameter can contain and handle any of the following Convergent Billing scalar types:

  • String
  • Integer
  • Real
  • Date.

If another type value (unknown, array, or hash table) is used, an error is raised. If a key in the hash table does not match a field in CUSTOMER_NODE_TRE_V view, an exception is raised.

 

Returns

An array of hashes consisting of customer node IDs and last modified dates, if successful; otherwise, an exception is raised (for example, if an unexpected key or value in the hash is found).

Remarks

The function constructs an SQL LIKE WHERE clause based on the specified parameters.

As all update functions access the customer node entities by CUSTOMER_NODE_ID, the function enables external systems to retrieve the CUSTOMER_NODE_ID if it has no knowledge of the ID.

The following table lists keys of each returned hash.

 

Keys of Each Returned Hash

Key

Data Type

Description

CUSTOMER_NODE_ID

Integer

Customer node ID of the customer matching the search criteria.

LAST_MODIFIED

Date

Last modified date of the customer matching the search criteria.
