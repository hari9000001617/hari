EntityEVFetchByTypeId&()

Syntax

EntityEVFetchByTypeId&(
    const Entity$,
    EntityTypeId&,
    const EffectiveDate~)

 

Description

Derives the ID of the entity validation for a specified Convergent Billing entity.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

Entity$

In: Convergent Billing entity for which entity validation is required.

It corresponds to a code label from the ENTITY_VALIDATION_TABLE reference type.

EntityTypeId&

In: ID of the entity type associated with Entity$.

EffectiveDate~

In: Date when the entity type is in effect.

It is used only for date-ranged tables.

 

Returns

ID of the entity validation associated with the specified Entity$ and EntityTypeId&; otherwise, an error is raised.

Remarks

To successfully return the ID of the entity validation:

  • The entity validation must be able to be derived for the entity at the specified date.
  • Entity$ must be one of the following:
    • Convergent Billing entity listed in the table below
    • ATLANTA_OPERATOR
    • ATLANTA_PROFILE.
  • A record must exist for the type table for the specified EntityTypeId&.

The following table contains the mapping of entities to their type tables and entity validation ID columns selected from the Convergent Billing type table.

For the ATLANTA_OPERATOR and ATLANTA_PROFILE entities, the ID of the entity validation associated with the SYSTEM_CLIENT configuration item is returned, if set.
