Statement Search Form

The following table lists automation parameters for the Statement Search form, data type, and supported functions.

The description column is provided to supplement the description in the INVOICE_TRE_V view, or to provide information about parameters not contained within the database view.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

ALWAYSDISPLAYLIST

Integer

Not applicable

Value of the Always Display List? check box. Valid values are:

  • 0: Displays either a detail form or a list of queries.
  • 1: List of queries is always displayed.

REUSEDETAILFORM

Integer

Not applicable

Specifies reuse of the detail form. Valid values are:

  • 0: A new form is used.
  • 1: The open search form is used.

SEARCHCLICK

Integer

Not applicable

When a value is entered for this parameter, the Search button is clicked on the form.

The value is irrelevant.

It must be specified last.

CLEARCLICK

Integer

Not applicable

Clears the search fields.

The value is irrelevant.

CUSTOMER_INVOICE_STR

String

Not applicable

Defines the Statement Number field.

ISSUE_DATE

Date

Not applicable

Date the statement was issued to the customer.

The issue date is normally the same as the issue date displayed on the statement.

EFFECTIVE_DATE

Date

Not applicable

Date used to determine the charges included on a statement.

The statement includes charges (standard and recurring) accrued up to, and including, the effective date.

PAYMENT_DUE_DATE

Date

Not applicable

Date the statement is due to be paid.

NODE_NAME

String

Not applicable

Customer node that received the statement.

ACCOUNT_NAME

String

Not applicable

Name of the account to which the statement was applied.

SERVICE_NAME

String

Not applicable

Number of the service for which to search.

STATEMENT_AMOUNT

String

Not applicable

Statement amount as either a numerical value or an SQL expression.

INVOICE_TYPE_ID

Integer

Not applicable

Type of the statement.

BILL_RUN_ID

Integer

Not applicable

ID of the bill run that generated the statement.

QA_IND_CODE

Integer

Not applicable

Flag to determine the type of bill run. Valid values are:

  • 1: Quality assurance or statement run.
  • NULL: True bill run.

REPORT_LEVEL_CODE

Integer

Not applicable

Report level of the customer node.
