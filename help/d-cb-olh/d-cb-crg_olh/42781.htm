fEV_QueryClose&()

Syntax

fEV_QueryClose&(
    queryId&,
    var lastModified~,
    const resolutionCode&,
    const resolutionText$)

 

Description

Closes an existing customer query in the database. The function is similar to and wraps biQueryClose&().

Context

The following groups of variables are available throughout all stages of processing a customer query close request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value CUSTQRYSVR.
    • evLastRowInd& contains the value 0 during pre-commit expressions; otherwise, it remains undefined.
    • evRowStatus$ contains the value Update.
  • Customer query variables.

Parameters

Parameter

Description

queryId&

Customer query ID.

An open customer query with the specified ID must exist; otherwise, an error is raised.

lastModified~

Last modified date of the query to be closed.

Initial value of this parameter must match the value in the pre-existing record matching the queryId&.

On successful return, the value is the date-time of this closure.

resolutionCode&

Resolution code, included for TRE compatibility.

If not specified, 0 is inserted to remain compatible with the Convergent Billing client.

resolutionText$

Resolution text.

If specified, the contents of the previous RESOLUTION_TEXT field is overwritten.

The parameter is mandatory if the field is null.

 

Processing

Assuming that all processing stages are enabled, customer query pre-commit expressions are evaluated before the customer query is closed.

On performing the close, the following sequence of actions occurs:

  1. Evaluate customer query in-transaction expressions.
  2. Queue and evaluate customer query post-commit expressions.

Returns

1 on success. If the close or pre-close validation steps fail, an error is raised.
