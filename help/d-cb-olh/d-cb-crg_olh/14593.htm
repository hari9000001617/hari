eval()

Syntax

eval(
    expr$,
    var result<type>)

 

Description

Parses and evaluates the content of the expr$ parameter as an EPM expression, and returns the result of the expression in result<type>.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

expr$

In: Text string containing an EPM expression.

result<type>

Out: Result of the evaluation of the expression.

<type> represents the data type of the parameter and is one of the valid EPM data types; for example, ?{} indicates an unknown hash.

 

Returns

1 if successful; otherwise, an error is raised.

An error is raised if the expression cannot be parsed or any error occurs while evaluating the expression.  An error is raised if the return type of the expression does not match the type of result<type>.

Remarks

The expression can reference global and local variables, and call functions, that are in the scope of the expression or function containing the reference to eval(). It also has access to any constants and public nested functions that have been imported by its enclosing expression or function. If the eval() call is used within a function, then any private nested functions defined in the function or its enclosing function will also be available.

Parsing an expression is a relatively expensive operation, and it is recommended that excessive use of this function be avoided because it could adversely impact system performance (especially if called within rating or billing). To reduce performance overhead, the function uses the expression cache.

eval_cache_maxsize() can be used to determine the size of the cache at runtime in the current process.
