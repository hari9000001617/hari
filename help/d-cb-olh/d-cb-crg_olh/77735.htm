EntitlementPeriodicAllocBatchProcess&()

Syntax

EntitlementPeriodicAllocBatchProcess&(
    const ScheduledFunctions$[], 
    const InterfaceNrs&[],
    const ScheduledFunctionParameters?[],
    const EffectiveOperators&[],
    const EffectiveDates~[],
    var ErrorId&[],
    var ErrorMessage$[])

Description

Performs batch processing of scheduled functions for entitlement allocations.

Attributes

Function Type

Local

Service

Local

Application Environment

Any

Context

Any

 

Parameter

Description

ScheduledFunctions$[]

In: Names of the of entitlement allocation scheduled functions that have been consolidated for batch processing.

InterfaceNr&[]

In: Interface numbers of the scheduled functions in the batch.

ScheduledFunctionParameters?[]

In: Two-dimensional array of the parameters passed to each scheduled function.

EffectiveOperators&[]

In: Effective operator associated with each scheduled function.

EffectiveDates~[]

In: Effective processing date associated with each scheduled function. These are not currently used.

ErrorId&[]

Out: ID of any error that occurred while processing each function.

ErrorMessage$[]

Out: Error message text of any error that occurred while processing each function.

 

Returns



Number of entitlement allocations that were successfully processed in this batch.  Details of any errors are returned in the ErrorId&[] and ErrorMessage$[] parameters. 


Remarks


This function provides automatic batch processing of scheduled functions for entitlement allocations.   Its interface conforms with that expected by the Scheduled Function Monitor (SFM) to support batch processing of scheduled functions.   The SFM.CoreBatchFunctions?[]() function returns this function as the batch function for zEntitlementPeriodicAllocGenRemote&() with a maximum batch size of 100. 


This function runs locally in the SFM, starts a transaction and calls biEntitlementPeriodicBatchProcess&().  If this function succeeds, it commits the transaction.  If this function fails, it rolls back the changes, logs a message and processes the functions individually as separate transactions just as the SFM would if batch processing wasn't in use.
