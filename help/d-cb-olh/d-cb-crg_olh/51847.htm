EXTERNAL_DEFN_TRE_V

The EXTERNAL_DEFN_TRE_V view displays the external definitions configured for use with Singleview.

External definitions are used to define the format of data that is imported into or exported out of Singleview. They provide a mapping from the internal representation of Singleview to an external representation of the data.

Column Definitions

Properties

Name of the view to be generated from this external definition.

Views must be unique system-wide.

A view name can only be specified if the external definition is flagged as external; that is, EXTERNAL_IND_CODE is set to 1.

VIEW_SECURITY_ENABLE_IND_
CODE

NUMBER (1)

Set to 1 (True) if the view generated for this external definition must enforce Singleview group security on the rows returned.

It can only be set to 1 if EXTERNAL_VIEW_NAME is specified. If not set to 1, it must be NULL.

If set, only the rows for which the Oracle user performing the query has Singleview group security view access are returned.

If the Oracle user performing the query is $ATADBACONNECT or $ATASVRCONNECT, the Oracle session's CLIENT_IDENTIFIER is used to identify the Singleview user.

STATUS_CODE

NUMBER (2)

Not null

Status of this external definition.

It is based on the EXTERNAL_DEFN_STATUS reference type.

EXTERNAL_IND_CODE

NUMBER (1)

If set to 1 (True), this external definition is suitable for the generation of interfaces to integrate with external systems.

If not set to 1, it must be NULL.

If NULL, it is only available internally. It is a building block for the generation of complete external definitions.

INTERFACE_GENERATION_CODE

NUMBER (2)

Status of the interface generation for this external definition.

It can only be populated if EXTERNAL_IND_CODE is set to 1.

It is based on the EXTERNAL_DEFN_GENERATION reference type.

SCHEMA_GENERATION_CODE

NUMBER (2)

Status of XML schema generation for this external definition.

It can only be populated if EXTERNAL_IND_CODE is set to 1.

It is based on the EXTERNAL_DEFN_GENERATION reference type.

VIEW_GENERATION_CODE

NUMBER (2)

Status of the view generation for this external definition.

It can only be populated if EXTERNAL_IND_CODE is set to 1 and EXTERNAL_VIEW_NAME is specified.

It is based on the EXTERNAL_DEFN_GENERATION reference type.

CONTENT_MODEL_CODE

NUMBER (2)

Not null

Type of content formed by the elements of this external definition.

It is based on the EXTERNAL_DEFN_CONTENT_MODEL reference type.

The table below lists available content types.

If ATTRIBUTE_GROUP_IND_CODE is set, only content model 1 (Simple Content) and 4 (All) are valid values.

CHOICE_EXT_DEFN_ELEMENT_
ID

NUMBER (8)

Element whose value determines the element in the current Choice content model to be selected.

It can only contain a value if CONTENT_MODEL_CODE is set to 3 (Choice).

MODEL_GROUP_IND_CODE

NUMBER (1)

Set to 1 (True) if this entry defines an XML schema model group.

It can only be set to 1 if EXTERNAL_IND_CODE and ATTRIBUTE_IND_CODE are NULL.

ATTRIBUTE_GROUP_IND_CODE

NUMBER (1)

Set to 1 (True) if this entry defines an XML attribute group.

It can only be set to 1 if EXTERNAL_IND_CODE and MODEL_GROUP_IND_CODE are NULL.

If set to 1, all rows in the EXTERNAL_DEFN_ELEMENT_TRE_V view associated with this external definition must be defined as attributes and the CONTENT_MODEL_CODE must be set to 1 (Simple Content) or 4 (All).

CB_ENTITY

VARCHAR (30)

Name of the Convergent Billing entity linked to this external definition.

It corresponds to the supported set of entity validation table names.

It is based on the ENTITY_VALIDATION_TABLE reference type.

Only one of CB_ENTITY or PE_CLASS_ID can be specified. If neither CB_ENTITY nor PE_CLASS_ID are specified, the external definition is an abstract definition and cannot have EXTERNAL_IND_CODE set to 1.

CB_ENTITY_TYPE_ID

NUMBER (8)

Reference to the specific Convergent Billing entity type (for example, CUSTOMER_NODE_TYPE_ID) that is being used for this external definition.

It can only be specified if CB_ENTITY is specified.

ENTITY_VALIDATION_ID

NUMBER (8)

Convergent Billing entity validation ID associated with this external definition.

The value is derived from CB_ENTITY_TYPE_ID, if specified.

PE_CLASS_ID

NUMBER (8)

Process Engine class associated with this external definition.

For information about the Process Engine classes that can be specified in an external definition, refer to the ED_CLASS_CONFIG_V view.

Only one of CB_ENTITY or PE_CLASS_ID can be specified. If neither CB_ENTITY nor PE_CLASS_ID are specified, the external definition is an abstract definition and can have EXTERNAL_IND_CODE set to 1.

PE_OBJECT_TYPE_ID

NUMBER (8)

Process Engine object type associated with this class.

If specified, only global facts or facts associated with this object type can be referenced by this external definition.

It can only be specified if PE_CLASS_ID is specified.

API_PROCESSING_SEQNR

NUMBER (3)

Order in which underlying external definition APIs are called for each external definition in a linked set of definitions.

The default processing order is:

  1. Hierarchy depth
  2. Element sequence.

If this processing order is not appropriate, it can be used to explicitly specify a different order in which the APIs are called for each external definition.

DATE_RANGED_IND_CODE

NUMBER (1)

Derived

Set to 1 (True) if the underlying Convergent Billing or Process Engine entity is date-ranged.

It is automatically defaulted based on the value of either CB_ENTITY or PE_CLASS_ID.

If not set to 1 (True), it is NULL.

DATE_RANGED_NEW_IND_CODE

NUMBER (1)

Set to 1 (True) if the default behaviour for updating a date-ranged record is to insert a new record as of the effective date of the transaction.

If NULL, the default behaviour is to update the record identified by the effective date of the transaction.

INPUT_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition is appropriate for use with input from external systems.

OUTPUT_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition is appropriate for use for output to external systems.

INSERT_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition is appropriate for the input of new data from an external system.

It can only be True if INPUT_IND_CODE is set to 1.

UPDATE_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition is appropriate to update data from an external system.

It can only be True if INPUT_IND_CODE is set to 1.

UPDATE_INSERT_IND_CODE

NUMBER (1)

Set to 1 (True) if the update interface generated for this external definition should automatically perform an insert if the update fails due to the referenced entity not existing in Singleview.

It can only be set to 1 if INPUT_IND_CODE and UPDATE_IND_CODE are set to 1.

If not set to 1, it must be NULL.

UPDATE_CHILD_SYNC_IND_
CODE

NUMBER (1)

Set to 1 (True) if the update interface generated for this external definition should automatically:

  • Delete child entities that are not referenced in the updated data supplied in the interface.
  • Insert child entities included in the data that do not already exist in Singleview.

It is usually set when the external system supplies all required data and Singleview needs to synchronise its data with the external system.

If not set to 1 (True), it must be NULL.

It can only be set to 1 if INPUT_IND_CODE and UPDATE_IND_CODE are set to 1.

DELETE_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition can be used for deleting data when specified by an external system.

It can only be set to 1 if INPUT_IND_CODE is set to 1.

SEARCH_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition can be used for receiving search parameters from an external system.

It can only be set to 1 if OUTPUT_IND_CODE is set to 1.

FETCH_IND_CODE

NUMBER (1)

Set to 1 (True) if this external definition can be used for retrieving data for use by an external system.

It can only be set to 1 if OUTPUT_IND_CODE is set to 1.

SOURCE_EXTERNAL_DEFN_ID

NUMBER (8)

ID of the external definition from which this external definition was copied for a generated external definition.

PARENT_EXTERNAL_DEFN_ID

NUMBER (8)

ID of the external definition for which this generation was performed for a generated external definition.

If SOURCE_EXTERNAL_DEFN_ID is specified, it is mandatory. Otherwise, it must be NULL.

FETCH_EXTERNAL_DEFN_ID

NUMBER (8)

If this is a search external definition, it references the fetch external definition to be used.

If this is an insert, update, or delete external definition, this definition can be used to define the format of the response, if one is required.

It must be set to a fetch external definition or a search external definition linked to a fetch external definition.

The specified external definition must have access to the input elements for this external definition to perform the necessary queries to construct the response.

WHERE_CLAUSE

CLOB

WHERE clause for filtering the rows to be retrieved for a search external definition.

Elements from the search external definition can be used as parameters in the WHERE clause.

It can only be specified if SEARCH_IND_CODE is set to 1.

ORDER_BY_CLAUSE

CLOB

ORDER BY clause for filtering the rows to be retrieved for a search external definition.

Elements from the search definition can be used as parameters in the WHERE clause.

It can only be specified if SEARCH_IND_CODE is set to 1.

PE_SEARCH_ID

NUMBER (8)

ID of the Process Engine search to be used for a search external definition on a Process Engine class.

Elements from the search definition can be used as parameters for the Process Engine search.

It can only be specified if SEARCH_IND_CODE is set to 1.

PE_SXP_SEARCH

CLOB

Process Engine SXP expression to be used to perform the search for a search external definition.

Elements from the search definition can be used as parameters for the expression.

It can only be specified if SEARCH_IND_CODE is set to 1.

PE_ABSTRACT_IND_CODE

Number (1)

Set to 1 (True) if this is an abstract external definition used in a Process Engine context; otherwise, it is used in a Convergent Billing context.

Applicable only to external definitions that are completely abstract; that is, the parent and children have no Convergent Billing entity or Process Engine class.

It can only be specified if CB_ENTITY and PE_CLASS_ID are set to NULL, and EXTERNAL_IND_CODE is set to 1.

COMMIT_IND_CODE

Number (1)

If this column is set to 1 (True), then the generated interfaces will be configured to execute as remote functions in a non-transactional server that will auto-commit all changes made before returning.

For interfaces that perform both insert or update and fetch operations, a commit will be performed after the insert or update has completed and before the fetch is performed.

 

Content Types for CONTENT_MODEL_CODE

Reference Code

Abbreviation

Description

1

Simple Content

Restricted to a single element, and zero or more attributes.

It is used to define additional restrictions on an element that can be reused in another definition, or to extend a single element with additional attributes.

2

Sequence

One or more elements that must appear in the order specified.

3

Choice

Two or more elements, only one of which can be present in the data.

The XML schema does not define how a choice is made.

The choice of the value can be either:

  • At the discretion of the external system supplying the data.
  • Determined by the value of any other Simple Content element in the current or linked external definition. For more information, refer to the EXTERNAL_DEFN_ELEMENT_TRE_V and EXTERNAL_DEFN_CHOICE_TRE_V views.

4

All

Two or more elements that can appear in any order.

This type of content is not appropriate for use with interfaces where elements are determined by their position in the data stream.
