ClientDynamicForm&()

Syntax

ClientDynamicForm&(
    l_FormCode$,
    l_ParamNames$[],
    l_ParamValues?[],
    l_ReturnParamNames$[],
    var l_ReturnParamValues?[])

 

Description

Displays a dynamic form, using either entity validation or a derived attribute to control validation and display characteristics.

Attributes

Application Environment

Data Entry

Context

Any

Method

Local

 

Parameter

Description

l_FormCode$

In: Form code to use for security on attribute types.

l_ParamNames$[]

In: Array of parameter names to display.

l_ParamValues?[]

In: Array of values corresponding to the parameter names specified in l_ParamNames$[].

l_ReturnParamNames$[]

In: Array of field names to be returned by the function.

l_ReturnParamValues?[]

Out: Array of returned values of fields specified in l_ReturnParamNames$[].

 

Returns

If the dynamic form is set to display modally:

  • The ID of the button pressed is returned.
  • Values are returned in l_ReturnParamValues?[] in the same order that they specified in l_ReturnParamNames$[]. If return parameters are specified for a Derived Attribute form, they are all returned in a single parameter as a two-dimensional array.

If the dynamic form is not set to display modally (FORM.MODAL=FALSE), 0 is returned.

Remarks

The following items can also be displayed.

  • Form caption
  • Prompt label
  • Entry and display fields
  • Grid lists
  • Buttons (including caption).

The caller can request return parameters in the parameter ReturnParamNames$[].

Buttons

Parameter Name

Properties

Description

BUTTON.ALIGNRIGHT

Boolean

Defaults to FALSE

Right alignment of the button.

If TRUE, the button is aligned to the right-hand edge of the form.

BUTTON.CANCEL

Boolean

Defaults to FALSE

Cancel state of the button.

If set to TRUE, pressing the button does not execute any validation expressions on the form.

The Close button on the menu bar effectively has BUTTON.CANCEL=TRUE, and returns an ID of 0.

BUTTON.CAPTION

String

Mandatory

Caption for a new button.

Setting BUTTON.CAPTION creates a new button and any further setting of BUTTON parameters act on that newly created button until BUTTON.CAPTION is set again.

BUTTON.DEFAULT

Boolean

Defaults to FALSE

Default state of the button.

If TRUE, the button is triggered when the Enter key is pressed and the cursor is in a field on the form.

BUTTON.ID

Integer

Mandatory

ID of the button.

If the form is called modally, the ID of the activated button is returned as l_ModalResult&[].

The Close button on the menu bar has an ID of 0.

BUTTON.NAVBUTTON

Boolean

Defaults to FALSE

State of the navigation button on the form.

If TRUE, the navigation button arrow is displayed on the form.

 

Form Structure

Parameter Name

Properties

Description

FORM.ALIGN

String

Specifies the position of the form.

Options are LEFT, RIGHT, TOP,and BOTTOM.

Tip:

To anchor a child form to three edges of a parent form, use FORM.ALIGN and FORM.ANCHORS.

FORM.ANCHORS.BOTTOM

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchor to the bottom of the parent form.

FORM.ANCHORS.LEFT

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the left of the parent form.

FORM.ANCHORS.RIGHT

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the right of the parent form.

Tip:

To horizontally resize the form when the parent form is resized, use FORM.ANCHORS.LEFT and .RIGHT.

FORM.ANCHORS.TOP

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the top of the parent form.

Tip:

To vertically resize the form when the parent form is resized, use FORM.ANCHORS.TOP and .BOTTOM.

FORM.BRINGTOFRONT

Boolean

Specifies that the form can be displayed on top of other active forms.

FORM.CAPTION

String

Mandatory

Caption displayed on the title bar of the form.

FORM.HEIGHT

Integer

Height of the form.

FORM.HELPFORNAME

String

Specifies that the form can be used to control the Form Field Help Dump form, and be available for F1 help and form help.

FORM.LEFT

Integer

Left location of the form.

FORM.MINHEIGHT

Integer

Minimum height of the form.

FORM.MINWIDTH

Integer

Minimum width of the form.

FORM.MODAL

Boolean

Defaults to TRUE

Specifies whether the form is displayed modally.

FORM.POSNFORMCODE

String

Creates a unique static value key used to identify the form and store its position in the system registry.

Note:

The string supports a maximum character length of 12 characters.

FORM.PROMPT

String

Text displayed in the prompt label.

FORM.REFRESH.INTERVAL

Integer

Refresh interval for the dynamic form.

The interval is only implemented for the web browser.

Note:

An interval of 0 does not activate the refresh timer.

FORM.REUSEFORMCODE

String

Code that identifies the form.

The form is reused if all of the following conditions are met:

  • FORM.REUSEFORMCODE is set.
  • The reuse flag is set (using ClientSetReuse&).
  • Another call is received to display a form with the same reuse form code.

If a form is reused, only parameters displayed after this parameter are used in the form.

FORM.TOP

Integer

Top location of the form.

FORM.WIDTH

Integer

Width of the form.

 

Entity Validation Definition Form

Parameter Name

Properties

Description

ENTITY_VALIDATION_ID

Int64

Mandatory

Entity validation ID used to display the form.

fieldname

(for example, RECEIPT_NR)

Data type of field

Values for all fields accessible in the entity validation definition.

Simple data type checking is performed.

Note:

To set the values as defaults, assign the required field value to the field name.

 

Derived Attribute Definition Form

Parameter Name

Properties

Description

DERIVED_ATTRIBUTE_ID

Int64

Mandatory

Derived attribute ID to use when the form is displayed.

FIELDS

Two-dimensional array

Default values for all fields as defined in the derived attribute.

Note:

FIELDS is a single parameter value that specifies a two-dimensional array (corresponding to the rows and columns in the grid).

Any mismatch in the number of columns between this parameter and the derived attribute definition, or the data types, specified for any of the fields, result in an automation parameter validation error.

GRID.AUTOSIZE

Boolean

Defaults to FALSE

Grid sizing preference.

If TRUE, the columns in the grid automatically size to the full width of the grid, ignoring the column widths specified for the derived attribute.

GRID.CANDELETE

Boolean

Defaults to FALSE

Row deletion preference.

If TRUE, rows can be deleted from the grid.

GRID.CANINSERT

Boolean

Defaults to FALSE

Row insertion preference.

If TRUE, rows can be inserted into the grid.

GRID.COLUMNCLICK
SORTING

Boolean

Defaults to FALSE

Column sorting preference.

If TRUE, columns can be sorted by clicking on the column header in the grid.

GRID.CURRENTROW

Boolean

Defaults to FALSE

Row return status.

If TRUE, ClientDynamicForm&() returns the selected row. If FALSE, ClientDynamicForm&() returns all rows in the grid.

GRID.EDITABLE

Boolean

Defaults to FALSE

Grid edit status.

If FALSE, the grid cannot be edited and the cursor cannot be placed inside the text field of the grid.

GRID.MULTISELECT

Boolean

Defaults to FALSE

Multiple row selection preference.

If TRUE, multiple rows can be selected in the grid.

GRID.ROWSELECT

Boolean

Defaults to FALSE

Row selection preference.

If TRUE, selecting a cell on the grid selects the row.

Note:

Do not use GRID.ROWSELECT in conjunction with GRID.EDITABLE. The combination prevents the user updating the data in the grid. For the correct behaviour, use GRID.CURRENTROW in conjunction with GRID.EDITABLE.

GRID.ROWVALIDATE

Boolean

Defaults to FALSE

Row validation preference.

If TRUE, when a validating button is clicked, row validation is performed on the row regardless of whether any changes have been made.

GRID.SIZEROWTODATA

Boolean

Defaults to FALSE

Cell text handling preference.

If TRUE, the grid wraps text in the cells and resizes the rows to fit the text.
