CUSTOMER_NODE_TRE_V

The CUSTOMER_NODE_TRE_V view is used by functions defined for the biCustomerNode service.

Searching with biCustomerNodeSearch&() is not permitted on the following fields:

  • CARD_SECURITY_CODE
  • CREDIT_LIMIT_CURRENCY_NAME
  • CURRENCY_NAME
  • CUSTOMER_PARTITION_NR
  • DO_NOT_PURGE_IND_CODE
  • DUPLICATE_NODE_NAME_IND_CODE
  • EXPECTED_REVENUE_CURRENCY_NAME
  • GL_CODE_NAME
  • INITIAL_ACCOUNT_BALANCE
  • INVOICE_FORMAT_ID
  • INVOICE_FORMAT_NAME
  • PRIME_ACCOUNT_ID
  • PRIME_ACCOUNT_NAME
  • TURNOVER_CURRENCY_NAME.

Billing Definitions

Column Name

Properties

Description

SCHEDULE_ID

NUMBER (15)

Standard invoice cycle that the customer node is assigned to. It defaults to the invoice cycle of the parent node.

If the customer node has a parent node, this must be equal to the parent node's SCHEDULE_ID over the date range.

If NULL, the customer node is associated with a prepaid without statements customer hierarchy.

For inserts, only one of PREPAID_IND_CODE or SCHEDULE_ID can be specified as a not-null value.

If not specified, and PREPAID_IND_CODE is not set, it is derived from the parent node for customer child nodes; otherwise, it defaults to a bill cycle with the earliest task start date within the customer node's date range.

For updates, it cannot be specified for customer nodes associated with prepaid without statements customer hierarchies.

Note:

It is not mandatory for customer nodes associated with a prepaid without statements customer hierarchy.

PREPAID_IND_CODE

NUMBER (1)

If set to 1 (True), this customer node is configured as prepaid without statements; otherwise, the value must be NULL.

For inserts, only PREPAID_IND_CODE or SCHEDULE_ID can be specified to be not null.

PREPAID_IND_CODE cannot be specified for updates.

BILLING_PRIORITY

NUMBER (4)

Priority associated with processing this customer node in a bill run.

When a group of customer nodes are processed on a bill run, higher priority nodes are processed first.

A lower numerical value indicates a higher priority. If specified, the priority must be greater than or equal to 1.

BILLING_COMPLEXITY

NUMBER (10)

Numerical value indicating the expected time for performing a bill run on this customer node.

BILLING_CONFIGURATION_CODE

NUMBER (8)

Billing configuration to be used for processing this customer node.

It can identify billing functions or servers to be called for the various stages in processing a bill run for this customer node.

It is based on the CUSTOMER_BILLING_CONFIGURATION reference type.

SUPPRESS_IND_CODE

NUMBER (1)

If set to 1 (True), this customer node hierarchy can be suppressed from standard bill runs and standard QA runs (depending on the value of SUPPRESS_UNTIL_ISSUE_DATE); otherwise, the value must be NULL.

It can only be set to 1 for root customer nodes.

It is ignored for immediate bill runs, complete bill runs, or QA bill runs.

SUPPRESS_BILL_CYCLE_COUNT

NUMBER (2)

If SUPPRESS_IND_CODE has the value 1, it can be used to specify the number of bill cycles for which invoice generation should be suppressed for this customer hierarchy.

It can only be specified if SUPPRESS_IND_CODE has a value of 1.

Specifying a value greater than the number of pending tasks for this customer's bill cycle is not possible. The maximum value entered can also be constrained based on the customer type.

SUPPRESS_UNTIL_ISSUE_DATE

DATE

If SUPPRESS_IND_CODE is 1, it is mandatory, and specifies the invoice issue date on or after which invoice generation recommences for standard and QA batch bill runs.

The maximum period for which invoice generation can be suppressed can be restricted based on the customer type.

INVOICE_FORMAT_ID

INVOICE_FORMAT_NAME

CARD_SECURITY_CODE

DO_NOT_PURGE_IND_CODE

NUMBER (1)

Pseudo-column

If set to 1 (True) for an update, no notification is sent to the customer node cache for any changes to the customer node.

It can be used for the bulk updating of customer nodes, where it is known that the customer node cache will be restarted on completion of the load.

If not specified for an update, or is specified with a value of 0 (False), the customer node cache is purged after the update is committed.

CUSTOMER_PARTITION_NR

NUMBER (2)

Pseudo-column

Insert only

If specified with a valid customer partition number for an insert, the customer node ID is derived using the sequence for that partition.

A maximum of 99 customer partitions are supported (1-99).

It is supported only for inserts where CUSTOMER_NODE_ID and PARENT_CUSTOMER_NODE_ID are not specified.

For information about how the customer partition number is determined if CUSTOMER_PARTITION_NR is not specified, refer to the biCustomerNodeInsert&() function in the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2)).

It is only applicable for multi-instance configurations.

DUPLICATE_NODE_NAME_IND_
CODE

NUMBER (2)

Pseudo-column

Insert and update only

If specified with a value of 1 (TRUE) for an insert or update, duplicate node names for child customers with the same parent are allowed.

If not specified for an update or insert, or specified with a value of 0 (FALSE), duplicate node names for child customers with the same parent are not allowed.

TRANSFERRED_ACCOUNT_ID

NUMBER (15)

Pseudo-column

Insert and update only

Pseudo-column used for inserts or updates.

Account to which invoice amounts for this account are rolled up at billing time.

It specifies an account outside of the current hierarchy that is responsible for paying statements for this account.
