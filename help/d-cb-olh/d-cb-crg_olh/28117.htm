Description

Removes a product instance from the database over the specified date range. The function is similar to and wraps the biProductInstanceDelete&() function.

Note:

A product instance cannot be deleted if there are any events or charges associated with it during the specified date range. Deletion of a product instance includes all associated entities, including services, facility group instances, and companion products; and de-allocates any equipment allocated to the product instance.

Context

The following groups of variables are available throughout all stages of processing a product instance delete request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value PRODSVR.
    • evLastRowInd& contains the value 0 during all stages of processing except for the evaluation of the product instance pre-commit expressions.
    • evRowStatus$ contains the value Delete.
  • Product instance
  • Product instance equipment

    The product instance equipment variables may only be available during the product instance equipment processing stages.

  • Product instance service

    The product instance service variables may only be available during the product instance service processing stages.

  • Facility group instance

    The facility group instance variables may only be available during the facility group instance processing stages.

  • Companion product instance

    The companion product instance variables may only be available during the companion product instance processing stages.

Parameters

Parameter

Description

prodInstId&

ID of the product instance being removed.

A product instance with the specified ID must exist in the database for the specified date range; otherwise, an error is raised.

Keys?{}

Keys used to identify the product instance being removed.

startDate~

StartDate~

Start of the date range for which the specified product instance must be removed.

If start date is not defined, the current date is used.

endDate~

EndDate~

End of the date range for which the specified product instance must be removed.

If end date is not defined, the end of time value is used.

IdentifierKeys?{}

Values of the product instance identifier fields derived from Keys?{}.

 

Processing

If Keys?{} is specified, the product instance key lookup function is called to retrieve the product instance ID and the effective date to use. If the lookup function does not find a unique product instance, an error is raised.

Assuming that all processing stages are enabled, the following is the order of evaluation of expressions and constraints before removal of the product instance:

  1. Bundle direct variables if the call is nested.
  2. Decrypt any encrypted values and mask any appropriate values.
  3. Evaluate product instance row expressions.
  4. Evaluate product instance pre-commit expressions.
  5. Evaluate product instance service pre-commit expressions for each service in creation order.
  6. Evaluate product instance equipment pre-commit expressions for each allocated equipment item in creation order.
  7. Evaluate facility group instance pre-commit expressions for each facility group instance.
  8. Evaluate companion product pre-commit expressions for each companion product instance.

On performing the deletion, the following sequence of actions occurs:

  1. Evaluate product instance service in-transaction expressions for each service in order of creation.
  2. Queue and evaluate product instance service post-commit expression for each service in order of creation.
  3. Evaluate product instance equipment in-transaction expressions for each allocated equipment item in creation order.
  4. Queue and evaluate product instance equipment post-commit expressions for each allocated equipment item in creation order.
  5. Evaluate facility group instance in-transaction expressions for each facility group instance.
  6. Queue and evaluate facility group instance post-commit expressions for each facility group instance.
  7. Evaluate companion product in-transaction expressions for each companion product.
  8. Queue and evaluate companion product post-commit expressions for each companion product.
  9. Evaluate product instance in-transaction expressions.
  10. Queue and evaluate product instance post-commit expressions.

After deletion and evaluation of post-commit expressions:

  1. Unbundle variables that were stored in step 1 before deletion.

Returns

On success, a non-zero value. If the deletion or any of the pre-delete validation steps fail, an error is raised.
