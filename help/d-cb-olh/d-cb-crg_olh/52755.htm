Schedule Detail Form

The following table lists automation parameters for the Schedule Detail form, data type, and supported functions.

The description column is provided to supplement the description in the SCHEDULE_TRE_V view, or to provide information about parameters not contained within the database view.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

BASELINE_DURATION

Integer

Insert

Update

Expected duration of tasks of this schedule in seconds.

Note:

The value must be greater than or equal to 1, and less than or equal to 2147483647.

DEPENDENCY_FAILURE_IND_
CODE

Integer

Insert

Update

Specifies that the failure of any prerequisite task updates the status of tasks of this schedule to Failure.

DESCRIPTION

String

Insert

Update

Purpose of the schedule.

EFFECTIVE_DATE

Date

Insert

Update

Date-time used as the effective date by the first task in the schedule.

The values of START_REPEAT_TYPE_CODE and REPEAT_UNITS are used to derive the value of EFFECTIVE_DATE for subsequent tasks.

EMAIL_ADDRESS

String

Insert

Update

Email address where the task results will be sent.

Note:

If email notification is not required, leave EMAIL_ADDRESS undefined or set it to either of the following:

  • /dev/null
  • nobody.

FIRST_DATE

Date

Insert

Update

Date-time when the first task in the schedule is to occur.

The values of START_REPEAT_TYPE_CODE and REPEAT_UNITS are used to derive the value of the starting date for all subsequent tasks in the schedule.

Caution:

This field cannot be blank.

GENERAL_1 - GENERAL_25

String

Insert

Update

General fields 1 - 25 that can be used as parameters for the scheduled tasks.

IMMEDIATE_START_IND_CODE

Integer

Insert

Update

Tasks of this schedule start immediately when all prerequisite conditions are satisfied.

In a schedule with no prerequisites, a task completes and the next scheduled task starts immediately.

INITIAL_HELD_IND_CODE

Integer

Insert

Update

Tasks of this schedule enter 'Held' status when ready to run.

INSTANCE_NAME

String

Insert

Update

Convergent Billing instance on which the task associated with the schedule was run.

LAST_DATE

Date

Insert

Update

Last date-time that can be used as the EFFECTIVE_DATE for a task.

If a value is not specified, tasks are scheduled indefinitely.

MAX_DURATION

Integer

Insert

Update

Maximum duration of tasks of this schedule in seconds.

Note:

The value must be greater than or equal to 1, and less than or equal to 2147483647.

PRESCHEDULE_COUNT

Integer

Insert

Update

Number of tasks scheduled in advance.

The value must be greater than zero.

PURGE_PERIOD

Integer

Insert

Update

Maximum number of days temporary data associated with a task of this type is kept.

REPEAT_UNITS

Integer

Insert

Update

Number of units of the type START_REPEAT_TYPE_CODE allowed between specific tasks.

REPLICATE_GROUP_CODE

Integer

Insert

Update

Specifies a set of instances to which the schedule is replicated.

REPLICATE_IND_CODE

Integer

Insert

Update

Replicates the schedule to a specified set of instances associated with the replication group.

SCHEDULE_NAME

String

Insert

Update

Name of the schedule.

SCHEDULE_TYPE_ID

Integer

Insert

ID used to define the type of task or function that is being scheduled.

SERVER_COMMAND_ARGS

String

Insert

Update

Additional server command-line parameters used when invoking the tasks of this schedule.

START_REPEAT_TYPE_CODE

Integer

Insert

Update

Type of repetition for a scheduled task. Valid values are:

  • 1 (month-start): nth day of week of kth week from start of the month
  • 2 (month-end): nth day of week of kth week from end of the month
  • 3 (month-day): nth day of the month
  • 4 (week): nth day of the week
  • 5 (day)
  • 6 (hour)
  • 7 (minute)
  • 8 (second).

Values 'n' and 'k' are derived from the FIRST_DATE parameter.

SUPPRESS_OVERDUE_TASK_
IND_CODE

Integer

Insert

Update

Specifies that pending tasks whose start date-time is less than the current date-time be suppressed.

TASK_CREATION_EXPR

String

Insert

Update

Optional expression executed immediately before a new task is inserted by the Scheduled Task Dispatcher (STD).
