ProductInstanceSvcInsert&()

Syntax

ProductInstanceSvcInsert&(
    ProductInstanceId&,
    StartDate~,
    var LastModified~,
    ServiceItemSeqnr&)

 

Description

Creates an optional service for the product instance from the specified start date-time to the end of time.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

ProductInstanceId&

In: ID of the product instance.

StartDate~

In: Start of the date-time range for the service item being inserted.

LastModified~

In: Last modified date-time of the product instance to be updated.

Out: Last modified date-time of the updated product instance.

ServiceItemSeqnr&

In: Product service item being inserted.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The specified product service type (identified by ServiceItemSeqnr&) may have optional services if the SERVICE_RANGE_MAX field in the PRODUCT_SERVICE_TYPE table is greater than the SERVICE_RANGE_MIN field, in which case:

  • An optional service has a SERVICE_RANGE_SEQNR field (in the PRODUCT_INSTANCE_SERVICE table) between SERVICE_RANGE_MIN + 1 and SERVICE_RANGE_MAX.
  • A mandatory service has a SERVICE_RANGE_SEQNR field between 1 and SERVICE_RANGE_MIN, assuming SERVICE_RANGE_MIN is greater than 0.

If the base product instance is assigned a product offer, the supported service range is determined based on the non-cancelled companion product instances for this base product instance according to the algorithm described for [bi]ProductSvcEquipRange?[]().

If the base product offer is not assigned a product offer, the supported service range is retrieved from the PRODUCT_SERVICE_TYPE table for this base product.

The specified product instance must be a base product having a status that is not Moved or Upgraded at any time in the product instance's history.

A service can be added to a product instance that has a status of Cancelled at a future date-time (that is, greater than StartDate~), in which case, the added service is cancelled at the future date-time to match to the product instance. The service is cancelled at the appropriate date-time when the product instance is completed again.

The start date-time for the insert must match the effective start date of any incomplete optional services or equipment; otherwise, an error is raised. To add an item using a different start date-time, the base product instance must be completed first.

The base product instance is flagged as being incomplete; that is, PRODUCT_INSTANCE_HISTORY records from StartDate~ to end of time have UNCOMPLETED_IND_CODE set to 1. The inserted PRODUCT_INSTANCE_SERVICE records have UNCOMPLETED_IND_CODE set to 1.

The next available optional SERVICE_RANGE_SEQNR value is used as the SERVICE_RANGE_SEQNR for the optional service for the product service type. If SERVICE_RANGE_MAX has been reached or optional services are not allowed for the product service type, an error is raised.

If the environment variable ATA_SERVICE_NORESEQ is set to 'Y', any gaps in the service range sequence numbers left by cancelling services are reused when the SERVICE_RANGE_MAX limit is reached. The lowest service range sequence number is chosen from any gaps effective for the specified StartDate~.

If the insert affects a period with multiple PRODUCT_INSTANCE_SERVICE records for the SERVICE_ITEM_SEQNR, the new service item may be allocated different SERVICE_RANGE_SEQNR values over the insert period based on the current greatest optional SERVICE_RANGE_SEQNR for the service item within each date range.

Any mandatory equipment items required by the product service type are also created.

The inserted PRODUCT_INSTANCE_SERVICE records have their UNCOMPLETED_IND_CODE set to 1.

On entry to the function, LastModified~ must be equal to the LAST_MODIFIED field of the PRODUCT_INSTANCE_HISTORY record identified by ProductInstanceId& as effective at StartDate~. On successful completion, LastModified~ is the new value of the LAST_MODIFIED field of the updated PRODUCT_INSTANCE_HISTORY record.

This function supports product instance history records with sub-day granularity, so StartDate~ can specify a time component.

The effective user must have insert security group access level for the customer associated with the record manipulated by this function call.

The customer security group access level required is determined by the effective start date of the product instance, or the effective date supplied by the active user.

GROUP_ACCESS_TRE_V view lists security group access level values.
