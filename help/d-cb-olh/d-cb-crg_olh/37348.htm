biInvoicePrepaidGenerate&()

Syntax

biInvoicePrepaidGenerate&(
    BillRunId&,
    EffectiveDate~,
    BillRunOperationId&,
    RootCustomerNodeList&[],
    UsageChargesBeforeBillDate&,
    var SuccessCustomerNodeList&[],
    var ErrorCustomerNodeList&[],
    var SuppressedCustomerNodeList&[],
    var Statistics?{})

 

Description

Generates invoice statements only for prepaid customer hierarchies.

Note:

The prepaid billing configuration must not rely on Bill Generation Process (BGP) functionality such as billing tariffs, subtotals, and invoice expressions.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

BillRunId&

In: ID of the bill run.

EffectiveDate~

In: Effective date of the bill run.

BillRunOperationId&

In: ID of the bill run operation.

RootCustomerNodeList&[]

In: List of one or more prepaid root customer nodes to be processed.

Note:

All services for a root customer node and any child node in a hierarchy must be prepaid.

UsageChargesBeforeBillDate&

In: If set to:

  • 1 (TRUE), the function processes usage charges before but not including the bill run effective date.
  • 0 (FALSE), processes usage charges before and including the bill run effective date.

SuccessCustomerNodeList&[]

Out: List of all root customer nodes successfully processed.

It is a subset of the RootCustomerNodeList&[] parameter and preserves the order of the RootCustomerNodeList&[] parameter.

ErrorCustomerNodeList&[]

Out: List of root customer nodes unsuccessfully processed or unable to be locked because of other billing processing the customer node.

SuppressedCustomerNodeList&[]

Out: List of root customer nodes whose invoices were suppressed.

A root customer node ID is added to this parameter if all prime accounts in the customer's hierarchy are suppressed. An invoice is suppressed if the account does not have charges.

Statistics?{}

Out: Hash containing the statistics gathered during the processing of the list of root customers.

The table below lists the parameter hash keys.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The function generates invoices using the specified effective date.

The function processes each root customer node in turn and does not perform any multi-processing within a root customer node. This may have throughput implications for processing large (corporate) hierarchies or high volume services.

The GLGuidance?[]()function is not invoked for any charges; therefore, all rating charges associated with the specified set of customers must only have Earned But Unbilled (EBUB) GL Guidance entities associated with them. Generation of invoices or statements by biInvoicePrepaidGenerate&()does not change the Sales Journal.

Note:

The following conditions apply:

  • The function cannot be called as part of a Quality Assurance bill run.
  • The function aborts if the number of errors detected during the operation exceeds the threshold provided by BillRunInvoicePrepaidMaxErrorsPerBatch&().
  • Charges updated by this function do not have the INVOICE_SECTION_NR and INVOICE_ITEM_NUMBER columns updated.

The following lists the Statistics?{} parameter hash keys.

 

Statistics?{} Parameter Hash Keys

Key

Description

TotalCharges

Number of charges processed by the bill run.

TotalCustomerNodes

Number of customer nodes processed by the bill run.

Invoices

Number of invoices generated for the bill run.

Statements

Number of statements generated for the bill run.

InvoiceAmount

Total of the invoiced amount generated by the bill run.

StatementAmount

Total of the statement amount generated by the bill run.
