biBillRunOperationNetSummary&()

Syntax

biBillRunOperationNetSummary&(
    BillRunId&,
    var FieldValues?[],
    DeriveFailStats&)

 

Description

Provides a short, net summary of a bill run, with only one line for each operation code performed, regardless of how many times the operation code was revoked and rerun, either completely or in part.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

BillRunId&

In: Bill run for which details are being retrieved.

DeriveFailStats&

In: If set to TRUE, the key statistics are recalculated directly from the INVOICE table.

FieldValues?[]

Out: Result set consisting of one row for each different operation code that was performed on the bill run.

Any revoke operations that have been performed are not included; their effects are combined into the row for the operation they correspond to.

OPERATION_COUNT and DURATION are calculated for each row, as the function executes, and the final values attached to the results set. The results are returned in a two-dimensional array in row, column order.

The table below lists the columns returned for each row.

 

Returns

Number of rows returned in FieldValues?[].

Remarks

If the billing operation was revoked and rerun in its entirety, the revoke counts are subtracted from the billing counts, providing an accurate net summary. However, if a partial revoke occurs or one or more operations fail, incorrect summary calculations may occur.

Setting the DeriveFailStats& parameter to True forces rederivation of some key statistics directly from the INVOICE table when operations fail. This ensures the correctness of the amount, invoices count, and statements count statistics returned for the Invoice/Statement generation (BGP) operation, and the amount and invoices count for the Apply Invoices operation.

All bill run operations for the bill run are retrieved from BILL_RUN_OPERATION and an empty results set is created. For each billing operation encountered that is not a rerun, a new row is inserted into the results set. If a rerun or revoke is encountered, the results set count for the corresponding row is updated.

The function relies on the SUCCESS_COUNT, INPUT_COUNT, and ERROR_COUNT columns being populated for all operations, including billing and revoke operations that have a status of failure.

During execution, any failed BGP or Apply Invoices operations or failed revoke of these operation types are noted.

If DeriveFailStats& is True and any BGP or revoke BGP operations fail, the BGP summary statistics that have been calculated are discarded and recalculated directly by querying the invoice table. Similarly, any apply or unapply of invoices that fail force the statistics for the Apply Invoices operation to be re-derived.

For BGP statistics, a query returns the sum of all invoices in the bill run for each currency. The currency and effective date of the bill run are retrieved and used to convert each summed amount to the currency of the bill run type. The converted figures are then added together to produce the amount statistic for the operation.

Further queries are used to derive the count of the invoice records, with and without a null statement amount, that correspond to the invoice and statement counts respectively.

The same derivation is used for the Apply Invoices statistics except for the added restriction that invoice records must be flagged with APPLIED_IND_CODE = 1 to be included in the calculations. The statements count is not required for this operation.

The following table lists the columns returned for each row.

 

Columns Returned for Each Row

Column

Description

BILL_RUN_OPERATION_ID

Operation ID for the first billing operation of this code in the bill run.

BILL_RUN_ID

Bill run ID.

BUN_RUN_OPERATION_CODE

Operation Code.

One row is returned for each different operation code in the original set of operations.

START_DATE

Earliest start date for the operation code in the bill run (including revoke operations).

END_DATE

Latest end date for the operation code in the bill run (including revoke operations).

INPUT_COUNT

Effective input count for the operation code in the bill run.

SUCCESS_COUNT

Effective success count for the operation code in the bill run.

ERROR_COUNT

Effective error count for the operation code in the bill run.

PRE_SKIP_COUNT

Effective number of skipped customers for the operation code in the bill run.

Note:

The revoke operation must be configured to correctly revoke skipped customers for PRE_SKIP_COUNT to be calculated correctly.

POST_SKIP_COUNT

Effective number of suppressed customers for the operation code in the bill run.

Note:

The revoke operation must be configured to correctly revoke suppressed customers for POST_SKIP_COUNT to be calculated correctly.

AMOUNT

Net amount billed for the operation code in the bill run, taking into account any revoked amounts.

OPERATION_COUNT (Derived)

Total number of billing and revoke operations of the operation code that have been performed.

DURATION (Derived)

Total duration of all billing and revoke operations of the operation code that have been performed.
