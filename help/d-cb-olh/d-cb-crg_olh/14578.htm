Description

Inserts equipment item details into the database.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

StartDate~

In: Start date-time for the new equipment item.

LastModified~

Out: Last modified date-time of the inserted equipment item.

FieldNames$[]

In: List of fields being inserted.

Field names are defined in EQUIPMENT_TRE_V view.

When using interface 1, string fields should be specified first, followed by integer, real, and date fields.

The minimum set of fields that must be specified are:

  • EQUIPMENT_TYPE_ID
  • EQUIPMENT_STATUS_CODE
  • NR_ITEM
  • SERIAL_NUMBER.

StringFieldValues$[]

In: Values of string fields to insert.

IntegerFieldValues&[]

In: Values of integer fields to insert.

RealFieldValues#[]

In: Values of real fields to insert.

DateFieldValues~[]

In: Values of date fields to insert.

FieldValues?[]

In: Values of fields to insert.

 

Returns

ID assigned to the new equipment item if successful; otherwise, an error is raised.

Remarks

If a value for EQUIPMENT_ID is not specified, the value is derived from the entity sequence of the specified CUSTOMER_PARTITION_NR. If a value for CUSTOMER_PARTITION_NR is not specified, the appropriate customer partition is determined from the RESERVED_CUSTOMER_NODE_ID. If a value for RESERVED_CUSTOMER_NODE_ID is not specified, the default partition of the instance performing the insert is used ($ATA_INSTANCE).

For serial number and network address details, at least two NR_ITEMS must be supplied: a 'from' serial number and a 'to' serial number.

If the EQUIPMENT_STATUS_CODE is being set to In-Transit, the ATLANTA_GROUP_ID is automatically set to the group associated with the RECEIVER_PERSON_ID. The SENDER_PERSON_ID is automatically assigned to the person associated with the current user. The TRANSIT_DATE is set to the current date-time.

Note:

This function prevents concurrent insertion of equipment with the same serial number but does not exclude concurrent insertion of:

  • Equipment with the same network address but different serial numbers
  • Ranges of equipment with overlapping serial numbers or network addresses.

The effective user must have insert security group access level for the group associated with the record manipulated by this function call.

GROUP_ACCESS_TRE_V view lists security group access level values.
