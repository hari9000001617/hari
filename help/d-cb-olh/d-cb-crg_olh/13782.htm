Parameter

Description

Subtotal@

In: ID of the subtotal.

If the subtotal name is xxx&, the blob ID is xxx@.

ServiceId&

In: ID of the service (SERVICE_ID).

EffectiveDate~

In: Any date in a specific subtotal period.

For example, if a subtotal is aggregated daily, any time within a day matches that day.

ReservedValue<T>

Out: Sum of any reservations against the subtotal in the effective period.

The type of this parameter matches the return type of the interface.

FromDate~

In: Start date of the subtotal aggregation period.

ToDate~

In: End date of the subtotal aggregation period.

 

Returns

Subtotal value of the function's return type if successful and aggregates exist; an undefined value of the function's return type if successful and no aggregates exist; otherwise, an exception is raised.

Remarks

Return types (<T>) can be integer (&), real (#), integer hash (&{}), or real hash (#{}).

Each variant of the function can be accessed through the trerate server (EventRateAndReturn&() and EventRateAndStore&()). Some of these interfaces can also be accessed outside of rating (as indicated below).

Interfaces 1 and 2

The ServiceId&() and EffectiveDate~() are implied from the event being rated.

For EventRateAndReturn&() in interface 1, the returned value contains reservations, excluding the current reservation. For EventRateAndStore&(), the returned value does not contain reservations.

In interface 2, the reserved value is returned separately in all cases. For EventRateAndReturn&(), the returned value excludes the current reservation.

Interface 3

Available outside of Rating if the subtotal periods are context-independent. For EventRateAndReturn&(), the returned value contains reservations, excluding the current reservation. For EventRateAndStore&(), the returned value does not contain reservations.

Interface 4

Available outside of Rating if the subtotal periods are context independent. In Interface 4 and 6 the reserved value is returned separately in all cases, and for EventRateAndReturn&(), the returned value excludes the current reservation.

Interface 5

For EventRateAndStore&() and non-rating use, the returned value does not contain reservations.

Interface 6

If the Subtotal@ variable is a slave subtotal, the FromDate~ and ToDate~ must align with the Effective_Start_Date and Effective_End_Date of the master rating subtotal.
