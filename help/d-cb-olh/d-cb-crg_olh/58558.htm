Description

Commits consumption of entitlements as reservations in the cache.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

Reservation$

In: Reservation ID associated with the event.

Mode&

In: Mode of tracking automatic reservation consumptions. Valid values are:

  • 0 (Reserve All): Consumed amounts are not tracked. Default.
  • 1 (Consume All): Consumed amounts are tracked. The reserved amount is automatically considered as being consumed.
  • 2 (Consume Update): Consumed amounts are tracked. When reservations are extended, the previously reserved amounts are added to the consumed amount. Consumed amounts accumulate, and additional amounts are reserved for the session.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

This function should be called following an EventRateAndReturn&() call where the caller wants to keep any consumptions as entitlement reservations. If this function is not called, or EntitlementEventRollback&() is called instead, any consumptions made during the EventRateAndReturn&() call are discarded.

This function is called automatically from SubtotalEventCommit&(). However, if rating subtotals are not in use, it must be called explicitly.

Amounts consumed using the automatic interfaces of EntitlementConsume&() are split into reserved and consumed components according to the mode specified. Amounts explicitly specified using the manual interfaces of EntitlementConsume&() are not affected by the consume mode. The automatic and manual components are added to give the reserved and consumed totals for the reservation.

When calculating the consumed amounts in Consume Update mode, only previous automatic reserved and consumed amounts are factored in. Any previous manually specified amounts are ignored.

Note:

The mode of tracking reservation consumptions is set on a per reservation request. The mode is ignored on subsequent calls for the specified reservation.
