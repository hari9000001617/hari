Description

Fetches information about an entity's available entitlement amount.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

Context&

In: Context of the entitlement being fetched. Valid values are:

  • 2 (Service),
  • 3 (CustomerNode),
  • 4 (Customer).

EntityId&

In: ID of the entity associated with the entitlement being fetched.

If Context& is set to:

  • 2, this parameter must be set to the service ID.
  • 3 or 4, this parameter must be set to the customer node ID.

Category&

In: Reference code of the entitlement category being fetched.

It must exist in the ENTITLEMENT_CATEGORY reference type.

SubCategory$

In: Sub-category of the entitlement being fetched.

EffectiveDate~

In: Effective date of the request.

Reserved#

Out: Total amount of reservations.

The current reservation is not included in this amount.

 

Returns

Total available amount of entitlements that matches the supplied parameters.

Remarks

If an interface is called without the EffectiveDate~ parameter, the effective date of the expression is used. In a Rating context, it is the charge start date of the event.

If consumed amounts are being tracked, the following rules apply:

  • The reservation amount is always automatically considered to be consumed.
  • Amounts consumed from reservations reduce the available amounts of entitlement allocations:
    • In rate and return mode, the available amount is reduced by the sum of all reserved and consumed amounts. The current reservation is excluded.
    • In rate and store mode, the available amount is reduced by the sum of all the consumed amounts. The current reservation is excluded.

The mode of tracking reservation consumption can be specified using EntitlementEventCommit&(). The default mode is Reserve All, where consumed amounts are not tracked.
