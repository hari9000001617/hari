ACCOUNT_TYPE_TRE_V

The ACCOUNT_TYPE_TRE_V view defines the account types.

Column Definitions

Column Name

Properties

Description

ACCOUNT_TYPE_ID

NUMBER (8)

Not null

ID of a type of account.

LAST_MODIFIED

DATE

Not null

Date-time that the record was last modified.

ACCOUNT_TYPE_NAME

VARCHAR (30)

Not null

Name of this type of account.

ACCOUNT_TYPE_DISPLAY_NAME

VARCHAR (60)

Not null

Name of account type displayed.

It defaults to ACCOUNT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

DESCRIPTION

VARCHAR (255)

Not null

Description of this type of account.

OPERATOR_ID

NUMBER (8)

Not null

ID of user who created or last modified the record.

GROUP_ID

NUMBER (8)

Not null

ID of security group associated with the record.

ACCOUNT_CLASS_CODE

NUMBER (2)

Not null

Class of the type of account. Valid values are:

  • 1 (Asset)
  • 2 (Liability).

Account class cannot be modified once an account of the type exists.

It is based on the ACCOUNT_CLASS reference type.

ACCOUNT_CATEGORY_CODE

NUMBER (2)

Not null

Category of the type of account. Valid values are:

  • 1 (Receivable)
  • 2 (Payable)
  • 3 (Deposit)
  • 4 (Credit)
  • 5 (Bank (Cash)).

The account category must be compatible with the account class.

It is based on the ACCOUNT_CATEGORY reference type.

ASSET_INCREASE_PROMPT

VARCHAR (40)

Not null

Label for the column on the Account History form that displays increases in the value of an asset account, or decreases in the value of a liability account.

For example, for a receivable account the value could be 'Charge'.

ASSET_DECREASE_PROMPT

VARCHAR (40)

Not null

Label for the column on the Account History form that displays decreases in the value of an asset account, or increases in the value of a liability account.

For example, for a receivable account the value could be 'Payment'.

CURRENCY_ID

NUMBER (8)

Not null

Currency in which balances for the type of account are maintained.

ACCOUNT_FORMAT_EXPR

VARCHAR (255)

Not null

EPM expression defining the format for generating the account number for a new account of this type.

The expression has access to customer node information, and the account ID generated for the account (using the variable @value&).

INVOICE_TYPE_ID

NUMBER (8)

Not null

Type of invoice associated with the account.

Invoice type must exist from the current date-time until the end of time.

ACCOUNT_RESERVATION_EV_ID

NUMBER (8)

ID of entity validation used to define layout of the Account Reservation tab on the Account History form.

The entity validation referenced must use ACCOUNT_RESERVATION_TRE_V validation table (displayed as Account Reservation TRE V by the client).

If not defined, the Account Reservation tab is not displayed.

VERSION_STR

VARCHAR (255)

Version identification string used to tag the entity.

It is provided for use by third party configuration management tools and is not directly used by Singleview.

MULTIPLE_IND_CODE

NUMBER (1)

Indicates if multiple accounts of this type are allowed on the same customer node.

If set to 1 (TRUE), multiple accounts of the same type can exist.

If not set to 1, it must be NULL.

Account types with this set to 1 cannot be specified as part of a charge category.

ENTITY_VALIDATION_ID

NUMBER (8)

ID of entity validation for the Account History form.

The entity validation referenced must use ACCOUNT_HISTORY validation table (displayed as Account by the client).

BALANCE_FUNCTION_DEFN_ID

NUMBER (10)

ID of account balance function (see the Finance Guide for Singleview), which is called when the balance, unbilled amount, or invoice unbilled amount of an account of this type is updated.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this entity.

Refer to the CATEGORY_V view for the list of available categories.

BALANCE_MANAGED_IND_CODE

NUMBER (1)

Enables the balance of an account of this type to be managed in real time.

If set to 1 (True), credit limit checks are performed when:

  • Events are generated for real-time tariffs.
  • Reservations are created for these events before charging.
  • The unbilled amount of the account is updated with a charge generated by a tariff configured for real-time account aggregation.

If not set to 1, it must be NULL.

 

Dispute Columns

Column Name

Properties

Description

DISPUTE_AVAIL_CREDIT_IND_
CODE

NUMBER (1)

Specifies whether the amount in dispute is added to the customer's available credit or ignored.

If set to 1 (True), a customer's available credit is increased by the total amount in dispute.

If NULL, disputed amounts are ignored when calculating the available credit.

It defaults to 1.

DISPUTE_ADJUSTMENT_TYPE_ID

NUMBER (8)

Default type of adjustment to automatically create on acceptance of a dispute against an account of this type.

DISPUTE_BA_FUNCTION_DEFN_ID

NUMBER (10)

Function that is called to derive the billed amount associated with a specified disputed charge.

If specified, the function must exist from the current date-time until the end of time.

DISPUTE_GL_FUNCTION_DEFN_ID

NUMBER (10)

Function that is called to modify the GL breakdown for a specified charge in dispute.

If not specified, the GLChargeGuidance&() function is called.

If specified, the function must exist from the current date-time until the end of time.

DISPUTE_ADJ_FUNCTION_DEFN_ID

NUMBER (10)

Function that is called to adjust the overall amount in dispute based on the set of itemised charges (if any), and return a receivable type breakdown (if applicable).

If specified, the function must exist from the current time until the end of time.
