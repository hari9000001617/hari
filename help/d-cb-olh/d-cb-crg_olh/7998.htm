RGP_NORMALISED_EVENT_V

The RGP_NORMALISED_EVENT_V view is used by the RGP in the generation of its rental events.

Column Definitions

Column Name

Properties

Description

PRODUCT_INSTANCE_ID

NUMBER (15)

Not null

ID of a product instance.

It is mandatory, and identifies that product instance associated with the tariff.

FAC_GROUP_INSTANCE_ID

NUMBER (15)

ID of a facility group instance.

Note:

Only one of the columns PRODUCT_INSTANCE_ID or FAC_GROUP_INSTANCE_ID can contain a value, enforced by C_RGP_NORMALISED_EVENT check constraint.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node associated with this product instance or facility group instance.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the root customer node associated with this product instance at the effective date-time of the bill run that generated this rental or adjustment event.

It supports rapid revoking of rental or adjustment events by root customer.

SERVICE_ID

NUMBER (15)

Not null

ID of the service associated with this product instance to be associated with this rental event.

The C_PARTY_INTERNAL_ID in the NORMALISED_EVENT table is set to this value.

EQUIPMENT_ID

NUMBER (15)

ID of an item of equipment associated with this event.

It only contains a value if the tariff's charge entity is equipment.

TARIFF_ID

NUMBER (8)

Not null

ID of a recurring tariff associated with this product instance.

VERSION_STR

VARCHAR (30)

Optional version ID associated with the tariff that defines what version of the tariff was used to generate this event.

BILL_RUN_ID

NUMBER (15)

Not null

ID of the bill run that caused the generation of this record.

For rental generation records, this is always the same as the BILL_RUN_ID in the RGP_FILE table.

For rental adjustment records, it is the ID of the bill run that generated the original record for which this record is an adjustment.

QA_IND_CODE

NUMBER (1)

Redundant

If this event was generated as part as part of a Quality Assurance bill run, this column is set to 1 (TRUE); otherwise, it must be null.

It can be derived from the BILL_RUN_ID in the RGP_FILE table (not the BILL_RUN_ID in this table).

For the purposes of calculating rental periods and adjustments, the RGP ignores all records in this table with this column set to 1.

PERIOD_START_DATE

DATE

Not null

Date-time used as the start date for the calculation of the rental duration period.

PERIOD_END_DATE

DATE

Not null

Date-time used as the end date for the calculation of the rental duration period.

END_DAY_OF_MONTH

NUMBER (2)

Not null

Day of the month when the RGP billed for this event.

The PERIOD_END_DATE may have a day of month less than this value if the month does not have this many days. For example, END_DAY_OF_MONTH may be 30, but PERIOD_END_DATE may be 28-02-2003.

For the purposes of prorating and calculation of future rental durations, if END_DAY_OF_MONTH is 30, 28-02-2003 to 30-03-2003 is considered to be 1 month.

STATUS_CODE

NUMBER (8)

Not null

Status of the product instance, facility group instance, or services over the rental duration period.

The entity referred to is dependent on the tariff definition.

For rentals associated with equipment, it is the status of the associated primary service.

LAST_MODIFIED

DATE

Not null

Date-time when the RGP generated this record.

DURATION

NUMBER

Not null

Duration of this rental as a fraction of the base rental period of the tariff.

It is usually 1.

 

Tariff Defined Columns

The following columns are optionally populated by assignment expressions associated with the tariff.

 

Column Name

Properties

Description

NORMALISED_EVENT_TYPE_
ID

NUMBER (8)

Type of event to be generated for recurring charges.

It is automatically populated by the RGP based on the details in the TARIFF_RECURRING table, but can be overridden or cleared by recurring tariff assignment expressions.

C_PARTY_NAME

VARCHAR (64)

Name of the C party, if known (for example, this can be provided by the switch for user-assisted calls).

C_PARTY_TON_CODE

NUMBER (8)

Charge party type of number.

C_PARTY_CELL_ID

VARCHAR (20)

Charge party network cell ID (for example, mobile cell).

C_PARTY_CARRIER_CODE

NUMBER (8)

Carrier associated with the chargeable party.

C_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the destination of the chargeable party to the finest granularity supported by the system.

Higher forms of location such as country and continent can be derived from this value.

C_PARTY_ROUTE

VARCHAR (80)

Incoming or outgoing route of the chargeable party.

 

Transitory Details

Column Name

Properties

Description

FULL_PATH

VARCHAR (40)

Holds all available details that relate to the route that a call passed through (for example, X.25 DNIC sequence for call).

It can alternatively be used to store the direction or type of call (for example, incoming, outgoing, internal, and transit).

CASCADE_CARRIER_CODE

NUMBER (8)

Carrier closest to the current network used in the completion of the call (may not be known).

 

Miscellaneous Details

Column Name

Properties

Description

EVENT_SOURCE

VARCHAR (40)

Source of the event.

This is usually the same as the EVENT_SOURCE specified in the NORMALISED_EVENT_FILE table.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

It is populated automatically by the RGP but can be overridden by the assignment expressions.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

It is populated automatically by the RGP but can be overridden using the assignment expressions.

VOLUME

NUMBER

Volume of data associated with the call.

The units are specified depending on the call type (for example, bytes or segments).

If this column is populated by assignment expressions, its contents should be specified as volume per unit duration.

PULSES

NUMBER (8)

Number of meter pulses associated with call.

If this column is populated by assignment expressions, its contents should be specified as pulses per unit duration.

CHARGE

NUMBER (15, 6)

Charge value.

If populated by assignment expressions, its contents should be specified as charge per unit duration.

CURRENCY_ID

NUMBER (8)

Currency for the above charge.

RATE_BAND

VARCHAR (40)

Time or rate band calculated for this call.

SWITCH_START_DATE

DATE

Date-time of the event used to retrieve service details and evaluate any rating expressions.

In Bill Cycle Issue Date (BIDS mode), for:

  • Tariffs and one-off charges, it is set to the effective date of the bill run.
  • Adjustments, it is set to the effective date of the bill run that generated the event being adjusted.

In Rental Event Period Start date (REPS mode):

  • For tariffs, one-off charges, and debit adjustments, it is set to the period start date (PERIOD_START_DATE) of the event.
  • For credit adjustments, it is set to the period start date (PERIOD_START_DATE) of the event being adjusted.
  • If ROUND_ENTITY_HISTORY is set to Nearest Day or Full Day:
    • For credit adjustments, if SWITCH_START_DATE of the event being adjusted is not NULL, it is set to the SWITCH_START_DATE of the event being adjusted.
    • Otherwise, it is set to the period start date (PERIOD_START_DATE) before rounding is performed.

GENERAL_1 - GENERAL_20

VARCHAR (4000)

General fields 1 - 20.

Their use is controlled by the normalised event type.

The fields are usually used by the assignment expressions to store details on which the rental tariff charge calculation depends.
