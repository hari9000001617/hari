Description

Delete rows of data from a global derived attribute table. The function internally calls the biDerivedAttributeArrayUpdate&() function to perform the deletions.

Context

The following groups of variables are available throughout all stages of processing a global derived attribute delete rows request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value GDAASVR.
    • evLastRowInd& contains the value 0 during all stages of processing, except the evaluation of the global derived attribute pre-commit expressions for the last row processed.
    • evRowStatus$ contains the value Delete for the global derived attribute processing stages.
  • Derived attributes.

Parameters

Parameter

Description

derivedAttributeId&

ID of the derived attribute to delete rows from.

seqNr&[]

For standard derived attribute tables, it:

  • Contains sequence numbers of rows to delete.
  • Determines the order row deletions are processed.
  • Must identify global derived attribute table rows.

For extended derived attribute tables, it contains the IDs of the extended derived attribute table rows to delete.

effectiveDate~

Effective date used to identify the derived attribute history record for the derived attribute table that is updated.

lastModified~

Initial value must match the last modified date-time of the derived attribute table identified by derivedAttributeId& and effectiveDate~.

On successful return, the value is the date-time of the deletions.

tableName$

Name of the derived attribute table from which the rows are deleted.

effectiveStartDate~

Date on which the deletions first become effective.

If not defined, the current date is used.

effectiveEndDate~

Date on which the deletions cease to be effective.

If not defined, the end of time value is used.

fieldValues?[]

fieldNames?[]

fieldValues?[]

Global derived attribute table rows to delete, identified by index values.

Accepted forms are:

  • Array of hashes of data for one or more global derived attribute table rows (fieldValues?[])
  • Array of field names and an array of arrays of field values, containing data for one or more global derived attribute table rows (fieldNames?[], fieldValues?[])
  • Pair of arrays of arrays of data for one or more global derived attribute table rows (fieldNames?[], fieldValues?[]).

Global derived attribute array data must contain a complete row of indexes for the derived attribute table. Result values must not be specified.

If a row of data in this parameter does not match any rows in the database, an error is raised.

The data can be keyed according to the corresponding column name in the DERIVED_ATTRIBUTE_EVS_V view. When keyed according to database column name, the data passed must match the type and value of the data stored in the database.

If attribute validation is enabled, data can be keyed by interface or display label specified in the appropriate entity validation, or in core defaults.

When keyed by a label, the data must match the type and value expected as defined in the appropriate attribute type. Denormalised fields must pass the denormalised string values. Other fields must pass data of the type defined in the attribute type.

noTre&

Flag to invoke the function when in non-TRE mode.

 

Processing

Assuming that all processing stages are enabled, the following is the order of evaluation of expressions and constraints before deletion of the global derived attribute rows:

  1. Bundle direct variables if the call is nested.
  2. If seqNr&[] is not specified:
    1. For standard derived attributes, derive the sequence number for each row.
    2. For extended derived attributes, derive the extended derived attribute table row ID for each row.
  3. Decrypt any encrypted values and mask any appropriate values.
  4. Apply attribute type defaults and validation.
  5. Evaluate global derived attribute pre-commit expressions for each new row of data to be deleted.

On performing the deletions, the following sequence of actions occurs:

  1. Evaluate global derived attribute in-transaction expressions for each row of data deleted.
  2. Queue and evaluate global derived attribute post-commit expressions for each row of data deleted.

After deletion and evaluation of post-commit expressions:

  1. Unbundle variables that were stored in step 1 before deletion.

Returns

On success, the number of rows deleted is returned. If any of the deletions or any of the pre-delete validation steps fail, an error is raised.
