Description

Interface 1 is used by the STD to notify the client of a specified event, without waiting for an acknowledgement that notification was received.

Interface 2 is used for notifying a client of a task completion event. If the client notification was unsuccessful (possibly because the client has logged off), notification is sent using email.

Attributes

Application Environment

Parameter

Description

ClientCbInstance$

In: Convergent Billing instance from which client request was sent to STD.

ClientId$

In: ID of the client to send notification to, returned by treClientGetId$() or biStatusClient&.

NotificationName$

In: Type of notification to send (received as event name).

For information about the types of notifications, refer to the table below.

FieldValues?[]

In: Parameters to be sent to the client.

The first two elements in the array are an error ID (integer) and an error message (string).

The 'Negative Response to Task Change Command' and 'Response for Task Completion' events have the task queue ID as the third array element.

MailCommand$

In: Mail command to send email notification, if the client does not receive event notification.

MailAddress$

In: Address for email notification.

TaskId&

In: ID of the completed task.

TaskStatus&

In: Status code for the completed task.

ScheduleName$

In: Name of the completed task's schedule.

MessageId&

In: ID of the message for first line of email body.

 

Returns

1 (TRUE) if successful; otherwise, 0 (FALSE).

Remarks

For task completion events that do not require client notification, the STD uses biStdEmailNotify&() to perform email notification.

The following acknowledgments can be sent by the STD:

  • Response to task change command
  • Negative response to task change command
  • Acknowledgment of schedule change command
  • Response for task completion.

The following table lists for each type of notification:

  • Values for the NotificationName$ parameter, where the task ID or schedule ID substituted in for <Task_Queue_ID> and <Schedule_ID> respectively.
  • Value of the third array entry for FieldValues?[] parameter.
