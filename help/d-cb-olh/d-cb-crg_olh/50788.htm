PERSON_TRE_V

The PERSON_TRE_V view is used by functions defined for the biPerson service.

Column Definitions

Column Name

Properties

Description

PERSON_ID

NUMBER (15)

Not null

Read-only

ID of a person.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that the row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

It is set to the value of the StartDate~ parameter specified when inserting a new person.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

It defaults to the end of time when a person record is inserted.

CUSTOMER_PARTITION_NR

NUMBER (2)

Pseudo-column

Used for inserts only

If specified with a valid customer partition number for an insert, the person ID is derived using the sequence for that partition.

A maximum of 99 customer partitions are supported (1-99).

For information about how the customer partition number is determined if CUSTOMER_PARTITION_NR is not specified, refer to the biPersonInsert&() function in the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2).
