biProductOfferOfferUpdate&()

Syntax

biProductOfferOfferUpdate&(
    ProductOfferId&,
    var LastModified~,
    OpCodes&[],
    const FieldNames$[],
    FieldValues?[])

 

Description

Inserts, updates or deletes a child product offer.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

ProductOfferId&

In: ID of the product offer whose children will be updated.

LastModified~

In/Out: Last modified date-time of the product offer record.

OpCodes&[]

In: Operations to perform on the row.

FieldNames$[]

In: Names of field from the PRODUCT_OFFER_OFFER_TRE_V view for which values have been specified for the update.

FieldValues?[]

In: Rows of Field values, where:

  • Each row is specified in the same order specified by OpCodes&[].
  • Each field value is specified in the same order specified in FieldNames$[].

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

Validations for the following conditions are performed on insert and update operations:

  • CHILD_PRODUCT_OFFER_ID must be specified.
  • SEQNR is must be specified only for inserts.
  • Duplicates cannot exist for a child product offer associated with the same parent.
  • A product offer cannot reference itself from a child product offer, either directly or indirectly through expanded children.
  • Child product offer records must have PRODUCT_ID set to NULL.
  • Mandatory child product offers must have an AVAILABLE_START_DATE and AVAILABLE_END_DATE that covers the entire product offer's available date range.
  • Optional child product offers must have an AVAILABLE_START_DATE and AVAILABLE_END_DATE that covers at least part of the product offer's available date range.
  • If the parent has HIDDEN_IND_CODE set to 1, its associated child product offers must have MANDATORY_IND_CODE set to 1, and HIDDEN_IND_CODE set to 1.
  • If the parent has SERVICE_SPECIFIC_IND_CODE set to 1, its associated child product offers must also have SERVICE_SPECIFIC_IND_CODE set to 1.
  • If CHILD_PRODUCT_OFFER_MIN is set on the parent product offer, the number of optional child product offers must be greater than 0 or the specified minimum, whichever is largest.
  • Where a child product offer has a tenant, the TENANT_ID of the child product offer must be the same as the TENANT_ID of the parent product offer.
  • If MANDATORY_IND_CODE is:
    • 1, OFFER_MIN must be greater than or equal to 1, if specified.
    • NULL, the OFFER_MIN must be 0, if specified.
    • Not specified, it is derived as 1 if the OFFER_MIN is greater than or equal to 1, otherwise NULL.
  • OFFER_MAX must be greater than or equal to OFFER_MIN; otherwise, NULL to indicate no maximum.
  • If OFFER_MAX is greater than 1, the associated child offer COMPANION_PRODUCT_MIN must be greater than or equal to 1, or MANDATORY_IND_CODE for one of the companion products associated with the child offer must be set to 1.
  • If OFFER_MIN and OFFER_MAX are not specified, their values are derived from MANDATORY_IND_CODE as follows:
    • OFFER_MIN is derived as 1, if the MANDATORY_IND_CODE is 1, otherwise 0.
    • OFFER_MAX is derived as 1.
  • MANDATORY_IND_CODE and DEFAULT_IND_CODE cannot both be 1.
