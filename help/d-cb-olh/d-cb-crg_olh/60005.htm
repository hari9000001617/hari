SERVICE_CV

The SERVICE_CV view defines details of all services.

Column Definitions

Column Name

Properties

Description

SERVICE_ID

NUMBER (15)

ID of the service.

SERVICE_NAME

VARCHAR (64)

Name associated with the service in terms of the most exact specification of the service in relation to the customer.

The format of this value usually corresponds to how the service is to be displayed on customer invoices (for example, MSIDN number for a GSM service).

It must be unique for the specified date range.

NETWORK_NAME

VARCHAR (64)

Name associated with the service in terms of the most exact specification of the service in relation to the network, if this is different from SERVICE_NAME.

The format of this value usually corresponds to how the service is referenced by switching equipment, CDRs, and events generated (for example, IMSE number for a GSM service).

EFFECTIVE_START_DATE

DATE

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

OPERATOR_LOGIN_NAME

VARCHAR (30)

Billing system user who last modified the service.

SERVICE_TYPE

VARCHAR (30)

Type of this service.

SERVICE_SUBTYPE

VARCHAR (40)

Subtype of this service (for example, Dial-up and Dedicated subtypes within the service type of X.28).

CUSTOMER_NODE_ID

NUMBER (15)

ID of the customer hierarchy element or node that this service belongs to.

SERVICE_STATUS

VARCHAR (40)

Current status of the service (for example, Active, Suspended, and Cancelled).

SERVICE_PRODUCT_OFFER_NAME

VARCHAR (60)

Name of the product offer assigned to this service.

SERVICE_PRODUCT_OFFER_ALL_DATE

DATE

Date-time at which the service was allocated to the product offer designated in SERVICE_PRODUCT_OFFER_NAME.

SERVICE_PREV_PROD_OFFER_NAME

VARCHAR (60)

Name of the previous product offer assigned to this service.

CONTRACT_REFERENCE

VARCHAR (40)

Reference to the contract number of the contract stored outside of Convergent Billing.

PERSON_ID

NUMBER (15)

Primary contact person for this service.

A_ADDRESS_ID

NUMBER (15)

Reference to the address details for the A-end of the service.

B_ADDRESS_ID

NUMBER (15)

Reference to the address details for the B-end of the service.

If the service does not require the destinations of a start and termination point, this reference is null.

REQUIRED_BY_DATE

DATE

Date-time on which the current action as specified by SERVICE_STATUS_CODE is due to be completed.

PROVISIONED_DATE

DATE

Date-time that the service was provisioned for use by the exchange staff.

ACTIVE_DATE

DATE

Date-time that the service became billable to the customer.

GRADE_OF_SERVICE

VARCHAR (40)

Speed of the service.

SERVICE_USAGE

VARCHAR (40)

Main usage type for the service where different types of usage are allowed.

COMMS_TYPE

VARCHAR (40)

Appropriate communications signalling type classification (for example, Synchronous and Asynchronous).

GL_CODE_NAME

VARCHAR (64)

GL code used for all financial transactions produced as part of this service.

PRODUCT_INSTANCE_ID

NUMBER (15)

Base product instance associated with the service.
