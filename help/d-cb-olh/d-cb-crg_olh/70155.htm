fCMN_TREProductInstanceCreateByOffer_R&()

Syntax

fCMN_TREProductInstanceCreateByOffer_R&(
    l_OfferStruct?{},
    l_ProdInst?{})

 

Description

Creates a product instance and related required product instance entities using product offer functionality.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_OfferStruct?{}

In: Hash containing the hierarchical product offer structure.

For more information about the parameter hash keys, refer to Remarks below.

l_ProdInst?{}

In/Out: Hash containing the values for all mandatory and optional fields for the product instance and related entities.

For more information about the parameter hash keys, refer to Remarks below.

 

Returns

New product instance ID, if successful; otherwise, an error is raised.

Remarks

The high-level processing is:

  1. Validate the top-level offer.
  2. Derive the base product ID.
  3. Process the product offer structure recursively:
    1. Derive the mandatory minimum companion product instances, with their product offer node IDs.
    2. Generate IDs of the product offer instances for companion products if the associated product offer is to be sold more than once.
    3. Assign parent product offer instance IDs for all companion products associated with the child offers of the set product offers.
    4. Confirm non-mandatory companion product instances specified on the node.
    5. If available, add non-mandatory companion product instances to the product instance hash.
  4. Confirm services required for service-specific product offers.
  5. If available, add service names for the associated service-specific companion products.
  6. Combine all derived companion products to the product instance hash.
  7. Validate service details.
  8. Combine the service details with companion products.
  9. Create the product instances and services.

Multiple Product Offer Instances

To create multiple service-specific offer instances of the same type, SERVICE_NAME must be specified for each service-specific offer on the highest level in l_OfferStruct?{} as required.

If the product offer requires multiple instances of a mandatory child offer, all instances of the mandatory child offer must be specified in l_OfferStruct?{}.

Default Product Offer Instances

Default product offers and companion products are not automatically included, and must be specified in the input offer structure if required.

An error is raised if:

  • A specified product offer within the offer structure is not available.
  • The top-level offer is not associated with a base product.
  • The ID of the product offer instance or parent product offer instance is specified in l_OfferStruct?{}.
  • A product offer is specified to be sold more than its maximum allowed.
  • A service is assigned to two separate product offers.
  • A service is not specified in both l_OfferStruct?{} and l_ProdInst?{}.

Data Type

{'PRODUCT_OFFER_SKU'}

[<ROW_NUMBER>]
{'<PRODUCT_INSTANCE_
TRE_V>'}

Various

Companion products for the product instance.

The companion products must be defined as compatible products in the product definition.

Only specify optional companion products. Mandatory companion products are automatically specified.

[<ROW_NUMBER>] is an array (0..n) of companion products.

Refer to PRODUCT_INSTANCE_TRE_V view for details of mandatory and optional fields. The data type of the hash entries correspond to the attribute types in the entity validation for a product.

[<ROW_NUMBER>]
{<OFFER_NODE>}

Various

Array of hashes containing each child offer node.

Child offers can contain a CHILD_PRODUCT OFFERS array representing the offer hierarchy. However, the leaf node in a offer hierarchy must only contain companion product instances.
