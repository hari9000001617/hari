ENTITLEMENT_TRE_V

The ENTITLEMENT_TRE_V view defines each current entitlement, including the entities associated to entitlements over time. Depending on the context, an entitlement is associated with a root customer node, child customer node, or service.

This view also records other details that are dependent on the entity with which the entitlement is associated.

Some of the columns in this view can be updated to have different values for different effective date-ranges. These are identified below. The remaining columns are not date-ranged.

PARTITION_NR

ENTITLEMENT_CATEGORY_

NUMBER (8)

Not null

Read-only

Reference code of the category to which the entitlement belongs.

It cannot be specified for updates.

It is based on the ENTITLEMENT_CATEGORY reference type.

ENTITLEMENT_TYPE_ID

NUMBER (8)

Not null

Read-only

ID of the entitlement type.

If not specified, it defaults from the EntitlementCategory derived attribute table when the entitlement is created.

It cannot be specified for updates.

ENTITLEMENT_TYPE_NAME

VARCHAR (30)

Not null

Read-only

Name of the entitlement type.

If not specified, it defaults from the EntitlementCategory derived attribute table when the entitlement is created.

It cannot be specified for updates.

ENTITLEMENT_TYPE_
DISPLAY_NAME

VARCHAR (60)

Not null

Read-only

Name displayed to customer service representatives.

Caution:

It must not be referenced by configuration code, because it may undergo language translation.

START_DATE

DATE

Not null

Date-time the entitlement became valid.

It cannot be specified for updates.

EXPIRY_DATE

DATE

Not null

Date-time after which the entitlement is no longer valid.

Value must be either:

  • Equal to or later than START_DATE.
  • One second earlier than START_DATE, to support back-dated cancellations.

If EXPIRY_DATE precedes START_DATE, new allocations are not created for the entitlement.

If the value equals the sunset date and time, the entitlement never expires.

INITIAL_ALLOCATION_
AMOUNT

NUMBER (22,6)

Initial amount to be allocated to the entitlement.

It must contain a value if ENTITLEMENT_CLASS_CODE is set to One-off.

It must be null if ENTITLEMENT_CLASS_CODE is set to Dynamic.

It must not be less than zero.

RECURRING_ALLOCATION_
AMOUNT

NUMBER (22,6)

Amount to be allocated at each repetition of the entitlement.

It must contain a value if ENTITLEMENT_CLASS_CODE is set to Recurring.

It must not be less than zero.

SUSPENDED_IND_CODE

NUMBER (1)

Set to 1 (True) when the entitlement is suspended.

Any existing allocations for the entitlement remain current, but any scheduled allocations are cancelled.

It must equal be set to 1 if not null.

If set to 1, ENTITLEMENT_CLASS_CODE cannot be set to One-off.

AUTO_ALLOC_FAILURE_IND_
CODE

NUMBER (1)

When set to 1 (True), an automatic allocation to the entitlement has failed, and no further automatic allocations will be attempted.

If not set to 1, it must have a value of NULL.

It cannot be specified for inserts.

NEXT_ALLOCATION_DATE

DATE

 

Start date of the next allocation to be created for the entitlement.

It must be NULL if SUSPENDED_IND_CODE is set to 1.

If it is not NULL, its value must be between START_DATE and EXPIRY_DATE inclusive.

It cannot be specified for inserts.

If specified as part of an update, the next allocation for this entitlement will be updated to occur at this date. If there is a scheduled function in place to create the next allocation, it is updated to reflect the changed next allocation date.

Specifying a value of NULL as part of an update disables the scheduling of the next allocation. If there is currently a next allocation scheduled, it is deleted.

If a non-NULL value is specified as part of an update, the entitlement cannot be aligned with the bill cycle of a customer.

LAST_ALLOCATION_DATE

DATE

Read-only

Start date of the last allocation that was created for the entitlement.

It contains a value only when ENTITLEMENT_CLASS_CODE is set to Recurring.

NEXT_ALLOC_SCHED_FUNC_ID

NUMBER (15)

ID of the scheduled function to be evaluated to create the next allocation for the entitlement.

It must be null if ENTITLEMENT_CLASS_CODE is not set to Recurring, or SUSPENDED_IND_CODE is set to 1.

For a recurring entitlement, it is automatically derived and cannot be set to a value. However, it can be set to null, which:

  • Deletes the scheduled function for the next allocation (if any).
  • Prevents the use of scheduled functions to create future allocations for this entitlement.
  • Causes subsequent allocations to be automatically created on demand during rating whenever this entitlement is accessed.

Caution:

Creating allocations during rating can negatively impact rating performance and is generally not recommended.

WARNING_SCHED_FUNC_ID

NUMBER (15)

Read-only

ID of the scheduled function to be evaluated when the entitlement is about to expire.

It contains a value only when WARNING_FUNCTION_DEFN_ID is not null.

EXPIRY_SCHED_FUNC_ID

NUMBER (15)

Read-only

ID of the scheduled function to be evaluated on expiry of the entitlement.

It contains a value only when EXPIRY_FUNCTION_DEFN_ID is not null.

NEXT_TASK_ID

NUMBER (15)

Read-only

ID of the task for the next bill run to be executed for the customer associated with the entitlement.

It contains a value only when entitlement is aligned with customer bill cycles.

The effective date-time of the task defines the START_DATE of the next allocation to be made to the entitlement.

RESERVATION

VARCHAR (40)

ID of the current reservation of an amount from an allocation of this entitlement.

It cannot be specified for inserts.

CHARGE_STATUS_CODE

NUMBER (2)

Current real-time charging status of this entitlement.

It is based on the ENTITLEMENT_CHARGE_STATUS reference type.

It cannot be specified on insert.

CHARGED_IND_CODE

NUMBER (1)

Indicates whether real-time charges have been generated for this entitlement.

If set to 1 (True), at least one real-time charge has been generated for this entitlement.

It must be NULL if CHARGE_STATUS_CODE is NULL. If not NULL, it must be 1.

It cannot be specified for inserts or updates.

FAIL_STATUS_UPDATE_DUE_
DATE

DATE

Date-time when the status of the companion product instance associated with the entitlement is due to be updated, following a reservation failure.

It contains a value only if DELAY_PERIOD is null and CHARGE_STATUS_CODE is set to 2 or 4.

It cannot be specified on insert.

FAIL_STATUS_SCHED_FUNC_
ID

NUMBER (15)

ID of the scheduled function used to perform the delayed update of the status of the companion product instance associated with the entitlement.

It contains a value only if FAIL_STATUS_UPDATE_DUE_DATE is not null.

It cannot be specified on insert.

Properties

NUMBER (6)

Duration of the advance period (seconds) for the entitlement.

ALLOC_GEN_FUNCTION_DEFN_

ALL_FIRST_USE_FUNCTION_

WARNING_ALLOC_FUNCTION_DEFN_ID

EXPIRY_FUNCTION_DEFN_ID

EXPIRY_ALLOC_FUNCTION_DEFN_ID

REAL_TIME_CANCEL_TARIFF_

Entitlement Category Columns

The following table lists the columns that are defaulted from the EntitlementCategory derived attribute table when entitlements are created.

 

Column Name

Properties

Description

UNIT_OF_MEASURE_CODE

NUMBER (2)

Not null

Units in which allocations to the entitlement are measured.

It is based on the ENTITLEMENT_UNIT_OF_MEASURE reference type.

ENTITLEMENT_CLASS_CODE

NUMBER (2)

Not null

A general classification of the entitlement. Valid values are:

  • 1 (Recurring)
  • 2 (One-off)
  • 3 (Dynamic).

It is based on the reference type ENTITLEMENT_CLASS reference type.

RECURRING_PERIOD

NUMBER (4)

Period of time between each repetition of the entitlement.

Value must and only be supplied if the ENTITLEMENT_CLASS_CODE is set to Recurring.

When specified, the value must be greater than zero.

RECURRING_PERIOD_UNIT_
CODE

NUMBER (1)

Units for time period contained in RECURRING_PERIOD.

If not null, units must be specified. If BILL_CYCLE_IND_CODE is set to 1, must be set to 0 (Bill Cycles).

It is based on the ENTITLEMENT_PERIOD_UNIT reference type.

VALIDITY_PERIOD

NUMBER (4)

Time period that each allocation to the entitlement remains valid.

When specified, the value must be greater than zero.

VALIDITY_PERIOD_UNIT_
CODE

NUMBER (1)

Units for the time period in VALIDITY_PERIOD.

If not null, the units must be specified.

It is based on the ENTITLEMENT_PERIOD_UNIT reference type.

WARNING_PERIOD

NUMBER (4)

Time period duration of the advanced warning of entitlement expiring.

The function identified by WARNING_FUNCTION_DEFN_ID is evaluated in advance of the expiry date-time.

Value cannot be less than zero.

WARNING_PERIOD_UNIT_
CODE

NUMBER (1)

Units for the time period contained in WARNING_PERIOD.

Must contain a value if WARNING_PERIOD is not null.

It is based on the ENTITLEMENT_WARNING_PERIOD_UNIT reference type.

WARNING_ALLOC_PERIOD

NUMBER (4)

Duration of the advance warning period before the entitlement allocation expires.

The function identified by WARNING_ALLOC_FUNCTION_DEFN_ID is evaluated at the start of this period.

Value cannot be less than zero.

WARNING_ALLOC_PERIOD_UNIT_
CODE

NUMBER (1)

Units for the advance warning period specified in WARNING_ALLOC_PERIOD.

Must contain a value if WARNING_ALLOC_PERIOD is not null.

It is based on the ENTITLEMENT_WARNING_PERIOD_UNIT reference type.

EXPIRE_ON_SUSPENSION_
IND_CODE

NUMBER (1)

Default effect on existing allocations for the entitlement if the entitlement is suspended.

It must have a value of 1 if not null.

When the value is set to 1, any existing allocations are expired when the scheduled allocations are cancelled.

When the value is null, existing allocations are unchanged.

CONSUMPTION_PRIORITY

NUMBER (2)

Not Null

Priority that determines the order in which allocations are consumed.

Its value must be greater than or equal to 1.

Lower numerical value is interpreted as a higher priority. 1 is the highest priority.

DELAY_PERIOD

NUMBER (4)

Period of time to delay an update to the status of a companion product instance, following a failure to reserve an allocation for this entitlement during the evaluation of a real-time recurring tariff.

Value must be greater than zero.

DELAY_PERIOD_UNIT_CODE

NUMBER (1)

Units for the time period in DELAY_PERIOD.

It must contain a value if DELAY_PERIOD is not null.

It is based on the ENTITLEMENT_DELAY_PERIOD_UNITS reference type.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General purpose fields.

Use of the general fields is controlled by the entitlement type, by way of entity validation.

 

Date-ranged Columns

The value of the following columns may vary over different date ranges.

 

Column Name

Properties

Description

EFFECTIVE_START_DATE

DATE

Not null

Date-time the entitlement became associated with the entity.

EFFECTIVE_END_DATE

DATE

Not null

Date-time one second before the entitlement ceased to be associated with the entity.

It must be later than or equal to EFFECTIVE_START_DATE.

SERVICE_ID

NUMBER (15)

ID for a service associated to the entitlement during the effective date range.

It must contain a value if the entitlement has a context of Service.

PRODUCT_INSTANCE_ID

NUMBER (15)

ID of a companion product instance that caused the entitlement to be created.

It must contain a value if the entitlement was created as a result of a product offer.

CUSTOMER_NODE_ID

NUMBER (15)

ID of the customer node associated to the entitlement for the effective date range.

It must contain a value if the entitlement has a context of Customer Node or Customer.

If the context is Customer, the customer must be a root customer node at all times during the effective date range.

ACCOUNT_ID

NUMBER (15)

ID of the account, to check during the effective date range, to determine when to reactivate the entitlement.

The specified account may not be the account charged for this entitlement.

SUB_CATEGORY

VARCHAR (255)

Optional.

Sub-category to assist in classifying the entitlement in the effective date range.
