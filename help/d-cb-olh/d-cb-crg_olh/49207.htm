treVariantCallAsync

C

TRE_RESULT treVariantCallASync(
    char *FnName,
    jtime_t EffectiveDate,
    INTSV Flags,
    INTSV NrParams,
    TRE_VARIANT *Arglist,
    double Timeout,
    INTSV *RequestId)

 

Object Pascal

Not applicable.

 

Perl

Not applicable.

 

Purpose

Provides a primary interface to invoking remote functions with the TRE asynchronously.

Parameters

Parameter

Description

FnName

In: Name of the remote function to call.

EffectiveDate

In: Effective date to use when retrieving the remote function definition.

Flags

In: Specifies how the function is called. Valid values are:

  • TRE_NOREPLY

    No reply sent, even if the function normally sends a reply.

  • TRE_NOBLOCK

    Do not block if the number of outstanding requests exceeds the maximum supported.

NrParams

In: Number of parameters required by the remote function.

Arglist

In: Array of parameter values passed to the function.

Timeout

In: Maximum time (in seconds) to wait for a response before cancelling it.

Timeout is only used if the TRE_NOREPLY flag is not set.

RequestId

Out: ID of the request.

If a response is expected, RequestId can be passed to treVariantGetReply() to retrieve the response. Otherwise, RequestId is set to 0.

 

Returns

Value

Cause

TRE_SUCC

The function was successful.

TRE_FAIL

A generic failure occurred (but not one of the other specific errors).

The TRE sets treErrno to indicate the type of error.

If TRE_NOBLOCK was set and a blocking condition existed, treErrno is set to TRE_WOULD_BLOCK.

TRE_NOTREGISTERED

The specified function is not registered for the specified date.

TRE_TIMEOUT

A blocking condition existed, TRE_NOBLOCK was not set, and the call could not be sent before its timeout expired.

 

Remarks

The value of EffectiveDate is ignored if this function call is part of a transaction, or treSetEffectiveDate() is called outside of a transaction.

The TRE_NOREPLY flag should be set if a response is not required or expected. If not set, RequestId is set and can be used to retrieve the response using treVariantGetReply(). Otherwise, RequestId is set to 0.

If the TRE_NOBLOCK flag is not set and the number of outstanding requests is equal to the maximum supported for a connection (50), calls to treVariantCallAsync() block until the number of outstanding requests drops below the maximum.

The passed function must have been cached by calling treFunctionCache(), or the TRE_NOTREGISTERED error is returned.

Usage for treVariantCallASync() is similar to treVariantCall().

Responses to asynchronous requests can be cancelled by calling treCancelReply().
