INV_CUSTOMER_ADJUSTMENT_V

The INV_CUSTOMER_ADJUSTMENT_V view displays details for credit and debit notes. The view is a combination of customer node and adjustment details. Only details for applied credit and debit notes can be retrieved.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_ID

NUMBER (15)

Not null

ID of an adjustment.

ACCOUNT_ID

NUMBER (15)

Not null

Account against which this adjustment was made.

LAST_MODIFIED

DATE

Not null

Date-time that the record was last modified.

ADJUSTMENT_NR

VARCHAR (30)

Not null

ID of the adjustment that can be referenced by users.

It is automatically generated, based on the ADJUSTMENT_FORMAT_EXPR of the adjustment type and the ADJUSTMENT_ID.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Type of adjustment.

If the adjustment is associated with a batch, it defaults to the adjustment type of the batch.

ADJUSTMENT_TYPE_DISPLAY_
NAME

VARCHAR (40)

Not null

Name of the adjustment type displayed.

It defaults to ADJUSTMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

Type of adjustment.

If the adjustment is associated with a batch, this adjustment type defaults to the adjustment type of the batch.

DESCRIPTION

VARCHAR (255)

Not null

Description of the adjustment.

It is not displayed on the invoice.

If this adjustment is associated with a batch, this defaults to the description of the batch.

ADJUSTMENT_TEXT

VARCHAR (80)

Not null

Text displayed on the customer invoice in the summary section with the details of the invoice adjustment.

It defaults to the text associated with the adjustment type.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User associated with creating adjustment.

AMOUNT

NUMBER (19,3)

Not null

Amount of the adjustment in the currency of the account.

A positive adjustment indicates a credit to the customer that must be included on the invoice.

A negative amount indicates a debit.

ADJUSTMENT_AMOUNT

NUMBER (19,3)

Not null

Original adjustment amount in the currency of the adjustment.

It is usually equal to AMOUNT, but the amount may differ if the adjustment was made in a different currency to the account.

ADJUSTMENT_CURRENCY_ID

NUMBER (8)

Not null

ID of the currency in which the adjustment was made.

ADJUSTMENT_DATE

DATE

Not null

Date-time that the adjustment was made.

ADJUSTMENT_RECEIVED_DATE

DATE

Not null

Date-time that the adjustment was applied to the account.

It allows adjustments to be backdated .

ADJUSTMENT_DUE_DATE

DATE

Date-time at which the adjustment was due.

It is usually only set for adjustments representing debit notes.

ADJUSTMENT_INVOICE_CODE

NUMBER (2)

Not null

Derived

Summary of the details in the ADJUSTMENT_INVOICE table for the TO_ACCOUNT_ID for this adjustment.

Valid values are:

  • 0 (Invoice): The adjustment is against a single invoice.
  • 1 (Split): The adjustment is against one or more invoices, adjustments, payments, and/or a direct credit or debit to the account.
  • 2 (Unallocated): The adjustment is a direct credit or debit to the account.
  • 3 (Adjustment): The adjustment is against another adjustment (for example, where a debit note is allocated to a credit note).
  • 4 (Payment): The adjustment is against a payment (for example, where a debit note is allocated to a payment).

ADJUSTMENT_INVOICE_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is Invoice (0), it is the ID of the single invoice in the ADJUSTMENT_INVOICE table for the adjustment.

CUSTOMER_INVOICE_STR

VARCHAR (30)

External ID of the invoice specified in ADJUSTMENT_INVOICE_ID.

It is set only when ADJUSTMENT_INVOICE_ID is not null.

ALLOCATED_ADJUSTMENT_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is Adjustment (3), it is the ID of the single adjustment in the ADJUSTMENT_INVOICE table for this adjustment.

ALLOCATED_ADJUSTMENT_NR

VARCHAR (30)

ID of the adjustment for which this adjustment was made (or example, adjustment receipt number for a debit or credit note).

It is populated only when ALLOCATED_ADJUSTMENT_ID is not null.

PAYMENT_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is Payment (4), it is the ID of the single payment in the ADJUSTMENT_INVOICE table for this adjustment.

PAYMENT_RECEIPT_NR

VARCHAR (30)

ID (for example, the customer payment receipt number) for the payment allocated to this adjustment.

It is populated only when PAYMENT_ID is not null.

ADJUSTMENT_GENERAL_1 -
ADJUSTMENT_GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be defined in the type definition table.

The constraints for each field are defined by the entity validation associated with their ADJUSTMENT_TYPE.

INVOICE_FORMAT_ID

NUMBER (10)

Optional reference to the invoice format.

It is mandatory if PENDING_GENERATION_IND_CODE or IMAGE_GENERATED_IND_CODE is specified.

It is used for credit or debit note adjustments that need to be sent to customers independently of the standard invoice cycle.

PENDING_GENERATION_IND_
CODE

NUMBER (1)

If set to TRUE (1), the adjustment is awaiting adjustment image generation (or regeneration).

IMAGE_GENERATED_IND_
CODE

NUMBER (1)

If set to TRUE (1), an image is generated for the adjustment.

PENDING_PRINT_IND_CODE

NUMBER (1)

If set to TRUE (1), the image for the adjustment is awaiting printing (or reprinting).

PRINTED_IND_CODE

NUMBER (1)

If set to TRUE (1), the image for the adjustment is printed (or emailed).

When the image is printed (or reprinted), it is automatically set to 1.

It cannot be specified as part of an Insert or Update operation.

STORED_IMAGE_TYPE_
CODE

NUMBER (2)

Type of image stored in the ADJUSTMENT table, derived from the value stored in INVOICE_FORMAT_ID at image creation.
