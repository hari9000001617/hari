Description

Generates XML schema for a specified namespace that has all its external definitions validated.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

ExternalDefnNamespaceId&

In: ID of the external definition namespace for which the external definition XML schema is generated.

ExternalDefnNamespaceName$

In: Name of the external definition namespace for which the external definition XML schema is generated.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

If a schema has previously been generated for the specified namespace, the previously-generated file is renamed with the current date-time as the extension, in the format <filename>.yyyymmddhhnnss.

The name of the XML schema file is derived from the value of NAMESPACE in the EXTERNAL_DEFN_NAMESPACE_TRE_V table row as follows:

  1. Remove all trailing forward slashes (/) from NAMESPACE, if required.
  2. From the resulting value, remove all characters before the last forward slash. If no forward slashes remain, use the full value from step 1.
  3. Add the .xsd extension.

For example, if NAMESPACE = /somedir1/somedir2/somename1/, the name of the XML schema file is derived as follows:

  1. /somedir1/somedir2/somename1
  2. somename1
  3. somename1.xsd

If NAMESPACE = somenameNoslashes, the name of the XML schema file is somenameNoslashes.xsd.

If the Singleview namespace has unqualified elements, then the elementFormDefault for the schema is set to unqualified, otherwise it is set to qualified.

The location of the XML schema file is derived from the value of SCHEMA_LOCATION in the EXTERNAL_DEFN_NAMESPACE_TRE_V table.

If SCHEMA_LOCATION is not specified, the location is derived from the value of the $ATA_DATA_SERVER_CONFIG variable and the xsd subdirectory. For example, if $ATA_DATA_SERVER_CONFIG = /home/dir1/config/, the location of the XML schema file is /home/dir1/config/xsd.

If the value of SCHEMA_LOCATION is specified with environment variables, the environment variables are substituted by their value. For example, if SCHEMA_LOCATION = $SOME_ENV_VAR/mydir/ and $SOME_ENV_VAR = /home/somedir, the location of the file XML schema file is /home/somedir/mydir.
