TARIFF_TRE_V

The TARIFF_TRE_V view contains details of all tariffs, including discounts and taxes.

Column Definitions

Column Name

Properties

Description

TARIFF_ID

NUMBER (8)

Not null

ID of the tariff.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

Not null

Name of the tariff.

VARCHAR (30)

Version ID used to tag the version of this entity.

DEFAULT_PRIORITY

NUMBER (4)

Default priority of this tariff when used in a product definition.

It is used by the Product Definition form to default the priority associated with this tariff.

It is not used for any other purpose.

If specified, the default priority must be greater than or equal to 1.

MASTER_TARIFF_ID

NUMBER (8)

Tariff with which this tariff shares details in the TARIFF_CONDITION_DEFN, TARIFF_CHARGE_DEFN, and TARIFF_CHARGE tables.

The following restrictions apply:

  • A tariff can only be a master tariff if it has a non-null CHARGE_EXPR.
  • NR_CONDITIONS and NR_CHARGE_COMPONENTS must be identical between a tariff and its master.
  • The tariff must have a CONTEXT_CODE and an APPLICATION_ENV_CODE greater than or equal to those of its master. A tariff that is the master of one or more other tariffs must have a CONTEXT_CODE and APPLICATION_ENV_CODE less than or equal to any of its children.
  • Only new tariffs can be assigned a master tariff. Existing tariffs cannot be updated to have a master tariff.
  • For the purposes of date-ranging, a tariff with a master tariff is kept in synchronisation with its master record, and any other tariffs for which it is a master. That is, the start and end dates for each record with a master tariff must be exactly the same as the start and end dates of the master tariff.
  • A tariff that has a non-null MASTER_TARIFF_ID cannot have any rows associated with it in the TARIFF_CONDITION_DEFN, TARIFF_CHARGE_DEFN, and TARIFF_CHARGE tables.

BILLABLE_IND_CODE

NUMBER (1)

Not null

If set to 1 (True), this tariff generates billable charges.

DONT_DISPLAY_IND_CODE

NUMBER (1)

If set to 1 (True), the values of this tariff are not included in the Invoice Item Detail form by default.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which this specification is used. Valid values are:

  • Rating environment
    • 1 (Normalised Event)
    • 2 (Service)
  • Billing environment
    • 1 (Normalised Event)
    • 2 (Service)
    • 3 (Customer Node)
    • 4 (Customer)
    • 5 (Charge)
  • TRE Table environment
    • 0 (Any).

CONTEXT_RATING, CONTEXT_BILLING, or CONTEXT_TRE_TABLE reference types define the possible values depending on the value of APPLICATION_ENV_CODE.

APPLICATION_ENV_CODE

NUMBER (2)

Not null

Application environment of this tariff.

It is based on the APPLICATION_TARIFF reference type.

NR_CONDITIONS

NUMBER (2)

Not null

Number of conditions associated with identifying the correct cost structure within this tariff, and must be between 0 and 10.

NR_CHARGE_COMPONENTS

NUMBER (2)

Not null

Number of charge components/expressions used to calculate the overall charge for this tariff, and must be between 0 and 20.

KEY_SUBSTITION_EXPR

VARCHAR (255)

Expression defining values to be substituted into the key variable.

It must be null if the tariff has a data type of Real.

If the data type is Real Array, it is mandatory. It defines the number of values substituted into KEY_VARIABLE_NAME.

If the data type is Real Hash, it is mandatory. It defines the string values to be substituted in KEY_VARIABLE_NAME.

KEY_VARIABLE_NAME

VARCHAR (66)

Name of local expression variable used in tariff expressions.

It can only, and must, contain a value if the data type is Real Array or Real Hash.

The variable name must start with the @ character, indicating that it is a local expression variable. It must be terminated by the appropriate type suffix:

  • & if the tariff is of type Real Array
  • $ if the tariff is of type Real Hash.

ELIGIBILITY_EXPRESSION_
LIST_ID

NUMBER (8)

List of conditional expressions that must be satisfied for this tariff to be evaluated.

If any of the conditions fail, the tariff is not applied.

If any of the expressions reference the key variable, all eligibility expressions are evaluated for each array index or key value as specified by KEY_SUBSTITUTION_EXPR.

If none of the expressions reference the key variable, the eligibility expression list is evaluated once only, before evaluating KEY_SUBSTITUTION_EXPR.

DEFAULT_CHARGE_EXPR

VARCHAR (255)

Not null

Default charge for this tariff if none of the conditional expressions match, or NR_CONDITIONS is zero.

CHARGE_EXPR

VARCHAR (255)

Expression for the charges associated with this tariff.

DEFAULT_INVOICE_TXT

VARCHAR (255)

Text that will appear for each charge created.

It is overridden by INVOICE_TXT in the TARIFF_CHARGE table.

CURRENCY_ID

NUMBER (8)

Currency of this subtotal.

A currency can only be specified if the data type is Real, Real Array, or Real Hash, and DONT_STORE_IND_CODE and CURRENCY_EXPR are null, and the APPLICATION_ENV_CODE is Billing.

The currency must exist for at least this date range.

CURRENCY_EXPR

VARCHAR (255)

Expression that defines the currency to be used for any charges generated by this subtotal.

It must return an integer value corresponding to a valid internal currency ID. An error is raised if a negative or undefined value is returned.

It may reference the key variable if it is defined. If it references the key variable, it is evaluated for each eligible key variable value. Therefore, different components of a tariff can be in different currencies.

If the key variable is not referenced, it is evaluated only once, and the same value is used for all key values.

SUPPRESS_ZERO_INC_CODE

NUMBER (1)

Not null

If set to 1 (True), the charge is calculated to be zero and no entry is recorded.

GLOBAL_EVENT_APPLY_IND_
CODE

NUMBER (1)

Not null

If set to 1 (True), it is applied to all events in the specified context and application environment.

All calculated charges are applied to the account(s) specified by the TO_ACCOUNT_ID (or FR_ACCOUNT_ID) of the charge category.

The account(s) may be overridden by the optional TO_ACCOUNT_ID_EXPR and FROM_ACCOUNT_ID_EXPR expressions.

SERVICE_NOT_FOUND_IND_
CODE

NUMBER (1)

If set to 1 (True), it is only applied if the service specified by C_PARTY_ID is not found in the database.

It can only be set to 1 (True) if CONTEXT_CODE is Normalised Event, APPLICATION_ENV_CODE is Rating, and GLOBAL_EVENT_APPLY_IND_CODE is set to 1.

CHARGE_CATEGORY_ID

NUMBER (1)

Charge category to which all charges associated with this tariff are applied.

It defines both account and general ledger details for the charge.

TO_ACCOUNT_ID_EXPR

VARCHAR (255)

Expression defining the To account for charges generated by this tariff.

It must return an integer value corresponding to a valid internal account ID.

FROM_ACCOUNT_ID_EXPR

VARCHAR (255)

Expression defining the From account for charges generated by this tariff.

It must return an integer value corresponding to a valid internal account ID.

TO_GL_CODE_ID

VARCHAR (255)

Expression defining the To GL code used for charges generated by this tariff.

It must return an integer value corresponding to a valid GL code ID.

FROM_GL_CODE_ID

VARCHAR (255)

Expression defining the From GL code for charges generated by this tariff.

It must return an integer value corresponding to a valid GL code ID.

RECEIVABLE_TYPE_ID

NUMBER (8)

Receivable type associated with all charges for this tariff.

The receivable type must exist for at least the date range of this tariff.

If specified, RECEIVABLE_TYPE_EXPR must be null.

RECEIVABLE_TYPE_EXPR

VARCHAR (255)

Defines the receivable type for any charges generated by this tariff.

If specified, RECEIVABLE_TYPE_ID must be null.

It must return an integer value corresponding to a receivable type ID.

If referencing the key variable, it is evaluated for each eligible key variable value. Otherwise, it is evaluated only once, and the same value is used for all key values.

INVOICE_SECTION_NR

NUMBER (8)

Not null

Invoice section number assigned to all charges generated by this tariff.

ASSIGNMENT_EXPRESSION_
LIST_ID

NUMBER (8)

List of assignment expressions to be evaluated for assigning values to direct variables associated with the CHARGE_V view.

Note:

Tariff assignment expressions are evaluated after the value of the tariff is calculated. Therefore, if the context of the tariff is not Charge, the charge expression cannot read the values of the direct variables associated with the CHARGE_INPUT_V view.

The calculated value of the tariff is also accessible in these expressions using its variable name. If these expressions reference the tariff, this reference is not included as a variable dependency.

Assignment expressions are evaluated before charge category proportioning to accounts is performed. General Ledger code, account details, and the charge proportion and the final charge value are not accessible.

If the context of the tariff is Charge (that is, the tariff is a billing tariff), the charge expression can access direct variables associated with the CHARGE_INPUT_V view. However, these variables contain values for the charge being processed, not the values calculated for the charge being generated.

These expressions may reference the key variable if it is defined. If any of them reference the key variable, they are evaluated for each eligible key variable value. Otherwise, they are evaluated only once for the tariff.

ACCOUNT_AGGREGATE_IND_
CODE

NUMBER (1)

If set to 1 (True), values calculated by this tariff are aggregated to the ACCOUNT.UNBILLED_AMOUNT column during rating. Otherwise, it is null.

It can only have a value of 1 if APPLICATION_ENV_CODE is Rating.

ACCOUNT_CHECK_EXPR

VARCHAR (255)

Expression that checks the balance of the account during rating.

It is evaluated after the charge is calculated, charge assignment expressions evaluated, and associated account(s) updated. It is evaluated for each charge proportion for each account for each key value (if applicable), after the charge has been aggregated to the account.

Direct variables available to the expression are the same as those for CHARGE_EXPR, with the addition of variables associated with the CHARGE_V and ACCOUNT_V views.

The key variable is not accessible in this expression. However, the value used to generate the charge (if any) is available as a direct variable in the CHARGE_V view.

DO_NOT_FIX_DEPENDENCIES

NUMBER (1)

Pseudo-column

Used for updates only

If specified with a 1 (TRUE), variable dependencies are not fixed.

It is used for multi-phase tariff updates to delay fixing of variable dependencies.
