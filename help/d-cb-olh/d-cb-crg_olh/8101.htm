Database Views

This section describes views defined in the Convergent Billing database for reports.

The following diagram displays the relationship between the entities accessed by the views.

 

Convergent Billing Views Entity Model

 

The following table lists the views provided. Full details of each view are provided in the sections after the table.

 

Database View for Reports

View Name

Description

ACCOUNT_CV

Displays account transaction history for each account in the system.

ADDRESS_CV

Displays historical and current details of all known addresses in the system.

ADJUSTMENT_CV

Displays adjustments that have been applied to accounts.

CALL_DATA_CV

Stores details on normalised events.

A normalised event may correspond to reconciled or unreconciled call data.

CALL_DATA_ERROR_CV

Stores details on normalised events (call data) that could not be processed.

CHARGE_CV

Displays details on all charges.

CURRENCY_CONVERSION_CV

Displays currency conversion rates used in the system.

CURRENCY_CV

Defines currencies known to the system.

CUSTOMER_CV

Defines customer node details.

CUSTOMER_QUERY_CV

Defines details relating to a customer query.

EQUIPMENT_CV

Displays details of equipment held in the Convergent Billing equipment inventory.

PAYMENT_CV

Defines the customer payments to be applied to invoices that have been produced.

PERSON_CV

Defines stored details of a person.

PRODUCT_INSTANCE_CV

Defines history of an instance of a product.

SERVICE_CV

Defines details of all services.
