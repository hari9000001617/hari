INV_COMPANION_PRODUCT_V

The INV_COMPANION_PRODUCT_V view provides details about companion product instances, which can be restricted to those associated with a specified base product instance or customer node.

The view is used by invoice templates as part of invoice image generation.

Column Definitions

Column Name

Properties

Description

PRODUCT_INSTANCE_ID

NUMBER (15)

Not null

ID of a companion product instance.

LAST_MODIFIED

DATE

Not null

Date-time when the row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Start of the effective date-time range for the row.

Note:

The effective start date-time may be greater than the effective start date-time of the companion product instance.

EFFECTIVE_END_DATE

DATE

Not null

End of the effective date-time range for the row.

Note:

The effective end date-time may be less than the effective end date-time of the companion product instance.

BASE_PRODUCT_INSTANCE_
ID

NUMBER (15)

Not null

ID of the base product instance associated with the companion product instance.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer hierarchy element or node to which the companion product instance belongs.

PRODUCT_INSTANCE_
STATUS_CODE

NUMBER (2)

Not null

Status of the companion product instance. Valid values are:

  • 0 (Product Creation): A temporary status during the creation of a product. When product creation is complete, the status is updated.
  • 1 (Provision Required): Provisioning must be performed.
  • 2 (Provisioned)
  • 3 (Active)
  • 4 (Suspended)
  • 5 (Temporary Disconnect Required): Deactivation must be performed.
  • 6 (Temporarily Disconnected)
  • 7 (Reactivation Required): Reactivation must be performed.
  • 8 (Cancellation Required): Deactivation must be performed.
  • 9 (Cancelled)
  • 10 (Moved)
  • 11 (Upgraded).

PRODUCT_INSTANCE_
STATUS_ABBREV

VARCHAR (60)

Not null

Status of the companion product instance. Valid values are:

  • 0 (Product Creation): A temporary status during the creation of a product. When product creation is complete, this status is updated.
  • 1 (Provision Required): Provisioning must be performed.
  • 2 (Provisioned)
  • 3 (Active)
  • 4 (Suspended)
  • 5 (Temporary Disconnect Required): Deactivation must be performed.
  • 6 (Temporarily Disconnected)
  • 7 (Reactivation Required): Reactivation must be performed.
  • 8 (Cancellation Required): Deactivation must be performed.
  • 9 (Cancelled)
  • 10 (Moved)
  • 11 (Upgraded).

PRODUCT_ID

NUMBER(8)

Not null

ID of the product associated with the companion product instance.

PRODUCT_NAME

VARCHAR (60)

Not null

Name of the product.

PRODUCT_DISPLAY_NAME

VARCHAR (120)

Not null

Name of the product displayed.

It defaults to PRODUCT_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

PRODUCT_DESCRIPTION

VARCHAR (255)

Not null

Description of the product.

PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer from which this companion product instance was generated.

The product offer must belong to the product offer hierarchy of the product offer assigned to this product instance's base product.

If the base product instance is:

  • Assigned a product offer, it must not be set to NULL.
  • Not assigned a product offer, it must be set to NULL.

PRODUCT_OFFER_NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product instance was allocated.

If PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PRODUCT_OFFER_
ALLOCATION_DATE

DATE

Date-time at which this product instance was allocated from the product offer specified in PRODUCT_OFFER_ID.

If PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PREVIOUS_PRODUCT_OFFER_
ID

NUMBER (8)

ID of the previous product offer associated with this product instance.

It is automatically modified when the product offer associated with a product is changed.

PREVIOUS_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product instance was previously allocated.

If PREVIOUS_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 associated with the companion product instance.
