Architecture Overview

The Java TRE API provides a mechanism for accessing TRE functions.

To access TRE functions, a TreConnection object is created that calls methods to perform the TRE functions.

The parameter and return types that are used when a remote function is called are described in Variable Types. This applies when using TreConnection.treVariantCall.

The Java API has been implemented so that a number of mechanisms can be developed, and one of them chosen at runtime. The mechanism chosen at runtime is specified in the EJIENVFILE configuration file. The connection method uses the tuxhandler socket listener, known as THP.

Secure communications over this connection are supported. Properties in the EJIENVFILE configuration file control the use of secure communications.
