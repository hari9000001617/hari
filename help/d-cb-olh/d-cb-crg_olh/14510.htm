Description

Returns the value of a customer node subtotal.

Parameter

Description

Subtotal@

In: Blob ID of the subtotal.

If the subtotal name is xxx&, the blob ID is xxx@.

CustomerNodeId&

In: ID of the customer node (CUSTOMER_NODE_ID).

EffectiveDate~

In: Any date in a given subtotal period.

For example, if a subtotal is aggregated daily, any time within a day matches that day.

ReservedValue<T>

Out: Sum of any reservations against the subtotal in the effective period.

The type of this parameter matches the return type of the function.

FromDate~

In: Start date of the subtotal aggregation period.

ToDate~

In: End date of the subtotal aggregation period.

 

Returns

Subtotal value of the function's return type if successful and aggregates exist; an undefined value of the function's return type if successful and no aggregates exist; otherwise, an error is raised.

Remarks

Each variant of the function can be accessed through the trerate server (EventRateAndReturn&()) and the ERT (EventRateAndStore&()). Some of these functions can also be accessed outside of Rating (as indicated below).

Return types can be integer (&), real(#), integer hash (&{}), or real hash (#{}).

Interface 1

Available only in Rating. The CustomerNodeId& and EffectiveDate~ are implied from the event being rated. For EventRateAndReturn&, the returned value contains reservations, excluding the current reservation. For EventRateAndStore&(), the returned value does not contain reservations.

Interface 2

Available only in Rating. The CustomerNodeId& and EffectiveDate~ are implied from the event being rated. The reserved value is returned separately in all cases. For EventRateAndReturn&(), the returned value excludes the current reservation.

Interface 3

Available in Rating. Available outside of Rating if the subtotal periods are context independent. For EventRateAndReturn&(), the returned value contains reservations, excluding the current reservation. For EventRateAndStore&(), the returned value does not contain reservations.

Interface 4

Available in Rating. Available outside of Rating if the subtotal periods are context independent. The reserved value is returned separately in all cases. For EventRateAndReturn&(), the returned value excludes the current reservation.

Interface 5

Available inside and outside of Rating. For EventRateAndReturn&, the returned value contains reservations, excluding the current reservation. For EventRateAndStore&() and for non-rating use, the returned value does not contain reservations.

Interface 6

Available inside and outside of Rating. The reserved value is returned separately in all cases. For EventRateAndReturn&(), the returned value excludes the current reservation.
