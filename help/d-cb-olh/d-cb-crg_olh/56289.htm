FileRegisterRecordUpdate&()

Syntax

FileRegisterRecordUpdate&(
    const FileRegisterRecordId&,
    var LastModified~,
    const FieldNames$[],
    FieldValues?[])

 

Description

Updates an audit record, or a record marked for delayed batch extraction.

Note:

If FILE_RECORD_STATUS_CODE is set to Success or Failure, the record cannot be updated.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

FileRegisterRecordId&

In: ID of the record.

LastModified~

In: Date-time the record was last updated.

Out: Date-time the record is updated.

FieldNames$[]

In: Names of the fields from the FILE_REGISTER_RECORD_TRE_V view whose values are to be updated.

FieldValues?[]

In: Field values in the same order that the field names are specified in FieldNames$[].

 

Returns

1 If successful; otherwise, an error is raised.

Remarks

The following table lists fields that are derived or set to their default values.

 

Fields and Default Values

Field Name

Value

OPERATION_DATE

current_date(), if FILE_RECORD_STATUS_CODE is 3 (Success) or 4 (Failure).

PENDING_CREATION_DATE

NULL if the updated record has FILE_RECORD_STATUS_CODE set to 3 (Success) or 4 (Failure).

 

The following table lists validation rules for the fields supplied in FieldNames$[].

Validation Rules

Validation Rule

Fields

Cannot be specified.
  • CREATION_DATE
  • CB_ENTITY
  • PE_CLASS_ID
  • ENTITY_ID
  • ENTITY_ID2
  • ENTITY_ID3
  • LAST_MODIFIED
  • OPERATION_DATE
  • OPERATOR_ID
  • PARTITION_NR
  • PENDING_CREATION_DATE
  • TEMPLATE_INTERFACE_DEFN_ID.

Can be specified with conditions.
  • EXTERNAL_DEFN_ID must be associated with the specified CB_ENTITY or PE_CLASS_ID.
  • FILE_RECORD_STATUS_CODE:
    • Can be set from Pending to another status, and from In Progress to Success or Failure.
    • If set to Success or Failure, FILE_REGISTER_ID, FILE_REGISTER_RECORD_NR, and INTERFACE_DEFN_ID must already be set or specified.
  • FILE_REGISTER_ID can be updated only if its current value is Null.
  • FILE_REGISTER_RECORD_NR:
    • Must be specified if FILE_REGISTER_ID is specified.
    • Can be updated only if its current value is Null.
  • INTERFACE_DEFN_ID:
    • Must be specified if FILE_REGISTER_ID is specified.
    • Must be set to the ID of an interface definition, not an interface template.
    • Must be associated with the template specified in TEMPLATE_INTERFACE_DEFN_ID.
    • Can be updated only if its current value is Null.
  • TASK_ID can be updated only if its current value is Null.
