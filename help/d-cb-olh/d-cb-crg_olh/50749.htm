RGP_EQUIPMENT_HISTORY_V

The RGP_EQUIPMENT_HISTORY_V view defines the details available for calculating rentals on equipment.

Column Definitions

Column Name

Properties

Description

EQUIPMENT_ID

NUMBER (15)

Not null

ID of an item of equipment.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

EQUIPMENT_TYPE_ID

NUMBER (8)

Not null

ID of the type of equipment.

EQUIPMENT_TYPE_NAME

VARCHAR (60)

Not null

ID of the type of equipment.

EQUIPMENT_TYPE_DISPLAY_NAME

VARCHAR (60)

Not null

Name of equipment type displayed.

It defaults to EQUIPMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

EQUIPMENT_STATUS_CODE

NUMBER (2)

Not null

Status of the equipment item (for example, Idle or In Use).

For a complete list of statuses, refer to the EQUIPMENT_TYPE table.

The list of possible values is dependent on the equipment type.

An equipment item can have a status of Idle (0) only if it is not associated with any rows in the PRODUCT_INSTANCE_EQUIP for the entirety of its date range, or the rows it is associated with are product instances with statuses of Cancelled, Moved, or Upgraded. It is checked by the DVP.

An equipment item can only have a status of In-Use (3) if it is associated with one or more rows in the PRODUCT_INSTANCE_EQUIP table for the entirety of its date range with product instance statuses other than Cancelled, Moved, or Upgraded. It is checked by the DVP.

REASON_CODE

NUMBER (8)

Optional code that can be used to define possible reasons for the current status of the equipment.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified this entry.

ATLANTA_GROUP_ID

NUMBER (8)

Not null

Security group that has access to this entry.

EQUIPMENT_SUBTYPE

VARCHAR (40)

Equipment subtype category to which this service belongs (for example, alpha-numeric and numeric display within equipment type pager).

It is recommended that EQUIPMENT_SUBTYPE be encoded (stored as numeric value) using a reference type.

EQUIPMENT_ORIGIN_CODE

NUMBER (4)

ID for where the equipment originally came from or whom it was supplied by (for example, from store and customer-supplied).

MAKE

VARCHAR (40)

Maker of the equipment.

It is recommended that MAKE be encoded (stored as a numeric value) using a reference type to decode its value.

MODEL

VARCHAR (40)

Model of the equipment.

It is recommended that MODEL be encoded (stored as a numeric value) using a reference type to decode its value.

NR_ITEMS

NUMBER (8)

Not null

Number of items of equipment associated with this record, usually 1.

It must be greater than or equal to 1, enforced by C_EQUIPMENT_HISTORY_RANGE check constraint.

SERIAL_NUMBER

VARCHAR (64)

Not null

Serial number or ID for the equipment item.

If NR_ITEMS is greater than 1, this defines the start of a contiguous range of serial numbers of this equipment.

The serial number must be unique within its equipment type.

It is checked by the DVP.

DECODED_SERIAL_NUMBER

VARCHAR (64)

If the equipment type has a decode expression defined, it stores the numeric value of the decoded serial number as a fixed length BCD string (the fixed length is dependent on the decoding expressions, with a maximum of 40 being supported).

The DVP checks that this column is not null if the equipment type has a SERIAL_NUMBER_DECODE_EXPR, but it cannot verify that the value is correct.

TO_SERIAL_NUMBER

VARCHAR (64)

Inclusive end of a contiguous range of serial numbers for this equipment.

The equipment type's SERIAL_NUMBER_DECODE_EXPR and SERIAL_NUMBER_ENCODE_EXPR define the format of this serial number and the meaning of a range.

It must have a value if NR_ITEMS is greater than 1, and it must be null if NR_ITEMS is equal to 1, enforced by C_EQUIPMENT_HISTORY_RANGE check constraint.

It can only have a value if its equipment type has SERIAL_NUMBER_DECODE_EXPR and SERIAL_NUMBER_ENCODE_EXPR expressions defined for it.

It is checked by the DVP.

DECODED_TO_SERIAL_NUMBER

VARCHAR (64)

If the equipment type has a decode expression defined, it stores the numeric value of the decoded serial number as a fixed length BCD string.

The DVP checks that this column is not null if TO_SERIAL_NUMBER is not null and the equipment type has a SERIAL_NUMBER_DECODE_EXPR, but it cannot verify that the value is correct.

NETWORK_ADDRESS

VARCHAR (64)

Network address programmed into the equipment item where this value does not change dynamically or cannot be reassigned while it is in use by a customer (for example, mobile data network LLI that can only be assigned by the network provider).

If a range of serial numbers is specified (NR_ITEMS > 1), it is usually null, unless network addresses are associated with the equipment before use.

DECODED_NETWORK_ADDRESS

VARCHAR (64)

If the equipment type has a network decode expression defined, and NETWORK_ADDRESS is not null, it stores the numeric value of the decoded network address as a fixed length BCD string.

It is checked by the DVP. The DVP cannot verify that the value is correct.

TO_NETWORK_ADDRESS

VARCHAR (64)

Inclusive end of a contiguous range of network addresses for this equipment only specified if NR_ITEMS > 1 and network addresses are associated with the equipment before use.

The equipment type's NETWORK_ADDRESS_FORMAT_CODE defines the format of this network address and the meaning of a range.

It must have a value if NR_ITEMS is greater than 1 and NETWORK_ADDRESS is not null.

It must be null if NR_ITEMS is equal to 1, enforced by C_EQUIPMENT_HISTORY_RANGE check constraint.

It can only have a value if its equipment type has NETWORK_ADDRESS_DECODE_EXPR and NETORK_ADDRESS_ENCODE_EXPR expressions defined for it. It is checked by the DVP.

DECODED_TO_NETWORK_ADDRESS

VARCHAR (64)

If the equipment type has a TO_NETWORK_ADDRESS specified, it stores the numeric value of the decoded to network address as a BCD string.

It is checked by the DVP. The DVP cannot verify that the value is correct.

RESERVED_CUSTOMER_NODE_ID

NUMBER (15)

ID of the customer node associated with the reservation, if this equipment was reserved for a customer.

Note:

The reservation of equipment for customers is independent of the equipment's status.

It must exist (not necessarily as a single record) for the entirety of this record's date range. It is checked by the DVP.

Equipment reserved for a customer node can only be allocated to that customer node or any customer node in the same hierarchy.

Conversely, equipment reserved for a customer node cannot be allocated to a customer node not in the same hierarchy for any period of the reservation. It is checked by the DVP.

In Transit Equipment Details

Column Name

Properties

Description

SENDER_PERSON_ID

NUMBER (15)

ID of the contact who sent this item of equipment.

It cannot be directly set, but is set to the person associated with the user who updated the equipment to an In-Transit status by the biEquipmentUpdate&() function.

Users who do not have a corresponding contact record cannot set equipment to an In-Transit status.

It must contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

The person must exist for at least the date range of this record. It is checked by the DVP.

RECIPIENT_PERSON_ID

NUMBER (15)

ID of the contact or group who is to receive this item of equipment.

It can only be set when equipment is being updated or in an In-Transit status. The group of the equipment is updated to the group of the assigned receiver.

It must contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

The person must exist for at least the date range of this record. It is checked by the DVP.

TRANSIT_ORGANISATION

VARCHAR (80)

Name of the transit organisation for sending the equipment to the recipient.

It can only contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_CONTACT_NUMBER

VARCHAR (30)

Contact phone number for the transit organisation.

It can only contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_REFERENCE_NUMBER

VARCHAR (30)

Reference or receipt number given by the transit organisation for the transportation of this equipment.

It can only contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.

TRANSIT_DATE

DATE

Date-time when the equipment's status was last updated to In-Transit.

It can only contain a value if the equipment has a status of In-Transit, enforced by C_EQUIPMENT_HISTORY check constraint.
