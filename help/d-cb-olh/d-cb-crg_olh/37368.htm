biBillRunInvoicePrepaidGenerate&()

Syntax

biBillRunInvoicePrepaidGenerate&(
    BillRunId&,
    EffectiveDate~,
    EffectiveDayOfMonth&,
    BillRunOperationId&,
    QAInd&,
    RootCustomerNodeList&[],
    var SuccessCustomerNodeList&[],
    var ErrorCustomerNodeList&[],
    var SuppressedCustomerNodeList&[],
    var OperationStatistics?{})

 

Description

Generates invoice statements only for prepaid customer hierarchies.

This is a billing configuration-compliant remote wrapper function around biInvoicePrepaidGenerate&().

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

BillRunId&

In: ID of the bill run.

EffectiveDate~

In: Effective date of the bill run.

EffectiveDayOfMonth&

Not used.

BillRunOperationId&

In: ID of the bill run operation.

QAInd&

Not used. QA bill runs are not supported for prepaid billing.

RootCustomerNodeList&[]

In: List of one or more prepaid root customer nodes to be processed.

All services for the root customer node and any child node in a hierarchy must be prepaid.

SuccessCustomerNodeList&[]

Out: List of all root customer nodes successfully processed.

It is a subset of the RootCustomerNodeList&[] parameter and preserves the order of the RootCustomerNodeList&[] parameter.

ErrorCustomerNodeList&[]

Out: List of root customer nodes unsuccessfully processed or unable to be locked because of other billing configuration processing the customer node.

SuppressedCustomerNodeList&[]

Out: List of root customer nodes whose invoices were suppressed.

A root customer node ID is added to this parameter if all prime accounts in the customer's hierarchy are suppressed. An invoice is suppressed if the account does not have charges.

OperationStatistics?{}

Out: Unknown EPM hash containing the statistics gathered during the bill run operation.

The table below lists the hash keys.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The function uses BillRunInvoicePrepaidUsageChargesBeforeBillDate&() to indicate whether or not to process charges up to but not including the bill run effective date.

The following table lists the OperationStatistics?{} parameter hash keys.

 

OperationStatistics?{} Parameter Hash Keys

Key

Value

AMOUNT

Total amount billed on invoices and statements generated in the bill run operation.

GENERAL_STATS1

Number of invoices generated by the bill run operation.

GENERAL_STATS2

Number of statements generated in the bill run operation.

GENERAL_STATS5

Number of customer nodes processed by the bill run operation.

GENERAL_STATS9

Number of original charges processed in the bill run operation.
