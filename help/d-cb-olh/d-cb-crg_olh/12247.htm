Description

Converts an EPM hash or array into an XML document.

Attributes

Function Type

Remote Synchronous

Service

biTrexml

Application Environment

Any

Context

Any

 

Parameter

Description

Data?{}

In: Hash containing the data to be converted to an XML string.

DefHeader$

In: Optional header to be placed after the XML declaration and before the parsed XML data.

If not defined, an empty string is passed.

Data?[]

In: Array containing data for the XML document when ordered parsing is used.

 

Returns

An XML document (in a string) if successful; otherwise, an error is raised.

Remarks

Interface 1 and 2 builds an XML document from the EPM hash. If a corresponding schema exists, it is used to generate the document.

Interface 3 and 4 builds an XML document from an EPM array preserving elements's order.

For more information about converting XML documents using the trexml server, refer to Tuxedo Guide for Singleview.

The following table lists error messages that can be generated during parsing.

 

Error Messages

Error Message

Description

Not well-formed (invalid token)

The current token could not be interpreted (possible corruption of the XML stream).

No element found

The XML data was passed in with no defined elements.

Duplicate Attribute

An attribute for a specific element was defined more than once.

Junk after document element

The XML data has junk characters after the last element.

XML declaration not well-formed

The XML header is corrupt or uses illegal syntax.

Mismatch tag

The XML tags do not match or are not synchronised.

Unknown encoding

The encoding in the XML header is not known to the parser.

 

The XML document is assembled by creating a default header with the content:

<?xml version="1.0" encoding="UTF-8"?>

The encoding value is specified by the LANG environment variable for the Singleview server. For example, if the LANG variable is ISO-8859, the default header is:

<?xml version="1.0" encoding="ISO-8859"?>

Elements in the hash are appended to the XML document. The function constructs the document using one of the following parsing methods:

  • Parsing with schema
  • Default parsing.

For more information about parsing methods, refer to the Tuxedo Guide for Singleview.

Handling Special Characters in Values

Special characters found when building a string are replaced with the corresponding XML entity. The special characters that are handled, and the corresponding XML entity, are displayed in the following table.

 

Special Characters and Equivalent XML Entity

Special Character

XML entity

<

&lt;

>

&gt;

'

&apos;

"

&quot;

&

&amp;

 

If the strings <![CDATA[ and ]]> are found when building a string, the handling of the special characters displayed in the table above is not performed and the characters between these strings are copied literally.

Handling Empty Elements

If an element is empty when building an XML string with an XML schema, the short notation <ElementName/> is used instead of the long notation <ElementName></ElementName>.
