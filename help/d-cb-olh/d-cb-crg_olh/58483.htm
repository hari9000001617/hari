TARIFF_RECURRING_TRE_V

 

The TARIFF_RECURRING_TRE_V view lists the details of recurring and one-off tariffs. It includes real-time tariffs.

Note:

Only recurring tariffs valid on the invoice date associated with the bill run are applied during billing.

Column Definitions

Column Name

Properties

Description

TARIFF_ID

NUMBER (8)

Not null

ID of the tariff.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

It always equals the effective start date for this tariff in TARIFF_TRE_V view.

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day. Time component always corresponds to one second before midnight (that is, 23:59:59).

RECURRING_CHARGE_PERIOD

NUMBER (2)

Recurring charge frequency.

For real-time tariffs, must be set to NULL.

If not set to NULL, it must have a value greater than 0.

RECURRING_CHARGE_UNIT_
CODE

NUMBER (1)

Units for applying recurring charges. Valid values are:

  • 0 (Days)
  • 1 (Weeks)
  • 2 (Months).

For example, if RECURRING_CHARGE_PERIOD is set to 3 and RECURRING_CHARGE_UNIT_CODE is set to 2 (Months), the charge is applied every three months.

ADVANCE_CHARGE_PERIOD

NUMBER (2)

For recurring charges, it defines the number of periods (in RECURRING_CHARGE_UNIT_CODE) for which the charge is applied in advance of the invoice issue date (or fixed date).

It can be negative.

For real-time tariffs, it must be set to NULL.

PRORATE_IND_CODE

NUMBER (1)

For recurring charges, it indicates if the charge is pro-rated.

For real-time tariffs, it must be set to NULL.

PRORATE_PERIOD

NUMBER (2)

Period in which pro-rating is performed.

For real-time tariffs, it must be set to NULL.

PRORATE_UNIT_CODE

NUMBER (1)

Units for the pro-rate period.

It is based on the PRORATE_UNIT reference type.

DAYS_PER_MONTH

NUMBER (8,6)

Number of days in a month used for performing partial month calculations.

It is only applicable if RECURRING_CHARGE_UNIT_CODE is set to 2 (Months); otherwise, it must be set to null.

If not defined, the full number of days in each month is used.

FIXED_DATE

DATE

Fixed date to use for recurring tariffs.

It controls the paid-to date used in rental calculations.

If a tariff is synchronised with a FIXED_DATE, the RGP finds the nearest day before the bill cycle date that is an exact number of fixed date periods after the FIXED_DATE, then adds exactly one ADVANCE_CHARGE_PERIOD and one RECURRING_CHARGE_PERIOD to obtain the charge period end date.

It can only contain a value if PRORATE_IND_CODE is 1 (True), and the RECURRING_CHARGE_UNIT_CODE is 2 (Months).

FIXED_DATE_PERIOD

NUMBER (2)

Period between fixed dates.

It can only contain a value, and must contain a value, if FIXED_DATE is not null.

It defaults to ADVANCED_CHARGE_PERIOD, and has the same units as ADVANCED_CHARGE_PERIOD.

For example, if FIXED_DATE is 1 Jan and both ADVANCED_CHARGE_PERIOD and FIXED_DATE_PERIOD are set to 12, the rental is charged from product instance activation date to Jan 1 for its first instance, and then from Jan 1 to Jan 1 for each period after that.

ACTIVE_IND_CODE

NUMBER (1)

If set to 1 (True), recurring events are only generated for this tariff when the product instance has a status of Active, and all eligibility expressions are satisfied.

If set to NULL, recurring charges are generated whenever the product instance has a status other than Product Creation, Cancelled, Moved, or Upgraded, and all eligibility expressions are satisfied. In this case, event generation is usually controlled by eligibility expressions.

For real-time and one-off charge tariffs, it must be set to null.

Default is 1 (True) for new recurring tariffs.

INTERIM_IND_CODE

NUMBER (1)

If set to 1 (True), events are generated for this tariff during interim bill runs; otherwise, this tariff is ignored during interim bill runs.

It can only be set to 1 or NULL.

For real-time tariffs, it must be set to NULL.

NORMALISED_EVENT_TYPE_ID

NUMBER (8)

Not null

Type of normalised event generated by the RGP for rental or one-off events based on this tariff.

It controls the list of valid values for EVENT_TYPE_CODE and ADJUSTMENT_EVENT_TYPE_CODE.

EVENT_TYPE_CODE

NUMBER (8)

Not null

Type of event generated by the RGP for normalised events associated with this tariff.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event sub-type to be generated by the RGP for normalised events based on this tariff.

The event sub-type is optional.

The EVENT_TYPE_CODE selected determines if the event sub-type is required, and the valid values to be used. This means that it is based on the CHILD_REFERENCE_TYPE_ID reference type associated with the EVENT_TYPE_CODE.

ADJUSTMENT_EVENT_TYPE_
CODE

NUMBER (8)

Type of event type generated by the RGP for adjustments to normalised events associated with this tariff.

If not specified, no adjustment events are generated for this tariff.

It can only contain a value for recurring tariffs.

ADJUSTMENT_EVENT_SUB_
TYPE_CODE

NUMBER (8)

Type of event sub-type generated by the RGP for adjustments to normalised events associated with this tariff.

The ADJUSTMENT_EVENT_TYPE_CODE determines whether the event sub-type and set of permissible values are generated.

It can only have a value if ADJUSTMENT_EVENT_TYPE_CODE is not null.

CHARGE_START_DATE_CODE

NUMBER (1)

Charge start date of rental events for this tariff. Valid values are:

  • 1 (Bill cycle issue date)
  • 2 (Rental event period start date).

Default is 1.

If set to:

  • 1, only the tariffs as they exist on that date, are applicable.
  • 2, a single rental event may be split up into multiple events due to changes in tariff.

For real-time tariffs, it must be set to NULL; otherwise, it must have a value.

CHARGED_ENTITY_CODE

NUMBER (1)

Not null

Billing entity to which this recurring or one-off tariff is to be applied. Valid values are:

  • 1 (Product)
  • 2 (Facility group)
  • 3 (Service)
  • 4 (Equipment)
  • 5 (Entitlement), only applicable for real-time tariffs.

It determines the direct variables available for eligibility expressions.

For real-time tariffs, it must be set to 1 or 5.

FACILITY_GROUP_ID

NUMBER (8)

ID of a facility group associated with this tariff.

Rental events for this tariff are only generated for this facility group.

It can only contain a value if CHARGED_ENTITY_CODE is set to 2.

Facility group must exist for at least the date range of this tariff.

ELIGIBILITY_EXPRESSION_
LIST_ID

NUMBER (8)

List of conditional expressions that must be satisfied for the RGP to generate a recurring or one-off event for this tariff.

Specified expressions have access to all functions with an application environment of Any (including remote functions) and direct variable from one of the following views, depending on the value of CHARGED_ENTITY_CODE:

  • RGP_PRODUCT_INSTANCE_V
  • RGP_FAC_GROUP_INSTANCE_V
  • RGP_SERVICE_HISTORY_V
  • RGP_EQUIPMENT_HISTORY_V.

Each expression must return an integer result.

ASSIGNMENT_EXPRESSION_
LIST_ID

NUMBER (8)

List of assignment expressions that control values assigned to fields in the event to be generated by the RGP.

The assigned values can be accessed by the tariff to control charge calculations, or checked by the RGP to determine when adjustment events must be generated.

Expressions access the same direct variables as ELIGIBILITY_EXPRESSION_LIST_ID, and also direct variables associated with the RGP_NORMALISED_EVENT_V view.

Note:

Expressions are evaluated only for rental events. For adjustment events, the values from the rental event being adjusted are copied into the adjustment event.

PRORATE_SPLIT_IND_CODE

NUMBER (1)

If set to 1 (true), events generated for this tariff that include a pro-rate period are split in two.

One event spans the pro-rate period and has a fractional duration, and the other event aligns with the end date of the charge period and has a duration of a whole number of charge periods.

If not set to 1, it must have a value of null.

For a real-time tariff, it must be set to NULL.

CHARGE_PERIOD_SPLIT_IND_
CODE

NUMBER (1)

Specifies whether events generated for this tariff can span multiple charge periods.

If set to 1 (True), events generated for this tariff that span multiple charge periods are split into multiple events.

Each generated event spans no more than 1 charge period and has a period end date that aligns with the end date of the charge period. If the original event included a prorate period, one event spans this prorate period and has a fractional duration, and the remaining events span exactly one charge period.

If not set to 1, must be NULL.

It can only be set if PRORATE_SPLIT_IND_CODE is set to 1 (True).

REAL_TIME_IND_CODE

NUMBER (1)

Specifies whether the tariff is a real-time tariff (see the Customer Care and Product Configuration Guide for Singleview Convergent Billing).

If set to 1 (True), this tariff is a real-time tariff. If the associated eligibility criteria are satisfied, an event is generated and rated for this tariff immediately instead of being generated at billing time by the RGP.

If not set to 1, it must be NULL.

It cannot have different values over the history of the tariff.

 

Real-Time Tariff Columns

Values for the following columns are based on the PRODUCT_INSTANCE_STATUS reference type.

 

Column Name

Properties

Description

TRIGGER_RESERVE_PI_
STATUS_CODE

NUMBER (8)

Trigger for an account reservation request.

When the status of a companion product instance is updated to this value, an account reservation request is generated.

It must be NULL if:

  • The tariff is for a cancellation charge.
  • REAL_TIME_IND_CODE is NULL.

TRIGGER_DEBIT_PI_
STATUS_CODE

NUMBER (8)

Trigger for an initial charge request.

When the status of a companion product instance is updated to this value, an initial debit/charge is created for an entitlement allocation.

It must be NULL if:

  • The tariff is for a cancellation charge.
  • REAL_TIME_IND_CODE is NULL.

TRIGGER_RETRY_PI_
STATUS_CODE

NUMBER (8)

Trigger for a charge regeneration request.

When the status of a companion product instance is updated to this value, an attempt is made to regenerate a charge for this tariff.

It must be NULL if:

  • The tariff is for a cancellation charge.
  • REAL_TIME_IND_CODE is NULL.

RESERVE_FAIL_PI_
STATUS_CODE

NUMBER (8)

Value to which the status of a companion product instance is set on failure of a reservation request.

It must be NULL if:

  • The tariff is for a cancellation charge.
  • REAL_TIME_IND_CODE is NULL.

RETRY_SUCCESS_PI_
STATUS_CODE

NUMBER (8)

Value to which the status of a companion product instance is set if a reservation request succeeds following a previous failure.

It must be NULL if:

  • The tariff is for a cancellation charge.
  • REAL_TIME_IND_CODE is NULL.

DO_NOT_FIX_DEPENDENCIES

NUMBER (1)

Pseudo-column

Used for updates only

If specified with a 1 (TRUE), variable dependencies are not fixed.

It is used for multi-phase tariff updates to delay fixing of variable dependencies.
