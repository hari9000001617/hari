biOperatorInsert&()

Syntax

biOperatorInsert&(
    var LastModified~,
    FieldNames$[],
    FieldValues?[])

 

Description

Creates a user and an associated Oracle user account, if necessary.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

LastModified~

In: Last modified date-time of the user.

Out: New last modified date-time of the updated user.

FieldNames$[]

In: Names of fields from OPERATOR_TRE_V view for which values have been specified for the insert operation.

Any fields not specified must be optional or have a default value.

FieldValues?[]

In: Values of the fields specified in FieldNames$[] listed in the same order.

 

Returns

ID of the new user if successful; otherwise, an error is raised.

Remarks

For information about validation requirements for user name (OPERATOR_LOGIN_NAME) and password (PASSWORD) fields, refer to the Security and Auditing Configuration Guide for Singleview.

The user calling the function must have appropriate privileges before proceeding with the requested insertion. They must have

  • Insert privilege on the security group associated with the user record.
  • Assignment privilege on the security group associated with the profile that is being assigned to the user.

Note:

Superuser privileges can only be enabled if the caller has superuser privilege.

If ENABLED_IND_CODE is set to True, the password is validated and an error raised if none was given. A standard insert of an OPERATOR_TRE_V record is then performed and any necessary changes for the underlying Oracle user for the user, including enabling, locking, or expiring the account, are also performed. The password is then set and the appropriate Oracle role is granted to the user.

An AUDIT_USER_INSERT audit message is logged with the date of the change and name of the user who was created successfully; otherwise, AUDIT_USER_INSERT_ERROR is logged.
