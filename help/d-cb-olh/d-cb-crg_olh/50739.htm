INV_SERVICE_HISTORY_PRODUCT_V

The INV_SERVICE_HISTORY_PRODUCT_V view is based on INV_SERVICE_HISTORY_V, but provides additional information about the base product associated with each service, and any details of any contract associated with the service.

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

Invoice on which this service is displayed.

SERVICE_ID

NUMBER (15)

Not null

ID for a service.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Start date-time of the service history record active at the effective date-time of the invoice.

EFFECTIVE_END_DATE

DATE

Not null

End date-time of the service history record active at the effective date-time of the invoice.

SERVICE_NAME

VARCHAR (64)

Not null

Name associated with the service involving the most exact specification of the service in relation to the network.

The format of this value usually corresponds to how the service is referenced by switching equipment and the CDRs and events that are generated; for example, the full country number for a voice service.

NETWORK_NAME

VARCHAR (64)

Name associated with the service involving the most exact specification of the service in relation to the network, if this is different from SERVICE_NAME.

The format of this value usually corresponds to how the service is referenced by switching equipment and the CDRs and events that are generated; for example, the full country number for a voice service, or IMSE number for a GSM service.

SERVICE_SUBTYPE

SERVICE_RANGE_SEQNR

NUMBER (4)

Not null

It can only contain a negative value if the service has a status of Cancelled.

SERVICE_USAGE

VARCHAR (40)

Main usage type for the service where different types of usage are allowed.

It is dependent on the type of the service.

COMMS_TYPE

VARCHAR (40)

Appropriate communications signalling type classification such as Synchronous or Asynchronous.

It is dependent on the type of the service.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

SERVICE_STATUS_ABBREV

VARCHAR (60)

Not null

Current status of the service. Valid values are:

  • 0 (Product Creation): A temporary status during the creation of a product. When product creation is complete, this status is updated.

  • 2 (Provisioned)
  • 3 (Active)
  • 4 (Suspended)
  • 5 (Temporary Disconnect Required): Deactivation must be performed.
  • 6 (Temporarily Disconnected)
  • 7 (Reactivation Required): Reactivation must be performed.
  • 8 (Cancellation Required): Deactivation must be performed.
  • 9 (Cancelled)
  • 10 (Moved)
  • 11 (Upgraded).

Of these statuses, the codes for Active, Cancelled, Moved, and Upgraded are used by the system and must always be supplied.

Of these statuses, the codes for Active, Cancelled, Moved, and Upgraded are used by the system and must always be supplied.

DATE

Date-time that the service became billable to the customer.

It should correspond to the date-time on which the customer has accepted the service (or at least been notified that the service is available).

It is also the date used as the basis for calculating any periodic charges for the service.

Contract Columns

Column Name

Properties

Description

CONTRACT_NAME

VARCHAR (40)

Name for the contract.

It is the external ID for the contract.

It is usually automatically assigned on contract creation by the contract types entity validation.

CONTRACT_CUSTOMER_
REFERENCE

VARCHAR (40)

Customer reference for the contract.

It is how the customer identifies the contract within their own accounts payable (or equivalent) system.

Convergent Billing does not enforce uniqueness on this column.

CONTRACT_TYPE_ID

NUMBER (8)

Type of contract entered into, and controls the products that can be associated with the contract, and also defaults many characteristics on contract creation.

CONTRACT_TYPE_NAME

VARCHAR (30)

Type of contract entered into, and controls the products that can be associated with the contract, and also defaults many characteristics on contract creation.

CONTRACT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Name of contract type displayed.

It defaults to CONTRACT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore may differ between installations.

CONTRACT_STATUS_CODE

NUMBER (8)

Status of this contract.

Values are client configurable.

The value defaults from the CONTRACT_TYPE. Valid values are:

  • 1 (In Negotiation)
  • 2 (Pending)
  • 3 (Active)
  • 8 (In Grace)
  • 9 (Cancelled).

CONTRACT_STATUS_ABBREV

VARCHAR (60)

Status of this contract.

Values are client configurable.

The value defaults from the CONTRACT_TYPE. Valid values are:

  • 1 (In Negotiation)
  • 2 (Pending)
  • 3 (Active)
  • 8 (In Grace)
  • 9 (Cancelled).

CONTRACT_START_DATE

DATE

Date-time that the contract first became active for this customer.

CONTRACT_DURATION

NUMBER (4)

Duration of the contract.

The default value is based on the contract type. If it is specified, CONTRACT_EXPECTED_EXPIRY_DATE is derived using the formula:

CONTRACT_EXPECTED_EXPIRY_DATE = CONTRACT_START_DATE + CONTRACT_DURATION

If CONTRACT_EXPECTED_EXPIRY_DATE is manually modified, CONTRACT_DURATION is cleared. It cannot be modified if the contract type has a FIXED_DURATION_IND_CODE of 1.

CONTRACT_DURATION_UNIT_
CODE

NUMBER (2)

Units of the contract duration. Valid values are:

  • 0 (Days)
  • 1 (Weeks)
  • 2 (Months)
  • 3 (Years).

CONTRACT_EXPECTED_EXPIRY_
DATE

DATE

Date-time when this contract is expected to expire, derived from the contract start date and contract duration if the contract duration is specified.

CONTRACT_ACTUAL_EXPIRY_
DATE

DATE

Date-time when the contract was terminated.

If the time is 00:00:00, the contract must have a status of Cancelled from this date-time; otherwise, the contract must have a status of Cancelled from midnight on the following day after this date-time.

CONTRACT_GRACE_EXPIRY_
DATE

DATE

Date-time after the EXPECTED_EXPIRY_DATE that a contract can remain in a non-cancelled status and not be renegotiated.

It can be derived from EXPECTED_EXPIRY_DATE and contract type, and manually modified by a user.

It is re-derived whenever the EXPECTED_EXPIRY_DATE is modified.

CONTRACT_SALES_CHANNEL_
CODE

NUMBER (8)

Sales channel through which this contract was obtained.

CONTRACT_GENERAL_1 - CONTRACT_GENERAL_10

VARCHAR (255)

General fields 1 - 10 for entering contract-related details.

Contents and value of these fields is specified by entity validation associated with the contract type.

 

Charge Partition Column

Column Name

Properties

Description

PARTITION_NR

NUMBER (2)

Not null

Customer partition number associated with charges for the service.

All charges associated with an invoice are associated with the same account (and therefore customer), so when customer partitioning is implemented, all charges for an invoice have the same PARTITION_NR.

It can be used as an additional selection criterion to improve access to the CHARGE table by eliminating irrelevant CHARGE table partitions.

 

Product Offer Column Definitions

Column Name

Properties

Description

PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer associated with this service.

The product offer must:

  • Be configured as service-specific.
  • Belong to the same hierarchy as the product offer assigned to the base product instance for this service.

If the service is not assigned a product offer, it must be set to NULL.

PRODUCT_OFFER_NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in PRODUCT_OFFER_ID was allocated.

If PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PREVIOUS_PRODUCT_OFFER_
ID

NUMBER (8)

ID of the previous product offer associated with this service.

It is automatically modified when the product offer associated with a service is changed.

PREVIOUS_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in PREVIOUS_PRODUCT_OFFER_ID was previously allocated.

If PREVIOUS_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer assigned to this service.

This product offer must belong to the product offer hierarchy of the product offer assigned to the base product instance for the service.

It must be NULL if the base product instance for the service is not assigned a product offer.

SERVICE_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in SERVICE_PRODUCT_OFFER_ID was allocated. This node must be associated with the product offer identified by SERVICE_PRODUCT_OFFER_ID.

If SERVICE_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PRODUCT_OFFER_
ALL_DATE

DATE

Date-time at which this service was allocated the product offer identified by SERVICE_PRODUCT_OFFER_ID.

It is usually the same as ACTIVE_DATE, but can be different if the product offer is changed.

If SERVICE_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PREV_PRODUCT_
OFFER_ID

NUMBER (8)

ID of the previous product offer associated with this service.

It is automatically modified when the product offer associated with a service is changed.

SERVICE_PREV_PROD_OFF_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in SERVICE_PREVIOUS_PROD_OFFER_ID was previously allocated.

If SERVICE_PREVIOUS_PROD_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.
