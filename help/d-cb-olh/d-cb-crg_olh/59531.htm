Description

Interface 1 fetches the default external definition elements that have not been stored in the database from a specified entity validation.

Interface 2 fetches values for external definition elements from ED_FACT_CONFIG_V view records with the specified ClassId& and ObjectTypeId&. Values are retrieved in FACT_ID order.

Only facts in the ClassId& associated with this object type are included in the generated external definition.

FieldNames$[] and FieldValues?[] as described in the parameter descriptions, and 1 if successful; otherwise, an error is raised.

Remarks

Interface 1

Generated external definition elements have properties based on the specified entity validation.

The following rules are used to generate the elements:

  • The elements are sequenced based on the order of columns in the EVS view associated with the Convergent Billing entity linked to the specified entity validation. This corresponds to the order of columns in the TRE view for the Convergent Billing entity. Non-stored attribute types are added to the end of the definition.
  • The name of each element is set to one of the following:
    • Interface label of the attribute type for the column, if specified.
    • Label of the attribute type if the Interface label is not specified and the label is unique in the definition.
    • Column name associated with the EVS view.
  • The characteristics for each element are derived from one of the following, in descending order of preference:
    • Attribute type for the column in the specified entity validation, if specified.
    • Attribute type for the column from the core EVS entity validation, if specified.
    • Database characteristics of the column from the EVS view if it has no attribute type associated with it.
  • Elements are not generated for GENERAL columns in the EVS view unless the columns are mapped to an attribute type in the entity validation. However, all other columns have elements generated for them.
  • If ViewName$ is specified, each element is given a column name equal to the first thirty characters of its element name. If the resulting name is not unique within this definition, the column name is not specified, resulting in the element not being included in the generated view by default.
  • For elements that derive their characteristics from an attribute type that has masking or encryption defined (for example DISPLAY_MASK_FUNCTION_DEFN_ID or ENC_ALGORITHM_CODE) the value of SENSITIVE_IND_CODE, in the EXTERNAL_DEFN_ELEMENT_V view, is set to 1.

Interface 2

The returned external definition element values are set as follows:

  • Name of each element is set to the fact name.
  • Interface label is set to the fact name.
  • If the ViewName$ parameter is defined, the column name is set to the:
    • Element name, if one of the following is True:
      • A stored Process Engine fact is specified and the element name is set.
      • No Process Engine fact is specified, a default value is specified, and the element name is set.
    • Fact name, if a stored Process Engine fact is specified and the element name is not set.
  • Mandatory indication code is set to TRUE and MIN_OCCURS is set to 1 if the fact record's MANDATORY_IND_CODE is set to 1.
  • The maximum length is set to the MAX_LENGTH returned from the ED_FACT_CONFIG_V view if the fact record's data type corresponds to an XML data type code of String.
  • MAX_OCCURS is set to 1.
  • Process Engine fact ID is set to the ID of the fact record.
  • Process Engine fact sequence number is determined by calling the ObjectFactSort@[]() Process Engine function.
  • If the fact record has masking or encryption defined, the value of SENSITIVE_IND_CODE in the EXTERNAL_DEFN_ELEMENT_V view is set to 1. This means that the attribute type returned by ED_FACT_CONFIG_V has either DISPLAY_MASK_FUNCTION_DEFN_ID or ENC_ALGORITHM_CODE defined.
  • XML and interface data type codes are derived from the fact's data type code.
