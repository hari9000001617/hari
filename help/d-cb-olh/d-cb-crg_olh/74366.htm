SFMCategoryDisable$[]()

Syntax

Interface 1

 

Interface 2

 
SFMCategoryDisable$[]()
 
 
 
SFMCategoryDisable$[](
    const CategoryList$[])

 

Description

Lists disabled Schedule Function Monitor (SFM) function categories previously disabled using interface 2. If a function categories list is supplied, this function also disables the SFM functions with those categories.

Attributes

Function Type

Local

Service

Local

Application Environment

Any

Context

Any

 

Parameter

Description

const CategoryList$[]

In: List of function categories to be disabled. The existing list of function categories disabled by this function is overwritten.

If CategoryList$[] is empty or undefined, disabled function categories that were enabled when the SFM was started are re-enabled.

Note:

Function categories must be specified using SCHEDULED_FUNC_CATEGORY reference type code labels (not their abbreviations).

Returns

An array containing SFM function categories that were disabled using this function before the current call to this function.

Remarks

After Singleview is started, calls to this function return an empty list until subsequent calls to interface 2 are used to disable function categories.

This function can re-enable only function categories it disabled. Other ways function categories can be enabled are:

  • The -c command-line option specified when the SFM process is started (see the System Reference Guide for Singleview).
  • The ScheduledFunctionMonitor interface definition (see the Configurer's Reference Guide for Singleview Convergent Billing).
