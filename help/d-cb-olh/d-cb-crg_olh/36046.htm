RTM_Alarm

The RTM_Alarm derived attribute table defines RTM alarms, which are tested at each iteration of the Real-time Monitor.

The table has an evaluation type of Table Lookup By Multiple (1-10) Keys.

Column Definitions

Column Name

Properties

Description

AlarmName

String (64)

Not null

Name used as a reference for the alarm test (for example, CpuMeltdown).

TimeBand

String (64)

Time band during which the alarm is tested.

It must be defined in the RTM_TimeBand derived attribute table.

If unspecified and an alarm with the same AlarmName with a defined TimeBand is not active, the alarm test is executed at all times. TimeBand values for the same AlarmName cannot overlap.

Enabled

Boolean

Not null

Activates or deactivates the alarm test.

The default value is FALSE.

Instance

String (9)

Instance name on which to execute the alarm test.

If unspecified, the alarm test is executed on every instance in the cluster.

MetricName

String (64)

Metric to test against a minimum or maximum value.

It is defined in the RTM_MetricName derived attribute table.

If a metric is specified, one or both of the MinThreshold or MaxThreshold values must be specified.

MinThreshold

Real

If the value for the metric falls below the minimum, the alarm is active.

MaxThreshold

Real

If the value for the metric rises above the maximum, the alarm is active.

TestExpression

String (256)

EPM expression that allows complex alarm tests.

The test expression has access to variables that hold the current values of derived statistics.  The expression can generate an alarm message that is used when the action is later performed, and when the message is written to the log.

Each metric defined in the RTM_MetricName derived attribute table is available.

The EPM variable name must be the metric name, appended with #, for real values, or $ for string values. Integer values are converted to real values.  The type must be the type of value held by the metric.

If a test expression is specified, a value cannot be specified in the MetricName, MinThreshold, or MaxThreshold columns.

The expression's return value can be either an integer or a string (so that a customer message can be specified).

For an integer expression, if the return value is:

  • TRUE, the alarm condition is active, and the standard message is used as the message.
  • FALSE, the alarm condition is not active.

For an expression returning a string, if the return value has a:

  • Non-zero length string, the alarm condition is active, and the returned string is used as the message.
  • Zero length string, the alarm condition is not active.

    Note:

    Calling abort from within the expression has the same effect as a string expression returning the abort message.

ActionList

String (256)

Comma-separated list of actions to perform if the alarm is active.

Actions are defined in the RTM_Action derived attribute table.

DampenCount

Integer

Number of additional consecutive iterations for which a test returns an alarm state before the alarm is active or deactivated.

For example, if the DampenCount equals 5, the alarm test must return:

  • An alarm condition on six consecutive iterations before the alarm is active
  • A false alarm condition for six consecutive tests before the alarm is deactivated.

OneOff

Boolean

Specifies when an alarm is activated.

If:

  • TRUE, the alarm is activated for each iteration that the alarm test returns an alarm condition.

    A one-off alarm does not execute an action if the alarm condition is FALSE.

  • FALSE, the alarm is activated when the alarm test indicates that an alarm condition exists, and deactivated when the alarm condition no longer exists.

    A periodic alarm may be in an activated state for a number of iterations.
