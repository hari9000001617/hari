ScheduledFunctionInsert&()

Syntax

ScheduledFunctionInsert&(
    var LastModified~,
    const FieldNames$[],
    FieldValues?[])

 

Description

Inserts a new scheduled function.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

LastModified~

Out: Last modified date-time of the new scheduled function record.

FieldNames$[]

In: List of fields from the SCHEDULED_FUNCTION_TRE_V view whose values are to be inserted.

Use INSTANCE_NAME to specify where the scheduled function is executed.

FieldValues?[]

In: Values for the fields in the same order that their names are specified in the FieldNames$[] array.

 

Returns

Scheduled function ID if successful.

If the scheduled function has a CATEGORY_CODE of 9 (Purge), the insertion may be suppressed by the Purge Target setting for the current instance. If the insertion is suppressed, null& is returned.

Otherwise, an error is raised.

Remarks

Note:

Scheduled functions can only be inserted using server-side configuration.

Mandatory FieldNames$[] values are:

  • FUNCTION_NAME
  • CATEGORY_CODE.

The following SCHEDULED_FUNCTION_TRE_V view fields cannot be specified on insert:

  • INTERFACE_NR
  • SCHEDULED_FUNCTION_ID
  • LAST_MODIFIED
  • OPERATOR_ID
  • OPERATOR_LAST_MODIFIED
  • CREATED_OPERATOR_ID
  • STATUS_CODE
  • ORIGINAL_SCHEDULED_START_DATE
  • PENDING_START_DATE
  • ORIGINAL_RETRY_COUNT
  • FIRST_START_DATE
  • START_DATE
  • ELAPSED_TIME
  • EVALUATION_COUNT
  • TOTAL_ELAPSED_TIME
  • LAST_ERROR_DATE
  • XID
  • SFM_CONFIG.

Caution:

If called in a non-transactional server, it is the caller's responsibility to either commit or roll back the database changes performed by this function.

If called to insert a scheduled function with a category of Purge, this function first checks the purge notification (Purge Target) setting for the originating instance in the InstanceStatus derived attribute table. If Purge Target is not set to 0 (PURGE_ALL), a whitelist check is performed. If the entity is:

  • On the whitelist, the purge restriction is ignored and the entity is purged on all instances.
  • Not on the whitelist, a purge scheduled function is only inserted for instances allowed by the Purge Target setting.

    If no purge scheduled functions are inserted, this function returns null& and no errors are raised.
