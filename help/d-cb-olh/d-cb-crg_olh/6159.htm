Variables Based on RGP_NORMALISED_EVENT_V

All direct variables based on the RGP_NORMALISED_EVENT_V view have an application environment of Rating and a context of Normalised Event.

Description

PeriodStartDate~

Date

Note:

Pulses&

Integer

Number of meter pulses associated with the call.

RateBand$

String (40)

Time or rate band calculated for the call.

SwitchStartDate~

Date

Date-time of the event used to retrieve service details and evaluate any rating expressions.

In Bill Cycle Issue Date (BIDS mode), for:

  • Tariffs and one-off charges, it is set to the effective date of the bill run.
  • Adjustments, it is set to the effective date of the bill run that generated the event being adjusted.

In Rental Event Period Start date (REPS mode):

  • For tariffs, one-off charges, and debit adjustments, it is set to the period start date (PERIOD_START_DATE) of the event.
  • For credit adjustments, it is set to the period start date (PERIOD_START_DATE) of the event being adjusted.
  • If ROUND_ENTITY_HISTORY is set to Nearest Day or Full Day:
    • For credit adjustments, if SWITCH_START_DATE of the event being adjusted is not NULL, it is set to the SWITCH_START_DATE of the event being adjusted.
    • Otherwise, it is set to the period start date (PERIOD_START_DATE) before rounding is performed.

Volume#

Real

Volume of data associated with the call.
