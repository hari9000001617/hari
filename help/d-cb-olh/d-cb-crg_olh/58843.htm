biProductInstanceOfferHierarchyFetch&()

Syntax

biProductInstanceOfferHierarchyFetch&(
    ProductInstanceId&,
    ProductOfferNodeId&,
    EffectiveStartDate~,
    EffectiveEndDate~,
    var Hierarchy?[])

 

Description

Retrieves the offer hierarchy for the top-level product offer node, with flags indicating the offer and companion children currently selected for the product instance.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

ProductInstanceId&

In: ID of the base product instance assigned the top-level offer.

ProductOfferNodeId&

In: ID of the top-level product offer node to be assigned to the base product instance.

EffectiveStartDate~

In: Effective start date-time of the base product instance.

EffectiveEndDate~

In: Effective end date-time of the base product instance.

Hierarchy?[]

Out: Table containing the entire offer hierarchy marked with the currently-active offer and companion selections.

The table below lists columns in order.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

The hierarchy is returned as a flattened two-dimensional array with each row returned representing either an offer node or companion product. A column containing the array row number for the parent of the current row is included, and used to expand the array back into tree form.

The tree is ordered with higher-level nodes first, followed by offer sequence number (the SEQNR column in PRODUCT_OFFER_OFFER_TRE_V) within the level. Rows for companion products that are direct children of an offer are listed immediately after the associated offer, in ascending product sequence number order (the SEQNR column in PRODUCT_OFFER_PRODUCT_TRE_V).

If multiple instances of a product offer exist, each product offer and companion product instance is also retrieved and listed under the initial instance.

The following table lists Hierarchy?[] columns in order.

 

Hierarchy?[] Columns Order

Column Name

Description

ROW_NR

Row number in the array, starting at 0.

PARENT_ROW_NR

Parent node row number in the array.

-1 is used for the top-level node.

COMPANION_IND_CODE

Set to 0 for an offer row, and 1 for a companion product row.

PRODUCT_OFFER_NODE_ID

Product offer node ID, included in both product and offer rows.

It is the parent offer if it is a product row.

PRODUCT_OFFER_ID

Product offer ID, for both product and offer rows.

It is the parent offer if it is a product row.

PRODUCT_INSTANCE_ID

Product instance ID, if a product row and the companion has been instantiated.

PRODUCT_ID

Product ID, if it is a product row.

SELECTED_IND_CODE

Indicates if the row is in use. It includes cancelled companion products.

It can be modified to update the hierarchy selections, and applied using biProductInstanceOfferHierarchyApply&().

The ACTIVE_IND_CODE can be used to filter companion products with a status other than Cancelled.

MANDATORY_IND_CODE

If the parent is selected, it indicates if the offer or product is mandatory.

SERVICE_SPECIFIC_IND_CODE

Offer flag indicating these companion products are service-specific.

CHILD_PRODUCT_OFFER_MIN

For offer validation, minimum number of optional child offers to select directly below it.

CHILD_PRODUCT_OFFER_MAX

For offer validation, maximum number of optional child offers to select directly below it.

COMPANION_PRODUCT_MIN

For offer validation, minimum number of optional companion products to select directly below it.

COMPANION_PRODUCT_MAX

For offer validation, maximum number of optional companion products to select directly below it.

HIDDEN_IND_CODE

This item should not be visible to the user.

It is always visible for mandatory selections.

AUTO_EXPAND_HIERARCHY_IND_CODE

For tree views, it has children automatically expanded.

COMPANION_MULTIPLE_IND_CODE

Indicates if more than one companion product instance is active on the base product instance.

ACTIVE_IND_CODE

Indicates if any product offers have product instances with a status other than Cancelled.

PRODUCT_OFFER_DISPLAY_NAME

Display name for the product offer.

For companion products, this column is populated with the display name of the associated product offer.

PRODUCT_OFFER_INSTANCE_ID

ID of a product offer instance associated with the product offer.

PARENT_PROD_OFFER_INSTANCE_ID

ID of the parent product offer instance.

DEFAULT_IND_CODE

Indicates if the companion product or child offer is selected by default by an external order management system when the parent is selected.

If MANDATORY_IND_CODE is 1, it is undefined .
