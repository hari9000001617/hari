CONTRACT_TRE_V

The CONTRACT_TRE_V view defines the accessible set of columns for searching on, inserting, and updating contracts.

Column Definitions

Column Name

Properties

Description

CONTRACT_ID

NUMBER (15)

Not null

Read-only

ID of a contract.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that this row was last modified.

CONTRACT_NAME

VARCHAR (40)

Not null

Read-only

Name for the contract.

It is the external ID for the contract.

It is automatically assigned on contract creation using the contract type's entity validation.

CUSTOMER_REFERENCE

VARCHAR (40)

Customer reference for the contract.

OPERATOR_ID

NUMBER (8)

Not null

Read-only

User who last modified the record.

It is automatically populated for inserts or updates.

GROUP_ID

NUMBER (8)

Not null

Group associated with the record.

It controls access to the record.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

It is set to StartDate~ for a new contract.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

It defaults to the end of time for a new contract.

CONTRACT_TYPE_NAME

VARCHAR (30)

Not null

Read-only

Type of contract of the account.

It controls the products that can be associated with the contract, and also defaults many characteristics when the contract record is created.

It cannot be specified if CONTRACT_TYPE_ID is specified.

CONTRACT_TYPE_DISPLAY_NAME

VARCHAR (60)

Not null

Name of contract type displayed.

It defaults to CONTRACT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

CONTRACT_TYPE_ID

NUMBER (8)

Not null

Read-only

Type of contract of the account.

It controls the products that can be associated with the contract, and also defaults many characteristics when the contract record is created.

It cannot be specified if CONTRACT_TYPE_NAME is specified.

ICON_ID

NUMBER (8)

Not null

Read-only

Icon associated with contracts of this type.

CONTRACT_FORM_CODE

NUMBER (3)

Not null

Read-only

Form used to maintain the details of this type of contract.

It is based on the CONTRACT_FORM reference type.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

Read-only

Details of the root customer node that have been entered into the contract.

It must exist as a root node for at least the effective date range of the record.

Note:

It cannot be modified using the biContractUpdate&() interface. It can be modified indirectly using customer node moves.

PERSON_ID

NUMBER (15)

Not null

Primary contact person for the contract.

Any correspondence that relates to the contract is addressed to this person.

It defaults to the primary contact of the root customer node.

ACCOUNT_MANAGER_OPERATOR_ID

NUMBER (8)

Account manager for the contract.

Account managers must be defined as users of the system.

It defaults from the contract type.

ASSIGNED_CSR_OPERATOR_ID

NUMBER (8)

User who nominally maintains the contract.

It defaults to who created the contract.

CONTRACT_STATUS_CODE

NUMBER (8)

Not null

Status of the contract. Valid values are:

  • 1 (In Negotiation)
  • 2 (Pending)
  • 3 (Active)
  • 8 (In Grace)

9 (Cancelled).

Values are client-configurable.

It defaults from the CONTRACT_TYPE.

Note:

Of these values, only the statuses of Active and Cancelled have significance. If a contract is Active, all companion product instances associated with the contract must also be Active for the Active date range of the contract if their base product instance is Active. Otherwise, they must have a status of Provisioned, or the same status as their base product instance. If a contract is cancelled, all companion product instances associated with the contract must also have a status of Cancelled. There is no relationship between contract status and the status of base product instances or services associated with the contract.

It is based on the CONTRACT_STATUS reference type.

START_DATE

DATE

Set to the date-time that the contract first became active for the customer.

The following rules apply:

  • The contract must have a status of Active at this date-time.
  • It is mandatory if the current contract's status is Active.
  • It should default or be updated if the contract status is updated to Active and this column is null.

    The default value should be set to the current date and time between EFFECTIVE_START_DATE and EFFECTIVE_END_DATE; otherwise, its default value should be set to EFFECTIVE_START_DATE.

CONTRACT_DURATION

NUMBER (4)

Duration of the contract.

The default value is based on the contract type.

If specified, EXPECTED_EXPIRY_DATE is derived using the formula:

EXPECTED_EXPIRY_DATE = START_DATE + CONTRACT_DURATION

It is enforced by check constraint C_CONTRACT_HISTORY.

If EXPECTED_EXPIRY_DATE is modified, it is cleared.

It cannot be modified if the contract type has a FIXED_DURATION_IND_CODE of 1.

CONTRACT_DURATION_UNIT_CODE

NUMBER (2)

Units of the contract duration. Valid values are:

  • 0 (Days)
  • 1 (Weeks)
  • 2 (Months)
  • 3 (Years).

The defaults are based on the contract type.

It cannot be modified if the contract type has a FIXED_DURATION_IND_CODE of 1.

It is based on the CONTRACT_DURATION_UNIT reference type.

EXPECTED_EXPIRY_DATE

DATE

Date-time that the contract is expected to expire.

It is derived from the contract start date and contract duration if the contract duration is specified.

ACTUAL_EXPIRY_DATE

DATE

Date-time that the contract was terminated.

The contract, if it exists, must have a status of Cancelled from this date-time (due to archiving it is not a requirement that the contract exist at this date-time). This is checked by the DVP.

It is set automatically by the server (based on the passed EFFECTIVE_START_DATE) when a contract is updated with a status of Cancelled.

It cannot be manually specified as a parameter to the biContractInsert&() or biContractUpdate&() functions.

A user or the system can terminate a contract early, because of failure of the customer to meet contract conditions or changes in customer requirements. The system can impose penalties for early contract termination.

The scheduler may set the date to a later value than the expected date due to the contract type having a grace period.

COMMENTS

VARCHAR (2000)

General field for storing information about the contract.

It is used only for complex or non-standard contracts.

OVERIDDEN_CONTRACTS

VARCHAR (2000)

Free form text list of contract entries that have been superseded by the contract.

RELATED_DOCUMENTS

VARCHAR (2000)

Free form text list of document entries, which relate to the contract.

SALES_CHANNEL_CODE

NUMBER (8)

Sales channel from which the contract was obtained.

It defaults from the contract type.

It is based on the SALES_CHANNEL reference type.

RENEGOTIATE_DATE

DATE

Date-time at or after which a 'contract to be re-negotiated' reminder is sent to the account manager.

It can be derived from EXPECTED_EXPIRY_DATE and the contract type, and manually modified.

It is re-derived whenever EXPECTED_EXPIRY_DATE is modified.

It must be null if EXPECTED_EXPIRY_DATE is null.

RENEGOTIATE_REMINDER_DATE

DATE

Read-only

Date-time that the last 'contract to be re-negotiated' reminder was sent to the account manager.

It is updated by the scheduler and cannot be modified by users.

EXPIRY_WARNING_DATE

DATE

Date-time at or after which a 'contract is about to expire' reminder is sent to the account manager.

It can be derived from EXPECTED_EXPIRY_DATE and the contract type.

It can also be manually modified by users.

It is re-derived whenever EXPECTED_EXPIRY_DATE is modified.

It must be null if EXPECTED_EXPIRY_DATE is null.

EXPIRY_WARNING_REMINDER_DATE

DATE

Read-only

Date-time that the last 'contract is about to expire' reminder was sent to the account manager.

It is updated by the scheduler and cannot be modified by users.

GRACE_EXPIRY_DATE

DATE

Date-time after the EXPECTED_EXPIRY_DATE that a contract is in a Cancelled status and cannot be re-negotiated.

It can be derived from EXPECTED_EXPIRY_DATE and the contract type.

It can be manually modified by a user.

It is re-derived whenever EXPECTED_EXPIRY_DATE is modified.

The schedules responsible for sending reminder messages are configurable to cancel any contracts that have passed their GRACE_EXPIRY_DATE. If a contract does not have a GRACE_EXPIRY_DATE, it is cancelled as soon as its EXPECTED_EXPIRY_DATE has elapsed.

It must be null if EXPECTED_EXPIRY_DATE is null.

SELECT_ALL_IND_CODE

NUMBER (1)

Not null

If set to 1 (True), the contract is associated with all services in the customer hierarchy that are available for selection (based on base products specified for the contract's type) and are not associated with a non-cancelled contract.

Any new service created with compatible base products is automatically added to the contract while its status is not cancelled.

It defaults to 0 (False).

It can be modified. Setting it from 1 to 0 causes all services associated with the contract to remain associated with the contract.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields for entering contract related details.

The contents and validation of these fields is specified by the entity validation associated with the contract type.
