PROGRESS_TRE_V

The PROGRESS_TRE_V view defines the available details for query progressions.

Column Definitions

Column Name

Properties

Description

CUSTOMER_QUERY_ID

NUMBER (15)

Not null

ID of a customer query.

LAST_MODIFIED

DATE

Not null

Date-time that the progression was last modified.

It is automatically maintained by the server.

PROGRESS_SEQNR

NUMBER (8)

Not null

Read-only

ID of the progress of the customer query.

It starts at 1 for a new customer query, and increments by 1 for each successive progression.

OPEN_QUERY_IND_CODE

NUMBER (1)

Set to 1 (TRUE) while the query is current for this work group; otherwise, set to null when the query has been processed by the work group.

It is indexed to speed retrieval time.

It is automatically maintained by the server and cannot be directly modified by updates.

WORK_FORCE_NAME

VARCHAR (30)

Not null

Read-only

Work group responsible for processing or progressing the query.

Note:

Only work groups to which the user has access can be viewed.

WORK_FORCE_DISPLAY_
NAME

VARCHAR (60)

Not null

Applies only to the work group responsible for processing or progressing the query.

It defaults to WORK_FORCE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore may differ between installations.

WORK_FORCE_ID

NUMBER (8)

Not null

Read-only

Work group responsible for processing or progressing the query.

Note:

Only work groups to which the user has access can be viewed.

OPEN_DATE

DATE

Not null

Read-only

Date-time that the query was directed to the work group.

RESPONSE_DUE_DATE

DATE

Not null

Date-time by which a response is due to the query by the work group.

PROGRESS_TEXT

VARCHAR (4000)

Redundant

Current progress of the query.

If PROGRESS_SEQNR is 1, this is CUSTOMER_QUERY.QUERY_TEXT. Otherwise, it is the RESOLUTION_TEXT of the previous progress for the query.

The redundancy is performed to simplify the display of relevant details to a work group. For simple queries, this can be null.

The query type and the person, customer, or service details may provide sufficient information.

ACTION_DATE

DATE

Read-only

Date-time that this work group completed its actions on the query.

It must be null if OPEN_QUERY_IND_CODE is 1, and must contain a not null value if OPEN_QUERY_IND_CODE is null.

ACTION_TEXT

VARCHAR (4000)

Action performed by this work group to complete its progression of the query.

OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified the progression.

It is automatically set and cannot be directly modified.

RESOLUTION_TEXT

VARCHAR (4000)

Text describing the resolution of the query.

It defaults to ACTION_TEXT, unless the work group does not close the query.

If the query is not closed, it should indicate the proposed actions required by the next work group to which this query is assigned.
