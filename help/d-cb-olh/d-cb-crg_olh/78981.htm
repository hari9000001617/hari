SUBTOTAL_TERM_TRE_V

The SUBTOTAL_TERM_TRE_V view contains the terms that are aggregated as part of a subtotal, and the conditions under which they are aggregated.

Subtotal terms can be updated using the biSubtotalTermsUpdate&() function.

Column Definitions

Column Name

Properties

Description

SUBTOTAL_ID

NUMBER (8)

Not null

ID of the subtotal.

SEQR

NUMBER (4)

Not null

Sequence number of the subtotal term instance.

LAST_MODIFIED

DATE

Not null

Date-time at which this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

It must have the same effective start date as a row in SUBTOTAL_HISTORY.

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

It must have the same effective end date as a row in SUBTOTAL_HISTORY.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which the subtotal term is evaluated. Valid values are:

  • 1 (Normalised Event)
  • 2 (Service)
  • 3 (Customer Node)
  • 4 (Customer)
  • 5 (Charge).

The value must be equal or lower than the context of the subtotal, or set to 5 (Charge).

The context of a subtotal term for a subtotal with an application environment of Rating cannot be Customer (4), Customer Node (3), or Service (2).

The CONTEXT_SUBTOTAL reference type define the valid values for this column.

SUBTOTAL_TERM_EXPR

VARCHAR (255)

Expression evaluated for the subtotal term. For example, tariff to be included in the subtotal.

Note:

The data type returned by each subtotal term must be the same as the data type defined for the subtotal.

KEY_EXPR

VARCHAR (255)

Key expression which determines the index (array) or key (hash) value.

It can only contain a value if the data type of the subtotal is array or hash.

If the data type of the subtotal is:

  • Array, this expression must return an integer value greater than or equal to 0.
  • Hash, this expression must return a non-empty string value.

The key expression is evaluated in the same context as the term expression. It has access to the same variables and functions as the subtotal term expression.

ELIGIBILITY_EXPRESSION_LIST_ID

NUMBER (8)

List of conditional expressions that must be satisfied for each instance of the variable to be aggregated into the subtotal.

These expressions are evaluated in the same context as the subtotal term expression, and have access to the same variables and functions as its subtotal term expression.

DO_NOT_FIX_DEPENDENCIES

NUMBER (1)

Pseudo-column

Update only

If set to 1 (TRUE), variable dependencies are not fixed.

It is used for multi-phase tariff updates to delay fixing of variable dependencies.
