UploadDirectories

The UploadDirectories derived attribute table defines the logical upload directories available for secure file transfer.

Each logical upload directory is mapped to a physical directory on one or more servers. Access to each logical upload directory is controlled by the user's security group.

Caution:

Normal users of the system should not be given permission to update or insert rows in this table.

The table has an evaluation type of Table Lookup By Multiple (1-10) Keys.

Column Definitions

Column Name

Properties

Description

UploadDirectory (Index 1)

String

Not null

Logical server directory from which files are uploaded.

The directory name can include / characters to represent a logical hierarchical path.

Caution:

It is recommended that the specified name not start with a / character

Environment variables are not supported for specifying this column.

Security Group (Result 1)

Integer

Not null

ID of the security group associated with this upload directory.

Only user with insert access to this security group can upload files to this directory.

ServerDirectory (Result 2)

String

Not null

Physical directory on the Singleview server that corresponds to this logical upload directory.

Note:

To be used, the directory must exist on the server. If the directory does not exist, an error occurs on upload.

Environment variables can be used to specify this column.

Files are only uploaded to this directory. To upload files to sub-directories or parent directories, specify separate rows in this table to access them.

Caution:

For security reasons, do not use the same physical directory for downloads and uploads.

Instance (Result 3)

String (9)

Instance name.

If specified, this upload directory exists only on this instance. Upload requests are routed to this instance as required.

If not specified, and a multi-instance deployment is used, the same server directory is assumed available on all instances in the cluster (by the use of a clustered file system or NFS). Upload requests for this directory can be satisfied by any instance in this cluster.

If necessary, use environment variables in the Server Directory column to define a different physical mapping of the server directory for each instance.
