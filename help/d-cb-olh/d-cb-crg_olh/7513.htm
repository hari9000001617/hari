SERVICE_CHARGE_CATEGORY_TRE_V

The SERVICE_CHARGE_CATEGORY_TRE_V view defines the set of accessible columns for inserting service charge category details.

Column Definitions

Column Name

Properties

Description

SERVICE_ID

NUMBER (15)

Not null

ID of a service.

CHARGE_CATEGORY_ID

NUMBER (8)

Not null

ID of a charge category.

SEQNR

NUMBER (3)

Not null

Sequence number for this combination of service and charge category.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

It is always equal to the EFFECTIVE_START_DATE of the associated service history record. This is checked by the DVP.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second. It must be greater than or equal to EFFECTIVE_START_DATE.

It is always equal to the EFFECTIVE_END_DATE of the associated service history record. This is checked by the DVP.

PROPORTION

NUMBER (10,6)

Not null

Percentage of the charges of the charge category for the service when apportioned to the following GL codes and accounts.

It defaults to 1.

For a specific service and charge category, the sum of the proportions must be 1.

FROM_ACCOUNT_ID

NUMBER (15)

Account from which the specified proportion of charges associated with the category for the service transferred.

FROM_GL_CODE_ID

NUMBER (8)

GL code from which the specified proportion of charges associated with the category for the service are transferred.

It must exist for at least this date range.

FROM_ACCOUNT_CLASS_CODE

NUMBER (2)

Account class associated with the FROM_ACCOUNT_ID. Examples are:

  • 1 (Asset)
  • 2 (Liability).

It is automatically derived by the service if not specified in an update or insert operation.

It is based on the ACCOUNT_CLASS reference type.

TO_ACCOUNT_ID

NUMBER (15)

Not null

Account to which charges are to be applied.

It is usually the customer node's prime account. However, for large corporate customers, it may be a specific account used for receiving charges of this type.

TO_GL_CODE_ID

NUMBER (8)

GL code to which any charges associated with the tariff is accredited.

It must exist for at least the date range.

TO_ACCOUNT_CLASS_CODE

NUMBER (2)

Not null

Derived

Account class associated with the TO_ACCOUNT_ID.

It is automatically derived by the service if not specified in an update or insert operation.

It is based on the ACCOUNT_CLASS reference type.
