INVOICE_TRANSACTION_TRE_V

The INVOICE_TRANSACTION_TRE_V view provides a list of payments and adjustments displayed on an invoice (not payments and adjustments applied to the invoice).

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

ID of the customer node invoice.

PAYMENT_ID

NUMBER (15)

ID of a payment included on the invoice.

ADJUSTMENT_ID

NUMBER (15)

ID of an adjustment included on the invoice.

TRANSACTION_DATE

DATE

Not null

Effective date on which this transaction was applied to the customer's account.

PAYMENT_BATCH_ID

NUMBER (15)

ID that defines a payment batch.

RECEIPT_NR

VARCHAR (30)

Official receipt number that is assigned to a payment.

PAYMENT_DATE

DATE

Date-time the payment was entered into the system.

PAYMENT_RECEIVED_DATE

DATE

Date-time the payment was received.

PAYMENT_FROM_ACCOUNT_ID

NUMBER (15)

Account the payment is deducted from.

PAYMENT_TO_ACCOUNT_ID

NUMBER (15)

Account the payment is applied to.

PAYMENT_INVOICE_CODE

NUMBER (2)

Summary of details in the PAYMENT_INVOICE table for the payment. Valid values are:

  • 0 (Invoice)
  • 1 (Split)
  • 2 (Credit).

It is based on the PAYMENT_INVOICE reference type.

PAYMENT_INVOICE_ID

NUMBER (15)

Derived

If the PAYMENT_INVOICE_CODE is 0, this is the ID of the single invoice in the PAYMENT_INVOICE table for the payment.

PAYMENT_AMOUNT

NUMBER (19,3)

Amount of the payment in the currency of the account.

PAYMENT_LOCATION_CODE

NUMBER (8)

Payment location category identifying how the payments are received (for example, at a customer service centre or bank, or by post).

It is based on the PAYMENT_LOCATION reference type.

PAYMENT_TYPE_NAME

VARCHAR (40)

Name of the payment method type.

PAYMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Name of the payment method type displayed.

It defaults to PAYMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

PAYMENT_TYPE_ID

NUMBER (8)

ID of the method of payment.

PAYMENT_STATUS_CODE

NUMBER (4)

Status of the payment. Valid values are:

  • 1 (Accepted)
  • 2 (Rejected).

It is based on the PAYMENT_STATUS reference type.

REJECTED_DATE

DATE

If the payment was rejected, the date-time on which the rejection occurred.

PAYMENT_REJECTED_CODE

NUMBER (8)

If the payment is rejected, it describes the reason for rejection (for example, a missing signature, inadequate funds, or exceeding the credit limit).

It is based on the PAYMENT_REJECTED reference type.

ADJUSTMENT_NR

VARCHAR (30)

ID of the adjustment, which is used as a reference.

ADJUSTMENT_BATCH_ID

NUMBER (15)

ID of the adjustment batch.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Standard category for the adjustment when entered.

If the adjustment is associated with a batch, the adjustment type defaults to the adjustment type of the batch.

ADJUSTMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Name of adjustment type displayed.

It defaults to ADJUSTMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Standard category for the adjustment when entered.

If the adjustment is associated with a batch, the adjustment type defaults to the adjustment type of the batch.

ADJUSTMENT_DESCRIPTION

VARCHAR (255)

Description of the adjustment.

It is not displayed on the invoice, but entered by the user.

The description may contain the justification for the adjustment.

If associated with a batch, it defaults to the description of the batch.

ADJUSTMENT_FROM_ACCOUNT_
ID

NUMBER (15)

Account from which this adjustment is deducted.

If specified, it defaults based on the adjustment type.

ADJUSTMENT_TO_ACCOUNT_ID

NUMBER (15)

Account to which this adjustment is to be applied.

If associated with a batch adjustment, the currency of the account must be the same as the currency associated with the batch adjustments.

ADJUSTMENT_TEXT

VARCHAR (80)

Text displayed in the summary section of the customer invoice (in addition to the details of the invoice adjustment).

It defaults to the text associated with the adjustment type.

ADJUSTMENT_INVOICE_CODE

NUMBER (2)

Summary of the details in the ADJUSTMENT_INVOICE table for the adjustment. Valid values are:

  • 0 (Invoice): Adjustment is against a single invoice.
  • 1 (Split): Adjustment is against one or more invoices or a direct credit or debit to the account.
  • 2 (Credit): Adjustment is a direct credit to the account. If the amount is negative, it is a debit.

It is based on the PAYMENT_INVOICE reference type.

ADJUSTMENT_INVOICE_ID

NUMBER (15)

If the ADJUSTMENT_INVOICE_CODE is 0, this is the ID of the single invoice in the ADJUSTMENT_INVOICE table for this adjustment.

ADJUSTMENT_AMOUNT

NUMBER (19,3)

Amount of the adjustment in the currency of the account.

A positive adjustment indicates a credit to the customer that must be included on the invoice. For example, if the customer had difficulties with their service in the last invoice period and the service rental charge for that invoice is refunded.

A negative amount indicates a debit.

The currency of the adjustment amount must be the same as the currency of the account.

ADJUSTMENT_DATE

DATE

Date-time the adjustment was made.

ADJUSTMENT_RECEIVED_DATE

DATE

Date-time the adjustment was applied to the account.

ADJUSTMENT_DUE_DATE

DATE

Date-time at which the adjustment was due.

It is usually only set for adjustments representing debit notes.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 controlled by the payment type or adjustment type (depending on whether the record is a payment or adjustment).

Typical values are items such as bank, cheque number, account name, card number, card type, or expiry date.
