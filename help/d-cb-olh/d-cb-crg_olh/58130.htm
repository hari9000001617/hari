PRODUCT_OFFER_HIERARCHY_V

The PRODUCT_OFFER_HIERARCHY_V view defines components of product offer hierarchies. A component can be a product offer or a companion product.

Each row in this view represents one of the following:

  • Product offer (that is, a child product offer) that is part of another product offer (that is, a parent product offer )
  • Product offer that is the root product offer of a product offer hierarchy (that is, it is not a child product offer)
  • Companion product that is part of a product offer.

This view is intended to support CSR operations using graphical representations of product offer hierarchies.

Column Definitions

Column Name

Properties

Description

PRODUCT_OFFER_HIERARCHY_
ID

NUMBER (12)

Not null

Derived

ID for a component (that is, product offer or companion product) of a product offer hierarchy.

PRODUCT_IND_CODE

NUMBER (1)

Not null

If set to:

  • 1 (TRUE), it represents a companion product that is part of a product offer.
  • 0 (FALSE), it represents a product offer (that may or may not be part of another product offer).

PRODUCT_OFFER_NODE_ID

NUMBER (8)

ID for a node in a hierarchy of product offers.

PRODUCT_OFFER_ID identifies the product offer referenced by this product offer node.

It is not NULL if PRODUCT_IND_CODE is 0.

PARENT_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the node of the product offer that contains this component.

It cannot have the same value as PRODUCT_OFFER_NODE_ID.

It is NULL if the product offer identified by PRODUCT_OFFER_ID (if not NULL) is a root product offer of a product offer hierarchy. Otherwise, the product offer identified by PRODUCT_OFFER_ID (if not NULL) must be a child product offer of the product offer associated with the parent node.

It is not NULL if PRODUCT_IND_CODE is 1.

HIERARCHY_LEVEL

NUMBER (8)

Not null

Derived

Level in the hierarchy of this component.

The root node of a hierarchy has a level of 1. Therefore, it must have a value of 1 if PARENT_PRODUCT_OFFER_NODE_ID is NULL, and must have a value greater than 1 otherwise.

If the value is not 1, it must be one greater than the value assigned to the node identified by PARENT_PRODUCT_OFFER_NODE_ID.

PRODUCT_OFFER_ID

NUMBER (8)

ID for a product offer.

It is not NULL if PRODUCT_IND_CODE is 0.

ENTITY_NAME

VARCHAR (60)

Not null

Name for this product offer or companion product.

ENTITY_DISPLAY_NAME

VARCHAR (120)

Not null

Name for the product offer or companion product as displayed to CSRs.

It should not be referenced by configuration code because it can be translated.

LAST_MODIFIED

DATE

Not null

Date-time at which this product offer or companion product was last modified.

OPERATOR_ID

NUMBER (8)

Not null

Operator who last modified this product offer or companion product.

GROUP_ID

NUMBER (8)

Not null

Operator group that has access to this product offer or companion product.

DESCRIPTION

VARCHAR (4000)

Not null

Description of the product offer or companion product.

AVAILABLE_START_DATE

DATE

Date-time at which this product offer becomes available to be used.

If PARENT_PRODUCT_OFFER_NODE_ID is NULL, this date-time is identical to the start date-time of the availability period of this product offer.

It is not NULL if PRODUCT_IND_CODE is 0.

If this product offer node has a parent product offer node (that is, PARENT_PRODUCT_OFFER_NODE_ID is not NULL), the following constraints apply if this product offer is:

  • A mandatory child product offer, the value of this column is the same as the start date/time of the availability period of the parent product offer node.
  • Not a mandatory child product offer, the value of this column is the later of two date-times; that is, the start date-time of the availability period of this product offer, and the start date-time of the availability period of the parent product offer node.

AVAILABLE_END_DATE

DATE

Date-time after which the product offer is no longer available.

If this product offer has no parent node, this date-time is identical to the end date-time of the availability period of this product offer.

It is not NULL if PRODUCT_IND_CODE is 0.

If this product offer node has a parent product offer node (that is, PARENT_PRODUCT_OFFER_NODE_ID is not NULL), and if this product offer is:

  • A Mandatory child product offer, the value of this column is the same as the end date-time of the availability period of the parent product offer node.
  • Not a mandatory child product offer, this date-time is the earlier of two date-times; that is the end date-time of the availability period of this product offer, and the end date-time of the availability period of the parent product offer node. The value of this column can be earlier than AVAILABLE_START_DATE. If so, this product offer cannot be selected as part of its parent product offer.

BASE_PRODUCT_ID

NUMBER (8)

ID of a base product with which this product offer is compatible.

It must be that of a base product for at least the availability period of the product offer.

It must be NULL for a child product offer (and therefore must be NULL for a product offer that is service-specific or hidden), or if PRODUCT_IND_CODE is 1.

It must not be NULL for a product offer that has no parent product offer. This constraint may be violated temporarily for example, while the product offer is pending assignment as a child product offer of another product offer.

PRODUCT_OFFER_SKU

VARCHAR (40)

Stock-keeping unit (SKU) for this product offer (if PRODUCT_IND_CODE is 0), or for the product offer that includes this companion product (if PRODUCT_IND_CODE is 1).

This is a short code that can be used by CSRs to quickly search for a product offer.

No two product offers with the same TENANT_ID can have the same PRODUCT_OFFER_SKU.

CHILD_PRODUCT_OFFER_MIN

NUMBER (4)

Minimum number of optional child product offers that must be selected as part of this product offer whenever it is selected for use.

It is checked during completion of a product instance.

It is NULL if PRODUCT_IND_CODE is 1.

If it contains a value, it must be greater than or equal to 0.

CHILD_PRODUCT_OFFER_MAX

NUMBER (4)

Maximum number of optional child product offers that can be selected as part of this product offer whenever it is selected for use.

It is checked during completion of a product instance.

It is NULL if PRODUCT_IND_CODE is 1.

It contains a value, it must be greater than or equal to CHILD_PRODUCT_OFFER_MIN (if specified).

COMPANION_PRODUCT_MIN

NUMBER (4)

Minimum number of optional companion products that must be selected as part of this product offer whenever it is selected for use.

It is checked during completion of a product instance. It is NULL if PRODUCT_IND_CODE is 1.

If it contains a value, it must be greater than or equal to 0.

COMPANION_PRODUCT_MAX

NUMBER (4)

Maximum number of optional companion products that can be selected as part of this product offer whenever it is selected for use.

It is checked during completion of a product instance. It is NULL if PRODUCT_IND_CODE is 1.

If it contains a value, it must be greater than or equal to COMPANION_PRODUCT_MIN (if specified).

CATEGORY_CODE

NUMBER (4)

Category to which this product offer belongs (if PRODUCT_IND_CODE is 0), or to which the product offer that includes this companion product belongs (if PRODUCT_IND_CODE is 1).

The BASE_INSTALL reference type PRODUCT_OFFER_CATEGORY defines the possible values.

SUB_CATEGORY_CODE

NUMBER (4)

Sub-category to which this product offer belongs (if PRODUCT_IND_CODE is 0), or to which the product offer that includes this companion product belongs (if PRODUCT_IND_CODE is 1).

The BASE_INSTALL reference type PRODUCT_OFFER_SUB_CATEGORY defines the possible values.

It can only contain a value if CATEGORY_CODE is not NULL.

CUSTOMER_NODE_TYPE_ID

NUMBER (8)

Can be used to constrain the selection of product offers that can be assigned to product instances belonging to a customer node of this type.

It is a selection criterion only; it is not mandated that only customer nodes of this type can have product instances that use this product offer.

It refers to the product offer containing this companion product, if PRODUCT_IND_CODE is 1.

SERVICE_SPECIFIC_IND_
CODE

NUMBER (1)

Not null

If set to 1 (TRUE), this product offer is a service-specific product offer and can only contain other service-specific product offers or service-specific companion products, and the product offer must be compatible with at least one service type.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 1; otherwise, it can have a value of either 0 or 1.

HIDDEN_IND_CODE

NUMBER (1)

Not null

If set to 1 (TRUE), this product offer is not displayed, and the following constraints apply:

  • This product offer cannot be an optional child product offer.
  • This product offer cannot include any optional companion products or optional child product offers.
  • Each descendent product offer of this product offer must also have a value of 1 (TRUE) for HIDDEN_IND_CODE.

It is usually set to 0 (FALSE) for a product offer that has no parent product offer but this may not apply for a product offer that is pending assignment as a child product offer of another product offer.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 1; otherwise, it can have a value of either 0 or 1.

AUTO_EXPAND_HIERARCHY_
IND_CODE

NUMBER (1)

Not null

If set to 1 (TRUE), any product offer hierarchy descended from this product offer is automatically displayed in an expanded mode by a GUI.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 1; otherwise, it can have a value of either 0 or 1.

VERSION_STR

VARCHAR (255)

Not null

Optional version identification string that can be used to tag this product offer.

It is provided for use by third party configuration management tools and is not directly used by the Convergent Billing.

It is NULL if PRODUCT_IND_CODE is 1.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this product offer.

See CATEGORY_V view for the list of available categories.

It is NULL if PRODUCT_IND_CODE is 1.

PRODUCT_OFFER_HELP_URL

VARCHAR2 (4000)

Optional Uniform Resource Locator (URL) of a resource (for example HTML document) that provides CSRs with more information about this product offer.

It is NULL if PRODUCT_IND_CODE is 1.

COMPONENT_DESCRIPTION

VARCHAR (255)

Optional description of the nature of the association between this component and the product offer to which it belongs.

It is NULL if PARENT_PRODUCT_OFFER_NODE_ID is NULL.

PRODUCT_OFFER_SEQNR

NUMBER (4)

Sequence number that controls the display order of this child product offer for this product offer.

It is not NULL if PRODUCT_IND_CODE is 0 and PARENT_PRODUCT_OFFER_NODE_ID is not NULL.

PRODUCT_OFFER_MAND_IND_
CODE

NUMBER (1)

Not null

If it is set to 1 (TRUE), this child product offer is a mandatory component of its parent product offer.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 1, or if PARENT_PRODUCT_OFFER_NODE_ID is NULL; otherwise, it can have a value of 0 or 1.

OFFER_MIN

NUMBER (4)

Not null Derived

Minimum number of times the same offer can be sold under each instance of a product offer.

If OFFER_MIN is:

  • Specified, the value must be greater than or equal to 0.
  • Not specified, the value is derived from MANDATORY_IND_CODE. If MANDATORY_IND_CODE is:
    • NULL, OFFER_MIN defaults to 0.
    • Not NULL, OFFER_MIN defaults to 1.

OFFER_MAX

NUMBER (4)

Maximum number of times the same offer can be sold under each instance of a product offer.

If OFFER_MAX is:

  • Specified, its value must be greater than or equal to OFFER_MIN, or NULL to indicate no maximum.
  • Not specified, its value defaults to 1 when OFFER_MIN is 0; otherwise, the value of OFFER_MIN.

PRODUCT_ID

NUMBER (8)

ID of a companion product that is part of the product offer that belongs to the product offer node identified by PARENT_PRODUCT_OFFER_NODE_ID.

The companion product is compatible with all other companion products that are part of this product offer.

ID must be that of a companion product for at least the availability period of the product offer.

It is not NULL if PRODUCT_IND_CODE is 1.

PRODUCT_SEQNR

NUMBER (4)

Sequence number that controls display order of this product in this product offer.

It is not NULL if PRODUCT_IND_CODE is 1.

PRODUCT_MANDATORY_IND_
CODE

NUMBER (1)

Not null

If set to:

  • 1 (TRUE), this companion product must be instantiated for a base product instance that is assigned either this product offer or a product offer that includes this product offer as a child product offer.
  • 0 (FALSE), the inclusion of this companion product is optional.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 0; otherwise, it can have a value of 0 or 1.

PRODUCT_MULTIPLE_IND_
CODE

NUMBER (1)

Not null

If set to:

  • 1 (TRUE), there can be more than one instance of this companion product active on a base product instance.
  • 0 (FALSE), only one instance of this companion product can be active on a base product instance.

It has a value of 0 (FALSE) if PRODUCT_IND_CODE is 0; otherwise, it can be 0 or 1.

PRODUCT_GENERAL_1 - PRODUCT_GENERAL_10

VARCHAR (255)

General fields 1 - 10.

They contain values that override default values for GENERAL_1 to GENERAL_10 for an instance of this companion product that is created as part of this product offer.

If PRODUCT_IND_CODE is 0, these fields are NULL .

DEFAULT_IND_CODE

NUMBER (1)

If set to 1 (TRUE), the component is automatically selected by default when ordering the product offer. The component can be deselected during the ordering process.

If it is not NULL, it must be 1 (TRUE).

If the component is mandatory, it must be NULL.
