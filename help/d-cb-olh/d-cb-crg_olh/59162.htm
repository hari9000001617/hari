ProductInstanceSvcEquipRange?[]()

Syntax

ProductInstanceSvcEquipRange?[](
    ProductInstanceId&,
    EffectiveDate~)

 

Description

Retrieves from the database the ranges of services and equipment items supported by the specified base product instance as at a specific date-time, adjusts these ranges based on definitions of the specified companion products, if appropriate, and returns them.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

ProductInstanceId&

In: ID of the base product.

EffectiveDate~

In: Effective date-time of database retrieval.

 

Returns

Array of unknown hashes. Each hash represents either a range of supported:

  • Services for a specific SERVICE_ITEM_SEQNR for the base product.
  • Equipment items for a specific combination of SERVICE_ITEM_SEQNR and EQUIPMENT_ITEM_SEQNR for the base product.

Each hash contains values of the following columns for a row in the PRODUCT_ORDER_V view, keyed by column name:

  • CREATION_ORDER
  • NETWORK_NAME_EQUIPMENT_CO
  • NETWORK_NAME_SERIALNR_IND_CODE
  • PRODUCT_SERVICE_TYPE_ID
  • SERVICE_ITEM_SEQNR
  • SERVICE_NAME_EQUIPMENT_CO
  • SERVICE_NAME_SERIALNR_IND_CODE
  • SERVICE_RANGE_MAX
  • SERVICE_RANGE_MIN
  • SERVICE_TYPE_ID.

For an equipment item, the hash also contains values for the following keys:

  • EQUIPMENT_ITEM_SEQNR
  • EQUIPMENT_RANGE_MAX
  • EQUIPMENT_RANGE_MIN
  • EQUIPMENT_TYPE_ID.

Remarks

Ranges of supported:

  • Services are specified by the SERVICE_RANGE_MIN and SERVICE_RANGE_MAX columns in the PRODUCT_SERVICE_TYPE table.
  • Equipment items are defined by the EQUIPMENT_RANGE_MIN and EQUIPMENT_RANGE_MAX columns in the PRODUCT_SERVICE_TYPE_EQUIP table.

The function returns an array that indicates the range of services and, for each service, the ranges of all related equipment items, which can be associated with an instance of the base product. If no companion products are supplied, the ranges are as defined by the definition of the base product; otherwise, the ranges can be altered by the definitions of the specified companion products. The returned array can be empty, indicating that no services or equipment are associated with an instance of the base product.

If two or more companion products specify different ranges for the same service type, the most restrictive range is used.

For example, if one companion product has SERVICE_RANGE_MIN of 2 and SERVICE_RANGE_MAX of 4, and another companion product for the same service type has SERVICE_RANGE_MIN of 3 and SERVICE_RANGE_MAX of 6, the resultant range is:

  • SERVICE_RANGE_MIN of 3, the maximum of the SERVICE_RANGE_MIN values.
  • SERVICE_RANGE_MAX of 4, the minimum of the SERVICE_RANGE_MAX values.

If the companion products do not specify one end of the range (that is, the SERVICE_RANGE_MIN or SERVICE_RANGE_MAX is undefined), the appropriate limit from the base product definition is used. If the resultant range has a minimum that is greater than its maximum, no service of that type is supported.

Equivalent rules apply to ranges for equipment types.

This function is called before instantiating a base product and its companion products, to determine the minimum and maximum number of services and equipment items created or allocated.

If BaseProductId& or any of the IDs in CompanionProductIds&[] does not identify a product as at the specified date-time, this function raises an error. It does not check that the supplied companion products are compatible either with each other or with the specified base product.
