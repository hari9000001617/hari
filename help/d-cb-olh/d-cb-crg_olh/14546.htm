DerivedTableCacheStatistics?{}()

Syntax

DerivedTableCacheStatistics?{}(
    const Cache$)

 

Description

Returns global derived attribute cache statistics for either the process cache (the process evaluating the function), or for the shared memory cache common to all processes on the instance.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

Cache$

In: Identifies whether the statistics are obtained for the process cache or the shared memory cache.

Valid values are:

  • Process (for global derived attribute tables)
  • Shared (for global-shared derived attribute tables).

 

Returns

Unknown hash containing the statistics of the requested cache if successful; otherwise, an error is raised.

Remarks

Each entry in the array Statistics?{'CachedTables'}[] that is returned in the unknown hash correlates to a derived attribute definition in the cache. Each array entry represents:

  • For untenanted derived attributes, a derived attribute table because there is a one-to-one relationship between untenanted derived attribute definitions and their tables.
  • For tenanted derived attributes, the rolled-up statistics of all tenanted derived attribute tables for the derived attribute definition's date range.

Note:

  • The statistics in Statistics?{'CachedTables'}[] that are described in the tables below as being at the derived attribute table level are at the derived attribute definition level.
  • The {'DateRangeArray'}[] array for each Statistics?{'CachedTables'}[] array entry can be used to identify the number of tenant-specific derived attribute tables cached for a tenanted derived attribute definition date range.
