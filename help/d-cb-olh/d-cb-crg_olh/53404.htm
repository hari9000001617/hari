TEMPLATE_TRE_V

The TEMPLATE_TRE_V view contains all details of templates that are known to the system. Templates are used by the IGP.

Column Definitions

Column Name

Properties

Description

TEMPLATE_ID

NUMBER (8)

Not null

ID of the template.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

The granularity of this column is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

The granularity of this column is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

TEMPLATE_NAME

VARCHAR (60)

Not null

Name of the template.

DESCRIPTION

VARCHAR (255)

Not null

Description of the template.

LAST_MODIFIED

DATE

Not null

Date-time at which this row was last modified.

OPERATOR_ID

NUMBER (8)

Not null

User that created or last modified this template.

GROUP_ID

NUMBER (8)

Not null

Group associated with, and controlling access to, this template.

VERSION_STR

VARCHAR (255)

Version ID used to tag this entity.

It is provided for use by third party configuration tools, and is not used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this entity.

Refer to the CATEGORY_V view for the list of available categories.

TEMPLATE_STATUS_CODE

NUMBER (2)

Not null

Status of this instance of the template. Valid values are:

  • 0 (Cancelled).
  • 1 (Active).

It is based on the TEMPLATE_STATUS reference type.

TEMPLATE_TYPE_CODE

NUMBER (4)

Not null

Type of template.

For invoice-related templates, this value defines the underlying view used to obtain the data and the WHERE_CLAUSE used to restrict the rows returned.

It is based on the TEMPLATE_TYPE reference type.

TEMPLATE_CONTEXT_CODE

NUMBER (2)

Redundant

Default context for processing the template.

It is based on the TEMPLATE_CONTEXT reference type.

VIEW_NAME

VARCHAR (30)

Name of the view associated with the template type.

The table below lists associations of views and template types.

WHERE_CLAUSE

VARCHAR (2000)

If specified, replaces the default WHERE_CLAUSE for the template type.

Parameters are specified within the WHERE_CLAUSE as expressions using the /+ <expression> +/ syntax that is used in templates.

It must be validated against the specified VIEW_NAME.

Parameter expressions must be validated within the template.

ORDER_BY_CLAUSE

VARCHAR (255)

ORDER BY clause.

If the view can return multiple rows, this clause defines the order in which the rows are retrieved.

It must be null if VIEW_NAME is null.

ELIGIBILITY_EXPR_LIST_
ID

NUMBER (8)

List of eligibility criteria before a template is created.

POST_ELIGIBILITY_EXPR_
LIST_ID

NUMBER (8)

List of eligibility criteria that are evaluated after the template is created.

TEMPLATE

CLOB

Original merge document.

It must be specified unless TEMPLATE_TYPE_CODE is set to 12.

STORED_IMAGE_TYPE_CODE

NUMBER (2)

Type of image generated from this template.

WORD_ORIGINAL_TEMPLATE

BLOB

Original merge document saved from Microsoft Word.

It must be specified unless TEMPLATE_TYPE_CODE is set to 12.

WORD_IGP_TEMPLATE

BLOB

Derived

Template definition used by the IGP.

It is only applicable if TEMPLATE_TYPE_CODE is set to 12.

It is re-derived when WORD_ORIGINAL_TEMPLATE is updated.

 

Template Types and Associated View Name

TEMPLATE_TYPE_CODE

View Name

Condition

0 - 8

Defaults but can be specified.

If the view is specified and the WHERE_CLAUSE is null, it must contain columns required by TEMPLATE_TYPE_CODE.

If a WHERE_CLAUSE is specified, it must be valid for the view.

9

No default.

If the WHERE_CLAUSE is null, the view must contain a TEMP_DATA_ID column.

If a WHERE_CLAUSE is specified, it must be valid for the view.

10

Defaults.

The view must contain an ADJUSTMENT_ID column.

11

Can be null.

If a VIEW_NAME is specified, a WHERE_CLAUSE must be also specified.

12

Must be null.

The WHERE_CLAUSE column must be null.
