biCustomerNodeContactUpdate&()

Syntax

biCustomerNodeContactUpdate&(
    CustomerNodeId&,
    OldEffectiveDate~,
    var LastModified~,
    EffectiveStartDate~,
    EffectiveEndDate~,
    Opcode&[],
    PersonId&[],
    FieldNames$[],
    FieldValues?[])

 

Description

Updates the contacts for the specified customer node contacts other than the primary contact.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

CustomerNodeId&

In: ID of the customer node whose contact details to be updated.

OldEffectiveDate~

In: Date-time used for retrieving customer node details.

LastModified~

In: Last modified date-time of the CUSTOMER_NODE_HISTORY row active at OldEffectiveDate.

Out: Last modified date-time of the CUSTOMER_NODE_HISTORY row active at EffectiveStartDate.

EffectiveStartDate~

In: Start of the date-time range for which changes are applied.

EffectiveEndDate~

In: End of the date-time range for which changes are applied.

Opcode&[]

Operations to perform on the row.

PersonId&[]

In: Array of person IDs whose contact details are updated.

The number of values in the Opcode&[] array must equal the number of values in the PersonId&[] array.

FieldNames$[]

In: Array of fields updated from CUSTOMER_NODE_CONTACT_TRE_V view.

FieldValues?[]

In: Two-dimensional array of field values.

 

Returns

Number of rows successfully processed, which should equal the size of Opcode&[]; otherwise, an error is raised.

Remarks

A last modified check and date-ranged update is automatically performed on the CUSTOMER_NODE_HISTORY record associated with this change, with the updated CUSTOMER_NODE_HISTORY last modified date-time being returned in LastModified~.

A date range update is also performed on the CUSTOMER_CONTACT table.  Multi-range updates are not supported; if either EffectiveStartDate~ or EffectiveEndDate~ fall outside the CUSTOMER_NODE_HISTORY record's date range, an error is raised.

A new contact that is a duplicate of the customer node's primary contact cannot be inserted.

The number of person IDs for an effective date-time determines the number of rows in the corresponding variable; the row number is the index. The first dimension of FieldValues?[] corresponds to the index of PersonId&[] and Opcode&[], and the second dimension corresponds to the index of FieldNames$[].

The effective user must have update security group access level for the group of the customer associated with the record manipulated by this function call. If the effective date of the update is less than the current date (00:00:00 timestamp), direct update security group access level is required.

GROUP_ACCESS_TRE_V view lists security group access level values.
