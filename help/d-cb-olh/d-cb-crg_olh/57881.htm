WSAEmail

The WSAEmail interface type is used for generating configuration settings for incoming and outgoing mail monitors in the Web Services Adapter.

Interface Type Parameters

Tab

Parameter

Type

Description

Operational Email

 

Supported Mail Monitors

List

Active mail monitors in the Web Services Adapter. Valid values are:

  • Incoming
  • Outgoing
  • Incoming and Outgoing.

SV Instance

Text

Singleview instance on which the interface is run.

Note:

Ignored if Cluster Managed is True.

EMAIL Sequence Number

Text

EMAIL configuration item sequence number.

The default is the next available EMAIL configuration item sequence number.

This value is also used to generate the REMOTE_SERVICE configuration item for an outgoing monitor.

Cluster Managed

List

Indicates if this instance is managed by a high availability configuration, and determines the number of REMOTE_SERVICE and EMAIL configuration items generated.

Default is False. If set to:

  • False, only one REMOTE_SERVICE and EMAIL configuration item is created
  • True, a REMOTE_SERVICE and EMAIL configuration item is created for each instance in the InstanceStatus derived attribute table.

Static Email

 

Config File Template

Text

Configuration template file used to produce the WSA configuration file.

The default path is $ATA_REL_JAVA_CONFIG/email.xml.

Use RSET

List

Indicates whether WSAEmail shall test its connection to the mail transport agent with the RSET command instead of the NOOP command.

The default is True. If set to:

  • False, the NOOP command is used
  • True, the RSET command is used.

Connection

SV Username

Text

User name used to connect to Singleview.

SV Password

Text

Password used with the user name.

TRE Client Name

Text

Name of client that is used to connect to Singleview.

The default is WSAEMAIL.

THP Instance

List

THP instance used by the Web Services Adapter to connect to Singleview.

Max Active THP Connections

Integer

Maximum number of concurrent connections.

Default value is 5.

Max Idle THP Connections

Integer

Maximum number of connections that can be idle before being closed.

Default value is 60.

Max Wait for THP Connection (s)

Integer

Length of time in seconds to wait for a connection.

Default value is 300.

THP Group

List

THP group to associate with the email configuration in a clustered environment.

Incoming

Note:

Displayed only if the Supported Mail Monitors field is set to Incoming, or Incoming and Outgoing on the Operational Email tab.

Protocol

List

Protocol used to fetch email. Valid options are:

  • IMAP
  • IMAPS
  • POP3
  • POP3S.

Host

[Text]

Machine running the IMAP or POP3 server.

Default is localhost.

Port

Text

Port on the machine running the IMAP or POP3 server.

It is specified as a port number or service name as defined in /etc/services.

Username

[Text]

User name of account that email is retrieved from.

Password

[Text]

Password used for the email account.

Poll Time (s)

Integer

Length of time in seconds between checks for new email.

Default value is 10.

Receive Function

Text

Function called to process new email.

The Mail.Receive&() function is provided as a template to be duplicated or modified.

TRE Listener

Note:

Displayed only if the Supported Mail Monitors field is set to Outgoing, or Incoming and Outgoing on the Operational Email tab.

Tuxedo Service Name

Text

Name of the Tuxedo service used to connect to outgoing mail monitors.

TREPROXY Server Group

Text

Tuxedo group associated with the treproxy server.

Default value is IMP.

Number Concurrent Requests

Integer

Number of treproxy servers to be created.

Default value is 5.

TREPROXY Port

Text

Port used to connect to the Web Services Adapter.

This field is specified as a port number or service name as defined in /etc/services.

Outgoing

Note:

Only displayed if the Supported Mail Monitors field is set to Outgoing, or Incoming and Outgoing on the Operational Email tab.

Protocol

List

Protocol used to send email. Valid options are:

  • SMTP
  • SMTPS.

Host

[Text]

Machine running the SMTP server.

Default is localhost.

Port

Text

Port on the machine running the SMTP server.

It is specified as a port number or service name as defined in /etc/services.

Username

Text

User name of the email account to use for sending email.

Password

Text

Password for the account sending email.

Minimum Idle Connections

Integer

Minimum number of idle connections in the pool.

Default value is 0.

Maximum Idle Connections

Integer

Maximum number of idle connections in the pool.

Default value is 1.

Max Active Connections

Integer

Maximum number of active connections in the pool.

Default value is 1.

Maximum Wait Time (s)

Integer

Length of time in seconds to wait for a connection to be established to send mail before failing.

The value is used to derive:

  • The TIMEOUT value associated with the REMOTE_SERVICE configuration item created for the treproxy server (the value is derived at 100% of the Maximum Wait Time)
  • The time to wait for:
    • A connection from the pool to be established (the value is derived at 80% of the Maximum Wait Time)
    • A connection the mail transport agent to be established (the value is derived at 60% of the Maximum Wait Time)
    • The mail transport agent to respond after sending data (the value is derived at 60% of the Maximum Wait Time).

Default value is 30.

Minimum Idle Time (s)

Integer

Length of time in seconds before a connection is marked as idle.

Default value is 300.
