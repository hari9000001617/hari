SERVICE_HISTORY_REP_V

The SERVICE_HISTORY_REP_V view displays details of all services with the resolution of codes, types, and so on.

Column Definitions

Column Name

Properties

Description

SERVICE_ID

NUMBER (15)

Not null

ID of a service.

LAST_MODIFIED

DATE

Not null

Date-time when this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

It must match an EFFECTIVE_START_DATE for the base product instance associated with the service. This is checked by the DVP.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

It must match an EFFECTIVE_END_DATE for the base product instance associated with the service. This is checked by the DVP.

ATLANTA_OPERATOR_LOGIN_NAME

VARCHAR (30)

Not null

User who modified the service.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who modified the service.

SERVICE_NAME

VARCHAR (64)

Not null

Name associated with the service involving the most exact specification of the service in relation to the network.

The format of this value usually corresponds to how the service is referenced by switching equipment and the CDRs and events generated (for example, the full country number for a voice service).

SERVICE_TYPE_NAME

VARCHAR (60)

Not null

Type of this service.

SERVICE_TYPE_DISPLAY_NAME

VARCHAR (60)

Not null

Name of service type displayed. Defaults to SERVICE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

SERVICE_TYPE_ID

NUMBER (8)

Not null

Type of this service.

SERVICE_SUBTYPE

VARCHAR (40)

Subtype of this service (for example, Dial-up and Dedicated subtypes within the service type of X.28).

Refer to the SERVICE_TYPE table for the list of valid subtypes.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer hierarchy element or node to which this service belongs.

SERVICE_STATUS_ABBREV

VARCHAR (60)

Not null

Current status of the service.

SERVICE_STATUS_CODE

NUMBER (8)

Not null

Current status of the service.

SERVICE_PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer assigned to this service.

SERVICE_PRODUCT_OFFER_NODE_
ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in SERVICE_PRODUCT_OFFER_ID was allocated.

SERVICE_PRODUCT_OFFER_ALL_
DATE

DATE

Date-time at which the service was allocated to the product offer designated in SERVICE_PRODUCT_OFFER_NAME.

SERVICE_PREV_PRODUCT_OFFER_
ID

NUMBER (8)

Name of the previous product offer assigned to this service.

SERVICE_PREV_PROD_OFF_NODE_
ID

NUMBER (8)

ID of the product offer hierarchy node from which the previous product offer in SERVICE_PREV_PROD_OFFER_ID was allocated.

PERSON_ID

NUMBER (15)

Primary contact person for this service.

Any correspondence that relates to this service is addressed to this person.

It defaults to the primary contact for the customer node.

CONTRACT_ID

NUMBER (15)

Contract under which this service is covered.

A_ADDRESS_ID

NUMBER (15)

Reference to the address details for the A-end of the service.

B_ADDRESS_ID

NUMBER (15)

Reference to the address details for the 'B-end' of the service.

If the service does not require the definition of a start and termination point, this reference is null.

REQUIRED_BY_DATE

DATE

Date-time on which the current action as specified by SERVICE_STATUS_CODE is due to be completed.

If the SERVICE_STATUS_CODE is 1, 5, 7, or 8, it is mandatory; otherwise, it is left at its previous value.

PROVISIONED_DATE

DATE

Date-time that the service was provisioned for use by the exchange staff.

ACTIVE_DATE

DATE

Date-time that the service became billable to the customer.

It should correspond to the date-time on which the customer has accepted the service (or at least been notified that the service is available).

It is the date used as the basis for calculating any periodic charges for the service.

GRADE_OF_SERVICE

VARCHAR (40)

Grade of the service.

The list of allowable baud rates is dependent on the type of the service.

SERVICE_USAGE

VARCHAR (40)

Main usage type for the service where different types of usage are allowed, dependent on the type of the service.

COMMS_TYPE

VARCHAR (40)

Appropriate communications signalling type classification such as Synchronous or Asynchronous, dependent on the service type.

GL_CODE_ID

NUMBER (8)

GL code used for all financial transactions produced as part of this service.

INDUSTRY_GENERAL_1 -
INDUSTRY_GENERAL_10

VARCHAR (255)

General fields 1 - 10 for use as common attributes for a specific type of service (for example, GSM services), irrespective of client customisation.

Client customisation should be performed using the GENERAL_1 to GENERAL_10 fields.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be defined in the type definition table.
