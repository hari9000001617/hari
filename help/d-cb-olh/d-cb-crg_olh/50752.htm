SERVICE_HISTORY_V

The SERVICE_HISTORY_V view is a view of the SERVICE_HISTORY table, with some information also obtained from PRODUCT_INSTANCE_HISTORY, CONTRACT_HISTORY, and ADDRESS_HISTORY.

It contains information associated with services required for rating or billing. The primary requirement of the definition is that for a given SERVICE_ID and date-time, only a single row must be returned by this query.

Contract Details

Column Name

Properties

Description

CONTRACT_NAME

VARCHAR (40)

Name for the contract.

It is the external ID for the contract that is usually automatically assigned on contract creation by the contract types entity validation.

CONTRACT_CUSTOMER_
REFERENCE

VARCHAR (40)

Customer reference for the contract.

It is how the customer identifies the contract within their own accounts payable (or equivalent) system.

Convergent Billing does not enforce uniqueness on this column.

CONTRACT_TYPE_ID

NUMBER (8)

Type of contract entered into.

It controls the products that can be associated with the contract, and also defaults many characteristics on contract creation.

CONTRACT_STATUS_CODE

NUMBER (8)

Status of this contract.

Values are client configurable. Valid values are:

  • 1 (In Negotiation)
  • 2 (Pending)
  • 3 (Active)
  • 8 (In Grace)
  • 9 (Cancelled).

The value defaults from the CONTRACT_TYPE.

CONTRACT_START_DATE

DATE

Date-time that the contract first became active for this customer.

CONTRACT_DURATION

NUMBER (4)

Duration of the contract.

The default value is based on the contract type.

If CONTRACT_DURATION is specified, CONTRACT_EXPECTED_EXPIRY_DATE is derived using the formula:

CONTRACT_EXPECTED_EXPIRY_DATE = CONTRACT_START_DATE + CONTRACT_DURATION

It is enforced by C_CONTRACT_HISTORY check constraint.

If CONTRACT_EXPECTED_EXPIRY_DATE is manually modified, CONTRACT_DURATION is cleared.

It cannot be modified if the contract type has a FIXED_DURATION_IND_CODE of 1.

CONTRACT_DURATION_
UNIT_CODE

NUMBER (2)

Units of the contract duration. Valid values are:

  • 0 (Days)
  • 1 (Weeks)
  • 2 (Months)
  • 3 (Years).

CONTRACT_EXPECTED_
EXPIRY_DATE

DATE

Date-time when this contract is expected to expire.

It is derived from the contract start date and contract duration, if the contract duration is specified.

CONTRACT_ACTUAL_
EXPIRY_DATE

DATE

Date-time when the contract was terminated.

If it has a time of 00:00:00, the contract must have a status of Cancelled from this date-time; otherwise, the contract must have a status of Cancelled from midnight on the following day after this date-time.

CONTRACT_GRACE_
EXPIRY_DATE

DATE

Date-time after the EXPECTED_EXPIRY_DATE that a contract can remain in a non-cancelled status and not be renegotiated.

It can be derived from the EXPECTED_EXPIRY_DATE and the contract type.

It can be manually modified by the user, and is re-derived whenever the EXPECTED_EXPIRY_DATE is modified.

CONTRACT_SALES_
CHANNEL_CODE

NUMBER (8)

Sales channel from which this contract was obtained.

CONTRACT_GENERAL_1 - CONTRACT_GENERAL_10

VARCHAR (255)

General fields 1 - 10 for entering contract related details.

The contents and validation of these fields is specified by entity validation associated with the contract type.

 

Product Offer Column Definitions

Column Name

Properties

Description

PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer associated with this service.

The product offer must:

  • Be configured as service-specific.
  • Belong to the same hierarchy as the product offer assigned to the base product instance for this service.

If the service is not assigned a product offer, it must be set to NULL.

PRODUCT_OFFER_NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in PRODUCT_OFFER_ID was allocated.

If PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PREVIOUS_PRODUCT_OFFER_
ID

NUMBER (8)

ID of the previous product offer associated with this service.

It is automatically modified when the product offer associated with a service is changed.

PREVIOUS_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in PREVIOUS_PRODUCT_OFFER_ID was previously allocated.

If PREVIOUS_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer assigned to this service.

This product offer must belong to the product offer hierarchy of the product offer assigned to the base product instance for the service.

It must be NULL if the base product instance for the service is not assigned a product offer.

SERVICE_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in SERVICE_PRODUCT_OFFER_ID was allocated. This node must be associated with the product offer identified by SERVICE_PRODUCT_OFFER_ID.

If SERVICE_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PRODUCT_OFFER_
ALL_DATE

DATE

Date-time at which this service was allocated the product offer identified by SERVICE_PRODUCT_OFFER_ID.

It is usually the same as ACTIVE_DATE, but can be different if the product offer is changed.

If SERVICE_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

SERVICE_PREV_PRODUCT_
OFFER_ID

NUMBER (8)

ID of the previous product offer associated with this service.

It is automatically modified when the product offer associated with a service is changed.

SERVICE_PREV_PROD_OFF_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product offer in SERVICE_PREVIOUS_PROD_OFFER_ID was previously allocated.

If SERVICE_PREVIOUS_PROD_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.
