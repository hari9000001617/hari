DISPUTE_CHARGE_TRE_V

The DISPUTE_CHARGE_TRE_V view provides details about individual disputed charges raised against an invoice or account.

Each row in the view defines details of a specific dispute raised against a charge.

Column Definitions

Column Name

Properties

Description

DISPUTE_CHARGE_ID

NUMBER (15)

Not null

ID of the disputed charge.

DISPUTE_ID

NUMBER (15)

Not null

ID of the dispute.

SEQNR

NUMBER (4)

Not null

Sequence number of the disputed charge.

It determines the order in which the disputed items are displayed.

LAST_MODIFIED

DATE

Not null

Date-time the record was last modified.

EXTERNAL_OBJECT_ID

NUMBER (15)

Reference object in an external schema (for example, a Process Engine case or issue) that created this dispute.

CHARGE_ID

NUMBER (15)

Not null

Read-only

ID of the charge in dispute.

On insert, the specified charge must exist, and be associated to the account and invoice associated with the dispute, or indirectly associated to an account and invoice through an account invoicing hierarchy.

Note:

The specified charge may be revoked or reprocessed as part of resolving the dispute, or before the dispute is archived.

CHARGE_PARTITION_NR

NUMBER (2)

Not null

Read-only

Partition number associated with this charge.

If not specified, it is derived from CHARGE_ID.

CHARGE_DATE

DATE

Not null

Date of the charge.

If not specified, it is derived from CHARGE_ID.

INVOICE_SECTION_NR

NUMBER (8)

Read-only

Invoice section number assigned to the charge in dispute.

It is automatically derived from CHARGE_ID.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number assigned to the charge in dispute.

It is automatically derived from the specified charge.

BILLABLE_AMOUNT

NUMBER (22,6)

Total billable amount associated with the charge.

If not specified, and a callback function is not defined for the account type, this amount is equal to the amount of the charge converted to the currency of the account.

The charge associated with this amount must be associated with a billable tariff.

Note:

A positive value indicates that the charge was debited from the account.

ORIGINAL_DISPUTED_AMOUNT

NUMBER (22,6)

Original billable amount in dispute.

If not specified, it defaults to BILLABLE_AMOUNT.

Note:

A positive value indicates the amount to be credited if the dispute is accepted.

DISPUTED_AMOUNT

NUMBER (22,6)

Billable amount still in dispute, or was accepted as being incorrect if the dispute is now accepted.

It defaults to ORIGINAL_DISPUTED_AMOUNT on insert.

DISPUTE_CLASSIFICATION_
CODE

NUMBER (8)

Classification code to indicate the type of dispute.

It is based on the DISPUTE_CLASSIFICATION reference type.

DISPUTE_RESOLUTION_CODE

NUMBER (8)

Resolution code to indicate how the dispute was resolved.

It is based on the DISPUTE_RESOLUTION reference type.

DISPUTE_CREATION_TXT

VARCHAR (4000)

Reason for raising the dispute.

DISPUTE_RESOLUTION_TXT

VARCHAR (4000)

Description of the resolution for the dispute.
