ATLANTA_GROUP_ACCESS_TRE_V

The ATLANTA_GROUP_ACCESS_TRE_V view is provided for compatibility with previous versions of Convergent Billing.

Note:

The GROUP_ACCESS_TRE_V should be used for all new development work.

The view returns the list of security groups to which a user has access, and should be used in any queries where the rows returned are to be restricted to the security groups a user has access to.

When used in TRE database services, the view should be restricted to the user calling the service.

Column Definitions

Column Name

Properties

Description

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

ID of a system user.

ATLANTA_GROUP_ID

NUMBER (8)

Not null

ID of the security group to which a user has access.

ACCESS_LEVEL

NUMBER (10)

Not null

Access level for the security group for the users with this profile.

It is interpreted as a bit mask.

Valid values are:

  • 1: Update records associated with the security group.
  • 2: Insert records using the security group.
  • 3: Delete records associated with the security group.
  • 7: View user, security group, and last modified details for records associated with the security group.
  • 8: Update the security group associated with the record.
  • 9: Perform direct updates on records associated with the security group (history forms).
  • 10: Perform range updates on records associated with the security group (history forms).
  • 11: Perform direct deletes on records associated with the security group (history forms).
  • 12: Assign records associated with the security group.
  • 13: View masked data that has been encrypted using the security group.
  • 14: Compare clear text value with data that has been encrypted using the security group for equality.
  • 15: View decrypted data that has been encrypted using the security group.
  • 16-31: Available to users for implementation of security group-based field-level security.

    Note:

    The existence of a security group in this table for a user implies that the user can view records in the security group.

For information about security groups, refer to the Security and Auditing Configuration Guide for Singleview.
