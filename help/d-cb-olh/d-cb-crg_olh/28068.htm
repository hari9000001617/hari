EVS Function Definitions

This section describes functions associated with the Entity Validation Server. All functions execute in the EPM interpreter.

EVS Functions

Local and remote versions are provided for most EVS functions. Remote versions of local functions are distinguished by using the same function name but including the prefix 'f'. Local functions avoid a transaction switching to a different server within the TRE, which usually occurs when calling a remote function; switching to a different server has a performance overhead.

A local version of a function can be called only within a TRE server that advertises the underlying TRE callback function. Otherwise, the remote version of the function must be called to route the call to the appropriate server. For example, EV_AdjustmentInsert&() can be called only from a server that advertises the biAdjustment service, which implements the AdjustmentInsert&() function.

EVS function interfaces that use the fieldValues?{} hash parameter to pass data to the functions also accept an element (in fieldValues?{}) with the CALL_PARAMS key. The data associated with the key is an untyped hash and assigned to the evsCallParams?{} global variable on entry to a function. The content of the hash is user-defined and provides a mechanism to make additional data available to entity validation expressions; the additional data is accessed in entity validation expressions by referencing the evsCallParams?{} variable.

Caution:

Do not call EVS functions in entity validation expressions.
