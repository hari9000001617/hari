ADJUSTMENT_TYPE_TRE_V

The ADJUSTMENT_TYPE_TRE_V view provides access to the types of adjustments known to Singleview.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

ID for a type of adjustment.

LAST_MODIFIED

DATE

Not null

Date-time at which this row was last modified.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Name of the adjustment type, as known to users.

ADJUSTMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Not null

Name of the adjustment type displayed.

It defaults to ADJUSTMENT_TYPE_NAME.

Caution:

Its value can be translated into a locale-specific language and therefore differ between implementations.

DESCRIPTION

VARCHAR (255)

Not null

General description of the adjustment type.

OPERATOR_ID

NUMBER (8)

Not null

ID of the user who last modified this record.

GROUP_ID

NUMBER (8)

Security group associated with, and controlling access to, the adjustment type.

DEFAULT_ADJUSTMENT_TEXT

VARCHAR (80)

Not null

Default text displayed in the summary section of the customer invoice.

The value can be overridden for individual adjustments.

CURRENCY_ID

NUMBER (8)

If specified, the currency in which all adjustments of this type are made. For batch adjustments, this defines the currency to be used.

The currency must exist from the current date-time until the end of time (checked by the DVP).

If the currency is NULL, adjustments of this type may be in any currency.

DEBIT_ACCOUNT_TYPE_ID

NUMBER (8)

Type of account from which charges are debited for a positive adjustment.

A value is only specified for adjustments that transfer amounts between two accounts.

A user can only specify a debit account for an adjustment if this column is specified.

DEBIT_ACCOUNT_ID

NUMBER (15)

Account from which charges are debited for a positive adjustment.

It can only contain a value if DEBIT_ACCOUNT_TYPE_ID is specified.

Only an account of the specified DEBIT_ACCOUNT_TYPE_ID can be specified.

CREDIT_ACCOUNT_TYPE_ID

NUMBER (8)

Type of account to which adjustments of this type are credited for a positive adjustment.

If specified, adjustments of this type can only be credited to accounts of this type.

DEBIT_GL_CODE_ID

NUMBER (8)

Default GL code to debit for positive adjustments of this type.

The GL code must exist from the current date-time until the end of time.

It is mandatory if the General Ledger subsystem is enabled.

If CURRENCY_ID is specified, the currency of this GL code must be the same as CURRENCY_ID.

CREDIT_GL_CODE_ID

NUMBER (8)

Default general ledger to credit for the portion of positive adjustments not allocated to invoices.

The GL code must exist from the current date-time until the end of time.

It is mandatory if the General Ledger subsystem is enabled.

The reporting entity and currency associated with this GL code must be the same as for DEBIT_GL_CODE_ID.

ADJUSTMENT_FORMAT_EXPR

VARCHAR (255)

Not null

Expression defining the format of adjustment numbers for this adjustment type.

The expression has access only to variables associated with the ADJUSTMENT table and the local variable @value& local variable.

Note:

@value& can be used to access the ADJUSTMENT_ID of the current adjustment.

ADJUSTMENT_BATCH_FORMAT_
EXPR

VARCHAR (255)

Not null

Expression defining the format of adjustment batch numbers for this adjustment type.

It has access only to variables associated with the ADJUSTMENT table and the local variable @value& local variable.

Note:

@value& can be used to access the ADJUSTMENT_ID of the current adjustment.

ALLOCATION_FUNCTION_DEFN_
ID

NUMBER (10)

ID of a function used to determine the initial allocations to invoices and receivable types for adjustments of this type.

For more information about allocation functions, refer to the Finance Guide for Singleview.

GL_ALLOCATION_FUNCTION_
DEFN_ID

NUMBER (10)

ID of a function used to perform General Ledger allocations for adjustments of this type (see the Finance Guide for Singleview).

The function is performed at the conclusion of applying an adjustment after all invoice allocations have been performed.

Returned information is used to populate the ADJUSTMENT_GL table for an adjustment.

ADJUST_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function used to determine the discrepancy adjust amount for adjustments of this type.

ENTITY_VALIDATION_ID

NUMBER (8)

Entity validation ID for all adjustments of this type.

If not null, the entity validation must have a table name of ADJUSTMENT.

VERSION_STR

VARCHAR (255)

Version identification string used to tag this entity.

It is provided for use by third party configuration management tools and is not directly used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this entity.

Refer to the CATEGORY_V view for the list of available categories.
