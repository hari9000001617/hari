Individual Payment Detail Form

The following table lists automation parameters for the Individual Payment Detail form, data type, and supported functions.

The description column is provided to supplement the description in the PAYMENT_TRE_V view, or to provide information about parameters not contained within the database view.

Data Type

Supported Functions

Description

Receipt number assigned to a payment.

Integer

Account that the payment is deducted from.

CREDIT_GL_CODE_ID

(FROM_GL_CODE_ID)

Integer

Insert

Update

GL code to credit with the payment amount.

It can only be updated if the payment is not applied.

FROM_GL_CODE_ID is deprecated and provided for backward compatibility only.

CREDIT_ACCOUNT_ID

(TO_ACCOUNT_ID)

Integer

Insert

Account against which the payment is being made.

This can only be specified if the PAYMENT_TYPE_ID has been supplied.

TO_ACCOUNT_ID is deprecated and provided only for backward compatibility.

An error occurs if CREDIT_ACCOUNT_ID is passed in before ADJUSTMENT_TYPE_ID.

(TO_GL_CODE_ID)

Integer

Insert

Update

GL code to debit with the payment amount.

It can only be updated if the payment is not applied.

TO_GL_CODE_ID is deprecated and provided only for backward compatibility.

Integer

Insert

Update

PAYMENT_LOCATION_CODE

Integer

Insert

Update

Preferred location where payments will be made.

PAYMENT_TYPE_ID

Integer

Insert

ID for the method of payment.

Note:

It is recommended that the this field be passed in first. Failure to do so might cause other fields to be ignored. An error occurs if TO_ACCOUNT_ID is passed in before PAYMENT_TYPE_ID.
