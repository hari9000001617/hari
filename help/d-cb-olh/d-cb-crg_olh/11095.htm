biAccountingYearUpdate&()

Syntax

biAccountingYearUpdate&(
    AccountYearId&,
    var LastModified~,
    const FieldNames$[],
    const FieldValues?[])

 

Description

Updates an accounting year.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

AccountingYearId&

In: ID of the accounting year to update.

LastModified~

In: Last modified date-time of the accounting year before the update.

Out: Last modified date-time of the accounting year after the update.

FieldNames$[]

In: Field names from the ACCOUNTING_YEAR table to be updated.

The field name DISCONTINUITY_IND_CODE can be specified to skip validation of the contiguousness of accounting years. Skipping this validation may be required when performing multiple or bulk inserts or updates.

If DISCONTINUITY_IND_CODE is specified, its corresponding value must be 1.

Updates to ACCOUNTING_CALENDAR_ID are propagated to all associated accounting periods.

FieldValues?[]

In: Field values in the same order that their names are specified in FieldNames$[].

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

FieldNames$[] and FieldValues?[] are validated by biAccountingYearValidate&() before updating the year.

Specifying DISCONTINUITY_IND_CODE is not recommended because it can cause gaps in accounting years. If DISCONTINUITY_IND_CODE is specified, it is important to ensure that the records are contiguous after all insert or update operations are complete.

The DISCONTINUITY_IND_CODE field must also be specified when adding the accounting years to fill the gaps. If gaps exist, accounting years cannot be modified using the Convergent Billing client, and DVP and GL guidance issues can arise.
