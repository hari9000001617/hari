ADJUSTMENT_INVOICE_TRE_V

The ADJUSTMENT_INVOICE_TRE_V view maps adjustments to invoices and receivable types.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_ID

NUMBER (15)

Not null

Read-only

ID for the adjustment.

ACCOUNT_ID

NUMBER (15)

Not null

Read-only

ID of an account against which the adjustment is being made.

SEQNR

NUMBER (5)

Not null

Read-only

Sequence number for this combination of account and adjustment.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that the record was last modified.

INVOICE_ID

NUMBER (15)

Read-only

ID of an invoice against which part of this adjustment was made.

If a credit or debit was made directly against an account, it is NULL.

The invoice must be associated with the specified ACCOUNT_ID. This is checked by the DVP.

INVOICE_LAST_MODIFIED

DATE

Read-only

Date-time that the invoice was last modified.

ALLOCATED_ADJUSTMENT_ID

NUMBER (15)

ID of another adjustment against which part of this adjustment was allocated.

It must be associated with the ACCOUNT_ID of this adjustment.

ADJUSTMENT_LAST_MODIFIED

DATE

Date-time at which the allocated adjustment was last modified.

It is mandatory for calls to biAdjustmentInvoiceUpdate&() where rows with an ALLOCATED_ADJUSTMENT_ID are specified.

It must be set to the last modified date-time of the allocated adjustment.

PAYMENT_ID

NUMBER (15)

ID of a payment against which part of this adjustment was allocated.

The payment must be associated with the specified ACCOUNT_ID.

PAYMENT_LAST_MODIFIED

DATE

Date-time at which this payment was last modified.

It is mandatory for calls to biAdjustmentInvoiceUpdate&() where rows with a PAYMENT_ID are specified.

It must be set to the last modified date-time of the allocated adjustment.

RECEIVABLE_TYPE_ID

NUMBER (8)

Read-only

ID of a receivable type against which part of this adjustment was made.

The receivable type must exist on the received date of the payment.

It is mandatory if an INVOICE_ID is specified and the invoice has one or more rows in the INVOICE_RECEIVABLE_TYPE table.

If RECEIVABLE_TYPE_ID and INVOICE_ID are specified, the combination must exist in the INVOICE_RECEIVABLE_TYPE table.

AMOUNT

NUMBER (19,3)

Not null

Amount of the adjustment against the invoice, adjustment or payment.

If the INVOICE_ID, ALLOCATED_ADJUSTMENT_ID, and PAYMENT_ID are NULL, the adjustment is applied directly against the account.

The currency of AMOUNT is the same as the account currency.
