Description

Calculates and returns charge information based on the event parameters set in the FieldNames$[] and <type>Values arrays.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

FieldNames$[]

In: Names of fields to set as the event.

Valid fields are from the NORMALISED_EVENT_V view.

The set of fields that must be specified is dependent on the rating configuration. For example, if a rating tariff was set up to verify EVENT_CLASS_CODE, EVENT_TYPE_CODE, and EVENT_SUB_TYPE_CODE, FieldNames$[] can be set to FieldNames$[] = ['NORMALISED_EVENT_TYPE_ID', 'C_PARTY_ID', 'CHARGE_START_DATE', 'EVENT_CLASS_CODE', 'EVENT_TYPE_CODE', 'EVENT_SUB_TYPE_CODE', 'DURATION'].

Note:

The mandatory fields required for the record to be inserted into the NORMALISED_EVENT or NORMALISED_EVENT_ERROR tables must be specified.

FieldValues?[]

In: Field values, in the same order that their names are specified in FieldNames$[].

ChargeFieldNames$[]

In: Charge fields to be returned for each charge calculation.

ChargeFieldValues?[]

Out: Charge values, corresponding to ChargeFieldNames$[].

Each row in the array corresponds to the requested charge fields.

DateValuesPerCharge&

Out: Number of date values returned for each charge.

ChargeDateValues~[]

Out: Resulting date values of the fields in ChargeFieldNames$[].

The beginning of each charge row is NrCharges& x DateValuesPerCharge&.

DebugLevel&

In: Level of rating debug to turn on for rating the event.

If set to 0, no debug tracing is activated.

The table below lists valid debug levels.

TraceFile$

Out: Path to the file containing the debug information (if DebugLevel& is set).

The file name has the format:

rating_<pid>_<yyyymmdd>_<hhnnss>_<usec>.trc

where:

  • <pid> is the process ID of the trerate process that handled the biEventRateAndReturn&() function.
  • <yyyymmdd> and <hhnnss> is the current date-time, respectively.
  • <usec> is the number of microseconds into the current second, to ensure each file name is unique.

The path to the file is the directory contained in the CALLER_PREFIX attribute of the TRERATE configuration item for the process rating the event.

ErrorId&

Out: Error ID if an error occurred during rating.

ErrorMessage$

Out: Error message if an error occurred during rating.

 

Returns

Interface 1

Number of charges (equal to NrCharges&) if successful; otherwise, an error is raised.

Interfaces 2 and 3

Number of charge values (equal to the number of rows in ChargeFieldValues?[]) if successful; otherwise, interface 2 returns -1. TraceFile$ is set so that debug tracing can also be obtained for events that error during rating.

Remarks

The number of values in each data type array is returned in that type's return parameter (for example, RealValuesPerCharge&).

The function can used for providing a quote for service usage. For example, to answer the question "How much will a 10 minute call to Pakistan at 9:45pm tonight cost?".

During calls to the function, subtotal values accessed include any current reservations in their returned values (except for the current reservation being processed).

If this function is called to determine the amount to reserve for an account and either AccountReservationInsert$() or AccountReservationUpdate&()executed (to create or update the account reservation), a call should be made to SubtotalEventCommit&() or EntitlementEventCommit&() to make reservations against rating subtotals and entitlements.

Note:

  • Reservations are available only as part of the separate Commerce Engine licence option.
  • The function only implicitly populates the direct variables from the NORMALISED_EVENT_V view that have been passed into it.
