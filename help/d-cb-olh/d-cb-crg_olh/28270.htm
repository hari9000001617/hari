fCMN_TREAdjustmentBatchCreate_R&()

Syntax

fCMN_TREAdjustmentBatchCreate_R&(
    l_LastModDate~,
    l_AdjustmentBatchDetails?{})

 

Description

The core TRE API provides mechanisms to insert adjustment batch records, individual adjustments for the batch, and invoice allocations for each of these adjustments.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_LastModDate~

Out: Date-time the record was last modified.

l_AdjustmentBatchDetails?{}

In: Hash containing the values for the adjustment creation.

 

Returns

New adjustment batch ID created.

Remarks

Adjustment batch creation allows the user to insert a complete batch that includes all subordinate objects in a single transaction. As is the case with all CBF APIs, what is in the hash map data structure determines CBF behaviour.

The high-level processing of the function is:

  1. Map client batch label names (if passed) with database (TRE) column names.
  2. Insert the initial empty adjustment batch using the existing TRE API.
  3. For each adjustment in the batch:
    1. Derive the adjustment type and batch ID.
    2. Insert the adjustment using the appropriate EVS API.
    3. If invoice allocations are supplied, allocate as specified otherwise auto-allocate.
  4. Apply adjustment batch, if specified.
  5. Retrieve adjustment LAST_MODIFIED date.
  6. Return the original adjustment batch ID.

The adjustment type is not mandatory. It can be derived from the batch record inserted (within the transaction).

The batch ID cannot be passed, and is derived internally after the successful batch insertion (within the transaction).
