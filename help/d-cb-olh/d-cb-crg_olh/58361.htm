EntitlementInsert&()

Syntax

EntitlementInsert&(
    StartDate~,
    var LastModified~,
    const FieldNames$[],
    FieldValues?[])

 

Description

Inserts an entitlement, and generates an initial allocation for the entitlement.

Attributes

Function Type

Local

Service

(Local)

Application Environment

Any

Context

Any

 

Parameter

Description

StartDate~

In: Effective start date-time for the new entitlement.

LastModified~

Out: Last modified date-time of the inserted entitlement.

FieldNames$[]

In: Names of fields from ENTITLEMENT_TRE_V view being inserted.

FieldValues?[]

In: Field values in the same order that their names are specified in FieldNames$[].

 

Returns

ID of the newly-created entitlement, if successful; otherwise, an error is raised.

Remarks

An entitlement, depending on its context, is associated with a service, customer, or a customer node. An entitlement belongs to a type and category. Entitlement types and categories define the attributes of the entitlement.

Defining Entitlements

The EntitlementCategory derived attribute can define the type of the entitlement, which in turn can define the context of the entitlement.

To define the category of the new entitlement, a value for the ENTITLEMENT_CATEGORY_CODE must be supplied to this function and must exist in the ENTITLEMENT_CATEGORY reference type. This value can correspond to the value of the Index 1 column for a row in the EntitlementCategory derived attribute table, and can determine the default values of the entitlement from the result fields of that row.

If the entitlement type is not defaulted through the entitlement category, it must be supplied.

If the context of the entitlement is:

  • Service, the value of SERVICE_ID must be supplied.
  • Customer Node, the value of CUSTOMER_NODE_ID must be supplied.
  • Customer, the value of CUSTOMER_NODE_ID must be supplied and the specified customer node must be a root customer node.

Either SERVICE_ID or CUSTOMER_NODE_ID must be specified, but not both.

If the entitlement has a class of:

  • Recurring, the value of RECURRING_ALLOCATION_AMOUNT must be supplied.
  • One-off, the value of INITIAL_ALLOCATION_AMOUNT must be supplied.
  • Dynamic, the values of RECURRING_ALLOCATION_AMOUNT and INITIAL_ALLOCATION_AMOUNT must not be supplied.

Some field values can be derived from the entitlement type and category. If no row is found in the EntitlementCategory derived attribute table matching the ENTITLEMENT_CATEGORY_CODE, the required field values must be supplied. If a row is found, any supplied field values will override those defaulted from the derived attribute table results.

Date-ranging

The StartDate~ parameter and the following fields from the ENTITLEMENT_TRE_V support a time component:

  • EFFECTIVE_START_DATE
  • EFFECTIVE_END_DATE.

The following fields from the ENTITLEMENT_TRE_V view support date ranging:

  • EFFECTIVE_START_DATE
  • EFFECTIVE_END_DATE
  • SERVICE_ID
  • PRODUCT_INSTANCE_ID
  • CUSTOMER_NODE_ID
  • SUB_CATEGORY.

Expiry Rules

Functions can be specified to be evaluated when an entitlement of the type is about to expire. The following rules apply:

  • If WARNING_FUNCTION_DEFN_ID and WARNING_PERIOD are specified, and the date-time is not in the past, the specified function is scheduled to be evaluated at an advance period of the expiry of the entitlement. The duration of the advance period is defined by the values specified in WARNING_PERIOD and WARNING_PERIOD_UNIT_CODE.
  • If WARNING_ALLOC_FUNCTION_DEFN_ID and WARNING_ALLOC_PERIOD are specified, and the date-time is not in the past, the specified function is scheduled to be evaluated at an advance period of the expiry of the associated allocation. The duration of the advance period is defined by the values specified in WARNING_ALLOC_PERIOD and WARNING_ALLOC_PERIOD_UNIT_CODE. If the date-time is in the past, the specified function is scheduled to be evaluated immediately.
  • If EXPIRY_ALLOC_FUNCTION_DEFN_ID is specified and the date-time is not in the past, the specified function is scheduled to be called when the associated allocation expires. If the date-time is in the past, the specified function is scheduled to be called immediately.

For recurring entitlements aligned with customer bill cycles, tasks for the bill run must be prescheduled for the whole validity period of the recurring entitlements.

Note:

If the entitlement is not suspended (SUSPENDED_IND_CODE is not 1), the EntitlementReload&() function is used to insert the entitlement into the temporal entitlements cache using the entitlement ID.
