CUSTOMER_NODE_HISTORY_V

The CUSTOMER_NODE_HISTORY_V view is an unrestricted view displaying customer details.

It is used by the billing engine and defines what information is available to the billing engine in the context of Customer Node.

Properties

Description

VARCHAR (200)

Not null

Not null

Derived

Duplicate names for customer nodes with the same parent may be permitted depending on the value of the SYSTEM_CLIENT configuration attribute DUPLICATE_CHILD_NODE_NAME.

Root customer nodes (that is, nodes with a PARENT_CUSTOMER_NODE_ID value of NULL) can have duplicate node names (see the System Configuration Guide for Singleview Convergent Billing).

OPERATOR_ID

GL_CODE_ID

NUMBER (8)

Not null

PARENT_CUSTOMER_NODE_ID

NUMBER (15)

Parent customer node for this node.

Only customer node types that support hierarchies can have a value in this column.

HIERARCHY_LEVEL

NUMBER (3)

Not null

Level in the hierarchy of this node.

The root node of a hierarchy has a level of 1. Therefore, it must have a value of 1 if PARENT_CUSTOMER_NODE_ID is null, and a value greater than 1 otherwise.

The values are enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

For values other than 1, the value must be one greater than the value assigned to the PARENT_CUSTOMER_NODE_ID, as verified by the DVP.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Derived

If this instance of a node has a PARENT_CUSTOMER_NODE_ID, this is the ID of the customer node at the root of the hierarchy.

If PARENT_CUSTOMER_NODE_ID is null, this column is also null.

Billing Details

Column Name

Properties

Description

SCHEDULE_ID

NUMBER (15)

Standard invoice cycle to which the customer node is assigned.

If this customer node has a parent node, it must be equal to the parent node's SCHEDULE_ID over the date range.

It is verified by the DVP.

Note:

It is not mandatory for customer nodes associated with a prepaid without statements customer hierarchy.

PREPAID_IND_CODE

NUMBER (1)

If set to 1 (True), this customer node is configured as prepaid without statements; otherwise, the value must be NULL.

BILLING_PRIORITY

NUMBER (4)

Priority associated with processing this customer node in a bill run.

When a group of customer nodes are processed on a bill run, higher priority nodes are processed first. A lower numerical value indicates a higher priority.

If specified, the priority must be greater than or equal to 1, and is enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

BILLING_COMPLEXITY

NUMBER (10)

Numerical value indicating the expected time for performing a bill run on this customer node.

It is not automatically populated or directly used by Convergent Billing.

BILLING_CONFIGURATION_
CODE

NUMBER (8)

Billing configuration for processing this customer node.

It may identify specific billing functions or servers to be called for the various stages in processing a bill run for this customer node.

SUPPRESS_IND_CODE

NUMBER (1)

If set to 1 (TRUE), this customer node hierarchy might be suppressed from standard bill runs and standard QA runs depending on the value of SUPPRESS_UNTIL_ISSUE_DATE; otherwise, the value is null.

It can only be set to 1 for root customer nodes, and is enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

It is ignored for immediate bill, complete bill runs for customer, or QA runs.

SUPPRESS_BILL_CYCLE_
COUNT

NUMBER (2)

If SUPPRESS_IND_CODE is 1, this column can be used to specify the number of bill cycles for which invoice generation should be suppressed for this customer hierarchy.

It is used at the time of entry to calculate SUPPRESS_UNTIL_ISSUE_DATE based on the effective dates of pending tasks for this customer's bill cycle.

It is not possible to specify a value greater than the number of pending tasks for this customer's bill cycle.

It can only contain a value if SUPPRESS_IND_CODE is 1.

SUPPRESS_UNTIL_ISSUE_
DATE

DATE

If SUPPRESS_IND_CODE is 1, it specifies the invoice issue date on or after which invoice generation recommences for standard and QA batch bill runs.

It is mandatory if SUPPRESS_IND_CODE is 1, and is enforced by the C_CUSTOMER_NODE_HISTORY check constraint.

DATE

INVOICE_AMOUNT

NUMBER (19,3)

Values stored in this column are at the limits of double precision.

STATEMENT_AMOUNT

NUMBER (19,3)

Total amount of charges associated with this statement.

It can only contain a value if INVOICE_AMOUNT is 0.

INVOICE_BALANCE_FORWARD

NUMBER (19,3)

Balance of the account before the application of this invoice.

INVOICE_ACCOUNT_BALANCE

NUMBER (19,3)

Account balance after this invoice was generated.

INVOICE_ACCOUNT_INITIAL_
DUE

NUMBER (19,3)

Initial amount due to be paid on this account.

It is usually equal to ACCOUNT_BALANCE, but may be different because of cents rounding or the presence of payment plans (not supported) for the account.

INVOICE_CURRENT_DUE

NUMBER (19,3)

Amount left to be paid on this invoice.

TOTAL_PAYMENTS

NUMBER (19,3)

Total payments made on this account between the issuing of the previous invoice and this invoice.

The payments are not made against this invoice.

The amount is usually displayed on this invoice, calculated by the billing engine.

TOTAL_ADJUSTMENTS

NUMBER (19,3)

Total adjustments made on this account between the issuing of the previous invoice and this invoice.

The adjustments are not made against this invoice.

The amount is usually displayed on this invoice, calculated by the billing engine.

EARLY_PAYMENT_DISCOUNT

NUMBER (19,3)

Early payment discount that applied for this invoice (if any).

The discount is derived by the billing engine (based on the INVOICE_TYPE table), and can be re-derived, based on details in the CHARGE table.

It is incorporated into the INITIAL_DUE calculated for this account.

PAYMENT_DUE_DISCOUNT

NUMBER (19,3)

Pay on time discount applied for this invoice (if any), derived by the billing engine, and can be re-derived, based on details in the CHARGE table.

It is incorporated into the INITIAL_DUE calculated for this account.

LATE_PAYMENT_CHARGE

NUMBER (19,3)

Late payment charge that can be applied to this account if payment is not made by the due date.

The charge is not incorporated into the INITIAL_DUE for this invoice, but can be applied to the account if payment is not made by the due date.

EARLY_PAYMENT_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the EARLY_PAYMENT_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set this column if necessary to record the account balance at the early payment date.

For example, this could be set by a collections or treatment module.

PAYMENT_DUE_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the PAYMENT_DUE_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set this column if necessary to record the account balance at the date the invoice is due to be paid.

For example, this could be set by a collections or treatment module.

INVOICE_GENERAL_1 - INVOICE_GENERAL_10

VARCHAR (255)

General fields 1 - 10 containing configurable details about the last invoice issued to this customer node.

Their use and meaning is controlled by entity validation on the invoice type. They are populated as required by invoice type expressions.
