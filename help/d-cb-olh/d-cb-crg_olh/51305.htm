NORMALISED_EVENT_ERROR_TRE_V

The NORMALISED_EVENT_ERROR_TRE_V view defines the set of fields that can be passed to the biNormalisedEventError* functions.

Column Definitions

Column Name

Properties

Description

NORMALISED_EVENT_ID

NUMBER (15)

Not null

ID of a normalised event.

LAST_MODIFIED

DATE

Not null

Date-time the record was last modified.

NORMALISED_EVENT_TYPE_ID

NUMBER (8)

Not null

Read-only

Normalised event type of this event.

PARENT_NORMALISED_EVENT_
ID

NUMBER (15)

ID of the parent event record that relates to this component.

It is only applicable to conference calls or calls with multiple components.

The parent event record is assigned the cost of its components and any direct cost associated with it.

NORMALISED_EVENT_FILE_ID

NUMBER (15)

Not null

Read-only

ID of the file from which this normalised event record was originally obtained.

FILE_RECORD_NR

NUMBER (10)

Not null

Read-only

Record number of this record in NORMALISED_EVENT_FILE_ID.

The number starts at 1 for the first record in a file and is incremented sequentially for each subsequent record.

EXTERNAL_FILE_RECORD_NR

NUMBER (10)

Read-only

Record number as specified in the original external event.

 

Reprocessing Details

Column Name

Properties

Description

ORIGINAL_FILE_ID

NUMBER (15)

Read-only

ID of the normalised event from which this event was originally processed.

It is only specified for reprocessed events.

ORIGINAL_FILE_RECORD_NR

NUMBER (10)

Read-only

Record number in the original normalised event file.

If the rating engine's version is:

  • 4 or later and ORIGINAL_FILE_ID is specified, it must contain a value.
  • Less than 4, it is always NULL.

REPROCESSED_COUNT

NUMBER (10)

Read-only

Number of times this event was reprocessed.

If ORIGINAL_FILE_ID is specified, it must contain a value greater than or equal to 1.

Chargeable (C) Party Details

Column Name

Properties

Description

C_PARTY_ID

VARCHAR (64)

Full national number or most precise ID for a chargeable party service.

It is usually equal to A_PARTY_ID, but is equal to B_PARTY_ID for reverse charge and 008 calls, and some other third party for conference calls.

C_PARTY_NETWORK_ID

VARCHAR (64)

Network ID of the chargeable party (C party), if any.

For a GSM service, it is the IMSI number.

At least one of the party details (C_PARTY_ID or C_PARTY_NETWORK_ID) must be specified.

The rating engine performs a lookup by the C_PARTY_ID. If it is defined, or returns no rows, a lookup by the C_PARTY_NETWORK_ID is performed, which can result in multiple matches.

C_PARTY_NAME

VARCHAR (64)

Name of the C party, if known (for example, the name provided by the switch for operator assisted calls).

C_PARTY_INTERNAL_ID

NUMBER (15)

ID of the chargeable party.

It is used by the RGP to guarantee unique identification of the service involved. If provided, it is used instead of the C_PARTY_ID or C_PARTY_NETWORK_ID to perform the service lookup.

Note:

External data feeds must not populate this column. If set and the EVENT_CLASS_CODE is not Usage (1), the billing engine includes any charges associated with this event in the current bill cycle even if they have a CHARGE_START_DATE after the bill effective date.

C_PARTY_TON_CODE

NUMBER (8)

C party type of number.

C_PARTY_CELL_ID

VARCHAR (20)

C party network mobile cell ID.

C_PARTY_CARRIER_CODE

NUMBER (8)

Carrier to which the call is destined.

C_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location of the chargeable party of the call to the finest supported granularity.

Higher forms of location such as country and continent can be derived from this value.

C_PARTY_ROUTE

VARCHAR (80)

Incoming or outgoing route of the chargeable party.

 

Transitory Details

Column Name

Properties

Description

FULL_PATH

VARCHAR (40)

Details relating to the route that a call passed through (for example, the X.25 DNIC sequence for the call).

CASCADE_CARRIER_CODE

Miscellaneous Details

Column Name

Properties

Description

SWITCH_START_DATE

DATE

Date-time on CDR and provided by the network switch equipment.

CHARGE_START_DATE

DATE

Date-time used for charging purposes.

It is usually derived from SWITCH_START_DATE.

PERIOD_START_DATE

DATE

Date-time of the start of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

PERIOD_END_DATE

DATE

Date-time of the end of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

EVENT_SOURCE

VARCHAR (40)

Source of the event.

If used, it is usually the same as the value specified in EVENT_SOURCE column of the NORMALISED_EVENT_FILE table.

EVENT_CLASS_CODE

NUMBER (2)

General classification of the event used for reporting purposes.

It is based on the EVENT_CLASS reference type.

BILL_RUN_ID

NUMBER (15)

Read-only

Bill run for which this event was generated, if applicable.

It is always populated for events generated by the RGP (rentals or adjustments).

It is used with ROOT_CUSTOMER_NODE_ID to allow rapid revoking of rental events and adjustments by root customers.

If populated, the BGP only processes this event as part of the specified bill run.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Read-only

ID of the root node of the customer hierarchy for which this event was generated.

It is only specified if BILL_RUN_ID is specified. It is used with BILL_RUN_ID to allow rapid revoking of rental events and adjustments by root customer.

Note:

If non-rental events are generated with BILL_RUN_ID and ROOT_CUSTOMER_NODE_ID specified, and rental events are revoked for the event's bill run and customer hierarchy, the non-rental events are deleted and must be regenerated.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

It is based on the reference type.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event sub-type.

The BASE_INSTALL reference type EVENT_SUBTYPE defines its values.

DURATION

NUMBER

Duration of call.

The units depend on the event type (for example, seconds).

VOLUME

NUMBER

Volume of data associated with the call.

The units depend on the call type (for example, bytes or segments).

If populated by events generated by the RGP, its contents are specified as volume per unit duration.

PULSES

NUMBER (8)

Number of meter pulses associated with the call.

If populated by events generated by the RGP, its contents are specified as pulses per unit duration.

CHARGE

NUMBER (22,6)

Charge as it was received from the switch.

The units are dependent on the switch.

If populated by events generated by the RGP, its contents are specified as charge per unit duration.

CURRENCY_ID

NUMBER (8)

Currency of CHARGE.

RATE_BAND

VARCHAR (40)

Time or rate band calculated for this call, typically based on its CHARGE_START_DATE.This can be supplied by the switch or derived during normalisation.

It can contain an encoded value.

 

General Purpose

Column Name

Properties

Description

GENERAL_1 - GENERAL_20

VARCHAR (4000)

Fields controlled by the normalised event type.

Values are usually used to derive normalised event attributes used for reporting.

PARTITION_NR

NUMBER (2)

Not null

Event error partition number associated with this event.

In single instance environments, all new records have a value of 1.

In multi-instance environments, it is populated by the ENM for guidance or normalisation errors, or by the ERT or ERO for rating errors.

The value is based on information in the ErrorEventPartitionRange derived attribute table.

It is part of the partitioning key for this view.
