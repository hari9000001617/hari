OPERATOR_TRE_V

The OPERATOR_TRE_V view provides details of the users defined in Singleview.

Column Definitions

Column Name

Properties

Description

OPERATOR_ID

NUMBER (8)

Not null

Read-only

ID of the user.

OPERATOR_FULL_NAME

VARCHAR (80)

Not null

Full name of the user.

OPERATOR_LOGIN_NAME

VARCHAR (30)

Not null

Name used by the user to log in to Singleview.

Note:

The operator login name:

  • Cannot contain apostrophe (') or quotation (") characters.
  • Can be modified only if the user account is disabled (ENABLED_IND_CODE is 0).

STATUS

VARCHAR (32)

Not null

Read-only

Status of the user's account when using internal authentication.

Internal authentication is implemented with Oracle user accounts and profiles.

Valid values are:

  • (DISABLED): Internal authentication is disabled. The user has no underlying Oracle account.
  • (ENABLED): Internal authentication is enabled. The user can log in with the appropriate password and has an underlying Oracle account.
  • (EXPIRED): User's password has expired. The password has expired either by explicit request or expiration of the assigned grace period. The password must be changed for the user to log in successfully at the next attempt.
  • (LOCKED): User's account is locked. Locking was due to an explicit request or excessive consecutive failed attempts at logging in. Attempts are rejected until the status is changed.
  • (EXPIRED and LOCKED): User's password has expired and account is locked.

The status of a user account is set by specifying the following values using the appropriate values using biOperatorInsert&() or biOperatorUpdate&():

  • ENABLED_IND_CODE
  • DISABLED_IND_CODE
  • LOCKED_IND_CODE
  • EXPIRED_IND_CODE

It is synchronised with the user's Oracle account when the user logs in.

ENABLED_IND_CODE

NUMBER (1)

Not null

It is TRUE (1) if the account is enabled for internal authentication and FALSE (0) if the account is disabled for internal authentication.

A password must always be specified, using PASSWORD, when enabling a user account.

DISABLED_IND_CODE

NUMBER (1)

Not null

It is TRUE (1) if the account is disabled from all types of Singleview authentication and FALSE (0) if Singleview authentication is enabled.

LOCKED_IND_CODE

NUMBER (1)

Not null

Locked flag.

It is TRUE (1) if the account is locked, and FALSE (0) if the account is unlocked.

It is only applicable if ENABLED_IND_CODE is TRUE (1).

EXPIRED_IND_CODE

NUMBER (1)

Not null

Password expired flag.

It is TRUE (1) if the user's password has expired, and FALSE (0) if the password is valid.

It is only applicable if ENABLED_IND_CODE is TRUE (1).

LOCK_DATE

DATE

Read-only

Date-time at which the user account was locked.

It contains a value only if LOCKED_IND_CODE is TRUE (1).

EXPIRY_DATE

DATE

Read-only

Date-time at which the user password expired.

It contains a value only if EXPIRED_IND_CODE is TRUE (1).

PERSON_ID

NUMBER (15)

Reference to a person entity that provides additional details about the user.

SUPERUSER_IND_CODE

NUMBER (1)

Not null

If set to 1 (TRUE), this user has the capability of performing TRE operations using the treOperatorSetId&() TRE API call.

It can be used in conjunction with treOperatorSetId&() for interfaces to connections from trusted third party applications operating in another Tuxedo domain (server to server connection), or from applications connecting as a Tuxedo client to multiplex multiple client requests over a single connection and maintaining user information for auditing purposes.
