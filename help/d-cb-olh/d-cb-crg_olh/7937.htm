ADJUSTMENT_V

The ADJUSTMENT_V view defines the adjustments that have been applied to accounts.

Most adjustments are performed on invoices, but can also be made on accounts for which invoices are not generated. If an adjustment refers to one or more invoices, associated rows are in the ADJUSTMENT_INVOICE table.

 

Column Definitions

Column Name

Properties

Description

Not null

ID of this adjustment.

ADJUSTMENT_BATCH_NR

VARCHAR (30)

Batch of which this adjustment is a part, if any.

ADJUSTMENT_BATCH_ID

NUMBER (15)

Batch of which this adjustment is a part, if any.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Standard category for the adjustment entered.

If this adjustment is associated with a batch, this adjustment type defaults to the adjustment type of the batch.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

Standard category for the adjustment entered.

If this adjustment is associated with a batch, this adjustment type defaults to the adjustment type of the batch.

DESCRIPTION

VARCHAR (255)

Not null

Description of the adjustment.

The description is not displayed on the invoice, but entered by the user. It can contain the justification for the adjustment.

If the adjustment is associated with a batch, the description defaults to that of the batch.

FROM_ACCOUNT_NAME

VARCHAR (30)

Account where this adjustment is deducted.

If specified, it usually defaults based on the adjustment type.

FROM_ACCOUNT_ID

NUMBER (15)

Account where this adjustment is deducted.

If specified, it usually defaults based on the adjustment type.

FROM_GL_CODE_ID

NUMBER (8)

GL code associated with the adjustment against FR_ACCOUNT_ID.

It is not null if FR_ACCOUNT_ID is not null, enforced by the C_ADJUSTMENT check constraint.

ADJUSTMENT_TEXT

VARCHAR (80)

Not null

Text displayed on the customer invoice in the summary section with the details of this invoice adjustment.

It defaults to the text associated with the adjustment type, but can be modified by the user.

TO_ACCOUNT_NAME

VARCHAR (30)

Not null

Account to which this adjustment applies.

TO_ACCOUNT_ID

NUMBER (15)

Not null

Account to which this adjustment applies.

TO_GL_CODE_ID

NUMBER (8)

Not null

GL code to be associated with this adjustment.

It defaults to the GL code associated with the adjustment type, but can be modified by the user.

ATLANTA_OPERATOR_LOGIN_
NAME

VARCHAR (30)

Not null

Billing system user who entered the adjustment.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

Billing system user who entered the adjustment.

AMOUNT

NUMBER (19,3)

Not null

Amount of the adjustment.

A positive adjustment indicates a credit to the customer that must be included on the invoice. For example, the customer had difficulties with their service in the last invoice period so the service rental charge for that invoice is being refunded.

A negative amount indicates a debit.

The currency of AMOUNT is the same as the currency of the adjustment.

ADJUSTMENT_DATE

DATE

Not null

Date-time when the adjustment was made.

CUSTOMER_QUERY_ID

NUMBER (15)

Optional reference to the customer query that caused this adjustment to occur.
