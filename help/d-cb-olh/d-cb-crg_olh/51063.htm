Variables Based on INV_INVOICE_V

All direct variables based on the INV_INVOICE_V view have an application environment of Invoicing and a context of Customer Node.

Invoice type expressions can update all direct variables except InvInvoiceId& and InvInvoiceEffectiveDate~.

Note:

Invoice values are not recalculated after invoice type expressions are evaluated, and changes to dependent values are ignored.

Variable Definitions

Variable

Type

Description

InvAccountBalance#

Real

Account balance after the invoice is generated.

If the invoice is associated with a liability account and handling of liability accounts is:

  • Enabled, the value is InvBalanceForward# - DeltaAmount.
  • Not enabled, the value is InvBalanceForward# + DeltaAmount.

DeltaAmount is equal to:

  • InvInvoiceAmount# for invoice accounts
  • InvStatementAmount# for statement accounts.

InvAccountCurrencyId&

Integer

Currency of the account, and in which the invoice is issued.

InvAccountClass&

Integer

Class that an invoice account belongs to. Valid values are:

  • 1 (Asset)
  • 2 (Liability).

InvAccountCategory&

Integer

Category this type of account belongs to. Valid values are:

  • 1 (Receivable)
  • 2 (Payable)
  • 3 (Deposit)
  • 4 (Credit)
  • 5 (Bank) (cash).

InvAccountId&

Integer

ID of the account to which the invoice is applied.

InvAccountInitialDue#

Real

Initial amount due for payment on the account.

Default is InvBalanceForward#.

InvAccountName$

String (30)

Name of the account.

InvAccountTypeId&

Integer

Type of account to which the invoice is applied.

InvAccountTypeName$

String

Name of the account to which the invoice is applied.

InvAccountTypeDisplayName$

String (60)

Name of type of account displayed.

It defaults to InvAccountTypeName$.

Caution:

The value of this column can be translated into a locale-specific language and therefore differ between implementations.

InvApplied&

Integer

Set to 1 (True) if the invoice is applied to the account.

Otherwise, the value of the variable is undefined.

InvBalanceForward#

Real

Balance on the account that was present before application of the invoice.

InvBillRunId&

Integer

ID of the bill run that generated the invoice.

InvConsolidatedInvoice&

Integer

Set to 1 (True) if the invoice is created as a consolidated invoice for one or more statements pending consolidation on the account.

If it is not set to 1, it must be Null.

InvConsolidationInvoiceId&

Integer

ID of the invoice created to consolidate this statement and other statements from the same account into a single invoice.

InvCurrentDue#

Real

Current amount to be paid on the invoice.

Default is InvInvoiceAmount#.

InvCustomerAccountInvoiceStr$

String (30)

Optional ID (unique within the account) generated for the invoice.

InvCustomerInvoiceStr$

String (30)

ID generated for the invoice.

Default is to_string(InvInvoiceId&,'#').

InvCustomerNodeId&

Integer

ID for the customer node for the invoice.

InvEarlyPaymentAccountBalance#

Real

Balance of the account at the early payment date.

InvEarlyPaymentDate~

Date

Date that early payment discounts are allowed.

InvEarlyPaymentDiscount#

Real

Amount of any early payment discount.

InvFormatId&

Integer

ID of the invoice format associated with the invoice image.

InvFormatName$

String (30)

Name for the invoice format associated with the invoice image.

InvGeneral1$ - InvGeneral10$

String (255)

General fields 1 - 10.

InvInvoiceAmount#

Real

Invoice amount.

Either InvInvoiceAmount# or InvStatementAmount# is populated, but not both.

Changes to InvInvoiceAmount# are not supported for receivable type accounts.

InvInvoiceEffectiveDate~

Date

Effective date of the bill run that generated the invoice.

It cannot be updated.

InvInvoiceFormatGeneral1$ -
InvInvoiceFormatGeneral10$

String (255)

General fields 1 - 10 associated with the invoice image for the customer node.

The meaning of these variables is dependent on the invoice format 1 - 10.

InvInvoiceId&

Integer

Invoice ID.

It cannot be updated.

InvInvoiceImageGenerated&

Integer

Set to 1 (True) if an image was generated for the invoice.

Otherwise, the value of the variable is undefined.

InvInvoiceTypeId&

Integer

Type of the invoice.

InvIssueDate~

Date

Date invoice is applied to the account.

Default is InvInvoiceEffectiveDate~.

It must be set to at least InvInvoiceEffectiveDate~ of the current bill run and less than InvInvoiceEffectiveDate~ of the next bill run for the customer so that the account balance forward is calculated correctly.

InvLastModified~

Date

Date-time the invoice was last modified.

InvOriginalPaymentDueDate~

Date

Date that the invoice is originally due for payment.

Default is InvIssueDate~.

InvPaymentDueAccountBalance#

Real

Balance of the account on the payment due date.

InvPaymentDueDate~

Date

Date that payment of the invoice is due.

Default is InvIssueDate~.

InvPaymentDueDiscount#

Real

Pay-on-time discount applied to the invoice.

InvPendingConsolidation&

Integer

Set to 1 (True) to convert the invoice to a statement pending consolidation.

The value is automatically set to 1 (True) by the BGP if one or more statements that already exist on the account are pending consolidation.

InvQA&

Integer

Set to 1 (True) if the invoice or statement is generated for a Quality Assurance or Statement-only bill run.

Otherwise, the value of the variable is undefined.

InvRunning&

Integer

Set to 1 (True) while the billing engine is calculating the value for the invoice.

Otherwise, the value of the variable is undefined.

InvSeqnr&

Integer

Sequence number that identifies a specific invoice image for the invoice.

InvStatementAmount#

Real

Total amount of charges associated with the statement, excluding the balance forward.

Either InvInvoiceAmount# or InvStatementAmount# is populated, but not both.

Changes to InvStatementAmount# are not supported for receivable type accounts.

InvSuppress&

Integer

Set to 1 (True) if the invoice is marked to be suppressed.

Otherwise, the value of the variable is undefined.

This variable cannot be set for statements, quality assurance bill runs, or if statements are pending consolidation.

Note:

If only some invoices in a customer hierarchy have this variable set for a bill run, suppression has no effect. However, the variable remains set for those invoices in the hierarchy that qualify for suppression.

InvTotalAdjustments#

Real

Total adjustments made on the account between the issuing of the previous invoice and this invoice.

InvTotalPayments#

Real

Total payments made on the account between the issuing of the previous invoice and the invoice.
