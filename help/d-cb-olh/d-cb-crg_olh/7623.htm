Programming Constraints

Programming with the TRE API in all languages must conform to some constraints.

Connection Management

Connections to the TRE must conform to the following:

  • An authenticated connection must be established using one of the connection management functions for C, Object Pascal, and Perl before any other TRE functions, except for treInit(), are called. The connection management functions to establish a connection are:
    • treConnect()
    • treConnectExternal()
    • treConnectExternalFlags()
    • treConnectExternalFlagsPE()
    • treConnectFlags()
    • treConnectFlagsPE()
    • treConnectNewPassword()
    • treConnectNewPasswordFlags().
  • The connection must be terminated using treDisconnect(), which must be the last TRE function call.

Note:

For applications connecting through a workstation listener, the Tuxedo environment must be set up before calling a connection management function. Set up the environment by either of the following methods:

  • Using a WSENVFILE file (see the System Configuration Guide for Singleview Convergent Billing)
  • Calling treInit() before other functions.

Memory Management

Memory for arrays and pass-by-reference variables must be allocated using treMalloc(), or re-allocated using treRealloc() if the size has changed, before any function call refers to the array or variable.
