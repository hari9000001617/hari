<DisputeGLFunctionName>&()

Syntax

<DisputeGLFunctionName>&(
    DisputeId&,
    Seqnr&,
    ChargeId&,
    PartitionNr&,
    ChargeDate~
    DisputedAmount#
    var GLAllocation?[])

 

Description

Modifies the GL breakdown for a specified charge in dispute. It is called automatically whenever disputed charge details are updated using the biDisputeChargeUpdate&() function. This is a template for a user-defined function.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

DisputeId&

In: ID of the dispute.

Seqnr&

In: Sequence numbers of the disputed charges.

ChargeId&

In: ID of the charge being disputed.

PartitionNr&

In: Partition number associated with the disputed charge.

ChargeDate~

In: Date of the charge.

DisputedAmount#

In: Amount of dispute.

GLAllocation?[]

Out: Array of hashes containing the GL allocation details.

Hash keys are listed in the table below.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

The following table lists keys of the hashes returrned in GLAllocation?[].

 

Array of Hashes for GLAllocation?[]

Key

Description

TRANSACTION_TYPE_CODE

Type of transaction associate with GL allocation.

Valid values are:

  • 30 (Adjustments Invoice): adjustment amounts against invoice.
  • 31 (Adjustments Credit): adjustment amounts to general credit.

AMOUNT

Amount of the transaction in the currency of the GL codes.

DEBIT_GL_CODE_ID

Debit GL code for this transaction.

CREDIT_GL_CODE_ID

Credit GL code for this transaction.

 

The function is used to override the default GL allocations associated with a billable charge in dispute.

If the <DisputeBAFunctionName>&() function is specified, it is usually necessary to specify this function to correctly perform the GL allocations. If not specified, the GLChargeGuidance&() function is called for the charge to derive its GL Allocations.

It must exist from the current date and time until the end of time.
