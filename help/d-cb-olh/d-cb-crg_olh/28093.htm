Description

Fetches a customer from the database. The function is similar to and wraps the CustomerNodeFetchById&() function.

Context

The following groups of variables are available throughout all stages of processing a customer fetch request, except where otherwise specified:

  • System client
  • Operator
  • Customer.

Parameters

Parameter

Description

customerNodeId&

ID of the customer being fetched.

A customer with the specified ID must exist in the database for the effective date specified; otherwise, an error is raised.

effectiveDate~

Effective date used to identify the historical customer record that is fetched.

fieldValues?{}

fieldNames$[]

fieldValues?[]

Data to be fetched for the specified customer.

The data may be keyed according to the corresponding column name in the CUSTOMER_NODE_TRE_V view and entity validation attribute type display or interface labels.

When keyed:

  • According to database column name, the data returned matches in type and value the data stored in the database.
  • By interface or display label, the data returned matches the type and value expected, as defined in the appropriate attribute type.

Reference type and SQL-based fields return the denormalised string values unless the data type suffix is appended to the field label. Other fields return data of the type defined in the attribute type.

 

Processing

Assuming that all processing stages are enabled, the following list is the order of evaluation of expressions and constraints before the fetch of the customer:

  1. Bundle direct variables if the call is nested.
  2. Retrieve all column values from the appropriate TRE view and assign to direct variables.
  3. Decrypt any encrypted values and mask any appropriate values.
  4. Resolve all variable values that are not null and that have entity validation applied to them.
  5. Assign these values to the values output hash.
  6. Assign appropriate nulls and values to output array (dependent on fieldNames$[] array contents passed).
  7. Unbundle variables stored in step 1.

Returns

1 on successful fetch; otherwise, an appropriate error.
