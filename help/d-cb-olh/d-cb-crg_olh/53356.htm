SUBTOTAL_TRE_V

The SUBTOTAL_TRE_V view contains all details of subtotals known to Singleview.

Subtotals can be queried, inserted, updated, and deleted using the biSUBTOTAL functions.

Column Definitions

Column Name

Properties

Description

SUBTOTAL_ID

NUMBER (8)

Not null

ID of the subtotal.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day. Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

SUBTOTAL_NAME

VARCHAR (67)

Not null

Name of the subtotal.

DESCRIPTION

VARCHAR (2000)

Not null

Description of the subtotal.

LAST_MODIFIED

DATE

Not null

Date-time at which this row was last modified.

OPERATOR_ID

NUMBER (8)

Not null

User who created or last modified this subtotal.

GROUP_ID

NUMBER (8)

Not null

Security group associated with, and controlling access to, this subtotal.

VERSION_STR

VARCHAR (255)

Version ID used to tag this entity.

It is provided for use by third party configuration tools and is not used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this entity.

Refer to the CATEGORY_V view for the list of available categories.

APPLICATION_ENV_CODE

NUMBER (2)

Not null

Application environment of this subtotal.

It is based on the APPLICATION_SUBTOTAL reference type.

MASTER_SUBTOTAL_ID

NUMBER (8)

Read-only

Subtotal from which the values of this subtotal are derived.

It can only contain a value if the subtotal has an application environment of Rating.

DATA_TYPE_CODE

NUMBER (2)

Not null

Possible data types for this subtotal.

It is based on the DATA_TYPE_SUBTOTAL and DATA_TYPE_SUBTOTAL_RATING reference types.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which this specification is used. Valid values are:

  • Rating environment
    • 1 (Normalised Event)
    • 2 (Service)
  • Billing environment
    • 1 (Normalised Event)
    • 2 (Service)
    • 3 (Customer Node)
    • 4 (Customer)
    • 5 (Charge)
  • TRE Table environment
    • 0 (Any).

CONTEXT_RATING, CONTEXT_BILLING, or CONTEXT_TRE_TABLE reference types define the possible values depending on the value of APPLICATION_ENV_CODE.

CURRENCY_ID

NUMBER (8)

Currency of this subtotal.

A currency can only be specified if the data type is Real, Real Array, or Real Hash, and DONT_STORE_IND_CODE is null, CURRENCY_EXPR is null, and the APPLICATION_ENV_CODE is Billing.

The currency must exist for at least this date range.

CURRENCY_EXPR

VARCHAR (255)

Expression that defines the currency to be used for any charges generated by this subtotal.

It must return an integer value corresponding to a valid internal currency identifier. An error is raised if a negative or undefined value is returned.

This expression may reference the key variable if it is defined. If it references the key variable, it is evaluated for each eligible key variable value. Therefore, different components of a tariff can be in different currencies.

If the key variable is not referenced, the expression is evaluated only once and the same value is used for all key values.

SUBTOTAL_STATUS_CODE

NUMBER (2)

Not null

Current status of this instance of the subtotal.

It is based on the SUBTOTAL_STATUS reference type.

ELIGIBILITY_EXPRESSION_
LIST_ID

NUMBER (8)

List of conditional expressions that must be satisfied for this subtotal to be calculated.

If any of these conditions fail, the subtotal is not calculated.

Its value is set to zero for referencing by other entities. No entries appear for the subtotal in the CHARGE table.

DEFAULT_INVOICE_TXT

VARCHAR (255)

Invoice text for each charge created by this subtotal.

It must be null if the subtotal has an application environment of Rating.

SUPPRESS_ZERO_INC_CODE

NUMBER (1)

No entry is recorded when the subtotal is zero.

It must be null if the subtotal has an application environment of Rating.

AGGREGATE_FUNCTION_CODE

NUMBER (1)

Type of aggregation for this subtotal.

It is based on the AGGREGATE_FUNCTION reference type.

AGGREGATE_IND_CODE

NUMBER (1)

Relevant only if the CONTEXT_CODE is Customer Node and the APPLICATION_ENV_CODE is Billing.

If 1 (True), the subtotal is calculated based on the aggregation of all charges to this customer node and all customer nodes below it in the hierarchy.

If 0 (False), the subtotal is calculated based only on charges directly associated with this customer node.

DO_NOT_AGGREGATE_EXPR

VARCHAR (255)

Conditional expression which prevents the value of this subtotal from being aggregated into the parent customer node's aggregated expression parser.

It can only contain a value if AGGREGATE_IND_CODE is 1 (True).

RUNNING_IND_CODE

NUMBER (1)

If 1 (True), the value of this subtotal can be used before its final value is calculated.

This allows for tariffs and other subtotals to be defined whose value changes based on the current amount processed to be defined.

RETAIN_IND_CODE

 

Relevant only if the APPLICATION_ENV_CODE is Billing and the CONTEXT_CODE is not Normalised Event or Charge.

If 1 (True), the values calculated for this subtotal are inserted into the SUBTOTAL_VALUE table.

It must be null or 0 (False) if the subtotal has a context of Normalised Event or Charge.

GLOBAL_IND_CODE

NUMBER (1)

If set to 1 (True), this subtotal is applied to all normalised events, services, customer nodes, or customers.

If set to 1 and the subtotal has an APPLICATION_ENV_CODE of Rating, the CONTEXT_CODE must be set to Global.

TOTAL_SERVICE_IND_CODE

NUMBER (1)

If set to 1 (True), all normalised events applicable to the service, in all accounts in the hierarchy, are examined.

It can only contain a value if the APPLICATION_ENV_CODE is Billing and the CONTEXT_CODE is Service.

DONT_STORE_IND_CODE

NUMBER (1)

If set to 1 (True), this subtotal is not stored in the CHARGE table.

It must be null if RETAIN_IND_CODE is 1 (True).

DONT_DISPLAY_IND_CODE

NUMBER (1)

If set to 1 (True), this subtotal is not displayed in the Invoice Item Detail Form.

It must be null if:

  • DONT_STORE_IND_CODE is 1.
  • The subtotal has an APPLICATION_ENV_CODE of Rating.

KEY_VARIABLE_NAME

VARCHAR (66)

Name of the local expression variable used (if any) in GL_Code or Receivable_Type expressions.

It can only contain a value if the subtotal is an array or hash type and DONT_STORE_IND_CODE is null.

If specified, the GL_CODE expression or RECEIVABLE_TYPE expression is evaluated for each defined array element or hash element in the subtotal, with the variable set to each index or key value in turn. This allows different GL_CODES or RECEIVABLE_TYPES to be associated with different elements in the array or hash.

GL_CODE_ID

NUMBER (8)

General Ledger code to be associated with values generated by this subtotal.

A GL_CODE can only be specified if a currency is specified and DONT_STORE_IND_CODE is null.

If specified, GL_CODE_EXPR must be null.

It must be null if the subtotal has an APPLICATION_ENV_CODE of Rating.

GL_CODE_EXPR

VARCHAR (255)

Expression used in the general ledger code, which must return a valid GL_CODE_ID.

It can only be specified if a currency is specified, DONT_STORE_IND_CODE is null, and GL_CODE_ID is null.

FROM_GL_CODE_ID

NUMBER (8)

GL_CODE to be associated with values generated by this subtotal.

A FROM_GL_CODE_ID can only be specified if a currency is specified, and DONT_STORE_IND_CODE is null.

The GL_CODE must exist for this date range.

If specified, FROM_GL_CODE_EXPR must be null.

It must be null if the subtotal has an APPLICATION_ENV_CODE of Rating.

FROM_GL_CODE_EXPR

VARCHAR (255)

Expression to define the From GL code, which must return a valid GL_CODE_ID.

It can only be specified if a currency is specified, and DONT_STORE_IND_CODE and FROM_GL_CODE_ID are null.

It must be null if the subtotal has an APPLICATION_ENV_CODE of Rating.

RECEIVABLE_TYPE_ID

NUMBER (8)

Receivable type associated with this subtotal.

It can only be specified if a currency is specified, and DONT_STORE_IND_CODE is null.

If specified, RECEIVABLE_TYPE_EXPR must be null.

It must be null if the subtotal has an APPLICATION_ENV_CODE of Rating.

RECEIVABLE_TYPE_EXPR

VARCHAR (255)

Expression defining the receivable type used for charges generated by this subtotal.

It can only be specified if a currency is specified, and if DONT_STORE_IND_CODE and RECEIVABLE_TYPE_ID are null.

It must be null if the subtotal has an APPLICATION_ENV_CODE of Rating (2).

INVOICE_SECTION_NR

NUMBER (4)

Invoice section number associated with all charges created using this subtotal.

It is mandatory if the subtotal has an APPLICATION_ENV_CODE of Billing (3); otherwise, it must be null.

ASSIGNMENT_EXPRESSION_
LIST_ID

NUMBER (8)

List of assignment expressions to be evaluated in order to assign values to direct variables associated with the CHARGE_V view.

It can only be specified if DONT_STORE_IND_CODE is null, and APPLICATION_ENV_CODE is Billing (3).

BILL_CYCLE_IND_CODE

NUMBER (1)

Applicable only if the subtotal has an APPLICATION_ENV_CODE of Rating (2).

If set to 1 (True), values for this rating subtotal are aggregated across periods corresponding to the customer's bill cycle.

If set to 1, FROM_DATE_EXPR and TO_DATE_EXPR must be null.

If not set to 1, it must be null.

If a MASTER_SUBTOTAL_ID is defined, the master subtotal must either have BILL_CYCLE_IND_CODE set to 1, or a date range corresponding to a single day.

RATING_CONTEXT_ACCESS_
IND_CODE

 

Applicable only if the subtotal has an APPLICATION_ENV_CODE of Rating (2), and BILL_CYCLE_IND_CODE is null.

If set to 1 (True), the date period expressions for this subtotal are allowed to access Rating application environment variables and functions.

If not 1 (True), it must be null.

FROM_DATE_EXPR

VARCHAR (255)

Defines the start of the date period to aggregate values for this rating subtotal.

It is only applicable if the subtotal has an APPLICATION_ENV_CODE of Rating (2), and BILL_CYCLE_IND_CODE is null.

If RATING_CONTEXT_ACCESS_IND_CODE is null, the expression has access only to functions with an application environment of Any and a context of Any.

It also has access to the local variable @EffectiveDate~, which is set to the effective date.

If RATING_CONTEXT_ACCESS_IND_CODE is 1 (True), all variables and functions with an application environment of Rating are accessible to the expression.

TO_DATE_EXPR

VARCHAR (255)

Expression that defines the end of the date period to aggregate values for this rating subtotal.

It is only applicable if the subtotal has an APPLICATION_ENV_CODE of Rating (2), and BILL_CYCLE_IND_CODE is null.

If RATING_CONTEXT_ACCESS_IND_CODE is null, the expression has access only to functions with an application environment of Any and a context of Any.

It also has access to the local variables @EffectiveDate~ and @FromDate~, which are set to the effective date, and the value calculated for FROM_DATE_EXPR.

If RATING_CONTEXT_ACCESS_IND_CODE is 1 (True), all variables and functions with an application environment of Rating are accessible to the expression.

RATING_ACCESS_VALUE_IND_
CODE

NUMBER (1)

If set to 1 (True), the value of this rating subtotal can be accessed during rating.

Applicable only if the subtotal has APPLICATION_ENV_CODE set to 2 (Rating).

DO_NOT_FIX_DEPENDENCIES

NUMBER (1)

Pseudo-column

Used for updates only

If specified with a 1 (TRUE), variable dependencies are not fixed.

It is used for multi-phase tariff updates to delay fixing of variable dependencies.
