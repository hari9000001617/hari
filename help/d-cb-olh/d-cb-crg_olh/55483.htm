Description

Fetches an interface template and the derived attribute tables that define the fields on its tabs.

Note:

Derived attribute tables that define grids on the interface template's tabs are excluded. Use fEV_InterfaceTemplateTableFetch?{}() or fEV_InterfaceTemplateTableFetch?[]() to fetch rows in associated derived attribute tables that define grids.

Context

The following groups of variables are available throughout all stages of processing an interface template update request, except where otherwise specified:

  • System client
  • Operator
  • Interface definition
  • Attribute type variables associated with the derived attribute tables to be fetched.

Parameters

Parameter

Description

InterfaceTemplateId&

ID of the interface template to be fetched.

FieldNames$[]

Names of the fields to be fetched.

Names must correspond to the name of:

  • A column in the INTERFACE_DEFN table.
  • The interface label of an attribute type associated with the derived attribute table's entity validation.

FieldValues?[]

Values corresponding to the specified field names.

When keyed:

  • According to database column name, the data returned matches data stored in the database in type and value.
  • By interface label, the data returned matches the type and value expected as defined in the appropriate attribute type.

Reference type and SQL-based fields return the denormalised string values unless the data type suffix is appended to the field label.

 

Processing

The following is the evaluation order for fetching an interface template:

  1. Bundle direct variables if the call is nested.
  2. Retrieve all related columns from INTERFACE_DEFN and INTERFACE_DEFN_DA_ARRAY and assign to their corresponding direct variables.
  3. Decrypt any encrypted values and mask any appropriate values.
  4. Resolve all variable values that are not null and have entity validation applied to them.
  5. Assign these values to the values output hash.
  6. If FieldNames$[] is specified, assign appropriate nulls and values to output array.
  7. Unbundle variables that were stored before the fetch.

Returns

1 if successful; otherwise, an error is raised.
