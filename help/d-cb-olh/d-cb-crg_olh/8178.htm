TablePartition

The TablePartition derived attribute table defines the tables in Convergent Billing that can be partitioned and managed by tools supplied with Convergent Billing.

Where partitioning is not in use, the table defines date-ranged tables that can be automatically archived and purged.

The table has an evaluation type of Table Lookup By Multiple (1-10) Keys.

The table is pre-populated for all applicable Convergent Billing tables. Entries can be added to the table as required for implementation-specific database tables.

Note:

If the table is modified in a non-partitioned environment, run the dbrebuild server utility after modification to create any required additional invisible indexes.

Column Definitions

Column Name

Properties

Description

Table Name (Index 1)

String (30)

Not null

If partitioning is:

  • Enabled, it is the name of a table that can be partitioned.
  • Not enabled, it is the name of a date-ranged table that can be automatically archived or purged.

The entry is forced to uppercase.

Table Abbreviation (Result 1)

String (20)

Not null

Unique abbreviation for the table, used to generate the table partition names.

If Table Name is less than or equal to 20 characters, Table Abbreviation should be the same as Table Name.

The entry is forced to uppercase.

It is ignored if partitioning is not enabled.

Entity Id Column Name (Result 2)

String (30)

Name of the column to be used to perform entity partitioning on the table.

The entry is forced to uppercase.

For example, for the NORMALISED_EVENT table, a value of PARTITION_NR is specified.

If both this column and Date Column Name are specified, this column must be set to PARTITION_NR.

Either this column or Date Column Name, or both, must be specified.

It is ignored if partitioning is not enabled.

Entity Range Table (Result 3)

String (64)

Derived attribute table that defines the numeric ranges to be used for entity partitioning of the table. Valid values are:

  • CustomerPartitionRange
  • EventPartitionRange
  • EventErrorPartitionRange.

It must contain a value only if Entity Id Column Name is specified.

It is ignored if partitioning is not enabled.

Date Column Name (Result 4)

String (30)

If partitioning is:

  • Enabled, it is the name of the column to be used to perform date partitioning on the table.
  • Not enabled, it is the name of the column to be used in archiving and purging data from the table.

The entry is forced to uppercase.

For example, for the NORMALISED_EVENT table, a value of CHARGE_START_DATE is specified.

Either this column or Entity Id Column Name, or both, must be specified.

No Partition Index (Result 5)

String (30)

Name of the index to be used in selecting the data to be archived or purged from the table.

If the index does not exist, it is created as an invisible index.

It is ignored if partitioning is enabled.
