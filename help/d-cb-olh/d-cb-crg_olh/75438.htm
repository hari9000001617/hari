Memory Pool Statistics

The following table lists statistics for memory pools used by the ACM, RSC, and TEC.

 

Memory Pool Statistics

Key

Type

Description

TotalAllocatedBytes

Integer

Total number of bytes allocated since the pool was created or reset.

This value can be less than the sum of the sizes of all allocated blocks, since there may be some wastage per block. For example, 1020 bytes from a 1024-byte block can be allocated to store a data item, with the remaining four bytes wasted.

TotalWastedBytes

Integer

Total number of bytes wasted in allocated blocks, as described in TotalAllocatedBytes.

TotalNrSuccAllocs

Integer

Total number of successful memory allocations from the pool since the pool was created or reset.

TotalNrFailAllocs

Integer

Total number of failed memory allocation attempts since the pool was created or reset.

AvgAllocatedBytes

Double

Average number of bytes in each successful allocation.

MaxAllocatedBytes

Integer

Maximum value of AllocatedBytes.

MaxAllocationSize

Integer

Largest successful allocation size in bytes.

MinAllocationSize

Integer

Smallest successful allocation size in bytes.

TotalBytes

Integer

Size of pool in bytes.

AllocatedBytes

Integer

Number of bytes currently allocated for data storage.

BlockOverheadBytes

Integer

Number of bytes of overhead per block.

FreeBytes

Integer

Unused memory in the pool in bytes.

MinFreeBytes

Integer

Amount of memory in the pool that will be kept free in order to reduce fragmentation of free blocks, in bytes. 

Allocation requests that cause the free memory to fall below this value will fail.

NrAllocatedBlocks

Integer

Number of allocated blocks in the pool.

NrFreeBlocks

Integer

Number of unallocated blocks in the pool.

BlockListStats

Array

Array of statistics from block lists, broken down by block size in bytes.

Each element is a hash containing statistics aggregated from a number of block lists. The contents of the hash are described in the Table 22.

FragStats

Array

Array of number of failed allocations due to fragmentation, broken down by fraction of free memory.

Each element is a hash containing the number of allocation attempts that failed when the portion of memory that was free fell within a particular range (for example, 5% to 10%). The contents of the hash are described in the Table 23.

ShardStats

Array

Array of unknown hashes, with each hash containing statistics on a shard from the pool.

The contents of the hash are described in the Table 24, with element 0 containing statistics for shard 0, element 1 containing statistics for shard 1, and so on.

 

The following table lists the keys in each hash representing block list statistics.

 

Table 22 Block List Statistics

Key

Type

Description

MaxBytes

Integer

Maximum block size in the lists from which these statistics were obtained, in bytes.

MinBytes

Integer

Minimum block size in the lists from which these statistics were obtained, in bytes.

TotalAllocatedBytes

Integer

Total number of bytes allocated for data storage.

TotalWastedBytes

Integer

Total number of bytes wasted.

TotalNrSuccAllocs

Integer

Total number of successful memory allocations from these block lists.

FreeBytes

Integer

Total number of bytes in unallocated blocks.

NrFreeBlocks

Integer

Total number of unallocated blocks.

UsedBytes

Integer

Total number of bytes in allocated blocks.

NrUsedBlocks

Integer

Total number of allocated blocks.

 

The following table lists the keys in each hash representing fragmentation statistics.

 

Table 23 Fragmentation Statistics

Key

Type

Description

MaxFractionFree

Double

Set when an allocation attempt failed, and the fraction of free memory is less than or equal to this value.

MinFractionFree

Double

Set when an allocation attempt failed, and the fraction of free memory is greater than this value.

NrFailAllocs

Integer

Number of failed allocation attempts that occurred when the fraction of free memory free was within this range.

Type

Description

TotalNrFailAllocs

TotalBytes

Integer

AllocatedBytes

Integer

Number of bytes currently allocated for storage of data in pool shard.

FreeBytes

Integer

Amount of currently unused memory in the pool shard, in bytes.
