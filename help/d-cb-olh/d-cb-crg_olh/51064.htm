Variables Based on INV_PAYMENT_V

All direct variables based on the INV_PAYMENT_V view have an application environment of Invoicing and a context of Customer Node.

Variable Definitions

Variable

Type

Description

InvPaymentAmount#

Real

Amount of the payment in the currency of the account.

InvPaymentCurrencyAmount#

Real

Payment amount using the currency of the payment.

It is equal to InvPaymentAmount#, unless the currency is different.

InvPaymentCurrencyId&

Integer

ID of the currency in which the payment is made.

InvPaymentDate~

Date

Date of the payment.

InvPaymentGeneral1$ -
InvPaymentGeneral10$

String

General payment fields 1 - 10.

InvPaymentInvoice$

String

ID of the single invoice for which the payment is made.

InvPaymentInvoice&

Integer

Status code for the invoice. Valid values are:

  • 1 (Invoice)
  • 2 (Split)
  • 3 (Credit).

InvPaymentInvoiceId&

Integer

ID of the single invoice in the PAYMENT_INVOICE table for the payment.

InvPaymentLocation$

String (40)

Location where the payment is received; for example, customer service centre, by post, or at a bank.

InvPaymentLocationCode&

Integer

Code for the location where the payment is received (for example, customer service centre, by post, or at a bank).

InvPaymentReceivedDate~

Date

Date the payment is received.

InvPaymentRejected$

String (40)

If the payment is rejected, it displays the type of rejection (for example, missing signature, inadequate funds, or exceeded credit limit).

InvPaymentRejectedCode&

Integer

Code for the payment rejection reason.

InvPaymentStatus$

String (40)

Text for the status of the payment.

InvPaymentStatusCode&

Integer

Code for the status of the payment. Valid values are:

  • 1 (Accepted)
  • 2 (Rejected).

InvPaymentTypeName$

String (30)

Type of payment.

InvPaymentTypeDisplayName$

String (60)

Name of the payment type displayed.

Defaults to InvPaymentTypeName$.

Caution:

The value of this column can be translated into a locale-specific language and therefore differ between implementations.

InvReceiptNr$

String (30)

Official receipt number assigned to a payment.

InvRejectedDate~

Date

Date the payment is rejected.
