INVOICE_TRE_V

The INVOICE_TRE_V view defines all invoices, statements, or statements pending consolidation known to the system.

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

ID of a customer node invoice.

LAST_MODIFIED

DATE

Not null

Date-time that the row was last modified.

CUSTOMER_INVOICE_STR

VARCHAR (30)

Not null

ID generated for the invoice.

INVOICE_TYPE_NAME

VARCHAR (30)

Not null

Redundant

Type of invoice.

INVOICE_TYPE_ID is derived from the account type of the invoice's account.

INVOICE_TYPE_DISPLAY_
NAME

VARCHAR (60)

Not null

Redundant

Name of account type displayed.

It defaults to INVOICE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

INVOICE_TYPE_ID

NUMBER (8)

Not null

Redundant

Type of invoice.

It is derived from the account type of the invoice's account.

BILL_RUN_ID

NUMBER (15)

Not null

ID of the bill run that generated the account.

RUNNING_IND_CODE

NUMBER (1)

Set to 1 while the billing engine is in the process of calculating the value for the invoice or statement.

It can be used to quickly identify incomplete invoices associated with a failed bill run that needs to be revoked.

QA_IND_CODE

NUMBER (1)

Set to 1 if this is an invoice or statement generated for a Quality Assurance or Statement only bill run.

It can be derived from the bill run when the invoice or statement is originally created. If it is not 1, it must be NULL.

Records with this set are never applied to accounts.

SUPPRESS_IND_CODE

NUMBER (1)

Set to 1 if this invoice is marked to be suppressed. It is set using the Invoice Type expressions.

It is not applicable and cannot be set for statements or Quality Assurance invoices.

If all invoices in a customer hierarchy have this set for a bill run, the invoices and statements are revoked including any rental charges and adjustments.

The CUSTOMER_NODE_BILL_RUN table indicates that invoice generation for the bill run was suppressed for this customer hierarchy.

If only some invoices in a customer hierarchy have this set for a bill run, it has no effect. It remains set for those invoices in the hierarchy that qualified for suppression.

Revoking suppressed invoices is performed by the billing process controller, not the BGP server.

ACCOUNT_NAME

VARCHAR (30)

Not null

ID of the account to which the invoice is applied.

ACCOUNT_ID

NUMBER (15)

Not null

ID of the account to which the invoice is applied.

INVOICED_ACCOUNT_NAME

VARCHAR (30)

Account into which all charges associated with the invoice have been calculated and applied.

INVOICED_ACCOUNT_ID can only contain a value if INVOICE_AMOUNT is 0 and STATEMENT_AMOUNT is not null.

If null and STATEMENT_AMOUNT is not null, this record is a statement for an 'Other Account'.

INVOICED_ACCOUNT_ID

NUMBER (15)

Account into which all charges associated with the invoice have been calculated and applied.

It can only contain a value if INVOICE_AMOUNT is 0 and STATEMENT_AMOUNT is not null.

If null and STATEMENT_AMOUNT is not null, this record is a statement for an 'Other Account'.

CUSTOMER_ACCOUNT_
INVOICE_STR

Optional invoice ID generated for the invoice.

CUSTOMER_NODE_NAME

VARCHAR (200)

Not null

Derived

Customer node that received the invoice.

CUSTOMER_NODE_ID is derivable from the ACCOUNT_ID.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

Derived

Customer node that received the invoice.

It is derivable from the ACCOUNT_ID.

GROUP_ID

NUMBER (8)

Not null

Derived

Security group that has access to the query.

IMAGE_GENERATED_IND_
CODE

NUMBER (1)

Set to True (1) when all images for the invoice have been successfully processed by the IGP, including images that have been suppressed.

It is set by the IGP.

APPLIED_IND_CODE

NUMBER (1)

Set to 1 (True) when the invoice is applied to the account by the apply_invoices server utility as appropriate; otherwise, the value is null.

When an invoice is applied, it cannot be modified in any way (except for flagging it for reprint).

Any account corrections must be made using adjustments displayed on the next invoice run for the customer.

EFFECTIVE_DATE

DATE

Not null

Redundant

Effective date of the bill run that generated the invoice.

ISSUE_DATE

DATE

Not null

Date that the invoice was issued.

It defaults, but is not necessarily equal, to the EFFECTIVE_DATE of the associated invoice run.

It can be set to a different date using invoice type expressions if required.

ORIGINAL_PAYMENT_DUE_
DATE

DATE

Not null

Date that the invoice was originally due to be paid.

PAYMENT_DUE_DATE

DATE

Not null

Date that the invoice is due to be paid.

It is usually the same as ORIGINAL_PAYMENT_DUE_DATE, but can be updated by users with appropriate security privileges.

EARLY_PAYMENT_DATE

DATE

Date for payment to receive an additional discount.

The discount will have been incorporated into the INVOICE_TOTAL.

INVOICE_AMOUNT

NUMBER (19,3)

Not null

Total amount of charges associated with the invoice, excluding the balance forward.

If handling of liability accounts is:

  • Enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.
  • Not enabled, a positive value is a debit for an asset account and a credit for a liability account.

Values stored in this column are at the limits of double precision.

STATEMENT_AMOUNT

NUMBER (19,3)

Total amount of charges associated with the statement.

It can only contain a value if INVOICE_AMOUNT is 0.

For secondary accounts, if handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.

BALANCE_FORWARD

NUMBER (19,3)

Not null

Balance on the account before application of the invoice.

ACCOUNT_BALANCE

NUMBER (19,3)

Not null

Account balance after this invoice was generated.

Note:

It is usually equal to INVOICE_AMOUNT + BALANCE_FORWARD, unless the invoice is a statement associated with an account other than the prime account, in which case it should be equal to STATEMENT_AMOUNT + BALANCE_FORWARD. Rounding of cents can cause this value to differ from the sum.

If handling of liability accounts is enabled and the account is a liability account, the value is set to either of the following:

  • BALANCE_FORWARD - INVOICE_AMOUNT
  • BALANCE_FORWARD - STATEMENT_AMOUNT.

ACCOUNT_INITIAL_DUE

NUMBER (19,3)

Not null

Initial amount due for payment on the account.

Note:

It is usually equal to ACCOUNT_BALANCE, but could be different due to cents rounding or the presence of payment plans (not supported) for the account.

CURRENT_DUE

NUMBER (19,3)

Not null

Derived

Current amount left to be paid on the invoice.

TOTAL_PAYMENTS

NUMBER (19,3)

Not null

Total payments made on the account between the issuing of the previous invoice and the invoice.

The payments are not recorded against the invoice.

The amount is usually displayed on the invoice, and is calculated by the billing engine.

TOTAL_ADJUSTMENTS

NUMBER (19,3)

Not null

Total adjustments made on the account between the issuing of the previous invoice and this invoice.

The adjustments are not recorded against the invoice.

The amount is usually displayed on the invoice, and is calculated by the billing engine.

EARLY_PAYMENT_DISCOUNT

NUMBER (19,3)

Early payment discount that applied for the invoice (if any).

It is derived by the billing engine (based on the INVOICE_TYPE table), and can be re-derived based on details in the CHARGE table.

The amount will have been incorporated into the INITIAL_DUE calculated for the account.

PAYMENT_DUE_DISCOUNT

NUMBER (19,3)

Pay-on-time discount that was applied for the invoice (if any).

It is derived by the billing engine, and can be re-derived based on details in the CHARGE table.

The amount will have been incorporated into the INITIAL_DUE calculated for the account.

LATE_PAYMENT_CHARGE

NUMBER (19,3)

Late payment charge applied to the account if payment is not made by the due date.

It is not incorporated into the INITIAL_DUE for the invoice.

It can be applied to the account if payment was not made by the due date.

EARLY_PAYMENT_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the EARLY_PAYMENT_DATE.

PAYMENT_DUE_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the PAYMENT_DUE_DATE.

UNBILLED_AMOUNT

NUMBER (19,3)

Not null

Derived

Total amount (in the currency of the account) rated by usage-based tariffs, with the ACCOUNT_AGGREGATE_IND_CODE set to 1 and associated with the invoice.

It is used when invoices are applied (or un-applied) to decrement (or increment) the ACCOUNT.UNBILLED_AMOUNT for the account associated with the invoice. The amount is calculated by the BGP.

Note:

The customer node hierarchy account aggregations are not taken into consideration when calculating this amount; that is, it can have a non-zero value for an account associated with a non-reporting node.

If handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied. If handling of liability accounts is not enabled, a positive value is a debit for an asset account and a credit for a liability account.

The updated_invoice_unbilled server utility validates this amount and corrects any discrepancies.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 controlled by the entity validation on the invoice type.

They are populated, as needed, by the invoice type expressions.

PENDING_CONSOLIDATION_
IND_CODE

NUMBER (1)

Set to 1 (True) if the statement is an invoice flagged by the BGP to be suppressed for the current bill cycle.

If set to 1:

  • STATEMENT_AMOUNT must be specified.
  • The statement image is not generated.

If not set to 1, it must be null.

CONSOLIDATION_INVOICE_
ID

NUMBER (15)

ID of the invoice created to consolidate the statement and other statements into a single invoice.

If specified:

  • STATEMENT_AMOUNT must be specified.
  • PENDING_CONSOLIDATION_IND_CODE must be null.

CONSOLIDATED_INVOICE_
IND_CODE

NUMBER (1)

If set to 1 (True):

  • The row is a consolidated invoice for one or more statements.
  • STATEMENT_AMOUNT must be null.

If not set to 1, it must be null.

DISPUTED_AMOUNT

NUMBER (19,3)

Not null

Derived

Total amount of disputed charges associated with the invoice.

A positive value indicates a credit to the account, irrespective of the class of the account and the setting of the ACCOUNT_ENABLED reference type.
