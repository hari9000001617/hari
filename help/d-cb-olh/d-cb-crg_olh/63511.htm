BroadcastFunctionCall&()

Syntax

BroadcastFunctionCall&(
    const FunctionName$,
    const FunctionParams?[],
    const ServerNames$[],
    const ProcessNames$[],
    ExcludeServers$[],
    ExcludeInstances$[],
    const MultiInstance&,
    const RemoteScheduled&)

 

Description

Sends messages to all instances of the specified Tuxedo servers and non-Tuxedo server processes to execute the specified function with the supplied parameters.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

FunctionName$

In: Name of the function to be called in each server.

If undefined or set to an empty string, no notifications are sent and a count of servers and processes that would be notified is returned.

Note:

Database access is required when performing a count, so this parameter must be specified during a database failover or when running in non-database mode.

FunctionParams?[]

In: Parameters to be passed to the function.

ServerNames$[]

In: Array of Tuxedo server names in which this function call is to be executed.

Tuxedo servers are identified by <ServerName> or by using the format specified for the ExcludeServers$[] parameter.

Examples of valid Tuxedo server names include trerodb, trergp:7.1, and MASTER::trebgp:7.10.

ProcessNames$[]

In: Array of non-Tuxedo server process and in which this function call is to be executed.

Processes are identified by <ProcessName> or by using the format specified for the ExcludeServers$[] parameter.

Examples of valid process names include bmp, and bkr:1.

If <ProcessNumber> is not specified, all processes with that name are notified.

ExcludeServers$[]

In: Array of Tuxedo servers and non-Tuxedo server processes to be excluded from performing this function call.

Tuxedo servers are identified using the format:

[<Instance>::]<ServerName>:<GroupId>.<ServerId>

If <Instance> is not specified, the exclude servers' filter is applied to all instances.

Non-Tuxedo server processes are identified using the format:

<ProcessName>:<ProcessNumber>

Note:

This parameter is not compatible with database failover, and must not be specified during a failover or when running in non-database mode.

ExcludeInstances$[]

In: Array of Convergent Billing instances to be excluded from broadcasting this function call.

MultiInstance&

In: Specifies whether or not the function should be broadcast to all Convergent Billing instances in the cluster.

If set to:

  • Non-zero value (TRUE), a message is sent to all instances.
  • 0 (FALSE), only the specified servers and processes on the current instance are notified.

If not specified, the value is set to TRUE.

RemoteScheduled&

In: Specifies whether or not broadcasts to remote instances are scheduled.

Note:

This is a deprecated parameter. Scheduled broadcast function calls on remote instances should be explicitly scheduled.

If set to TRUE:

  • MultiInstance& must be set to TRUE.
  • The <Instance>::biBroadcastFunctionCall&() function is scheduled for each active remote instance with a start delay time of ORACLERACPropagationDelay&() seconds specified.

If not specified, the value is set to FALSE. If set to FALSE, the <Instance>::biBroadcastFunctionCall&() function is immediately called for each active remote instance.

 

Returns

If FunctionName$ is set to:

  • A value and this function is successful, the number of responders is returned.
  • An empty string or is undefined, the number of servers and processes that would be notified is returned.

Otherwise, an error is raised and a warning message is logged for each active remote instance.

Remarks

This function is used by many of the purge functions to send purge messages to the appropriate set of servers.

The notification rate is throttled to a configured number of notifications per second (within an instance). The configured number is provided by the BroadcastFunctionMaxNotificationsPerSecond&() function. However, if FunctionName$ is set to DatabaseFailover&() or BMPDatabaseFailover&(), the notifications are sent immediately.

Tuxedo Servers

For each Tuxedo server specified in ServerNames$[], this function calls the biStatusServer&() function to determine the set of servers running with that name. For each server found that is not specified in ExcludeServers$[], a biTrcInvoke event is posted to execute the function specified in the FunctionName$ parameter.

Non-Tuxedo Server Processes

For each non-Tuxedo server process, a query is performed on the CONFIGURATION_ITEM table to find all configured processes with the name specified in ProcessNames$[] if any of the following are true:

  • ExcludeServers$[] is specified
  • FunctionName$ is undefined or set to an empty string.

For each process found that is not specified in ExcludeServers$[], an event of the format <ProcessName>:<ProcessNumber> is posted to get the process to the function specified in the FunctionName$ parameter. If a configuration item cannot be found for a specified <ProcessName>:<ProcessNumber>, all processes of the specified <ProcessName> are notified.

Note:

To avoid querying the database during a database failover or when running in non-database mode, ensure that both of the following are true:

  • ExcludeServers$[] is not specified
  • FunctionName$ is set to a valid function name.

If ExcludeServers$[] is not specified, the specified processes are notified without querying the database.
