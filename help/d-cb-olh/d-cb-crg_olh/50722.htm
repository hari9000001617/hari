RECEIVABLE_TYPE_TRE_V

The RECEIVABLE_TYPE_TRE_V view contains historical and current information about receivable types.

Column Definitions

Column Name

Properties

Description

RECEIVABLE_TYPE_ID

NUMBER (8)

Not null

ID of receivable type.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

RECEIVABLE_TYPE_NAME

VARCHAR (64)

Not null

Name of receivable type for this date range.

RECEIVABLE_TYPE_DISPLAY_
NAME

VARCHAR (64)

Not null

Name of receivable type displayed.

It defaults to RECEIVABLE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

DESCRIPTION

VARCHAR (2000)

Not null

Description of this receivable type.

LAST_MODIFIED

DATE

Not null

Date-time the record was last modified.

OPERATOR_ID

NUMBER (8)

Not null

ID of user who created or last modified the record.

GROUP_ID

NUMBER (8)

Not null

ID of security group associated with, and controlling access to, the record.

VERSION_STR

VARCHAR (255)

Version ID string of receivable type.

It is provided for use by third party configuration management tools, and is not used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this receivable type.

Refer to the CATEGORY_V view for the list of available categories.

PRIORITY

NUMBER (4)

Not null

Priority given to receivable type in allocating outstanding payments and adjustments.

A lower numerical value is considered to be a higher priority.

It is used at discretion of the allocation algorithm.

GL_CODE_ID

NUMBER (8)

General Ledger code associated with all payments and adjustments made against this receivable type.

DEFAULT_IND_CODE

NUMBER (1)

If set to 1, (TRUE), this receivable type is the system default receivable type.  Otherwise, it must be null.

If one or more receivable types are defined for a given date-time, one of them must be set to TRUE.
