ENTITLEMENT_TYPE_TRE_V

The ENTITLEMENT_TYPE_TRE_V view defines the types of entitlements available in Singleview.

Column Definitions

Column Name

Properties

Description

ENTITLEMENT_TYPE_ID

NUMBER (8)

Not null

ID of a type of entitlement.

LAST_MODIFIED

DATE

Not null

Date-time at which this row was last modified.

OPERATOR_ID

NUMBER (8)

Not null

Operator that created or last modified this record.

GROUP_ID

NUMBER (8)

Not null

Group associated with, and controlling access to, this variable.

ENTITLEMENT_TYPE_NAME

VARCHAR (30)

Not null

Name of the entitlement type.

ENTITLEMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Not null

Name of the entitlement type displayed to the customer.

It must not be referenced by configuration code, it may undergo language translation.

It defaults to ENTITLEMENT_TYPE_NAME.

DESCRIPTION

VARCHAR (4000)

Not null

Description of the entitlement type.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which entitlements of the type operate. Valid values are:

  • 2 (Service)
  • 3 (Customer Node)
  • 4 (Customer).

ALLOC_GEN_ADVANCE_PERIOD

NUMBER (6)

Not null

Default 300

Duration of the advance period (seconds) for entitlements of the type.

Allocations can be generated automatically for an entitlement in advance of the entitlement start date.

If a system-wide advance period is specified, the value is added to the value of this column to obtain the duration of the advance period for generation of allocations of entitlements of the type.

The value cannot be less than zero.

ALLOC_GEN_FUNCTION_DEFN_
ID

NUMBER (10)

ID of a function to be evaluated to generate an allocation for an entitlement of the type.

If null, the EntitlementAllocGen&() default function is applied.

ALL_FIRST_USE_FUNCTION_
DEFN_ID

NUMBER (10)

ID of a function to be evaluated on the first use of an allocation for an entitlement of the type.

The function is evaluated during rating of a normalised event.

WARNING_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function to be evaluated when an entitlement of the type is about to expire.

WARNING_ALLOC_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function to be evaluated when an allocation for an entitlement of the type is about to expire.

EXPIRY_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function to be evaluated when an entitlement of the type expires.

EXPIRY_ALLOC_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function to be evaluated when an allocation for an entitlement of the type expires.

BILL_CYCLE_IND_CODE

NUMBER (1)

If set to 1 (True), entitlements of the type are aligned with bill cycles for customers.

If not null, it must be set to 1.

PRORATE_IND_CODE

NUMBER (1)

If set to 1 (True), prorating applies to entitlements of the type.

If not null, it must be set to 1.

PRORATE_FUNCTION_DEFN_ID

NUMBER (10)

ID of a function used to prorate allocations to entitlements of the type.

If null, the EntitlementAllocProrate#() default function is used to prorate.

ENTITY_VALIDATION_ID

NUMBER (8)

Set of validation rules, labels, and hint text, to be applied to each entitlement of the type.

If not null, it must reference an entity with a VALIDATION_TABLE_NAME of ENTITLEMENT.

REAL_TIME_RECURRING_
TARIFF_ID

NUMBER (8)

ID of a real-time tariff used to generate recurring charges for entitlements of this type.

If specified, the tariff must exist from the current date-time until the end of time.

REAL_TIME_ACTIVATION_
TARIFF_ID

NUMBER (8)

ID of a real-time tariff used to generate real-time activation charges for entitlements of this type.

If specified, the tariff must exist from the current date-time until the end of time.

REAL_TIME_CANCEL_TARIFF_
ID

NUMBER (8)

ID of a real-time tariff used to generate real-time cancellation charges for entitlements of this type.

If specified, the tariff must exist from the current date-time until the end of time.

VERSION_STR

VARCHAR (255)

Version identification string.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with the entity.
