ADJUSTMENT_BATCH_TRE_V

The ADJUSTMENT_BATCH_TRE_V view defines the columns for inserting and querying biAdjustmentBatch services.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_BATCH_ID

NUMBER (15)

Not null

Read-only

ID that defines an adjustment batch.

ADJUSTMENT_BATCH_NR

VARCHAR (30)

Not null

Read-only

ID for the batch adjustment that can be referenced by users.

It is automatically generated, based on the ADJUSTMENT_BATCH_ID.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Read-only

Type of adjustments being made in the batch.

All adjustments in a batch must be of the same type.

ADJUSTMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Not null

Name of adjustment type displayed.

It defaults to ADJUSTMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

Read-only

Type of adjustments being made in the batch.

All adjustments in a batch must be of the same type.

CURRENCY_SYMBOL

VARCHAR (5)

Read-only

Currency to be used for all adjustments associated with the batch.

It defaults to the currency ID of the adjustment type.

If no currency is specified, this batch can contain payments in different currencies.

CURRENCY_ID

NUMBER (8)

Read-only

Currency to be used for all adjustments associated with the batch.

It defaults to the currency ID of the adjustment type.

If no currency is specified, this batch can contain payments in different currencies.

TOTAL_AMOUNT_CURRENCY_
SYMBOL

VARCHAR (5)

Not null

Read-only

Currency of TOTAL_AMOUNT.

If not specified at insert, the value of CURRENCY_ID is used.

TOTAL_AMOUNT_CURRENCY_ID

NUMBER (8)

Not null

Read-only

Currency of TOTAL_AMOUNT.

If not specified at insert, the value of CURRENCY_ID is used.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that the record was last modified.

DESCRIPTION

VARCHAR (255)

Not null

Read-only

General description of this set of batch adjustments.

BATCH_TYPE_CODE

NUMBER (8)

Not null

Read-only

Type of batch adjustments being performed here.

It is based on the BATCH_TYPE reference type.

BATCH_DATE

DATE

Not null

Read-only

Date-time that the batch adjustment details were entered into the system.

Note:

If not specified, it is automatically defaulted to the current date-time.

OPERATOR_ID

NUMBER (8)

Not null

Read-only

User who entered the batch adjustment details.

All batch adjustments and other automatic billing system processes operate under a common user ID.

BATCH_STATUS_CODE

NUMBER (8)

Not null

Read-only

Status of the adjustment batch being entered.

When a set of batch adjustments is verified they can no longer be modified as part of the batch processing.

It automatically defaults to Entered for new batches.

Valid values are:

  • 1 (Entered): Details have been entered by the user or are in the process of being entered by the user. None of the adjustments in the batch have been applied.
  • 2 (Applied): All adjustments in the batch have been applied to the account. Details can no longer be modified.
  • 3 (Partially Applied): Some of the adjustments in the batch have been applied to the account. Some adjustments are future-dated and were not applied.

It is automatically derived by the biAdjustmentBatch... functions.

It is based on the BATCH_STATUS reference type.

TOTAL_AMOUNT

NUMBER (19,3)

Not null

Read-only

Sum of all adjustments made under the batch.

It is derived from the ADJUSTMENT table.

If the batch contains adjustments in different currencies, conversion from each adjustment currency to the currency specified in TOTAL_AMOUNT_CURRENCY_ID occurs.

It cannot be specified when inserting a batch adjustment.

It defaults to 0 on insertion, and is incremented or decremented for each adjustment inserted or deleted.

ENTERED_COUNT

NUMBER (8)

Not null

Read-only

Number of adjustments in the batch with a status of Entered or Future-dated.

It is automatically derived by the biAdjustment... and biAdjustmentBatch... functions.

APPLIED_COUNT

NUMBER (8)

Not null

Read-only

Number of adjustments in the batch with a status of Applied.

It is automatically derived by the biAdjustment... and biAdjustmentBatch... functions.
