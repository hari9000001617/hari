EXTERNAL_DEFN_ELEMENT_TRE_V

The EXTERNAL_DEFN_ELEMENT_TRE_V view displays the elements in an external definition.

Column Definitions

Column Name

Properties

Description

EXTERNAL_DEFN_ELEMENT_
ID

NUMBER (8)

Not null

ID of an external definition element.

EXTERNAL_DEFN_ID

NUMBER (8)

Not null

ID of the external definition that the element belongs to.

For more information, refer to EXTERNAL_DEFN_TRE_V view.

SEQNR

NUMBER (4)

Not null

Order of elements within the external definition.

Note:

It must form a contiguous range starting at 1.

SRC_EXTERNAL_DEFN_
ELEMENT_ID

NUMBER (8)

ID of the source external definition element that this element was generated from.

Note:

The element must be part of an external definition that has been generated from a source external definition, and the value must be a valid element associated with the source. Otherwise, it must be null.

ELEMENT_NAME

VARCHAR (60)

Name of the element.

If specified, this must be unique within the external definition. It can only be specified if ATTRIBUTE_NAME is null.

Note:

If SRC_EXTERNAL_DEFN_ELEMENT_ID is specified, the value must be the same as the value of ELEMENT_NAME in the source external definition.

COLUMN_NAME

VARCHAR (30)

Name of the element's column if the external definition is used to create a view. The default value is ELEMENT_NAME.

It can only be specified if ELEMENT_NAME or ATTRIBUTE_NAME are specified, and XML_DATA_TYPE_CODE is specified.

Note:

It must be unique within this external definition.

ATTRIBUTE_NAME

VARCHAR (60)

Name of the attribute.

It can only be specified if ELEMENT_NAME is null.

Note:

If SRC_EXTERNAL_DEFN_ELEMENT_ID is specified, the value must be the same as the value of ATTRIBUTE_NAME in the source external definition.

ATTR_EXTERNAL_DEFN_
ELEMENT_ID

NUMBER (8)

ID of the element to which the attribute belongs.

It can only be specified if ATTRIBUTE_NAME is specified. If specified, the referenced element must be a part of this definition.

Note:

If specified, the combination of ATTRIBUTE_NAME and ATTR_EXTERNAL_DEFN_ELEMENT_ID must be unique.

If ATTRIBUTE_NAME is specified and this column is null:

  • This attribute must be a part of an attribute group.
  • The combination of EXTERNAL_DEFN_ID and ATTRIBUTE_NAME must be unique.

DESCRIPTION

VARCHAR (4000)

Description of the element or attribute.

XML_DATA_TYPE_CODE

NUMBER (3)

XML simple data type of the element or attribute.

It is based on the EXTERNAL_DEFN_XML_DATA_TYPE reference type.

The supported data types are a subset of the full set of XML schema built-in data types.

It must be specified for a validated external definition if REF_EXTERNAL_DEFN_ID and REF_EXTERNAL_DEFN_ELEMENT_ID are null, and ELEMENT_NAME or ATTRIBUTE_NAME are specified. This constraint is not enforced for a non-validated definition.

It must be null if both ELEMENT_NAME and ATTRIBUTE_NAME are null.

If SRC_EXTERNAL_DEFN_ELEMENT_ID is specified, its value must be the same as the value of the XML_DATA_TYPE_CODE of the source external definition element.

REF_EXTERNAL_DEFN_ID

NUMBER (8)

ID of the external definition referenced by this element.

If XML_DATA_TYPE_CODE is:

  • Null, the referenced external definition is embedded in the current definition at this element position. If ELEMENT_NAME is also null, the referenced external definition is embedded at the current nesting level; otherwise, it is embedded under ELEMENT_NAME.
  • Not null, no embedding occurs. The value associated with this element is assumed to be a reference (for example, an internal identifier) to the entity associated with REF_EXTERNAL_DEFN_ID.

If this external definition and REF_EXTERNAL_DEFN_ID have views associated with them, a foreign key between the two views is created for the column associated with this element.

If REF_EXTERNAL_DEFN_ELEMENT_ID is specified, it must be null.

If ATTRIBUTE_TYPE_ID and PE_FACT_ID are null for this element, REF_EXTERNAL_DEFN_ID must have the same entity (CB_ENTITY or PE_CLASS_ID) as either of the following:

  • The external definition this element is associated with.
  • An external definition that is linked to the external definition this element is associated with.

If ATTRIBUTE_TYPE_ID or PE_FACT_ID are specified for this element, they must be a reference to an entity (scalar or array), and REF_EXTERNAL_DEFN_ID must reference the same entity.

REF_EXTERNAL_DEFN_
ELEMENT_ID

NUMBER (8)

ID of a linked external definition from which this element obtains its value.

If REF_EXTERNAL_DEFN_ID is specified, it must be null.

If XML_DATA_TYPE_CODE is null, this element is not exposed in any external interface. It is used to provide a value for this entity for a Convergent Billing or a Process Engine interface.

If specified on input, the value of the referenced element is passed through to the Convergent Billing or the Process Engine interface. For output interfaces, the referenced value can be used to perform a fetch on the entity, if no other direct links are provided to define search or fetch criteria.

If XML_DATA_TYPE_CODE is:

  • Null, INTERFACE_LABEL must be specified.
  • Not null and the referenced element is a part of the interface, the value of this element must match the value of the referenced element. For output interfaces, the value is duplicated from the referenced element to return its value. For input interfaces, the two supplied values must be identical.

If the referenced element is associated with a fetch external definition (FETCH_IND_CODE is set to 1) and this element is a part of a different external definition, the fetch is performed before this definition is called.

CHOICE_DEFAULT_IND_CODE

NUMBER (1)

If set to 1 (True), it is the default choice element for a choice content model.

CHOICE_VALUE

VARCHAR (255)

Exact value of the choice element that will result in the use of this element.

It can only be set if EXTERNAL_DEFN_ID is of a choice content model.

The value specified must be compatible with the data type of the choice element. If the choice element is not mandatory, it can be null.

If there are multiple potential values that could result in the use of this element, the additional non-null values can be specified in the EXTERNAL_DEFN_CHOICE_TRE_V view.

MANDATORY_IND_CODE

NUMBER (1)

If set to 1 (True), the following applies:

  • Element is mandatory.
  • NILLABLE_IND_CODE must be NULL.

A non-null value must be specified on input or output.

Note:

If MANDATORY_IND_CODE and NILLABLE_IND_CODE are set, MANDATORY_IND_CODE is used.

EFFECTIVE_DATE_IND_CODE

NUMBER (1)

If set to 1 (True), it defines the effective date to be used for query or access on this date-ranged entity.

This element can have an attribute type or fact associated with it.

KEY_IND_CODE

NUMBER (1)

If set to 1 (True), it:

  • Defines part of the lookup key for this entity.
  • Must have an INTERFACE_LABEL specified for it unless EFFECTIVE_DATE_IND_CODE is also set to 1 (True).

NILLABLE_IND_CODE

NUMBER (1)

If set to 1 (True), the following applies:

  • Element supports the XML schema instance nil attribute being specified to indicate that a null value has been specified.
  • MANDATORY_IND_CODE must be NULL.

    Note:

    If MANDATORY_IND_CODE and NILLABLE_IND_CODE are set, MANDATORY_IND_CODE is used.

MIN_OCCURS

NUMBER (6)

Not null

Minimum number of occurrences.

The value specified must be greater than or equal to 0. The default value is 1. A value of 0 implies that the element is optional.

If it is greater than or equal to 1, the element is mandatory, regardless of the value of MANDATORY_IND_CODE.

If MANDATORY_IND_CODE is set to 1, it must have a value greater than or equal to 1.

MAX_OCCURS

NUMBER (6)

Maximum number of occurrences.

The default value is 1. If undefined, there is no maximum. This corresponds to a value of 'unbounded' when defined in an XML schema.

DEFAULT_VALUE

VARCHAR (255)

Default value to be applied to this element.

On input, it is used if this element is not specified, or if the element is mandatory and an empty string or NULL value has been specified.

On output, it is used if the element is not linked to a Convergent Billing attribute type or Process Engine fact, or it is linked but a NULL value was returned from Convergent Billing or Process Engine for this element.

The XML Schema format for the value is used to store the value in this column. For example, for date-time values, the format yyyy-mm-ddThh:mm:ss is used.

FIXED_VALUE

VARCHAR (255)

Fixed value that is always used for this element.

On input, this element must be specified and its value must match the value specified in this column.

If ELEMENT_NAME and ATTRIBUTE_NAME are null, this element is not specified on input and FIXED_VALUE is used as the default value of a Convergent Billing attribute type or a Process Engine fact as part of this interface.

On output, this value is always output for this element. If this element is linked to an attribute type or fact, the value of the attribute type or fact must match this value.

The XML Schema format for the value is used to store the value in this column. For example, for date-time values, the format yyyy-mm-ddThh:mm:ss is used.

ATTRIBUTE_TYPE_ID

NUMBER (8)

ID of the Convergent Billing attribute type that is associated with this element.

If this column and REF_EXTERNAL_DEFN_ID are specified, ATTRIBUTE_TYPE must have a REFERENCE_ENTITY_NAME that equals the value of CB_ENTITY for the REF_EXTERNAL_DEFN_ID.

If REF_EXTERNAL_DEFN_ID is not specified, one of the following must apply to this attribute type:

  • No ENTITY_NAME specified.
  • An ENTITY_NAME specified that matches the entity of this external definition, or an external definition linked to this definition.
  • FETCH_IND_CODE set to 1 (True).

PE_FACT_ID

NUMBER (20)

ID of the Process Engine fact associated with this element

If this column and REF_EXTERNAL_DEFN_ID are specified, PE_FACT_ID must correspond to an object reference or binref whose associated class equals the PE_CLASS_ID for the REF_EXTERNAL_DEFN_ID.

If REF_EXTERNAL_DEFN_ID is not specified, this fact must reference the same class as this external definition, or an external definition linked to this definition.

INTERFACE_LABEL

VARCHAR (64)

Name of this element used when interfacing to Singleview APIs.

For an element linked to a Process Engine fact, it must be set to the name of the fact.

For an element linked to a Convergent Billing attribute type or a Convergent Billing external definition, it is one of the following:

  • Name of the INTERFACE_LABEL associated with the attribute type, if specified.
  • Name of the LABEL associated with this attribute type, if it has no INTERFACE_LABEL, and LABEL is unique for the Convergent Billing entity.
  • Convergent Billing column name for this Convergent Billing entity. If an attribute type is specified for this element, it must be associated with this column name using entity validation.

INTERFACE_DATA_TYPE_
CODE

NUMBER (2)

Redundant

EPM data type expected for the value of this element being passed to or from Singleview APIs.

It is based on the DATA_TYPE_ATTRIBUTE_TYPE reference type.

For an element linked to a:

  • Convergent Billing attribute type, it is always equal to the DATA_TYPE_CODE of the attribute type.
  • Process Engine fact, it is derived from the type of the fact.

FACT_PROCESS_SEQNR

NUMBER (3)

Order in which Process Engine facts associated with this external definition are to be processed by Process Engine.

It must be specified for external definitions that support insert or update interfaces.

PARAMETER_NAME

VARCHAR (60)

Name of a parameter defined in the search SQL, if this external definition is for a search.

PE_PARAM_ID

NUMBER (20)

ID of a Process Engine parameter.

It can only be specified if the associated external definition is a search associated with a Process Engine class that references a saved search, or a Process Engine SXP expression.

DENORMALISE_IND_CODE

NUMBER (1)

Set to 1 (True) if this element is to be converted to or from an integer representation to a string representation of the value.

It can only be set to 1 if the XML_DATA_TYPE_CODE is a string data type and either of the following applies:

  • An attribute type based on a reference type is specified for ATTRIBUTE_TYPE_ID.
  • PE_FACT_ID is specified and corresponds to an object reference.

NAME_IND_CODE

NUMBER (1)

Set to 1 (True) if this element is denormalised and the string representation should be based on the NAME field for the configuration entity rather than the DISPLAY_NAME field.

For Convergent Billing attribute types associated with a reference type, it is interpreted as the string representation being based on CODE_LABEL rather than the ABBREVIATION.

ENUMERATION_IND_CODE

NUMBER (1)

Set to 1 (True) if the XML schema representation of this element should explicitly define an enumeration for this element containing all possible values.

If DENORMALISE_IND_CODE is NULL, the enumeration is a list of numeric values. If it is 1 (True), the enumeration is a list of strings.

It can only be set to 1 (True) if either of the following applies:

  • An attribute type based on a reference type is specified for ATTRIBUTE_TYPE_ID.
  • PE_FACT_ID is specified and corresponds to an object reference.

The enumeration must consist of all:

  • Reference codes of the associated reference type.
  • Configured entities of the specified REFERENCE_CONFIG_ENTITY_CODE as of the current date and time.
  • Objects associated with the same class as the object referenced by PE_FACT_ID.

DISPLAY_FORMAT

VARCHAR (80)

Display format for this attribute.

It is only applicable on output if the XML data type is string and the corresponding attribute type or Process Engine fact is not a string.

For supported formats, refer to the to_string() EPM function.

FETCH_IND_CODE

NUMBER (1)

Set to 1 (True) if the value of a Convergent Billing attribute type or Process Engine fact must be retrieved after the insert or update operation has completed, or before the delete operation is performed.

It allows:

  • Additional information about the entity that has been inserted, updated, or deleted to be accessible to linked external definitions.
  • Information provided in the interface to be modified by Convergent Billing and Process Engine, and for the modified information to be made available to linked external definitions.

If not set to 1 (True), it must be NULL.

It can only be set to 1 if:

  • INTERFACE_LABEL is specified.
  • The associated external definition supports insert, update, or delete operations.

SENSITIVE_IND_CODE

NUMBER (1)

Set to 1 (True) if this element contains sensitive data (for example, when using a masked or encrypted attribute type or fact).

It allows external definition interfaces to manage protected data.

It must be Null or 1.
