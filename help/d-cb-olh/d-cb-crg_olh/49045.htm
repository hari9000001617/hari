biDocumentImageRetrieve$()

Syntax

biDocumentImageRetrieve&(
    DocumentImageId&,
    ImageType$,
    Regenerate&,
    var CachedCopy&)

 

Description

Retrieves the specified document image from the cache, or regenerates the image file.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

DocumentImageId&

In: ID of required document image.

ImageType$

In: Type of image (for example, docx).

Regenerate&

In: If set to TRUE, ignore any cached images and regenerate the image file.

CachedCopy&

Out: Set to TRUE if a cached copy is found and returned.

Set to FALSE if an image file is generated.

It is FALSE if Regenerate& is set to TRUE.

 

Returns

File name of the image prefixed by the CALLER_PREFIX attribute of the DOCUMENT_IMAGE configuration item; otherwise, an error is raised.

Remarks

If no image exists and Regenerate& is TRUE, the function generates an image for the specified document called DocumentImageId&.ImageType$.

If ImageType$ is not specified, the function uses the DisplayImageType derived attribute table.

In a clustered environment, the DOCUMENT_IMAGE configuration item, on the instance where the request is processed, is used to retrieve a document image file. In a non-clustered environment, the DOCUMENT_IMAGE configuration item with a sequence number of 1 is used.

The function uses the following attributes from the selected DOCUMENT_IMAGE configuration item to determine the file name of a document image:

  • CACHE_DIRECTORY
  • CALLER_PREFIX
  • INSTANCE.

The function calls the ivp server utility (-i option)to generate the image. A lookup on the DisplayImageMethod derived attribute table is used to determine the script name parameter by using :

  • An Entity Image Type of Document
  • An Output Image Type retrieved from the DOCUMENT_IMAGE table
  • Required Display Image Type.

If a row is returned, script name is passed to the ivp using the -script option to generate the image. If no row is returned, the ivp uses view_inv_<display_image_type>.
