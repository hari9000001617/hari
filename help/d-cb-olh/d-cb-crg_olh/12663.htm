biInvoiceRevokeParallel&()

Syntax

biInvoiceRevokeParallel&(
    BillRunId&,
    EffectiveDate~,
    BillRunOperationId&,
    RevokeParallelConfigItemSeqnr&,
    const RootCustomerNodeList&[],
    var SuccessCustomerNodeList&[],
    var ErrorCustomerNodeList&[],
    var Statistics?{})

 

Description

Performs the revoke invoices or statements billing operation for a set of customers in parallel.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

BillRunId&

In: Internal ID of the bill run being processed.

EffectiveDate~

In: Effective date of the bill run.

BillRunOperationId&

In: Internal ID of the bill run operation being processed for the revoking of customer invoices.

RevokeParallelConfigItemSeqnr&

In: Sequence number for the INVOICE_REVOKE_PARALLEL configuration item to use when revoking the invoices.

RootCustomerNodeList&[]

In: List of root customer node IDs to have their invoices revoked.

SuccessCustomerNodeList&[]

Out: List of root customer node IDs that successfully had their invoices revoked.

ErrorCustomerNodeList&[]

Out: List of root customer node IDs that failed to have all or their invoices revoked.

Statistics?{}

Out: Unknown EPM hash containing the statistics gathered during the processing of the list of root customers.

The table below lists the parameter hash keys.

 

Returns

Number of customers successfully processed; otherwise, an error is raised.

Remarks

The function is used when multi-processing capability is required. For each customer, it finds all invoices or statements on the bill run and performs the following operations:

  • Deletes all charges associated with the invoice or statement generated by the billing engine.
  • Updates all charges associated with the invoice or statement generated by the rating engine.
  • Deletes the invoice or statement record with the receivable type breakdown (if any).

Because of the many charges that may be associated with an invoice or statement, period commits are performed during the first and second operation.

A lock is obtained on each customer as it is processed, and the outcome of the revoke is recorded in the database.

The maximum number of child processes spawned by the function is controlled by the INVOICE_REVOKE_PARALLEL configuration item (with the sequence number specified in the InvoiceRevokeParallelItemSeqnr& parameter). The MAX_CHILD_PROCESSES attribute contains the maximum number of child processes that can be spawned.

The following table lists the Statistics?{} parameter hash keys.

 

Statistics?{} Parameter Hash Keys

Key

Value

InvoicesDeleted

Number of invoices deleted during the revoke.

StatementsDeleted

Number of statements deleted during the revoke

InvoiceAmount

Sum of the invoice amounts revoked.
