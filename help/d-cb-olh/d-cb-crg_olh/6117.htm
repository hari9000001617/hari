Edit Masks for Attribute Types

The edit masks supported by Convergent Billing define the exact format of the data to be entered in a field, and are typically used only for string data types.

Note:

Fields with an edit mask and fields with a display mask (referred to as 'masked fields') are not the same. Edit masks define how data is entered, and display masks define how data is displayed (see the Security and Auditing Configuration Guide for Singleview).

For each character that is expected, the edit mask can define exactly what type of character is expected (for example, letter, number, or either) and whether it is mandatory or optional. Edit masks are useful for entering and displaying information like credit card numbers or phone numbers in a predefined format.

An edit mask is made up of one, two, or three fields, separated by semi-colons. The first field is the mask itself (that restricts the characters that can be entered).

The second field determines whether any literal characters included in the mask are saved as part of the data. Literal characters are any fixed characters that the user does not input, but are displayed as part of the field to assist data entry.

The third field contains the character used to represent a character not yet entered by the user. For examples of use of these fields, refer to the table below that lists mask examples.

The table below lists the characters that can be used to create an edit mask for an attribute type. These characters make up the first field in the mask (if more than one field is being used).

 

Characters Used to Create Edit Masks

Character

Use

Description

!

Exclamation mark

If included, optional characters not entered are represented as leading blanks.

If excluded, optional characters are represented as trailing blanks.

>

'Greater than' symbol

If included, all characters following the > character are formatted in upper case until either the end of the mask is reached or until a '<' character is encountered.

<

'Less than' symbol

If included, all characters following the < character are formatted in lower case until either the end of the mask is reached or until a '>' character is encountered.

<>

'Less than' and 'greater than' symbols

If this pair of characters is entered, no case checking is performed and the data is formatted using the case in which a user enters the data.

\

Backslash

Allows one of the mask characters to be specified as literal in the data.

The character following the backslash is regarded as the literal character.

L

Upper-case 'L'

Requires an alphabetic character in this position.

For most western countries, this character must be A-Z or a-z.

l

Lower-case 'l'

Permits an alphabetic character in this position, but does not require one.

A

Upper-case 'A'

Requires an alphanumeric character in this position.

For most western countries, this character must be A-Z, a-z, or 0-9.

a

Lower-case 'a'

Permits an alphanumeric character in this position, including characters such as '#', '@', '-', but does not require one.

C

Upper-case 'C'

Requires a character in this position (including characters such as '#', '@', '-').

c

Lower-case 'c'

Permits a character in this position, but does not require one.

0

Zero

Requires a numeric character in this position.

9

Nine

Permits a numeric character in this position, but does not require one.

#

Hash symbol

Permits a numeric character or a plus (+) or minus (-) sign in this position, but does not require one.

:

Colon

Separates hours, minutes, and seconds in times.

If the character that separates hours, minutes, and seconds is different in the International settings of the Control Panel utility, that character is used in place of the colon (:).

/

Forward slash

Separates months, days, and years in dates.

If the character that separates months, days, and years is different in the International settings of the Control Panel utility, that character is used in place of the forward slash (/).

;

Semicolon

Separates the fields in the mask, if there is more than one.

_

Underscore

Automatically inserts a blank character in the edit box.

When a user enters data in the field, the cursor skips the blank character.

 

The following table lists mask examples.

 

Mask Examples

Mask

Description

0000_0000_0000_0000

Forces the user to enter 16 digits and spaces them automatically for ease of viewing (for example, 2345_7777_8888_6543).

LL0000

Forces the user to enter two alphabetic characters (A-Z or a-z) followed by four digits.

The user must enter a digit in all of the positions where a '0' is displayed, even if that digit is zero.

LL9999

Forces the user to enter two alphabetic characters with the option of entering up to four digits after the characters.

The user cannot enter more than four digits.

>LL9999

Same as the previous example, except when the user enters an alphabetic character, it is automatically converted to upper case and the user cannot enter a lower-case character.

(00) 0000-0000;0

The mask has two fields:

  • The first field allows the user to enter a series of digits (all of which are mandatory) formatted like a phone number.
  • The second field (the zero after the semi-colon) means the number entered is stored in the database without the brackets or dash, for example, '0732188888'.

!9999;;*

The mask has three fields:

  • The first field allows the user to enter up to four digits (which are optional). The exclamation mark means that any digits not entered are represented as blanks at the beginning of the string. For example, if '44' was entered into the field, '**44' is displayed after the user leaves the field.
  • The second field is ignored, because there is nothing in the field.
  • The third field means that any digits not entered are represented by an asterisk, so before anything is entered into the field '****' is displayed in the field.

!99999.99

Forces the user to enter a number with two decimal places.

The number entered must contain five numbers before the decimal point and two after.

A disadvantage of using edit masks to control the entry and format of numbers is that the user cannot enter a number greater than the specified mask (in this example, 99,999). The use of display formats and edit formats may be more appropriate.

For an example of how to restrict the number of decimal places entered using display formats, edit formats, and validation expressions, refer to the Configuration Fundamentals Guide for Singleview Convergent Billing.

!#99999.99

According to the previous example, except the hash ('#') allows a negative sign ('-') to be entered. The negative sign is optional.

00/00/0000

Forces the user to enter a date in the format DD/MM/YYYY (for example, 27/05/1996).

The mask does not validate the date entered; therefore, if the attribute type has a data type of string (as opposed to date), the user could enter any digits in the field (for example, 99/99/9999).

\<9999\>

The user can enter up to four digits in the field.

The number is surrounded by the less than symbol ('<') and the greater than symbol ('>') as the forward slash symbol ('\') specifies that the following character is the literal character, not the specifier.
