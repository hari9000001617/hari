INV_CHARGE_SERVICE_V

The INV_CHARGE_SERVICE_V view displays all subtotal and tariff details for the specified service.

Only tariffs and subtotals with a context of Service are retrieved.

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

ID of the invoice with which these charges are associated.

SERVICE_ID

NUMBER (15)

Not null

Service with these charges.

CHARGE

NUMBER (22,6)

Value of this charge.

PROPORTION

NUMBER (15,6)

Proportion of this charge that was allocated to this customer.

It contains a value if TARIFF_ID is not null.

CURRENCY_ID

NUMBER (8)

ID of the currency in which this charge was made.

It is mandatory for all charges, but is not required for subtotals that are aggregations of non-currency amounts.

TARIFF_ID

NUMBER (8)

Tariff, discount, or tax that generated this charge, if any.

TARIFF_NAME

VARCHAR (30)

Name of the tariff, discount, or tax that generated this charge, if any.

TARIFF_TYPE_CODE

NUMBER (2)

Numeric code that defines the type of the tariff.

SUBTOTAL_ID

NUMBER (8)

Subtotal, or total calculation that generated this charge.

SUBTOTAL_NAME

VARCHAR (30)

Name of the subtotal that generated this charge, if any.

SUBTOTAL_DATE

DATE

Date-time calculated by the given subtotal, if this subtotal generates a date-time result.

It can only contain a value if SUBTOTAL_ID is not null and CHARGE is null.

It is checked by the DVP.

EFFECTIVE_START_DATE

DATE

Effective start date of the tariff or subtotal associated with the call.

EFFECTIVE_END_DATE

DATE

Effective end date of the tariff or subtotal associated with the call.

INVOICE_SECTION_NR

NUMBER (8)

Invoice section number associated with this charge.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number associated with this charge.

INVOICE_TXT

VARCHAR (255)

Optional invoice text that can be displayed in association with this charge.

KEY_VALUE

VARCHAR (255)

Hash key value or array index value associated with this charge for this tariff or subtotal.

It is only populated for charges generated by hash or array tariffs and subtotals.

For hash based tariffs or subtotals, it stores the string hash key associated with this charge.

For array tariffs or subtotals, it stores the integer array index associated with this charge.

GENERAL_1 - GENERAL_20

VARCHAR (255)

General fields 1 - 20.

They can be used to store intermediate calculations and lookups performed as part of tariff evaluation during rating or billing (for example, distance band, time band, and multi-time band flag).

The information contained in these columns is accessible by the BGP and the IGP.

Only charges associated with tariffs can populate them.

They are always null for charges associated with subtotals, payments, or adjustments.

 

Charge Partition Column

Column Name

Properties

Description

PARTITION_NR

NUMBER (2)

Not null

Customer partition number associated with a service level charge generated by a subtotal or tariff.

All charges associated with an invoice are associated with the same account (and therefore customer), so when customer partitioning is implemented, all charges for an invoice have the same PARTITION_NR.

It can be used as an additional selection criterion to improve access to the CHARGE table by eliminating irrelevant CHARGE table partitions.
