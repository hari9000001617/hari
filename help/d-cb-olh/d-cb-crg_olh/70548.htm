DERIVED_ATTRIBUTE_ARRAY_TRE_V

The DERIVED_ATTRIBUTE_ARRAY_TRE_V view defines the contents of global derived attribute tables.

Column Definitions

Column Name

Properties

Description

DERIVED_ATTR_ARRAY_EXT_
ID

NUMBER (10)

Read-only

ID of the derived attribute table row.

Automatically allocated for new table rows.

Note:

Only applicable for derived attributes configured to support extended features.

DERIVED_ATTRIBUTE_ID

NUMBER (8)

Not null

Read-only

ID of the derived attribute.

TENANT_ID

NUMBER (8)

Read-only

ID of the tenant.

It must contain a value when the derived attribute table is tenantable and the implementation is tenanted. Otherwise, it is not applicable.

For new records, automatically set to match the tenant of the user creating the record.

EFFECTIVE_START_DATE

DATE

Not null

Date-time the derived attribute table row becomes effective.

Its finest granularity is one second.

It always matches the effective start date of the derived attribute table to which the row belongs.

It is automatically set to the effective start date of the insert or update operation.

EFFECTIVE_END_DATE

DATE

Not null

Date-time one second before the derived attribute table row ceases to be effective.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

It always matches the effective end date of the derived attribute table to which the row belongs.

It is automatically set to the effective start date of the insert or update operation.

SEQNR

NUMBER (7)

Sequence number of the derived attribute table row within its effective date range.

It controls ordering for display purposes.

It must be specified for new records, but then becomes read-only.

Its contiguity is automatically maintained during insert and delete operations.

Note:

It is only applicable for derived attributes that are not configured to support extended features.

LAST_MODIFIED

DATE

Not null

Date-time when the derived attribute table row was last modified.

It is automatically set to the current date-time when the table row is inserted or updated.

DESCRIPTION

VARCHAR (80)

Optional description for the derived attribute table row.

OPERATOR_ID

NUMBER (8)

ID of the user who last modified this derived attribute table row.

It is automatically set to the ID of the user inserting or updating the table row.

Note:

It is only applicable for derived attributes configured to support extended features.

GROUP_ID

NUMBER (8)

ID of the security group that controls view and update access to the derived attribute table row.

It must contain a value if row-level security is enabled for the derived attribute table.

If not specified when inserting a new derived attribute table row, it defaults to the primary security group associated with the user.

Note:

It is only applicable for derived attributes configured to support extended features.

HASH_KEY

VARCHAR (80)

Hash of the index column values.

It is used to efficiently validate the uniqueness of derived attribute table row indexes based on the evaluation type of the derived attribute table.

It is generated as follows:

  1. A string is created by joining the values of the index columns, separated by colons.

    For example, if the values red, blue, NULL, and green are stored in the INDEX1_VALUE to INDEX4_VALUE fields, the string red:blue::green is created.

  2. A hash is created from the string using the SHA-1 algorithm resulting in a blob. The blob digest is then converted back into a string using Base64 encoding.

    Note:

    It is automatically derived and cannot be specified when inserting, updating, or deleting derived attribute table rows.

INDEX1_VALUE - INDEX10_VALUE

VARCHAR (255)

Values for performing lookups on the derived attribute table.

The number of available columns and their purpose is defined using the entity validation associated with the derived attribute table.

Note:

Index values do not include expressions.

RESULT1_VALUE - RESULT20_VALUE

VARCHAR (255)

Values for storing results for lookups on the derived attribute table.

The number of available columns and their purpose is defined using the entity validation associated with the derived attribute table.

Note:

Result values can include expressions.

VERSION_STR

VARCHAR (255)

Optional version identification string that can be used to tag a derived attribute table row.

It is used by configuration management tools.

Note:

It can only contain a value if the derived attribute table supports row-level version information using extended features.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this derived attribute table row.

A list of available categories is stored in CATEGORY_V view.

Note:

It can only contain a value if the derived attribute table supports row-level category information using extended features.
