Account History Variables (ACCOUNT_HISTORY_TRE_V)

The following table lists the entity validation variables associated with the ACCOUNT_HISTORY_TRE_V view.

Variable Definitions

Variable

View Field

Type

Description

evAcchAccountBalance#

ACCOUNT_BALANCE

Real

Account balance after the invoice is generated.

It is usually equal to INVOICE_AMOUNT + BALANCE_FORWARD, unless the invoice is a statement associated with an account other than the prime account, in which case it should be equal to STATEMENT_AMOUNT + BALANCE_FORWARD.

evAcchAccountId&

ACCOUNT_ID

Integer

ID of an account.

evAcchAccountInitialDue#

ACCOUNT_
INITIAL_DUE

Real

Initial amount due on the account.

It is usually equal to ACCOUNT_BALANCE, but can be different due to cents rounding or the presence of payment plans for the account.

evAcchAdjustmentAlloc
AdjustmentId&

ADJUSTMENT_
ALLOC_
ADJUSTMENT_ID

Integer

If ADJUSTMENT_INVOICE_CODE is 3, it is mandatory; otherwise, it must be null.

It is the ID of the single adjustment in the ADJUSTMENT_INVOICE table for the adjustment.

evAcchAdjustmentAmount#

ADJUSTMENT_
AMOUNT

Real

Amount of the adjustment in the currency of the account.

A positive adjustment indicates a credit to the customer that must be included on the invoice.

A negative amount indicates a debit.

ADJUSTMENT_AMOUNT includes any allowed adjustments associated with the currency conversion.

evAcchAdjustmentBatchId&

ADJUSTMENT_
BATCH_ID

Integer

ID of the adjustment batch.

evAcchAdjustmentCredit
Amount#

ADJUSTMENT_
CREDIT_AMOUNT

Real

Amount of the adjustment not allocated to any invoice.

evAcchAdjustmentDate~

ADJUSTMENT_DATE

Date

Date-time the adjustment was made.

evAcchAdjustment
Description$

ADJUSTMENT_
DESCRIPTION

String

Description of the adjustment.

It is not displayed on the invoice, but entered by the user.

ADJUSTMENT_DESCRIPTION may contain the justification for the adjustment.

If the adjustment is associated with a batch, it defaults to the description of the batch.

evAcchAdjustmentEnable
Aging&

ADJUSTMENT_
ENABLE_AGING_
IND

Integer

If set to 1 (TRUE), any unallocated amount associated with the adjustment is aged according to the aging rules of the ADJUSTMENT_TO_ACCOUNT_ID's account type.

It can only be set to 1 if ADJUSTMENT_FROM_ACCOUNT_ID is null; that is, aging cannot be enabled for transfer adjustments.

If not set to 1, it must be null.

It is usually only set for adjustments representing debit notes.

evAcchAdjustmentFrom
AccountId&

ADJUSTMENT_
FROM_ACCOUNT_ID

Integer

Account from which the adjustment is deducted.

If specified, it normally defaults based on the adjustment type.

evAcchAdjustmentId&

ADJUSTMENT_ID

Integer

Adjustment that has caused the CURRENT_BALANCE to be updated.

evAcchAdjustmentInvoice&

ADJUSTMENT_
INVOICE_CODE

Integer

Summary of the details in the ADJUSTMENT_INVOICE table for this adjustment. Valid values are:

  • 0 (Invoice): Adjustment is against a single invoice.
  • 1 (Split): Adjustment is against one or more invoices, adjustments, payments, direct credit, or direct debit.
  • 2 (Unallocated): Adjustment is a direct credit (amount is positive) or direct debit (amount is negative) to the account, or the TO_ACCOUNT_ID is not a prime account for a customer node.
  • 3 (Adjustment): The adjustment is against another adjustment, and is usually a debit note allocated to a credit note, or the reverse.
  • 4 (Payment): The adjustment is against a payment, and is usually a debit note allocated to a payment.

evAcchAdjustmentInvoice
Id&

ADJUSTMENT_
INVOICE_ID

Integer

If ADJUSTMENT_INVOICE_CODE is 0, it is mandatory; otherwise, it must be null.

It is the ID of the single invoice in the ADJUSTMENT_INVOICE table for the adjustment.

evAcchAdjustmentNr$

ADJUSTMENT_NR

String

ID of this adjustment, which is used as a reference.

It is automatically generated, based on the ADJUSTMENT_FORMAT_EXPR of the adjustment type and the ADJUSTMENT_ID.

evAcchAdjustmentPayment
Id&

ADJUSTMENT_
PAYMENT_ID

Integer

If ADJUSTMENT_INVOICE_CODE is 4, it is mandatory; otherwise, it must be null.

It is the ID of the single payment in the ADJUSTMENT_INVOICE table for the adjustment.

evAcchAdjustmentText$

ADJUSTMENT_
TEXT

String

Text displayed on the customer invoice in the summary section, in addition to the details of the invoice adjustment.

ADJUSTMENT_TEXT defaults to the text associated with the adjustment type.

evAcchAdjustmentTo
AccountId&

ADJUSTMENT_TO_
ACCOUNT_ID

Integer

Account to which the adjustment is to apply.

If the account is associated with a batch adjustment, the currency of the account must be the same as the currency associated with the batch adjustments.

evAcchAdjustmentTypeId&

ADJUSTMENT_
TYPE_ID

Integer

Standard category for the adjustment when entered.

If the adjustment is associated with a batch, the adjustment type defaults to the adjustment type of the batch.

evAcchAppliedDate~

APPLIED_DATE

Date

Date-time the transaction was applied or the payment rejected.

evAcchBalanceForward#

BALANCE_FORWARD

Real

Balance of the account before the application of the invoice.

evAcchBillRunId&

BILL_RUN_ID

Integer

ID of the bill run that generated the invoice.

evAcchCurrentBalance#

CURRENT_BALANCE

Real

Current balance of the account between the dates specified.

evAcchCurrentDue#

CURRENT_DUE

Real

Current amount left to be paid on the invoice.

The current amount is equal to INVOICE_AMOUNT minus any entries for the invoice in the PAYMENT_INVOICE and ADJUSTMENT_INVOICE tables.

evAcchCustomerAccount
InvStr$

CUSTOMER_
ACCOUNT_INVOICE_
STR

String

Optional ID generated for the invoice.

If defined, the ID is only unique within the account.

evAcchCustomerInvoice
Str$

CUSTOMER_
INVOICE_STR

String

ID generated for the invoice.

evAcchEarlyPaymentAcc
Balance#

EARLY_PAYMENT_
ACCOUNT_BALANCE

Real

Balance of the account at the EARLY_PAYMENT_DATE.

evAcchEarlyPaymentDate~

EARLY_PAYMENT_
DATE

Date

Date by which the customer must make a payment to receive an additional discount.

The discount is included in the INVOICE_TOTAL.

evAcchEarlyPayment
Discount#

EARLY_PAYMENT_
DISCOUNT

Real

Early payment discount applied for the invoice (if any).

evAcchEffectiveEndDate~

EFFECTIVE_END_
DATE

Date

One second before the date-time the next transaction occurred on the account.

evAcchEffectiveStart
Date~

EFFECTIVE_START_
DATE

Date

Date-time the transaction occurred.

Multiple transactions can have the same effective start date-time.

evAcchGeneral1$ -
evAcchGeneral10$

GENERAL_1 -
GENERAL_10

String

General fields 1 - 10 controlled by the payment type or adjustment type (depending on whether the record is a payment or adjustment).

Typical values are items such as bank, cheque number, account name, card number, card type, or expiry date.

evAcchGLUploadedInd&

GL_UPLOADED_IND_
CODE

Integer

Set to 1 (TRUE) if the transaction is uploaded to the General Ledger system; otherwise, it is null.

Note:

Uploaded transactions cannot be modified.

evAcchImageGenerated
Ind&

IMAGE_GENERATED_
IND_CODE

Integer

Set to 1 (TRUE) by the IGP when all invoice images for the invoice have been generated, resulting in one or more rows existing in the INVOICE_CONTENTS table for the invoice.

Otherwise, it is null.

evAcchInvoiceAmount#

INVOICE_AMOUNT

Real

Total amount of charges associated with the invoice, excluding the balance forward.

evAcchInvoicedAccount
Id&

INVOICED_
ACCOUNT_ID

Integer

Account in which all charges associated with the invoice are held.

INVOICED_ACCOUNT_ID can only contain a value if INVOICE_AMOUNT is 0 and STATEMENT_AMOUNT is not null.

Note:

If null and STATEMENT_AMOUNT is not null, the record is a statement for an 'Other Account'. In this case, the STATEMENT_AMOUNT is applied to the account rather than the INVOICE_AMOUNT.

evAcchInvoiceId&

INVOICE_ID

Integer

Invoice that has caused the CURRENT_BALANCE to be updated for the account.

evAcchLatePayment
Charge#

LATE_PAYMENT_
CHARGE

Real

Late payment charge applied to the account if payment is not made by the due date.

evAcchOriginalPayment
DueDate~

ORIGINAL_
PAYMENT_DUE_DATE

Date

Date the invoice was originally due to be paid.

evAcchPaymentAlloc
AdjustmentId&

PAYMENT_ALLOC_
ADJUSTMENT_ID

Integer

If PAYMENT_INVOICE_CODE is 3, this is the ID of the single adjustment in the PAYMENT_INVOICE table for the payment.

Otherwise, it is null.

evAcchPaymentAmount#

PAYMENT_AMOUNT

Real

Amount of the payment in the currency of the account.

evAcchPaymentBatchId&

PAYMENT_BATCH_ID

Integer

ID that defines a payment batch.

evAcchPaymentCredit
Amount#

PAYMENT_CREDIT_
AMOUNT

Real

Amount of the payment not allocated to any invoice.

PAYMENT_CREDIT_AMOUNT is always in the currency of the account.

evAcchPaymentDate~

PAYMENT_DATE

Date

Date-time the user entered the payment.

evAcchPaymentDue
AccountBalance#

PAYMENT_DUE_
ACCOUNT_BALANCE

Real

Balance of the account at the PAYMENT_DUE_DATE.

evAcchPaymentDueDate~

PAYMENT_DUE_DATE

Date

Date the invoice is due to be paid, and is usually the same as ORIGINAL_PAYMENT_DUE_DATE.

evAcchPaymentDue
Discount#

PAYMENT_DUE_
DISCOUNT

Real

'Pay-on-time' discount applied for the invoice (if any).

evAcchPaymentFrom
AccountId&

PAYMENT_FROM_
ACCOUNT_ID

Integer

Account from which the payment is taken.

If specified, it usually defaults based on the payment type.

evAcchPaymentId&

PAYMENT_ID

Integer

Payment that has caused the CURRENT_BALANCE to be updated for the account.

evAcchPaymentInvoice&

PAYMENT_INVOICE_
CODE

Integer

Summary of details in the PAYMENT_INVOICE table for the payment.

It is based on the PAYMENT_INVOICE reference type.

evAcchPaymentInvoice
Id&

PAYMENT_INVOICE_
ID

Integer

If PAYMENT_INVOICE_CODE is 0, this is the ID of the single invoice in the PAYMENT_INVOICE table for the payment; otherwise, it is null.

evAcchPaymentLocation&

PAYMENT_
LOCATION_CODE

Integer

Payment location category identifying how payments are received (for example, at a customer service centre or bank, or by post).

evAcchPaymentRejected&

PAYMENT_
REJECTED_CODE

Integer

If the payment is rejected, this column describes the reason for rejection (for example, a missing signature, inadequate funds, or exceeding the credit limit).

evAcchPaymentStatus&

PAYMENT_STATUS_
CODE

Integer

Status of the payment. Valid values are:

  • 1 (Accepted)
  • 2 (Rejected).

evAcchPaymentToAccount
Id&

PAYMENT_TO_
ACCOUNT_ID

Integer

Account to which the payment is applied.

evAcchPaymentTypeId&

PAYMENT_TYPE_ID

Integer

ID of the method of payment.

evAcchPreviousBalance#

PREVIOUS_BALANCE

Real

Previous balance on the account. If SEQNR is 1, this must be 0.

evAcchReceiptNr$

RECEIPT_NR

String

Official receipt number assigned to a payment.

RECEIPT_NR corresponds to accounting requirements outside the billing system (for example, government enforced).

evAcchRejectedDate~

REJECTED_DATE

Date

If the payment is rejected, the date is recorded in REJECTED_DATE.

evAcchSeqNr&

SEQNR

Integer

Number that identifies the chronological sequence in which transactions are applied to the account.

evAcchStatementAmount#

STATEMENT_AMOUNT

Real

Total amount of charges associated with the statement.

STATEMENT_AMOUNT can only contain a value if INVOICE_AMOUNT is 0.

evAcchTotal
Adjustments#

TOTAL_
ADJUSTMENTS

Real

Total adjustments made on the account between the issue of the previous invoice and this invoice.

evAcchTotalPayments#

TOTAL_PAYMENTS

Real

Total payments made on the account between the issue of the previous invoice and this invoice.
