Variables Based on INV_DUNNING_V

All direct variables based on the INV_DUNNING_V view have an application environment of Invoicing and a context of Any.

Variable Definitions

Variable

Type

Description

InvDunningAccountBalance#

Real

Balance of the account.

InvDunningAccountInitialDue#

Real

Initial amount due for payment on the account.

InvDunningAccountName$

String (30)

Account name.

InvDunningAcctId&

Integer

ID of the account to which this invoice was applied.

InvDunningAction&

Integer

Code for the dunning action taken on the invoice.

InvDunningBalanceForward#

Real

Balance on the account before the application of the invoice.

InvDunningCreditLimit#

Real

Credit limit for the customer node.

InvDunningCurrent&

Integer

Code for the current dunning action.

InvDunningCurrentDue#

Real

Current amount due on the invoice.

InvDunningCustAccInvoiceStr$

String (30)

Optional invoice ID generated for the invoice.

InvDunningCustNodeId

Integer

ID of the customer node associated with the instance.

InvDunningCustomerInvoiceStr$

String (30)

ID generated for the invoice.

InvDunningCustomerName$

String (80)

Name associated with the node.

InvDunningCustomerTypeName$

String (30)

Name of the customer type for the node.

InvDunningCustomerTypeDisplay
Name$

String (60)

Name of the customer type displayed.

It defaults to InvDunningCustomerTypeName$.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

InvDunningEarlyPayDiscount#

Real

Early payment discount for the invoice.

InvDunningEarlyPaymentBalance#

Real

Balance of the account at the early payment date.

InvDunningEarlyPaymentDate~

Date

Date at which early payment discounts are granted.

InvDunningEffectiveDate~

Date

Effective date for the information.

InvDunningFamilyName$

String

Family name of the customer node.

InvDunningImageGenerated&

Integer

Set to 1 (TRUE) when an invoice image has been generated for the invoice.

Otherwise, the value of the variable is undefined.

InvDunningInvoiceAmount#

Real

Total amount of charges associated with the invoice, excluding the balance forward.

InvDunningInvoiceId&

Integer

ID that defines a specific customer node invoice.

InvDunningIssueDate~

Date

Date that the invoice is issued.

InvDunningPaymentDueBalance#

Real

Balance of the account at the payment due date.

InvDunningPaymentDueDate~

Date

Date that payment of the invoice is due.

InvDunningPaymentDueDiscount#

Real

Pay-on-time discount that is applied to the invoice.

InvDunningPersonOfficialName$

String

Official name of the customer node.

InvDunningPostalCity$

String (80)

City of the node's correspondence address.

InvDunningPostalCountry$

String (80)

Country of the node's correspondence address.

InvDunningPostalGeneral1$ -
InvDunningPostalGeneral10$

String (255)

General fields 1 - 10.

InvDunningPostalGeocode$

String (80)

Optional encoding of the geographic location of the address.

InvDunningPostalLine1$

String (80)

Line 1 of the node's correspondence address.

InvDunningPostalLine2$

String (80)

Line 2 of the node's correspondence address.

InvDunningPostalState$

String (80)

State of the node's correspondence address.

InvDunningPostalSuburb$

String (80)

Suburb of the node's correspondence address.

InvDunningPostalXPos#

Real

Numeric representation of the location of the address.

InvDunningPostalYPos#

Real

Numeric representation of the location of the address.

InvDunningPostalZPos#

Real

Numeric representation of the location of the address.

InvDunningPostCode$

String (10)

Postal code of the node's correspondence address.

InvDunningSeqnr&

Integer

ID of a specific row, given a TEMP_DATA_ID.

InvDunningTaskQueueId&

Integer

ID of the task that generated the information.

InvDunningTitle$

String

Title of the primary contact person for this customer node.

InvDunningTotalAdjustments#

Real

Total adjustments made against the account between the issuing of the previous invoice and this invoice.

InvDunningTotalPayments#

Real

Total payments made on the account between the issuing of the previous invoice and this invoice.
