biAdjustmentImageRetrieve$()

Syntax

biAdjustmentImageRetrieve$(
    AdjustmentId&,
    ImageType$,
    Regenerate&,
    var CachedCopy&

 

Description

Returns the file name of an adjustment image.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

AdjustmentId&

In: ID of the adjustment for which the image is required.

ImageType$

In: Image format to generate.

If the parameter is not specified, the image type is determined by the DisplayImageType derived attribute table. For more information, refer to Remarks.

Regenerate&

In: Value that specifies whether an image is regenerated.

If TRUE, the image is regenerated; otherwise, the cached image is returned.

CachedCopy&

Out: Value that specifies whether the file name of a cached image was returned.

If TRUE, the cached image file name is returned; otherwise, FALSE if the image was regenerated.

 

Returns

Fully qualified file name of the image; otherwise, an error is raised.

Remarks

The function uses the following attributes from the selected ADJUSTMENT_IMAGE configuration item to determine the file name of an adjustment image:

  • CACHE_DIRECTORY
  • CALLER_PREFIX
  • INSTANCE.

In a multi-instance environment, the function uses the INSTANCE attribute to determine the correct configuration item to use.

The file name of the generated adjustment image uses the following format:

<caller_prefix><AdjustmentId&>.<ImageType$>

where:

<caller_prefix> is the file location and access mechanism of the adjustment image.

If ImageType$ is not specified, the DisplayImageType derived attribute table is used to determine the correct image to generate. The table maps each OUTPUT_IMAGE_TYPE reference type to the corresponding DISPLAY_IMAGE_TYPE reference type. If the derived attribute table does not contain a valid mapping for the required OUTPUT_IMAGE_TYPE, an error is raised.

The function calls the ivp server utility (-a option)to generate the image. A lookup on the DisplayImageMethod derived attribute table is used to determine the script name parameter by using:

  • An Entity Image Type of Adjustment
  • An Output Image Type retrieved from the ADJUSTMENT table
  • Required Display Image Type.

If a row is returned, script name is passed to the ivp using the -script option to generate the image. If no row is returned, the ivp uses view_inv_<display_image_type>.
