CUSTOMER_CV

The CUSTOMER_CV view defines the unique customer node details.

Column Definitions

Column Name

Properties

Description

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node associated with this instance.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

CUSTOMER_NODE_TYPE

VARCHAR (30)

Type of this customer node.

NODE_NAME

VARCHAR (200)

Name associated with this node.

ATLANTA_GROUP_NAME

VARCHAR (60)

Security group with access to this customer node.

OPERATOR_LOGIN_NAME

VARCHAR (30)

Oracle user name of the user who created or last modified this record.

GL_CODE_NAME

VARCHAR (64)

GL code associated with all charges for this customer node.

PARENT_CUSTOMER_NODE_ID

NUMBER (15)

Parent customer node for this node.

HIERARCHY_LEVEL

NUMBER (3)

Level in the hierarchy of this node.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

If this instance of a node has a parent_customer_node_id, it is the ID of the customer node at the root of the hierarchy.

CUSTOMER_NODE_STATUS

VARCHAR (60)

Status of the customer node (for example, Prospect, Active, and Cancelled).

ACTIVE_DATE

DATE

Date-time that the customer first became an active subscriber to the network.

Its finest granularity is one second.

PERSON_ID

NUMBER (15)

Primary contact person for this customer node.

PRIME_ACCOUNT_ID

NUMBER (15)

Primary account associated with this customer node.

REPORT_LEVEL

VARCHAR (60)

Level of financial reporting that is required for this customer node (for example, Invoice, Statement, and No Reporting).

POSTAL_ADDRESS_ID

NUMBER (15)

Reference to the address details for all postal correspondence regarding this customer node, including invoices/statements.

SITE_ADDRESS_ID

NUMBER (15)

Reference to the physical location of this customer, or division of the customer.

INVOICE_SCHEDULE_NAME

VARCHAR (30)

Standard invoice cycle that the customer node had been assigned to.

TURNOVER

NUMBER (19,3)

Revenue for the last financial year for this customer node.

TURNOVER_CURRENCY_ID

NUMBER (8)

Currency for the above turnover value.

CREDIT_LIMIT

NUMBER (19,3)

Amount of credit that a customer node has at any one time.

CREDIT_LIMIT_CURRENCY_ID

NUMBER (8)

Currency for the above credit limit value.

CREDIT_RATING

VARCHAR(60)

Indicator for what rating a customer has regarding invoicing and payment collection.

TAX_CLASS

VARCHAR (60)

Tax class to which this customer node is associated.

PAYMENT_METHOD

VARCHAR (60)

Official method of payment for the customer node (for example, cash and cheque).

PAYMENT_LOCATION

VARCHAR (60)

Official location where payments will be made.

BANK

VARCHAR (60)

Customer's bank or financial institution used for invoice payments.

BRANCH

VARCHAR (60)

Branch of the financial institution used by the customer node.

BANK_ACCOUNT_NUMBER

VARCHAR (60)

Customer's account number used for automatic payments.

BANK_ACCOUNT_REF

VARCHAR (15)

Account reference number quoted by customer with their payment at banking institutions and other third party payment collection points.

CARD_TYPE

VARCHAR (60)

Type of credit card used for payments, and as default information for payment.

CARD_NUMBER

VARCHAR (60)

Main or default credit card number used for making payments.

CARD_EXPIRY_DATE

DATE

Date that the main or default credit card expires.

ASSIGNED_OPERATOR_NAME

VARCHAR (30)

User assigned to liaise with this customer node in all matters.

COMPANY_NUMBER

VARCHAR (15)

Official company or business registration number.

INDUSTRY_TYPE

VARCHAR (60)

Type of industry or activity the customer node is involved in.

REGION

VARCHAR (60)

Geographic region that the customer node is located in.
