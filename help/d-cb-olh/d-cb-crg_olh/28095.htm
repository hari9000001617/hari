Description

Allocates an item of equipment to a product instance. The function is similar to and wraps the biEquipmentAllocate&() function.

Note:

The function always uses the value 1 for the service and equipment range sequence numbers. If any other values are required, use the fEV_EquipmentAllocateRange&() function.

Context

The following groups of variables are available throughout all stages of processing an equipment allocate request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value EQUI_ALLSVR.
    • evLastRowInd& contains the value 0 during all stages of processing except for the evaluation of the product instance equipment and product instance service pre-commit expressions.
    • evRowStatus$ contains the value Update.
  • Product instance equipment.

Parameters

Parameter

Description

prodInstId&

ID of the product instance to which equipment is being allocated.

A product instance with the specified ID must exist in the database for the specified effectiveDate~; otherwise, an error is raised.

effectiveDate~

Date on which the allocation of the specified equipment item to the specified product instance first becomes effective.

The specified:

  • Product instance must exist on the date specified.
  • Equipment item must exist and be idle on the specified date.

lastModified~

Initial value must match the value in the existing product instance record matching the specified prodInstId& and effectiveDate~.

On successful return, the value is the date-time of this allocation.

equipmentId&

ID of the equipment item being allocated.

An equipment item with the specified ID must exist in the database for the specified effectiveDate~; otherwise, an error is raised.

equipLastModified~

Value must match the value in the existing equipment item record for the specified equipmentId& at effectiveDate~.

creationOrder&

Equipment item being allocated.

It corresponds to an entry in the product definition.

callParams{}?

User-defined data for entity validation, assigned to the evsCallParams?{} global variable on entry to the function for resolution of entity validation expressions.

 

Processing

Assuming that all processing stages are enabled, the following list is the order of evaluation of expressions and constraints before allocation of the equipment item:

  1. Bundle direct variables if the call is nested.
  2. Decrypt any encrypted values and mask any appropriate values.
  3. Evaluate product instance equipment row expressions.
  4. Evaluate product instance service pre-commit expressions for all services affected by the allocation (completed product instances only).
  5. Evaluate product instance equipment pre-commit expressions.
  6. Unmask any masked values and encrypt any variable values configured for encryption.

On performing the insertion, the following sequence of actions occurs:

  1. Evaluate product instance service in-transaction expressions for all services affected by the allocation (completed product instances only).
  2. Queue and evaluate product instance service post-commit expressions for all services affected by the allocation (completed product instances only).
  3. Evaluate product instance equipment in-transaction expressions.
  4. Queue and evaluate product instance equipment post-commit expressions.

After allocation and evaluation of post-commit expressions:

  1. Unbundle variables that were stored in step 1 before allocation.

Returns

On success, a non-zero value. If the allocation or any of the pre-allocate validation steps fail, an error is raised.
