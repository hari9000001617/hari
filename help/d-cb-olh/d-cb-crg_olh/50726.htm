INVOICE_FORMAT_TRE_V

The INVOICE_FORMAT_TRE_V view contains information of all available invoice formats.

Column Definitions

Column Name

Properties

Description

INVOICE_FORMAT_ID

NUMBER (8)

Not null

ID of invoice format.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

INVOICE_FORMAT_NAME

VARCHAR (30)

Not null

Name of this invoice format.

INVOICE_FORMAT_DISPLAY_
NAME

VARCHAR (60)

Not null

Name of invoice format displayed.

It defaults to INVOICE_FORMAT_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

DESCRIPTION

VARCHAR (255)

Description of invoice format.

LAST_MODIFIED

DATE

Not null

Date-time the record was last modified.

OPERATOR_ID

NUMBER (8)

Not null

ID of user who created or last modified the record.

GROUP_ID

NUMBER (8)

Not null

Security group associated with, and controlling access to, this record.

VERSION_STR

VARCHAR (255)

Optional version ID string of this INVOICE_FORMAT.

It is provided for use by third party configuration management tools and is not used by the core application.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this invoice format.

Refer to the CATEGORY_V view for the list of available categories.

INVOICE_FORMAT_STATUS_
CODE

NUMBER (2)

Not null

Status of this instance of an invoice format. Valid values are:

  • 0 (Cancelled)
  • 1 (Active).

It is defined by reference type INVOICE_FORMAT_STATUS.

VIEWABLE_IND_CODE

NUMBER (1)

Set to 1 (True), if images generated with this format are viewable.

STORED_IMAGE_TYPE_CODE

NUMBER (2)

Type of image stored in the INVOICE_CONTENTS table for an invoice in this format.

For bulk output, the OUTPUT_IMAGE_TYPE_CODE must match the STORED_IMAGE_TYPE_CODE.

TEMPLATE_ID

NUMBER (8)

ID of the TeX template used for this invoice format.

The template references other templates within its definition.

The referenced template must have a template type of Invoice/Statement_Image (0) or Adjustment_Image (10).

ENTITY_VALIDATION_ID

NUMBER (8)

Validation rules, labels and hint text when defining invoice formats for a customer node.

If not null, it must reference a row with VALIDATION_TABLE_NAME of CUSTOMER_NODE_INV_FORMAT.

It is only applicable if the specified template is type Invoice/Statement_Image.
