Transaction Engine API

This section describes the Application Programming Interface (API) to the Transaction Engine (TRE). The TRE is a configured installation of the Tuxedo transaction processing middleware (see the Tuxedo Guide for Singleview) and performs the following functions:

  • Acts as the middle tier of a three tier system in a client/server environment.
  • Provides a mechanism for the system to communicate with external applications.
  • Enables external applications to access the database.

Function role security is enforced for all TRE API calls originating from a Tuxedo client. The TRE message contains an indicator identifying the request from a client or server. Requests originating from Tuxedo servers include a digital signature that is verified by the Tuxedo server receiving the request.

The TRE API is available for the following programming languages:

  • C
  • Perl
  • Object Pascal
  • Java.

Information about the API is provided for C, Perl, and Object Pascal programming languages.

It is assumed that the reader has a thorough knowledge of these languages.

The TRE C API interfaces to the Tuxedo C API to perform remote function calls. The Perl and Object Pascal APIs are wrappers for the C API.

The TRE is used by client and server processes and can also be used by external applications using the API, for the following purposes:

  • Calling TRE functions.
  • Grouping function calls into transactions, with the option to commit or roll back the transaction.
  • Queuing transactions.
  • Subscribing to TRE events, defining an event handler, and checking for events.
  • Posting TRE events.
