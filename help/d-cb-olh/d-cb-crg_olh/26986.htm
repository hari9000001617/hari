fCMN_TREProductInstanceCreate_R&()

Syntax

fCMN_TREProductInstanceCreate_R&(
    l_ProdInst?{})

 

Description

Creates a product instance and related product instance entities required when selling a product.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_ProdInst?{}

In: Hash containing the values for all mandatory and optional fields for the product instance and related entities.

For more information, refer to Remarks below.

 

Returns

New product instance ID, if successful; otherwise, an error is raised.

Remarks

The high-level processing is:

  1. Retrieve the product definition to be generated.
  2. Retrieve the product order rules from Convergent Billing configuration.
  3. Insert the product instance.
  4. If the base product instance has a product offer, instantiate any optional companion products.
  5. Use the creation and order rules to insert mandatory and optional services and equipment.
  6. Complete the product instance.
  7. Insert and populate other optional entities in the following order:
    1. Non-standard facility groups
    2. Product-based service lists
    3. Companion products.

Various

Details for the product instance using fields from PRODUCT_INSTANCE_TRE_V with the field name as the hash key (represented by <PRODUCT_INSTANCE_TRE_V>); for example:

l_ProdInst&{'PRODUCT_ID'} := 9999;

{'NODE_NAME'}

String

Name of the customer node to create the product instance for.

{'CUSTOMER_NODE_ID'}

Integer

Customer node ID of the customer for which to create the product instance.

CUSTOMER_NODE_ID overrides NODE_NAME.

{'ACCOUNT_ID'}

Integer

Can be used with NODE_NAME to reduce the likelihood of ambiguity of multiple identical customer node names existing.

If ACCOUNT_ID is specified, ACCOUNT_NAME cannot be specified.

{'ACCOUNT_NAME'}

String

Can be used with NODE_NAME to reduce the likelihood of ambiguity of multiple identical customer node names existing.

If ACCOUNT_NAME is specified, ACCOUNT_ID cannot be specified.

[<SERVICE_RANGE_SEQNR>]
{'<SERVICE_TRE_V>'}

Various

Service details for the product instance using fields from SERVICE_TRE_V with the field name as the <SERVICE_TRE_V> hash key.

The data type of the hash entries correspond to the service fields.

[<SERVICE_ITEM_SEQNR>] is an array (1..n) of service items.

[<SERVICE_RANGE_SEQNR>] is an array (1..n) of services ranges.

[<SERVICE_ITEM_SEQNR>]
[<SERVICE_RANGE_SEQNR>]
{'EQUIPMENT'}
[<EQUIP_ITEM_SEQNR>]
[<EQUIP_RANGE_SEQNR>]

Unknown

[<EQUIP_ITEM_SEQNR>] is an array (1..n) of equipment items.

[<EQUIP_RANGE_SEQNR>] is an array (1..n) of equipment ranges.

Only required for optional equipment (mandatory equipment is automatically allocated).

The data type of the element is either integer or hash. Any other data type causes an error to be raised.

If the data type is integer, a value of 1 specifies automatic allocation of the first available equipment item of the appropriate type.

If the data type is hash, identifies specific equipment for allocation whether equipment is optional or mandatory. Equipment is identified by SERIAL_NUMBER or NETWORK_ADDRESS; the elements of the hash use these strings as keys and the associated values identify the item of equipment.

[<SERVICE_RANGE_SEQNR>]
{'LISTS'}
{'<TABLE_NAME>'}
[<ROW_NUMBER>]
{'<COLUMN_NAME>'}

Various

Lists for the service.

<TABLE_NAME> refers to the derived attribute table name.

[<ROW_NUMBER>] is an array (0..n) of list rows for the table.

Values for the columns for each row of a list are specified in the {<COLUMN_NAME>} hash. The key for an entry can either be the display label of a column's attribute type (if defined), or a column name (for example, INDEX1_VALUE) from the database definition for the derived attribute table content (DERIVED_ATTRIBUTE_ARRAY).

The EVS section has more information about specifying keys and values.

[<ROW_NUMBER>] is an array (0..n) of facility groups.

Companion products (which must be defined as compatible products in the product definition) for the product instance.

[<ROW_NUMBER>] is an array (0..n) of companion products.

Refer to PRODUCT_INSTANCE_TRE_V view for details of mandatory and optional fields. The data type of the hash entries correspond to the attribute types in the entity validation for a product.

{'CONTRACT'}
{'<CONTRACT_TRE_V>'}

Various

Contracts for the product instance using fields from CONTRACT_TRE_V view with field names as keys for {<CONTRACT_TRE_V>}.

The data type of the hash entries correspond to the attribute types in the entity validation for a contract.

If either CONTRACT_ID or CONTRACT_NAME are specified and the contract exists in Convergent Billing, the contract ID is used for the service being created.

If the contract does not exist in Convergent Billing, the contract is created and the service created is assigned to the contract. CONTRACT_ID overrides CONTRACT_NAME.

{'FROM_ACCOUNT_IDS'}[]

Optional array of account IDs.

It is required if the product definition requires creation of charge category information with credit (transfer) account information.

Unknown

Optional hash specifying user-defined data for entity validation.
