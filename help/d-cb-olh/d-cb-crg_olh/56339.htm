Description


Extracts data from or loads data into Singleview using custom-defined formatting.


Note:
The custom-format functions used to extract or load data must be user-defined and specified in the interface definition.


Attributes






Application Environment



Any







Context



Any







Method



Local








 






Parameter



Description







ExternalDefnId&



In:  ID of the external definition.







InterfaceDefnId&



In:  ID of the interface definition defined using the BatchOutbound interface type for a batch extraction, or the BatchInbound interface type for a batch load.







Operation$



In:  Name of the operation to perform.  Valid values are:


  • Insert
  • Update
  • Delete (for a batch load)
  • Fetch (for a batch extraction).






Request?{}



In:  Data to be converted.


If Header& is set to FALSE, contains a hash element structure in a format compatible with the external definition specified by ExternalDefnId&; otherwise, it is empty.


Note:
It is applicable only for batch extractions.







Line$



In:  For batch loads, line from the input file to be processed.


Out:  For batch extractions, line to be written to the output file. 







Headers$[]



In:  Array of strings that represent the headers:


  • From the external definition 
  • Decoded from the first line of the input line.






Audit&



In:  If set to TRUE, auditing is enabled.







Header&



In: Determines the content of Line$.  If set to:


  • TRUE, Line$ contains:
    • Column headers instead of the data, for a batch extraction  
    • Header line of the file to be loaded, for a batch load.
  • FALSE, Line$ contains:
    • Custom-formatted data, for a batch extraction
    • Data line, for a batch load.






ControlInfo?{}



In: Hash containing Control Information.


This ControlInfo?{} has the following keys:


  • FILENAME, name of the input file being  processed.
  • RECORD_NR, current record number in the input file.







 


Returns


1 (TRUE) if successful; otherwise, an error is raised.


For batch loads, returns 0 (FALSE) if the user-defined load function indicates that the line should be ignored or skipped. 


Remarks


For batch extractions, the data is converted into the custom format by a user-defined function that matches the syntax of the <CustomExtractFormatFuncName>&() function.  


For batch loads, the data is read from a custom formatted file by a user-defined function that matches the syntax of the <CustomLoadFormatFuncName>&() function.
