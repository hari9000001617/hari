CUSTOMER_QUERY_SEARCH_V

The CUSTOMER_QUERY_SEARCH_V view is used to perform searches relating to a customer query.

Column Definitions

Column Name

Properties

Description

CUSTOMER_QUERY_ID

NUMBER (15)

Not null

ID of the customer query.

CUSTOMER_QUERY_NR

VARCHAR (30)

Not null

ID of the query, used to identify the query.

CUSTOMER_QUERY_TYPE_ID

NUMBER (8)

Not null

Number for defining the type of query registered.

CUSTOMER_QUERY_TYPE_NAME

VARCHAR (40)

Not null

Name of the customer query type.

CUSTOMER_NODE_ID

NUMBER (15)

ID of a customer hierarchy element or node to which this query relates.

When the query does not relate to an established customer, this field can be left as null.

PERSON_ID

NUMBER (15)

Not null

ID of a person in the billing system.

If the query does not relate to an established contact person, a new value for this field is created by establishing a new person in the contact information database.

SERVICE_ID

NUMBER (15)

ID of the service that the query relates to, if known or logical to make such an association.

ACCOUNT_ID

NUMBER (15)

ID of an account that the query relates to, if the query does relate to an account.

INVOICE_ID

NUMBER (15)

ID of the invoice that the query relates to, if the query relates to an invoice.

It can only contain a value if an account is specified, enforced by the C_CUSTOMER_QUERY check constraint.

AMOUNT

NUMBER (19,3)

Amount (usually in the currency of the account) being queried.

It can only contain a value if an account is specified, enforced by the C_CUSTOMER_QUERY check constraint.

CURRENCY_ID

NUMBER (8)

If an amount is being queried, this is the currency of the amount. CURRENCY_ID defaults to the currency of the account.

It can only contain a value if an account is specified, enforced by the C_CUSTOMER_QUERY check constraint.

ATLANTA_GROUP_ID

NUMBER (8)

Not null

Security group that has access to this query problem.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who recorded this customer query or complaint.

QUERY_STATUS_ABBREV

VARCHAR (60)

Not null

Holds the value of the query progression status. Valid values are:

  • 1 (Opened)
  • 2 (Closed).

QUERY_STATUS_CODE

NUMBER (8)

Not null

Holds the value of the query progression status. Valid values are:

  • 1 (Opened)
  • 2 (Closed).

QUERY_WORK_FORCE_NAME

VARCHAR (30)

If the query is not closed, this is the work group responsible for investigating this query.

It is obtained from the CUSTOMER_QUERY_PROGRESS table.

If the query has a status of Closed, this column is null.

QUERY_WORK_FORCE_
DISPLAY_NAME

VARCHAR (60)

Name of the work group responsible for investigating this query displayed to users.

It defaults to QUERY_WORK_FORCE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

QUERY_WORK_FORCE_ID

NUMBER (8)

Redundant

If the query is not closed, this is the work group responsible for investigating this query.

It is obtained from the CUSTOMER_QUERY_PROGRESS table.

If the query has a status of Closed, it is null.

PROGRESS_SEQNR

NUMBER (8)

Redundant

Highest sequence number in the CUSTOMER_QUERY_PROGRESS table for this query.

If the query is not progressed because it was resolved by the user, it is null.

OPEN_DATE

DATE

Not null

Date that a query is added to the system in response to written or verbal information received from a customer or other party.

RESPONSE_DUE_DATE

DATE

Not null

Date-time by which an initial response to this query is expected.

It is based on the nature of the query (the query type), and the contract details associated with the service on which this query is based.

RESPONSE_GIVEN_DATE

DATE

Date-time when an initial response is given for this query.

NEXT_CALLBACK_DATE

DATE

Not null

Date-time by which a user should call the customer back to report on the status of the query.

For a new query, it defaults to RESPONSE_DUE_DATE.

RESOLUTION_DUE_DATE

DATE

Not null

Date-time when this query is due to be resolved.

It is based on the nature of the query (the query type), and the contract details associated with the service on which this query is based.

CLOSE_DATE

DATE

Date-time when the query was completely resolved and requires no further action.

QUERY_TEXT

VARCHAR (255)

Text that describes the initial query or problem that a customer or other party reported.

RESOLUTION_TEXT

VARCHAR (255)

Description of the resolution of the problem.

If the query is not able to be resolved immediately, this column is initially null.

On completion of the query, this text is set to the final resolution.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10.

The labels, hint text, and validation associated with each of these columns is determined from their attribute types defined in the CUSTOMER_QUERY_TYPE table.
