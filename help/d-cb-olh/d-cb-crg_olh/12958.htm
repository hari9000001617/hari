Description

Allocates contributory adjustments to invoices, other adjustments, and receivable types.

Note:

Interface 1 is deprecated, and is used only for backward compatibility.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

AccountId&

In: ID of the account being allocated a payment.

TotalAmount#

In: Total amount of the payment.

PaymentTypeId&

In: ID of the payment type.

EffectiveDate~

In: Date-time that the payment is applied to the account.

InvoiceId&[]

In: (Deprecated) Performs allocations for the specified invoice only.

Out: Returns the set of invoices for which allocations have been made.

If a single invoice ID is supplied, the allocation is only performed for that invoice.

InvoiceLastModified~[]

Out: (Deprecated) Last modified date-time of the updated invoices.

ReceivableTypeId&[]

Out: (Deprecated) ID of the receivable type.

Amount#[]

Out: (Deprecated) Amount of each payment.

Allocations?[]

In (optional): Single array, containing a single key of INVOICE_ID or ADJUSTMENT_ID for the allocation.

If specified, allocations are performed for the specified ID.

Out: Array of hashes containing results of allocations.

The table below lists the entries present in the hash.

 

The following table lists Allocations?[] hash keys.

 

Allocations?[] Hash Keys

Keys

Description

INVOICE_ID

Invoice against which an allocation will be/was applied.

ADJUSTMENT_ID

Adjustment against which an allocation will be/was applied.

LAST_MODIFIED

Receivable type of the specified allocation.

This key is only present if INVOICE_ID or ADJUSTMENT_ID is specified.

RECEIVABLE_TYPE_ID

Receivable type (if applicable) of the specified allocation.

AMOUNT

Amount of the allocation in the currency of the account.

 

Note:

A maximum of one INVOICE_ID or ADJUSTMENT_ID key can be present in each hash. If no ID key is present, AMOUNT is regarded as unallocated and debited or credited to the account specified in AccountId&.

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

Allocations are made to the oldest invoice or adjustment first, with allocations to receivable types being distributed within each invoice or adjustment based on its contribution to the amount due.
