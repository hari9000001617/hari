Client DLLs

The TRE API is used by Singleview Windows clients. It is included in 32-bit dynamic linked libraries (DLLs). The same source code is used for both the server implementation of the TRE API and the DLLs.

The following two versions of the TRE DLL are available:

  • tre32.dll
  • tre32_64.dll.

Both are 32-bit DLLs, and the clients are 32-bit Windows applications. tre32.dll supports 32-bit integers and tre32_64.dll supports 64-bit integers. 64-bit integer support allows the client to process Convergent Billing entity IDs that exceed the 32-bit integer size limit.

The _WINDOWS compiler macro is used to identify Windows-specific code. INT64BIT distinguishes code for 64-bit integer support from code for 32-bit integers. INTSV is the variant data type that resolves to long for server and 32-bit integer client builds, and _int64 for 64-bit integer client builds.

The DLL builds are performed on the server, using a command-line Visual C++ compiler executed under a Windows emulator.
