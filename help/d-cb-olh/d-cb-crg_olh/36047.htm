RTM_MetricName

The RTM_MetricName derived attribute table defines a mapping between the MetricName value in the RTM_Alarm derived attribute table and the Perl expression used to access the value in the RTM real-time data hash.

The table has an evaluation type of Table Lookup By Multiple (1-10) Keys.

 

Column Definitions

Column Name

Properties

Description

MetricName

String (64)

Not null

Name of the metric value.

The name must start with a letter, and can be followed by letters, digits, or underscores (Perl and EPM naming convention).

PerlTerm

String (255)

Not null

Part of the Perl expression used to access the metric value in the real-time data hash.

For example, the following implies that the metric value is accessed using the $l_rt_data->{Unix}->{CPUUsage}->{CPUIdle} Perl expression:

{Unix}->{CPUUsage}->{CPUIdle}

For Commerce Engine requests, the general access path is:

{CE}->{Request}->{<Request_Name>}->{<MetricName>}

For DCEA, request names are defined in the function specified by the FN_REQUEST_NAME configuration item attribute of the DCEA configuration item.

 

The following table lists example mappings between the MetricName value in the RTM_Alarm derived attribute table and the Perl expression used.

 

Example Mappings

MetricName

Perl Expression

DebitAverageLatency

{CE}->{Request}->{CE_DEBIT_UNIT}->{AverageLatency}

AccountUsedPercentage

{Cache}->{Account}->{UsedPercentage}

QueryBalanceResponseSentRate

{CE}->{CE_QUERY_BALANCE}->{ResponseSentRate}

 

Default Metrics

For information about default metrics delivered with Convergent Billing, refer to the System Configuration Guide for Singleview Convergent Billing.

Note:

  • The default metrics delivered with Convergent Billing apply only to the General Purpose Commerce Engine Adapter (GPCEA) and eServ Commerce Engine Adapter (ECEA). Metrics are not provided for the Diameter Commerce Engine Adapter (DCEA).
  • GPCEA has reservation and debit. DCEA has initial request, update request, and termination request. This means that DCEA requires one more set of metrics for the initial request and updates to the request names.
