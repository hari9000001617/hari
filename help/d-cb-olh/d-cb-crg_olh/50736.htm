INV_DUNNING_V

The INV_DUNNING_V view defines details for generating dunning or credit check letters and invoice reprints for customers.

Column Definitions

Column Name

Properties

Description

TEMP_DATA_ID

NUMBER (15)

Not null

ID of the task that generated this information.

TASK_QUEUE_ID

NUMBER (15)

Not null

ID of the task that generated this information.

It is identical to TEMP_DATA_ID, which is required by the IGP, and the TASK_QUEUE_ID by the General Output Process (GOP) for invoice output.

SEQNR

NUMBER (6)

Not null

ID for a specific row, given a TEMP_DATA_ID.

EFFECTIVE_DATE

DATE

Not null

Date-time for which this dunning information was generated.

INVOICE_ID

NUMBER (15)

ID that defines a customer node invoice.

INVOICE_SEQNR

NUMBER (3)

Sequence number of an invoice image for this invoice.

If this dunning entry does not reference a specific invoice image, this column is null.

DUNNING_ACTION_CODE

NUMBER (8)

Code representing the dunning action taken on this invoice. Valid values are:

  • 1 (Late fee applied)
  • 2 (Overdue).

CURRENT_IND_CODE

NUMBER (1)

If the DUNNING_ACTION_CODE refers to a temporary state of an invoice, it is either 1 (TRUE), or 0 (FALSE), as follows:

  • 1 indicates that this dunning action represents the current state of this invoice.
  • 0 indicates that the invoice was in this state at some point, but is no longer in this state.

CUSTOMER_INVOICE_STR

VARCHAR (30)

ID that was generated for this invoice.

The format depends on the specific implementation, but is usually derived from INVOICE_ID.

ACCOUNT_ID

NUMBER (15)

Not null

ID of the account to which this invoice was applied.

If INVOICE_ID is null, this is the customer node's prime account.

ACCOUNT_NAME

VARCHAR (30)

Not null

Name of this account.

CUSTOMER_ACCOUNT_
INVOICE_STR

VARCHAR (30)

Optional ID generated for this invoice.

If defined, the ID is unique only within the account.

Its use is implementation-dependent.

IMAGE_GENERATED_IND_
CODE

NUMBER (1)

Set to True (1) when all images for the invoice have been successfully processed by the IGP, including images that have been suppressed. It is set by the IGP.

APPLIED_IND_CODE

NUMBER (1)

Set to 1 (TRUE) when this invoice has been applied to the account.

It is set by the apply_invoices server utility, as appropriate.

After an invoice is applied, it cannot be modified in any way (except for flagging it for reprint). Any account corrections must be made by adjustments displayed on the next invoice run for the customer.

The DVP checks that an entry exists in the ACCOUNT_HISTORY table for each invoice with this flag set.

ISSUE_DATE

DATE

Date on which this invoice was issued.

ORIGINAL_PAYMENT_DUE_
DATE

DATE

Date on which this invoice was originally due to be paid.

PAYMENT_DUE_DATE

DATE

Date on which this invoice is due to be paid.

It is usually the same as ORIGINAL_PAYMENT_DUE_DATE, but can be updated by privileged users.

EARLY_PAYMENT_DATE

DATE

If the customer pays before this date, they receive an additional discount.

The discount will have been incorporated into the INVOICE_TOTAL.

INVOICE_AMOUNT

NUMBER (19,3)

Total amount of charges associated with this invoice, excluding the balance forward.

If handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.

If handling of liability accounts is not enabled, a positive value is a debit for an asset account and a credit for a liability account.

Values stored in this column are at the limits of double precision.

BALANCE_FORWARD

NUMBER (19,3)

Balance on the account that was present before application of this invoice.

ACCOUNT_BALANCE

NUMBER (19,3)

Derived

Account balance after this invoice was generated.

ACCOUNT_INITIAL_DUE

NUMBER (19,3)

Initial amount due to be paid on this account.

It is usually equal to ACCOUNT_BALANCE, but may be different because of cents rounding or the presence of payment plans for the account.

CURRENT_DUE

NUMBER (19,3)

Derived

Current amount left to be paid on this invoice.

It is checked by the DVP.

TOTAL_PAYMENTS

NUMBER (19,3)

Total payments made on this account between the issuing of the previous invoice and this invoice.

The payments are not made against this invoice.

The amount is usually displayed on this invoice, and calculated by the billing engine.

TOTAL_ADJUSTMENTS

NUMBER (19,3)

Total adjustments made on this account between the issuing of the previous invoice and this invoice.

The adjustments are not made against this invoice.

The amount is usually displayed on this invoice, and calculated by the billing engine.

EARLY_PAYMENT_DISCOUNT

NUMBER (19,3)

Derived

Early payment discount (if any) applied for this invoice.

The discount is derived by the billing engine, and can be re-derived, based on details in the CHARGE table.

PAYMENT_DUE_DISCOUNT

NUMBER (19,3)

Derived

Pay-on-time discount (if any) that was applied for this invoice.

The discount is derived by the billing engine, and can be re-derived, based on details in the CHARGE table.

EARLY_PAYMENT_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the EARLY_PAYMENT_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set this column if necessary to record the account balance at the early payment date. For example, it could be set by a collections or treatment module.

PAYMENT_DUE_ACCOUNT_
BALANCE

NUMBER (19,3)

Derived

Balance of the account at the PAYMENT_DUE_DATE.

It is not set by any Convergent Billing processes, but implementation-specific processing can be used to set this column if necessary to record the account balance at the date the invoice is due to be paid. For example, it could be set by a collections or treatment module.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node associated with this instance.

CUSTOMER_NODE_TYPE_NAME

VARCHAR (30)

Not null

Type of this customer node instance.

CUSTOMER_NODE_TYPE_
DISP_NAME

VARCHAR (60)

Not null

Name of customer node type displayed.

It defaults to CUSTOMER_NODE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

CUSTOMER_NODE_TYPE_ID

NUMBER (8)

Not null

Type of this customer node instance.

NODE_NAME

VARCHAR (200)

Not null

Name associated with this node.

PARENT_CUSTOMER_NODE_ID

NUMBER (15)

Parent customer node for this node.

Only customer node types that support hierarchies can have a value in this column.

HIERARCHY_LEVEL

NUMBER (3)

Not null

Level in the hierarchy of this customer node.

The root node of a hierarchy has a level of 1.

ROOT_CUSTOMER_NODE_ID

NUMBER (15)

Derived

If this instance of a node has a PARENT_CUSTOMER_NODE_ID, this is the ID of the customer node at the root of the hierarchy.

If PARENT_CUSTOMER_NODE_ID is null, this column is also null.

CUSTOMER_NODE_STATUS_
ABBREV

VARCHAR (60)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-listed).

CUSTOMER_NODE_STATUS_
CODE

NUMBER (4)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-listed).

CREATED_DATE

DATE

Not null

Date that the customer was first added to the system, with a status of Prospect (or any other status).

ACTIVE_DATE

DATE

Date that the customer first became an active subscriber to the network; that is, when their status first changed to Active.

 

Primary Contact Details

Column Name

Properties

Description

PERSON_ID

NUMBER (15)

Not null

Primary contact person for this customer node.

Any correspondence that relates to this customer node is addressed to this person.

Other contacts are referred to in the CUSTOMER_CONTACT secondary contacts list table that displays the contact person's ID and the reason for the association with the customer node.

TITLE

VARCHAR (60)

Title of the person (Mr, Dr, Miss, Mrs, Ms, Sir, Lord, <NULL>).

FAMILY_NAME

VARCHAR (110)

Family name or surname.

GIVEN_NAMES

VARCHAR (80)

Person's first name and/or middle name.

OFFICIAL_NAME

VARCHAR (200)

Not null

Official name to be used in corresponding with this person.

The naming conventions (for example, family, first, or last), specified for individual installations, defaults this value according to the information stored in other naming fields.

GENDER_ABBREV

VARCHAR (60)

Code for whether the person is male or female. Valid values are:

  • 0 (Male)
  • 1 (Female).

GENDER_CODE

NUMBER (1)

Code for whether the person is male or female. Valid values are:

  • 0 (Male)
  • 1 (Female).

PERSON_STATUS_ABBREV

VARCHAR (60)

Not null

Status of this person.

The reference type for the values is obtained from the PERSON_TYPE table. Valid values are:

  • 0 (Cancelled)
  • 1 (Prospect)
  • 2 (Subscriber)
  • 3 (Business Contact)
  • 4 (Subscriber and Business Contact)
  • 5 (VIP)
  • 6 (Bad Risk).

PERSON_STATUS_CODE

NUMBER (2)

Not null

Status of this person.

The reference type used for the values is obtained from the PERSON_TYPE table. Valid values are:

  • 0 (Cancelled)
  • 1 (Prospect)
  • 2 (Subscriber)
  • 3 (Business Contact)
  • 4 (Subscriber and Business Contact)
  • 5 (VIP)
  • 6 (Bad Risk).

CREDIT_LIMIT

NUMBER (19,3)

Amount of credit that a customer node can have at any one time.

It is only required if REPORT_LEVEL indicates that invoices are to be produced.

CREDIT_LIMIT_CURRENCY_
ID

NUMBER (8)

Currency for the above credit limit value.

CREDIT_RATING_ABBREV

VARCHAR (60)

Indicator of customer rating in accounting matters including invoicing and payment collection.

VIP customers are given the highest rating of 1.

The credit rating is defaulted by the CUSTOMER_NODE_TYPE.

CREDIT_RATING_CODE

NUMBER (4)

Indicator of customer rating in accounting matters including invoicing and payment collection.

VIP customers are given the highest rating of 1.

The credit rating is defaulted by the CUSTOMER_NODE_TYPE.

CREDIT_COMMENTS

VARCHAR (255)

Comments relating to a customer's credit rating and the assigned limit, and provides reasons and justifications for the assigned values.

TAX_CLASS_ABBREV

NUMBER (4)

Tax class with which this customer node is associated.

SALES_CHANNEL_ABBREV

VARCHAR (60)

Sales channel for this customer node.

SALES_CHANNEL_CODE

NUMBER (4)

Sales channel for this customer node.

COMPANY_NUMBER

Official company or business registration number (for example, A.C.N).

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be defined in the type definition table.

The constraints of each column are defined by the details of its CUSTOMER_NODE_TYPE.
