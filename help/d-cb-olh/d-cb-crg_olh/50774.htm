TASK_TRE_V

The TASK_TRE_V view defines the valid columns for use by the biTask TRE functions.

The biTask TRE functions are used for viewing, updating, and deleting tasks.

In the column descriptions, the following definitions apply:

  • 'This task' refers the task identified by TASK_ID.
  • 'The schedule' refers the schedule identified by SCHEDULE_ID.
  • 'The schedule type' refers the schedule type identified by SCHEDULE_TYPE_ID.
  • 'This task's prerequisites' refers to the prerequisite tasks defined for this task in the TASK_PREREQUISITE_TRE_V.

SCHEDULED_START_DATE

DATE

Not null

Date-time this task is scheduled to start.

For a newly-inserted task, it is derived from the schedule's REPEAT_UNITS and can be modified by the schedule's TASK_CREATION_EXPR.

It can be updated only if TASK_STATUS_CODE is 1 (Pending), 5 (Stalled), or 8 (Held).

BASELINE_START_DATE

DATE

Not null

Read-only

Date-time this task is expected to start.

It is set when the task is created.

If a task does not have any prerequisites, it equals SCHEDULED_START_DATE. Otherwise, it is based on SCHEDULED_START_DATE, and the BASELINE_START_DATE and BASELINE_DURATION fields of this task's prerequisite tasks.

It is usually static for the existence of a task. However, it is updated when SCHEDULED_START_DATE or the relevant baseline details on prerequisite tasks are updated.

START_DATE

DATE

Not null

Read-only

For a pending, stalled, or held task, it is the date-time this task is scheduled to start. Otherwise, it is the date-time the task started.

For a newly-created task, it defaults to BASELINE_START_DATE. However, as prerequisite tasks are run, it is updated to closer approximate the start date of the task until the task is run.

If TASK_STATUS becomes 5 (Stalled) or 8 (Held), START_DATE is updated periodically.

EFFECTIVE_DATE

DATE

Not null

Date-time used to select the related information stored in the database.

It defines the effective date for record definitions and the calculation of relevant charges.

BASELINE_DURATION

NUMBER (10)

Not null

Expected duration of this task in seconds.

It defaults from the schedule, but can be overridden by:

  • The baseline duration expression (BASELINE_DURATION_EXPR) on the SCHEDULE_TYPE_TRE_V view if the expression does not evaluate to a constant.
  • Entity validation.
  • The user.

If the schedule type is configured using the TaskAverageDuration&() function, for new tasks it is derived from the duration of the last 'n' successfully completed tasks on the schedule.

It is usually static for the life of a task.

It must be greater than or equal to 1, and less than or equal to 2147483647.

MAX_DURATION

NUMBER (10)

Maximum permitted duration of this task in seconds.

The TLM terminates running tasks that exceed MAX_DURATION.

It defaults from the schedule, but can be overridden by:

  • The maximum duration expression (MAX_DURATION_EXPR) on the SCHEDULE_TYPE_TRE_V view if the expression does not evaluate to a constant.
  • Entity validation.
  • The user.

It must be one of the following:

  • NULL.
  • Greater than or equal to BASELINE_DURATION, and less than 2147483647.

If NULL, the task runs indefinitely unless it terminates itself.

BASELINE_END_DATE

DATE

Not null

Read-only

Redundant

Date-time this task is expected to end.

It is set when the task is created.

It is calculated as follows:

BASELINE_END_DATE = BASELINE_START_DATE + BASELINE_DURATION - 1

DURATION

NUMBER (10)

Not null

Read-only

Expected or actual duration of the task in seconds.

It must be greater than or equal to 1, and less than 2147483647.

For a newly-created task, it defaults to BASELINE_DURATION.

When a task is running and the time taken for the task to run exceeds DURATION, the STD updates it to the time taken for the task to run.

When a task completes, it is updated to the complete duration of the task.

END_DATE

DATE

Not null

Redundant

Read-only

Before a task completes, it is the date-time this task is expected to end.

For a completed task, it is the date-time this task completed with success or failure.

It is calculated as follows:

END_DATE = START_DATE + DURATION - 1

TASK_STATUS_CODE

NUMBER (8)

Not null

Current execution status of this task.

Valid values are defined in Table 6.

User-initiated updates to the TASK_STATUS_CODE are supported by biTaskUpdate&().

Table 7 lists valid TASK_STATUS_CODE transitions. For example, the first row of the table indicates that if TASK_STATUS_CODE is 1 (Pending), the user is permitted to update TASK_STATUS_CODE to 8 (Held).

When it is updated from 3 (Success), 4 (Failure), or 7 (Stopped) to 1 (Pending), this update is interpreted as a resubmission of the task. Depending on the schedule type's configuration this update may result in the creation of a new task.

It is based on the TASK_STATUS reference type.

TASK_STATUS_REASON_CODE

NUMBER (3)

Read-only

Code that provides further explanation of this task's status.

It is populated only when TASK_STATUS_CODE is one of the following:

  • 4 (Failure)
  • 5 (Stalled)
  • 6 (Stopping)
  • 7 (Stopped)
  • 8 (Held).

It is based on the TASK_STATUS_REASON reference type.

Table 8 to Table 12 list the TASK_STATUS_REASON_CODE codes and their corresponding meanings.

PREREQUISITE_TASK_ID

NUMBER (15)

Read-only

If TASK_STATUS_REASON_CODE equals 5 (Task prerequisite), it specifies the prerequisite task that resulted in this task's status.

If multiple prerequisite tasks contributed to this task's status, one of the tasks is chosen to populate it.

DEPENDENT_RESOURCE_NAME

VARCHAR (60)

Read-only

If TASK_STATUS_REASON_CODE equals 6 (Resource compatibility) or 7 (Resource profile limit), it specifies the name of a resource that resulted in this task's status. 

If multiple resources contributed to this task's status, one of the resources is chosen to populate it.

DEPENDENT_RESOURCE_KEY

VARCHAR (255)

Read-only

If TASK_STATUS_REASON_CODE equals 6 (Resource compatibility) or 7 (Resource profile limit), it defines the name of the resource key of DEPENDENT_RESOURCE_NAME that resulted in this task's status.

If multiple resource keys contributed to this task's status, one of the resource keys is chosen to populate it.

It is populated only when DEPENDENT_RESOURCE_NAME is populated.

PROCESS_IDENTIFIER

NUMBER (10)

Read-only

ID of the server process running this task.

The TLM process populates it when the TLM starts the task.

It is populated only when TASK_STATUS equals 2 (Running) or 6 (Stopping).

PURGE_PERIOD

NUMBER (8)

Maximum number of days that the task and any of its associated data will be kept online after it has completed.

It defaults to the value contained in the schedule definition.

If not specified, the task is maintained indefinitely online.

IMMEDIATE_START_IND_CODE

NUMBER (1)

Contains a bit mask of settings that control the immediate starting of a task, as listed in Table 13.

It defaults from the task's schedule.

INITIAL_HELD_IND_CODE

NUMBER (1)

If it equals 1 (TRUE) when this task is scheduled to run, TASK_STATUS_CODE is set to 8 (Held) and TASK_STATUS_REASON_CODE is set to 11 (user request).

Manual intervention is necessary to start processing the task.

If populated, the HELD_NOTIFICATION_EXPR column in the SCHEDULE_TYPE_TRE_V view is evaluated (for example, to notify users of tasks entering 8 (Held) status).

If it equals NULL when this task can be run, the task executes immediately.

If it does not equal 1 (TRUE), it must be NULL.

It defaults from the schedule.

It can be updated only for tasks either with a status of 1 (Pending) or tasks being updated to a status of 1.

PREREQUISITE_FAILURE_IND_
CODE

NUMBER (1)

If it equals 1 (TRUE), the failure of a prerequisite task condition causes the STD to update TASK_STATUS to 4 (Failure) rather than the default, 8 (Held).

Note:

The resubmission of the failed prerequisite task does not result in the resubmission of this task.

If it is not 1 (TRUE), it must be NULL.

It defaults from the schedule.

RESUBMITTED_TASK_ID

NUMBER (15)

Read-only

If this task was created resulting from resubmission of a failed task, it equals the TASK_ID of the failed task.

If resubmission only updates the failed task, it is set to this task's TASK_ID.

PENDING_UPDATE_IND_CODE

NUMBER (1)

Read-only

Set to 1 (TRUE) when a task is updated and the update requires the regeneration of the task's attributes (for example, resources).

While set, the task's attributes are possibly out of date.

It is set by a biTaskUpdate&() call, and cleared by the STD after it has regenerated the task's attributes.

If not set to 1 (TRUE), it must be NULL.

SERVER_COMMAND_ARGS

VARCHAR (255)

Additional server command-line parameters used when invoking this task.

If specified, they are added to any command-line parameters specified in the schedule type.

It defaults from this task's schedule.

UPDATE_EFF_DATE_IND_CODE

NUMBER (1)

Pseudo-column

Used for updates only

If set to 1 (TRUE), the EFFECTIVE_DATE of the task referenced by NEXT_TASK_ID of an entitlement can be updated.

DO_NOT_NOTIFY_IND_CODE

NUMBER (1)

Pseudo-column

Used for updates only

Set to 1 (TRUE) if notification is not required to be sent to the STD about the changes to the schedule.

It can be used for bulk loading of schedules where the schedule is not running or will be restarted on completion of the load.

If specified, RESPONSE_IND_CODE cannot be specified.

RESPONSE_IND_CODE

NUMBER (1)

Pseudo-column

Used for updates only

If specified, the STD sends an asynchronous notification back to the caller as soon as the next pending task for the schedule has completed.

CLIENT_INSTANCE_NAME

VARCHAR (9)

Pseudo-column

Name of instance to which the Tuxedo client that made this request is connected.

Specify only if the biTaskUpdate&() call is being made to an instance different from that on which the client is connected, and an acknowledgement or response has been requested.

CLIENT_ID

VARCHAR (40)

Pseudo-column

ID of the Tuxedo client that made this request.

Note:

It must be specified if CLIENT_INSTANCE_NAME is specified.

GENERAL_1 - GENERAL_25

VARCHAR (255)

General fields 1 - 25 for specifying parameters for scheduled tasks.

Table 13 IMMEDIATE_START_IND_CODE Bit Mask Settings

Bit

Description
