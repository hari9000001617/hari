Description

Inserts an interface definition and the derived attribute tables that define the fields on its tabs.

The interface definition must be based on an existing interface template, and the interface type is derived from the interface template on which it is based.

Note:

Derived attribute tables that define grids on the interface definition's tabs are excluded. Use fEV_InterfaceDefinitionTableInsert&() to insert rows in associated derived attribute tables that define grids.

Context

The following groups of variables are available throughout all stages of processing an interface definition insert request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value INTDSVR.
    • evLastRowInd& contains the value 0 during all stages of processing except for the evaluation of the pre-commit expressions.
    • evRowStatus$ contains the value Insert.
  • Interface definition
  • Attribute type variables associated with the derived attribute tables to be inserted.

Parameters

Parameter

Description

InterfaceTemplateId&

ID of the interface template on which to base the new interface definition.

InterfaceTemplateName$

Name of the interface template on which to base the new interface definition.

LastModified~

Date-time the record was inserted.

The initial value of this parameter is ignored.

FieldNames$[]

Names of the fields to be inserted.

Names must correspond to the name of:

  • A column in the INTERFACE_DEFN table.
  • The interface label of an attribute type associated with the derived attribute table's entity validation.

FieldValues?[]

The values corresponding to the specified field names.

The data passed must match the data type and value expected by the field.

Denormalised fields must pass the denormalised string values.

FieldValues?{}

Hash containing the field names and values.

The restrictions for FieldNames$[] and FieldValues?[] apply.

IdentifierKeys?{}

Populated on successful return with the values of the interface definition identifier fields derived from FieldValues?{}.

 

Processing

Assuming that all processing stages are enabled, the following is the order of evaluation of expressions and constraints before the insertion of the new interface definition.

Note:

Derived attribute tables are only processed if they are associated with at least one inserted field element. If all values for the variables associated with the derived attribute table are undefined at the point of insertion, the row is not inserted.

  1. Bundle direct variables if the call is nested.
  2. Initialise all direct variable values from the interface template.
  3. Apply attribute type defaults and validation.
  4. Apply specified field values to corresponding direct variables.
  5. Evaluate derived attribute defaults by tab sequence number.
  6. Check the mandatory fields defined by the derived attribute tables, in tab sequence number order.
  7. Evaluate attribute type validation expressions associated with derived attribute tables, in tab sequence number order.
  8. Evaluate any derived attribute table row expressions, in tab sequence number order, if fields on that tab have been inserted.
  9. Evaluate any interface definition row expressions.
  10. Evaluate any derived attribute pre-commit expressions in tab sequence number order, if fields on that tab have been inserted.
  11. Evaluate any interface definition pre-commit expressions.
  12. Encrypt the necessary variable values.

Upon performing the insertion, the following sequence of actions occur:

  1. Evaluate derived attribute table in-transaction expressions, in tab sequence number order, if fields on that tab have been inserted.
  2. Evaluate interface definition in-transaction expressions.
  3. Queue and evaluate derived attribute table post-commit expressions, in tab sequence number order, if fields on that tab have been inserted.
  4. Queue/evaluate interface definition post-commit expressions.

After the insertion and the evaluation of the post-commit expressions, the following steps occur:

  1. Unbundle variables that were stored before the insertion.

Returns

ID of the new interface definition; otherwise, an error is raised.
