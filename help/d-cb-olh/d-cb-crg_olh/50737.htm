INV_OTHER_INVOICES_V

The INV_OTHER_INVOICES_V view displays details of other invoice accounts and their associated customer details.

It is a combination of the information available in the INV_INVOICE_V and the INV_CUSTOMER_NODE_HISTORY_V views, but is used for invoices whose details need to be included in this invoice's results.

APPLIED_IND_CODE

NUMBER (1)

It is set by the apply_invoices server utility as appropriate.

Any account corrections must be made by adjustments displayed on the next invoice run for the customer.

EARLY_PAYMENT_DATE

INVOICE_AMOUNT

NUMBER (19,3)

Not null

Total amount of charges associated with this invoice, excluding the balance forward.

If handling of liability accounts is:

  • Enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.
  • Not enabled, a positive value is a debit for an asset account and a credit for a liability account.

Values stored in this column are at the limits of double precision.

STATEMENT_AMOUNT

NUMBER (19,3)

Total amount of charges associated with this statement.

It can only have a value if INVOICE_AMOUNT is 0.

For secondary accounts, if handling of liability accounts is enabled, a positive value is always a debit, irrespective of the class of the account to which it is being applied.

NUMBER (19,3)

PENDING_CONSOLIDATION_IND_
CODE

NUMBER (1)

Indicates if the statement was switched from an invoice to a statement pending consolidation.

If set to 1, STATEMENT_AMOUNT must be specified.

CONSOLIDATION_INVOICE_ID

NUMBER (15)

ID of the invoice created to consolidate this statement, and other statements into a single invoice.

If specified, STATEMENT_AMOUNT must be specified and PENDING_CONSOLIDATION_IND_CODE must be null.

CONSOLIDATED_INVOICE_IND_
CODE

NUMBER (1)

Indicates if this invoice has been created as a consolidated invoice for one or more statements.

If set to 1:

  • STATEMENT_AMOUNT must be null.
  • One or more rows on the INVOICE table must:
    • Reference this INVOICE_ID as their CONSOLIDATION_INVOICE_ID.
    • Have the sum of all STATEMENT_AMOUNT columns equal to the INVOICE_AMOUNT of this invoice.

PARENT_CUSTOMER_NODE_ID

NUMBER (3)

NUMBER (4)

Not null

Status of this instance of the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)

  • 4 (Cancelled)
  • 5 (Black-listed).

CREATED_DATE

DATE

Not null

Date that the customer was first added to the system, with a status of Prospect (or any other status).

ACTIVE_DATE

DATE

Date that the customer first became an active subscriber to the network; that is, when their status first changed to Active.

REPORT_LEVEL_CODE

NUMBER (2)

Not null

Level of financial reporting required for this customer node. Valid values are:

  • 1 (Invoice)
  • 2 (Statement)
  • 3 (No Reporting)
  • 4 (Transferred Statement).

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 containing invoice or statement specific details.

Their use and meaning is controlled by entity validation on the invoice type.

They are populated as required by invoice type expressions.

CUSTOMER_GENERAL_1 - CUSTOMER_GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be used to store customer node specific information.

The definition and constraints for each column are defined by the customer's type.

REPORT_GENERAL_1 - REPORT_GENERAL_10

VARCHAR (255)

General fields 1 - 10 that can be populated by the invoice hierarchy scripts as required.
