fCMN_TREProdInstServiceListRowAdd_R&()

Syntax

fCMN_TREProdInstServiceListRowAdd_R&(
    l_ProductInstanceId&,
    l_ServiceId&,
    l_TableName$,
    l_EffectiveDate~,
    l_LastModDate~,
    l_Array?[])

 

Description

Adds a new row to a specified service list.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_ProductInstanceId&

In: Product instance ID of the product instance to which the service belongs.

l_ServiceId&

In: Service ID of the list.

l_TableName$

In: Derived attribute table name.

l_EffectiveDate~

In: Effective date from when the addition should take effect.

If not specified, the current date is used.

l_LastModDate~

In: Last modified date of the service to which the list rows are being added.

It must match the last modified date for the service record being updated; otherwise, an exception is raised.

The last modified date can be retrieved using fCMN_TREProdInstServiceSearch_R?[]().

l_Array?[]

In: Array of hashes containing the values for the rows to be added.

 

Returns

1 if successful; otherwise, an exception is raised.

Remarks

The following table lists key of the hash in the l_array?[] parameter.

 

l_array?[] Parameter Hash Key

Key

Data Type

Description

[<ROW_NR>]
{<COLUMN_NAME>}

Various.

The list element column's data type is dependent on the attribute type in the entity validation for the derived attribute.

<COLUMN_NAME> refers to the name of the column for the list, as it is displayed on the Service Details form in the Convergent Billing client.

The valid values for the fields depend on the attribute type used for the column.

<ROW_NUMBER> is an array of list rows for the table from 0..n.
