INV_NORMALISED_EVENT_NAME_V

The INV_NORMALISED_EVENT_NAME_V view displays identical details to INV_NORMALISED_EVENT_V, except that it is queried by SERVICE_NAME, rather than SERVICE_ID.

It does not display charge details of events associated with global tariffs. The INV_NORMALISED_EVENT_V is used for those charge details.

This view is used by the IGP, and the Bill Run Detail form.

Chargeable (C) Party Details

Column Name

Properties

Description

C_PARTY_ID

VARCHAR (40)

Full national number or most precise ID for a chargeable party service.

It is usually equal to A_PARTY_ID, but is equal to B_PARTY_ID party for reverse charge and 008 calls, and some other third party for conference calls.

C_PARTY_NETWORK_ID

VARCHAR (40)

Network ID of the chargeable party (C party), if any. For a GSM service, this corresponds to the IMSI number.

At least one of the party details (C_PARTY_ID or C_PARTY_NETWORK_ID) must be specified, although it is not enforced for performance reasons.

C_PARTY_NAME

VARCHAR (40)

Name of the C party, if known; for example, this can be provided by the switch for user-assisted calls.

C_PARTY_INTERNAL_ID

NUMBER (15)

ID of the chargeable party, used by the RGP to guarantee identification of a service.

If C_PARTY_INTERNAL_ID is provided, it is used instead of the C_PARTY_ID or C_PARTY_NETWORK_ID to perform the service lookup.

Note:

External data feeds must not populate this column. If it is set and the EVENT_CLASS_CODE is not Usage (1), the billing engine includes any charges associated with this event in the current bill cycle, even if they have a CHARGE_START_DATE after the bill cycle effective date.

C_PARTY_TON_CODE

NUMBER (8)

Charge party type of number.

C_PARTY_CELL_ID

VARCHAR (20)

Charge party network cell ID (Example: mobile cell).

C_PARTY_CARRIER_CODE

NUMBER (8)

Carrier to which the call is destined.

C_PARTY_LOCATION_CODE

NUMBER (8)

Geographic location that specifies the origin of the call to the finest granularity supported by the system.

Higher forms of location such as country and continent can be derived from this value.

C_PARTY_ROUTE

VARCHAR (80)

Incoming or outgoing route of the chargeable party.

 

Miscellaneous Event Details

Column Name

Properties

Description

FULL_PATH

VARCHAR (40)

Available details relating to the route that a call used; for example, X.25 DNIC sequence for call.

It can alternatively be used to store the direction or type of call (for example, Incoming, Outgoing, Internal, Transit).

CASCADE_CARRIER_CODE

NUMBER (8)

Carrier closest to the current network that was used in the completion of the call (may not be known).

CHARGE_START_DATE

DATE

Date-time for charging purposes, usually derived from the switch date.

PERIOD_START_DATE

DATE

Date-time of the start of this event, used for events that extend over a period of time (for example, events associated with recurring charges).

PERIOD_END_DATE

DATE

Date-time of the end of this event, used for events that extend over a period of time (for example, events associated with recurring charges).

EVENT_CLASS_CODE

NUMBER (2)

General classification of the event used for reporting purposes. Valid values are:

  • 1 (Usage event): The event relates to the usage of a service
  • 2 (Recurring event): The event represents a rental period, typically generated by the RGP.
  • 3 (One-off or ad hoc event): These include automatically generated one-off events (generated by the RGP), and ad hoc events generated by users or other sources.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

DURATION

NUMBER

Duration of call. The units depend on event type.

VOLUME

NUMBER

Volume of data associated with call.

The units depend on the call type.

If populated by events generated by the RGP, its contents should be specified as volume per unit duration.

PULSES

NUMBER (8)

Number of meter pulses associated with call.

If populated by events generated by the RGP, its contents should be specified as pulses per unit duration.

RATE_BAND

VARCHAR (40)

Time or rate band calculated for this call.

Properties

Name of the tariff, discount, or tax that generated this charge, if any.

TARIFF_TYPE_CODE

NUMBER (8)

Numeric code that defines the type of the tariff.

SUBTOTAL_ID

NUMBER (8)

Subtotal, or total calculation that generated this charge.

SUBTOTAL_NAME

VARCHAR (30)

Name of the subtotal that generated this charge, if any.

SUBTOTAL_DATE

DATE

Date-time calculated by the subtotal, if this subtotal generates a date-time result.

It can only contain a value if SUBTOTAL_ID is not null and CHARGE is null.

It is checked by the DVP.

EFFECTIVE_START_DATE

DATE

Effective start date of the tariff or subtotal associated with the call.

EFFECTIVE_END_DATE

DATE

Effective end date of the tariff or subtotal associated with the call.

INVOICE_SECTION_NR

NUMBER (8)

Invoice section number associated with this charge.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number associated with this charge.

INVOICE_TXT

VARCHAR (255)

Optional invoice text that can be displayed in association with this charge.

KEY_VALUE

VARCHAR (255)

Hash key value or array index value associated with this charge for this tariff or subtotal.

It is only populated for charges generated by hash or array tariffs and subtotals.

For:

  • Hash-based tariffs or subtotals, it stores the string hash key associated with this charge.
  • Array tariffs or subtotals, it stores the integer array index associated with this charge.

 

Event General Columns

Column Name

Properties

Description

GENERAL_1 - GENERAL_20

VARCHAR (4000)

General fields 1 - 20 for normalised event data.

Their use is controlled by the normalised event type.

Typical values are for the derivation of derived normalised event attributes that need to be stored for reporting, or decoding of location codes.

 

Charge General Columns

Column Name

Properties

Description

CHARGE_GENERAL_1 - CHARGE_GENERAL_20

VARCHAR (255)

General fields 1 - 20 for charge data.

They can be used to store intermediate calculations and lookups performed during rating (for example, distance band, time band, multi-time band flag).

The information is accessible by the BGP and the IGP.

 

Charge Partition Column

Column Name

Properties

Description

PARTITION_NR

NUMBER (2)

Not null

Customer partition number associated with the charge generated by the normalised event.

All charges associated with an invoice are associated with the same account (and therefore customer), so when customer partitioning is implemented, all charges for an invoice have the same PARTITION_NR.

It can be used as an additional selection criterion to improve access to the CHARGE table by eliminating irrelevant CHARGE table partitions.
