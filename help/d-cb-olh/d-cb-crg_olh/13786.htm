Description

Updates a service record.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

ServiceId&

In: ID of the service to be updated.

OldEffectiveDate~

In: Date-time used for identifying the record to be updated.

LastModified~

In: Last modified date-time of the service to be updated.

Out: Last modified date-time of the updated service.

FieldNames$[]

In: List of fields to be updated from the SERVICE_TRE_V view.

Specify string fields first, followed by integer, real, and date fields.

FieldValues?[]

In: Field values, in the same order that their names were specified in FieldNames$[].

StringFieldValues$[]

In: Values of string fields to update.

IntegerFieldValues&[]

In: Values of integer fields to update.

RealFieldValues#[]

In: Values of real fields to update.

DateFieldValues~[]

In: Values of date fields to update.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The names of the fields to be updated are passed in the FieldNames$[] array.

A service cannot be updated to or from a status of Product Creation.

A service can be updated to a status of Cancelled only if the service status following the end date of the cancellation is Cancelled, Moved, or Upgraded.

If a service is updated to a status of Cancelled, this function checks if:

  • The service is the only service associated with the base product instance.
  • Unallocated service or equipment items are associated with the base product instance.
  • Cancellation of the service violates the minimum service requirements for:
    • The base product instance
    • An associated companion product instance if a product offer is assigned to the base product instance.

If any of the above conditions is met, and the base product instance for the service is:

  • Assigned a product offer, the base product instance is flagged as requiring re-completion.
  • Not assigned a product offer, the service cannot be updated to a status of Cancelled.

When a service is added to a future-cancelled product instance, during the process of re-completing the product instance, the added service is cancelled at the same date-time the product instance becomes cancelled.

Updating a service to a status of Cancelled re-derives the status of any associated equipment items.

Note:

If there are multiple services with the same SERVICE_NAME for a given date-time range, only one can have a status of Active.

A successful update results in biServicePurge&() being called to clear data for the updated service from the service cache, including the service ID, service name, and network name caches, as appropriate, unless DO_NOT_PURGE_IND_CODE is set to 1.

A service can be assigned to a product offer, subject to the same conditions on the product offer as for inserting a service.

Note:

To use the supplied product offer, the effective user must have at least view security group access level for the group associated with the product offer.

The effective user must have update security group access level for the customer associated with the record manipulated by this function call.

The customer security group access level required is determined by the effective start date of the product instance, or the effective date supplied by the active user. If the effective date of the update is earlier than the current date (00:00:00 timestamp), direct update security group access level is required.

GROUP_ACCESS_TRE_V view lists security group access-level values.
