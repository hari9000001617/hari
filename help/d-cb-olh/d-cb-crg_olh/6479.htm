Customer Hierarchy Form

The following table lists automation parameters for the Customer Hierarchy form, data type, and supported functions.

Note:

The automation parameters for this form do not correspond to any specific TRE View.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

CUSTOMER_NODE_ID

Integer

View

Customer node added to the outline view of the hierarchy.

Passing multiple values to the form enters a number of customers.

SERVICE_ID

Integer

View

Service in the detail view of the form, and the corresponding customer in the outline view of the form.

If multiple values are passed to the form, the last value corresponds to the item that receives focus.

Specifying this field has no effect if the Display Entities field disables the display of services in the hierarchy.

CUSTOMER_QUERY_ID

Integer

View

Customer query in the detail view, and the corresponding customer in the detail view of the form.

Specifying this field has no effect if the Display Entities field disables the display of customer queries in the hierarchy.

DisplayEntities

Integer

View

Entities displayed in the detail view.

It is equivalent to the check boxes displayed on the form. Valid values are:

  • 1 (Contacts)
  • 2 (Products)
  • 4 (Services)
  • 8 (Equipment)
  • 16 (Open Queries)
  • 32 (Closed Queries)
  • 64 (Contracts)
  • 128 (Child Customers)
  • 256 (Cancelled Customers)
  • 512 (Cancelled Products)
  • 1024 (Cancelled Services)
  • 2048 (Reserved Equipment)
  • 4096 (Cancelled Contracts)
  • 8192 (Companion Products)
  • 16384 (Facility Groups).

Combinations of entities are displayed by adding the values.
