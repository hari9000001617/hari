CustomerNodeInvFormatUpdate&()

Syntax

CustomerNodeInvFormatUpdate&(
    CustomerNodeId&,
    OldEffectiveDate~,
    var LastModified~,
    EffectiveStartDate~,
    EffectiveEndDate~,
    const Opcode&[],
    const Seqnr&[],
    const FieldNames$[],
    const FieldValues?[])

 

Description

Inserts, updates, or deletes invoice formats for a specified customer node.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

CustomerNodeId&

In: ID of the customer node whose invoice format is to be updated.

OldEffectiveDate~

In: Date-time used to retrieve the correct CUSTOMER_NODE_HISTORY row.

LastModified~

In: Last modified date-time of the CUSTOMER_NODE_HISTORY row effective as at OldEffectiveDate~.

Out: Last modified date-time of the CUSTOMER_NODE_HISTORY row effective as at EffectiveStartDate~.

EffectiveStartDate~

In: Start of date-time range for which changes are to be applied.

EffectiveEndDate~

In: End of date-time range for which changes are to be applied.

Opcode&[]

Operations to perform on the row.

Seqnr&[]

In: Sequence number of the row being inserted, updated or deleted from the CUSTOMER_NODE_INV_FORMAT table.

FieldNames$[]

In: Names given to the set of fields being inserted or updated.

The same fields must be specified for all rows.

FieldValues?[]

In: Two-dimensional array of field values in [Row][Field] order being inserted, updated or deleted.

An empty row may be specified for records being deleted.

 

Returns

Number of rows successfully processed (which should equal the size of Opcode&[]); otherwise, an error is raised.

Remarks

A last modified check and date-time ranged update is automatically performed on the CUSTOMER_NODE_HISTORY record associated with this change, with the updated CUSTOMER_NODE_HISTORY last modified date-time being returned in LastModified~.

A date-time range update is also performed on the CUSTOMER_NODE_INV_FORMAT table. Multi-range update is not supported; therefore, if either the EffectiveStartDate~ or EffectiveEndDate~ fall outside the CUSTOMER_NODE_HISTORY record's date-time range, an error is raised.

The field names to be updated can be placed in any order in the FieldNames$[] array, but must be valid fields from CUSTOMER_NODE_INV_FORMAT_TRE_V view.

The update operations that can be performed on a customer nodes' invoice format are 1 (Insert), 2 (Update), and 3 (Delete) and are represented by the value placed in the Opcode&[] array. If multiple updates are to be performed on this customer node, the number of the values in the Seqnr&[] array must equal the number of values in the Opcode&[] array.

The effective user must have update security group access level for the group of the customer associated with the record manipulated by this function call. If the effective date of the update is less than the current date (00:00:00 timestamp), direct update security group access level is required.

GROUP_ACCESS_TRE_V view lists security group access level values.
