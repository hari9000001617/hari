EventRateAndReturnTariff&()

Syntax

EventRateAndReturnTariff&(
    TariffId&,
    const FieldNames$[],
    const FieldValues?[],
    const ChargeFieldNames$[],
    var ChargeFieldValues?[])

 

Description

Calculates and returns charge information using a specific tariff.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

TariffId&

In: ID of the tariff used to rate the event.

FieldNames$[]

In: Field names to set for the event. Only fields from the NORMALISED_EVENT_V view are valid.

Note:

The mandatory fields required for the event to be inserted into the NORMALISED_EVENT or NORMALISED_EVENT_ERROR tables must be specified.

FieldValues?[]

In: Field values for the event, in the same order as the names are specified in FieldNames$[].

ChargeFieldNames$[]

In: Charge fields to return from the charge calculations.

ChargeFieldValues?[]

Out: Charge values, corresponding to the ChargeFieldNames$[] parameter.

The first dimension corresponds to a charge.

The second dimension corresponds to the requested charge fields.

 

Returns

Charge values, if successful; otherwise, an error is raised.

Remarks

The return value corresponds to the number of rows returned in ChargeFieldValues?[].

If TariffId& is specified, the arrays are rated against that tariff. The function evaluates the tariff using the supplied event details and the service details retrieved using the last call to EventRateAndReturn&().

If TariffId& is not specified, the function evaluates all tariffs for the services associated with the normalised event resulting from the last call to EventRateAndReturn&().

If the function is called to determine the amount to reserve for an account, and either AccountReservationInsert$(), which creates the reservation, or AccountReservationUpdate&(), which updates the reservation, are executed, call SubtotalEventCommit&() or EntitlementEventCommit&() to reserve any rating subtotal and entitlements.

Note:

C-party details contained in the FieldNames$[] and FieldValues?[] parameters do not determine the set of services associated with the event.
