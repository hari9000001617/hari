fEVS_PostCommitError&()

Syntax

fEVS_PostCommitError&(
    entityType$,
    entityId&,
    secondaryId&,
    startDate~,
    endDate~,
    rowStatus$,
    var notify?{},
    errorId&,
    errorMsg$)

 

Description

Error handler function called in the event of an error occurring during post-commit processing.

Note:

This function is a Singleview-supplied sample function.

Parameter

Parameter

Description

entityType$

Name of the entity associated with the type of entity that was processed (for example, CUSTSVR and PERSSVR).

entityId&

Numeric ID of the entity that was processed.

secondaryId&

Secondary numeric ID for the entity being processed, if required.

For most entities, it is undefined. However, for customer and service derived attribute lists, it is the derived attribute ID of the data being processed.

startDate~

Effective start date of the entity that was processed, if the entity is date-ranged.

If the entity is not date-ranged, it must be undefined.

endDate~

Effective end date of the entity that was processed, if the entity is date-ranged.

If the entity is not date-ranged, it must be undefined

rowStatus$

Row status of the operation that was performed. Valid values are:

  • Insert
  • Update
  • Delete.

notify?{}

evsNotify?{} hash parameter made available during the post-commit processing.

It can be used to pass extra information to the post-commit notification callback functions.

errorId&

Numeric ID of the error that occurred.

errorMsg$

Error message of the error that occurred.

 

Remarks

The supplied parameters form an event. The event name is formed by a colon-separated list, containing the following fields in the specified order:

  1. The prefix EVS_POST_ERR
  2. Entity type name (for example, Customer, Person, and Service)
  3. Row status (Insert, Update, or Delete)
  4. Numeric ID of the entity (for example, 3001856).

For example, for an update to a customer with the customer node ID 3001856, the event name is EVS_POST_ERR:Customer:Update:3001856.

The remaining parameters form an event array in the following order:

  1. Effective start date of the entity that was processed
  2. Effective end date of the entity that was processed
  3. Numeric error ID for any error that occurred
  4. Error message for any error that occurred
  5. evsNotify?{} hash.

Any parameters that are not relevant (for example, start date for non-date-ranged entities) must be specified as undefined values.

Returns

Undefined.
