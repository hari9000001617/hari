fCMN_TREPaymentBatchCreate_R&()

Syntax

fCMN_TREPaymentBatchCreate_R&(
    l_LastModDate~,
    l_PaymentBatchDetails?{})

 

Description

The core TRE API provides mechanisms to insert payment batch records, individual payments for the batch, and invoice allocations for each of these payments.

Attributes

Application Environment

Any

Context

Any

 

Parameter

Description

l_LastModDate~

Out: Date-time the record was last modified.

l_PaymentBatchDetails?{}

In: Hash containing the values for the payment creation.

Each name key and element pair in l_PaymentBatchDetails?{} form a name value pair. An example is the following parameter value:

l_PaymentBatchDetails?{'BATCH_APPLY_IND_CODE'} := 1;

For more information, refer to Remarks below.

 

Returns

New payment batch ID created.

Remarks

Payment batch creation allows the user to insert a complete batch that includes all subordinate objects in a single transaction. As with all CBF APIs, what is in the hash map data structure determines CBF behaviour.

The high-level processing of the function is:

  1. Map client batch label names (if passed) with database (TRE) column names.
  2. Insert the initial empty payment batch using the existing TRE API.
  3. For each payment in the batch:
    1. Derive the payment type and batch ID.
    2. Insert the payment using the appropriate EVS API.
    3. If invoice allocations are supplied, allocate as specified; otherwise, auto-allocate.
  4. Apply payment batch, if specified.
  5. Retrieve payment LAST_MODIFIED date.
  6. Return the original payment batch ID.
