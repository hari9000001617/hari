ACCOUNT_V

The ACCOUNT_V view defines the account details stored in the account cache and available for real-time aggregate checking by the rating engine.

Column Definitions

Column Name

Properties

Description

ACCOUNT_ID

NUMBER (15)

Not null

ID of an account.

ACCOUNT_NAME

VARCHAR (30)

Not null

Name for the account, as known by the customer and displayed in the user interface.

LAST_MODIFIED

DATE

Not null

Date-time when this record was last modified.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

ID of the customer node responsible for this account.

ACCOUNT_TYPE_ID

NUMBER (8)

Not null

ID of the type of account.

ACCOUNT_CURRENCY_ID

NUMBER (8)

Not null

Currency of this account.

ACCOUNT_BALANCE

NUMBER (19,3)

Not null

Derived

Current balance of this account.

For new records it should be set to 0.

It is derivable from the ACCOUNT_HISTORY table, being equal to the CURRENT_BALANCE of the highest numbered transaction entry for the account.

It is checked by the DVP.

BALANCE_DATE

DATE

Not null

Derived

Date-time when the account balance was last updated.

It is derivable from the ACCOUNT_HISTORY table, being equal to the EFFECTIVE_START_DATE of the highest numbered transaction entry for the account.

It is checked by the DVP.

CREDIT_LIMIT

NUMBER (19,3)

Redundant

Credit limit of this account (if any).

For a prime account of a customer node, this is equal to CUSTOMER_NODE_HISTORY.CREDIT_LIMIT (after any required currency conversion is performed).

Note:

It is strongly recommended that a customer node's credit limit be specified in the same currency as their prime account.

It is checked by the DVP.

UNBILLED_AMOUNT

NUMBER (19,3)

Not null

Derived

Total amount (in the currency of the account) rated by usage-based tariffs with the ACCOUNT_AGGREGATE_IND_CODE set to 1, but not yet applied to the account.

The values of UNBILLED_AMOUNT and ACCOUNT_BALANCE can be used to provide a real-time indication of the total amount owed by the customer for this account.

Note:

Customer node hierarchy account aggregations are not taken into consideration when calculating the total amount owed; that is, UNBILLED_AMOUNT can have a non-zero value for an account associated with a non-reporting node. To determine the total unbilled amount for an invoice point in a customer hierarchy, it is necessary to sum the UNBILLED_AMOUNTS of its contributory accounts.

If handling of liability accounts is enabled, a positive UNBILLED_AMOUNT value is always a debit, irrespective of the account class; otherwise, a positive UNBILLED_AMOUNT value is a debit for an asset account and a credit for a liability account.

The update_unbilled_account server utility validates the value of UNBILLED_AMOUNT and updates any discrepancies.

ACCOUNT_ACTION_CODE

NUMBER (8)

Code indicating what action was taken (if any) in relation to the unbilled amount for an account.

Possible values for this are installation-dependent, but could indicate that:

  • A service was temporarily disconnected (or is scheduled for disconnection).
  • An immediate invoice issued.
  • A warning letter or message was sent to the customer (or is scheduled to be sent).

It is always null for new accounts, and only set using the biAccountUpdate&() function, or, if specified, by the account balance function for the account type.

It is based on the ACCOUNT_ACTION reference type.

ACCOUNT_ACTION_DATE

DATE

Date-time when the ACCOUNT_ACTION_CODE was last set or cleared.

It is set automatically by biAccountUpdate&() and cannot be directly modified, except, if specified, by the account balance function for the account type.

INVOICE_UNBILLED_AMOUNT

NUMBER (22,6)

Derived

Total amount (in the currency of the account) rated by tariffs with the ACCOUNT_AGGREGATE_IND_CODE set to 1, but not yet applied to the account.

It not only includes charges generated for the account, but those in all Statement and No Reporting accounts below the account in the customer hierarchy (for which the account is the invoice point).

It can only contain a value, and must contain a value, if the:

  • Account is the prime account for a customer node.
  • Customer node has a report level of Invoice for the current date-time.

If it contains a value, and all accounts are the same currency and have the same account class, the value is equal to:

UNBILLED_AMOUNT + SUM(UNBILLED_AMOUNT where INVOICE_ACCOUNT_ID = ACCOUNT_ID)

The fix_invoice_account server utility also validates the value of INVOICE_UNBILLED_AMOUNT and updates any discrepancies.

INVOICE_ACCOUNT_ID

NUMBER (15)

Derived

Account to which all unbilled charges are rolled up at billing time if billing is performed with an effective date equal to the current date-time.

It can only contain a value, and must contain a value, if the:

  • Account is the prime account for a customer node.
  • Customer node has a report level of Statement or None for the current date-time.

Because of the above rules, it is not valid for both INVOICE_UNBILLED_AMOUNT and INVOICE_ACCOUNT_ID to be specified for the same account.

The fix_invoice_account server utility validates the value of INVOICE_ACCOUNT_ID and updates any discrepancies.

SUBTYPE_CODE

NUMBER (8)

Optional code to classify a secondary account associated with a customer node.

Entity validation specified by the account type controls possible values.

ACCOUNT_STATUS_CODE

NUMBER (8)

Optional code that defines the account status.

Entity validation specified by the account type controls possible values.

FIRST_USE_DATE

DATE

Date-time the account was first used.

It can be used for prepaid accounts to indicate when the account was first activated.

ACCOUNT_RESTRICTION_DATE

DATE

Date-time the account is due to change to a restricted mode of operation. For example, the account can only receive calls, or make calls to the call centre.

BALANCE_EXPIRY_DATE

DATE

Date-time the current balance of the account is due to expire.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10 containing configurable details for the account.

Entity validation specified by the account type controls display and meaning of the columns.

TRANSFERRED_ACCOUNT_ID

NUMBER (15)

Account to which invoice amounts for this account are rolled up at billing time.

It specifies an account outside of the current hierarchy that is responsible for paying statements for this account.

It can only contain a value, and must contain a value, if both the following apply:

  • Account is the prime account of a root customer node
  • Customer node has a report level of Transferred Statement for the current date-time.

Its account type must match this account.
