TARIFF_CACHE_V

The TARIFF_CACHE_V view lists tariff information that has been cached. This information can be retrieved using the TariffFetchById?() function.

For a specific TARIFF_ID and date-time, only a single row is returned from this view.

Column Definitions

Column Name

Properties

Description

TARIFF_ID

NUMBER (8)

Not null

ID of the tariff.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day. Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day. Time component always corresponds to one second before midnight (that is, 23:59:59).

TARIFF_NAME

VARCHAR (67)

Not null

Name of the tariff.

TARIFF_CLASS_CODE

NUMBER (2)

Class of the tariff.

It is based on the TARIFF_CLASS reference type.

If set to null, this tariff is not accessed by the RGP.

TARIFF_TYPE_CODE

NUMBER (8)

Not null

Type of tariff.

It is not used by the rating or billing engines. It is based on the TARIFF_TYPE reference type.

MASTER_TARIFF_ID

NUMBER (8)

ID of the master tariff of this tariff.

If set, it is used instead of TARIFF_ID to retrieve tariff details.

The following restrictions apply:

  • A tariff can only be a master tariff if it has a non-null CHARGE_EXPR.
  • NR_CONDITIONS and NR_CHARGE_COMPONENTS for the tariff and its master must match.
  • Tariff must have a CONTEXT_CODE and an APPLICATION_ENV_CODE greater than or equal to those of its master.
  • A master tariff can only be assigned on insert.
  • The date range of a tariff and its master must match.

If set, TARIFF_ID cannot be associated with any rows in the TARIFF_CONDITION_DEFN, TARIFF_CHARGE_DEFN, or TARIFF_CHARGE tables.

BILLABLE_IND_CODE

NUMBER (1)

Not null

If set to 1 (true), the tariff generates billable charges.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context in which this tariff operates.

It is based on the CONTEXT_TARIFF reference type.

APPLICATION_ENV_CODE

NUMBER (2)

Not null

Application environment in which the tariff operates.

It is based on the APPLICATION_TARIFF reference type.

KEY_SUBSTITUTION_EXPR

VARCHAR (255)

Expression defining the values to be substituted into the key variable.

Where the tariff has a data type of:

  • Real, it must be null.
  • Real Array, it is mandatory and must return an integer result. It defines the number of values substituted in KEY_VARIABLE_NAME.
  • Real Hash, it is mandatory and must return a string array result. It defines the string values to be substituted in KEY_VARIABLE_NAME.

KEY_VARIABLE_NAME

VARCHAR (66)

Name of the local expression variable used in tariff expressions.

It can only and must contain a value if the data type is Real Array or Real Hash.

Variable name must start with the @ character, indicating a local expression variable. It must be terminated by the appropriate type suffix:

  • & (integer) for a tariff with a data type of Real Array
  • $ (string) for a tariff with a data type of Real Hash.

DEFAULT_CHARGE_EXPR

VARCHAR (255)

Not null

Default charge expression for this tariff if none of the conditional expressions match or NR_CONDITIONS is zero.

The expression must return a real value.

If the expression references the key variable, it is evaluated for each key variable value.

CHARGE_EXPR

VARCHAR (255)

Expression for the charges associated with this tariff.

The charge components in the expression are referenced as the local variables '@charge1#' to '@charge10#' (depending on the number of charge components).

Key variable can be referenced if applicable.

Note:

For discounts, it must generate a negative value.

The following rules apply:

  • It must be null if NR_CHARGE_COMPONENTS is 0. DEFAULT_CHARGE_EXPR is used to evaluate the value of this tariff.
  • It can only be null if NR_CHARGE_COMPONENTS is 0 or 1.
  • It cannot reference charge variables greater than NR_CHARGE_COMPONENTS.
  • It must return a real result.

DEFAULT_INVOICE_TXT

VARCHAR (255)

Not null

Invoice text that will appear for each charge created by this tariff.

It is overridden by the INVOICE_TXT column in the TARIFF_CHARGE table.

CURRENCY_ID

NUMBER (8)

Currency in which the charge of tariff is calculated.

Either CURRENCY_ID or CURRENCY_EXPR must be specified, but not both.

CURRENCY_EXPR

VARCHAR (255)

Expression defining the currency to be used for any charges generated by this tariff.

If it is specified, CURRENCY_ID must be null.

It must return a positive integer value corresponding to a valid internal currency identifier.

If it:

  • References the key variable, it is evaluated for each eligible key variable value. Therefore, different components of a tariff can be in different currencies.
  • Does not reference the key variable, the expression is evaluated only once and the same currency is used for all key values.

SUPPRESS_ZERO_IND_CODE

NUMBER (1)

Not null

If set to 1 (True) and the resultant charge is calculated to be zero, no charge is recorded.

GLOBAL_EVENT_APPLY_IND_
CODE

NUMBER (1)

Not null

If set to 1 (True), it is applied for all events in the context and application environment matching the tariff's eligibility criteria.

All calculated charges are applied to the account specified by the TO_ACCOUNT_ID and FR_ACCOUNT_ID (if specified) of the charge category.

The account(s) may be overridden by the TO_ACCOUNT_ID_EXPR and FROM_ACCOUNT_ID_EXPR expressions.

SERVICE_NOT_FOUND_IND_
CODE

NUMBER (1)

If set to 1 (True), it is only applied if the service specified by the C_PARTY_ID in the normalised event was not found in the database.

It can only be set to 1 (True) if:

  • CONTEXT_CODE is set to Normalised Event.
  • APPLICATION_ENV_CODE is set to Rating.
  • GLOBAL_EVENT_APPLY_IND_CODE is set to 1.

Otherwise, it must be NULL.

CHARGE_CATEGORY_ID

NUMBER (8)

Not null

Charge category to which all charges associated with this tariff are applied.

It defines both account and general ledger details for the charge.

TO_ACCOUNT_ID_EXPR

VARCHAR (255)

Expression defining the To account to be used for any charges generated by this tariff.

It must return an integer value corresponding to a valid account ID.

FROM_ACCOUNT_ID_EXPR

VARCHAR (255)

Expressions defining the From account to be used for any charges generated by this tariff.

It must return an integer value corresponding to a valid account ID.

TO_GL_CODE_ID_EXPR

VARCHAR (255)

Expression defining the To GL code to be used for any charges generated by this tariff.

It must return an integer value corresponding to a valid GL code ID.

FROM_GL_CODE_ID_EXPR

VARCHAR (255)

Expression defining the From GL code to be used for any charges generated by this tariff.

If specified, it overrides the GL code details inferred by the charge category.

It must return an integer value corresponding to a valid GL code ID.

RECEIVABLE_TYPE_ID

NUMBER (8)

Receivable type associated with all charges generated by this tariff. It must exist for at least the date range of this tariff.

If specified, RECEIVABLE_TYPE_EXPR must be NULL.

RECEIVABLE_TYPE_EXPR

VARCHAR (255)

Expression that defines the receivable type to be used for any charges generated by this tariff.

If specified, RECEIVABLE_TYPE_ID must be NULL.

It must return an integer value corresponding to a receivable type ID.

INVOICE_SECTION_NR

NUMBER (8)

Not null

Invoice section number assigned to all charges generated by this tariff.

ACCOUNT_AGGREGATE_IND_
CODE

NUMBER (1)

If set to 1 (True) values calculated by this tariff are aggregated during rating.

It can only be set to 1 if the APPLICATION_ENV_CODE is set to Rating.

Otherwise, it must be NULL.

ACCOUNT_CHECK_EXPR

VARCHAR (255)

Expression used to check the balance of the account during rating.

It can perform actions if the balance exceeds preset amounts.

The key variable cannot be accessed by this expression.

It can only contain a value if ACCOUNT_AGGREGATE_IND_CODE is set to 1 (True).

 

Recurring Tariff Details

Column Name

Properties

Description

NORMALISED_EVENT_TYPE_
ID

NUMBER (8)

Not null

Type of normalised event generated by the RGP for rental or one-off events based on this tariff.

It also controls the list of possible values for EVENT_TYPE_CODE and ADJUSTMENT_EVENT_TYPE_CODE.

EVENT_TYPE_CODE

NUMBER (8)

Not null

Event type generated by the RGP for normalised events based on this tariff.

It is based on the EVENT_TYPE reference type.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event sub-type to be generated by the RGP for normalised events based on this tariff.

It is based on the EVENT_SUBTYPE reference type.

Valid values are dependent on EVENT_TYPE_CODE.

REAL_TIME_IND_CODE

NUMBER (1)

Read only

If set to 1 (True), it is a real-time tariff.

If the associated eligibility criteria are satisfied, an event is generated and rated for this tariff immediately, instead of being generated at billing time by the RGP.

If not set to 1, it must be NULL.
