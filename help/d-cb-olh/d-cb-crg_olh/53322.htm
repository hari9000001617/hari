DERIVED_ATTRIBUTE_TRE_V

The DERIVED_ATTRIBUTE_TRE_V view contains historical and current information about the derived attributes in Singleview.

Column Definitions

Column Name

Properties

Description

DERIVED_ATTRIBUTE_ID

NUMBER (8)

Not null

ID of the derived attribute.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its granularity is one day.

Time component always corresponds to midnight (that is, 00:00:00 hours).

EFFECTIVE_END_DATE

DATE

Not null

Date-time that the record ceases to be effective.

Its granularity is one day.

Time component always corresponds to one second before midnight (that is, 23:59:59).

DERIVED_ATTRIBUTE_NAME

VARCHAR (67)

Not null

Name of the derived attribute including its data type suffix.

Only valid variable names can be specified (see the Configuration Fundamentals Guide for Singleview Convergent Billing).

LAST_MODIFIED

DATE

Not null

Date-time that the row was last modified.

DESCRIPTION

VARCHAR (2000)

Not null

Description of the derived attribute.

OPERATOR_ID

NUMBER (8)

Not null

ID of the user who last modified this record.

GROUP_ID

NUMBER (8)

Not null

Security group associated with, and controlling access to, this record.

VERSION_STR

VARCHAR (255)

Version ID string used to tag this entity.

It is provided for use by third party configuration management tools, and is not directly used by Singleview.

CATEGORIES

VARCHAR (255)

Comma-separated list of categories associated with this derived attribute.

Refer to the CATEGORY_V view for the list of available categories.

DATA_TYPE_CODE

NUMBER (2)

Not null

Return data type of this attribute.

It is based on the DATA_TYPE_DERIVED_ATTRIBUTE reference type.

For a derived attribute with an evaluation type of 6 (List), it must be set to one of:

  • 0 (Integer)
  • 8 (Integer Array)
  • 16 (Integer Hash).

APPLICATION_ENV_CODE

NUMBER (2)

Not null

Environment types where this attribute is used.

It is based on the APPLICATION_DERIVED reference type.

If set to 6 (TRE Table), the derived attribute can only be used by the TRE server, and cannot be accessed directly.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context of this derived attribute. Valid values are:

  • Rating environment
    • 1 (Normalised Event)
    • 2 (Service)
  • Billing environment
    • 1 (Normalised Event)
    • 2 (Service)
    • 3 (Customer Node)
    • 4 (Customer)
    • 5 (Charge)
  • TRE Table environment
    • 0 (Any).

Depending on the value of APPLICATION_ENV_CODE, the possible values for this column are defined by the CONTEXT_RATING, CONTEXT_BILLING, or CONTEXT_TRE_TABLE reference type.

EVALUATION_TYPE_CODE

NUMBER (2)

Not null

Type of evaluation performed on the derived attribute.

It is based on the EVALUATION_TYPE reference type.

KEY_SUBSTITUTION_EXPR

VARCHAR (255)

Expression that defines the values to be substituted into the key variable.

It must be null if the derived attribute has a scalar data type.

If the derived attribute:

  • Has an evaluation type of 0 (Simple), it is optional; otherwise, it is mandatory.
  • Is an array, this expression must return an integer. This value is the number of integer values to be substituted into KEY_VARIABLE_NAME.
  • Is a hash, it must return a string array for substitution into KEY_VARIABLE_NAME.

KEY_VARIABLE_NAME

VARCHAR (66)

Name of the local variable to be used in subsequent derived attribute expressions.

It can only contain a value, and must contain a value, if KEY_SUBSTITUTION_EXPR is specified.

This value must start with @ and end with:

  • & if it is an array
  • $ if it is a hash.

DEFAULT_VALUE_EXPR and derived attribute index expressions can reference this variable.

STORAGE_CONTEXT_CODE

NUMBER (2)

Storage context of this derived attribute.

Depending on the value of APPLICATION_ENV_CODE, the possible values for this column are defined by the STORAGE_CONTEXT_RATING, STORAGE_CONTEXT_BILLING, or STORAGE_CONTEXT_TRE_TABLE reference type.

Valid values are:

  • 1 (Facilities)
  • 2 (Service)
  • 3 (CustomerNode)
  • 4 (Customer)
  • 5 (Global)
  • 6 (Global-Shared).

Where storage context is:

  • 1 (Facilities) or 2 (Service), CONTEXT_CODE must be set to 1, 2, or 5.
  • 3 (CustomerNode), CONTEXT_CODE must be set to 1, 2, 3, or 5.
  • 4 (Customer), CONTEXT_CODE can be set to any value.
  • 5 (Global) or 6 (Global-Shared), CONTEXT_CODE must be set to 0 (Any).

MASTER_DERIVED_ATTRIBUTE_
ID

NUMBER (8)

Master derived attribute of this derived attribute. It can only be specified on insert.

The following columns must be identical in the master and child derived attributes:

  • EVALUATION_TYPE_CODE
  • STORAGE_CONTEXT_CODE
  • TABLE_NAME
  • TABLE_NAME_DESCRIPTION
  • ENTITY_VALIDATION_ID
  • EXTENDED_IND_CODE
  • ROW_GROUP_SECURITY_IND_CODE
  • TENANT_IND_CODE
  • DISABLE_DATE_RANGING_IND_CODE.

TABLE_NAME

VARCHAR (64)

Name of the table associated with this derived attribute.

It can only be specified and is mandatory for derived attributes with one of the following evaluation types:

  • Single Range
  • Multiple (1-10) Keys
  • Best Match
  • Searching
  • List.

DISPLAY_TABLE_NAME

VARCHAR (64)

Name of the derived attribute table displayed to users.

It can be translated.

It defaults to TABLE_NAME.

TABLE_NAME_DESCRIPTION

VARCHAR (2000)

Description of the table name.

It is mandatory if TABLE_NAME is mandatory.

ENTITY_VALIDATION_ID

NUMBER (8)

ID of the entity validation record.

It is mandatory if TABLE_NAME is mandatory.

SEARCH_ENTITY_VALIDATION_
ID

NUMBER (8)

ID of the entity validation record that defines the columns in the derived attribute table on which searches can be performed.

RESULT_SEQNR

NUMBER (2)

RESULT column number from the derived attribute table.

It is mandatory for the following evaluation types:

  • Single Range
  • Multiple (1-10) Keys
  • Best Match
  • Searching.

It must be null for the following evaluation types:

  • Simple Expression
  • Single Value Returned
  • Multiple Values Returned
  • List with a storage context of Facilities.

It is optional for List evaluation types with other storage contexts.

MAX_TABLE_SIZE

NUMBER (5)

Maximum number of rows that can be entered in any table associated with this derived attribute.

It is optional if TABLE_NAME is specified; otherwise, it must be Null.

If not defined, there is no limit to the number of rows allowed.

VALUE_ATTRIBUTE_TYPE_ID

NUMBER (8)

Attribute type associated with the resulting attribute type values for this derived attribute.

If defined, value expressions must be constants and the data type of the attribute type must be the same as DATA_TYPE_CODE.

DEFAULT_VALUE_EXPR

VARCHAR (255)

Not null

Default value of this attribute if none of the conditions are satisfied or no array index is defined.

If VALUE_ATTRIBUTE_TYPE_ID is specified, this default value is not an expression but a valid value based on VALUE_ATTRIBUTE_TYPE_ID.

WILDCARD_STRING

VARCHAR (255)

String used to define a wildcard only if the evaluation type is Table Lookup by Searching.

EXTENDED_IND_CODE

NUMBER (1)

Read-only

If set to 1 (TRUE):

  • The derived attribute table can support extended features such as multi-tenancy, row-level group security, and disabled date ranging.
  • The derived attribute table rows are stored in the DERIVED_ATTR_ARRAY_EXT table instead of the DERIVED_ATTRIBUTE_ARRAY table.
  • STORAGE_CONTEXT_CODE must be set to Global or Global-Shared.

If it is not specified for a new derived attribute, it is automatically set to 1 if any of the following are set to 1:

  • ROW_GROUP_SECURITY_IND_
    CODE
  • ROW_CATEGORY_IND_CODE
  • TENANT_IND_CODE
  • DISABLE_DATE_RANGING_IND_
    CODE.

If not set to 1, it must be NULL.

ROW_GROUP_SECURITY_IND_
CODE

NUMBER (1)

Read-only

If set to 1 (TRUE):

  • The derived attribute table supports row-level group security.
  • EXTENDED_IND_CODE is set to 1.

If it is not specified for a new derived attribute, it is automatically set to 1 if any of the following are set to 1:

  • ROW_CATEGORY_IND_CODE
  • TENANT_IND_CODE.

If not set to 1, it must be NULL.

ROW_CATEGORY_IND_CODE

NUMBER (1)

Read-only

If set to 1 (TRUE):

  • The derived attribute table supports row-level group category and version information.
  • ROW_GROUP_SECURITY_IND_
    CODE and EXTENDED_IND_CODE are set to 1.

If not set to 1, it must be NULL.

TENANT_IND_CODE

NUMBER (1)

Read-only

If set to 1 (TRUE):

  • The derived attribute table is tenantable.
  • STORAGE_CONTEXT_CODE must be set to Global-Shared
  • ROW_GROUP_SECURITY_IND_
    CODE and EXTENDED_IND_CODE are set to 1.

For a tenantable derived attribute table, if the implementation is:

  • Tenanted, the derived attribute table is tenanted.

    A separate set of rows is stored for this derived attribute table for each tenant.

  • Untenanted, the derived attribute table is extended and untenanted.

If set to NULL in a tenanted implementation, all users share the derived attribute table and have view-only access.

If not set to 1, it must be NULL.

DISABLE_DATE_RANGING_IND_
CODE

NUMBER (1)

Read-only

If set to 1(TRUE):

  • The date-range splitting of this derived attribute and its derived attribute table contents is prohibited.
  • EXTENDED_IND_CODE is set to 1.

If not set to 1, it must be NULL.
