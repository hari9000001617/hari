Description

Update a row of data in a customer derived attribute array, identified by the supplied keys.

Context

The following groups of variables are available throughout all stages of processing customer derived attribute update row request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$
    • evLastRowInd&, available at all stages of processing except evaluation of customer and derived attribute pre-commit expressions.
    • evRowStatus$
  • Customer
    • Available only during customer processing stages and evaluation of customer derived attribute pre and post-commit expressions.
  • Derived attribute .
    • Available only during customer derived attribute processing stages.

Parameters

Parameter

Description

Keys?{}

Keys that identify the customer derived attribute row to update.

LastModified~

Initial value of this parameter must match the last modified date of the customer record specified by FieldValues?{}.

If not defined, represents the date-time of this update.

FieldValues?{}

Data to be updated for the customer derived attribute array row.

If an effective start state is not specified, the current date will be used. If an effective end date is not specified, the end of time value will be used.

Index values cannot be updated. To update index values, the row must be deleted and re-inserted with new values.

The data may be keyed according to column name. When keyed by column name, the data passed must match the data type and value expected by the database column.

If attribute validation is in use, data may be keyed by the interface or display label specified in the appropriate entity validation, or core defaults. When keyed by a label the data passed must match the data type and value expected by appropriate attribute type. Denormalised fields must pass denormalised string values.

IdentifierKeys?{}

Populated on successful return with the values of the CUSTOMER_NODE_DA_ARRAY identifier fields derived from Keys?{}.

 

Processing

The entity lookup function specified for customer derived attributes is called to determine the customer node ID, derived attribute ID, sequence number, and effective date for the update of the customer derived attribute array row.

If a row is not found, an error is raised. Processing continues as if fEV_CustomerDAUpdateRows&() was called for a single row, using the determined IDs and dates as parameters.

Returns

A non-zero value if update was successful; otherwise, an appropriate error.
