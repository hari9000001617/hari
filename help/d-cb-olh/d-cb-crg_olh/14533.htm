debug_on()

Syntax

Interface 1

 

Interface 2

 
debug_on()

 

 
debug_on(
    DebugLevel&)

Description

Enables function call stack debugging and the debug() function.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

DebugLevel&

In: Valid values are:

  • 0 (Disable Debug): Same as calling debug_off().
  • 1 (Light Debug): Function parameter values and return values are not included in the trace output.
  • 2 (Full Debug): Function parameter values and return values are included in the trace output.
  • 3 (Light Debug with call stack entries): Same as 1 (Light Debug) with trace output for call stack entries and exits and exits made by Singleview.
  • 4 (Full Debug with call stack entries): Same as 2 (Full Debug) with trace output for call stack entries and exits and exits made by Singleview.

DebugLevel$ is optional. Full Debug is enabled by default if the parameter is not specified.

 

Returns

1 (TRUE).

Remarks

The function does not cause compiled code to be re-parsed with debug() enabled.

The existence of a debug_on() and debug_off() statements in code is checked at parse time so that the debug() call is generated at parse time if a debug_on() statement is present earlier in the expression of function (even if EPM_FUNCTION_DEBUG is not set at the time of parsing). The following statements always cause the debug() statement to be generated at parse time and invoked at runtime:

debug_on(); debug(...); debug_off();
