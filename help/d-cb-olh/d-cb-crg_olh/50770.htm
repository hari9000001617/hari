PRODUCT_INSTANCE_ORDER_TRE_V

The PRODUCT_INSTANCE_ORDER_TRE_V view provides details about the services and equipment associated with, or should be associated with, product instances.

The details in this view cannot be directly updated.

Column Definitions

Column Name

Properties

Description

PRODUCT_INSTANCE_ID

NUMBER (15)

Not null

ID of a product instance.

LAST_MODIFIED

DATE

Not null

Date-time that this row was last modified.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

If this is a companion product instance, it must correspond to a date for the base product instance associated with the product instance.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

If this is a companion product instance, it must correspond to a date for the base product instance associated with the product instance.

CREATION_ORDER

NUMBER (4)

Not null

Order in which equipment and services should be allocated to the product instance.

SERVICE_ITEM_SEQNR

NUMBER (4)

Not null

ID of the service type within the product.

SERVICE_RANGE_MIN

NUMBER (4)

Minimum number of service instances with the specified SERVICE_ITEM_SEQNR created on an instance of the product.

The following constraints apply:

  • It is mandatory for base products, and always null for companion products.
  • The minimum value is 1 if SERVICE_ITEM_SEQNR = 1, or 0 otherwise. That is, there must be at least a single instance of the first service on a product.

SERVICE_RANGE_MAX

NUMBER (4)

Maximum number of service instances with the specified SERVICE_ITEM_SEQNR that can be created on an instance of the product.

The following constraints apply:

  • It must contain a value if SERVICE_RANGE_MIN contains a value, and must be null if SERVICE_RANGE_MIN is null.
  • If it contains a value, it must be greater than or equal to SERVICE_RANGE_MIN.

EQUIPMENT_ITEM_SEQNR

NUMBER (4)

ID of the equipment type within a service type of a product.

EQUIPMENT_RANGE_MIN

NUMBER (4)

Minimum number of items of equipment with the specified EQUIPMENT_ITEM_SEQNR that must be allocated to the corresponding service instance of the product.

The following constraints apply:

  • It is mandatory for base products, and always null for companion products.
  • The value must be greater than or equal to 0.

EQUIPMENT_RANGE_MAX

NUMBER (4)

Maximum number of items of equipment with the specified EQUIPMENT_ITEM_SEQNR that must be allocated to the corresponding service instance of the product.

The following constraints apply:

  • It must contain a value if EQUIPMENT_RANGE_MIN contains a value, and must be null if EQUIPMENT_RANGE_MIN is null.
  • If it contains a value, it must be greater than or equal to EQUIPMENT_RANGE_MIN.

SERVICE_ID

NUMBER (15)

ID of the service associated with the product instance, for this creation order.

It can only be null if UNCOMPLETED_IND_CODE is 1, or if the product instance has a status of Moved.

SERVICE_RANGE_SEQNR

NUMBER (5)

Range sequence number of the service within the product, which identifies a service within SERVICE_ITEM_SEQNR.

It can only contain a negative value if the service has a status of Cancelled.

For base product instances, it must have a value between 1 and SERVICE_RANGE_MAX for the corresponding record in the PRODUCT_SERVICE_TYPE table.

For companion product instances, the value is controlled by the corresponding set of rows of its base product instance. For example, if the user has created 5 GSM services on the base (SERVICE_RANGE_SEQNRs of 1 to 5), and the companion product definition has the GSM service, five rows exist in this table for the companion product instance, with SERVICE_RANGE_SEQNRs of 1 to 5.

A companion product cannot be associated with instance that only has some of the services within a range.

SERVICE_TYPE_ID

NUMBER (8)

Type of service required for the product instance.

SERVICE_TYPE_NAME

VARCHAR (60)

Type of service required for the product instance.

SERVICE_TYPE_DISPLAY_
NAME

VARCHAR (60)

Name of service type displayed.

It defaults to SERVICE_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

SERVICE_FORM_CODE

NUMBER (3)

Form that maintains the details of this type of service.

It is based on the SERVICE_FORM reference type.

SERVICE_ICON_ID

NUMBER (8)

Windows icon associated with this type of service.

SERVICE_NAME_EQUIPMENT_
CO

NUMBER (4)

Creation order of the item of equipment from which the service name is allocated for the service.

If not specified, the service is not allocated a unique name automatically by the system.

It defaults based on the service type.

It is optional for base products and always null for companion products.

The value must correspond to a row in the PRODUCT_SERVICE_TYPE_EQUIP table for the product and date range.

SERVICE_NAME_SERIALNR_
IND_CODE

NUMBER (1)

Contains a not null value only if SERVICE_NAME_EQUIPMENT_CO is not null.

If set to 1 (TRUE), the serial number is used to define the service name; otherwise, the network address is used.

NETWORK_NAME_EQUIPMENT_
CO

NUMBER (4)

Creation order of the item of equipment from which the network name for this service is allocated.

If not specified, the service is not allocated a network name automatically by the system.

It is optional for base products and always null for companion products.

The value must correspond to a row in the PRODUCT_SERVICE_TYPE_EQUIP table for the product and date range.

NETWORK_NAME_SERIALNR_
IND_CODE

NUMBER (1)

Contains a not null value only if NETWORK_NAME_EQUIPMENT_CO is not null.

If set to 1 (TRUE), the serial number is used to define the network name; otherwise, the network address is used.

It defaults based on the service type.

EQUIPMENT_ID

NUMBER (15)

ID of an item of equipment associated with the product instance, for this creation order.

EQUIPMENT_RANGE_SEQNR

NUMBER (4)

Range sequence number of the equipment within the product.

It identifies an item of allocated equipment within its SERVICE_ITEM_SEQNR, SERVICE_RANGE_SEQNR, and EQUIPMENT_ITEM_SEQNR.

For base product instances, it must have a value between 1 and EQUIPMENT_RANGE_MAX for the corresponding record in the PRODUCT_SERVICE_TYPE_EQUIP table.

For companion product instances, it is controlled by the corresponding set of rows of its base product instance. For example, if the user has allocated 5 GSM phones on the base (EQUIPMENT_RANGE_SEQNRs of 1 to 5), and the companion product definition has the GSM service, five rows exist in this table for the companion product instance, with EQUIPMENT_RANGE_SEQNRs of 1 to 5.

A companion product cannot be associated with an instance that only has some of the equipment within a range.

EQUIPMENT_TYPE_ID

NUMBER (8)

Type of equipment required for the CREATION_ORDER of this product instance.

EQUIPMENT_TYPE_NAME

VARCHAR (60)

Type of equipment required for the CREATION_ORDER of this product instance.

EQUIPMENT_TYPE_DISPLAY_
NAME

VARCHAR (60)

Name of equipment type displayed.

It defaults to EQUIPMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

EQUIPMENT_ICON_ID

NUMBER (8)

Windows icon associated with this type of equipment.

SEARCH_ENTITY_
VALIDATION_ID

NUMBER (8)

Optional set of labels, hint text, and defaults for the search fields to be used on the Equipment Allocation form for the allocation of this equipment to an instance of the product.

Unlike other forms, the order of the attribute types associated with this entity validation specifies the order in which the attribute types be displayed on the allocation form.

If specified, it must have a VALIDATION_TABLE_NAME of EQUIPMENT_HISTORY and a PARENT_TABLE_NAME of null or PRODUCT_INSTANCE_HISTORY.

MAX_ROWS_DISPLAYED

NUMBER (3)

Optional maximum number of rows of idle equipment to display on the Equipment Allocation form for the allocation of this equipment to an instance of the product.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

Customer node that owns this instance of the product.
