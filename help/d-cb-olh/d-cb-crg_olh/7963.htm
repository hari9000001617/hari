INV_CHARGE_OTHER_INVOICES_V

The INV_CHARGE_OTHER_INVOICES_V view:

  • Displays customer node charge details of all other invoice accounts.
  • Includes invoices with a No Reporting level.

Column Definitions

Column Name

Properties

Description

NUMBER (15)

Not null

Customer node for which this set of invoices was retrieved.

INVOICE_SEQNR

NUMBER (3)

Not null

Sequence number of a specific invoice format and invoice image for this customer node.

Refer to the INVOICE_CONTENTS tables.

SEQNR

NUMBER (8)

Not null

Sequence number (from the INV_REPORT_ACCOUNTS_T table) for defining the order in which the accounts are displayed on this customer node's invoice.

ID generated for this invoice.

BILL_RUN_ID

Not null

CHARGE

Value of this charge.

PROPORTION

Proportion of this charge that was allocated to this customer.

NUMBER (8)

Name of the subtotal that generated this charge, if any.

SUBTOTAL_DATE

DATE

Date-time calculated by the given subtotal, if this subtotal generates a date-time result.

It can only contain a value if SUBTOTAL_ID is not null and CHARGE is null.

EFFECTIVE_START_DATE

DATE

Not null

Effective start date of the tariff or subtotal associated with the call.

EFFECTIVE_END_DATE

DATE

Not null

Effective end date of the tariff or subtotal associated with the call.

INVOICE_SECTION_NR

NUMBER (8)

Invoice section number associated with this charge.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number associated with this charge.

INVOICE_TXT

VARCHAR (255)

Optional invoice text that can be displayed in association with this charge.

KEY_VALUE

VARCHAR (255)

Hash key value or array index value associated with this charge for this tariff or subtotal.

It is only populated for charges generated by hash or array tariffs and subtotals.

For hash based tariffs or subtotals, it stores the string hash key associated with this charge.

For array tariffs or subtotals, it stores the integer array index associated with this charge.

GENERAL_1 - GENERAL_20

VARCHAR (255)

General fields 1 - 20.

They can be used to store intermediate calculations and lookups performed as part of tariff evaluation during rating or billing (for example, distance band, time band, multi-time band flag).

The information contained in them is accessible by the BGP and the IGP.

Only charges associated with tariffs can populate them. They are always null for charges associated with subtotals, payments, or adjustments.

 

Charge Partition Column

Column Name

Properties

Description

PARTITION_NR

NUMBER (2)

Not null

Customer partition number associated with a customer node level charge for all other invoice accounts of the customer.

Because all charges associated with an invoice are associated with the same account (and therefore customer), when customer partitioning is implemented, all charges for an invoice have the same PARTITION_NR.

It can be used as an additional selection criterion to improve access to the CHARGE table by eliminating irrelevant CHARGE table partitions.
