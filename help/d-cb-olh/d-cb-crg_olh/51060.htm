Variables Based on INV_CUSTOMER_NODE_HISTORY_V

All direct variables based on the INV_CUSTOMER_NODE_HISTORY_V view have an application environment of Invoicing and a context of Customer Node.

Variable Definitions

Variable

Type

Description

InvAssignedOperatorId&

Integer

ID of the user assigned the customer.

InvBankAccountNumber$

String

Customer's bank account number.

InvBankAccountRef$

String

Customer's bank account reference number.

InvBankCode&

Integer

Code or number of the bank or financial institution where a customer has their account.

InvBranchCode&

Integer

Code or number that identifies the location of the bank or financial institute where a customer has their account.

InvCardExpiryDate~

Date

Date when the main or default credit card will expire.

InvCardNumber$

String

Main or default credit card number used for making payments.

InvCardTypeCode&

Integer

Code or number which identifies the credit card type used by the customer node.

InvCompanyNumber$

String

Official company or business registration number (for example, ACN).

InvCreditComments$

String

Comments relating to a customer's credit rating and the assigned limit.

It provides reasons and justification for the assigned values.

InvCreditLimitCurrencyId&

Integer

Currency for the customer's credit limit value.

Note:

It is strongly recommended that a customer node's credit limit be specified in the same currency as the customer's prime account.

InvCreditRatingCode&

Integer

Indicator for how a customer is treated regarding financial accounting practices including invoicing and payment collection.

InvCurrencyId&

Integer

Preferred currency of this customer node to be used for all invoice and statement generation.

InvCurrencySymbol$

String

Currency symbol associated with the customer.

InvCustomerActiveDate~

Date

Date that the customer became active.

InvCustomerCreatedDate~

Date

Date the customer is created.

InvCustomerCreditLimit#

Real

Customer's credit limit.

InvCustomerNodeGeneral1$ -
InvCustomerNodeGeneral10$

String

General fields 1 - 10.

InvCustomerNodeHierarchy
Level&

Integer

Level in the hierarchy of the customer node. The root node of a hierarchy has a level of 1.

InvCustomerNodeName$

String

Name of the customer node.

InvCustomerNodePersonId&

Integer

ID of the primary contact person for the customer node.

InvCustomerNodeReportLevel&

Integer

Level of reporting required for the customer. Valid values are:

  • 1 (Invoice)
  • 2 (Statement)
  • 3 (No reporting)
  • 4 (Transferred Statement).

InvCustomerNodeSeqnr&

Integer

A sequence number that identifies a specific invoice image for the customer.

InvCustomerNodeStatusCode&

Integer

Status code for the customer node. Valid values are:

  • 1 (Prospect)
  • 2 (Credit Check Required)
  • 3 (Active)
  • 4 (Cancelled)
  • 5 (Black-list).

InvCustomerNodeTypeId&

Integer

ID of the customer node type for the node.

InvCustomerNodeTypeName$

String (30)

Name of the customer node type for the node.

InvCustomerNodeTypeDisplay
Name$

String (60)

Name of the customer node type displayed. Defaults to InvCustomerNodeTypeName$.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

InvEffectiveDate~

Date

Effective date for the information.

InvIndustryCode&

Integer

Code that is used to identify what type of industry or activity the customer node is involved in.

InvParentCustomerNodeId&

Integer

ID of the parent customer node for the node.

InvPaymentLocationCode&

Integer

Identifies the preferred location where payments will be made; for example, postal delivery, third party institution such as a bank or post office, or in person at a specific or general customer service centre.

InvPaymentMethod&

Integer

ID of the official method of payment for the customer.

InvPersonBirthDate~

Date

Date of birth of the person.

InvPersonDisabilityType&

Integer

Disability type, if any, of the person.

InvPersonDriversLicence$

String

Driver's licence of the person.

InvPersonEmailAddress$

String

Standard email address of the person.

String

Marital status of the person.

InvPersonMobilePhoneNumber$

Nationality of the person.

String

InvPersonPreferredName$

String

InvPersonProfession&

Integer

Residential status of the person.

InvPersonSocialSecurityNo&

Social security number of the person.

InvPersonSpokenLanguage&

Integer

First language of the person.

Integer

Status code of the primary contact for the node. Valid values are:

  • 0 (Cancelled)
  • 1 (Prospect)
  • 2 (Subscriber)
  • 3 (Business Contact)
  • 4 (Subscriber & Business Contact)
  • 5 (VIP)
  • 6 (Bad Risk).

InvPersonTitle$

String

Title of the primary contact for the node.

InvPersonTypeId&

Integer

InvPersonWorkPhoneNumber$

String

InvPersonWrittenLanguage&

Integer

Integer

ID of the postal address used for the node.

String

General fields 1 - 10 for storing address-related information.

Numeric representation of the location of the address.

String

Optional primary ID of this customer node.

InvPrimaryIdentifier2$

String

Second optional ID of the customer node.

InvPrimeAccountId&

Integer

Primary account associated with this customer node.

InvPrimeAccountName$

String

Name of the node's primary account.

InvRegionCode&

Integer

Code or map grid reference used to identify the geographic region where the customer node is located.

The region can be based on physical address details such as suburb.

InvRootCustomerNodeId&

Integer

ID of the root customer node for the node.

InvSalesChannelCode&

Integer

Sales channel for this customer node.

InvTaxClassCode&

Integer

Real

Integer
