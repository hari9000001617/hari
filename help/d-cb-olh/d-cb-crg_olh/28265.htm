Payment Batch CBF Interface

The Convergent Billing payment batch entity has several related components that are treated as separate by the Convergent Billing API.

The purpose of the function described in this section is to supply a comprehensive set of encapsulated APIs as a functional extension of the Convergent Billing API set.

The function keeps the payment batch as the central entity. The creation of CBF APIs to update subordinate payment batch entities is superfluous as EVS provides all necessary functionality.

Local and remote versions are provided for functions. Remote versions have the same function name but include an '_R' suffix. Because the functional behaviour is the same, the functions are documented together.

Remote functions call the local function but are defined so that the remote function executes in a transactional server. Executing in a transactional server ensures that any subsequently called functions are included as part of the remote function transaction. The result is that if any function called by the remote function fails and transaction rollback is invoked, the effects of all called functions are rolled back. This ensures that the transaction either completes in its entirety, or not at all.

Note:

The local CBF APIs (for example, fCMN_TREPaymentBatchCreate&()) are designed to be called from their remote wrappers (for example, fCMN_TREPaymentBatchCreate_R&()). Local CBFs must be executed in a transactional server and cannot be called from the Expression Test form or any other non-transactional component (such as entity validation).
