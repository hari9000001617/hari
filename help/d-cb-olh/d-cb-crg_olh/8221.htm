IDL Notation

The interface to properties and methods detailed in the following sections are expressed in Interface Description Language (IDL).

The syntax is in the following format:

[operation, dispatch_id] HRESULT _stdcall
method_name([modifier, retval] PARAM_TYPE param_name)

 

The following table displays the elements of the IDL syntax.

 

IDL Field Descriptions

Field Name

Description

Possible Values

Value Description

operation

Specifies how the method operates.

If it has a value, the method acts as a property.

propget

Retrieves a value from the property.

propput

Assigns a value to the property.

dispatch_id

Dispatch ID.

It is a unique number that can be used as an alternative to the name of the method.

0x1 to 0x19

Standard methods and properties.

0x20 to 0x39

Application specific properties.

0x40 onwards

Application specific methods.

method_name

Name of the method or property.

 

 

modifier

Specifies how the parameters passed by the method are handled.

in

The controlling process (server) sends a parameter to the client process.

out

The controlling process (server) is passed a parameter from the client process.

In, out

A parameter is passed in both directions.

retval

Specifies that a value be returned.

 

 

PARAM_TYPE

Data type of parameter.

 

 

param_name

Name of the parameter.

 

 

 

Example 1

Example 1 is an example of a property definition represented in IDL:

[propget, id(0x0000002B)] HRESULT _stdcall
PropertyName([out, retval] VARIANT);

The property definition describes a read-only method called PropertyName, which is a VARIANT data type, with a hexadecimal dispatch ID of 2B.

The presence of propget indicates that the property will retrieve a value from the automation server.

Example 2

Example 2 is an example of a method definition represented in IDL:

[id(0x0000002C)] HRESULT _stdcall MethodName([in] BSTR x, [in,out] VARIANT_BOOL *y, [out] VARIANT *z);

The method definition describes a method called MethodName that has the following parameters:

  • x, which is used as a return value from the client process.
  • y, which is both exported and used for a return value from the client process.
  • z, which is used to export a value from the controlling process.

The method has a hexadecimal dispatch ID of 2C.
