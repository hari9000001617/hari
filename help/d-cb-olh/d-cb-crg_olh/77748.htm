biEntitlementPeriodicAllocBatchProcess&()

Syntax

biEntitlementPeriodicAllocBatchProcess&(
    const ScheduledFunctionParameters?[],
    const EffectiveOperators&[],
    var ErrorId&[],
    var ErrorMessage$[])

Description

Performs batch processing of scheduled functions for entitlement allocations.

Note:

It is intended to be called by EntitlementPeridocAllocBatchProcess&() and should not be called directly.

Attributes

Function Type

Remote

Service

biFnEvaluateRW

Application Environment

Any

Context

Any

 

Parameter

Description

ScheduledFunctionParameters?[]

In: Two-dimensional array of the parameters passed to each scheduled function.

EffectiveOperators&[]

In: Effective operator associated with each scheduled function.

ErrorId&[]

Out: ID of any error that occurred while processing each function.

ErrorMessage$[]

Out: Error message text of any error that occurred while processing each function.

 

Returns



Number of entitlement allocations that were successfully processed in this batch.  Details of any errors are returned in the ErrorId&[] and ErrorMessage$[] parameters. 


If the batch fails and the transaction is rolled back, an error is raised.


 


Remarks


This function is the remote interface for performing batch processing of scheduled functions for entitlement allocations.   


For each completed entitlement, it checks the information returned to track pending entitlement updates and cache reloads.  After all entitlements have been processed, it issues a single SQL update for all pending entitlement updates and a single biEntitlementReload&() call to update the entitlement cache with the changes.


In the event that an error is raised during the creation of an allocation, the function checks if a rollback is required.  If a rollback is not required, the error details are returned in ErrorId&[] and ErrorMessage$[] and the function continues processing the remaining entitlements in the batch.  If a rollback is required, the error message is wrapped in an entitlement batch failure error message and this function raises an error, causing the transaction to be rolled back.
