treEventSetHandlerx

C

TRE_RESULT treEventSetHandlerx(void (*Handler)(char *EventName, TRE_VARIANT *FieldValues))

 

Object Pascal

function treEventSetHandlerx(Handler: TtreEventHandler): TRE_RESULT

 

Perl

Not applicable.

 

Purpose

Defines the handler function in the client for receiving event notifications in the client's current context.

Parameters

Parameter

Description

Handler

In: Name of the locally-defined function used as the event handler.

The function is only called when treEventCheck() is called.

It may be called 0 or more times by a single treEventCheck() call, depending on the number of pending events to the client.

 

Returns

TRE_SUCC if successful; otherwise, TRE_FAIL. The TRE sets treErrno to indicate the type of error.

Remarks

If the Handler value is null, the handler becomes undefined. Only one handler may be called.

For the C interface, the function defines the handler function in the client for receiving event notifications in the client's current context. The Handler is only called depending on the method of event notifications specified in the flag's parameter of treConnectFlags().

For the Object Pascal interface, the function type is defined as follows:

TtreEventHandlerx = procedure (EventName: Pchar; Fieldvalues:
    PtreVariant);
    cdec1;

 

Events posted using the older functions treEventPost() and treEventPostWait() can be interpreted by this handler. The ErrorId, ErrorMsg, and five string parameters are translated to elements 0 to 6 of the FieldValues array.

The method of event notification specified when calling treConnectFlags() determines if the function is called:

  • TRE_POLL (default)

    Called as a result of polling to check for waiting messages. Can be called zero or more times at each poll interval, depending on the number of pending events to the client.

  • TRE_SIGNAL (UNIX only)

    Called immediately when the message arrives. This method is not valid when the TRE is used in multi-context mode.

  • TRE_THREAD (UNIX only)

    Called immediately when the message arrives.

Unknown hash

Class-specific attribute values, determined by the value of the TA_CLASS attribute.

The keys of the hash correspond to the names of the attributes and the value for each key is either of the following:

  • The value of the related attribute (if the related attribute has only one value in the event)
  • An array of all the values of the related attribute (if the related attribute has two or more values in the event).

If the event class does not support specific attributes, the hash is empty.
