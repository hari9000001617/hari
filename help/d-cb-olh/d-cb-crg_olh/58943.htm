biProductOfferValidateFields&()

Syntax

biProductOfferValidateFields&(
    OpCode&,
    const Fields?{})

 

Description

Validates the fields passed to biProductOfferInsert&() or biProductOfferUpdate&() functions.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

OpCode&

Operations to perform on the row.

Fields?{}

In: Hash of field names to be validated, from the PRODUCT_OFFER_TRE_V views.

 

Returns

1 if successful; otherwise, an error is raised.

Remarks

Mandatory validation is performed on the following:

  • AVAILABLE_START_DATE
  • PRODUCT_OFFER_NAME
  • DESCRIPTION.

Validation of the following conditions is performed:

  • The combination of PRODUCT_OFFER_NAME and TENANT_ID must be unique.
  • The combination of PRODUCT_OFFER_DISPLAY_NAME and TENANT_ID must be unique.
  • The combination of PRODUCT_OFFER_SKU and TENANT_ID must be unique.
  • TENANT_ID must be set to the ID of the effective user's associated tenant.

    If it does not have an associated tenant, TENANT_ID can be null or the ID of any tenant that the effective user can view.

  • CHILD_PRODUCT_OFFER_MIN must be greater than or equal to 0.
  • CHILD_PRODUCT_OFFER_MAX must be greater than or equal to CHILD_PRODUCT_OFFER_MIN.
  • COMPANION_PRODUCT_MIN must be greater than or equal to 0.
  • COMPANION_PRODUCT_MAX must be greater than or equal to COMPANION_PRODUCT_MIN.
  • CATEGORY_CODE must be a valid code from the PRODUCT_OFFER_CATEGORY reference type.
  • SUB_CATEGORY CODE must be a valid code from the PRODUCT_OFFER_SUB_CATEGORY reference type.
  • SUB_CATEGORY_CODE can only be populated if CATEGORY_CODE is not null.
  • PRODUCT_ID cannot be populated if SERVICE_SPECIFIC_IND_CODE or HIDDEN_IND_CODE is not null.
  • PRODUCT_ID must not identify a product that is:
    • Associated with a companion product that is automatically assigned using a contract type.
    • Compatible with another base product.
    • A companion product.
