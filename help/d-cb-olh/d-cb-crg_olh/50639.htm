Description

Inserts a row of data in a customer derived attribute array.

Context

The following groups of variables are available throughout all stages of processing customer derived attribute insert row request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$
    • evLastRowInd&, available all stages of processing except evaluation of customer and derived attribute pre-commit expressions
    • evRowStatus$.
  • Customer
    • Available only during customer processing stages and evaluation of customer derived attribute pre and post-commit expressions.
  • Derived attribute.
    • Available only during customer derived attribute processing stages.

Parameters

Parameter

Description

StartDate~

Date on which the insertion becomes effective.

If not specified, this parameter defaults to the current date.

FieldValues?{}

Data to be inserted into the customer derived attribute array.

Data must specify values for the primary IDs of the customer derived attribute array. If the identified derived attribute array cannot be found, an error is raised. If a sequence number is not specified, a new row is added to the end of the derived attribute array. If an effective end date is not specified, the end of time value is used.

The data can be keyed according to column name. When keyed by column name, the data passed must match the data type and value expected by the database column.

If attribute validation is in use, data can be keyed by interface or display label specified in the appropriate entity validation, or core defaults. When keyed by a label, the data must match the data type and value expected by the appropriate attribute type. Denormalised fields must pass denormalised string values.

LastModified~

Initial value of this parameter must match the last modified date of the customer record identified by FieldValues?{}.

If not defined, the last modified value in CUSTOMER_NODE_HISTORY_V for the specified StartDate~ is used.

On successful return, LastModified~ represents the date-time of this insertion.

IdentifierKeys?{}

Populated on successful return with the values of the CUSTOMER_NODE_DA_ARRAY identifier fields.

 

Processing

The entity lookup function specified for customer derived attributes is called to determine the customer node ID, derived attribute ID, sequence number, and effective date of the customer derived attribute row. If a row is not found an error is raised.

Processing continues as if fEV_CustomerDADeleteRows&() was called for a single row, using the determined IDs and dates as parameters.

Returns

A non-zero if insertion was successful; otherwise, an appropriate error.
