Normalised Event Error Detail Form

The following table lists automation parameters for the Normalised Event Error Detail form, data types, and supported functions.

The description column is provided to supplement the description in the NORMALISED_EVENT_V view, or to provide information about parameters not contained within the database view.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

A_PARTY_CARRIER_CODE

Integer

Update

Carrier from which the call arrived.

A_PARTY_CELL_ID

String

Update

A party mobile cell ID.

A_PARTY_ID

String

Update

Full, national number of the calling party (A party).

A_PARTY_LOCATION_CODE

Integer

Update

Geographic location specifying the origin of the call to the finest granularity supported by the system.

Higher forms of location, such as country and continent, can be derived from this value.

A_PARTY_NAME

String

Update

Name of the A party, if known.

A_PARTY_NETWORK_ID

String

Update

Network ID of the calling party (A party), if any.

For a GSM service, this corresponds to the IMSI number.

A_PARTY_ROUTE

String

Update

Incoming route of CDR.

A_PARTY_TON_CODE

Integer

Update

A party type of number, indicating whether it is international, local, STD, or some other type.

B_PARTY_CARRIER_CODE

Integer

Update

Carrier to which call is destined (may not be known).

B_PARTY_CELL_ID

String

Update

B party mobile cell ID.

B_PARTY_ID

String

Update

Full national number or most precise ID for the B party service.

B_PARTY_LOCATION_CODE

Integer

Update

Geographic location specifying the destination of the call to the finest granularity supported by the system.

Higher forms of location, such as country and continent, can be derived from this value.

B_PARTY_NAME

String

Update

Name of the B party, if known.

B_PARTY_NETWORK_ID

String

Update

Network ID of the called party (B party), if any.

For a GSM service, this corresponds to the IMSI number.

B_PARTY_ROUTE

String

Update

Outgoing route of CDR.

B_PARTY_TON_CODE

Integer

Update

B party type of number.

CASCADE_CARRIER_CODE

Integer

Update

Carrier closest to the current network that was used in the completion of the call.

CHARGE

Real

Update

Charge as it was received from the switch. The units are dependent on the switch.

CHARGE_START_DATE

Date

Update

Date-time used for charging purposes.

CURRENCY_ID

Integer

Update

Currency for the above charge.

C_PARTY_CARRIER_CODE

Integer

Update

Carrier associated with the chargeable party.

C_PARTY_CELL_ID

String

Update

Charge party network cell ID (for example, mobile cell).

C_PARTY_ID

String

Update

Full national number or the most precise ID for a chargeable party service.

It is usually equal to the A_PARTY_ID, but is equal to the B_PARTY_ID for reverse charge calls, 008 calls, and some other third party for conference calls.

C_PARTY_INTERNAL_ID

String

Update

ID for the chargeable party.

It is used by the Rental Generation Process (RGP) to guarantee identification of the service involved.

If this is provided, it is used instead of the C_PARTY_ID or the C_PARTY_NETWORK_ID to perform the service look-up.

C_PARTY_LOCATION_CODE

Integer

Update

Geographic location that specifies the origin of the call to the finest granularity supported by the system.

Higher forms of location, such as country and continent, can be derived from this value.

C_PARTY_NAME

String

Update

Name of the C party, if known.

C_PARTY_NETWORK_ID

String

Update

Network ID of the chargeable party (C party), if any.

For a GSM service, this corresponds to the IMSI number.

At least one of the party details (C_PARTY_ID or C_PARTY_NETWORK_ID) must be specified.

C_PARTY_ROUTE

String

Update

Incoming or outgoing route of the chargeable party.

C_PARTY_TON_CODE

Integer

Update

Charge party type of number.

DURATION

Integer

Update

Duration of call.

The units depend on event type (for example, seconds).

EVENT_CLASS_CODE

Integer

Update

General classification of the event that is used for reporting purposes. Valid values are:

  • 1 (Usage event): The event is in relation to the usage of a service.
  • 2 (Recurring event): The event represents a rental period, typically generated by the RGP.
  • 3 (One-off or ad hoc event): Events include automatically generated one-off events (generated by the RGP), and ad hoc events generated by users or other sources.

EVENT_SOURCE

String

Update

Source of the event.

EVENT_SUB_TYPE_CODE

Integer

Update

Event subtype.

EVENT_TYPE_CODE

Integer

Update

Type of event.

FULL_PATH

String

Update

Available details for the route that a call used (for example, X.25 DNIC sequence for call).

GENERAL_1 - GENERAL_20

String

Update

General fields 1 - 20.

Typical values are for the derivation of derived, normalised event attributes that need to be stored for reporting.

PERIOD_END_DATE

Date

Update

Date-time of the end of this event.

It is used for events that extend over some period of time (such as events associated with recurring charges).

PERIOD_START_DATE

Date

Update

Date-time of the start of this event.

It is used for events that extend over some period of time (such as events associated with recurring charges).

PULSES

Integer

Update

Number of meter pulses associated with call.

If populated by events generated by the rental generation process, its contents should be specified as pulses per unit duration

RATE_BAND

String

Update

Time or rate band calculated for this call (typically based on its CHARGE_START_DATE).

It may be supplied by the switch, or derived during normalisation.

An encoded value can be used.

SWITCH_START_DATE

Date

Update

Date-time on CDR as produced by the network switch equipment.

VOLUME

Integer

Update

Volume of data associated with call.

The units depend on the call type (for example, bytes or segments).

If populated by events generated by the rental generation process, its contents should be specified as volume per unit duration.
