ClientDynamicFormList&()

Syntax

ClientDynamicFormList&(
    l_ParamNames$[],
    l_ParamValues?[])

 

Description

Displays a dynamic form in the client that displays a read-only list.

Attributes

Application Environment

Data Entry

Context

Any

Method

Local

 

Parameter

Description

l_ParamNames$[]

In: Array of the parameter names to display.

l_ParamValues?[]

In: Array of the values corresponding to the parameter names.

 

Returns

ID of the button pressed if the dynamic form is set to display modally (FORM.MODAL=TRUE); otherwise, 0 (FALSE).

Remarks

The following items can also be displayed:

  • Form caption
  • Prompt label
  • Buttons (including caption).

No validation takes place on this form; it is simply used to display information for the user.

Buttons

Parameter Name

Properties

Description

BUTTON.ALIGNRIGHT

Boolean

Defaults to FALSE

Right alignment of the button.

If TRUE, the button is aligned to the right-hand edge of the form.

BUTTON.CANCEL

Boolean

Defaults to FALSE

Cancel state of the button.

If set to TRUE, pressing the button does not execute any validation expressions on the form.

The Close button on the menu bar effectively has BUTTON.CANCEL=TRUE, and returns an ID of 0.

BUTTON.CAPTION

String

Mandatory

Caption for a new button.

Setting BUTTON.CAPTION creates a new button and any further setting of BUTTON parameters act on that newly created button until BUTTON.CAPTION is set again.

BUTTON.DEFAULT

Boolean

Defaults to FALSE

Default state of the button.

If TRUE, the button is triggered when the Enter key is pressed and the cursor is in a field on the form.

BUTTON.ID

Integer

Mandatory

ID of the button.

If the form is called modally, the ID of the activated button is returned as l_ModalResult&[].

The Close button on the menu bar has an ID of 0.

BUTTON.NAVBUTTON

Boolean

Defaults to FALSE

State of the navigation button on the form.

If TRUE, the navigation button arrow is displayed on the form.

 

Form Structure

Parameter Name

Properties

Description

FORM.ALIGN

String

Specifies the position of the form.

Options are LEFT, RIGHT, TOP, BOTTOM.

Tip:

To anchor a child form to three edges of a parent form, use FORM.ALIGN and FORM.ANCHORS.

FORM.ANCHORS.BOTTOM

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchor to the bottom of the parent form.

FORM.ANCHORS.LEFT

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the left of the parent form.

FORM.ANCHORS.RIGHT

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the right of the parent form.

Tip:

To horizontally resize the form when the parent form is resized, use FORM.ANCHORS.LEFT and .RIGHT.

FORM.ANCHORS.TOP

Ignored

Resizes a child form relative to the size of the parent form.

Sets the form anchors to the top of the parent form.

Tip:

To vertically resize the form when the parent form is resized, use FORM.ANCHORS.TOP and .BOTTOM.

FORM.BRINGTOFRONT

Boolean

Specifies that the form can be displayed on top of other active forms.

FORM.CAPTION

String

Mandatory

Caption displayed on the title bar of the form.

FORM.HEIGHT

Integer

Height of the form.

FORM.HELPFORNAME

String

Specifies that the form can be used to control the Form Field Help Dump form, and be available for F1 help and form help.

FORM.LEFT

Integer

Left location of the form.

FORM.MINHEIGHT

Integer

Minimum height of the form.

FORM.MINWIDTH

Integer

Minimum width of the form.

FORM.MODAL

Boolean

Defaults to TRUE

Specifies whether the form is displayed modally.

FORM.POSNFORMCODE

String

Creates a unique static value key used to identify the form and store its position in the system registry.

Note:

The string supports a maximum character length of 12 characters.

FORM.PROMPT

String

Text displayed in the prompt label.

FORM.REFRESH.INTERVAL

Integer

Refresh interval for the dynamic form.

The interval is only implemented for the web browser.

Note:

An interval of 0 does not activate the refresh timer.

FORM.REUSEFORMCODE

String

Code that identifies the form.

The form is reused if all of the following conditions are met:

  • FORM.REUSEFORMCODE is set.
  • The reuse flag is set (using ClientSetReuse&).
  • Another call is received to display a form with the same reuse form code.

If a form is reused, only parameters displayed after this parameter are used in the form.

FORM.TOP

Integer

Top location of the form.

FORM.WIDTH

Integer

Width of the form.

 

List Area

Parameter Name

Properties

Description

LIST.ADD_DISPLAY_FIELD

String

Display field to add to the list.

This parameter can be used as an alternative to LIST.DISPLAY_FIELDS.

LIST.COLUMN_CLICK_
SORTING

Boolean

Defaults to FALSE

Specifies how lists are sorted.

If set to TRUE, clicking on a column in the list sorts the list by that column.

LIST.DATA

Mandatory

Array

Displays a two-dimensional array of results in the grid.

LIST.DISPLAY_FIELDS

Mandatory

String

Fields to display in the grid.

The field names are separated using the \r\n characters (carriage return and linefeed).

LIST.ENTITY_NAME

String

Allows the Details button to display the form for this entity when the list displays items all of the same entity type (for example, REFERENCE_TYPE).

LIST.ENTITY_NAME_
FIELDNAME

String

When the list displays items of different entity types, this value can be set to the field that contains the entity name.

LIST.FIELDS

Mandatory

Array

Displays a two-dimensional array of field information in the grid.

Each field requires the following three elements: field name, display label for the user, and display size.

All fields passed in to LIST.DATA must be accounted for by LIST.FIELDS, but they do not need to be displayed in the list (using the LIST.DISPLAY_FIELDS parameter).

Note:

The display size parameters are overridden if any of the column-sizing parameters are set (for example, LIST.SIZE_COLS_TO_GRID_WIDTH).

LIST.GROUP_ID_FIELDNAME

String

ATLANTA_GROUP_ID of the record.

The Details button is disabled if the current user does not have group privileges to view this item.

LIST.INSTANCE_ID_
FIELDNAME

String

Main field ID in the resulting list.

If the Details button is pressed, this field is used as the ID to display the selected entity.

LIST.INSTANCE_NAME_
FIELDNAME

String

User-friendly field name for the Details button hint text.

LIST.SAVE_COL_SIZES

Boolean

Defaults to FALSE

Column width settings.

If set to TRUE, column sizes are persistent and reused when the form is next opened.

LIST.SAVE_COL_SIZES relies on FORM.POSNFORMCODE being set.

LIST.SHOW_ASSIGN

Boolean

Defaults to TRUE

Specifies whether the Assign button is displayed on the form.

If set to TRUE, the form displays the Assign button.

LIST.SIZE_COLS_TO_GRID_
WIDTH

Boolean

Defaults to FALSE

Specifies how columns are displayed in the grid.

If set to TRUE, column widths adjust so all columns are displayed in the grid (without requiring the horizontal scroll bar).

Tip:

Before this parameter is activated, ensure that it will not make the columns too narrow and obstruct data in the rows.

LIST.SIZE_ROWS_TO_DATA

Boolean

Defaults to FALSE

Specifies how column width is affected by text size.

If set to TRUE, each row in the grid automatically resizes itself, and the text in the cells word wraps to the available width of the cell.
