BatchInbound

This interface type is used to configure file loading logic.

Note:

Users of this interface type must have insert and update security group access level for:

  • ED_AUTO, the security group associated with automatically-generated external definition functions.
  • IFD_AUTO, the security group associated with automatically-generated interface definition functions.

 

Interface Type Parameters

Tab

Parameter

Type

Description

File Process Manager

Processing Directory

[Text]

Path of the directory where files to be loaded are saved.

It can contain environment variables.

File Pattern

[Text]

UNIX shell file name pattern used to select the files to be processed.

File Group

[Text]

UNIX group associated with the file.

If specified and Local Server File Secure is set to Yes, the created directories and processed files use File Group as their UNIX group. The directory's group is set only on creation of the directories, and the file's group is set at the start of the file processing.

For more information, refer to the System Configuration Guide for Singleview Convergent Billing.

Action on Success

[List]

Action to perform on the processed file and, if File Availability Method is set to Control File, the control file, if the file is successfully processed.

Valid values are:

  • Delete File

    File and, if applicable, the control file, are deleted.

  • Rename File

    File and, if applicable, the control file, are renamed to the name specified by Archive File Pattern.

  • Rename and Move File

    File and, if applicable, the control file, are renamed to the name specified by Archive File Pattern and moved to the directory specified by Archive Directory.

It is based on the BIMFPMIN_SUCCESS_ACTION_TYPE reference type.

Archive Directory

[Text]

Name of the Singleview file system directory into which files are moved if processing is successful.

The parameter:

  • Can contain environment variables.
  • Is mandatory if Action on Success is set to Rename and Move File; otherwise, it is ignored.
  • Is created if it does not exist.

Archive File
Pattern

[Text]

Name to which the file is renamed.

Mandatory and only applicable if Action on Success is set to one of the following:

  • Rename File
  • Rename and Move File.

If File Availability Method is set to Control File, the file name pattern is applied to the processed file and the control file.

The file name pattern:

  • Must contain the <filename> tag, which is the file name of the processed file including the extension.
  • Can contain the following tags:
    • <basename>: file name of the processed file excluding the extension.
    • <task>: ID of the task that processed the file.
    • <date>: current date in the format yyyymmdd.
    • <datetime>: current date-time in the format yyyymmddhhnnss.

Action on Error

[List]

Action to perform on the processed file and, if File Availability Method is set to Control File, the control file, if an error occurs during processing.

Valid values are:

  • None

    File and, if applicable, the control file, are unchanged.

  • Rename File

    File and, if applicable, the control file, are renamed to the name specified by Error File Pattern.

  • Rename and Move File

    File and, if applicable, the control file, are renamed to the name specified by Error File Pattern and moved to the directory specified by Error Directory.

  • Error Records File

    A new file containing the error records is generated in the directory specified by Error Directory.

It is based on the BIMFPMIN_ERROR_ACTION_TYPE reference type.

Error Directory

[Text]

Name of the Singleview file system directory into which files can be moved if an error occurs during processing.

The parameter:

  • Can contain environment variables.
  • Is mandatory if Action on Error is set to Rename and Move File; otherwise, it is ignored.
  • Is created if it does not exist.

Error File Pattern

[Text]

Name to which the errored file is renamed.

Only applicable if Action on Error is set to one of the following:

  • Rename File
  • Rename and Move file.

The error file name pattern is applied to the processed file and to the control file, if File Availability Method is set to Control File.

The error file pattern:

  • Must contain the <filename> tag: the file name of the processed file and, if applicable, the control file, including the extension.
  • Can contain the following tags:
    • <basename>: file name of the processed file excluding the extension.
    • <task>: ID of the task that processed the file.
    • <date>: current date in the format yyyymmdd.
    • <datetime>: current date-time in the format yyyymmddhhnnss.

Batch Loader

Transaction
Control

[List]

Group of records processed as one transaction.

Valid values are:

  • Record: Performs one transaction for each record that is processed. Equivalent to having Records per Transaction set to 1.
  • File: For each file processed, all records in the file are processed as one transaction. The Records per Transaction field is ignored.
  • Run: All records for all processed files are processed as one transaction.
  • Records per Transaction: Records are processed as defined in the Records per Transaction field.

The default value is Records per Transaction.

It is based on the BIMBPIN_TRANSACTION_CONTROL_TYPE reference type.

Records per Transaction

[Integer]

Number of records processed as one transaction when Transaction Control is set to Records per Transaction.

By default, it is set to 1000. If undefined, it is treated as set to 0. Valid values are:

  • 0: Records are processed in the same way as if Transaction Control is set to Run.
  • 1: Records are processed in the same way as if Transaction Control is set to Record.
  • Greater than 1: The specified number of records are processed as one transaction, and committed at the end of the each processed file.

If the batch loading process encounters an error, the current transaction is rolled back.

Transaction Timeout (s)

[Integer]

Maximum time, in seconds, that a batch loader transaction can be active.

After this time, the transaction fails.

The value must be set depending on the values of Transaction Control and Records per Transaction.

Child Processes

[Integer]

Number of child processes that are used to process a file.

Each record in a file can potentially be processed in a different child process.

It is applicable only if Transaction Control is set to Record, or Records per Transaction is set to 1.

File Process Manager Static

File Processing Order

List

Order in which the files are processed if multiple files match the specified file pattern.

Valid values are:

  • Last Modified

    Files are processed in ascending date order (oldest to most current).

  • Name

    Files are processed in ascending alphabetical order.

Based on the BIMFPMIN_FILE_PROCESS_ORDER_TYPE reference type.

File Availability Method

List

Method used to confirm that the file is ready to be loaded into Singleview.

Valid values are:

  • Last Modified

    The period of time since the file was last modified is greater than the value specified in Last Modified Wait Period.

  • Control File

    A control file is present and, if Checksum is specified, the content matches the computed file checksum.

  • Filename

    The file name matches the specified File Pattern.

It is based on the BIMFPM_FILE_COMPLETENESS_TYPE reference type.

Last Modified Wait Period (s)

[Integer]

Amount of time in seconds that must have elapsed since a file was last modified before it is made available for processing.

If File Availability Method is set to Last Modified, Last Modified Wait Period is mandatory; otherwise, it is not applicable.

Control File Pattern

[Text]

Name of the control file.

It is only applicable if File Availability Method is set to Control File.

The file name pattern can contain the following tags:

  • <basename>: file name of the processed file excluding the extension.
  • <filename>: file name of the processed file including the extension.

File Type

List

File data group classification (for example, Payments).

Checksum

[Text]

Checksum algorithm used on the processed file to validate the control file content.

It is based on the CHECKSUM_TYPE reference type.

Local Server File Secure

Yes/No

Specifies if UNIX directory and file permissions are to be set for files on the local server file system.

The directory's permissions are set to allow only Singleview and users in the UNIX group specified by File Group to access the directory and its files.

Directory group and permissions are set only on creation.

File group and permissions are set at the start of file processing.

Allow Duplicates

Yes/No

Specifies if duplicate file names are allowed for file processing.

If set to Yes, the same file name can be reprocessed.

Batch Loader Static

Audit

Yes/No

Enables auditing on the batch loading.

If it is enabled, external definition operations are performed with auditing enabled.

Header

[List]

Specifies if the file to be processed contains a file header. Valid values are:

  • No Header (file has no header).
  • Skip Header (file header is ignored).
  • Use Header (file has a header and is used for validation and ordering of data to be loaded).

File Format

List

Format of the processed file.

It is based on the FILE_FORMAT_TYPE reference type.

Custom Format Function

[Text]

Name of the custom formatting function used to read a custom-formatted file.

Contents of the file are read one line at a time.

It is only used if File Format is set to Custom.

The custom-formatting function must match the syntax of the <CustomLoadFormatFuncName>&() function.

Character
Encoding

List

Character set of the processed file.

Encoding rules apply.

It is based on the BIMBP_CHARACTER_ENCODING_TYPE reference type.

Error Threshold

[Integer]

Number of errored records allowed for the run; when it is reached, the batch load process terminates.

It is applicable only if Error Filter Function is specified; otherwise, the batch load process behaves as if Error Threshold is set to 1 (process terminates at the first error).

If it is undefined or set to 0, unlimited errored records are allowed.

Skip Threshold

[Integer]

Number of skipped records allowed for the run.

When it is reached, the batch load process terminates.

It is applicable only if Error Filter Function or Record Filter Function is specified.

If it is undefined or set to 0, unlimited skipped records are allowed.

Record Retry
Delay(s)

[Integer]

Minimum number of seconds that the batch load process waits, before retrying a record that failed with an error code that is compatible with retries.

It is applicable only if Error Filter Function is specified.

Record Retry Limit

[Integer]

Parameter that is passed to Error Filter Function to allow the function to control the number of retry attempts.

It is applicable only if Error Filter Function is specified.

Record Filter Function

[Text]

Name of a custom filter function used to filter record data before loading the file into Singleview.

The function must be of an integer type.

If it is not defined, the batch load operation is attempted.

It is applicable only if Transaction Control is set to Record, or Records per Transaction is set to 1.

The record filter function must match the syntax of the <CustomLoadRecordFilterFuncName>&() function.

Error Filter
Function

[Text]

Name of a user-defined error function used to filter errors into a category to be processed by the batch load.

The function must return an integer value.

If it is not specified, the batch load process behaves as if Error Threshold is set to 1 (process terminates at the first error).

It is applicable only if Transaction Control is set to Record, or Records per Transaction is set to 1.

The user-defined error function must match the syntax of the <CustomLoadErrorFilterFunc>&() function.

External Definition Static

Operation

List

External definition operation to be performed when the file is loaded.

It is based on the BIMBPIN_ED_OPERATION_TYPE reference type.

Namespace

List

ID of the external definition namespace.

External Definition

List

External definition used to load the data.

The external definition must:

  • Be specific for input.
  • Be valid for the selected Operation.
  • Belong to the specified Namespace.
