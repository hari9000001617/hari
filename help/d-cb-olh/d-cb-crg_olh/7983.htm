NORMALISED_EVENT_FILE_V

The NORMALISED_EVENT_FILE_V view is used by the ERO process to perform threshold-driven behaviour during rating.

Column Definitions

Column Name

Properties

Description

NORMALISED_EVENT_FILE_ID

NUMBER (15)

Not null

ID of a normalised event file.

FILENAME

VARCHAR (255)

Not null

If the file was processed on the rating machine, the name of the file (excluding the directory component).

If the file is a logical file (corresponding to a logical grouping of records passed to Convergent Billing by a third party), the name is defined by the remote system.

The name should be unique system-wide and is usually based on the system date-time.

ATLANTA_OPERATOR_ID

NUMBER (8)

Not null

User who initiated the processing of the file.

TASK_QUEUE_ID

NUMBER (15)

ID of the task that rated the file, if applicable.

NORMALISED_EVENT_FT_ID

NUMBER (8)

Not null

Normalised event file type associated with the file and specifies the mechanism by which the events in the file were decoded.

Note:

It can only be NULL if INPUT_EVENT_COUNT is 0.

PROCESS_START_DATE

DATE

Not null

Date-time at which events from the file commenced to be processed.

PROCESS_END_DATE

DATE

Date-time at which all events from the file were processed.

EVENT_SOURCE

VARCHAR (40)

Not null

ID of the source from which the group of normalised events originated.

RATER_VERSION

VARCHAR (20)

Version of the rating engine that rated the file (for example, 4.00.07).

EXTERNAL_INPUT_EVENT_
COUNT

NUMBER (10)

Number of events that should have been processed in association with the file.

It can be defined in the file header, or specified by the external event feed. For some logical grouping of normalised events, it may not be specified.

When rating has completed, it should equal INPUT_EVENT_COUNT. If not equal, the DVP reports a warning that the file may be corrupted.

INPUT_EVENT_COUNT

NUMBER (10)

Not null

Redundant

Number of events that have been processed in association with the file.

It is calculated and updated each time the record is updated during rating of the file.

Its value is the sum of DECODED_EVENT_COUNT and CORRUPT_EVENT_COUNT.

DECODED_EVENT_COUNT

NUMBER (10)

Not null

Redundant

Number of events from the file decoded successfully during normalisation.

An event is considered to be decoded when the values of its data attributes have been assigned successfully to direct variables associated with the NORMALISED_EVENT_V view.

It is calculated and updated each time the record is updated during rating of the file.

The value is the sum of DISCARDED_EVENT_COUNT, RATED_EVENT_COUNT, UNRATED_EVENT_COUNT, NORM_ERROR_EVENT_COUNT, RATING_ERROR_EVENT_COUNT, and TRACER_EVENT_COUNT.

CORRUPT_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file that could not be decoded during normalisation.

An event is considered to be decoded when the values of its data attributes have been assigned successfully to direct variables associated with the NORMALISED_EVENT_V view.

It is updated each time the record is updated during rating of the file.

DISCARDED_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file discarded during normalisation or discarded after being rated.

It is updated each time the record is updated during rating of the file.

RATED_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file successfully rated.

It is updated each time the record is updated during rating of the file.

UNRATED_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file flagged by an ENM process as not to be rated, and are stored in the NORMALISED_EVENT table but have no associated charges.

It is updated each time the record is updated during rating of the file.

TRACER_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file flagged as tracer events by an ENM process.

It is updated each time the record is updated during rating of the file.

NORM_ERROR_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file that could not be normalised by an ENM process because of an error condition.

It is updated each time the record is updated during rating of the file.

RATING_ERROR_EVENT_COUNT

NUMBER (10)

Not null

Number of events from the file that could not be rated by one or more ERT processes because of an error condition.

It is updated each time the record is updated during rating of the file.

REPROCESSED_EVENT_COUNT

NUMBER (10)

Not null

Number of successfully rated events that have been reprocessed and deleted from the NORMALISED_EVENT table, and no longer directly associated with the file.

It is set to 0 by the rating engine.

REPROCESSED_ERROR_EVENT_
COUNT

NUMBER (10)

Not null

Number of error events that have been reprocessed and deleted from the NORMALISED_EVENT_ERROR table, and no longer directly associated with the file.

It is set to 0 by the rating engine.

DELETED_EVENT_COUNT

NUMBER (10)

Not null

Number of successfully rated events that have been deleted (with their charges) and not reprocessed.

It is used to track revoking of rental events and rental adjustment events associated with a given customer hierarchy.

It is set to 0 by the rating engine.

DELETED_ERROR_EVENT_
COUNT

NUMBER (10)

Not null

Number of error events that have been deleted and not reprocessed.

It may contain a value if an event is deleted using the Convergent Billing client or schedule types, or revoking of errored rental events and errored rental adjustment events associated with a given customer hierarchy.

It is set to 0 by the rating engine.

NEF_EXPORT_ID

NUMBER (15)

ID of an export file that contains events exported from this file.

It can only be populated for complete records if PROCESS_END_DATE is specified.

EXPORTED_EVENT_COUNT

NUMBER (10)

Number of events associated with the export file exported to the file specified by NEF_EXPORT_ID.

It can only be populated if NEF_EXPORT_ID is specified.

The value must be less than or equal to the sum of RATED_EVENT_COUNT and UNRATED_EVENT_COUNT.

EXPORTED_CHARGE_COUNT

NUMBER (10)

Number of charges associated with the events exported to the export file specified by NEF_EXPORT_ID.

It can only be populated if NEF_EXPORT_ID is specified.

EXPORTED_RECORD_COUNT

NUMBER (10)

Number of records exported to the export file specified by NEF_EXPORT_ID.

It can only be populated if NEF_EXPORT_ID is specified.

The value is the sum of EXPORTED_CHARGE_COUNT and the number of exported unrated events.

MIN_CHARGE_START_DATE

DATE

Minimum charge start date for normalised events from this file.

It is not updated for revoked, archived, or reprocessed events.

If the sum of RATED_EVENT_COUNT and UNRATED_EVENT_COUNT is greater than zero, it contains a value.

MAX_CHARGE_START_DATE

DATE

Maximum charge start date for normalised events from this file.

It is not updated for revoked, archived, or reprocessed events.

If the sum of RATED_EVENT_COUNT and UNRATED_EVENT_COUNT is greater than zero, it contains a value.
