DISPUTE_TRE_V

The DISPUTE_TRE_V view provides details about monetary disputes raised against an invoice or account.

Column Definitions

Column Name

Properties

Description

DISPUTE_ID

NUMBER (15)

Not null

ID of the dispute.

DISPUTE_NR

VARCHAR (60)

Not null

ID of the dispute that can be referenced by users.

LAST_MODIFIED

DATE

Not null

Date-time the record was last modified.

OPERATOR_ID

NUMBER (8)

Not null

ID of the user who last modified this record.

GROUP_ID

NUMBER (8)

Not null

Security group with access to the dispute record.

If not specified for a new dispute, it defaults to the security group associated with the customer node for the dispute.

ACCOUNT_ID

NUMBER (15)

Not null

ID of the account against which the dispute has been raised.

INVOICE_ID

NUMBER (15)

ID of the invoice against which the dispute has been raised.

It must be applied to ACCOUNT_ID, and be an invoice against a primary account, or a statement against a secondary account.

PERSON_ID

NUMBER (15)

ID of the contact person who raised the dispute.

Any correspondence that relates to the dispute is addressed to this contact person. The person record must exist for at least the duration of the dispute record.

QUERY_ID

NUMBER (15)

ID of the customer query associated with the dispute.

EXTERNAL_OBJECT_ID

NUMBER (15)

Reference object in an external schema (for example, a Process Engine case or issue) that created this dispute.

OPEN_DATE

DATE

Not null

Date-time that the dispute was added to the system.

If not specified, it defaults to current date-time.

CLOSE_DATE

DATE

Not null

Date-time at which the dispute was resolved.

ITEMISED_DISPUTE_IND_CODE

NUMBER (1)

If set to 1 (TRUE), the dispute contains a set of itemised charges recorded in the DISPUTE_CHARGE_TRE_V view.

If not set to 1 (TRUE), it must be NULL.

ORIGINAL_DISPUTED_AMOUNT

NUMBER (19,3)

Not null

Original amount in dispute.

It cannot be specified for itemised disputes.

For itemised disputes, the value is automatically derived from the sum of the ORIGINAL_DISPUTED_AMOUNT of its itemised charges and any specified ORIGINAL_ADJUST_AMOUNT.

Note:

A positive amount indicates a credit to the customer's account.

ORIGINAL_ADJUST_AMOUNT

NUMBER (19,3)

Not null

Read-only

Adjustment amount based on the set of itemised charges specified, including applicable taxation or discounts.

It is dependent on configuration and is based on the account type.

DISPUTE_AMOUNT

NUMBER (19,3)

Not null

Amount in dispute while its status is Open.

For non-itemised disputes, it defaults to ORIGINAL_DISPUTED_AMOUNT if not specified when inserting a dispute. This value can change on investigation of the dispute.

For itemised disputes, it is automatically derived from the sum of the itemised charges in dispute, and cannot be specified.

ADJUST_AMOUNT

NUMBER (19,3)

Not null

Read-only

Adjustment amount based on the set of itemised charges specified, including taxation or discounts that are applicable to the set of disputed charges.

It is dependent on configuration and is based on the account type.

DISPUTE_STATUS_CODE

NUMBER (8)

Not null

Status of the dispute.

If set to 1 or 2 and the dispute is not itemised, DISPUTE_AMOUNT must be greater than zero.

If set to 3, DISPUTE_AMOUNT must be zero.

It is based on the DISPUTE_STATUS reference type.

DISPUTE_CLASSIFICATION_
CODE

NUMBER (8)

Classification code to indicate the type of dispute.

It is based on the DISPUTE_CLASSIFICATION reference type.

DISPUTE_RESOLUTION_CODE

NUMBER (8)

Resolution code to indicate how the dispute was resolved.

It is based on the DISPUTE_RESOLUTION reference type.

DISPUTE_CREATION_TXT

VARCHAR (4000)

Description of the reason for raising the dispute.

DISPUTE_RESOLUTION_TXT

VARCHAR (4000)

Description of the dispute resolution.

ADJUSTMENT_TYPE_ID

NUMBER (8)

ID of the adjustment type generated if this dispute is resolved.

If not specified, it defaults to the account type.

ADJUSTMENT_ID

NUMBER (15)

ID of the adjustment generated as part of the acceptance of the dispute.

The adjustment can be generated using configuration, or automatically generated on acceptance of the dispute, based on the specified adjustment type.

It contains a value only if DISPUTE_STATUS_CODE is set to 2 (Accepted).
