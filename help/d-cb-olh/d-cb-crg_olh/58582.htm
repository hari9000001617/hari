fEV_ProdInstComplete&(

Description

Completes the creation of a product instance from the specified startDate~.

This function is similar to, and wraps the biProductInstanceComplete&() function.

Context

The following groups of variables are available throughout all stages of processing a product instance complete request, except where otherwise specified:

  • System client
  • Operator
  • Miscellaneous
    • evFormCode$ contains the value PRODSVR.
    • evLastRowInd& contains the value 0 during all stages of processing, except for the evaluation of the product instance pre-commit expressions.
    • evRowStatus$ contains the value Update.
  • Product instance
  • Product instance equipment

    The product instance equipment variables may only be available during the product instance equipment processing stages.

  • Product instance service

    The product instance service variables may only be available during the product instance service processing stages.

  • Facility group instance

    The facility group instance variables may only be available during the facility group instance processing stages.

  • Companion product instance

    The companion product instance variables may only be available during the companion product instance processing stages.

Parameters

Parameter

Description

startDate~

Date on which the completion of this product instance first becomes effective.

prodInstId&

ID of the product instance being completed.

A product instance with the specified ID must exist in the database for the specified start date; otherwise, an error is raised

lastModified~

Initial value of lastModified~ must match the value in the pre-existing record matching the specified prodInstId& and startDate~.

On successful return, the value is the date-time of this completion.

fromAccountTypeId&[]

If the return value is 0, this array contains an array of IDs of account types for which a matching account is required to be able to successfully complete the creation of this product instance.

fromAccountId&[]

Array of IDs of the accounts to be used for completion of this product instance.

callParams?{}

Hash of user-defined data for entity validation.

It is assigned to the evsCallParams?{} global variable for evaluation.

A key of SAVE_CHARGE_FAIL_ENTL_IND_CODE with value 1 can be specified to indicate that entitlements that have real-time activation or recurring charges are to be saved even if subscriber has insufficient funds (no entitlement allocation is created in this case).

 

Processing

Assuming that all processing stages are enabled, the following is the order of evaluation of expressions and constraints before completion of the product instance:

  1. Bundle direct variables if the call is nested.
  2. Decrypt any encrypted values and mask any appropriate values.
  3. Complete the product instance.
  4. Evaluate product instance pre-commit expressions.
  5. Evaluate product instance service pre-commit expressions for each service in creation order.
  6. Evaluate product instance equipment pre-commit expressions for each equipment item in creation order.
  7. Evaluate facility group pre-commit expressions for each facility.
  8. Evaluate companion product pre-commit expressions for each companion product instance whose status has been updated.

If evProdProductOfferId& is defined and the evProdCmpDoNotSyncId& variable is not TRUE, the following sequence occurs:

  1. Insert associated entitlements for activated companion products.

    While inserting entitlements, if the companion product instance status is:

    • Not Active, insert entitlements with SUSPENDED_IND_CODE as 1.
    • Active, insert entitlements with SUSPENDED_IND_CODE as NULL.
  2. Synchronise entitlements of previously activated companion product instances with a non-cancelled status for re-completion.
  3. For each companion product instance and each associated entitlement, attempt to generate reservations for real-time activation and recurring tariffs if the reservation status of the tariff matches the product's initial status.

On performing the completion, the following sequence occurs:

  1. Evaluate product instance service in-transaction expressions for each service in order of creation.
  2. Queue and evaluate product instance service post-commit expressions for each service in order of creation.
  3. Evaluate product instance equipment in-transaction expressions for each item in order of creation.
  4. Queue and evaluate product instance equipment post-commit expressions for each item in order of creation.
  5. Evaluate facility group in-transaction expressions.
  6. Queue and evaluate facility group post-commit expressions.
  7. Evaluate companion product in-transaction expressions for each companion product instance whose status has been updated.
  8. Queue and evaluate companion product post-commit expressions for each companion product instance whose status has been updated.
  9. Evaluate product instance in-transaction expressions.
  10. Queue and evaluate product instance post-commit expressions.

After completion and evaluation of post-commit expressions:

  1. Unbundle variables that were stored in step 1 before completion.

Returns

A non-zero value on success, and zero if account information is required to successfully complete this product instance; otherwise an error is raised.

Remarks

The product instance is updated from a status of Product Creation to the configured initial status specified in the product definition.

A product instance can only be completed when all required equipment is allocated and all required services have been created or allocated; otherwise, an error is raised.
