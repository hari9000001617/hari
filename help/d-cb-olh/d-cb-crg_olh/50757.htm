ADJUSTMENT_TRE_V

The ADJUSTMENT_TRE_V view defines the set of columns that can be specified to the biAdjustment TRE services.

Column Definitions

Column Name

Properties

Description

ADJUSTMENT_ID

NUMBER (15)

Not null

Read-only

ID of the adjustment.

LAST_MODIFIED

DATE

Not null

Read-only

Date-time that the record was last modified.

ADJUSTMENT_NR

VARCHAR (30)

Not null

Read-only

ID of the adjustment that can be referenced by users.

If not specified, it is automatically generated based on the ADJUSTMENT_FORMAT_EXPR of the adjustment type and the ADJUSTMENT_ID.

ADJUSTMENT_BATCH_ID

NUMBER (15)

Read-only

ID of adjustment batch.

It is mandatory when inserting adjustments associated with a batch.

ADJUSTMENT_TYPE_NAME

VARCHAR (40)

Not null

Read-only

Name of the adjustment type.

If the adjustment is associated with a batch, this adjustment type defaults to the adjustment type of the batch.

If ADJUSTMENT_TYPE_ID is specified, it cannot be specified.

ADJUSTMENT_TYPE_DISPLAY_NAME

VARCHAR (60)

Not Null

Name of adjustment type displayed.

It defaults to ADJUSTMENT_TYPE_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

ADJUSTMENT_TYPE_ID

NUMBER (8)

Not null

Read-only

ID of the adjustment type.

If the adjustment is associated with a batch, it defaults to the adjustment type of the batch.

If ADJUSTMENT_TYPE_NAME is specified, it cannot be specified.

ADJUSTMENT_STATUS_CODE

NUMBER (4)

Not null

Read-only

Indicates status of the adjustment.

Values can be specified for a status when inserting a new adjustment.

If not specified, status defaults to:

  • 0 (Entered): For all other adjustments.
  • 1 (Applied): For new individual adjustments with an ADJUSTMENT_RECEIVED_DATE <= ADJUSTMENT_DATE and sufficient detail to apply the new adjustment to an invoice (for more information, refer to biAdjustmentInsert&() in the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2)).
  • 3 (Future Dated): For new individual or batch adjustments with an ADJUSTMENT_RECEIVED_DATE > ADJUSTMENT_DATE.

It is based on the ADJUSTMENT_STATUS reference type.

DESCRIPTION

VARCHAR (255)

Not Null

Description of the adjustment.

Note:

It is not displayed on the invoice, but is entered by the user. If this adjustment is associated with a batch, this defaults to the description of the batch.

FROM_ACCOUNT_NAME

VARCHAR (30)

Read-only

Account from which the adjustment is deducted.

If not specified, it usually defaults based on the adjustment type.

If the adjustment type specifies a From account, the type of this account must match the type specified.

If FROM_ACCOUNT_ID is specified, it cannot be specified.

FROM_ACCOUNT_ID

NUMBER (15)

Read-only

Account from which the adjustment is deducted.

If not specified, it usually defaults based on the adjustment type.

If the adjustment type specifies a From account, the type of this account must match the type specified.

If FROM_ACCOUNT_NAME is specified, it cannot be specified.

FROM_GL_CODE_NAME

VARCHAR (64)

GL code associated with the adjustment against FROM_ACCOUNT_ID.

It can be specified if FROM_ACCOUNT_ID is NULL.

It can only be updated if the adjustment was not applied.

If FROM_GL_CODE_ID is specified, it cannot be specified.

FROM_GL_CODE_ID

NUMBER (8)

GL code associated with the adjustment against FROM_ACCOUNT_ID.

It can be specified if the FROM_ACCOUNT_ID is NULL.

It can only be updated if the adjustment was not applied.

If FROM_GL_CODE_NAME is specified, it cannot be specified.

ADJUSTMENT_TEXT

VARCHAR (80)

Not Null

Text displayed on the customer invoice in the summary section with the details of the invoice adjustment.

It defaults to the text associated with the adjustment type.

TO_ACCOUNT_NAME

VARCHAR (30)

Not null

Read-only

Account to which the adjustment applies.

If the adjustment type specifies a To account, the type of this account must match the type specified.

If associated with a batch adjustment, the currency of the account must be the same as the currency associated with the batch adjustments.

If TO_ACCOUNT_ID is specified, it cannot be specified.

TO_ACCOUNT_ID

NUMBER (15)

Not null

Read-only

Account to which the adjustment applies.

If the adjustment type specifies a To account, the type of this account must match the type specified.

If associated with a batch adjustment, the currency of the account must be the same as the currency associated with the batch adjustments.

If TO_ACCOUNT_NAME is specified, it cannot be specified.

TO_GL_CODE_NAME

VARCHAR (64)

GL code to be associated with the adjustment.

TO_GL_CODE_ID defaults to the GL code associated with the adjustment type, but it can be modified by the user.

If TO_GL_CODE_ID is specified, it cannot be specified.

TO_GL_CODE_ID

NUMBER (8)

GL code to be associated with the adjustment.

It defaults to the GL code associated with the adjustment type, but it can be modified by the user.

If TO_GL_CODE_NAME is specified, it cannot be specified.

ADJUSTMENT_INVOICE_CODE

NUMBER (2)

Not null

Read-only

Summary of the details in the ADJUSTMENT_INVOICE table for the TO_ACCOUNT_ID for this adjustment.

It is based on the PAYMENT_INVOICE reference type and accepts all possible values in the reference type.

It is not usually specified for inserts, but the following values are permitted for calls to biAdjustmentInsert&():

  • 0 (Invoice), if an INVOICE_ID is also specified and the specified invoice has no receivable type breakdown.
  • 2 (Unallocated), if AUTO_ALLOCATE_IND_CODE is not specified. In this case, the adjustment is treated as a single credit amount to the account with no receivable type allocations.
  • 3 (Adjustment), if ALLOCATED_ADJUSTMENT_ID is also specified. The adjustment is against another adjustment; it is usually a debit note allocated to a credit note or the opposite.
  • 4 (Payment), if PAYMENT_ID is also specified. The adjustment is against a payment; it is usually a debit note allocated to a payment.

If AUTO_ALLOCATE_IND_CODE is passed to biAdjustmentInsert&() as TRUE, it must not be specified, and is automatically derived.

INVOICE_ID

NUMBER (15)

Read-only

If the ADJUSTMENT_INVOICE_CODE is 0 (Invoice), it is the ID of the single invoice in the ADJUSTMENT_INVOICE table for the adjustment.

ALLOCATED_ADJUSTMENT_ID

NUMBER (15)

Read-only

If the ADJUSTMENT_INVOICE_CODE is 3 (Adjustment), it is the ID of the single adjustment in the ADJUSTMENT_INVOICE table.

However, to simplify interfaces, the ALLOCATED_ADJUSTMENT_ID can be specified for calls to biAdjustmentInsert&(), provided ADJUSTMENT_INVOICE_CODE is also specified with a value of 3 (Adjustment).

Refer to ADJUSTMENT_INVOICE_CODE for further restrictions.

PAYMENT_ID

NUMBER (15)

Read-only

If the ADJUSTMENT_INVOICE_CODE is 4 (Payment), it is the ID of the single payment in the ADJUSTMENT_INVOICE table.

However, to simplify interfaces, the PAYMENT_ID can be specified for calls to biAdjustmentInsert&(), provided that ADJUSTMENT_INVOICE_CODE is also specified with a value of 4 (Payment).

Refer to ADJUSTMENT_INVOICE_CODE for further restrictions.

It is used only for producing debit and credit notes.

OPERATOR_ID

NUMBER (8)

Not null

Read-only

User who entered the adjustment.

CURRENCY_SYMBOL

VARCHAR (5)

Not Null

Currency of the adjustment.

If not specified for a new adjustment, it defaults to the currency of the adjustment type if specified; otherwise, it defaults to the currency of the adjustment To account.

If the adjustment type specifies a currency, it must be set to the adjustment type currency. Batch adjustments that specify a currency must match this column and be able to be converted to the currency specified in TOTAL_AMOUNT_CURRENCY_ID.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

If CURRENCY_ID is specified, it cannot be specified.

CURRENCY_ID

NUMBER (8)

Not Null

Currency of the adjustment.

If not specified for a new adjustment, it defaults to the currency of the adjustment type if specified; otherwise, it defaults to the currency of the adjustment To account.

If the adjustment type specifies a currency, it must be set to the adjustment type currency. Batch adjustments that specify a currency must match it, and be able to be converted to the currency specified in TOTAL_AMOUNT_CURRENCY_ID.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

If CURRENCY_NAME is specified, it cannot be specified.

CURRENCY_CONVERSION_DATE

DATE

Not Null

Date-time used when converting amounts between the adjustment currency and the currency used in the To or From account.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

AMOUNT

NUMBER (19,3)

Not Null

Amount of the adjustment in the currency of the adjustment.

A positive adjustment indicates a credit to the customer that must be included on the invoice; for example, the customer had difficulties with their service in the last invoice period so the service rental charge for that invoice is being refunded.

A negative amount indicates a debit. The currency of the adjustment amount must match the currency of the account.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

FROM_AMOUNT

NUMBER (19,3)

Negative of the adjustment amount in the currency of the From account.

It must be specified if FROM_ACCOUNT_ID is specified and not specified if FROM_ACCOUNT_ID is not specified.

If not specified when updating and any of CURRENCY_ID, AMOUNT, or CURRENCY_CONVERSION_DATE has changed, it is recalculated.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

FROM_TOTAL_AMOUNT

NUMBER (19,3)

Read-only

Adjustment amount against the From account.

It equals FROM_ACCOUNT + FROM_ADJUST_AMOUNT.

TO_AMOUNT

NUMBER (19,3)

Not Null

Adjustment amount in the currency of the To account.

If not specified on insert, it is calculated from AMOUNT using CurrencyConversion#.

If not specified when updating and any of CURRENCY_ID, AMOUNT, or CURRENCY_CONVERSION_DATE has changed, it is recalculated.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

TO_TOTAL_AMOUNT

NUMBER (19,3)

Not null

Read-only

Adjustment amount against the To account.

It equals TO_ACCOUNT + TO_ADJUST_AMOUNT.

ADJUSTMENT_DATE

DATE

Not null

Read-only

Date-time that the adjustment was made.

ADJUSTMENT_RECEIVED_DATE

DATE

Not Null

Date-time that the adjustment is applied to the account.

It allows users to backdate adjustments.

If it is greater than the current date-time, ADJUSTMENT_STATUS_CODE must have a value of 3.

It can only be updated if the adjustment was not applied.

CUSTOMER_QUERY_ID

NUMBER (15)

Optional reference to the customer query that caused the adjustment to occur.

AUTO_ALLOCATE_IND_CODE

NUMBER (1)

Pseudo-column

If specified, automatic allocation of the adjustment amount to invoices and receivable types is performed.

IGNORE_MAX_ADJUST_IND_CODE

NUMBER (1)

Pseudo-column

If specified, MAX_ADJUSTMENT_AMOUNT is ignored.

FROM_ADJUST_GL_CODE_NAME

VARCHAR (64)

GL code associated with adjustment amounts against the From account.

It defaults from the adjustment type and can only be updated if the adjustment was not applied.

If FROM_ADJUST_GL_CODE_ID is specified, it cannot be specified.

FROM_ADJUST_GL_CODE_ID

NUMBER (8)

GL code associated with adjustment amounts against the From account.

It defaults from the adjustment type, and can only be updated if the adjustment was not applied.

If FROM_ADJUST_GL_CODE_NAME is specified, it cannot be specified.

TO_ADJUST_GL_CODE_NAME

VARCHAR (64)

GL code associated with adjustment amounts against the To Account.

TO_ADJUST_GL_CODE_ID defaults from the adjustment type, and can only be updated if the adjustment was not applied.

If TO_ADJUST_GL_CODE_ID is specified, it cannot be specified.

TO_ADJUST_GL_CODE_ID

NUMBER (8)

GL code associated with adjustment amounts against the To account.

It defaults from the adjustment type, and can only be updated if the adjustment was not applied.

If TO_ADJUST_GL_CODE_NAME is specified, it cannot be specified.

FROM_ADJUST_AMOUNT

NUMBER (19,3)

Amount that the From account is to be adjusted to allow for discrepancies.

An amount of zero indicates a discrepancy does not exist.

If FROM_ACCOUNT_ID is specified, it must be specified.

If FROM_ACCOUNT_ID is not specified, it cannot be specified.

If not specified during an update to FROM_AMOUNT, it is automatically recalculated.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

TO_ADJUST_AMOUNT

NUMBER (19,3)

Not Null

Amount that the To account is to be adjusted to allow for discrepancies.

An amount of zero indicates a discrepancy does not exist.

If not specified during an update to TO_AMOUNT, it is automatically recalculated.

It can only be updated if the adjustment was not applied. Updating it may cause a change in the total amount of the adjustment.

INVOICE_FORMAT_NAME

VARCHAR (30)

Name of the adjustment note invoice format.

It is mandatory if PENDING_GENERATION_IND_CODE or IMAGE_GENERATED_IND_CODE is specified.

It is used for credit or debit notes that need to be sent to customers independently of the standard invoice cycle.

If INVOICE_FORMAT_ID is specified, it cannot be specified.

INVOICE_FORMAT_DISPLAY_NAME

VARCHAR (60)

Name of invoice format displayed.

It defaults to INVOICE_FORMAT_NAME.

Caution:

The value can be translated into a locale-specific language and therefore differ between implementations.

INVOICE_FORMAT_ID

NUMBER (10)

ID of the adjustment note invoice format.

It is mandatory if PENDING_GENERATION_IND_CODE or IMAGE_GENERATED_IND_CODE is specified.

It is used for credit or debit notes that need to be sent to customers independently of the standard invoice cycle.

If INVOICE_FORMAT_NAME is specified, it cannot be specified.

ENABLE_AGING_IND_CODE

NUMBER (1)

If set to 1 (True), unallocated amounts associated with an adjustment are aged according to the account type aging rules of TO_ACCOUNT_ID.

Aging cannot be enabled for transfer adjustments.

It can only be set to 1 if FROM_ACCOUNT_ID is NULL.

If set to 1, the ADJUSTMENT_DUE_DATE must be specified.

If not set to 1, it must be set to NULL.

ADJUSTMENT_DUE_DATE

DATE

Date-time at which the adjustment was due.

It can only have a value if ENABLE_AGING_IND_CODE is set to 1.

It is usually only set for adjustments representing debit notes.

PENDING_GENERATION_IND_CODE

NUMBER (1)

If set to 1 (True), the adjustment is awaiting adjustment image generation (or regeneration).

IMAGE_GENERATED_IND_CODE

NUMBER (1)

If set to 1 (True), an image is generated for the adjustment.

It cannot be specified as part of an Insert or Update operation.

It is populated automatically when the adjustment image is generated or regenerated.

PENDING_PRINT_IND_CODE

NUMBER (1)

If set to 1 (True), the image for the adjustment is awaiting printing (or reprinting).

It can only be set to 1 if an adjustment image was generated or an image is pending generation.

PRINTED_IND_CODE

NUMBER (1)

Read-only

If set to 1 (True), the image for the adjustment was printed or emailed.

When the image is printed (or reprinted), it is automatically set to 1.

STORED_IMAGE_TYPE_CODE

NUMBER (2)

Read-only

Type of image stored in ADJUSTMENT_IMAGE, derived from the value stored in INVOICE_FORMAT_ID at image creation.

It is based on the OUTPUT_IMAGE_TYPE reference type, and set by biAdjustmentImageGenerate&().

ADJUSTMENT_IMAGE

BLOB

Read-only

Generated image for the adjustment, formatted in accordance with the invoice format specified in INVOICE_FORMAT_ID.

It is set by biAdjustmentImageGenerate&().

DISPUTE_ID

NUMBER (15)

ID of the dispute for which the adjustment was made.

If specified, the dispute must have a status of 1 (Accepted) and reference this adjustment.

It can only be specified for adjustments with a status of 1 (Applied).

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields that can be defined in the type definition table.

The constraints for each field are defined by the entity validation associated with its adjustment type.
