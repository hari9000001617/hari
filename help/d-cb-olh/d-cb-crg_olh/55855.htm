SOAPInbound

The SOAPInbound interface type is used for generating configuration settings for inbound interfaces defined using external definitions. The interface type is used to generate an inbound SOAP interface to be used by the Web Services Adapter.

Static SOAP

Config File Template

Text

Used for interface types where the configuration is based on a template file.

The default path is $ATA_REL_SERVER_CONFIG/tresoap/tresoap.xml.

Note:

This file can be customised by implementation teams. If so, it should be renamed and copied to an implementation-specific directory. For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

Disable Attachments?

List

Enables or disables SOAP with Attachments (SwA) in the SOAP message.

Default is Default.

If set to:

  • Yes, SwA is disabled
  • No, SwA is enabled
  • Default, SwA is defined by the enableSwA parameter in the axis2.xml configuration file.

THP Group

[List]

THP group to associate with the SOAP interface configuration in a clustered environment. It is used to determine the THP to use on each instance when the SOAP configuration is generated.

If Cluster Managed is set to True, this field is mandatory. Otherwise, it must not be set.

Cluster Managed

List

Indicates if this instance is managed by a high availability configuration, and determines the number of TRESOAP and REMOTE_SERVICE configuration items generated.

Default is False.

If set to:

  • False, only one of each configuration item is created.
  • True, one of each configuration item is created for each instance in the InstanceStatus derived attribute table.

THP SSL Verify

[List]

If set to True, the security certificate provided by the THP must be verified.

Note:

Applicable only if SSL_CONFIGURATION and SSL_ENABLED are specified in the THP configuration item.

Host IP Address

[Text]

IP address of the SOAP host.

This is used in high availability implementations to identify the virtual IP address (VIP) used by this SOAP interface.

Authentication realm for Basic and Digest authentication schemes.

HTTP Authentication Password File

Authentication password file, containing inbound user name/password for Basic and Digest authentication schemes.

If the authentication scheme is not None, this parameter must be set.

Note:

For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

SSL

Keystore Location

[Text]

File path to the keystore that contains the identification certificate.

Note:

For multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

If no file path is configured, $ATA_DATA_SERVER_CONFIG/tresoap<seqnr>/tresoap.keystore is used by default.

If the keystore name is supplied without a path, the $ATA_DATA_SERVER_CONFIG/tresoap<seqnr> path will be used by default.

The keystore can be created using the keytool utility that is included in the Java standard edition SDK (see the Security and Auditing Configuration Guide for Singleview).

Keystore Password

[Text]

Password used to access the keystore.

Key Password

[Text]

Password used to access the key in the keystore.

Truststore Location

[Text]

File path to the truststore that contains the trusted certificates.

Note:

In multi-instance environments, this file needs to be accessible by all instances, that is, a common path on all environments. This can be an environment variable that resolves to the same directory on each node.

This parameter can be specified as an environment variable.

The truststore can be created using the keytool utility that is included in the Java standard edition SDK (see the Security and Auditing Configuration Guide for Singleview).

Truststore Password

[Text]

Password used to access the truststore.

Client Authentication?

Yes/No

If Yes is specified, client trust must be established using the truststore when a connection is established.

Static Service Specific

Service Name

[Text]

Service name and group used by the adapter.

 

 

External Definition

[List]

Name of external definition associated with the service group.

Security Mode

[List]

Mode of security to use. Valid values are:

  • Administrative (integration performed as a Singleview super user)
  • Fixed (integration performed with a specified user name and password)
  • Dynamic (integration performed with the initiating operator's user name and password).

Username Token Required

Yes/No

If Yes is specified, a user name token is required.

This parameter must be set to Yes if Security Mode is set to Dynamic.

Verification Function

[Text]

Function used to verify the user name token. The function must have the following syntax:

<FunctionName>&(
    var controlInfo?{},
    const username$,
    const password$)

Function Prefix

[Text]

Prefix used to map interface functions to the service.

If Link WSDL to External Definition schema? is set to Yes, it is a sub-string of '<NS_PREFIX>:WSA<ED_NAME>.' starting from the beginning of <NS_PREFIX>.

If Service Name has more than one instance of the same value, the defined Function Prefix values must be the same for those instances.

If interface functions are generated using an external definition, the prefix is the external definition namespace name.

The':' prefix is used to ensure that the external definition namespace is not interpreted as an XML schema namespace in the generated Web Service Definition Language (WSDL).

Description

[Text]

Description of the external definition.

Auditing Enabled

Yes/No

If Yes is specified, external definitions auditing is enabled.

Link WSDL to External Definition schema?

Yes/No

If Yes is specified, when a service is added, a schema file is generated in the directory specified by $ATA_DATA_SERVER_CONFIG/tresoap<TRESOAP Sequence Number>.

The schema file has the name SOAPServiceName.xsd.

Use qualified default namespaces?

Yes/No

If Yes is specified, the external definitions specified in the External Definition field use default qualified namespaces, that is, no prefix.
