ScheduledFunctionUpdate&()

Syntax

ScheduledFunctionUpdate&(
    ScheduledFunctionId&,
    var LastModified~,
    FieldNames$[],
    FieldValues?[])

 

Description

Updates a scheduled function.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

ScheduledFunctionId&

In: ID of the scheduled function.

LastModified~

In: Last modified date-time of the scheduled function to be updated.

The date-time records must be in the same order as the ScheduledFunctionId&[] records.

Out: Last modified date-time of the updated scheduled function.

FieldNames$[]

In: List of fields from the SCHEDULED_FUNCTION_TRE_V view whose values are to be updated.

FieldValues?[]

In: Values for the fields in the same order that their names are specified in the FieldNames$[] array.

 

Returns

Returns 1 if successful; otherwise, an error is raised.

Remarks

The following restrictions apply to scheduled function updates:

  • If a scheduled function has status of Running or Success, it cannot be updated.
  • If SCHEDULED_START_DATE is provided, the value cannot be more than 15 minutes in the past from the current date-time.
  • If RETRY_COUNT is provided:
    • It must be greater than or equal to zero.
    • And it is greater than zero, RETRY_PERIOD must also be greater than zero.
  • If LAST_ERROR_MESSAGE_ID is provided, LAST_ERROR_MESSAGE must be provided.
  • EFFECTIVE_OPERATOR_ID can only be specified as a value different from the current operator if the current operator has appropriate privileges.
  • If FUNCTION_NAME is provided with no PARAMETERS, the existing PARAMETERS is overwritten to Null.

The following SCHEDULED_FUNCTION_TRE_V view fields cannot be updated:

  • INTERFACE_NR
  • SCHEDULED_FUNCTION_ID
  • LAST_MODIFIED
  • OPERATOR_ID
  • OPERATOR_LAST_MODIFIED
  • CREATED_OPERATOR_ID
  • STATUS_CODE
  • PARTITION_NR
  • ORIGINAL_SCHEDULED_START_DATE
  • ORIGINAL_RETRY_COUNT
  • FIRST_START_DATE
  • START_DATE
  • ELAPSED_TIME
  • EVALUATION_COUNT
  • TOTAL_ELAPSED_TIME
  • LAST_ERROR_DATE
  • SEQUENTIAL_IND_CODE
  • XID
  • SFM_CONFIG.

The following table lists default values used for SCHEDULED_FUNCTION_TRE_V view fields on update.

 

Default Values on Update

Field

Defaults to

LAST_MODIFIED

current_date()

ATLANTA_OPERATOR_ID

treOperatorGetId()

OPERATOR_LAST_MODIFIED

LAST_MODIFIED

INTERFACE_NR

If FUNCTION_NAME is changed, it is derived from the FUNCTION_NAME and the number of PARAMETERS.

This combination must match an existing function interface.

STATUS_CODE
  • 5 (Failure (Retries pending)), if STATUS_CODE is set to 4 (Failure) and either of the following is true:
    • SCHEDULED_START_DATE is updated.
    • RETRY_COUNT is updated with a non-zero value.
  • 4 (Failure), if any of the following are true:
    • LAST_ERROR_MESSAGE_ID is updated, and STATUS_CODE is set to 1 (Pending), and EVALUATION_COUNT, ELAPSED_TIME, and TOTAL_ELAPSED_TIME are equals 0.
    • RETRY_COUNT is updated to 0, and STATUS_CODE is 5 (Failure (Retries Pending)).

PENDING_START_DATE

SCHEDULED_START_DATE, if STATUS_CODE is set to 1 (Pending) or 5 (Failure (Retries Pending)).

LAST_ERROR_DATE

current_date(), if LAST_ERROR_MESSAGE_ID is updated.

 

Caution:

If called in a non-transactional server, it is the caller's responsibility to either commit or roll back the database changes performed by this function.
