QUOTE_EVENT_TRE_V

The QUOTE_EVENT_TRE_V view provides details of events and charges associated with a temporary invoice generated as part of a quote request.

Column Definitions

Column Name

Properties

Description

INVOICE_ID

NUMBER (15)

Not null

Quote invoice ID associated with the charge, if any.

CHARGE_ID

NUMBER (15)

Not null

Charge record associated with the normalised event.

CUSTOMER_NODE_ID

NUMBER (15)

Temporary customer node associated with the charge, if any.

It is present for all charges calculated by the billing engine (irrespective of context).

NODE_NAME

VARCHAR (200)

Not null

Name associated with the customer node.

LAST_MODIFIED

DATE

Not null

Date-time that the charge was last modified.

INVOICE_EFFECTIVE_DATE

DATE

Effective date of the bill run that generated this invoice.

Miscellaneous Details

Column Name

Properties

Description

CHARGE_START_DATE

DATE

Date-time for charging purposes.

It is usually derived from the switch date.

CHARGE_DATE

DATE

Not null

Date-time for the calculation of the charge.

PERIOD_START_DATE

DATE

Date-time of the start of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

PERIOD_END_DATE

DATE

Date-time of the end of this event.

It is used for events that extend over a period of time (for example, events associated with recurring charges).

EVENT_CLASS_CODE

NUMBER (2)

General classification of the event for reporting purposes.

Valid values are:

  • 1 (Usage event): The event is in relation to the usage of a service
  • 2 (Recurring event): The event represents a rental period, typically generated by the RGP.
  • 3 (One-off or ad hoc event): These include automatically generated one-off events (generated by the RGP), and ad hoc events generated by users or other sources.

It is based on the EVENT_CLASS reference type.

EVENT_TYPE_CODE

NUMBER (8)

Type of event.

It is based on the EVENT_TYPE reference type.

EVENT_SUB_TYPE_CODE

NUMBER (8)

Event subtype.

It is based on the EVENT_SUBTYPE reference type.

DURATION

NUMBER

Duration of call.

The units depend on event type.

VOLUME

NUMBER

Volume of data associated with call.

The units depend on the call type (for example, bytes or segments).

If populated by events generated by the RGP, its contents should be specified as volume per unit duration.

PULSES

NUMBER (8)

Number of meter pulses associated with call.

If populated by events generated by the RGP, its contents should be specified as pulses per unit duration.

RATE_BAND

VARCHAR (40)

Time or rate band that was calculated for this call.

 

Event General Columns

Column Name

Properties

Description

GENERAL_1 - GENERAL_20

VARCHAR (4000)

General fields 1 - 20 containing normalised event data.

Their use is controlled by the normalised event type.

Typical values are for the derivation of derived normalised event attributes that need to be stored for reporting, or decoding of location codes.

 

Charge Details

Column Name

Properties

Description

CHARGE

NUMBER (22,6)

Value of this charge.

PROPORTION

NUMBER (22,6)

Proportion of the total charge that this charge corresponds to.

It only contains a value if the charge is associated with a tariff and is usually 1.

CURRENCY_ID

NUMBER (8)

ID of the currency in which this charge was made.

It is mandatory for all charges, but not required for subtotals that are aggregations of non-currency amounts.

TARIFF_ID

NUMBER (8)

Tariff, discount, or tax that generated this charge, if any.

TARIFF_NAME

VARCHAR (33)

Name of the tariff, discount, or tax that generated this charge, if any.

TARIFF_TYPE_CODE

NUMBER (8)

Numeric code that defines the type of the tariff.

It is based on the TARIFF_TYPE reference type.

TARIFF_TYPE_NAME

VARCHAR (60)

Abbreviation corresponding to the TARIFF_TYPE_CODE.

SUBTOTAL_ID

NUMBER (8)

Subtotal, or total calculation that generated this charge.

SUBTOTAL_NAME

VARCHAR (30)

Name of the subtotal that generated this charge, if any.

TARIFF_OR_SUBTOTAL_NAME

VARCHAR (33)

Not null

Name of the tariff or subtotal for each event level charge.

DONT_DISPLAY_IND_CODE

NUMBER (1)

Indicator that the charge should not be displayed.

CONTEXT_CODE

NUMBER (2)

Not null

Processing context associated with the tariff or subtotal. Valid values are:

  • 1 (Normalised Event)
  • 5 (Charge).

It is based on the CONTEXT_SUBTOTAL reference type.

INVOICE_SECTION_NR

NUMBER (8)

Invoice section number associated with this charge.

INVOICE_ITEM_NR

NUMBER (8)

Invoice item number associated with this charge.

INVOICE_TXT

VARCHAR (255)

Optional invoice text that can be displayed in association with this charge.

KEY_VALUE

VARCHAR (255)

Hash key value or array index value associated with this charge for this tariff or subtotal.

It is only populated for charges generated by hash or array tariffs and subtotals.

For:

  • Hash-based tariffs or subtotals, it stores the string hash key associated with this charge.
  • Array tariffs or subtotals, it stores the integer array index associated with this charge.

 

Charge General Columns

Column Name

Properties

Description

CHARGE_GENERAL_1 -
CHARGE_GENERAL_20

VARCHAR (255)

General fields 1 - 20 containing charge data.

They can be used to store intermediate calculations and lookups performed during rating (for example, distance band, time band, multi-time band flag).

The information is accessible by the Bill Generation Process (BGP) and the Invoice Generation Process (IGP).
