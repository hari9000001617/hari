SCMStats?{}()

Syntax

SCMStats?{}(
    CacheName$)

 

Description

Returns current statistics for the specified cache.

Attributes

Application Environment

Any

Context

Any

Method

Local

 

Parameter

Description

CacheName$

Name of the cache for which statistics are to be returned.

 

Valid values for CacheName$ are listed in the following table.

 

Values for CacheName$

Value

Description

Service

Cache of details of each service, indexed by service identifier (ID).

ServiceName

Cache of IDs of services, indexed by service name.

NetworkName

Cache of IDs of services, indexed by service network name.

ServiceDA

Cache of Derived Attribute Tables and Companion Product instances for services.

 

Returns

Cache statistics for the specified cache.

Remarks

This function is used by the cache statistics gathering function biCacheMonitor&() when logging information to the TREMON process.

The following table lists statistics returned.

 

Statistics Returned by SCMStats?{}

Key

Type

Description

Created

Date

Date-time the cache was created.

Size

Integer

Total size of this cache in bytes, including the cache header.

PoolSize

Integer

Size of the memory pool of the cache in bytes.

This is the amount of memory available for storing data.

Its value equals the value specified for the relevant attribute in the SERVICE_CACHE configuration item.

Accesses

Integer

Number of attempts to retrieve an item.

Hits

Integer

Number of retrieval attempts that found the required item.

Inserts

Integer

Number of items inserted (includes replacements).

Updates

Integer

Number of items updated.

Only the ServiceLoad&() function causes existing items to be updated.

Replacements

Integer

Number of times the least-recently-used (LRU) item has had to be purged from the cache and replaced by a new item.

Purges

Integer

Number of explicit purges of items.

TotalData

Integer

Total data currently cached in bytes that is the sum of the sizes of all stored items.

It excludes indexing and management overheads.

Searches

Integer

Total number of internal searches for an item.

HitTraversals

Integer

Total number of blocks scanned before the item was found.

MissTraversals

Integer

Total number of blocks scanned before it is determined that the item is not in the cache.

ChainedBlockInserts

Integer

Number of additional blocks inserted into the cache due to the insertion of large items (over 8 KB) whose allocation has been split over multiple blocks.

It is included only if this number is not zero.

ChainedBlockDeletes

Integer

Number of additional blocks deleted due to the purging or replacement of large items (over 8 KB) whose allocation has been split over multiple blocks.

It is included only if this number is not zero.

PoolAvailSize

Integer

Amount of current free space in the cache in bytes.

On initial creation, this is 24 bytes greater than PoolSize.

PoolMinAvailSize

Integer

Amount of memory that is kept free to reduce fragmentation of free blocks.

Any allocation request that causes free memory to fall below this value causes one or more replacements to occur first. The requested memory is then allocated.

For the ServiceDA cache, this value equals one-tenth of the value of PoolSize. For every other cache this value is zero.

PoolUsedSize

Integer

Amount of space allocated to data in bytes.

The equation PoolUserSize + PoolAvailSize = PoolSize + 24 always holds.

The value returned includes memory management and indexing overheads. These overheads are 64 bytes per cached item for all caches except the ServiceName cache, for which the overhead is 88 bytes per item.

The ServiceName cache has two indexes per cached item, whereas all other caches have a single index per item.

PoolMaxUsedSize

Integer

Maximum value of PoolUsedSize since the cache was first created or last flushed.

PoolAllocatedBlockCnt

Integer

Number of used blocks.

The number of used blocks is equal to:

Inserts + ChainedBlockInserts - (Purges + Replacements + ChainedBlockDeletes).

PoolFreeBlockCnt

Integer

Number of free blocks.

On initial creation, there is one free block with a size equal to PoolAvailSize.

As blocks are allocated and freed, the number of free blocks increase due to fragmentation of the free memory.

Neighbouring free blocks are automatically coalesced where possible to reduce this impact.

PoolTotalNrSuccAllocs

Integer

Total number of successful memory allocations in the pool.

It should equal Inserts.

PoolTotalNrFailAllocs

Integer

Total number of failed allocations in the pool.

Failed allocations can occur due to insufficient contiguous free memory of the required size. They result in LRU items being purged until enough free contiguous space is available.

It should generally equal Replacements.

PoolTotalAllocatedBytes

Integer

Total number of bytes allocated from the pool (since the cache was last flushed) for storing data.

This may be less than the sum of the sizes of all blocks ever allocated since there may be some wastage per block.  For example, 1020 bytes from a 1024-byte block may be allocated to store a data item; the remaining four bytes are wasted.

PoolTotalWastedBytes

Integer

Total number of bytes wasted in allocation from the pool since the cache was last flushed.

For more information on wastage refer to the description for PoolTotalAllocatedBytes.

PoolMaxAllocationSize

Integer

Number of bytes in the largest-ever allocation from the pool (since the cache was last flushed).

PoolMinAllocationSize

Integer

Number of bytes in the smallest-ever allocation from the pool (since the cache was last flushed).

PoolBlockListStats

Array

Array of statistics for the lists of free and used blocks in the memory pool.

The pool contains several lists of free blocks.

Each element in the array is a hash that contains statistics aggregated from one or more of these lists.

A sample value for PoolBlockListStats is:

, { FreeBytes => 183328, MaxBytes => 4096, MinBytes => 2049, NrFreeBlocks => 68, NrUsedBlocks => 162, TotalAllocatedBytes => 1074924484, TotalNrSuccAllocs => 380083, TotalWastedBytes => 24639348, UsedBytes => 485664 }

where the keys in each of the hashes of this array are described in the table below.

PoolFragStats

Array

Array of hashes, each hash containing statistics on the number of failed pool allocation requests due to fragmentation.

For example, a hash of {MinFractionFree => 0.05, MaxFractionFree => 0.10, NrFailedAllocations => 500 } indicates that 500 allocation requests failed when between 5% and 10% of pool memory was free.

For the ServiceDA cache, the sum of all of the counts in this array may be less than PoolTotalNrFailAllocs, because some allocation requests may fail due to the unavailability of reserved free memory rather than due to memory fragmentation.

For every other cache, the sum of all of the counts in this array equals PoolTotalNrFailAllocs.

Checks

Integer

Optional key tracks the number of times an integrity check has been performed on the cache.

It is only present if one or more checks have been performed.

Recoveries

Integer

Optional key to track the number of times cache recovery has been performed.

It is only output if recovery is performed.

If a cache recovery is performed, but no correction is required, the recovery is counted as a check and not a recovery.

Flushes

Integer

Number of times the cache has been flushed.

This optional key is only present if the cache has been flushed at least once.

LastFlushDate

Date

Date-time the cache was last flushed.

This optional key is only present if the cache has been flushed.

 

The following table lists PoolBlockListStats integer arrays.

 

PoolBlockListStats Integer Arrays

Array

Description

MaxBytes

Maximum size (in bytes) of blocks in the lists from which the statistics were obtained.

MinBytes

(Integer) Minimum size (in bytes) of blocks in the lists from which the statistics were obtained.

TotalAllocatedBytes

Total number of bytes allocated (since pool was created or reset) from the block lists for storage of data.

TotalWastedBytes

Total number of bytes wasted in blocks allocated from the lists.

TotalNrSuccAllocs

Total number of successful memory allocations from the block lists since the pool was created/reset.

FreeBytes

Total number of bytes in currently unallocated blocks in the lists.

NrFreeBlocks

Total number of currently unallocated blocks in the lists.

UsedBytes

Total number of bytes in blocks currently allocated from the lists.

NrUsedBlocks

Total number of blocks currently allocated from the lists.
