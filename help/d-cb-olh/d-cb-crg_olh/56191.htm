FILE_REGISTER_RECORD_TRE_V

The FILE_REGISTER_RECORD_TRE_V view stores record-level processing details for the Batch Extractor.

The view is used for auditing extracted files and defining records to be extracted as part of a delayed batch extraction.

Column Definitions

Column Name

Properties

Description

FILE_REGISTER_RECORD_ID

NUMBER (15)

Not null

Read-only

Derived

ID of the file register record.

It is automatically derived on insert.

LAST_MODIFIED

DATE

Not null

Read-only

Derived

Date-time when this record was last modified.

It is automatically derived on insert and update.

OPERATOR_ID

NUMBER (8)

Not null

Read-only

Derived

ID of the user who last modified this record.

It is automatically derived on insert and update.

PARTITION_NR

NUMBER (2)

Not null

Read-only

Derived

Customer partition number associated with the file register record.

It can be specified on insert; otherwise, it is set to the default partition for the instance on which the insert was performed.

It forms part of the partitioning key for this table.

CREATION_DATE

DATE

Not null

Read-only

Date-time when the file register record was created.

It can be specified on insert; otherwise, it defaults to the current date-time.

It forms part of the partitioning key for this table.

PENDING_CREATION_DATE

DATE

Read-only

Derived

Date-time the record entered Pending status.

Set to:

  • CREATION_DATE, if FILE_RECORD_STATUS_CODE is Pending
  • NULL, after the record is extracted.

FILE_RECORD_STATUS_CODE

NUMBER (2)

Not null

File record extraction status level.

Based on the FILE_RECORD_STATUS reference type.

It can be specified on insert; otherwise, it defaults to Pending.

It can only be updated from:

  • Pending to another status
  • In Progress to Success or Failure.

TEMPLATE_INTERFACE_
DEFN_ID

NUMBER (8)

Not null

Read-only

ID of the interface template used to extract this record.

The interface template must be based on a BIM interface type.

It must be specified on insert.

FILTER_KEY

VARCHAR (255)

String used for later identification and filtering of records.

It cannot be updated after FILE_RECORD_STATUS_CODE is set to Success or Failure.

CB_ENTITY

VARCHAR (30)

Read-only

Convergent Billing entity associated with the external definition.

If not specified, PE_CLASS_ID must be specified.

It can be specified on insert.

PE_CLASS_ID

NUMBER (8)

Read-only

Process Engine class associated with the external definition.

If not specified, CB_ENTITY must be specified.

It can be specified on insert.

ENTITY_ID

NUMBER (15)

Not null

Read-only

ID that forms a unique key to identify the record to be extracted.

It must be specified on insert.

ENTITY_ID2

NUMBER (15)

Not null

Read-only

ID that forms a unique key to identify the record to be extracted.

It can be specified on insert.

ENTITY_ID3

NUMBER (15)

Not null

Read-only

ID that forms a unique key to identify the record to be extracted.

It can be specified on insert.

EFFECTIVE_DATE

DATE

Date-time of the extracted record, if the record is date-ranged.

If not specified on insert, it defaults to the effective date of the calling process.

FILE_REGISTER_ID

NUMBER (15)

ID of the file record as specified in the FILE_REGISTER_TRE_V.

It can be specified on insert or update if FILE_RECORD_STATUS_CODE is being set to a value other than Pending.

If specified, FILE_REGISTER_RECORD_NR must be specified. After it is set, it cannot be updated.

FILE_REGISTER_RECORD_NR

NUMBER (10)

Record number in the file associated with FILE_REGISTER_ID.

The first record in a file has FILE_REGISTER_RECORD_NR set to 1.

It can be specified on insert or update if FILE_RECORD_STATUS_CODE is being set to a value other than Pending.

If specified, FILE_REGISTER_ID must be specified.

After it is set, it cannot be updated.

INTERFACE_DEFN_ID

NUMBER (8)

ID of the interface definition created using the interface template specified by TEMPLATE_INTERFACE_DEFN_ID.

It must be specified on insert or update if FILE_RECORD_STATUS_CODE is not Pending. After it is set, it cannot be updated.

EXTERNAL_DEFN_ID

NUMBER (8)

ID of an external definition that defines the data used for a batch extraction or a batch load.

It can be specified on insert or update if FILE_RECORD_STATUS_CODE is being set to a value other than Pending.

After it is set, it cannot be updated.

The external definition must match the values for CB_ENTITY or PE_CLASS_ID.

OPERATION_DATE

DATE

Read-only

Date-time of the batch extraction.

Set to the current date-time of the calling process when the status is set to Success or Failure.

TASK_ID

NUMBER (15)

ID of the task that processed this record.

It can be specified on insert or update if FILE_RECORD_STATUS_CODE is being set to a value other than Pending.

After it is set, it cannot be updated.
