Java Transaction Engine API

This section provides an overview of the Java Application Programming Interface (API) to the Transaction Engine (TRE).

For full details of the Java TRE API, refer to the Java documentation in the Singleview release folder at the following locations:

  • Enterprise Java Interface: <release_version>\java\doc\eji\index.html
  • Java Bridge: <release_version>\java\doc\tresrv\index.html

The Java API is distinct from the C, Perl, and Object Pascal API because Java is platform independent. It is also known as the Enterprise Java Interface (EJI). The term Java TRE API is used in this section.

The TRE is used by client and server processes.

The TRE can also be used by external applications using the API, for the following purposes:

  • Calling TRE functions
  • Grouping function calls into transactions, with the option to commit or roll back the transaction
  • Queuing transactions
  • Subscribing to events, defining an event handler, and checking for events
  • Posting TRE events.

Note:

It is assumed that the reader has a thorough knowledge of the Java programming language.
