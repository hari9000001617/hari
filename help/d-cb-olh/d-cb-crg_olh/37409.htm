DerivedTableContentsWithFilter?[]()

Syntax

DerivedTableContentsWithFilter?[](
    const TableName$,
    EffectiveDate~,
    const Keys?[])

 

Description

Returns a filtered two-dimensional unknown array from the specified derived attribute table with a storage context of Global or Global-Shared.

Attributes

Function Type

Local Built-in

Service

(Local)

Application Environment

Any

Context

Any

 

Parameter

Description

TableName$

In: Table name of the derived attribute lookup table to query.

The lookup table can be cached in either the process or shared memory caches depending on the storage context.

Note:

The table must be of type Table Lookup by Multiple (1-10) Keys or Table Lookup by Searching.

EffectiveDate~

In: Date for which the derived attribute table contents are required.

Keys?[]

In: Filter criteria to use.

It must match the defined number of indexes for the table.

Undefined values in the array are interpreted as a wildcard (that is, a match for anything corresponding to the key element).

 

Returns

Two-dimensional unknown array in [row][column] order containing the contents of the rows from the specified derived attribute table that match the filtering criteria specified in Keys?[].

Note:

Rows are not returned in SEQNR order.

An empty unknown array is returned if no rows match the filtering criteria.

An error is raised if the:

  • Specified derived attribute table does not exist at EffectiveDate~.
  • Evaluation type is not of type Table Lookup by Multiple (1-10) Keys or Table Lookup by Searching.
  • Index key length does not match the number of indexes defined for the table.
  • Index columns are associated with an encrypted attribute type.

Remarks

Values returned from a results column may be encrypted. Encrypted data must be explicitly decrypted using biAttributeValueDecrypt$(). Index columns do not support encrypted attributes types.

A number of filtered results are possible from this derived attribute table using differently populated Keys?[] arrays. NULL is referred to as an undefined element in the two-dimensional unknown array. The results in the following table are returned.

The result for row one is returned because it is an exact match of all index columns in the Keys?[] array.

["QLD", "4000", NULL]

[["QLD", "4000", "Peak", 0.20]],
[["QLD", "4000", "Off-peak", 0.15]]

Results for rows one and two are returned because the match is determined by index columns one and two.

[NULL, "4000", NULL]

[["QLD", "4000", "Peak", 0.20]],
[["QLD", "4000", "Off-peak", 0.15]]

Results for rows one and two are returned because the match is determined by index column two.

[NULL, NULL, NULL]

[["QLD", "4000", "Peak", 0.20]],
[["QLD", "4000", "Off-peak", 0.15]],
[["QLD", "4001", "Peak", 0.22]],
[["QLD", "4000", "Peak", 0.25]]

All rows are returned.

This result is identical to the result that would be returned by the DerivedTableContents?[]() function.
