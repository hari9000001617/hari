PERSON_HISTORY_SEARCH_V

The PERSON_HISTORY_SEARCH_V view defines information required for the person search forms.

Column Definitions

Column Name

Properties

Description

PERSON_ID

NUMBER (15)

Not null

ID of a person.

EFFECTIVE_START_DATE

DATE

Not null

Date-time that the record becomes effective.

Its finest granularity is one second.

EFFECTIVE_END_DATE

DATE

Not null

Date-time of the last second that the record is effective.

The record ceases to be effective one second after the EFFECTIVE_END_DATE.

Its finest granularity is one second.

It must be greater than or equal to EFFECTIVE_START_DATE.

PERSON_TYPE_ID

NUMBER (8)

Not null

ID of the type of person that also defines what values (if any) must be entered into the ten general fields, and the reference type for the PERSON_STATUS_CODE.

If only one person type is defined, this field automatically defaults.

PRIMARY_IDENTIFIER

VARCHAR (80)

Optional primary ID of the person.

It can be the same as another column in this table.

The type of value in this field may be dependent on the person type or values in other columns.

PRIMARY_IDENTIFIER2

VARCHAR (80)

Optional secondary primary ID of the person.

It can be the same as another column in this table.

Its value may be dependent on the person type or values in other columns.

Both primary IDs are usually not specified, but both can be, if required.

TITLE

VARCHAR (60)

Title of the person.

FAMILY_NAME

VARCHAR (110)

Family name or surname.

FAMILY_NAME_UPPERCASE

VARCHAR (110)

Derived

Family name with all characters converted to upper case, used for case-insensitive searching.

FAMILY_NAME_SOUNDEX

VARCHAR (10)

Derived

Soundex value of the FAMILY_NAME column, derived from FAMILY_NAME using a customised version of the soundex algorithm.

It is used for soundex-based searches using FAMILY_NAME.

GIVEN_NAMES

VARCHAR (80)

Person's first name and/or middle name.

GIVEN_NAMES_UPPERCASE

VARCHAR (80)

Derived

Person's first names and/or middle name converted to upper case, used for case-insensitive searching.

GIVEN_NAMES_SOUNDEX

VARCHAR (10)

Derived

Soundex value of the GIVEN_NAMES column, derived from GIVEN_NAMES using a customised version of the soundex algorithm.

It is used for soundex based searches using GIVEN_NAMES.

OFFICIAL_NAME

VARCHAR (200)

Not null

Official name to be used in corresponding with this person.

The naming conventions (for example, family, first, or last), specified for individual installations, defaults this value according to the information stored in other naming fields.

OFFICIAL_NAME_UPPERCASE

VARCHAR (200)

Not null

Derived

Set to UPPERCASE(OFFICIAL_NAME)

It is enforced by C_PERSON_HISTORY check constraint.

PREFERRED_NAME

VARCHAR (80)

Preferred name of the person, used verbally.

PERSON_STATUS_CODE

NUMBER (2)

Not null

Status of this person. Valid values are:

  • 0 (Cancelled)
  • 1 (Prospect)
  • 2 (Subscriber)
  • 3 (Business Contact)
  • 4 (Subscriber & Business Contact)
  • 5 (VIP)
  • 6 (Bad Risk).

The reference type used for possible values is obtained from the PERSON_TYPE table.

HOME_ADDRESS_ID

NUMBER (15)

Home, postal, and work addresses of the person.

These are indexes into the ADDRESS details table to enable multiple database areas to reference the same address definition.

For example, all of the above addresses may reference the same definition for a person who works from home.

POSTAL_ADDRESS_ID

NUMBER (15)

Home, postal, and work addresses of the person.

These are indexes into the ADDRESS details table to enable multiple database areas to reference the same address definition.

For example, all of the above addresses may reference the same definition for a person who works from home.

WORK_ADDRESS_ID

NUMBER (15)

Home, postal, and work addresses of the person.

These are indexes into the ADDRESS details table to enable multiple database areas to reference the same address definition.

For example, all of the above addresses may reference the same definition for a person who works from home.

HOME_PHONE_NR

VARCHAR (30)

Standard telephone voice and fax network dial numbers.

No spaces are present in the numbers to allow standard formatting and display on reports.

At least one of the above numbers must be present.

WORK_PHONE_NR

VARCHAR (30)

Standard telephone voice and fax network dial numbers.

No spaces are present in the numbers to allow standard formatting and display on reports.

At least one of the above numbers must be present.

MOBILE_PHONE_NR

VARCHAR (30)

Standard telephone voice and fax network dial numbers.

No spaces are present in the numbers to allow standard formatting and display on reports.

At least one of the above numbers must be present.

FAX_PHONE_NR

VARCHAR (30)

Standard telephone voice and fax network dial numbers.

No spaces are present in the numbers to allow standard formatting and display on reports.

At least one of the above numbers must be present.

PASSPORT

VARCHAR (40)

Passport ID.

SOCIAL_SECURITY_NUMBER

VARCHAR (40)

Social security number.

DRIVERS_LICENCE

VARCHAR (40)

Driver's licence of the person.

 

Search Related Field

Column Name

Properties

Description

SEARCH_IND_CODE

NUMBER

Code of the mode of the search form.

A value of 0 indicates a search operation; a value of 1 indicates a new operation.
