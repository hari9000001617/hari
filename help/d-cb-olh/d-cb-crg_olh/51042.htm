RGP_PRODUCT_INSTANCE_V

The RGP_PRODUCT_INSTANCE_V view is used to generate rentals for product instances. It is applicable to both base and companion product instances.

PRODUCT_INSTANCE_
STATUS_CODE

NUMBER (8)

Not null

Current status of this product.

It mirrors service statuses. Valid values include:

  • 0 (Product Creation): A temporary status during the creation of a product). When product creation is complete, this status is updated.
  • 1 (Provision Required): Provisioning must be performed.
  • 2 (Provisioned)
  • 3 (Active)
  • 4 (Suspended)
  • 5 (Temporary Disconnect Required): Deactivation must be performed.
  • 6 (Temporarily Disconnected)
  • 7 (Reactivation Required: Reactivation must be performed.
  • 8 (Cancellation Required): Deactivation must be performed.
  • 9 (Cancelled)
  • 10 (Moved)
  • 11 (Upgraded).

Other statuses can be added by implementation teams.

Of these statuses, the codes for Active, Cancelled, Moved, and Upgraded are used by the system and must always be supplied. The numeric code for Cancellation Required is also used but does not need to be supplied.

The numeric code for Provisioned is also significant in the following rules; however, it is not necessary to provide such a status.

The following rules apply to the statuses of product instances and their related entities.

  • Changing the status of a product instance to Cancelled, Moved, or Upgraded changes the status of its services, companion product instances, and facility group instances to Cancelled, Moved, or Upgraded respectively, irrespective of their current status.
  • Changing the status of a product instance to Active changes the status of all its services, companion product instances, and facility group instances to a status of Active if they have either the same status as the product instance, or a status of Provisioned; otherwise, their statuses are not changed.
  • Changing the status of a product instance to any status other than Cancelled changes the status of all its services, companion product instances, and facility group instances that had the same status as the product instance to the new status.
  • A service, companion product instance or facility group instance cannot have a status of Active unless their base product instance has a status of Active. It is checked by the DVP.
  • A service, companion product instance, or facility group instance must have a status of Cancelled, Moved, or Upgraded if their base product instance has a status of Cancelled, Moved, or Upgraded respectively. It is checked by the DVP.
  • A service must have a status of Product Creation if its base product instance has a status of Product Creation. It is checked by the DVP.
  • A base product instance with a status of Product Creation cannot have any companion product instances or facility group instances. It is checked by the DVP. The default facility group instances, and upgraded facility group instances and companion products instances are populated on product completion.

CUSTOMER_NODE_ID

NUMBER (15)

Not null

Customer node that owns this instance of the product that must exist for at least this date range.

It is checked by the DVP.

BASE_PRODUCT_INSTANCE_
ID

NUMBER (15)

If PRODUCT_ID corresponds to a companion product, this column must be specified and is the ID of the base product instance for this companion product instance.

It is checked by the DVP.

FROM_PRODUCT_INSTANCE_
ID

NUMBER (15)

If this product instance was created as a replacement or a product instance, either by a move or an upgrade, this is the ID of the product instance that was replaced.

When the PRODUCT_INSTANCE_ID has a status of Active, it must have a status of Moved or Upgraded, and is eventually deleted; therefore, this value may not reference a valid product instance. The DVP checks that in the event the product instance does exist:

  • That the referenced product instance has a TO_PRODUCT_INSTANCE_ID. As a single product instance could have been split to create this and other product instances, the referenced product instance's TO_PRODUCT_INSTANCE_ID may not be equal to PRODUCT_INSTANCE_ID.
  • That if this product instance has a status of Active, the other product instance has a status of Moved or Upgraded.

FROM_PRODUCT_ID

NUMBER (8)

Product ID associated with the FROM_PRODUCT_INSTANCE_ID, if this product instance was created as part of an upgrade.

It is not redundant because at some point after a replacement, the FROM_PRODUCT_INSTANCE_ID is archived off the system.

It must contain a value for product upgrades, and must be null for product moves. It is checked by the DVP.

TO_PRODUCT_INSTANCE_ID

NUMBER (15)

If this product instance was replaced by a move or upgrade, this is the ID of the product instance that replaced it.

When it has a status of Active, this product instance must have a status of Moved or Upgraded and is eventually deleted. It is checked by the DVP.

To simplify range updating, a foreign key does not exist on this column. Instead, the DVP checks that the referenced product instance has a FROM_PRODUCT_INSTANCE_ID.

As this product instance could be one of many that have been merged into a single product instance, the referenced product instance's FROM_PRODUCT_INSTANCE_ID may not be equal to PRODUCT_INSTANCE_ID.

TO_PRODUCT_ID

NUMBER (8)

Redundant

Product ID associated with the TO_PRODUCT_INSTANCE_ID, if this product instance was replaced as part of an upgrade.

It is redundant, being obtained from TO_PRODUCT_INSTANCE_ID.

It must contain a value for upgraded product instances, and must be null for moved product instances, enforced by C_PRODUCT_INSTANCE_HISTORY check constraint.

GENERAL_1 - GENERAL_10

VARCHAR (255)

General fields 1 - 10.

 

Product Offer Columns

Column Name

Properties

Description

PRODUCT_OFFER_ID

NUMBER (8)

ID of the product offer associated with this base product instance, or the product offer from which this companion product instance was generated.

For base product instances, product offer must be compatible with the base product.

For companion product instances:

  • The product offer must belong to the product offer hierarchy of the product offer assigned to this product instance's base product.
  • If the base product instance is assigned a product offer, it must not be set to NULL.
  • If the base product instance is not assigned a product offer, it must be set to NULL.

PRODUCT_OFFER_NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product instance was allocated.

If PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PRODUCT_OFFER_
ALLOCATION_DATE

DATE

Date-time at which this product instance was allocated from the product offer specified in PRODUCT_OFFER_ID.

For base product instances, it must be set to NULL.

For companion product instances, if PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.

PREVIOUS_PRODUCT_OFFER_
ID

NUMBER (8)

ID of the previous product offer associated with this product instance.

It is automatically modified when the product offer associated with a product is changed.

PREVIOUS_PRODUCT_OFFER_
NODE_ID

NUMBER (8)

ID of the product offer hierarchy node from which the product instance was previously allocated.

If PREVIOUS_PRODUCT_OFFER_ID is:

  • NULL, it must be set to NULL.
  • Not NULL, it must not be set to NULL.
