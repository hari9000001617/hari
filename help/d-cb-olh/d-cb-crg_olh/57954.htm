Mail.Receive&()

Syntax

Mail.Receive&(
    const MessageID$,
    const FromAddr$,
    const Sender$,
    const ReplyTo$,
    const ReturnTo$,
    const SentDate~,
    const InReplyToMsgId$,
    const References$[],
    const Headers${},
    const Subject$,
    const To$[],
    const Cc$[],
    const Message$[],
    const MimeType$[],
    const Inline?[],
    const Attach?[])

 

Description

Receives email into Singleview using an incoming mail monitor in the Web Services Adapter. This function is provided as a template to be duplicated or modified by implementation teams.

Attributes

Application Environment

Any

Context

Any

Method

Remote

 

Parameter

Description

MessageID$

In: ID of the message.

This value is generated by the sender's email system, and is used to respond to the received email, where the reply is considered as part of the same email conversation.

FromAddr$

In: Email address of the person sending the email.

Sender$

In: Person sending the email.

ReplyTo$

In: Email address specified by sender to be used when replying to this email.

ReturnTo$

In: Email address used to receive bounce notifications.

SentDate~

In: Date the email was sent.

InReplyToMsgId$

In: Message ID of the email being replied to.

References$[]

In: Array of message IDs for all previous emails associated with the email.

This array does not include the MessageID$ value.

Headers${}

In: Header containing values from the email.

The hash key is the header name, and the hash value is the header value.

Subject$

In: Subject of the email.

To$[]

In: Array of email addresses receiving the email.

Cc$[]

In: Array of email addresses copied on the email.

Message$[]

In: Array of email message contents received.

Each value is a separate MIME format.

MimeType$[]

In: Array of MIME format email messages contained in the Message$[] parameter.

Inline?[]

In: Array of hashes of attachments listed as files to be displayed inline in the email message (for example, pictures embedded inline within the email).

For information about the hash keys, refer to the table below.

Attach?[]

In: Array of hashes of files attached to the email.

For information about how attachments are handled, and hash keys, refer to the table below.

 

Returns

1 (TRUE) if successful; otherwise, an error is raised.

Remarks

The saveattachments property in the Config File Template specfies the method used to handle attachments:

  • If false, attachments are supplied as data streams to this function with the data of the attachment supplied as a string or BLOB. This method is useful for small attachments, up to 100kb.
  • If true, Web Services Adapter saves the attachments as a file on the Singleview file system. The contents of the attachment are written to this file and Mail.Receive&() calls the file name of this file to retrieve the attachments. File path or URL values are supplied.

Note:

The saveattachments property is set to false in the $ATA_REL_JAVA_CONFIG/email.xml template file.

For more information about defining incoming mail monitors, refer to the WSAEmail interface type in the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 1)

The following table lists hash keys in the Inline?[] and Attach?[] parameters.

 

Inline?[] and Attach?[] Parameter Hash Keys

Key

Value

type

MIME format used for the attachment.

id

ID of the attachment.

This key is mandatory for inline attachments.

filename

File name for the attachment specified by the sender.

Some attachments may not have file names.

data

String or BLOB containing contents of the attachment.

file

Name of the file on the Singleview server where the attachment was saved.

url

URL of the attachment file stored on the Singleview server.

This value is defined by the attachmenturl property in the email.xml configuration file, which must be updated before it can be used.
