AccountClose&()

Syntax

AccountClose&(
    AccountId&,
    var LastModified~,
    CloseDate~)

 

Description

Closes the account specified by AccountId&.

Attributes

Application Environment

Any

Context

Any

Method

Built-in

 

Parameter

Description

AccountId&

In: ID of the account to be updated.

LastModified~

In: Last modified date-time of the account to be updated.

Out: Last modified date-time of the updated account.

CloseDate~

In: Closure date-time of the account.

 

Returns

If successful, 1 (TRUE) with LastModified~ set to the last modified date-time of the account, or Null if the account was deleted; otherwise, an error is raised if the:

  • Value of AccountId& is not the ID of any account.
  • Value supplied for LastModified~ does not equal the value of LAST_MODIFIED for this account.
  • Balance of the account is not zero.
  • Account is the prime account for its associated customer node.
  • Value of CloseDate~ is not within the date range of the last history record for this account because either of the following has occurred:
    • The account was closed at a date or time earlier than CloseDate~.
    • A transaction was performed on this account at a date-time later than CloseDate~.

Remarks

An account can be closed only if its balance is zero, and it is not the primary account for its associated customer node.

If the account can be closed and CloseDate~ is the same as the date on which the account was created, the account is deleted.

If the account is deleted, the biAccountPurgeLocal&() function is called to replicate the account deletion to any separate rating instance.

The effective user must have update security group access level for the customer associated with the record manipulated by this function call.

GROUP_ACCESS_TRE_V view lists security group access level values.
