One-Off Task Definition Form

The following table lists automation parameters for the One-Off Task Definition form, data type, and supported functions.

The description column is provided to supplement the description in the SCHEDULE_TRE_V view, or to provide information about parameters not contained within the database view.

Parameter Definitions

Parameter or Field Name

Data Type

Supported Functions

Description

ADD_BILL_RUN_SCHEDULE_TYPES_
TO_COMBO

Integer

Insert

Specifies whether to add the bill run schedule types (schedule types with a task code of 5 or 6) to the schedule type combo.

If set to any value, the schedule types are added to the field.

The value passed is ignored.

BASELINE_DURATION

Integer

Insert

Expected duration of the task in seconds.

Note:

The value must be greater than or equal to 1, and less than or equal to 2147483647.

DEPENDENCY_FAILURE_IND_CODE

Integer

Insert

Specifies that the failure of any prerequisite task updates the status of the task to Failure.

EFFECTIVE_DATE

Date

Insert

Effective date-time used for the task.

EMAIL_ADDRESS

String

Insert

Email address where the task results will be sent.

Note:

If email notification is not required, leave EMAIL_ADDRESS undefined or set it to either of the following:

  • /dev/null
  • nobody.

GENERAL_1 - GENERAL_25

String

Insert

General fields 1 - 25.

IMMEDIATE_START_IND_CODE

Integer

Insert

Specifies that the task start immediately when all prerequisite conditions are satisfied.

A task with no prerequisites starts when the previous task in the schedule is completed.

INITIAL_HELD_IND_CODE

Integer

Insert

Specifies that the task enter Held status when ready to run.

INSTANCE_NAME

String

Insert

Convergent Billing instance on which all tasks associated with the schedule are to be run.

MAX_DURATION

Integer

Insert

Maximum duration of the task in seconds.

Note:

The value must be greater than or equal to 1, and less than or equal to 2147483647.

PURGE_PERIOD

Integer

Insert

Maximum number of days temporary data associated with the task is kept.

REPLICATE_GROUP_CODE

Integer

Insert

Specifies a set of instances to which the schedule is replicated.

REPLICATE_IND_CODE

Integer

Insert

Replicates the schedule to a specified set of instances associated with the replication group.

SCHEDULE_TYPE_ID

Integer

Insert

ID used to define the type of task or function that is being scheduled.

SERVER_COMMAND_ARGS

String

Insert

Additional server command-line parameters used when invoking the task.
