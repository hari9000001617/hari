Customer Type Definition Form

The Customer Type Definition form is used to view, define, and update a customer type.

Specifies any defaults or entity validation for customers specified with that type. If required, an icon can be specified to identify each customer type.

Context: Convergent Billing, Customer Management

See also Customer and Icon.

The upper part of the form allows entry and display of basic details about the customer type, such as its name and description, charge category details, and icon used to represent the customer type.

The lower part of the form consists of the following tabs, allowing more specific details to be entered and viewed:

Settings

The Settings tab is used to:

  • Specify the form to be used to enter customer details and defaults for various fields (for example, prime account type, report level, GL code, tax class).
  • Specify the address format to be used for the customer type.
  • Define the expression used to generate account numbers.

The use of the GL code specified on this tab is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance.

Credit and Payment

The Credit and Payment tab is used to enter default credit limits, payment methods, and payment due period, and early payment period for customers of a specified type.

Billing

The Billing tab is used to specify the maximum number of cycles or amount of time for which invoicing may be suppressed for root customers of the relevant customer type.

Entity Validation

The Entity Validation tab is used to select entity validation definitions to customise the Customer Detail form, and its associated forms, for customers of a specific type.

Derived Attribute Tables (List)

The fields on this tab are used to identify derived attributes to be included in the Customer Detail form. A table row is inserted for each derived attribute to be included.

The form can be displayed in one of three form modes, with the current mode being indicated in the form’s title.

A form can be displayed in one of the following modes:

Form Mode

Description

View Only mode

Allows information held in Convergent Billing to be viewed, but not updated (modified).

Insert mode

Allows new information to be recorded (entered) in Convergent Billing.

Update mode

Allows information held in Convergent Billing to be viewed and updated (modified).

Associated Actions

Viewing a Customer Type

To view a customer type:

  1. Select the Maintenance Customer Care Customer Types option. The Customer Type Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the defined customer types matching the specified criteria is displayed.
  4. Select the appropriate customer type and click the Detail... button, or double-click the appropriate customer type. The Customer Type Definition form is displayed in View Only mode.

Need more information on customer types?

Updating a Customer Type

To update a customer type:

  1. View the appropriate customer type.
  2. Click the Update button on the toolbar. The Customer Type Definition form is displayed in Update mode.
  3. Update the customer type details as required.
  4. Click the Save button on the toolbar.

Need more information on customer types?

Defining a Customer Type

To define a customer type:

  1. Select the Maintenance Customer Care Customer Types option. The Customer Type Search form is displayed.
  2. Click the New... button. The Customer Type Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Need more information on customer types?

Deleting a Customer Type

To delete a customer type:

  1. View the appropriate customer type.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the customer type is to be deleted.
