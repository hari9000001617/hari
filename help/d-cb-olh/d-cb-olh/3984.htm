Variables Declared with the @ Symbol

Local variables with scope only within a single expression are implicitly declared by Convergent Billing using the @ symbol. The variables are declared for use in expression in the client application.

@charge1<type> to @charge10<type>

Contains value that refers to charges defined in the Tariff Definition form. Each variable refers to one of the charges defined in the grid on the Charges tab (for more information, refer to the Customer Care and Product Configuration Guide for Singleview Convergent Billing).

 

A variable with scope only within a single expression can be declared using the @ symbol. The most common example of this is the @value<type> variable, used in the Convergent Billing client for encoding and decoding sequence or serial numbers.

Users can define an expression to format the account number for a customer node by entering an expression such as:

'ACC' + to_string (@value&, '00000000')

The expression takes the account ID of the account being declared in the database, converts it to a string of eight numerals, and prefixes it with the letters 'ACC'.
