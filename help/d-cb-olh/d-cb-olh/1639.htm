Data Types

A variable's data type can be either simple (for example, an integer or string) or composite (for example, an unknown hash or integer array).

Simple variables can be of six data types: integer, real, string, date, blob (binary large object), and unknown. The EPM enforces strict typing, and the appropriate type declaration character must suffix variable names, as follows.

 

Data Type

Description

Type Declaration Character

Range

INTEGER

Equivalent to UNIX 'long'. Signed Integer.

&

-2,147,483,647

to

2,147,483,648

REAL

Equivalent to UNIX 'double'. Floating point number.

# (or none)

-1.79 x 10308 to 1.79 x 10308

STRING

Sequence of characters enclosed by single quotes.

$

String size is limited only by the available memory.

DATE

Date and time.

~

01 Jan 0001 00:00:00 to 30 Dec 9999 23:59:59
(24 hour time notation).

BLOB

Byte array.

@

Maximum number of bytes is 232 –1.

UNKNOWN

Unknown scalar type. Values of any other scalar type may be assigned to a variable declared to be of this type.

?

N/A

 

Note:

  • If no type declaration character is appended to a variable name, the variable is assumed to be a Real value.
  • The type declaration character forms part of the variable name; therefore, a$, a&, a#, and a~ are all distinct variables.
  • In Convergent Billing, variables (except local variables) are subject to date ranging; this means that a variable has a start date and an end date, which together define the period for which the variable is applicable. Often the end date is not defined, which means that the variable is applicable indefinitely.
