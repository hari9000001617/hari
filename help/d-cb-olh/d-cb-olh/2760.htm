Derived Attribute Definition Form

The Derived Attribute Definition form is used to view, define, and maintain a derived attribute.

Facility for deriving a new variable based on the values of other variables, which can be achieved by:

  • Using a simple expression
  • Conditional evaluation (similar to functions)
  • List evaluation type
  • Table look-up based on single, multiple, searching, or range indexes.

Context: Convergent Billing

See also Evaluation Type, Master Derived Attribute, Product Definition, Slave Derived Attribute, Keyed Derived Attribute, and Variable.

The upper part of the form displays, or allows entry of, general information about the derived attribute, such as derived attribute name, description, start date and end date.

The lower part of the form consists of the following tabs for more information to be displayed or entered:

Details

The Details tab is used to select information for the application environment of the derived attribute, context in which it will be used, mechanism used to derive the value returned, storage context, and data type returned.

Lookup Table

The Lookup Table tab is used to define details for a derived attribute defined with any of the table lookup evaluation types, or an evaluation type of Lists.

The tab specifies:

  • Details for the lookup table including table name, description, maximum number of rows in the table, result column number (result column the derived attribute will return), and the validation for the derived attribute.
  • Entity validation record that defines columns in the derived attribute table on which searches can be performed (if the storage context of the derived attribute is Global or Global Shared).

The Table… button is enabled only if Global, Global Shared, or Interface Definition is selected in the Storage Context field on the Detail tab.

Note:

If the value in Storage Context is Facility, this tab is not displayed.

Indexes

The Indexes tab is used to define the parameters for the index column in the lookup table. Each index is specified on a separate row. The tab is only displayed if the selected evaluation type is 'List', or one of the table lookup evaluation types.

Note:

If the value in Storage Context (on the Details tab) is Facility, this tab is not displayed.

Conditions

The tab is used to define conditional expressions, associated value expressions and, optionally, tariff split condition expressions.

It is only displayed if the selected evaluation type is one of the following:

  • Conditional: Single Value Returned
  • Conditional: Multiple Values Returned.

If specifying conditions for a derived attribute with an evaluation type of Conditional: Single Value Returned, the order of the conditional expressions is important because the value returned is the value associated with the first conditional expression that evaluates to TRUE.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

Associated Actions

Viewing a Derived Attribute

To view a derived attribute:

  1. Select the Maintenance Tariffs Derived Attributes Definition option. The Derived Attribute Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the defined derived attributes matching the specified criteria is displayed.
  4. Select the appropriate derived attribute and click the Detail... button, or double-click the appropriate derived attribute. The Derived Attribute Definition form is displayed in View Only mode.

Defining a Derived Attribute

To define a derived attribute:

  1. Select the Maintenance Tariffs Derived Attributes Definition option. The Derived Attribute Search form is displayed.
  2. Click the New... button. The Derived Attribute Definition form is displayed in Insert mode.
  3. Complete the data entry fields as required.

    Evaluation Type: Conditional: Single Value Returned

    Evaluation Type: Conditional: Multiple Values Returned

    Evaluation Type: List

    Evaluation Type: Simple Expression

    Evaluation Type: Table Lookup by Best Match

    Evaluation Type: Table Lookup by Multiple (1-10) Keys

    Evaluation Type: Table Lookup by Searching

    To define a derived attribute having an evaluation type of Table Lookup by Searching:

    1. View the Derived Attribute Definition form.
    2. Complete the data entry fields on the Details tab. Ensure that the Evaluation Type field is set to Table Lookup by Searching.
    3. Select the appropriate storage context.
    4. Complete the appropriate data entry fields.
    5. Select the Lookup Table tab. Select the appropriate entity validation definition in the Validation field, ensuring that the definition is defined with at least one index and one result column.
    6. If a user-specified wildcard is required, enter the wildcard value in the Wildcard String field on the Lookup Table tab. Otherwise, leave the field blank and undefined values are treated as wildcards.
    7. If required, enter the result column number and maximum number of rows.
    8. Select the Indexes tab. Enter the appropriate index expression.
    9. Click the Save button on the toolbar. If a storage context of Global or Global-Shared is used, continue with the procedure.
    10. Click the Table… button on the Lookup Table tab. The Derived Attribute Table form is displayed with the defined index and result columns.
    11. Populate the table with the required lookup (index) and result values.
      • If a user-specified wildcard is set in the Wildcard String field, ensure that the wildcards entered in the index columns match the value in the field.
      • If Wildcard String field is not set, any undefined index columns are treated as wildcards.
      • A row can be included in the derived attribute table with all index columns specified as undefined wildcards. This is the default row returned if no matches are found.
    12. Click the Save button on the toolbar.

Updating a Derived Attribute

Caution:

Modifying a derived attribute may result in corruption of data and/or forms that use the derived attribute.

To update a derived attribute:

  1. View the appropriate derived attribute.
  2. Click the Update button on the toolbar. The Derived Attribute Definition form is displayed in Update mode.
  3. Update the derived attribute details as required.
  4. Click the Save button on the toolbar.

Need more information on derived attributes?

Deleting a Derived Attribute

To delete a derived attribute:

  1. View the appropriate derived attribute.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the derived attribute is to be deleted.
