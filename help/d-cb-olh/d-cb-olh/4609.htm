TRE Named Servers

TRE named servers advertise services available to clients. Each service is made up of one or more remote functions that carry out some operation within the service. For example, the service designed to work with the person entity in the Convergent Billing database (biPerson) contains functions for locating, adding, updating, and deleting a person record in the database.

The named servers are:

  • Authentication Server (authsvr)
  • Bill Generation Server (trebgp)
  • Database Read-Only Server (trerodb)
  • Invoice Generation Server (treigp)
  • Monitor Services Server (tremon)
  • Rating Server (trerate)
  • Remote Services Server (treproxy)
  • Rental Generation Server (trergp)
  • Simple Network Management Protocol (SNMP) trap server (tresnmp)
  • Socket Interface Server (tresok)
  • Transactional Rating Server (trerate_tran)
  • Transactional Server (trerwdb)

All of these servers have common services, as follows:

  • biPurge

    Receives events sent out when cached configuration (for example, functions or derived attribute tables) changes.

  • biTrcInvoice

    Receives events that invoke a specified EPM function in the server with a given set of parameters. biTrcInvoice is used to gather performance statistics from the servers.

  • biTrcMem

    Processes the event that outputs a memory report for the server.

  • biTrcTre

    Processes the event that activates or deactivates TRE message tracing.
