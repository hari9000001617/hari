How Do I - Customer Care Tasks - Queries and Progressions

Select the option most appropriate to the required task.

Create a Query

A query can be created for a:

  • Customer
  • Prospective customer (for example, a person requesting information about services), or a contact.

The procedure to create a query is similar for both types of customer, except that for potential customers and contacts the name and address of a contact need to be specified. The details are entered automatically by Convergent Billing if a query is created for an existing customer.

Need more information on queries?

Create a Query for a Potential Customer

A query raised by a potential customer or contact is recorded on the Customer Query form. The form can be accessed by selecting either of the:

  • New… button on the Customer Query Search form.
  • Customer Care Queries New Query option.

To create a query:

  1. Display the Customer Query form using one of the above methods.
  2. Enter text about the query in the field displayed on the Request tab. Ensure that as much information as possible is entered to assist in the resolution of the query.
  3. In the Query Type field, select a query type appropriate to the customer's enquiry.

    Tip:

    For help in selecting a query type appropriate to the customer's enquiry, highlight one of the query type options in the drop-down list and click the Query Type Help... button adjacent to the Query Type field. A description of that query type is displayed in a pop-up window and a list of questions to ask the customer about their query.

  4. If required, select the Contact tab and assign a new contact for this query.

    Use the Search... button to locate the appropriate contact and the Assign button to assign that contact to the query, or use the New… button to record a new contact to be assigned.

    Note:

    A contact must be assigned to the query before it can be saved.

  5. If the query relates directly to one of the customer's services or contracts, complete the fields on the Service tab.

    Use the Search… button to locate the appropriate service and the Assign button to assign that service to the query.

  6. Select the General tab. If this tab contains any fields, complete each of the fields.

    Note:

    The presence of fields on this tab depends on the query type specified.

  7. Select the Dates tab. The dates displayed on this tab are entered automatically based on the query type selected.
  8. The Response Due field specifies the date-time that the customer should receive a response to the query, not the date by which the query must be resolved. For example, if the query is a complex one, the Response Due date indicates when the customer should be contacted regarding the progress of the query.
  9. The Resolution Due field specifies the date-time by which the query must be resolved.
  10. Click the Save button on the toolbar. The Customer Query form is displayed in View Only mode.
  11. A unique number is automatically assigned to the query when the query details are saved, and displayed in the Query Number field on the Customer Query form.

    Note:

    The format of the query number is based on the specified query type.

  12. If the query can be resolved immediately, close the query.
  13. If the query cannot be resolved immediately, progress the query.

Create a Query for a Customer

A query raised by a customer is recorded on the Customer Query form. The form can be accessed by conventional methods, and by:

  • Dragging and dropping the Query icon onto the appropriate customer (or customer node) in the Customer Hierarchy form.
  • Clicking on the New Query button on the Customer Detail form.

The method chosen depends on which form is displayed and preferred method.

To create a query using the Customer Hierarchy form:

  1. Ensure that the name of the customer for whom the query is to be created is displayed in the left-hand panel of the Customer Hierarchy form.
  2. Drag the Query icon and drop it onto the appropriate customer's name. To do this, click the Query icon and, while holding down the left mouse button, move the cursor until the required customer's name is highlighted. Now drop the icon by releasing the mouse button.

     

    Symbol

    Description

    This icon can be dragged and dropped on one of the customer nodes listed in the left-hand panel of the Customer Hierarchy form to create a new query.

     

    As the icon is dragged into the left-hand panel of the Customer Hierarchy form the cursor changes from one depicting a stop symbol to one depicting an arrow. If the icon is dropped when the stop symbol is displayed, no action is taken by Convergent Billing.

     

    Symbol

    Description

    This cursor symbol is used to indicate that the Query icon can be dropped onto one of the customer nodes listed in the left-hand panel of the Customer Hierarchy form.

    This cursor symbol is used to indicate that dropping the Query icon will be ignored by Convergent Billing; that is, no action will be taken.

     

    When dropped onto the appropriate customer name, the Customer Query form is displayed in Insert mode.

  3. Enter text about the query in the field displayed on the Request tab. Ensure that as much information as possible is entered to assist in the resolution of the query.
  4. In the Query Type field, select a query type appropriate to the customer's enquiry.

    Tip:

    If assistance is required in selecting a query type appropriate to the customer's enquiry, highlight one of the query type options in the drop-down list and click the Query Type Help... button adjacent to the Query Type field. A description of that query type is displayed in a pop-up window and a list of questions to ask the customer about their query.

  5. If required, select the Contact tab and assign a new contact for this query.

    Use the Search... button to locate the appropriate contact and the Assign button to assign that contact to the query, or use the New… button to record a new contact to be assigned.

    Note:

    A contact must be assigned to the query before it can be saved.

  6. If the query relates directly to one of the customer's services or contracts, complete the fields on the Service tab.

    Use the Search... button to locate the appropriate service and the Assign button to assign that service to the query.

  7. Select the General tab. If this tab contains any fields, complete the appropriate fields.

    Note:

    The fields displayed on this tab depend on the query type specified.

  8. Select the Dates tab.

    The dates displayed on this tab are entered automatically based on the query type selected.

    The Response Due field specifies the date-time that the customer should receive a response to the query, not the date by which the query must be resolved. For example, if the query is complex, the Response Due date indicates when the customer should be contacted regarding the progress of the query.

    The Resolution Due field specifies the date-time by which the query must be resolved.

  9. Click the Save button. The Customer Query form is displayed in View Only mode.

    A unique number is automatically assigned to the query when the query details are saved, and displayed in the Query Number field on the Customer Query form.

    Note:

    The format of the query number is based on the specified query type.

  10. If the query can be resolved immediately, close the query.
  11. If the query cannot be resolved immediately, progress the query.

To create a query using the Customer Detail form:

  1. View the appropriate customer’s details.
  2. Click the New Query… button. The Customer Query form is displayed in Insert mode.
  3. Enter text about the query in the field displayed on the Request tab. Ensure that as much information as possible is entered to assist in the resolution of the query.
  4. In the Query Type field, select a query type appropriate to the customer's enquiry.

    Tip:

    If assistance is required in selecting a query type appropriate to the customer's enquiry, highlight one of the query type options in the drop-down list and click the Help... button adjacent to the Query Type field. A description of that query type is displayed in a pop-up window and a list of questions to ask the customer about their query.

  5. If required, select the Contact tab and assign a new contact for this query.

    Use the Search... button to locate the appropriate contact and the Assign button to assign that contact to the query.

  6. If the query relates directly to one of the customer's services or contracts, complete the fields on the Service tab.

    Use the Search... button to locate the appropriate service and the Assign button to assign that service to the query.

  7. Select the General tab. If this tab contains any fields, complete the appropriate fields.

    Note:

    The fields displayed on this tab depend on the query type specified.

  8. Click the Save button on the toolbar. The Customer Query form is displayed in View Only mode.

    Convergent Billing automatically assigns a unique number to the query when the query details are saved, and displayed in the Query Number field on the Customer Query form.

    Note:

    The format of the query number is based on the specified query type.

  9. If the query can be resolved immediately, close the query.
  10. If the query cannot be resolved immediately, progress the query.

View a Query

To view a query:

  1. Select the Customer Care Queries Manage Queries option. The Customer Query Search form is displayed.
  2. Enter the query number, work group, or query type (or any combination).
  3. Select the Query option in the Search for group and either the All or Opened option in the Status group. Select the Opened option if you want to restrict the search to unresolved queries (that is, still open). Select the All option if you want the search to include both open (unresolved) and closed (resolved) queries.
  4. Click the Search button. Any queries that match the specified selection criteria are retrieved and listed on the Customer Query List form.
  5. To view a specific query, double-click the query or highlight the query and click the Details... button. The Customer Query form is displayed in View Only mode with details of the selected query.

Determine the Status of a Query

To determine the current status (open or closed) of a query, use one of the following methods:

  • Search for and display the appropriate query. The current status of the query is displayed in the Status field on the Customer Query form.
  • List the queries for a specific customer on the Customer Hierarchy form.

    The current status of each query is displayed in the Status column in the right-hand panel of the form.

Both the Open Queries and Closed Queries display options on the Customer Hierarchy form may need to be selected to view all queries associated with a customer.

Progress a Query

To progress a query:

  1. View the appropriate customer query.
  2. Click the Progress button. A message is displayed, prompting you to confirm that you want to progress the query to the specified work group. By default, the work group displayed is that associated with the query type selected.
  3. If required, select an alternative work group.
  4. Click Yes to progress the query to the nominated work group.

Resolve (Close) a Query

A query recorded in Convergent Billing can be resolved in the following ways:

  • Immediately

    If you can provide the enquirer with the details they require, without having to forward the query to a work group to get it resolved, you can enter text on the Resolution tab and close the query; there is no need to progress the query further.

  • By progressing it to a work group

    If you cannot provide the enquirer with sufficient information, you must progress the query to the appropriate work group.

To resolve a query without progressing it to a work group:

  1. Ensure that the appropriate query is displayed on the Customer Query form.
  2. Select the Resolution tab and enter text detailing how the enquiry was resolved.
  3. Click the Save button. A unique query number is assigned to the query.
  4. If the query needs a response date and time assigned to it, click the Update button, then click the Respond button.
  5. Click the Save button. The date and time that the query was responded to can then be viewed on the Dates tab.
  6. Click the Resolve button. When prompted, confirm that the query is being closed.
  7. Click Yes to close the query. As a result of this action, the query status is set to 'Closed'.

To progress a query to a work group:

  1. Ensure that the appropriate query is displayed on the Customer Query form.
  2. Click the Query Progressions… button. The Customer Query Progress History form is displayed.
  3. Select the most recent progression and click the Details… button. The Customer Query Progress Detail form is displayed in View Only mode.
  4. Click the Update button on the toolbar. The Customer Query Progress Detail form is displayed in Update mode.
  5. Select the Resolution tab and enter text detailing how the enquiry was resolved.
  6. Click the Save button on the toolbar.
  7. If the query was resolved, click the Resolve button. When prompted, click the Yes button to close the query.
  8. If the query is not resolved, click the Progress button to forward the query to another work group.
  9. When prompted, select a work group and click Yes. As a result of this action, a progression is created and the resolution text from the query is copied to the Progress tab of the new progression.

View Customer Usage Details

To view a customer’s usage details:

  1. Select the Accounts Account History option. The Account History form is displayed in View Only mode.
  2. Enter the appropriate account number in the Account Number field and tab out of the field or click the Refresh History button. The account’s transaction details are displayed.
  3. Click the Usage Details… button. The Usage Detail Search form is displayed.
  4. Enter the appropriate search criteria.
  5. Specify the appropriate type of usage details to be included in the search by selecting the appropriate option in the Search for group.
  6. To include all events associated with the specified customer node and its child nodes, select the Show all usage liability? check box.
  7. Click the Search… button. The Call Usage Display form is displayed. One or more service numbers are listed in the Service Number column.
  8. To view the usage details of a specific service number, double click on the appropriate number. The usage details are displayed in the Usage Details grid on the right-hand side of the form.

List Progressions

A list of the progressions associated with a specific query can be obtained by:

  • Searching for the progressions associated with a specific query. The progressions are listed on the Customer Query Progress List form.
  • Click the Query Progressions button on the Customer Query form. The progressions are listed on the Customer Query Progress History form.

Need more information on progressions?

View a Progression

View a progression by:

  • Searching for the progressions associated with a query and then viewing the relevant progression.
  • Clicking the Query Progressions button on the Customer Query form and then viewing the relevant progression.

To search for and view a progression:

  1. Select the Customer Care Queries Manage Queries option. The Customer Query Search form is displayed.
  2. Enter the query number.
  3. Select Progress-All in the Search For field.
  4. Click the Search button. Any progressions associated with the specified query are retrieved and listed on the Customer Query Progress List form.
  5. To view a specific progression, double-click the progression or highlight the progression and click the Details... button. The Customer Query Progress Detail form is displayed with details of the selected progression.

To access and view a progression using the Customer Query form:

  1. Search for and display the query with the relevant progressions. The selected query is displayed on the Customer Query form.
  2. Click the Query Progressions… button. Any progressions associated with the specified query are listed on the Customer Query Progress List form.
  3. To view a specific progression, double-click the progression or highlight the progression, and click the Details... button. The Customer Query Progress Detail form is displayed with details of the selected progression.

Need more information on progressions?

Work on a Progression

To work on a progression:

  1. View the appropriate customer query.
  2. Click the Query Progressions… button. The Customer Query Progress History form is displayed with each of the progressions associated with the specified query.
  3. Select the last progression and click the Details... button. The Customer Query Progress Detail form is displayed.
  4. Click the Update button. The Customer Query Progression Detail form is displayed in Update mode.
  5. In the Action tab, record any actions taken to resolve the query.

    Note:

    Record the actions taken even if the query is not fully resolved

  6. In the Resolution tab, record the results of the actions taken to resolve the query. The record of actions taken is used as the basis for any further progressions or as the resolution text of the original query.
  7. Click the Save button on the toolbar.
  8. If the query has been resolved, resolve the progression.
  9. If the query cannot be resolved, forward the progression.

Need more information on progressions?

Forward a Progression

To forward a progression to another work group:

  1. Ensure that the appropriate progression is displayed on the Customer Query Progress form and that the details have been saved.
  2. Click the Progress button.
  3. When prompted, select a new work group and click Yes. As a result of this action, a new progression is created and the resolution text from the current progression is copied to the Progress tab of the new progression.

Need more information on progressions?

Resolve a Progression

Ensure that work on the progression is completed.

To resolve a progression (and therefore resolve the associated query):

  1. Ensure that the details on the Customer Query Progress Detail form have been saved.
  2. Click the Resolve button. A prompt is displayed for confirming that the query is to be closed.
  3. Click Yes to close the query. As a result of this action, the resolution text of the progression is copied to the Resolution tab of the associated query and the query status is set to Closed.
