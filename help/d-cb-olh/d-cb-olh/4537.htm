Rating Circuit

Using the rating server (trerate) interface, events can be rated and their resultant charges returned to the caller. An event that is associated with a reservation of customer credit is assigned a unique reservation ID.

Note:

The use of the Event Socket Interface (ESI) for a rating circuit is deprecated.

Rating Pipeline

A rating pipeline requires a specially configured Event Normalisation (ENM) process, listening on a named pipe or a TCP/IP socket connection for an incoming event. When the ENM receives an event, the ENM normalises it and passes it to an Event Rating (ERT) process for rating in the normal way.

The resulting event and the charge records are passed to an Event Rating Output (ERO) process, which transmits the data to its configured destination. The destination can be directly to the database, a remote host using a TCP/IP socket, or SQL*Loader files for later upload to the database. If normalisation fails, or if the ERT fails to rate an event, an error event is passed to the ERO and no charges are output.

Note:

For the:

  • ENM, reading events from a TCP/IP socket is deprecated and provided only for backward compatibility.
  • ERO, outputting normalised events and charges to a remote host using a TCP/IP socket, or to SQL*Loader files is deprecated and provided only for backward compatibility. SQL*Loader is not supported for rating events that perform real-time account updates or use rating subtotals.
