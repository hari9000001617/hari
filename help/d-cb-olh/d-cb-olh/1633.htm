Functions

A function is a specialised variable that takes zero or more parameters as input and returns a single value as output. A function has the same basic properties as other variables (Name, Start Date, End Date, Description, Data type, Context, and Application Environment), but also includes properties specific to functions (for example, function definitions).

  • Syntax
  • Parameters
  • Calls.

In Convergent Billing, expressions and functions are similar, in that both process code to return a value, and both are evaluated by the EPM. The difference between them is that a function is passed zero or more parameters.

Functions in Convergent Billing are similar to functions in other programming languages as they can:

  • Assign a name to a sequence of one or more expressions.
  • Be called and executed when required; for example, to validate a service number, a sequence of checks on an entered value must be performed. Because the sequence of steps is required in more than one place, a function can be written to identify the sequence of steps and give the sequence of steps a name. The function can then be called from multiple places within Convergent Billing.

The following types of function are available.

 

Type

Description

EPM built-in

Intrinsic part of EPM (for example, abs()). EPM built-in functions are executed by the EPM parser within the current process.

EPM callback

EPM extensions built into various processes and TRE servers. EPM callback functions are specific to particular server or client processes and execute locally within the specific process, if available in that process.

User-defined

Written using EPM (may be supplied with Convergent Billing or user created). User-defined functions invoked by a remote function call can be called by external applications using the TRE Application Programming Interface (API).

Transaction Engine (TRE) Application Programming Interface (API). The API provides the interface between the TRE and a remote host in the form of TRE function calls for the following programming languages:

  • C
  • Object Pascal
  • Perl
  • Java.

Context: Convergent Billing, Process Engine

See also Transaction Engine.

 

Functions can be defined to execute locally or remotely:

  • Local execution occurs in the context of, or local to, the function caller (for example, the client or a TRE server). Remote execution takes place outside the context of, or remote from, the function caller.
  • Remote execution occurs in a server of the TRE that is advertising the Tuxedo service associated with the function.

For more information about functions and function definition, refer to the Configuration Fundamentals Guide for Singleview Convergent Billing.
