Data Interface Language

The Convergent Billing Data Interface Language (DIL) is used to define the structure of an incoming event record data stream to the interpreter in the Event Normalisation (ENM) process. The interpreter breaks the data stream down into its constituent records, and each record into its constituent fields. Rules and expressions can then be used to perform calculations and assign data from these fields into direct variables.

The ENM uses these direct variables to construct normalised events, which can then be rated and stored in the database. If no rules or expressions have been defined for an input field, the interpreter moves on to the next field; in effect, the value of such a field is ignored, though its length is used to advance through the data stream.

The DIL is loosely based on the protocol language known as Abstract Syntax Notation One (ASN.1), defined by the International Telecommunications Union (ITU). For the complete definition of ASN.1, refer to the ITU-T Specification of Abstract Syntax Notation One (ASN.1) X.208 Fascicle VII.4.

Note:

Though primarily intended for parsing event data, the DIL is not limited to this sole function. The DIL can be used, for example, to process payment records, calling TRE remote functions to insert the payment details into the database.

The DIL is used to decode a sequence of bytes that make up one or more event records, breaking each record up into its constituent fields. Where necessary, these fields can be manipulated, using the Convergent Billing Expression Parser Module (EPM), and eventually converted to the Direct Variables that make up a Normalised Event. The resulting event can then be rated, to generate charges for the appropriate service.

A DIL definition is made up of keywords, type definitions, identifiers, semantics, and included definitions.
