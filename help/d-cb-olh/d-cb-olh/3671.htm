Rerated File Search Form

The Rerated File Search form is used to search for one or more rerated files, using a specific field, criteria, and method.

File that has been rerated. Rerating may be done if errors occurred during the rating process. Files may also need to be re-rated if product or tariff changes were made.

Context: Convergent Billing

See also Tariff, Rated Event, and Rating Engine.

The lower part of the form is used to list rerated files returned by the search that match the specified criteria.

Rerated files are used for reprocessing of normalised events received with an error status. By default, all records active on the date the search is performed are included in the search.

Multiple instances of the form can be displayed by clearing the Reuse form? check box.

Determines whether or not multiple instances of a detail form are displayed when a specific object is selected in the right-hand panel of the form and the Detail… button is clicked.

Note:

To display multiple instances of the appropriate form, ensure that the check box is not selected.

Associated Actions

Viewing a Normalised Event Error

To view a normalised event error:

  1. Select the Server Tasks Events View/Assign Error Events option. The Normalised Event Error Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the normalised error events matching the specified criteria is displayed.
  4. Select the appropriate normalised error event and click the Detail… button, or double-click the appropriate normalised error event.
  5. The Normalised Event Error Detail form is displayed in View Only mode.

Updating a Rerated File

To update a rerated file:

  1. View the appropriate rerated file.
  2. Click the Update button on the toolbar. The Rerated File Detail form is displayed in Update mode.
  3. Update the rerated file details as required.
  4. Click the Save button on the toolbar.

Need more information on normalisation?

Defining a New Rerated File

To define a rerated file:

  1. View the appropriate normalised event error.
  2. Click the Update button on the toolbar.
  3. Click the Assign New button. The New Rerated File form is displayed.
  4. Enter the appropriate details in the File Name and Event Source fields and click the Create button. The New Rerated File form is closed, and the error event is assigned to the new file.
  5. Click the Save button on the toolbar.

Need more information on normalisation?

Deleting a Rerated File

To delete a rerated file:

  1. View the appropriate rerated file.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the rerated file is to be deleted.
