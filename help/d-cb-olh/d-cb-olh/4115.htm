Display the Customer Hierarchy Form

To display the Customer Hierarchy form, use one of the following methods:

  • Perform a customer search ensuring that the Always Display Hierarchy option is selected. Use the Reuse last Hierarchy?option to display single or multiple instances of the Customer Hierarchy form.

    If multiple instances of the Customer Hierarchy form are required for display (for example, to focus a search for a specific customer) the Reuse last hierarchy? option should not be selected.

    Caution:

    If selected, the Reuse last Hierarchy? option overrides the display of the customer hierarchy in a new form; the customer hierarchy is displayed in the existing form.

    If a single customer record is returned from a search, that customer’s details are displayed on the Customer Detail form. If more than one customer record is returned, the records are listed in the left-hand panel of the Customer Hierarchy form. The required record can then be selected and its details displayed on the Customer Detail form.

    If selected, the Always Display Hierarchy? option overrides the automatic display of the Customer Detail form when only a single record is retrieved; in such cases the Customer Hierarchy form is displayed.

  • Click the Hierarchy… button on the Customer Detail form.

Expand a Customer Hierarchy

To expand a customer node:

  1. Highlight the customer node to be expanded.
  2. Click the Expand button or double-click the customer node name.

To fully expand a customer hierarchy, repeat steps 1 and 2 for every parent node.

Alternatively, click the '+' symbol to the left of the node name.

Note:

Only parent nodes (nodes having one or more child nodes) can be expanded. They have a '+' symbol to the left of the node name.

Expanding a node after it is collapsed displays all child nodes, regardless of whether a child node matches the initial search criteria used to select the (customer) nodes. Because there may be a large number of child nodes, the maximum number of child nodes displayed can be limited by configuration. If a parent node that has more child nodes than the limit is expanded, a warning message is displayed.

Collapse a Customer Node

To collapse a customer node:

  1. Highlight the customer node to be collapsed.
  2. Click the Collapse button or double-click the customer node name.
  3. Alternatively, click the '-' symbol to the left of the node name.

To collapse a customer's entire hierarchy:

  1. Highlight the root customer node.
  2. Click the Collapse button or click the '-' symbol to the left of the node name.

Note:

Only parent nodes (nodes having one or more child nodes) can be collapsed, and only if they are expanded.

Display More than one Customer Hierarchy

When moving a product between nodes in different customer hierarchies, or when moving a customer node between customer hierarchies, both customer hierarchies must be displayed.

To do this:

  1. Using the Customer Search form, search for the first instance of the customer hierarchy. Ensure that the Always Display Hierarchy? option is selected. The source Customer Hierarchy form is displayed.
  2. Using the Customer Search form, search for the second instance of the customer hierarchy. Ensure that the Always Display Hierarchy? option is selected, and Reuse Last Hierarchy? is not selected. The destination Customer Hierarchy form is displayed.
  3. Expand each hierarchy until the customer node containing the product to be moved (the source node) and the customer node to receive the moved product (the target node) are displayed.

Create a Child Customer Node

To create a child customer node:

  1. Display the appropriate customer node in the left-hand panel of the Customer Hierarchy form.
  2. Drag and drop the Customer icon onto the appropriate customer node in the left-hand panel. The Customer Detail form is opened in Insert mode.
  3. As the icon is dragged into the left-hand panel of the Customer Hierarchy form, the cursor changes from depicting a stop symbol to an arrow symbol. If the icon is dropped when the stop symbol is displayed, no action is taken by Singleview Convergent Billing.
  4. Complete the customer details in the upper part of the form.
  5. Select and then complete each tab relevant to the customer type specified.
  6. Repeat steps 2 to 4 for each child node.
  7. Click the Save button on the toolbar.

If a customer node is created or updated with a start or end date outside of the displayed hierarchy's date, the Customer Hierarchy form is refreshed to the new date. The refreshed form displays only the updated node and the node's parent from the record's effective start date. Any sibling or other nodes displayed previously are not displayed.

A child customer node inherits the customer type of the parent node. Many fields on the Customer Detail form default according to the customer type. If the customer type is later changed, the fields default to the new customer type, except for the Invoice Format field.

Move a Customer Node

Restrictions on Moving a Customer Node

Move Multiple Customer Nodes

A child customer node can be moved:

  • To another location within the same hierarchy
  • Out of a hierarchy, creating a new root customer node
  • To a different hierarchy.

To perform one of these actions, the following privileges must be available:

  • Update privileges on the customer being moved.
  • View privileges on the customer node that will become the new parent customer node of the node being moved.
  • View privileges for blacklisted customers (if the customer node to be moved is blacklisted).
  • Privileges to move products or child customer nodes within a hierarchy.
  • Privileges to move products or child customer nodes between hierarchies.

Depending on the type of customer node move, preferences can be set to determine how the move is performed.

If changes to the customer records are required directly following the move, preferences can also be set to display the Customer Detail form for each of the moved customer nodes.

If this is the first move since logging in, the Move Preferences form is displayed automatically. If not, and the move preferences need to be changed, they can be changed manually. The following procedure assumes that the move preferences have been set.

To move a customer node:

  1. Display and fully expand the customer hierarchy for the appropriate customer. Use two instances of the Customer Hierarchy form if necessary.
  2. If moving a customer node within a hierarchy or to a different hierarchy, select the customer node and drag and drop it on to the appropriate customer node.
  3. If moving a customer node out of a hierarchy (that is, a child node is to become a root node), select the child node and drag and drop it onto the hierarchy node. The Move Confirm form is displayed.
  4. Enter the appropriate date range. Depending on security settings, the Start Date and End Date fields consist of a single component, the date, or two components, date and time. Click the Continue button.

    The customer node being moved has its start date usually set to the customer hierarchy search date. If this date is less than the current date and the user does not have historical update privileges, the start date is set to the current date.

  5. The forms displayed throughout the rest of the move are determined by the move preferences previously selected.

Note:

The invoice cycle for all customers being moved is adjusted to be the invoice cycle of the new root customer node.

Move Multiple Customer Nodes

Restrictions on Moving a Customer Node

Move a Customer Node

Multiple customer nodes can be moved only if they are a group of child nodes under the same parent node. For example, if MegaWork's Melbourne and Sydney offices are both to be moved from the Sales Division and located directly under the head office in the hierarchy, both child nodes can be highlighted and moved simultaneously.

If this is the first move since logging in, the Move Preferences form is displayed automatically. If not, and the move preferences need to be changed, they can be changed manually. The following procedure assumes that the move preferences have been set.

To move multiple customer nodes:

  1. Display and fully expand the customer hierarchy for the appropriate customer. Use two instances of the Customer Hierarchy form if necessary.
  2. If moving child nodes within a hierarchy or to a different hierarchy, select the appropriate nodes from the right-hand panel of the Customer Hierarchy form and drag and drop them onto the appropriate customer node on the left-hand panel of the form.

    To select:

    • A group of customer nodes next to each other, hold down the Shift key, select the first node in the group, then the last node in the group. All nodes in between the first and the last are automatically highlighted.
    • Multiple customer nodes not grouped together, hold down the Ctrl key, and then click each relevant node.
  3. If moving customer nodes out of a hierarchy (that is, a group of child nodes are to become root nodes), select the nodes from the right-hand panel of the Customer Hierarchy form and drag and drop them onto the hierarchy node on the left-hand panel of the form. The Move Confirm form is displayed.
  4. Enter the appropriate date range and click the Continue button.
  5. The forms displayed throughout the rest of the move are determined by the move preferences previously selected.

If one or more of the nodes fail to move, preferences can be set to do either of the following:

  • Cancel all the moves.
  • Move the nodes that can be moved and display a report detailing which moves failed, and why.

Set Customer Node Move Preferences

Depending on the type of customer move, preferences can be set to determine how the move is performed.

If multiple customer nodes are being moved, and one or more of the nodes fail to move, preferences can be set to do either of the following:

  • Cancel all the moves
  • Move the nodes that can be moved and display a report detailing which moves failed and why.

If changes to the customer records are required directly following the move, preferences can also be set to display the Customer Detail form for each of the moved customer nodes.

The Move Preferences form is displayed automatically the first time a move is performed after logging in. If the preferences need to be set again, the form can be accessed manually.

To set move preferences manually:

  1. Display the Customer Hierarchy form.
  2. Select Customer Hierarchy Move Preferences options. The Move Preferences form is displayed.
  3. To set cancel all moves if one fails, select 'Back-out all moves' in the On failed move group.
  4. To move all possible nodes and to display a move report once the move is complete, select 'Continue and show Move Report after all items have been processed' from the On failed move group.
  5. To display the Customer Detail form for each of the moved customer nodes, select 'Display details form in update mode after each individual customer move'.
  6. Click OK.

Move a Product Instance between Customer Nodes

If this is the first move since logging in, the Move Preferences form is displayed automatically. If not, and move preferences need to be changed, they can be changed manually. The following procedure assumes that move preferences have been set.

When moving product instances between nodes in different customer hierarchies, use multiple instances of the Customer Hierarchy form to display both customer hierarchies simultaneously.

Restrictions on Moving a Product Instance

The following restrictions apply when moving a product instance:
  • A product cannot be upgraded or moved if the product instance was created as part of a product offer.
  • A product cannot be moved if its owning customer or target customer is blacklisted at any time during the move period and the user does not have access to view blacklisted customers on the Customer Form.
  • A product cannot be upgraded or moved if the product instance is cancelled, moved, or upgraded at the effective start date or any later date.
  • A product must be assigned to a customer node. A product dragged over a root node or blank space of the hierarchy panel cannot be dropped.
  • Some information regarding the lists and charge categories may need to be entered (when prompted) before the product instance move can be completed.

Caution:

If a product instance is moved, any changes made to it before or after the move start date are not carried over in the move. For example, if the product instance is moved at an effective date before an equipment allocation change, following the move the variation of equipment allocation is not reflected. A warning is displayed so the user can cancel the move.

To move a product instance:

  1. Display the source node in the left-hand panel of the first instance of the Customer Hierarchy form. Ensure that the details for the source node are displayed in the right-hand panel. Select the Products option from the Display for Customer group to ensure that the source product is displayed.
  2. Display the target node in the left-hand panel of the second instance of the Customer Hierarchy form.
  3. Drag and drop the source product instance (located in the right-hand panel of the first instance of the Customer Hierarchy form) onto the target node (displayed in the left-hand panel of the second instance of the Customer Hierarchy form). Ensure that the product instance and not the associated service is dragged and dropped.

    To drag and drop a product instance, click the product instance and, while holding down the left mouse button, move the cursor into the left-hand panel until the target node is highlighted. Now drop the product instance onto the target node by releasing the mouse button.

    As the product instance is dragged from the right-hand to the left-hand panel, the cursor changes from one depicting a stop symbol to one depicting an arrow. If the icon is dropped when the stop symbol is displayed, no action is taken.

  4. The Move Confirm form is displayed. Enter the appropriate date range and click the Continue button.

Deactivate a Customer

To deactivate a customer:

  1. Cancel each product instance associated with the customer.
  2. View the appropriate customer's details.
  3. Click the Delete button on the toolbar. The Cancel Customer form is displayed.
  4. Set the date on which the cancellation is to take effect. Depending on security settings, the date consists of a single component, the date, or two components, date and time. Only the last customer detail history record can be cancelled. The date must be between the start and end dates of the history record. By default, the cancellation date is today's date at 12:00:00 AM.
  5. Select the appropriate Final Invoice Details options.

    The available options are used to specify whether the final invoice is to be generated immediately or included in the next invoice cycle.

    If the customer type references the Customer Cancel entity validation, the Generate final invoice immediately? option is enabled. Select this option to generate and print the final invoice immediately.

    If the Generate final invoice immediately? option is selected, the Show Details… button is enabled. This button enables new values to be entered in any of the 10 General entity validation fields that have been configured. Clicking the Hide Details… button makes these details disappear.

    If invoice generation is not immediate, select the printer to which the invoice should be sent.

    If any outstanding transactions should be included in the next invoice cycle, select the appropriate cycle in the Invoice Cycle for final invoice field. An invoice cycle for final invoice should be chosen if a customer node in a customer hierarchy is being cancelled.

    Note:

    Prepaid without statements customers do not generate invoices. Therefore, the Final Invoice Details options are not displayed for a prepaid without statements customer.

  6. Click the Continue Cancellation button. The message 'Customer cancelled successfully' is displayed.
  7. Click the OK button to remove the confirmation message. Depending on the selected Final Invoice Details group options, one or more post-cancellation tasks may be performed.

Deactivate a Customer Hierarchy

To delete a customer hierarchy:

  1. Display and fully expand the required customer hierarchy.
  2. Deactivate each child node at the lowest hierarchy level.
  3. Repeat step 2 for each level in the hierarchy until only the root customer node remains.
  4. Deactivate the root customer node.

Note:

If the customer account is being finalised, the customer record can be deactivated (cancelled) rather than being deleted.

  2. View the appropriate customer's details.
  3. Click the History button on the toolbar. The Customer History form is displayed.
