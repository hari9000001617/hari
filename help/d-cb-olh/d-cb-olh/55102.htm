Audit Search Form

The Audit Search form is used to search for audit records to track changes to selected tables in the database, using a specific table, user, range of audit creation dates, or SQL Where clause.

The Table drop-down list enables the returned audit records to be restricted to the selected table, and it contains only those tables configured for auditing.

The User drop-down list enables the returned audit records to be restricted to the selected user.

Multiple instances of the Audit Detail form can be displayed by deselecting the Reuse last audit form option.

If multiple instances of the detail form are required for display (for example, to access details for several audit records) the Reuse last audit form? option should not be selected.

If selected, the Reuse last audit form? option overrides the display of information in a new form; in such cases, the details are displayed in the existing form.

Associated Action

Viewing Audit Records

To view audit records:

  1. Select the Maintenance Security View Audit Records option.
  2. Select the database table with audit records to be viewed from the drop-down list in the Table field.
  3. Optionally, enter any other search criteria to restrict the search.
  4. Click the Audit… button to retrieve the audit records. The Audit form displays a grid with one row for each audit record and a column for each field in the database table.

Note:

More than one audit record window can be open at the same time. If this is required, enter new selection criteria in the Audit Search form and click the Audit… button. However, if the new selection is to overwrite the previous selection, select the Reuse last audit form? check box before clicking the Audit… button.
