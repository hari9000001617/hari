Product Upgrade: Allocation Form

The Product Upgrade: Allocation form is used to allocate service and equipment components from one product instance (source product instance) to another (target product instance). Optional services and equipment can also be added to the target product instance.

Specific combination of services and the associated tariff structure offered to a customer based on the framework provided by the corresponding product definition. The original product definition is preserved, with the product instance being tailored to a specific customer's requirements. Any changes made to the product definition affect only product instances created after the changes are made, not existing product instances.

Referred to as a Purchased Product in Customer Management.

Context: Convergent Billing, Customer Management

See also Product Definition and Product Model.

The upper part of the form displays the source product instance service and equipment components.

The lower part of the form displays service and equipment components allocated to the target product instance.

The form is used when:

  • Splitting a multi-service source product instance into two or more separate target product instances
  • Merging two or more separate source product instances into a single multi-service target product instance
  • Upgrading products.

The form provides options, using buttons in the lower part of the form, to:

  • Allocate source product instance components to the target product instance.
  • Allocate selected components.
  • De-allocate selected components.

Individual components can also be dragged from the source product instance and dropped onto the equivalent target product instance component.

Associated Actions

Splitting a Product Instance Manually

Notes and Restrictions
The following terminology is used in the procedures given for splitting and merging product instances:
  • Source product instance (from which services or equipment are to be allocated)
  • Target product instance (to which services and equipment in the source product instance are allocated).

A product instance can only be split or merged if:

  • It was not created as part of a product offer.
  • Update security privileges for the Product Instance Detail form are available.
  • Compatible base products are available.
  • The product is configured to allow splitting or merging.

To split a product instance manually:

  1. View the appropriate (source) product instance.
  2. Select the Upgrade/Move Details tab and click the Update button. The Upgrade button on the tab is enabled.
  3. Click the Upgrade button. A message is displayed prompting selection of the target product.
  4. Select the target product from the drop-down list and click the Yes button. The Product Upgrade: Allocation form is displayed.
  5. To allocate all source product instance components, click the Allocate All button. All components in the source product instance are allocated to the specified target product instance. The Upgrade Status field for each of these components is now set to Used.
  6. To allocate individual source product components, select one of the source product instance components and the equivalent target product instance component and click the Allocate button. The Upgrade Status field for this component is now set to Used.
  7. Repeat this procedure for each component required by the target product instance. If a component is incorrectly allocated, select it and click the Deallocate button.
  8. Click the Done button. A message is displayed to:
    • Indicate the number of source product instance components allocated and the number of components still to be allocated.
    • Prompt whether the remaining components should be allocated to another product instance.
  9. Click the Yes button. A message is displayed prompting selection of the next target product instance.
  10. Repeat steps 4 to 7 for the next target product instance.

When all components of the source product instance have been allocated, the source product instance is split and each of the target product instances are saved with their new components.

On completion of the split, the source product instance is cancelled when each of the target product instances becomes active.

Splitting a Product Instance Automatically

Notes and Restrictions
The following terminology is used in the procedures given for splitting and merging product instances:
  • Source product instance (from which services or equipment are to be allocated)
  • Target product instance (to which services and equipment in the source product instance are allocated).

A product instance can only be split or merged if:

  • It was not created as part of a product offer.
  • Update security privileges for the Product Instance Detail form are available.
  • Compatible base products are available.
  • The product is configured to allow splitting or merging.

To split a product instance automatically:

  1. View the appropriate (source) product instance.
  2. Select the Upgrade/Move Details tab and click the Update button. The Upgrade button on the tab is now enabled.
  3. Click the Upgrade button. A message is displayed prompting selection of the target product.
  4. Select the target product from the drop-down list and click the Yes button. A further message is displayed to:
    • Indicate the number of source product instance components allocated and components still to be allocated.
    • Specify whether to allocate the remaining components to another product instance.
  5. Click the Yes button. A message is displayed prompting selection of the next target product instance.
  6. Select the target product instance from the drop-down list and click the Yes button.
  7. Repeat steps 4 to 6 until the product split is complete. On completion, the Product Instance Detail form is displayed in View Only mode for the last target product instance.

On completion of the split, the source product instance is cancelled when each of the target product instances becomes active.

Merging a Product Instance

Notes and Restrictions
The following terminology is used in the procedures given for splitting and merging product instances:
  • Source product instance (from which services or equipment are to be allocated)
  • Target product instance (to which services and equipment in the source product instance are allocated).

A product instance can only be split or merged if:

  • It was not created as part of a product offer.
  • Update security privileges for the Product Instance Detail form are available.
  • Compatible base products are available.
  • The product is configured to allow splitting or merging.

To merge a product instance:

  1. Assign the target product to the appropriate customer by dragging the product icon onto the customer listed in the Customer Hierarchy form. The Product Instance Detail form is displayed.
  2. On the Detail tab, select one of the service component entries and click the Allocate button. A message is displayed prompting for confirmation that the product dates are correct.
  3. Click the Yes button. A list of services associated with product instances of the specified product type assigned to this customer is displayed on the Service List form.
  4. Select the service to be merged into the target product instance and click the Assign button. The Product Upgrade: Allocation form is displayed. The selected service component is allocated to the target product instance. Either allocate all the source product instance components to the target product instance or allocate individual source product instance components.
  5. To allocate all source product instance components, click the Allocate All button. All components in the source product instance are allocated to the specified target product instance. The Upgrade Status field for each of these components is now set to 'Used'.
  6. To allocate individual source product components, select one of the source product instance components and the equivalent target product instance component and click the Allocate button. The Upgrade Status field for this component is now set to 'Used'. Repeat this procedure for each component required by the target product instance. If a component is incorrectly allocated, select it and click the Deallocate button.
  7. Click the Done button. The Product Instance Detail form is displayed for the target product instance.

    To perform a full merge, repeat steps 2 to 6 for each of the other services required by the target product instance, then save the target product instance. The Status field of the target product instance is now set to 'Active'; the source product instances are set to 'Cancelled'.

    To perform a partial merge, create each of the other services required by the target product instance, allocate equipment (where necessary), then save the target product instance.
