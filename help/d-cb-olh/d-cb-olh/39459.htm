How Do I - Operations - Error Events

Select the option most appropriate to the required task.

View a Normalised Event Error

To view a normalised event error:

  1. Select the Server Tasks Events View/Assign Error Events option. The Normalised Event Error Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the normalised error events matching the specified criteria is displayed.
  4. Select the appropriate normalised error event and click the Detail… button, or double-click the appropriate normalised error event.
  5. The Normalised Event Error Detail form is displayed in View Only mode.

View a Normalised Event Error from a Normalised File

To view a normalised event error from the Normalised File Detail form:

  1. View the appropriate normalised file.
  2. Select the Error Events tab to display any error events returned for the file.
  3. To display details of the error event, highlight the error event in the grid and click the Details… button.
  4. The Normalised Event Error Detail form is displayed in View Only mode.

Update a Normalised Event Error

To update a Normalised Event Error:

  1. View the appropriate normalised event error.
  2. Click the Update button on the toolbar. The Normalised Event Error Detail form is displayed in Update mode.
  3. Update the normalised event error details as required.
  4. Click the Save button on the toolbar.

Need more information on normalisation?

Delete a Normalised Event Error

To delete a Normalised Event Error:

  1. View the appropriate normalised event error.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the normalised event error is to be deleted.

Correct Corrupted Events

Events that cannot be parsed by the ENM are written to the error directory as corrupted events. To resolve corrupted events:

  1. Go to the error directory.
  2. View the event to determine the cause of the corruption.
  3. Correct the event data. In severe cases, it may be necessary to refer the problem to the originator of the event.
  4. Move the corrected file to the input directory.
  5. Rename the corrected file to reflect its origin and status.
  6. Rate the file.

Correct Error Events

If an error is detected during normalisation or rating, the event is called an error event. To resolve error events:

  1. View the normalised event error
  2. Determine if the event is required. If not, run a Purge Error Events task to purge the event. If it is, continue.
  3. Determine what caused the error. It may be in the event data or a tariff, derived attribute, function, or other rating component.
  4. Resolve the error by correcting the appropriate component or event data.
  5. For a single error event, assign the event to a rerated file. For multiple error events, run the appropriate schedule type to reprocess the events.

Correct Incorrect Charges

If normalised events and charges are created without apparent error, but the charges are consistently incorrect:

  1. Determine what caused the incorrect charge generation. It may be in a tariff, derived attribute, function, or other rating component.
  2. Correct the element causing the incorrect charge generation.
  3. Determine the extent of reprocessing required. This depends on the number of events that need to be reprocessed and their effect on any rating subtotals.
  4. Run the appropriate schedule type to rerate or reprocess the events according to reprocessing required. For possible schedule types, refer to the System Operations Guide for Singleview Convergent Billing.
