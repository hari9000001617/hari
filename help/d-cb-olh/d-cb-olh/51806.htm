External Definition Namespace Definition Form

The External Definition Namespace Definition form is used to create, update, and delete external definition namespaces. External definition namespaces ensure the uniqueness of associated external definitions.

Associated Actions

Defining an External Definition Namespace

To define an external definition namespace:

  1. Select the Maintenance Server External Definition External Definition Namespace option. The External Definition Namespace Search form is displayed.
  2. Click the New... Button. The External Definition Namespace form is displayed.
  3. Enter a namespace name in the Namespace Name field. The namespace name must be unique.
  4. Enter a namespace abbreviation prefix in the Prefix field.
  5. If a specific location is required, enter an absolute path in the Schema Location field that describes the location of the XML schema for the namespace.
  6. If generating an XML schema with unqualified elements, select the Unqualified Elements? checkbox
  7. Complete the Description field for the namespace, if required.
  8. Click the Save button on the toolbar.

Viewing an External Definition Namespace

To view an external definition namespace:

  1. Select the Maintenance Server External Definition External Definition Namespace option. The External Definition Namespace Search form is displayed.
  2. Enter the search criteria and select the search method.
  3. Click the Search button. Namespaces matching the search criteria are displayed in the lower section of the form.
  4. Select the namespace to update and click the Detail... button. The External Definition Namespace form is displayed in View only mode.

Generating an External Definition

External definitions can be generated from XML schema files.

Restrictions apply when generating external definitions from XML schema files (see the System Configuration Guide for Singleview Convergent Billing).

Caution:

XML schema files must be validated with a third party tool before importing them using the EDXmlSchemaRead&() function. This function performs basic checks for conformance to external definition specifications, but assumes that the XML is valid and correctly aligned with XML Schema 1.0.

To generate external definitions from an XML schema file:

  1. Select the Server Tasks One-Off Task option. The One-Off Task form is displayed.
  2. In the Schedule Type field, select the External Definition XML Schema Read schedule type from the drop-down list.
  3. In the Schema Files field, enter the name of the XML schema file.

    The name of the XML schema file cannot have UNIX shell meta characters, and cannot be a directory.

  4. In the ED Generation Mode field, select the generation mode from the drop-down list.
  5. Click the Save button.
  6. Complete and validate the generated external definitions (see the System Configuration Guide for Singleview Convergent Billing).

Validate an External Definition

To validate an external definition:

  1. Select the Maintenance Server External Definition External Definition option. The External Definition Search form is displayed.
  2. Enter the search criteria and select the search method.
  3. Click the Search button. External definitions matching the search criteria are displayed in the lower section of the form.
  4. Select the required external definition and click the Detail... button. The Viewing External Definition form is displayed in View only mode.
  5. On the Settings tab, click the Validate... button.

    For information about validation checks that are performed, refer to the System Configuration Guide for Singleview Convergent Billing.

Generating XML Schema

To generate XML schema from an external definition namespace, all external definitions in the namespace need to be validated.

To generate XML schema from an external definition namespace:

  1. Select the Maintenance Server External Definition External Definition Namespace option. The External Definition Namespace Search form is displayed.
  2. Enter the search criteria and select the search method.
  3. Click the Search button. Namespaces matching the search criteria are displayed in the lower section of the form.
  4. Select the required namespace and click the Detail... button. The External Definition Namespace form is displayed in View only mode.
  5. Click the Generate XML Schema... button.

    The name of the XML schema file is derived from the value of Namespace Name.

    The location of the XML schema file is derived from the value of Schema Location, if specified. If Schema Location is not specified, the default location is $ATA_DATA_SERVER_CONFIG/xsd. If the value of Schema Location is specified with environment variables, the environment variables are substituted by their values.
