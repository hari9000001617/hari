Create a Customer

Customer details are recorded on the Customer Detail form, which can be accessed by:

  • Using the Customer Search form
  • Dragging and dropping the customer icon into the left-hand panel of the Customer Hierarchy form.

Although both methods can be used to create customers, the second method is primarily used to create customer nodes within a customer hierarchy.

To create a customer using the Customer Search form:

  1. Select the Customer Care Customers New Customer option. The Customer Search form is displayed.
  2. Enter the appropriate search criteria on the Details tab. If part or all of the customer's address is available, click the Address tab and select an address format from the drop-down list, then enter the address details.
  3. Ensure that the Create a new customer option is selected and click the New… button. The system performs a search of the database and then displays either the Customer Detail form or the Customer List form.

    When the New… button on the Customer Search form is clicked, a search of the database is performed based on the information specified on the search form.

    • If no records are found that match the criteria, the Customer Detail form is displayed in Insert mode.
    • If one or more records that match the information about the search form are found, those records are displayed on the Customer List form. Details of each record (using the Details… button) can then be displayed, and if an appropriate match does not exist, a new record can be created by clicking the New… button on the Customer List form; the Customer Detail form is then displayed in Insert mode.

    Any information entered on the Customer Search form is inserted into the appropriate fields on the Customer Detail form irrespective of the mechanism by which the Customer Detail form is displayed.

    If the Customer List form is displayed, view each of the records displayed. If an appropriate match does not exist, click the New… button on the Customer List form. The Customer Detail form is displayed in Insert mode.

  4. Complete the basic customer details in the upper part of the form.
  5. Select and then complete each tab relevant to the customer type specified.
  6. Click the Invoice Formats… button on the Invoicing tab. The Customer Invoice Format form is displayed.
  7. Select an invoice format.
  8. Click the Save button on the toolbar.

To create a customer using the technique of dragging and dropping:

  1. Display the Customer Hierarchy form.
  2. To create a:
    • root customer node, drag and drop the Customer icon onto the Hierarchy node (located at the top of the left-hand panel).
    • child customer node, drag and drop the Customer icon onto the appropriate customer node in the left-hand panel.

      Node that is a direct subordinate (or child) of a node (referred to as a parent customer node) located at the next highest level of the customer hierarchy.

      Context: Convergent Billing, Customer Management

      Note:

      A child customer node can also be the parent customer node of a node (or nodes) located at a lower level in a customer hierarchy.

      See also Customer Hierarchy, Parent Customer Node, and Root Customer Node.

    In both cases, the Customer Detail form is opened in Insert mode.

    As the icon is dragged into the left-hand panel of the Customer Hierarchy form, the cursor changes from depicting a stop symbol to depicting an arrow. If the icon is dropped when the stop symbol is displayed, no action is taken by the system.

     

    Symbol

    Description

    This cursor symbol is used to indicate that the customer icon can be dropped onto one of the customer nodes (listed in the left-hand panel of the Customer Hierarchy form) or the hierarchy node.

    This cursor symbol is used to indicate that dropping the customer icon will be ignored by Convergent Billing; that is, no action will be taken.

     

  3. Complete the basic customer details in the upper part of the form.
  4. Select and then complete each tab relevant to the customer type specified.
  5. Click the Save button on the toolbar.

Create a Customer Hierarchy

To create a customer hierarchy:

  1. Create a root customer node.
  2. Create one or more child customer nodes.
  3. Create one or more child customer nodes for each customer node created in step 2.
  4. Create one or more child customer nodes for each customer node created in step 3.

View Customer Details

To view a customer's details:

  1. Select the Customer Care Customers Manage Customer option. The Customer Search form is displayed.
  2. Enter the name of the customer in the Customer Name field.
  3. Enter additional search criteria in the other fields, if required.
  4. If part or all of the customer's address is available, select an address format from the drop-down list, then enter the details.
  5. Select one of the following options:
    • Always Display Hierarchy?

      If a single customer record is returned from a search, that customer’s details are displayed on the Customer Detail form. If more than one customer record is returned, the records are listed in the left-hand panel of the Customer Hierarchy form. The required record can then be selected and its details displayed on the Customer Detail form.

      If selected, the Always Display Hierarchy? option overrides the automatic display of the Customer Detail form when only a single record is retrieved; in such cases the Customer Hierarchy form is displayed.

    • Reuse Last Hierarchy? option to list customers on single or multiple instances of the Customer Hierarchy form.

      If multiple instances of the Customer Hierarchy form are required for display (for example, to focus a search for a specific customer) the Reuse last hierarchy? option should not be selected.

      Caution:

      If selected, the Reuse last Hierarchy? option overrides the display of the customer hierarchy in a new form; the customer hierarchy is displayed in the existing form.

  6. Select the Search for customers option and click the Search… button.

    If the search returns:

    • A single customer record, the Customer Detail form is displayed (if the Always Display Hierarchy? option was not selected).
    • More than one customer record, or the Always Display Hierarchy? option was selected, the Customer Hierarchy form is displayed listing each of the customer records that matched the search criteria in a new form. If the Reuse Last Hierarchy? option was selected, the customer records are displayed on the last Customer Hierarchy form.
  7. To view:
    • Items associated with a specific customer record, click the customer name in the left-hand panel; the items (contacts, products, services, equipment, queries, and contracts) associated with that customer are listed in the right-hand panel (if all available display options are selected).

       

      Example 1

       

      To list all queries associated with the current customer node, select the Open Queries and Closed Queries display options.

      Example 2

       

      To list all components associated with the current customer node, select all thirteen display options.

       

    • Details of a specific customer record, double-click the appropriate customer record, or select the record and click the Details… button. The Customer Detail form is displayed in View Only mode.

Deactivate a Customer

To deactivate a customer:

  1. Cancel each product instance associated with the customer.
  2. View the appropriate customer's details.
  3. Click the Delete button on the toolbar. The Cancel Customer form is displayed.
  4. Set the date on which the cancellation is to take effect. Depending on security settings, the date consists of a single component, the date, or two components, date and time. Only the last customer detail history record can be cancelled. The date must be between the start and end dates of the history record. By default, the cancellation date is today's date at 12:00:00 AM.
  5. Select the appropriate Final Invoice Details options.

    The available options are used to specify whether the final invoice is to be generated immediately or included in the next invoice cycle.

    If the customer type references the Customer Cancel entity validation, the Generate final invoice immediately? option is enabled. Select this option to generate and print the final invoice immediately.

    If the Generate final invoice immediately? option is selected, the Show Details… button is enabled. This button enables new values to be entered in any of the 10 General entity validation fields that have been configured. Clicking the Hide Details… button makes these details disappear.

    If invoice generation is not immediate, select the printer to which the invoice should be sent.

    If any outstanding transactions should be included in the next invoice cycle, select the appropriate cycle in the Invoice Cycle for final invoice field. An invoice cycle for final invoice should be chosen if a customer node in a customer hierarchy is being cancelled.

    Note:

    Prepaid without statements customers do not generate invoices. Therefore, the Final Invoice Details options are not displayed for a prepaid without statements customer.

  6. Click the Continue Cancellation button. The message 'Customer cancelled successfully' is displayed.
  7. Click the OK button to remove the confirmation message. Depending on the selected Final Invoice Details group options, one or more post-cancellation tasks may be performed.

    The available options are used to specify whether the final invoice is to be generated immediately or included in the next invoice cycle.

    If the customer type references the Customer Cancel entity validation, the Generate final invoice immediately? option is enabled. Select this option to generate and print the final invoice immediately.

    If the Generate final invoice immediately? option is selected, the Show Details… button is enabled. This button enables new values to be entered in any of the 10 General entity validation fields that have been configured. Clicking the Hide Details… button makes these details disappear.

    If invoice generation is not immediate, select the printer to which the invoice should be sent.

    If any outstanding transactions should be included in the next invoice cycle, select the appropriate cycle in the Invoice Cycle for final invoice field. An invoice cycle for final invoice should be chosen if a customer node in a customer hierarchy is being cancelled.

    Note:

    Prepaid without statements customers do not generate invoices. Therefore, the Final Invoice Details options are not displayed for a prepaid without statements customer.

Change Primary Customer Contact

To replace (change) the primary customer contact:

  1. View the appropriate customer's details.
  2. Open the Contact tab. Details of the current primary customer contact are displayed.
  3. Click the Update button on the toolbar. The Customer Detail form is displayed in Update mode.

    Note:

    Depending on available privileges and the value in the Start Date field of the current record, a Confirm window may be displayed.

  4. If details for the replacement contact are recorded in the system, click the Search... button. The Person Search form is displayed.
  5. Search for and display the details of the replacement contact in the Person Detail form.

    Tip:

    If the search retrieves more than one contact, the Person List form is displayed. Select the replacement contact from the list and click the Assign button on the Person List form. A message is displayed to confirm that the nominated contact is to be assigned to the specified customer. Continue at step 6 of this procedure.

  6. If details for the replacement contact are not recorded in the system, click the New... button. The Person Search form is displayed.
  7. Enter any relevant information about the contact in the search criteria fields and click the New… button. The Person Detail form is displayed in Insert mode.
  8. Record the details for the replacement contact and then save them. A message is displayed prompting confirmation that the nominated contact is to be assigned to the specified customer.
  9. Click the Yes button to replace the details of the current primary customer contact with those of the replacement contact.
  10. Click the Save button on the toolbar.

Assign a Secondary Customer Contact

To assign a secondary customer contact:

  1. View the appropriate customer's details.
  2. Click the Other Contacts... button on the Contact tab. The Customer Contacts form is displayed in View Only mode, listing the secondary contacts (if any) assigned to the specified customer.
  3. Click the Insert button on the toolbar. The Customer Contacts form is displayed in Update mode with the cursor positioned at the start of a blank row.
  4. If details for the contact to be assigned are recorded in the system, click the Search... button. The Person Search form is displayed. Search for and display the details of the contact to be assigned in the Person Detail form.

    Tip:

    If the search retrieves more than one contact, the Person List form is displayed. Select the replacement contact from the list and click the Assign button on the Person List form. A prompt is displayed to confirm that the nominated contact is to be assigned to the specified customer. Continue at step 6 of this procedure.

  5. If details for the contact are not recorded in the system, click the New... button. The Person Search form is displayed. Alternatively, the Person List form may be displayed if the details entered on the Person Search form are similar to another person in the database. Record the details for the contact and then save the details. A message is displayed prompting confirmation to assign the nominated contact to the specified customer.
  6. Click the Yes button.
  7. Repeat steps 3 to 6 for each contact to be assigned to the customer.
  8. Click the Save button on the toolbar.

Change a Secondary Customer Contact

To change (replace) a secondary customer contact:

  1. View the appropriate customer's details.
  2. Click the Other Contacts... button on the Contact tab. The Customer Contacts form is displayed in View Only mode, listing the secondary contacts (if any) assigned to the specified customer.
  3. Click the Update button on the toolbar. The Customer Contacts form is displayed in Update mode.

    Note:

    Depending on available privileges and the value in the Start Date field of the current record, a Confirm window may be displayed before the Customer Contacts form is displayed.

  4. If details for the replacement contact have been recorded in the system, click the Search... button. The Person Search form is displayed. Search for and display the details of the replacement contact in the Person Detail form.

    Tip:

    If the search retrieves more than one contact, the Person List form is displayed. Select the replacement contact from the list and click the Assign button on the Person List form. A message is displayed for confirmation that the nominated contact be assigned to the specified customer. Continue at step 6 of this procedure.

  5. If details for the replacement contact are not recorded in the system, click the New... button. The Person Search form is displayed. Enter any relevant information about the contact in the search criteria fields and click the New… button. The Person Detail form is displayed in Insert mode. Record the details for the replacement contact and then save those details.

    A message is displayed for confirmation that the nominated contact be assigned to the specified customer.

  6. Click the Yes button to replace the contact. Click the No or Cancel button to leave the existing contact.
  7. Click the Save button on the toolbar.

Remove a Customer Contact

A primary customer contact cannot be removed, only replaced (changed). Secondary customer contacts can be removed, replaced, or modified whenever required.

To remove (unassign) a secondary customer contact:

  1. View the appropriate customer's details.
  2. Click the Other Contacts... button on the Contact tab. The Customer Contacts form is displayed listing the secondary contacts (if any) assigned to the specified customer.
  3. Select the row containing the contact to be removed.
  4. Click the Delete button on the toolbar. The Customer Contacts form is placed in Update mode and the specified contact is removed.

    Note:

    Depending on privileges and the value in the Start Date field, a Confirm window may be displayed before the Customer Contacts form is displayed.

  5. Click the Save button on the toolbar.

View an Account

The method to access and view a customer’s account depends on whether the account number is known.

If the account number is known:

  1. Select the Accounts Account History option. The Account History form is displayed.
  2. Enter the account number in the Account Number field.
  3. Press Tab or click the Refresh History button. Transaction details associated with the specified account are displayed including the account balance.

If the account number is not known:

  1. View the Customer Details Form for the customer.
  2. To view the primary account, click the Account History… button. The Account History form is displayed with details of the primary account including the account balance.
  3. To view a secondary account, click the Other Accounts… button. The Other Accounts for Customer form is displayed listing the secondary accounts (if any) assigned to the specified customer.
  4. Double-click the account to be viewed or select the account and click the Account History… button. The Account History form is displayed with details of the specified secondary account.

Create a Secondary Account

To create a secondary account:

  1. View the Customer Details form for the appropriate customer.
  2. Click the Update button on the toolbar.
  3. Click the Other Accounts... button. The Other Accounts for Customer form is displayed listing the secondary accounts (if any) assigned to the specified customer.
  4. Click the Insert button on the toolbar. The Other Accounts for Customer form is displayed in Update mode.

    Note:

    Depending on privileges and the start date of the record, a Confirm window may be displayed.

  5. Enter the account number (if known) and/or select the account type. Multiple accounts of the same type can be added to a customer node, if allowed by the account type.
  6. Click the Save button on the toolbar. The account details are saved and the form is reopened in View Only mode. If an account number is not entered, Convergent Billing assigns a number before saving the account details.
  7. To enter additional secondary accounts, repeat Steps 3 to 5.
  8. Close the Other Accounts form.

Create a Query

A query can be created for a:

  • Customer
  • Prospective customer (for example, a person requesting information about services), or a contact.

The procedure to create a query is similar for both types of customer, except that for potential customers and contacts the name and address of a contact need to be specified. The details are entered automatically by Convergent Billing if a query is created for an existing customer.
