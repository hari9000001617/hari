Transaction Engine (TRE) API

The TRE is a configured installation of the Tuxedo transaction processing middleware, and the TRE API is available for C, Perl, Object Pascal, and Java programming languages. The TRE performs the following functions:

  • Acts as the middle tier of a three-tier system in a client/server environment.
  • Provides a mechanism to communicate with external applications.
  • Enables external applications to access the database.

The TRE is used by client and server processes. The TRE can also be used by external applications by means of the API, for the following purposes:

  • Calling TRE functions.
  • Grouping function calls into transactions, with the option to commit or roll back the transaction.
  • Queuing transactions.
  • Subscribing to TRE events, defining an event handler, and checking for events.
  • Posting TRE events.

For a summary of the TRE API function calls, refer to the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 1).

A detailed description of each TRE API function is included in Transaction Engine API.

Programming Constraints

Programming with the TRE API in all languages is subject to the following constraints:

  • The first TRE function call must be treConnect.
  • The last TRE function call must be treDisconnect.
  • Memory for arrays and pass-by-reference variables must be allocated using treMalloc or re-allocated using treRealloc if the size has changed, before any function call refers to the array or variable.
