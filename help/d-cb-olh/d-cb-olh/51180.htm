Table Partition Definition Form

The Table Partition Definition form is used to view, define, and maintain a table partition. The tables that can be partitioned are defined in the TablePartition derived attribute table.

Improves the management and the performance of the rating, billing and archive processes. Database table scalability can often be compromised when tables that hold large quantities of data are seldom accessed, or when data is accessed very frequently. Convergent Billing provides the capability to define and manage table partitions.

Context: Convergent Billing

See also Billing Engine and Rating Engine.

If partitioning is implemented, the CHARGE, NORMALISED_EVENT, NORMALISED_EVENT_ERROR, SUBTOTAL_RATING_VALUE, SUBTOTAL_RATING_DELTA, and NEF_GL_GUIDANCE_DELTA (not required if the GL subsystem is not enabled) tables must always be partitioned for operational reasons (for example, archiving and upgrading).

The upper part of the form is used to enter the names of the table partition and base table used for the partition to be specified.

The lower part of the form consists of the following tabs for viewing and updating the partition table's details:

Details

The Details tab allows the following to be specified:

  • Desired status of the partition.

    The default for new partitions is 'Dropped'.

  • Table partition schedule for performing required partition changes.
  • Schedule task used to perform any required changes to the partition.

    The task defaults to the first pending task for the schedule whenever the Schedule field is changed.

  • Operation to be performed on the table, in addition to the operations required to change the partition's Actual Status to match the Desired Status.

Note:

Compressing partitions is available only if Oracle Advanced Compression is activated.

The following fields are automatically populated and cannot be modified:

  • Actual Status
  • Operation Outcome
  • Operation Step
  • Contains Unbilled charges?
  • Data Tablespace
  • Index Tablespace
  • Exchange Table
  • Archive File
  • Compressed?

Partition Ranges

The Partition Ranges tab displays table columns and date or entity ranges used to partition the table. The table columns cannot be modified. The date and entity ID ranges can be modified using the Desired fields.

The View Graph... button is used to view information about the current and related partitions.

Statistics

The Statistics tab displays the statistics for the partition in reverse date order. All data on this tab is read-only.

The Uninvoiced Count column is only displayed for partitions associated with the CHARGE table.

The following columns are only displayed for partitions associated with the CHARGE or NORMALISED_EVENT tables:

  • Last Sort Count(s)
  • Sort Date
  • Last Sort Duration.

The Task Details... button opens the Task Details form with the task responsible for generating the selected row's statistics. This button is only available to users with sufficient security group privileges.

The form can be displayed in one of three form modes, the current mode being indicated in the form's title.

Associated Actions

Viewing a Table Partition

To view a table partition:

  1. Select the Maintenance Server Table Partitioning option. The Table Partition Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of table partition names matching the specified criteria is displayed.
  4. Select the appropriate table partition name and click the Detail... button, or double-click the appropriate table partition name. The Table Partition Definition form is displayed in View Only mode.

Need more information on table partitioning?

Defining a Table Partition

To define a table partition:

  1. Select the Maintenance Server Table Partitioning option. The Table Partition Search form is displayed.
  2. Click the New... button. The Table Partition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Need more information on table partitioning?

Updating a Table Partition

To update a table partition:

  1. View the appropriate table partition.
  2. Click the Update button on the toolbar. The Table Partition Definition form is displayed in Update mode.
  3. Update the table partition details as required.
  4. Click the Save button on the toolbar.

    Note:

    The From Date and To Date in the Desired group, and From Entity Id and To Entity Id in the Desired group values can only be updated if the Actual Status field is set to 'Dropped' and the Desired Status field is set to other than 'Dropped'.

Need more information on table partitioning?

Deleting a Table Partition

To delete a table partition:

  1. View the appropriate table partition.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the table partition is to be deleted.

Note:

The table partition can only be deleted if the Actual Status field is set to Dropped.
