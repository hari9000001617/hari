How Do I - Security Tasks - Audit Reports

Select the option most appropriate to the task.

Define an Auditing Configuration Item

To define an auditing configuration item:

  1. Select the Maintenance Server Configuration Items option. The Configuration Item Search form is displayed.
  2. Click the New… button. The Configuration Item Definition form is displayed in Insert mode.
  3. Select AUDIT_TABLE from the Configuration Item Type drop-down list. The Configuration Item Name and Sequence Number fields are automatically populated.
  4. Complete the Description field.
  5. Select the required value for the Enabled attribute in the Base Attributes grid.
  6. Select the table to be audited as the Table Name attribute in the Base Attributes grid.
  7. Select the required value for the Include New Data in Audit attribute in the Base Attributes grid.
  8. Click the Save button on the toolbar.

View Audit Records

To view audit records:

  1. Select the Maintenance Security View Audit Records option.
  2. Select the database table with audit records to be viewed from the drop-down list in the Table field.
  3. Optionally, enter any other search criteria to restrict the search.
  4. Click the Audit… button to retrieve the audit records. The Audit form displays a grid with one row for each audit record and a column for each field in the database table.

Note:

More than one audit record window can be open at the same time. If this is required, enter new selection criteria in the Audit Search form and click the Audit… button. However, if the new selection is to overwrite the previous selection, select the Reuse last audit form? check box before clicking the Audit… button.

Edit Configuration Items

To edit a configuration item for a database table to set auditing on or off:

  1. Select the Maintenance Server Configuration Items option. The Configuration Item Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.

    For example, to find a list of configuration items, select Configuration Item Type from the drop-down list in the Search field, and enter the required value AUDIT_TABLE in the Search Criteria field.

  3. Click the Search button. A list of the defined configuration items matching the specified criteria is displayed.
  4. Select the appropriate configuration item and click the Detail… button, or double-click the configuration item to be changed. The Configuration Item form is displayed in View Only mode.
  5. Click Update on the toolbar. The appropriate configuration item is displayed in Update mode.
  6. Update the form:
    1. Select TRUE or FALSE as required for the value of the Enabled attribute in the Base Attributes grid.
    2. Select the appropriate table.
    3. Select TRUE or FALSE as required for the value of the Include New Data in Audit attribute in the Base Attributes grid.
    4. Select TRUE to save all data for both the new and old record.
  7. Click Save on the toolbar.

Reset Oracle Triggers

To control the auditing process by resetting the Oracle triggers:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Click the New… button. The Schedule Definition form is displayed in Insert mode.
  3. Select Reconfigure Auditing from the Schedule Type drop-down list.
  4. Complete the parameters for a one-off task.
  5. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  6. Click the Submit button. The new audit configuration takes effect as soon as the Reconfigure Auditing task has completed successfully.
