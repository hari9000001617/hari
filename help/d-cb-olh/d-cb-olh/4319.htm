How Do I - Product Configuration Tasks - Product Definition

Select the option most appropriate to the required task.

View a Product Definition

To view a product definition:

  1. Select the Maintenance Products Product Definition option. The Product Definition Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of product definitions matching the specified criteria is displayed.
  4. Select the appropriate product definition and click the Detail... button, or double-click the appropriate product definition. The Product Definition form is displayed in View Only mode.

Prepare for Product Definition

Before attempting to define a product, ensure that all the following components required by that product are available:

  • At least one service type
  • Equipment types required by the specified service types
  • Tariffs to calculate the charges (or benefits) for each of the services specified
  • Charge categories to be associated with each of the specified tariffs
  • Entitlement types to be associated with the product
  • Subtotals required to aggregate the specified costed event records
  • Generic Convergent Billing components required by each of the above product components (for example, derived attributes, functions, reference types, and attribute types)
  • Invoice sections or invoice templates.

If the product is not to be used with product offers, facility groups facility groups containing a minimum of one facility can be defined for a base product.

Define a Companion Product

To define a companion product:

Specify Service Types and Equipment Types

To identify service types and equipment types for inclusion in the product:

  1. Select the Service Types tab on the Product Definition form.
  2. In the upper part of the tab, select the required service type and complete the details for that table row.

    When a service type is selected, default values may be inserted in the Minimum Range, Maximum Range, and Creation Order fields. The default values are defined by the equipment type associated with the specified service type. These values can be overridden if required.

    Tip:

    The number inserted in the Creation Order field determines the order in which the service types are displayed. Choose an order that reflects the normal order in which users create a product, or the order in which the services must be created. For example, if equipment is shared by more than one service, the service type that controls the service number should be positioned before other associated service types.

  3. In the lower part of the tab, select the equipment type to be associated with the specified service type and complete the details for each type selected. If a default equipment type is specified for a specific service type, that equipment type is displayed in the list of equipment types.
  4. Repeat steps 2 and 3 for each service type required in the product.

Specify Tariffs

To include a tariff in the product definition:

  1. Select the Tariffs tab on the Product Definition form.
  2. Select the tariff from the drop-down list of tariffs.
  3. Press the Tab key. If available, values are entered in the Default Invoice Text, Charge Category, and Priority columns.

    Note:

    A value is only entered in the columns if default text was specified when the tariff was defined.

  4. If required, enter text (or change the text) in the columns.

    Tip:

    If unsure what priority to give a tariff, select a number that allows the product to be flexible. For example, if there is a possibility of adding companion products with tariffs in the future, select a number that will allow room to assign both lower and higher priorities to other tariffs.

  5. Repeat steps 2 to 4 for each of the other tariffs defined for this product.

Note:

Tariffs associated with entitlement types are automatically included in the product definition and do not need to be specified separately.

Charge Category Examples

A charge category specifies an account type and, optionally, an account, to which a tariff’s charge or benefit is posted, and is an essential part of tariffing. Another account type can also be defined and, optionally, an account, to which a balancing (opposite) entry can be posted (known as the ‘from’ account). The term ‘tariff/charge category pair’ is often used to emphasise that the two objects are independent, but always go together.

Charge categories cannot reference account types that permit multiple accounts of the same type on a customer node. The account inferred from an account type assigned to a charge category must be unambiguous.

A charge category instance is created for every tariff used by a service whenever a product containing the service is sold, creating a product instance. The charge category instance is located at the customer node, the customer (root node), or the service level, depending on the context defined for the associated tariff. A different instance of the charge category can exist at each level, for use by tariffs with different contexts. A charge category instance is created using values from the default charge category specified in the appropriate tariff configuration. The values used can be overridden.

Note:

Only one instance of the charge category is created at each location, no matter how many services reference the tariff. Any defaults for services after the first are ignored.

A charge category instance can be re-defined so that proportions of a charge are posted to more than one account. They can also be edited after creation, to redirect payments to a different account.

When the rating or billing engine processes an event, the process involves looking up details of the service stored at the customer node, identifying the applicable tariffs, and then using the tariffs to calculate the value of each charge or benefit. The appropriate charge category instance is then used to identify the account. The combination of the charge and the account information makes up the charge record in the CHARGE table of the database.

A charge category instance can also define a GL code to assign to the charge or benefit, if required. However, using charge categories to assign GL codes is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance (for more information, refer to the Finance Guide for Singleview).

Modify a Tariff/Charge Category Pair

The association between a tariff and a charge category can be modified by:

  • Changing the charge category configured as part of a tariff.
  • Overriding the charge category when the tariff is included in a product definition.

To change the charge category configured as part of a tariff:

  1. View the appropriate tariff configuration.
  2. Select the Settings tab.
  3. Click the Update button on the toolbar. The Tariff Configuration form is displayed in Update mode.
  4. Select an alternative charge category from the drop-down list.
  5. Click the Save button on the toolbar

To override the charge category in a product definition:

  1. View the appropriate product definition.
  2. Select the Tariffs tab.
  3. Click the Update button on the toolbar. The Product Definition form is displayed in Update mode.
  4. Select an alternative charge category for the appropriate tariff.
  5. Click the Save button on the toolbar.

  2. Select the subtotal from the drop-down list of subtotals.
  3. Press the Tab key. If available, values are entered in the Default Invoice Text column and the GL Code column.

  5. Repeat steps 2 to 4 for each of the other subtotals defined for this product.

Specify Compatible Products

Note:

If the product is to be used with product offers, each base product needs to be configured to be compatible with any companion products with which it will be sold, and no compatibility needs to be specified on companion products.

To specify a compatible product:

  1. Select the Compatibility tab on the Product Definition form.
  2. Create a new table row by clicking the Insert button on the toolbar.
  3. In the Compatible Product column, select the required product from the drop-down list of products.

    Note:

    The list from which a compatible product can be selected is restricted to those products (base and companion) defined in Convergent Billing.

  4. If in step 3 a base product was entered, press the Tab key and set the Reuse permitted? field to either Yes or No.

    The setting of the Reuse permitted? field is used only when upgrading a base product (and merging/splitting a product):. It is is set to:

    • Yes, services and equipment specified in the existing product instance can be transferred to (reused in) the new product instance when the upgrade is performed.
    • No, services and equipment cannot be transferred to the new product instance when the upgrade is performed.

    Whether the services and equipment are reused depends on the service type and equipment type specified in the existing product and new product. To determine this, a simple match is performed. If the service type exists in both the old product instance and the new product instance, the service is reused (if the Reuse permitted? field is set to Yes). A similar match is performed for equipment types.

    If a base product was entered, press the Tab key and set the Manual Allocation? field to either Yes or No.

    If in step 3 a companion product was selected, press the Tab key twice to move the cursor into the Description column.

    Caution:

    Do not select a setting in the Reuse permitted? field; it must remain blank. To delete a value set in this field, press the Backspace key.

  5. Enter a description for the specified product.
  6. Repeat steps 2 to 5 for each of the other products to be specified as compatible with the product being defined.

Specify Facility Groups

Note:

Facility groups are not compatible with product offers.

To include a facility group in the product definition:

  1. Select the Facility Groups tab on the Product Definition form.
  2. Select the required facility group from the drop-down list of facility groups.
  3. Press the Tab key. If available, values are entered in the Included as standard? and Status columns.
  4. If the specified facility group is to be automatically included in every product instance, set the value of the Included as standard? field to Yes. If the facility group is to be available, but not automatically included in a product instance, set the value to No.
  5. Select the appropriate initial status of the facility group in the Status field.
  6. Repeat steps 2 to 5 for each of the other facility groups to be included in the product.

Enable Product Merging and Splitting

The processes of merging and splitting products are similar to allowing product upgrades and downgrades.

Note:

Product merging and/or splitting can be performed only on base products not associated with product offers. It cannot be performed on companion products.

To enable product merging and splitting:

  1. Follow the steps for defining a base product.
  2. Ensure the product definition contains two or more service types (and optional associated equipment types) when service types and equipment types are specified.
  3. Follow the steps for specifying base product compatibility, ensuring that the product definition is compatible with any associated base products.
  4. Set the Reuse permitted? field settings to Yes on the Compatibility tab for each compatible product, to allow service and equipment to be transferred to the new product instance when the split/merge is performed.

Update a Product Definition

Note:

Information in a product definition cannot be changed if it will invalidate product instances based on that product definition.

Effect of Updating Product Components on Product Instances

 

Product Component

Effect of Update on Product Instances

Tariffs

Available immediately. Used next time events are rated.

Note:

Example of how changing a tariff can adversely affect charges.

When updating a product definition, the following changes cannot be made:

  • Removing service or equipment types
  • Increasing service or equipment type minimum values
  • Decreasing service or equipment type maximum values
  • Adding new service or equipment types to product definitions with a minimum
    value > 0.
  • Changing a tariff outside of the start/end date of the bill period.

    Changes to a tariff should be restricted to the start or end of a bill period, ensuring that charges are billed correctly. The following is an example of when a tariff is changed in the middle of a bill cycle.

    • Product 1 is effective 01/01/00 to 15/01/02 and contains Tariff 1.
    • Product 1 is effective 16/01/01 to end of time and contains Tariff 2.

    If the product is sold to a customer on 01/01/01 and the bill run is scheduled for 30/01/01, the charges for the bill run are based on Tariff 2. The bill run picks up the tariff that is effective at the time of the bill run and bases charges for the billing period on that tariff.

Charge Categories

Existing charge categories available immediately.

Additional charge categories only available if the server process to update charge categories is run.

Subtotals

Available immediately. Used next time rated events are aggregated.

Service Types

Updating not permitted if product instances exist.

Equipment Types

Updating not permitted if product instances exist.

Facility Groups

Available to existing product instances, but only when product instance is itself updated.

Facilities

Available immediately in facility groups if the facility is used in a product instance.

Derived Attributes

Existing derived attributes available immediately.

Additional derived attributes only available if the process to update lists is run.

Product Group

None.

Product Icon

Available immediately.

Product Compatibility

None.

None.

 

Limitations when updating a product definition

To update a product definition:

  1. Define any new product components, or modify product components required in the updated product.
  2. View the appropriate product definition.
  3. Click the Update button on the toolbar. The Product Definition form is displayed in Update mode.
  4. Update the product definition details as required.
  5. Click the Save button on the toolbar.
  6. Notify the system or billing administrator that a product has been updated. For the updated product to be available in the next rating/billing operation, some server tasks must be performed.

Add a Companion Product to a Product Offer

Note:

To be available for adding to a product offer, a companion product must be added to the compatibility list for the base product associated with the top-level product offer in the hierarchy.

To add a companion product to a product offer:

  1. Edit the appropriate product offer.
  2. Select the Companion Products tab.
    If the base product is associated with a product offer, the Companion Products tab is used to view and change the status of companion product instances selected for this base product instance. Companion product instances are added using the Offer tab and can be created before the product instance has been completed. Any companion instances created on an incomplete product instance receive the initial status of Product Creation.

    If the base product is not associated with a product offer, the Companion Products tab is used to select one or more companion products to be added to this instance of the base product. The available companion products are defined when the product on which this instance is based, is defined. Companion products can be added to this tab only when the product instance is completed.

  3. Select one of the rows in the product table, click the Insert button, select the product to be added to the offer from the Companion Product drop-down list, and select either Yes or No from the Mandatory drop-down list.

    Note:

    If a companion product is configured as mandatory, it is automatically assigned with the product offer.

  4. If the companion product has a derived attribute table associated with it, edit the derived attribute table.

    Note:

    Any derived attribute tables configured in the companion product definition as mandatory must be populated with at least one row.

  5. Repeat steps 3 and 4 for other companion products to be added to the product offer.
  6. Enter a minimum and maximum number of companion products that must be selected when the product offer is sold.
  7. If necessary, rearrange the order of the products in the product table using the Up and Down buttons.

    Note:

    The order in which the products are defined determines the order in which those product's icons are displayed when that product group is displayed on the Customer Hierarchy form.

  8. Click the Save button on the toolbar.

Post-product Definition Tasks

When a product has been defined, the following further tasks need to be performed before the product can be tested and implemented.

Base Products

  1. Create a top-level product offer based on the product.
  2. Add the product to one or more product groups.
  3. Notify the System Administrator or Billing Administrator that a new product has been created. For the product to be available in the next rating/billing operation, a number of server tasks must be performed.

Companion Products

  1. Add the product to one or more product offers.
  2. Notify the System Administrator or Billing Administrator that a new product has been created. For the product to be available in the next rating/billing operation, a number of server tasks must be performed.
