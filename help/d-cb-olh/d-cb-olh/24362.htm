Authentication Server (authsvr)

The Authentication server provides a means of validating client entry to the TRE and enforces licensing conditions. Each machine in a Singleview cluster must have its own Authentication server and, therefore, its own LICENCE configuration item.

If neither a configuration item sequence number nor a configuration item name is provided, the configuration details of LICENCE1 are loaded.

Command-line usage for the Authentication server is:

authsvr [-- [-c <EnvConnectName>] <ConfigItemName>|<ConfigItemSeqNr>]

Parameters

Parameter

Description

-c <EnvConnectName>

Name of the environment variable holding the database connection string the Authentication server uses to connect to the database.

If not specified, the Authentication server uses the connection string specified in the ATASVRCONNECT environment variable.

<ConfigItemName>

Optional.

Configuration item name of a LICENCE configuration item, and is used for retrieving configuration information.

<ConfigItemSeqNr>

Optional.

Sequence number of a LICENCE configuration item, and is used to retrieve configuration information. The LICENCE string can optionally be used to prefix the sequence number for readability purposes.
