External Definition Form

The External Definition form is used for configuring and validating external definition entities. Validated external definitions can have their interfaces generated from the form.

The form's tabs allow for configuration of every aspect of the external definition and its interfaces, including:

  • Namespace
  • Associations with related external definitions
  • Content model and XML representation
  • Types of query interfaces to be generated
  • Linked Convergent Billing entity or Process Engine object
  • Search method
  • Elements or attributes that make up the external definition, if it is a complex structure.

The lower part of the form consists of the following tabs:

Settings

Description

The Description tab of the External Definition form is used to describe the current external definition.

Entity

The Entity tab of the External Definition form defines the Convergent Billing entity or Process Engine object that an external definition is linked to.

CB Group

The CB group defines a Convergent Billing entity and entity validation. The entity validation and associated attribute types define how the data is transformed between internal and external representations including generated views. Attribute types associated with the entity validation can be mapped to external definition elements and attributes on the Element tab.

For a Convergent Billing entity, after an entity validation is selected, the list of associated elements displayed in the Element tab is automatically generated from the entity validation's list of attributes.  Previously-generated elements, not those manually inserted, are removed first.  If the Entity Validation field is cleared, previously-generated elements are also removed from the list of elements in the Element tab.

PE Group

The PE group defines a Process Engine class and type object. Facts associated with the object can be mapped to external definition elements and attributes.

For a Process Engine class, after a value for Object Type is selected,the list of associated elements displayed in the Element tab is automatically generated for the selected Class and Object Type combination.  Previously-generated elements, not those manually inserted, are removed first.  If the Object Type field is cleared, previously-generated elements are also removed from the list of elements in the Element tab.

For a Process Engine class with no Object Type selected, a complete list of elements associated with the selected Class can be automatically generated.  In this case, the Generate Elements button is displayed in the bottom right-hand corner of the Element tab. Clicking this button generates the elements.

Abstract Group

The options in the Abstract group indicate if the external definition is an abstract definition used in a Convergent Billing or a Process Engine context. The indicators are used in real-time interfaces for routing inbound messages through Convergent Billing or Process Engine. This group is only available if the External? check box in the Settings tab is selected.

API Processing

The Sequence Number field in the API Processing group controls the order in which the interfaces of an external definition and its linked definitions are called. Interfaces in a linked set of external definitions are called in an ascending numbered sequence.

External Definition Form

Element Tab

Search

The External Definition Search form is used for searching and listing one or more external definitions.

The search criteria and method are specified in the upper section of the form. The lower section lists the external definitions returned by the search that match the specified criteria.

Clicking the Detail... Button displays the External Definition form for the external definition selected in the lower section.

Multiple external definition definitions can be viewed simultaneously by deselecting the Reuse Form? check box.

Determines whether or not multiple instances of a detail form are displayed when a specific object is selected in the right-hand panel of the form and the Detail… button is clicked.

Note:

To display multiple instances of the appropriate form, ensure that the check box is not selected.

Associated Actions

Defining an External Definition

To define an external definition:

  1. Select the Maintenance Server External Definition External Definition option. The External Definition Search form is displayed.
  2. Click the New... Button. The External Definition form is displayed in Insert mode.
  3. Enter the external definition configuration.
  4. Click the Save button on the toolbar.

Note:

Default external definition configurations can be generated from entity validation using the ExternalDefnGenerate&() function.

Need more information on external definitions?

Generating an External Definition View

To define an external definition that is eligible for view generation:

  1. Select the Maintenance Server External Definition External Definition option. The External Definition Search form is displayed.
  2. Click the New... Button. The External Definition form is displayed in Insert mode.
  3. Complete the View Name field.
  4. In the View Generation drop-down list, select Pending.
  5. Configure remaining fields as required.
  6. Validate the configuration using the Validate... button on the Settings tab.
  7. Click the Save button on the toolbar.
  8. Run the dbsync server utility on the command line.

View generation can be initiated manually by a call to the biExternalDefnViewGenerate&() function.

Generating an External Definition Interface

To define an external definition and generate interfaces for it:

  1. Select the Maintenance Server External Definition External Definition option. The External Definition Search form is displayed.
  2. Click the New... Button. The External Definition form is displayed in Insert mode.
  3. Select the External? check box.
  4. In the Interface Generation drop-down list, select Pending.
  5. In the Content Model drop-down list, select a content model other than Simple Content.
  6. Select relevant check boxes from the Access group to configure the interface functions to be generated.
  7. Configure other fields as required.
  8. Click the Save button on the toolbar.
  9. Click the Validate... button on the Settings tab. The validation process is initiated.
  10. In the Status drop-down list, select Validated to ensure that validation continues.
  11. Click the Generate Interface... button. The interface generation process is initiated.

    Note:

    The Interface Generation field is updated with the results of the interface generation process.
