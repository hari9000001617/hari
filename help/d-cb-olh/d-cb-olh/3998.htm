Variable Definition History Form

The Variable Definition History form is used to view a:

  • List of records for the specified variable.

    Named entity containing a value that is referenced in Convergent Billing. A variable is a field in a database record or a more complex expression based entity (containing other variables, calculated values, and constants).

    Convergent Billing supports the following types of variable:

    • Local variables
    • Direct variables
    • Entity validation variables
    • Functions
    • Derived attributes
    • Subtotals
    • Tariffs.

    Variables have specialised properties such as data type, context, and application environment, and are generally used in expressions.

    Context: Convergent Billing

    See also Application Environment, Context, Data Type, Derived Attribute, Direct Variable, Entity Validation, Expression, Function, Lists, Subtotal, and Tariff.

    The list of records is displayed in chronological order and includes the current record, one or more historical records (if available), and one or more future records (if defined).

    Record having a start date in the future.

    Context: Convergent Billing, Customer Management

    See also Current Record and Historical Record.

    Record having an end date before today.

    Context: Convergent Billing

    See also Current Record and Future Record.

    Record having a start date before or equal to today, and an end date equal to or after today.

    Context: Convergent Billing, Customer Management, Process Engine

    See also Future Record and Historical Record.

  • Specific record and, if appropriate privileges are available, to update records.

The form can be displayed in one of three form modes, with the active mode indicated in the status bar.

A form can be displayed in one of the following modes:

Form Mode

Description

View Only mode

Allows information held in Convergent Billing to be viewed, but not updated (modified).

Insert mode

Allows new information to be recorded (entered) in Convergent Billing.

Update mode

Allows information held in Convergent Billing to be viewed and updated (modified).

Viewing an Historical Data Record

To view a historical record:

  1. Display the details of the current data record on the appropriate form. For example, to display the details of a customer, access and display the Customer Detail form with the customer's details.
  2. Click the History button on the toolbar. The appropriate history form is displayed listing each of the historical records associated with the current record.
  3. To view a specific historical record, double-click the appropriate entry in the list of historical records, or select one of the entries and click the Details... button. The relevant form is displayed with details of the historical record.
