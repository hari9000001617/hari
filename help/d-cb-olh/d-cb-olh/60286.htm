Customer Charge Category Form

The Customer Charge Category form is used to view and update a list of charge categories for the specified customer.

Person or organisation using one or more products. Each customer is given a unique primary account against which invoices are raised.

Context: Convergent Billing, Customer Management

See also Account, Contact, Customer Node, Customer Type, Product, and Product Offer.

Identifies the default account type to which a calculated tariff charge or benefit is posted (allocated). For services, the charge category also identifies a specific account number. The association between a tariff and a charge category is specified when the tariff is defined and is referred to as the tariff/charge category pair.

By default, the charge category associated with a tariff is used each time that tariff is used. However, the charge category can be overridden when defining a customer type, service type, customer node, or service.

Context: Convergent Billing, Configuration Studio

See also Account Type and Tariff.

The list of charge categories is displayed in chronological order and includes Charge Category, Number, Proportion, To Account Number, To General Ledger, To Account Class, From Account Number, From General Ledger, and From Account Class fields.

Note:

Using charge categories to assign GL codes is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance.

The form can be displayed in one of two form modes, with the current mode indicated in the form’s title.

Associated Actions

Viewing a Charge Category

To view a charge category:

  1. Select the Maintenance Accounts Charge Categories option. The Charge Category Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the defined charge categories matching the specified criteria is displayed.
  4. Select the appropriate charge category and click the Detail... button, or double-click the appropriate charge category. The Charge Category Definition form is displayed in View Only mode.

Charge Category Examples

A charge category specifies an account type and, optionally, an account, to which a tariff’s charge or benefit is posted, and is an essential part of tariffing. Another account type can also be defined and, optionally, an account, to which a balancing (opposite) entry can be posted (known as the ‘from’ account). The term ‘tariff/charge category pair’ is often used to emphasise that the two objects are independent, but always go together.

Charge categories cannot reference account types that permit multiple accounts of the same type on a customer node. The account inferred from an account type assigned to a charge category must be unambiguous.

A charge category instance is created for every tariff used by a service whenever a product containing the service is sold, creating a product instance. The charge category instance is located at the customer node, the customer (root node), or the service level, depending on the context defined for the associated tariff. A different instance of the charge category can exist at each level, for use by tariffs with different contexts. A charge category instance is created using values from the default charge category specified in the appropriate tariff configuration. The values used can be overridden.

Note:

Only one instance of the charge category is created at each location, no matter how many services reference the tariff. Any defaults for services after the first are ignored.

A charge category instance can be re-defined so that proportions of a charge are posted to more than one account. They can also be edited after creation, to redirect payments to a different account.

When the rating or billing engine processes an event, the process involves looking up details of the service stored at the customer node, identifying the applicable tariffs, and then using the tariffs to calculate the value of each charge or benefit. The appropriate charge category instance is then used to identify the account. The combination of the charge and the account information makes up the charge record in the CHARGE table of the database.

A charge category instance can also define a GL code to assign to the charge or benefit, if required. However, using charge categories to assign GL codes is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance (for more information, refer to the Finance Guide for Singleview).

Defining a Charge Category

To define a default charge category:

  1. Select the Maintenance Accounts Charge Categories option. The Charge Category Search form is displayed.
  2. Click the New... button. The Charge Category Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. If required, assign the appropriate account number.
  5. Click the Save button on the toolbar.

Updating a Charge Category

To update a default charge category:

  1. View the appropriate charge category.
  2. Click the Update button on the toolbar. The Charge Category Definition form is displayed in Update mode.
  3. Update the charge category details as required.
  4. Click the Save button on the toolbar.

Need more information on charge categories?

Deleting a Charge Category

To delete a default charge category:

  1. View the appropriate charge category.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the charge category is to be deleted.
