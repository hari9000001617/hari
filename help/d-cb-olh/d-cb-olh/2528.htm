Customer Query Progress Form

The Customer Query Progress form is used to:

  • View details of a progression.
  • Record information in the Action and Resolution tabs.

The upper part of the form displays basic details of the associated query, such as the type of query, date on which the query was raised, and current status of the query.

The lower part of the form consists of the following tabs used to enter and view specific details about the progression:

Progress

The Progress tab displays a copy of the query text (entered in the Request tab of the original query) or a copy of the progress text (entered in the Resolution tab of the previous progression).

Contact

The Contact tab is used to view the recorded details about the person making the enquiry.

Customer

The Customer tab is used to view the caller’s name, address, and contact number.

Service

If the query relates directly to one of the customer’s services, the Service tab can be used to specify that service.

Contract

The Contract tab is used to view the details about the contract associated with this query.

Action

The Action tab is used to record any actions taken to resolve the original query. Actions should be recorded on this tab even if full resolution os the query has not been achieved.

Resolution

The Resolution tab is used to enter details about how the query was resolved (fully or partially).

If the progression is to be forwarded to another work force, the text entered on this tab is used as the progress text for the next progression. If the progression (and therefore the query) is now resolved, the text is used as the resolution text of the original query.

Information must be entered in the Resolution tab before the query can be progressed or resolved.

General

The General tab is used to view the information entered when the query was created.

Note:

The presence and number of fields on this tab depends on the query type selected.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

A form can be displayed in one of the following modes:

The Respond button is enabled when the form is in Update mode, the query is not closed, and a response date not set.

The Progress button is enabled when the form is in View Only mode, the query is not closed, and no information is entered in the Resolution tab.

The Resolve button is enabled when the form is in View Only mode, the query is not closed, it was not progressed, and information entered in the Resolution tab.

Associated Actions

Creating a Query

A query can be created for a:

  • Customer
  • Prospective customer (for example, a person requesting information about services), or a contact.

The procedure to create a query is similar for both types of customer, except that for potential customers and contacts the name and address of a contact need to be specified. The details are entered automatically by Convergent Billing if a query is created for an existing customer.

Viewing a Query

To view a query:
  1. Select the Customer Care Queries Manage Queries option. The Customer Query Search form is displayed.
  2. Enter the query number, work force, or query type (or any combination).
  3. Select the Query option in the Search for group and either the All or Opened option in the Status group.
    1. Select the Opened option if the search should be restricted to unresolved queries (that is, still open).
    2. Select the All option if the search should include both open (unresolved) and closed (resolved) queries.
  4. Click the Search button. Any queries that match the specified selection criteria are retrieved and listed on the Customer Query List form.
  5. To view a specific query, double-click the query or highlight the query and click the Details... button. The Customer Query form is displayed in View Only mode with details of the selected query.

Progressing a Query

To progress a query:
  1. View the appropriate customer query.
  2. Click the Progress button. A message is displayed, prompting confirmation that the query is to be progressed to the specified work force. By default, the work force displayed is that associated with the query type selected.
  3. If required, select an alternative work force.
  4. Click Yes to progress the query to the nominated work force.

Resolving a Query

A query recorded in Convergent Billing can be resolved in the following ways:
  • Immediately

    If the enquirer can be provided with the details they require, without having to forward the query to a work force to get it resolved, enter text on the Resolution tab and close the query; there is no need to progress the query further.

  • By progressing it to a work force

    If the enquirer cannot be provided with sufficient information, progress the query to the appropriate work force.

To resolve a query without progressing it to a work force:

  1. Ensure that the appropriate query is displayed on the Customer Query form.
  2. Select the Resolution tab and enter text detailing how the enquiry was resolved.
  3. Click the Save button. A unique query number is assigned to the query.
  4. If the query needs a response date and time assigned to it, click the Update button, then click the Respond button.
  5. Click the Save button. The date and time that the query was responded to can then be viewed on the Dates tab.
  6. Click the Resolve button. When prompted, confirm that the query is being closed.
  7. Click Yes to close the query. As a result of this action, the query status is set to 'Closed'.

To progress a query to a work force:

  1. Ensure that the appropriate query is displayed on the Customer Query form.
  2. Click the Query Progressions… button. The Customer Query Progress History form is displayed.
  3. Select the most recent progression and click the Details… button. The Customer Query Progress Detail form is displayed in View Only mode.
  4. Click the Update button on the toolbar. The Customer Query Progress Detail form is displayed in Update mode.
  5. Select the Resolution tab and enter text detailing how the enquiry was resolved.
  6. Click the Save button on the toolbar.
  7. If the query was resolved, click the Resolve button. When prompted, click the Yes button to close the query.
  8. If the query is not resolved, click the Progress button to forward the query to another work force.
  9. When prompted, select a work force and click Yes. As a result of this action, a progression is created and the resolution text from the query is copied to the Progress tab of the new progression.

Viewing a Progression

View a progression by:

  • Searching for the progressions associated with a query and then viewing the relevant progression.
  • Clicking the Query Progressions button on the Customer Query form and then viewing the relevant progression.

To search for and view a progression:

  1. Select the Customer Care Queries Manage Queries option. The Customer Query Search form is displayed.
  2. Enter the query number.
  3. Select Progress-All in the Search For field.
  4. Click the Search button. Any progressions associated with the specified query are retrieved and listed on the Customer Query Progress List form.
  5. To view a specific progression, double-click the progression or highlight the progression and click the Details... button. The Customer Query Progress Detail form is displayed with details of the selected progression.

To access and view a progression using the Customer Query form:

  1. Search for and display the query with the relevant progressions. The selected query is displayed on the Customer Query form.
  2. Click the Query Progressions… button. Any progressions associated with the specified query are listed on the Customer Query Progress List form.
  3. To view a specific progression, double-click the progression or highlight the progression, and click the Details... button. The Customer Query Progress Detail form is displayed with details of the selected progression.
