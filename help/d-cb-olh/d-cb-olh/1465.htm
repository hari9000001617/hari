Context-sensitive Help

Context-sensitive help is a feature that provides:

  • Context-sensitive form help for each form.
  • Context-sensitive field help for each form component (fields, buttons, check boxes, and options).

Customising Context-sensitive Help

Convergent Billing context-sensitive help can be customised in the following ways:

  • Convergent Billing forms can have a customised HTML page associated with them.
  • Dynamic forms can have customised field or form help.
  • Field help can be customised for core Convergent Billing forms and dynamic forms.
  • Field help can be associated with an attribute type.

    The help entry associated with an attribute type overrides the context-sensitive help for fields to which the attribute type is applied.

Customising context-sensitive help is performed in Convergent Billing using the following forms:

  • Help Entry Definition form, which is used to define the help text for fields.
  • Form Definition form, which groups the help entry definitions for forms and allows alternative HTML files to be specified.

Attribute Type Definition form, which can associate a field help entry with the attribute type.

For more information about customising context-sensitive help, refer to the Configuration Fundamentals Guide for Singleview Convergent Billing.
