Date Ranging

Date ranging is the mechanism for recording period-specific data for date-ranged entities (not all entities are date-ranged). When a date-ranged entity is created, it is given an effective start date and an effective end date. The dates define the period, or date range, for which data of the entity is effective and is represented as a date-range record.

The start date typically defaults to the current date and the end date is usually unspecified, signifying that the date range has no end. Although an end date that spans to the end of time is displayed as blank, the field is recorded as the maximum date definable in the system (30-Dec-9999 23:59:59).

When an entity is updated, a number of different types of update can be performed depending on the user's access level. The update usually applies to the current record (the date-range record that spans the current date). The update sets the effective end date for the current record, closing that record, and a copy of the record with the specified changes is created with new start and end dates.

Security feature that determines what a user can do and what data they are permitted to access. Security is based on form roles and groups, which in combination, determine a user's privileges. Each user is assigned a specific profile that defines the level of access for each role and group.

The start date defaults to the current date, and the end date is unspecified. As a result of the update, the newly created date-ranged record is the current record, and the previous record is an historical record with a defined date range. A user with the appropriate update access privilege can change an historical record (that is, backdate changes).

Over a period of time, an entity is likely to change a number of times. For example, the contact person changes for a customer. Each time an entity is updated, a new current record and another historical record are created. The timespan of the entity is defined by contiguous date ranges that specify historical, current, and future versions of the data. By creating date-range records, changes to an entity can be tracked, and the entity viewed as the entity exists at a specific point in time; that is, at an effective date.

The default granularity for date ranging is one day (date granularity). That is, changes are recorded for specified effective start and end dates. Some customer-related entities can support date ranging with a granularity of one second (date-time granularity). For entities that support date-time granularity, the time can be specified for each of the effective start and end dates. Date-time granularity allows Convergent Billing to reflect the status of an entity as it changes throughout a day.
