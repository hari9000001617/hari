General Ledger Guidance Entity

General ledger (GL) guidance entities map charges associated with normalised event file and invoice transactions to the affected GL codes in the Sales Journal. GL guidance entities define the GL codes affected by a specific type of transaction. The GL codes affected by a transaction differ depending on the type of transaction and whether the transaction results in accrued, actual, deferred, or earned revenue.

GL guidance entities must be configured to define the GL codes affected by a specific type of normalised event file or invoice transaction when revenue is recognised. GL guidance entities, rather than GL codes, are associated with normalised event file or invoice transactions.

Convergent Billing supports the following types of GL guidance entities:

  • Accrual
  • Actual
  • Deferral
  • Non-billable.

The types are not explicitly displayed in the Convergent Billing client. Each type is distinguished by the combination of Earned But Unbilled (EBUB), Billed and Earned (BE), Billed but Unearned (BUE), and Earned but Unbilled (EBUB) Pending Consolidation GL code pairs.

For more information about GL guidance entities, refer to the Finance Guide for Singleview.
