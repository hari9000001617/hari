Variables

Convergent Billing supports the following types of variables (see the Configuration Fundamentals Guide for Singleview Convergent Billing):

  • Local Variables
  • Direct Variables
  • Entity Validation Variables.

Functions, Derived Attributes, Tariffs, and Subtotals are also regarded as variables in Convergent Billing because they return values. With the exception of local variables whose scope is confined to the current expression block or function, the scope of a Convergent Billing variable is controlled by two attributes, the Application Environment and the Context.

Environment in which a variable or function is used. Must be one of the following:

  • Any (for functions only)
  • Billing
  • Data Entry
  • EVS (for Entity Validation Server (EVS) functions)
  • GL Guidance
  • IGP Report
  • Invoicing
  • PE
  • Rating
  • TRE Table (unavailable for functions).

Note:

Functions with an application environment of Any can be called from Convergent Billing or Process Engine.

The PE application environment is specific to Process Engine functions and direct variables defined in Process Engine. Functions with an application environment of PE are read-only and cannot be deleted. All other application environments are specific to functions and variables defined in Convergent Billing.

Functions with an application environment of EVS are read-only and cannot be deleted. They cannot be specified or updated in Convergent Billing or Process Engine.

Context: Convergent Billing, Process Engine

See also Context and Variable.

Excluding all type suffixes, a name can have a length of up to 64 characters. The first character of a variable name must be an upper or lower case letter or the '_' character. Subsequent characters can also include numbers and the '.' and ':' characters.

An application environment of Any and a context of Any can only be given to a function that references only the parameters it is passed or other Any/Any functions. Variables with other application environments or contexts may not be available in all circumstances in which the function could be called.

The PE application environment is reserved for Process Engine functions, and cannot be specified or updated in Convergent Billing.

Variables can be hidden from display using encryption or masking.
