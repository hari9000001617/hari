Notes on the Comparison Operator

The EPM comparison operator (<=>) compares two values if they are of the same data type. Zero is returned if the values being compared are the same. A positive integer is returned if the first value is greater than the second. A negative integer is returned if the first value is less than the second.

Notes on Array and Hash Operations

Arrays are compared by examining corresponding elements, or by comparing with an undefined value if there is no corresponding element in one of the arrays. An undefined value is compared after all other values, which is consistent with the ordering of NULL values by Oracle. An undefined value is not equivalent to the default value for an undefined element.

A runtime error is raised if Array elements cannot be ordered. For example, an integer cannot be compared to a string, and a string cannot be compared to a hash. A runtime error is also raised if a comparison is made between unequal hash elements.

Arrays are equal if they are the same size, and each element is equal with the corresponding element in the second Array.

Two hashes are equal if they are the same size, they have the same set of keys, and the corresponding key values in each hash are equal.
