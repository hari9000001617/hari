Related Topics

Task Prerequisites

Schedules and one-off tasks can be defined with prerequisites; that is, tasks that must be in a specified state before another task will run. Prerequisite tasks allow complex operational frameworks to be defined to promote process automation and ensure tasks are run in the correct order.

For example, Tasks A and B are prerequisites for Task C, and Task C must be completed before Task D can begin.

Prerequisite tasks do not have to be completed successfully to trigger the dependent task. Tasks can also be defined to start whenever their prerequisite conditions are satisfied, regardless of the specified start date-time.

Schedules

Schedules are required to run tasks at specific times and frequencies in Convergent Billing. Schedules are used to manage all billing tasks such as the rating of events, and the preparation and printing of invoices. Schedules are also used to generate reports. Tasks can be set up to run automatically at predefined intervals or at a specified time and date. The schedule definition is determined in part by the associated schedule type.

When setting up a schedule, an instance on Convergent Billing can be specified to run the schedule. If an instance is not specified, the schedule is run on the default instance defined for the scheduler. (The default instance is defined in the STD configuration item.)

Preconfigured functions can be automatically executed on commencement or completion of a scheduled task, and notification messages provided to assist with system operations.

A schedule specifies:

  • When a task is to be run for the first time.
  • The Convergent Billing instance the schedule is run on.
  • The effective date (used as a base to compute the effective date for tasks that are created).
  • How often the task should be repeated.
  • What program the task needs to perform its function, and what parameters should be passed to it.
  • Selection parameters such as whether a report should include data on all customer nodes or only include data on a specific customer node.
  • Display parameters, such as whether the data included in a report is to be sorted, the sort order, and the sort sequence.
