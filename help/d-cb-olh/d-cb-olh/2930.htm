Normalised Event File Type

The normalised event file type defines the decoding methods used to convert raw events into normalised events. The normalised event file type used for each type of incoming raw event is recorded as an attribute for the Event Normalisation (ENM) process.

Each ENM has a FILE_TYPE configuration attribute which specifies the default normalised event file type supported by the ENM, but this file type can be overridden by any other file type for each normalised event file to be processed.

Decoding of incoming events uses:

  • Built-in decoding methods
  • Data Interface Language (DIL)
  • Additional processing, which can be carried out on decoded records before the values are committed to the database.

    After decoding by built-in decoding or DIL decoding, the direct variables can be further modified using the Expression Parser Module (EPM). Further modification may be required, for example, to translate a value supplied by the switch in its own format into the format required by Convergent Billing.

    Parses and executes expressions. Used to make comparisons, validate input, and calculate values. The EPM provides built-in functions, including mathematical, type conversion, string processing, date functions, subtotal history, invoice data, and form automation.

    Context: Convergent Billing, Process Engine

    See also Built-in Function, Expression, and Form Automation.

    Additional processing is usually required after using the built-in decoding methods. With DIL decoding it is generally more convenient to define additional processing as part of the DIL specification, rather than applying additional expressions after the decoding is complete.
