Event Normalisation

Convergent Billing provides facilities to input events from an almost unlimited number of source types to support the production of a single, consolidated bill. The sources can include call data records, switch data records, and pre-billed records of services from other service providers for re-billing.

In addition to handling 'external' events, Convergent Billing also generates internal events (using a process known as the Rental Generation Process), which are used as the basis for obtaining recurring charges (for example, line and appliance rentals).

Whatever its source, the event data is converted into a standard record format in a process referred to as event normalisation. The records resulting from this process are referred to as normalised events. Each data field in a normalised event is stored as a direct variable and is therefore available for use in tariff calculations during event rating.
