Product Instance Detail Form

The Product Instance Detail form is used to enter, view, and update details about a product instance.

Specific combination of services and the associated tariff structure offered to a customer based on the framework provided by the corresponding product definition. The original product definition is preserved, with the product instance being tailored to a specific customer's requirements. Any changes made to the product definition affect only product instances created after the changes are made, not existing product instances.

Referred to as a Purchased Product in Customer Management.

Context: Convergent Billing, Customer Management

See also Product Definition and Product Model.

The upper part of the form displays the product name, status of this instance of the product, and start and end dates (where relevant).

The lower part of the form consists of the following tabs for entering more specific details:

Detail

The Detail tab of the Product Instance Detail form lists items that must be created for the specified product. Typically, this tab lists the service types for which a service must be created and any associated equipment types. An item of each type listed on this tab must be completed before the product instance is complete and made active.

The following buttons can be used:

  • Create… button to create a service of the required service type.

    Enabled only when one of the service types is selected.

  • Allocate… button to allocate an existing item of equipment to the specified service.

    Enabled only when one of the equipment types is selected.

  • Add Service… button to select an optional service.

    Enabled only when there are optional services available.

  • Add Equipment… button to select an optional equipment item.

    Enabled only when optional equipment items are available.

  • Details… button to view more details on the equipment or service.

The information on this tab displays the current status of each component of the product instance.

Offers (available if the base product is associated with a product offer)

The Offers tab is used to:

  • View details about the product offers and companion products associated with the product instance.
  • Add new product offers or companion products to the current product instance.
  • Add another instance of the selected offer node and its children to the current product instance.
  • Move companion product instances between compatible product offers.

Facility Groups (available if the base product is not associated with a product offer)

The Facility Groups tab is used to:

  • Select one or more of the facility groups to be used by this instance of the current base product (and companion products if any have been added to the base product).
  • Enter details required by that facility group for the current product instance.

Note:

The available facility groups are defined when the product (on which this instance is based) is defined.

Facility groups are listed on this tab only when the product instance is completed.

Companion Products

If the base product is associated with a product offer, the Companion Products tab is used to view and change the status of companion product instances selected for this base product instance. Companion product instances are added using the Offer tab and can be created before the product instance has been completed. Any companion instances created on an incomplete product instance receive the initial status of Product Creation.

If the base product is not associated with a product offer, the Companion Products tab is used to select one or more companion products to be added to this instance of the base product. The available companion products are defined when the product on which this instance is based, is defined. Companion products can be added to this tab only when the product instance is completed.

Upgrade/Move Details

The Upgrade/Move Details tab is used to:

  • Replace the current base product (on which this instance is based) with an alternative, compatible base product. The compatible base products are defined when the product (on which this instance is based) is defined.
  • View details of any product moves.

Industry

The Industry tab is used to display other fields to provide additional (general) information about this product instance, if configured.

General

The General tab displays other fields to provide additional (general) information about this product instance.

Note:

If fields are present on this tab, they should be completed.

The lower part of the form is also used to:

  • View and enter details of any new up-front charges required before the product instance can be activated (using the New Up-Front Charge… button).
  • View details of the customer associated with the product instance (using the Customer Details… button),
  • View changes made to the product instance record (using the Changes… button).

The form can be displayed in one of three form modes, with the current mode being indicated in the form’s title.

A product instance is considered complete if all required service and equipment information has been entered. Only completed product instances can be made active (that is, available for use and inclusion in a bill cycle).

Associated Actions

Viewing a Product Instance

To view summary information about each product instance:

  1. Display the Customer Hierarchy form for the appropriate customer.
  2. Select Products or Companion Products options from the Display for customer group. Summary information about each product instance associated with the specified customer is displayed in the right-hand panel.

Creating a Product Instance

Products are typically implemented with product offers, to simplify product configuration and product instance creation.

If a Product Offer drop-down list is not displayed on the Product Instance Detail form, the product is not associated with a product offer. In this case, follow procedure 2 to create the product instance.

Procedure 1

To create a product instance from a product associated with a product offer:

  1. Assign a product to a customer or customer node.
  2. Select a product offer to use with the product instance. The Offer tab is displayed.
  3. Expand the Offer Hierarchy tree, if required.
  4. Select optional child product offers and companion products by ticking the check boxes beside their entry in the tree. Mandatory child product offers and companion products will be selected automatically.

    Note:

    Placement of a companion product in the tree may affect the charges and entitlements configured for the product. Check configuration in the Companion Products tab to ensure that the desired settings are available.

  5. Click the Apply Changes button. The Companion Products tab is displayed.
  6. Select each of the companion products displayed, and add any further configuration information required.
  7. Allocate equipment to each service (where required).
  8. Create the required services.

    Note:

    One service must be created before the product instance is complete and can become active.

  9. If required, select one or more of the available optional product components.
  10. If required, select one or more of the available lists on the Customer Detail or Service Detail forms and create list items for each of those lists.
  11. To save the product instance, click the Save button on the toolbar.

Procedure 2

To create a product instance from a product not associated with a product offer:

  1. Assign a product to a customer or customer node.
  2. Allocate required equipment to each service.
  3. Create the required services.

    Note:

    One service must be created before the product instance is complete and can become active.

  4. If required, select one or more of the available optional product components.
  5. If required, select one or more of the available facility groups.
  6. If required, select one or more of the available companion products.
  7. If required, select one or more of the available lists on the Customer Detail or Service Detail forms and create list items for each of those lists.
  8. If required, record any up-front charges associated with the product.
  9. Click the Save button on the toolbar.

Modify a Product Instance

A product instance can be modified by:
  • Changing its status by activating, suspending, cancelling, and reactivating the instance.
  • Changing the product offer that the product instance is associated with.
  • Changing the status of a specific service within the product instance, by activating, suspending, temporarily disconnecting, cancelling, and reactivating a service.
  • Changing the status of a facility group, by activating, suspending, cancelling, and reactivating the facility group.
  • Adding one or more of the available facility groups.
  • Changing the status of a companion product.
  • Adding one or more of the available companion products.
  • Changing (upgrading or downgrading) the base product.

Deleting a Product Instance

Caution:

A product instance can only be removed/deleted if it has a status of Product Creation. If a product instance is not in this state, deleting a product instance is identical to cancelling that instance.

To delete an incomplete product instance (a product instance with a status of Product Creation):

  1. View the appropriate product instance.
  2. Click the Delete button on the toolbar.
  3. When prompted, specify the date from which to delete the product instance. Depending on security settings, the date consists of a single component, the date, or two components, the date and time.
  4. When prompted, click the Yes button to confirm that the product instance is to be deleted. The product instance and all associated services are deleted.

Upgrading a Product Instance

Options on the Upgrade/Move Details tab of the Product Instance Detail form cannot be selected while creating a product instance. Product instances that are created as part of a product offer, or have been previously moved, upgraded, or cancelled, cannot be upgraded.

To upgrade a product:

  1. View the appropriate product instance.
  2. Select the Upgrade/Move Details tab.
  3. Click the Update button on the toolbar. The Product Instance Detail form is displayed in Update mode.
    • The From Product button is enabled if the product was merged or is the result of an upgrade.
    • The To Product button is only enabled if the product instance was previously upgraded.
    • Clicking the From Product button displays details of the product from which the product instance being viewed was upgraded or moved.
    • Clicking the To Product button displays details of the product to which the product instance being viewed was upgraded or moved.
  4. Click the Upgrade button. A message is displayed prompting selection of the product to which the current product instance is to be upgraded.
  5. Select the Detail tab and, if necessary, update the services required to make the new product instance active.
  6. Complete the appropriate fields on the Facility Groups and Companion Product tabs for the upgraded product instance.
  7. Click the Save button on the toolbar.

Creating a Service

To create a service:

  1. View the relevant product instance.
  2. Highlight one of the service types listed.
  3. Click the Create… button. A message is displayed prompting confirmation that the start and end dates for the product instance are correct.
  4. Click the Yes button to continue creating the service with the specified dates. The Service Detail form is displayed.
  5. If the dates are not correct, click the No or Cancel button to return to the Product Instance Detail form to change the dates.
  6. Complete the form for the selected service.
  7. Click the Save button on the toolbar. Control is returned to the Product Instance Detail form.
  8. To allocate an optional service to the product instance, click the Add Service… button. The Optional Product Component form is displayed.
  9. Select the optional component and click the Add button. The additional component is added to the Product Instance Detail form, the Optional Product Component form is closed, and control is returned to the Product Instance Detail form.
  10. Complete steps 2 to 6 for each additional component to be allocated to the product instance.
  11. Click the Save button on the toolbar.

Move Companion Product Instances between Product Offers

The companion product must be associated with the origin and destination product offers.

To move a companion product instance from one product offer node to another:

  1. View the product instance for the customer.
  2. Click on the Offers tab.
  3. Ensure that both the origin and destination product offer nodes are visible.
  4. Hold down the CTRL key and drag and drop the instance to the target offer node.

    To drag and drop product instances:

    1. Select the product instances and hold down the left mouse button.
    2. Move the cursor into the left-hand panel until the target node is highlighted.
    3. Release the mouse button to drop the product instance onto the target node.

    As the product instances are dragged from the right-hand to the left-hand panel, the cursor changes from depicting a stop symbol to depicting an arrow. If the icon is dropped when the stop symbol is displayed, no action is taken.

    The Move Confirm form is displayed.

  5. Click the Continue button. The Move Progress form is displayed.

Need more information on product offers?

Remove a Companion Product Instance from a Product Offer

To remove a companion product instance from a product offer node:

  1. View the product instance for the customer.
  2. Click on the Offers tab.
  3. Expand the product offer node.
  4. Deselect the check box next to the companion product instance being removed. All instances of the companion product in that node are removed.
  5. Click the Apply Changes button.

To remove a single companion product instance from a product offer node containing more than one instance of the companion product:

  1. View the product instance for the customer.
  2. Click the Companion Products tab.
  3. Do either of the following:
    • Delete the record by selecting it and clicking the Delete button.
    • Cancel the product instance by selecting Cancelled in the Status drop-down list.

Change the Service Associated with a Product Offer

Changing the service for any child in the same service-specific sub-tree changes it for all product offers and companion products in the sub-tree. However, only the offer node at the top of the sub-tree is assigned to the service.

To change the service associated with a product offer:

  1. View the product instance for the customer.
  2. Click on the Offers tab.
  3. Click the Update button on the toolbar. The Product Instance Detail form is displayed in Update mode.
  4. Select the product offer to be updated.
  5. Select the required service in the Service Specific drop-down list.
  6. Click on the Apply Changes button.

Need more information on product offers?

Associate a Service with a Product Offer

A service can be associated with a product offer if the offer is:

  • Not a top-level offer
  • Configured as a service-specific offer

To associate a service with a product offer:

  1. Display the Product Offer Definition form.
  2. Ensure that the Service Specific? checkbox is selected.
  3. Select a service from the Service drop-down list.
  4. Click the Save button on the toolbar.
