Views

Convergent Billing provides a comprehensive logical data model that can be externally accessed using views. Views provide a universal, stable, and less complex interface than the physical database.

Views provide:

  • Reduced complexity

    The physical data model for Singleview is a meta model and therefore difficult for third parties to access. The logical data model is designed to minimise the meta model effect.

  • Data abstraction

    Views represent the data in a format that reflects the service provider's conceptual view of the data (for example, customers, contacts, accounts, or products).

  • Access that can be upgraded

    Any direct external access to the physical tables cannot be upgraded by core upgrades. The logical data model provides an abstraction layer that helps protect investment placed in external development.

Convergent Billing provides the following types of views:

  • Normal
  • TRE
  • Database

TRE views and database views can be customised, allowing implementation-specific views to be preserved during upgrades. For example, hints can be added to view definitions for performance tuning. A knowledge of SQL and the purpose and use of each view is required for customising views. For more information, refer to the Configuration Fundamentals Guide for Singleview Convergent Billing.

String Columns and Unicode

The string columns in the Convergent Billing database have fixed byte sizes. Characters encoded using UTF-8 may be single-byte or multi-byte. Therefore, if UTF-8 encoding is being used (that is, Unicode is enabled in Convergent Billing), the effective number of characters that can be stored in a string column may be fewer than the column size, depending on the characters that make up the string.
