Entity Validation

Entity validation is the key to extensively configuring the customer care environment of Convergent Billing to look and perform as required. Entity validation provides control over what information is captured by a user, and how it is captured. This is achieved by adding and removing fields from generic Convergent Billing forms, customising generic Convergent Billing fields, and adding field security, field validation, intelligent defaulting, and context-sensitive help.

Entity validation can tailor an entity's data model for integration with external systems. Non-stored attributes that are derived from an entity's stored attributes can be added, external aliases for attributes can be assigned, and the sets of key attributes that identify an entity instance can be configured.

Reference types, attribute types, and key field functions are components for entity validation. Customising a field using entity validation involves associating an attribute type with that field. Configuring the keys that identify an entity instance requires specification of new key field functions.

Note:

Customising fields includes adding new fields. The customer care forms include a General tab, which contains invisible general fields that can be customised to become visible.

The attribute type defines many of the changes to the field such as whether it is based on a reference type, its format, characteristics, hint text, context-sensitive help, field security, field external alias, field validation, and field defaults (more extensive validation and field defaults can be set up within entity validation). The attribute also specifies the addition and defines the derivation of non-stored attributes.

After associating an attribute type with a field, the field can be customised further by entity validation. Fields can also be defined to change their characteristics or default values based on a value entered in a separate field.

Note:

An attribute type can be added to entity validation without a corresponding form field if it defines a derived non-stored attribute.

For more information on entity validation, refer to the Configuration Fundamentals Guide for Singleview Convergent Billing.
