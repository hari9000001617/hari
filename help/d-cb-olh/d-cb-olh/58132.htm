Product Definition Form

The Product Definition form is used to enter, view, and update details of a product definition.

Combination of product components that form a specific type of product. Individual product instances based on the product type or definition can be created.

Context: Convergent Billing, Customer Management, Configuration Studio

See also Base Product, Product, Product Component, and Product Offer.

The upper part of the form displays basic details about the definition such as its name, description, and period for which it is applicable.

The lower part of the form consists of the following tabs for entering and viewing more specific product definition details:

Settings

The Settings tab provides drop-down lists for selecting:
  • Default work force to handle any queries that may result from using the product
  • Initial status of the product
  • Icon to be used to identify the product
  • Product entity validation, used to override the display options for the selected product definition.
  • Service and equipment entity validation, used to override the display options for the selected services and equipment.

The Settings tab is also used to specify whether this is a base or companion product. For companion products, the tab is used to specify if more than one active instance of the same companion product can be assigned to a single base product.

Services

The fields on the Services tab are used to identify the service types and associated equipment types required for this product.

The upper part identifies the service types and the lower part the equipment types.

For a base product the mandatory status of the equipment types and service types is determined by its minimum and maximum ranges. The number of instances of the minimum range indicates the number of equipment or services that are mandatory, a zero minimum range indicates that the equipment or service is not mandatory. The range between the minimum and maximum ranges indicate the number of optional equipment or services that are available. The default values for the minimum and maximum ranges is 1.

Note:

At least one service type must be assigned for each product.

Tariffs

The fields on the Tariffs tab are used to define tariffs and associated charge categories so they are made available in each instance of the product for each service. Tariffs can appear only once in the grid.

Real-time tariffs associated with entitlements are displayed in the Tariff field. These tariffs cannot be changed in the Tariffs tab.

Subtotals

The fields on the Subtotals tab are used to define the subtotals to be made available in each instance of the product for each service.

Note:

Tariffs should not be included in the list of subtotals; a separate tab is provided for tariffs.

The following columns are available:

  • Subtotal, which is used to specify the name of each subtotal defined for the product.

    Specify every defined subtotal, even though each product instance may require only a subset of the available subtotals. The subtotals used by any one product instance depend on the service types selected and the eligibility of each subtotal.

  • Default Invoice Text, which is optional, for specifying text to be displayed on an invoice for each calculated subtotal.

    Any text entered in this field overrides the default text defined for the subtotal.

  • GL Code, which is optional, for specifying the GL code to be associated with the subtotal value.

    By default, the code displayed in this field is that specified for the subtotal. The specified code can be overridden with another or removed from the field.

The use of the GL code specified on this tab is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance.

Compatibility

The fields on the Compatibility tab are used to specify companion products that are compatible with the base product, and can be used in product offers associated with the base product.

If products are added on this tab for a companion product, the product is not available to product offers.

Facility Groups

The fields on the Facility Groups tab is used to identify facility groups that have been defined for the product, whether those facility groups should be included as standard, and whether each facility group is available.

Identifying a facility group means identifying the facilities provided by the facility group. If facility groups are defined correctly, some facility groups can be offered as standard and some as optional, depending on the facilities that each facility group provides.

Derived Attribute Tables (Lists)

The fields on the Derived Attribute Tables (Lists) tab are used to identify each of the derived attributes to be included in the product definition. A table row is inserted for each derived attribute to be included.

Payment Items

The fields on the Payment Items tab is used to:

  • Identify payment items to be included in the product definition.
  • Specify expressions that evaluate to the amount and label to be associated with the payment.

The tab is displayed only for base products because payment items can only be defined for base products.

Offer Tables

The Offer Tables tab defines the derived attribute tables used to store pricing data for companion products associated with product offers. A derived attribute table may be shared by more than one companion product.

The tab is displayed only if the Companion Product check box is selected.

Entitlement Types

The Entitlement Types tab defines the entitlement types available for use with a companion product (see the Customer Care and Product Configuration Guide for Singleview Convergent Billing).

Changes to the tab are reflected on the Tariffs tab, with the list of real-time tariffs associated with the product automatically updated to match the tariffs specified by the current entitlement types.

The tab is only displayed if the Companion Product check box is selected.

Note:

Entitlement types are configuration objects that are delivered as part of an implementation, and are not expected to be updated by the service provider, and are created and maintained in Configuration Studio.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

Associated Actions

Viewing a Product Definition

To view a product definition:

  1. Select the Maintenance Products Product Definition option. The Product Definition Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of product definitions matching the specified criteria is displayed.
  4. Select the appropriate product definition and click the Detail... button, or double-click the appropriate product definition. The Product Definition form is displayed in View Only mode.

Need more information on products?

Defining a Base Product

To define a base product:
  1. Select the Maintenance Products Product Definition option. The Product Definition Search form is displayed.
  2. Click the New... button. The Product Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields in the upper part of the form.

    Tip:

    If a required component was not defined while specifying the product definition, define it and then return to creating the product definition.

  4. Complete the appropriate data entry fields on the Settings tab.
  5. Select each of the other tabs in turn and complete the data entry fields or table rows as required.

    Specifying Service Types and Equipment Types

    Specifying Tariffs

    Specifying Subtotals

    Specifying Compatible Products

    Specifying Facility Groups

    Specifying List Types

  6. Click the Save button on the toolbar.
  7. Perform the necessary post-product definition tasks.

Need more information on products?

Defining a Companion Product

To define a companion product:
  1. Select the Maintenance Products Product Definition option. The Product Definition Search form is displayed.
  2. Click the New... button. The Product Definition form is displayed in Insert mode.
  3. In the upper part of the form, enter the name of the companion product, its description, and the period for which it is to be in effect.
  4. Complete the appropriate fields on the Settings tab, and ensure that the Companion Product? option is selected.
  5. Complete the appropriate fields on the Tariffs tab, Subtotals tab, and optionally, the Compatibility tab, Derived Attribute Tables (Lists) tab, Offer Tables tab, and Entitlement Types tab.

    The Offer Tables tab defines the derived attribute tables used to store pricing data for companion products associated with product offers. A derived attribute table may be shared by more than one companion product.

    The tab is displayed only if the Companion Product check box is selected.

  6. Click the Save button on the toolbar.
  7. Perform the appropriate post-product definition tasks.

Updating a Product Definition

Note:

Information in a product definition cannot be changed if it will invalidate product instances based on that product definition.

Effect of Updating Product Components on Product Instances

 

Product Component

Effect of Update on Product Instances

Tariffs

Available immediately. Used next time events are rated.

Note:

Example of how changing a tariff can adversely affect charges.

When updating a product definition, the following changes cannot be made:

  • Removing service or equipment types
  • Increasing service or equipment type minimum values
  • Decreasing service or equipment type maximum values
  • Adding new service or equipment types to product definitions with a minimum
    value > 0.
  • Changing a tariff outside of the start/end date of the bill period.

    Changes to a tariff should be restricted to the start or end of a bill period, ensuring that charges are billed correctly. The following is an example of when a tariff is changed in the middle of a bill cycle.

    • Product 1 is effective 01/01/00 to 15/01/02 and contains Tariff 1.
    • Product 1 is effective 16/01/01 to end of time and contains Tariff 2.

    If the product is sold to a customer on 01/01/01 and the bill run is scheduled for 30/01/01, the charges for the bill run are based on Tariff 2. The bill run picks up the tariff that is effective at the time of the bill run and bases charges for the billing period on that tariff.

Charge Categories

Existing charge categories available immediately.

Additional charge categories only available if the server process to update charge categories is run.

Subtotals

Available immediately. Used next time rated events are aggregated.

Service Types

Updating not permitted if product instances exist.

Equipment Types

Updating not permitted if product instances exist.

Facility Groups

Available to existing product instances, but only when product instance is itself updated.

Facilities

Available immediately in facility groups if the facility is used in a product instance.

Derived Attributes

Existing derived attributes available immediately.

Additional derived attributes only available if the process to update lists is run.

Product Group

None.

Product Icon

Available immediately.

Product Compatibility

None.

Payment Items

None.

 

Limitations when updating a product definition

To update a product definition:

  1. Define any new product components, or modify product components required in the updated product.
  2. View the appropriate product definition.
  3. Click the Update button on the toolbar. The Product Definition form is displayed in Update mode.
  4. Update the product definition details as required.
  5. Click the Save button on the toolbar.
  6. Notify the system or billing administrator that a product has been updated. For the updated product to be available in the next rating/billing operation, some server tasks must be performed.

Deleting a Product Definition

Caution:

Deleting a product definition invalidates product instances based on that product definition.

To delete a product definition:

  1. View the appropriate product definition.
  2. Click the Delete button of the toolbar.
  3. When prompted, click the Yes button to confirm that the product definition is to be deleted.
