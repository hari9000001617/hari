Online Help for Singleview Convergent Billing 9.1

Security

Convergent Billing security defines which users have access to information and the type of access they have.

Client security is based on profiles that define security privileges for types of user. Each user in Convergent Billing is associated with a single profile.

The profile defines whether a user is authenticated using an internal or external method. User profiles also control login access for internal authentication.

A user can be associated with a person stored in the database, providing additional information about the user. For more information about user, refer to the Security and Auditing Configuration Guide for Singleview.

Profiles are designed to provide access to the data and Convergent Billing forms required to carry out a user's function. A profile is made up of one or more of the following four elements:

  • Security groups, which define the database entities that a user can access.
  • Security form roles, which define the menu items and forms available to a user.
  • Security function roles, which define the database remote functions that a user can access.
  • Work groups, which provide a mechanism for the management of customer queries.

Access levels define the level of security for the following profile entities:

  • Security Groups, restricting the operations a user can perform on data; for example, updating, inserting, and deleting records, or updating and deleting historical records.
  • Form roles, restricting the kind of operations a user can perform using a specific form; for example, viewing, inserting, updating, and deleting records. Some forms have form-specific operations; for example, restricting the ability to insert future-dated payments using the Payment Details form.
  • Work groups, restricting a user's ability to view, insert, or update queries for a work group.

A user definition cannot be defined until the appropriate security profile is defined. The security profile cannot be defined until the appropriate form roles, security groups, and work groups have been defined. If a person record needs to be associated to a user definition, the person record must be defined before the user definition is defined.

Users

Convergent Billing users are assigned a security profile, which provides the security privileges needed to perform specific tasks.

The information recorded for each user includes:

  • A user name and password, used for logging in to Convergent Billing. Passwords are recorded and maintained by Convergent Billing only if internal authentication is enabled for the user.
  • A security profile, which defines the user's access to Convergent Billing and the database, and access privileges to data, menu items, and forms.
  • Personal information such as the full name of the user, email address, and contact telephone numbers. A user record can also be assigned to a person record (of person type 'Employee'), where additional personal information can be recorded.
  • Additional company-specific information, defined using entity validation.
