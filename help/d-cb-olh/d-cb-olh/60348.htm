Rating and Billing Process

Many billing systems restrict data input to call data records, and in some cases, to specific types of call data record. In Convergent Billing, any event that results in a charge or benefit being passed on to a customer can be input to, or generated within, the system.

Events can originate from any source of charge information, including network elements, other service providers, other mediation systems, customer contacts such as one-off rebates, service enquiries, and faults). All such events can be input to Convergent Billing if the format of the data is known and is consistent for each type of input. Because Convergent Billing does not restrict data input to call data records, the term event record refers to a record that contains information that may result in a charge or benefit being passed on to a customer.

The full rating and billing process is divided into the following logical operations:

  • Balance Management, which provides service authorisation, advice of charge information, and customer credit reservations.
  • Event Normalisation, in which input data is converted into a standard record format. The output from this stage is a set of normalised event records.
  • Event Rating, in which the normalised event records are rated (costed) by applying the appropriate tariffs. The output from this stage is a set of costed event charges.
  • Event output, in which the costed event records are stored in the Convergent Billing database.
  • Billing, in which the costed event records are aggregated and any applicable discounts are applied, using Convergent Billing server processes. The output from this stage is a set of billing data records.
  • Invoicing, in which billing data is combined with customer detail records and incorporated into an invoice or statement.
  • Invoice distribution or output, in which invoices and statements are printed or converted for electronic distribution.
