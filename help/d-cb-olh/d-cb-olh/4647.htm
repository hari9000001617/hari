Unicode Support

Unicode is a character encoding system designed to support all languages, regardless of platform, program, and programming language. It provides a unique number, called a code point, for every character.

Convergent Billing supports Unicode in the following ways:

  • The Convergent Billing database, server, and TRE support Unicode, allowing data to be entered, viewed, and manipulated in multiple languages.
  • The Convergent Billing database and server support UTF-8 encoding.
  • The TRE supports both UTF-8 encoding and UTF-16 encoding, enabling integration with many applications that use UTF-16, including Windows.
  • The C, Pascal, and Java TRE clients support UTF-8 and UTF-16.
  • The Perl TRE clients support UTF-8.
  • A single Convergent Billing instance is capable of supporting multiple Windows clients, each using different character sets (however, only one character set per PC is supported).

    Therefore, in an internationalisation context, data can be entered, viewed, and manipulated in the Convergent Billing client in any language supported by the character set of the PC's locale.

The following constraints apply to Convergent Billing's support of Unicode:

  • The Convergent Billing client is not Unicode enabled, so when Unicode is used, conversions are performed between Unicode and the character set as defined by the PC's locale.
  • Separate client machines are required for each distinct character set as each Convergent Billing client can only correctly display characters corresponding to the locale for which the PC is configured. In practice, a user can only work with one character set as configured on the PC they are working on.

Note:

ASCII is available in all character sets.

String columns in the Convergent Billing database are of fixed byte size; for example, the customer name column is 80 bytes. As UTF-8 encoding is a variable length byte encoding between one and four bytes, depending on the characters stored in a column, the effective number of characters stored in a column may be fewer than the column size.

Process Engine and Process Engine business solutions (for example, Customer Management) do not support Unicode.

As Process Engine is not supported for Unicode, Process Engine cannot be used with a Convergent Billing installation.

For more information, refer to the Internationalisation Configuration Guide for Singleview.
