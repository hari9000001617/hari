Output Device Definition Form

The Output Device Definition form defines the UNIX command used to send the output to the required device. Therefore knowledge of UNIX, the Convergent Billing environment, and device configuration to work with output devices is required.

The form includes the following information:

  • Output Image Type, which defines the format of the image stored in the database.
  • Run Command, which is processed in conjunction with the Pipe images to run command? and Concatenate Images? options.
  • Start Command, which is an optional UNIX command executed before a batch of images is output.

    For example, the start command can be used to instruct the printer to select the correct stationery.

  • Stop Command, which is an optional UNIX command that is executed when each batch is completed.

    For example, the stop command can be used to advise a user that the print run has finished. It can also be used to track the batch number, or where in the entire run a specific batch is.

Options and outcomes of the run command are listed in the following table.

 

Option

Result

Neither

Run Command (taking $file as an argument) is executed after each image is output.

Concatenate

Run Command is executed after all images in the batch have been concatenated into $file.

Note:

The only file type that can be concatenated is .txt.

Pipe

Output image is piped to the Run command.

Both

Same as Pipe, except that the pipe is not closed, and is reopened between images.
