Server Processes

Convergent Billing uses a three-tier architecture, in which clients connect to the server by means of the Transaction Engine (TRE), an implementation of Tuxedo middleware. The Tuxedo processes together make up the TRE, used for communication and inter-process control. Server operations specific to Convergent Billing are controlled by a series of processes, all under the control of the Backend Monitor (BMP) process. A control program, the Expression Control Program (ECP), is available for process control from the UNIX command line and in scripts.

All Convergent Billing processes and Tuxedo need to be running for Convergent Billing server operations. The server processes are:

  • Billing
  • Rating
  • Other Server Processes.

Convergent Billing provides other processes used in bill generation and printing. Bill generation and printing processes are run as required, rather than continuously.

Most server functions such as rating, invoice generation, database validation, and archiving, are run as independent tasks. These tasks are usually scheduled from a client workstation, using the STD process, and run by the TLM process at the correct time. The UNIX command line is not required for day-to-day operations.
