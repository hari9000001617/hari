User Definition Form

The User Definition form is used to view, define, and update user definitions; the current mode of the form is indicated in the form's title.

A form can be displayed in one of the following modes:

Form Mode

Description

View Only mode

Allows information held in Convergent Billing to be viewed, but not updated (modified).

Insert mode

Allows new information to be recorded (entered) in Convergent Billing.

Update mode

Allows information held in Convergent Billing to be viewed and updated (modified).

User-defined, is the entity that controls access to Singleview and its elements. A user's access to forms and data, and the range of permitted operations (for example, update or delete), is controlled using a profile. User is shared between Convergent Billing and Process Engine.

Context: Convergent Billing, Process Engine, Configuration Studio

See also Access Level, Security Group, User Profile, Form Role, and Work Group.

The upper part of the form is used to enter general details about the user such as their user name, full name, profile, and email address. The Last Login, Last Logout, and Current Sessions fields, are read-only and display the user's login details.

The lower part of the form:

  • Displays and maintains the user's account status.
  • Records auxiliary attributes such as phone number.
  • Allows for assignment of a person entity to record more information.
  • Includes the Password button to change the user's password.

    Note:

    The button is enabled only if the user has an Oracle account (Enabled? field is selected).

Auxiliary attributes are recorded on the following tabs:

Settings

The Settings tab is used to enter additional details about the user (for example, phone and fax numbers). If more information is required, the user can be entered into Convergent Billing as a person, and assigned to the user definition.

General

The General tab is used to add additional fields to the Operator Definition form using entity validation.

If internal authentication is in use, user login to Convergent Billing is managed by the user's account status. If external authentication is in use, user login is managed by the application providing external authentication; for example, access using single sign-on is managed using Microsoft Active Directory.

The data and forms that a user has access to are controlled by access levels assigned to the elements (for example, form role) of the security profile.

Associated Actions

Viewing a User

To view a user:
  1. Select the Maintenance Security Users option. The User Definition Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of users matching the specified criteria is displayed.
  4. Select the appropriate user and click the Detail… button, or double-click the appropriate user. The User Definition form is displayed in View Only mode.

Caution:

If Process Engine is enabled, user accounts must be defined, updated, and deleted in Process Engine, using the User Tool to ensure integrity of additional Process Engine data.

Defining a User

To define a user:
  1. Select the Maintenance Security Users option. The User Definition Search form is displayed.
  2. Click the New… button. The User Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Note:

The Password button is only available if:

  • All the required fields have been completed and the record has been saved
  • Internal authentication is enabled in the security profile.

    Caution:

    If Process Engine is enabled, user accounts must be defined, updated, and deleted in Process Engine using the User Tool to ensure integrity of additional Process Engine data.

 

Need more information on security?

Updating a User

To update a user's access:
  1. View the appropriate user definition.
  2. Click the Update button on the toolbar. The User Definition form is displayed in Update mode.
  3. Update user details as required.
  4. Click the Save button on the toolbar.

Caution:

If Process Engine is enabled, user accounts must be defined, updated, and deleted in Process Engine using the User Tool to ensure integrity of additional Process Engine data.

Need more information on security?

Deleting a User

To delete a user:

  1. View the appropriate user definition.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the user definition is to be deleted.

Note:

A user cannot be deleted if it is referenced by an entity.

Checking user references during a delete operation can cause timeouts. To avoid this, CSG International recommends disabling users by deselecting the Enabled? checkbox.

Caution:

If Process Engine is enabled, user accounts must be defined, updated, and deleted in Process Engine using the User Tool to ensure integrity of additional Process Engine data.
