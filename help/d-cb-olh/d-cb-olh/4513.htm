Licensing

Convergent Billing is subject to licensing restrictions; the conditions of a licence are negotiated between the Convergent Billing user and CSG International.

Each licence is based on three factors:

  • A period of validity, from the granting of the licence to an expiry date.
  • A specified machine and, optionally, a specific database on that machine.
  • A maximum permitted workload, defined in terms of the number of concurrent users, active customers, active customer nodes, and active service instances.

A licence key can be affected by changes to the specified machine, causing a new key to be required. Changes to hardware such as RAM, CPUs, or disks do not affect the licence key. Changes that may require a new licence key are:

  • Changes to the motherboard
  • Firmware updates
  • New system software installed (AIX)
  • Change to the IP address.

The licence is supplied in the form of a software key, from which a configuration item is constructed in the Convergent Billing database. A Tuxedo server, known as the Authentication server, is responsible for checking the validity and usage level of the licence. Detection of an invalid licence (one that has expired or been tampered with) can have two outcomes:

  • On startup, the Authentication server does not start, and does not permit the other server processes to start. Clients are not able to log in under these circumstances.
  • In normal operation, the Authentication server continues to run. If the key has been tampered with, the previous licence information is retained. However, if either the Event Rating Process (BKR) or the Bill Generation Process (BGP) detect that the licence has expired (or that the licence limits have been exceeded), these processes do not start.

Each machine in a Convergent Billing cluster must have its own Authentication Server and, therefore, its own LICENCE configuration item. The Authentication Server also monitors the workload, issuing warnings as the maximum permitted level for each entity (user, customer, customer node, or service instance) is approached. The following action is taken when the workload reaches its maximum permitted levels:

  • If the number of concurrent users is exceeded, additional users are not permitted to log in.
  • If the number of active services is exceeded, rating and billing are disabled.
  • If the number of active customers or customer nodes is exceeded, billing is disabled.

However, Convergent Billing will remain available for work on entities created within the workload limit.

The licence does not distinguish between temporary and permanent services. For example, GSM roamers and open access services (IGF) are treated in the same way as permanent services when the Authentication Server assesses workload.

The Authentication server also checks the currency of the licence each time a user logs in. If an invalid licence signature is detected or licence recalculation is overdue, the Authentication Server automatically initiates a task to perform a licence recalculation. Users are unable to log in until the licence recalculation task is successful. No further licence recalculations are issued by the Authentication server for 30 minutes.

Note:

The Convergent Billing Invoice Design Tool is a separately licensed item.
