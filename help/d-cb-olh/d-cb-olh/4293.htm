How Do I - Product Configuration Tasks - Derived Attributes

Select the option most appropriate to the required task.

View a Derived Attribute

To view a derived attribute:

  1. Select the Maintenance Tariffs Derived Attributes Definition option. The Derived Attribute Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the defined derived attributes matching the specified criteria is displayed.
  4. Select the appropriate derived attribute and click the Detail... button, or double-click the appropriate derived attribute. The Derived Attribute Definition form is displayed in View Only mode.

Define a Derived Attribute (General Procedure)

To define a derived attribute:

  1. Select the Maintenance Tariffs Derived Attributes Definition option. The Derived Attribute Search form is displayed.
  2. Click the New... button. The Derived Attribute Definition form is displayed in Insert mode.
  3. Complete the data entry fields as required.

    Evaluation Type: Conditional: Single Value Returned

    Evaluation Type: Conditional: Multiple Values Returned

    Evaluation Type: List

    Evaluation Type: Simple Expression

    Evaluation Type: Table Lookup by Best Match

    Evaluation Type: Table Lookup by Multiple (1-10) Keys

    Evaluation Type: Table Lookup by Searching

    To define a derived attribute having an evaluation type of Table Lookup by Searching:

    1. View the Derived Attribute Definition form.
    2. Complete the data entry fields on the Details tab. Ensure that the Evaluation Type field is set to Table Lookup by Searching.
    3. Select the appropriate storage context.
    4. Complete the appropriate data entry fields.
    5. Select the Lookup Table tab. Select the appropriate entity validation definition in the Validation field, ensuring that the definition is defined with at least one index and one result column.
    6. If a user-specified wildcard is required, enter the wildcard value in the Wildcard String field on the Lookup Table tab. Otherwise, leave the field blank and undefined values are treated as wildcards.
    7. If required, enter the result column number and maximum number of rows.
    8. Select the Indexes tab. Enter the appropriate index expression.
    9. Click the Save button on the toolbar. If a storage context of Global or Global-Shared is used, continue with the procedure.
    10. Click the Table… button on the Lookup Table tab. The Derived Attribute Table form is displayed with the defined index and result columns.
    11. Populate the table with the required lookup (index) and result values.
      • If a user-specified wildcard is set in the Wildcard String field, ensure that the wildcards entered in the index columns match the value in the field.
      • If Wildcard String field is not set, any undefined index columns are treated as wildcards.
      • A row can be included in the derived attribute table with all index columns specified as undefined wildcards. This is the default row returned if no matches are found.
    12. Click the Save button on the toolbar.

Evaluation Type: List

To define a derived attribute with an evaluation type of List:

  1. View the Derived Attribute Definition form.
  2. Complete the data entry fields as required. Ensure that the Evaluation Type field is set to List.
  3. Select the appropriate storage context.
  4. Complete the appropriate data entry fields.
  5. If a storage context of Facility Group was selected, click the Save button on the toolbar and disregard the remainder of the procedure because the Lookup Table tab and Indexes tab are not displayed.
  6. Select the Lookup Table tab. Select the appropriate entity validation definition in the Validation field, ensuring that the definition is defined with a single index column, of any data type.
  7. If required, enter the result column number and maximum number of rows.
  8. Select the Indexes tab. Enter the appropriate index expression (the expression must contain an @value<data type> variable, where <data type> is the data type of the attribute type for the index column).
  9. Click the Save button on the toolbar. If a storage context of Global or Global-Shared was used, continue with the procedure.
  10. Click the Table… button on the Lookup Table tab. The Derived Attribute Table form is displayed with the defined index and result columns.
  11. Populate the table with the required lookup (index) and result values.
  12. Click the Save button on the toolbar.

Define a Master Derived Attribute

A derived attribute only becomes a 'master' when it is referenced by another (slave) derived attribute. To define a master derived attribute, define a derived attribute in the usual way.

Note:

The master derived attribute must not have an end date selected.

Define a Slave Derived Attribute

To define a slave derived attribute:

  1. View the Derived Attribute Definition form.
  2. In the Master Derived Attribute field, select the name of the derived attribute to act as the 'master' for this 'slave'. Details associated with the specified master derived attribute are displayed on the current form.

    Note:

    The Master Derived Attribute field is enabled only if one of the table lookup evaluation types is specified.

  3. Change the value of the Result Column Number field to a number other than one. The number one is used to identify the result column associated with the master derived attribute.
  4. Complete the data entry fields as required.

    Evaluation Type: Simple Expression

    Evaluation Type: Conditional: Single Value Returned

    Evaluation Type: Conditional: Multiple Values Returned

    Evaluation Type: List

    Evaluation Type: Table Lookup by Best Match

    Evaluation Type: Table Lookup by Multiple Keys

    Evaluation Type: Table Lookup by Searching

    Evaluation Type: Table Lookup by Single Range.

  5. Click the Save button on the toolbar.

Access a Derived Attribute Lookup Table

Convergent Billing provides facilities to:

  • List lookup tables (using the Derived Attribute Table Search form), to use a specific lookup table.
  • View a specific lookup table (using the Derived Attribute Definition form).

To search for a lookup table:

  1. Select the Maintenance Tariffs Derived Attributes Table Maintenance option. The Derived Attribute Table Search form is displayed.
  2. Specify search criteria and click the Search button. Lookup tables that match the specified criteria are listed in the lower part of the form.
  3. To view a specific lookup table in more detail, select the table name in the list of lookup tables and click the Detail... button, or double-click the lookup table name. The Derived Attribute Table form is displayed in View Only mode.

To view a specific lookup table:

  1. View the appropriate derived attribute.
  2. Click the Table… button on the Lookup Table tab. The Derived Attribute Table form is displayed in View Only mode.

Update the Index Expression Rows for a Derived Attribute

If entity validation is being used for a derived attribute and the number of columns or the data types of the columns have changed, the index expression rows on the Indexes tab of the Derived Attribute Definition form must be updated.

To update the index expression rows:

  1. View the appropriate derived attribute.
  2. Click the Update button on the toolbar. The Derived Attribute Definition form is displayed in Update mode.
  3. Click the Save button on the toolbar.

    A message is displayed indicating that the number of index rows defined for the validation selected is not the same as defined in the Indexes tab.

  4. Click the OK button to repopulate the indexes. The additional index rows are displayed in the Indexes tab.
  5. Configure the index expressions as required.
  6. Click the Save button on the toolbar.

This procedure can be followed for individual, master, and child derived attributes and must be done for each affected derived attribute definition. Updating the index expressions on a master definition does not automatically update the index expressions on its children.
