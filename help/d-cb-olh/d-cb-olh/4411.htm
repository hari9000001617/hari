How Do I - Operations - Bill Runs

Select the option most appropriate to the required task.

Perform a Bill Run using a Schedule

To perform a bill run using a schedule:

  1. Select the Server Tasks Schedule option. The Schedule Search form is displayed.
  2. Select the Create a new schedule option and click the New… button.
  3. The Schedule Definition form is displayed in Insert mode.
  4. Complete the appropriate fields on the form. For possible schedule types, refer to the System Operations Guide for Singleview Convergent Billing.
  5. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  6. Submit the schedule.

Perform a Bill Run using a One-off Task

To perform a bill run using a one-off task:

  1. Select the Server Tasks One-off Task option. The One-Off Task Definition form is displayed.
  2. Select the appropriate bill run from the Schedule Type drop-down list. The lower part of the form displays the parameters associated with the bill run.
  3. Enter the appropriate parameters for the bill run.
  4. Click the Save button on the toolbar. The bill run is saved and a one-off task is submitted.
  5. Click the Display Task… button to open the Task Detail form and monitor the progress of the bill run.
  6. When the bill run is complete, a notification is sent by email.

Modify a Bill Run

The method of searching for and viewing the details of a bill run depends on if the bill run was set up:

  • Using a schedule.
  • As a one-off task.

If the bill run was set up using a schedule:

  1. Select the Server Tasks Schedule option. The Schedule Search form is displayed.
  2. Enter or select the appropriate search criteria.
  3. Click the Search… button. A list of schedules matching the specified criteria is displayed.
  4. Select the appropriate schedule and click the Details… button or double-click the appropriate schedule. The Schedule Definition form is displayed in View Only mode.
  5. Click the Update button on the toolbar. The form is displayed in Update mode.
  6. Make the appropriate changes to the bill run.
  7. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  8. Select the appropriate submit option.

     

    Option

    Used To

    Regenerate tasks?

    Cancel any pending jobs associated with the original schedule. Regenerate tasks? is only used when modifying a schedule.

    Be notified on receipt?

    Direct the scheduler to open a window on the workstation to notify the user when the submitted schedule is received at the server.

    Be notified on completion?

    Direct the scheduler to open a window on the workstation to notify when the first scheduled task is completed. User must be logged in to Convergent Billing at the time the notification is sent for it to be displayed. A notification of the completion of any remaining tasks is performed using email (to the schedule's email address).

     

  9. Submit the schedule.

If the bill run was set up as a one-off task:

  1. Select the Server Tasks Bill Runs option. The Bill Run Search form is displayed.
  2. Enter or select the appropriate search criteria.
  3. Click the Search… button. A list of one-off tasks matching the specified criteria is displayed.
  4. Select the appropriate one-off task, and click the Details… button or double-click the appropriate one-off task. The Bill Run Detail form is displayed in View Only mode.
  5. Select the Detail tab, and in the Creation Task group, click the Details… button. The Task Details form is displayed in View Only mode.
  6. Click the Update button on the toolbar. The form is displayed in Insert mode.
  7. Make the appropriate changes.
  8. Click the Save button on the toolbar.

Note:

Entitlements that are aligned with customer bill cycles are dependent on the effective date of the bill run task. Changing the effective date of a bill run task may require a large number of entitlements to be rescheduled. The Task Details form detects entitlement dependencies and raises a warning if such a change is attempted.

View a Bill Run

To view bill run details:

  1. Select the Server Tasks Bill Runs option. The Bill Run Search form is displayed.
  2. Enter or select the appropriate search criteria.
  3. Click the Search… button.

    If the search returns only a single bill run, the Bill Run Details form is displayed in View Only mode (if the Always Display List? option is not selected).

    If a single bill run record is returned from a search, the details of that record are displayed on the Bill Run Detail form. If more than one bill run record is returned, the records are listed on the Bill Run List form. The required record can then be selected and its details displayed on the Bill Run Detail form.

    If selected, the Always Display List? option overrides the automatic display of the Bill Run Detail form when only a single record is retrieved; in such cases the Bill Run List form is displayed.

    If the search returns more than one bill run, or the Always Display List? option is selected, the Bill Run List form is displayed.

  4. Select the appropriate bill run and click the Details… button, or double-click the appropriate bill run. The Bill Run Details form is displayed in View Only mode.

Perform an Operation on a Bill Run

The Perform Operation… button is enabled only if the user has:

  • Sufficient privileges to access the One-Off Task Definition form
  • Schedule creation privileges for at least one of the operation tasks supported by the current bill run type.

To perform an operation on a bill run:

  1. View the appropriate bill run details.
  2. Click the Perform Operation… button. The Perform Operation form is displayed.
  3. Highlight the customer billing operation to be performed, and click the Submit button. The One-Off Task Definition form is displayed.
  4. Click the Save button on the toolbar to submit the task.
  5. Click the Display Task… button to open the Task Details form and monitor the progress of the task.
  6. When the task is complete, a notification is sent by email.

Display a List of Failed Customers

To display a list of customers that failed on the bill run:

  1. View the appropriate bill run details.
  2. Click the Failed Customers… button. The Failed Customer List form is displayed in View Only mode.
  3. To view more specific details on why the customer failed on the bill run, click the Details… button. The Bill Runs for Customer form is displayed in View Only mode.
