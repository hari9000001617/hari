Service Types

When creating a product offering for a customer, one or more services must be specified as part of a product offering. As the services available at any one time vary depending on locality and availability, it is impractical for a product administrator to define a product using specific services. For this reason, it is necessary to define the service components of a product more generically in terms of service types.

Defining the type (or category) of service has the following advantages over hardcoding a service into a product offering:

  • Flexibility

    The user can offer one of several equivalent services, rather than being restricted to a single service offering.

  • Maintainability

    The list of available services (having the specified service type) can be modified whenever required without affecting the product offering.

  • Easy introduction of new services

    New services can be introduced whenever they are required, without affecting the costing of the product offering.

  • Up-to-date

    The user is always able to select the current services associated with a specific service type.

  • Customisation

    The list of available services can be customised to reflect local conditions and current market trends.

Service types can also define generic and default values for the services associated with that type and to define any service attributes for a specific service. For example, default values for the types and quantities of equipment required by services of the specified service type can be defined.

Defining service types to be used in a product is a two step process:

  • Determine whether the service type exists.
  • Define a new service type (if required).

Note:

For inclusion in a product definition, it is only necessary to define the service type to be used; it is not necessary to define the services to be used as these have no bearing on how the product is defined or rated. The services associated with a service type are defined by a user when they create a service instance for each service type specified in the product instance.

When a companion product is created, the service type of the base product must also be included in the companion product's list of service types; otherwise, the companion products tariff is not applied. Other service types can be listed in the companion product definition, but the base product's service type is mandatory.
