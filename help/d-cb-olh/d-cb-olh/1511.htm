Online Help for Singleview Convergent Billing 9.1

How Do I - General Operations

This section describes general operations performed by all users.

Display/Hide Hint Text

To turn on the display of hint text for each field and field value hints for grid fields, select the Options Show Hints option (when selected, a tick is displayed to the left of the option).

To turn off the display of hint text for each field and field value hints for grid fields, deselect the Options Show Hints option.

To display the hint text for a field:

  1. Ensure that the display of hint text is turned on.
  2. Position the cursor over the field. Hint text for the field is displayed either below the field or below and to the right of the field.

To display the field value hints for a field in a grid:

  1. Ensure that the display of hint text is turned on.
  2. Position the cursor over the field in a grid. The field value for that field is displayed below the field.

Obtain Help on Error Messages

Whenever an error occurs, an error message is displayed in a pop-up window. For example, an error message is displayed if a mandatory field is not completed.

To obtain further details about the error:

  1. Click the Help button on the Error window. A Message Help window is displayed detailing the cause of the error and actions to correct it.
  2. After reading the details on the Message Help window, click one of the following:
    • Retain button to return to the Error window, leaving the Message Help window open for reference.
    • Close button to return to the Error window, closing the Message Help window.
  3. Click the OK button on the Error window.
  4. Fix the problem causing the error before continuing.

Refresh Security Settings

By default, any changes to security settings take effect only when next logging in to Convergent Billing. If the changes need to take effect immediately, update (refresh) security settings.

To update (refresh) the security settings for the current Convergent Billing session, do either of the following:

  • Select the File Refresh Security Settings menu option.
  • Press Ctrl-F5.

Change Convergent Billing Password

Note:

Setting a password is relevant only for users using internal authentication. Consult the system administrator to determine if passwords are:

  • Authenticated internally
  • Case-sensitive.

To change a Convergent Billing password:

  1. Select the File Change Password option. The Set/Change Password form is displayed.
  2. Enter current password in the Old Password field.
  3. Enter new password in the New Password and Verify New Password fields.
  4. Click the OK button.

The new password becomes effective at the next log in to Convergent Billing.

Caution:

If Process Engine is enabled, it is highly recommended that the password is updated in Process Engine using the User Tool to ensure integrity of additional Process Engine data.

Related Topic

Rules for Singleview Passwords

Setting a password is valid only for users using internal authentication.

Note:

Consult the system administrator to determine if passwords are:

  • Authenticated internally
  • Case-sensitive.

The new password must comply with the rules of the validation function associated with the security profile. The default rules require the new password to:

  • Be at least six characters in length.
  • Not include the @ or " characters.
  • Include at least one non-alphabetical character.
  • Be different from the old password.

There is no maximum limit on the number of characters in a password.

If password reuse rules are enabled by the System Administrator (in the user's profile), a password can be reused if the following conditions are met since the password was last used:

  • The password was changed the minimum number of times specified in the Reuse Max field.
  • The number of days specified in the Reuse Time field have elapsed.

Enter Data

To enter data, a form must be displayed in either Insert or Update mode.

Enter data into a field by:

  • Keying in the data.
  • Selecting an option on the drop-down list associated with that field.
  • Pasting copied data into the field.

Convergent Billing provides a number of ways to enter a date.

Insert Data in a Grid

Data Grids

On some forms, one or more records can be entered. For example, on the Customer Contacts form, details for a number of secondary contacts can be included. The details for each secondary contact are entered on a single row. To enter more than one secondary contact, a new row must be inserted into the list of entries and then the appropriate details entered.

As each row is a separate data record consisting of a series of fields, the list of records is referred to as a grid or table.

Field value hints are available to display the full contents of a grid field. Field value hints are useful for viewing data in fields not wide enough to display the full value.

To insert a new row, do either of the following:

  • Before the current grid row, click the Insert button on toolbar.
  • After the last grid row, position the cursor in the last grid row and press the Down Arrow key.

Whichever method is used, a new row is created to enter appropriate details.

Search for a Data Record

To perform a search:

  1. Select the menu option appropriate to the type of search required.
  2. Enter search criteria as required.
  3. Select a search method.
  4. Select a display format option, if required.
  5. Click the Search... button.

Tips for Searching

  • The more specific the search criteria entered, the fewer records returned.
  • Whenever possible, limit a search to the relevant record.

    For example, when searching for a query, progression, payment, invoice, or adjustment, enter the number of the query, progression, receipt, invoice, or adjustment.

  • Ensure that the record to be retrieved exists on the effective search date.

    By default, only records effective on the date that the search is performed are included in the search. On some search forms, an alternative date can be specified in the Effective Date or Search Date field, on which date records must be effective to be included in the search. Specifying a date is useful if the search is for records effective at a past date (that is, historical records), or records that were created, but become effective at a future date.

    On some search forms, the field can consist of either a single component (date), or two components (date and time). Granularity (date or date-time) of the field is set when Convergent Billing is installed. The default value for the field is affected by the granularity, and is described in the associated field help.

  • Searches are usually case-sensitive.

    Searches for customers and contacts (using the Customer Search and Person Search forms) are not case-sensitive. For example, entering a search string of 'Carol' on either the Customer Search or Person Search form, is equivalent to entering 'carol' or 'CaRoL'.

  • If unsure of the exact spelling of a person's name, select the Soundex search option (when available).

    The Soundex search option (available on the Customer Search and Person Search forms) is useful if the exact spelling of a customer's or person's name is not known.

    If this option is selected when the search is performed, any records that sound like the name entered in the search criteria are retrieved. For example, if the name 'Newtown' is entered in the Family Name search field on the Person Search form, and the Soundex option selected, records for 'Newton', 'Newtown', and 'Newtonne' could be retrieved.

  • Whenever possible, use the search options available on some search forms to restrict the search to specific records.

    For example, the Customer Query Search form can be used to specify that the search should include only active (open) queries, rather than both active and resolved (closed) queries.

  • If a search fails to retrieve the required record, perform at least one other search using alternative search criteria or the same criteria, but a different search method.

Specify Search Criteria

Search forms are used to specify one or more criteria on which to base a search.

Note:

Only data records that match the specified criteria are retrieved. Leaving the search criteria fields blank results in a general (non-specific) search being performed with possibly a large number of records being retrieved.

If a search that may result in a large number of records being retrieved is performed, a Confirm window asking to confirm that that search should continue is displayed.

To continue with the search, click the Yes button. If more than 100 records are retrieved, only the first 100 records are displayed.

To cancel the search, click the No or Cancel button and then enter more specific search criteria to narrow down the search. For example, rather than specifying only the customer type when searching for a customer (which retrieves every customer record having that customer type), specify the customer's name.

Specify a Search Method

The search method determines how stringent a search is.

The following table lists search methods.

 

Search Method

Description

Exact Match (default)

By default, only records that match the search string exactly are retrieved.

For example, if the search string 'Carol Bailey' is entered when searching for a customer, only the record that matches it exactly is retrieved.

Records such as 'Karol Bailey' or 'Carol Bayley' are not retrieved.

To perform a less stringent search, use the Wildcard search method.

Wildcard

To make a search less stringent and therefore increase the likelihood of the required record being retrieved, use one or more wildcards.

Convergent Billing supports the following wildcards:

  • _ (underscore), which substitutes for a single character.
  • % (percentage), which substitutes for zero or more characters.

Examples.

 

Single character substitution

Ba_ley

Retrieves any record starting with the character string 'Ba' or 'ba' and the 4th, 5th, and 6th characters being 'ley'.

_arol

Retrieves any record having the character string 'arol' immediately after the first character.

Multiple character substitution

Carol B%

Retrieves any record starting with the character string 'Carol B'.

C%y

Retrieves any record starting with the character string 'C' and ending in 'y'.

%ley

Retrieves any record ending with the character string 'ley'.

Combinations

%rol%

Retrieves any record containing the character string 'rol'.

_rol%

Retrieves any record where the 2nd, 3rd, and 4th characters in a character string are 'rol'.

 

SQL

The Structured Query Language (SQL) search method is allowed on some search forms.

Conditional criteria can be specified so that only records that match the specified conditions are retrieved. For example, an SQL search can be used to retrieve only records within a certain range of values.

This method is very powerful and can be complex. It requires good understanding of the syntax of the SQL WHERE clause.

Delete a Data Record

Restrictions on Deleting Data Records

A data record can usually only be deleted from the database if it is not referenced by other data in the database. For example, details of a contact cannot be deleted, if that contact is assigned to a customer.

Each Convergent Billing entity (for example, customers, contacts, products, services, and queries) is governed by its own set of rules and restrictions. For example, a contact cannot be deleted if that contact is not assigned to a customer and/or has no queries assigned to it.

In practice therefore, these restrictions mean that only a data record that has just been created can be deleted. In fact, the facility to delete a data record is provided only so that a data record entered in error can be deleted. Data records no longer active must be archived.

To delete a data record:

  1. Display the form relevant to the data record to be deleted in View Only mode.
  2. Click the Delete button on the toolbar. Assuming that the record is not referenced by another entity in the database, a message is displayed prompting confirmation that the specified record is to be deleted.
  3. Click the Yes button to delete the data record. A message that the record is deleted successfully is displayed.

Drag and Drop Items

The technique of dragging and dropping is used to select a graphical item, drag it to a new location, and then drop it. Drag and drop is used in Convergent Billing when the Customer Hierarchy form is displayed to:

  • Create a new customer record.
  • Assign a base product to an existing customer.
  • Assign a contract to an existing customer.
  • Create a new query for an existing customer.
  • Re-organise the relationship between customer nodes.
  • Move a base product between customer nodes.

In the first four cases, dropping the appropriate graphical item results in the corresponding form being displayed in Insert mode to allow the item to be created. For example, dragging the customer icon into the left-hand panel of the Customer Hierarchy form and dropping it results in the Customer Detail form being displayed. Similarly, dragging a product icon over a customer name in the left-hand panel and dropping it displays the Product Instance Detail form.

In last two cases, dragging and dropping simply involves changing the relationship between items stored in the Convergent Billing database.

Configure Caching for Performance

Client PCs require regular attention and configuration for optimum Convergent Billing performance.

For efficiency and performance, static data and operational queries can be cached to a local PC, speeding the execution next time data is selected. Caching follows the 'first in, first out' rule. 'First in, first out', also known as FIFO, means that when the cache is full, the oldest item in the cache is deleted first.

Associated Actions

Saving the Cache

When first logging in to Convergent Billing, the query cache is empty. The first time an activity is performed, the queries are reconstructed and executed to retrieve the required data. Therefore, the first time an activity is performed, it is slower than the subsequent executions of the same activity during that session.

Alternatively, the query cache can be saved to a file, increasing the performance of first use. The saved query cache is loaded automatically when Convergent Billing is started.

To save the cache to disk:

  1. Select the File Cache Status… option. The Cache Status form is displayed.
  2. Select the Save cache to disk? check box.

A saved cache is unique to the user who saves it, allowing multiple users to save caches on one PC.

Clearing the Cache

If the Save cache to disk? check box is not selected, the cache is cleared during every log-off operation. The cache needs to be cleared after configuration or security changes have been made.

The preferred methods of clearing the cache are to select the File Refresh Security Settings option or press Ctrl-F5. It is also possible to clear the cache by clicking the Clear Cache button on the Cache Status form.

  1. Select the File Cache Status… option. The Cache Status form is displayed.
  2. Select the Save cache to disk? check box.

A saved cache is unique to the user who saves it, allowing multiple users to save caches on one PC.

Configuring the Cache

The default size for the cache is 80.

Users who perform regular customer hierarchy searches and product instance creations need a cache size of '100'. All users can set the cache size to this level if desired.

When configuring functions, reference types, and attributes, it is advisable to set the cache size to '0'. A cache size of '0' alleviates the requirement to manually clear the cache by continually refreshing the security settings.

The cache details are found on the Cache Status form. To open the Cache Status form, select the File Cache Status… option from the menu options.

Note:

It is important to reset the cache to the recommended size of '100' when configuration is complete.

Starting or Stoping the Cache

The cache is on by default. To turn it off, set the Desired Cache Size to 0.

Save the Cache

When first logging in to Convergent Billing, the query cache is empty. The first time an activity is performed, the queries are reconstructed and executed to retrieve the required data. Therefore, the first time an activity is performed, it is slower than the subsequent executions of the same activity during that session.

Alternatively, the query cache can be saved to a file, increasing the performance of first use. The saved query cache is loaded automatically when Convergent Billing is started.

To save the cache to disk:

  1. Select the File Cache Status… option. The Cache Status form is displayed.
  2. Select the Save cache to disk? check box.

A saved cache is unique to the user who saves it, allowing multiple users to save caches on one PC.

Clear the Cache

If the Save cache to disk? check box is not selected, the cache is cleared during every log-off operation. The cache needs to be cleared after configuration or security changes have been made.

The preferred methods of clearing the cache are to select the File Refresh Security Settings option or press Ctrl-F5. It is also possible to clear the cache by clicking the Clear Cache button on the Cache Status form.
