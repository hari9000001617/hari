Attribute Type Definition Form

The Attribute Type Definition form is used to enter, view, and update an attribute type.

Controls the representation of an entity's attributes.

For users of Singleview Convergent Billing, an attribute type defines the format, characteristics, and validation rules for a form field.

Attribute types are the major components for entity validation. Based on a reference type, they also define the representation of an entity's attributes externally, allowing the entity's data model to be tailored for integration with external systems. They can also define additional non-stored attributes that are derived from an entity's stored attributes.

Attribute types are defined independently of the data entry fields to which they relate, which allows the same attribute type to be used for multiple fields.

Context: Convergent Billing, Process Engine

See also Derived Attribute, Derived Non-Stored Attribute, and Reference Type.

The upper part of the form displays, or allows entry of, basic details about the attribute type.

The lower part of the form consists of the following tabs:

Display

The Display tab is used to:

  • Specify how entered data is to be displayed in a field (for example, field label, field width, display format, an edit mask, and alignment of data entered).
  • Associate help with the field by entering hint text, and selecting a help entry definition.

Properties

The Properties tab is used to define the properties of a field by specifying the type of attribute type, using a various options. For example, the attribute type can be based on a reference type, SQL query, expression, data type, or the original field.

The attribute type can also be based on whether the field is mandatory, how the field is displayed (visible, enabled, can be updated), type of data that can be entered, restrictions on the data that can be entered, default value, and field validation.

Security

The Security tab is used to configure:

  • Fields to be masked or encrypted.

    Note:

    This option is not available for attribute types based on reference type, SQL query, or expression.

  • Field security to determine users who can view a field, users who can update a field, and whether a field is disabled or enabled for specified users.

SQL

The fields on the SQL tab are used to specify a general SQL statement that defines values that can be set for this attribute type. Defining the values is similar to defining a reference type in that it results in a drop-down list being displayed for the attribute type.

The SQL tab is enabled only when the SQL Query option is selected in the Attribute type is based on field.

When using an SQL query or expression, items can be defined as available or unavailable for selection from the drop-down list. To enable an item, return a numeric field as the final column in the query or expression array. If this column returns 1, the item appears in the drop-down list; otherwise, the item is hidden.

Expression

This tab is used to specify an expression that defines the possible values that can be displayed for the attribute type. Defining an expression is similar to defining a reference type in that it results in a drop-down list being displayed for the attribute type.

The Expression tab is enabled only when the Expression option is selected in the Attribute type is based on group on the Properties tab.

Integration

The Integration tab is used to define attribute types that tailor a Convergent Billing entity's data model for integration with external systems.

Configurers can:

  • Specify an attribute as a key field that, in conjunction with other key fields, identifies an entity instance.
  • Define an interface label that can be used to access a field of a entity.
  • Add a new derived non-stored attribute to an entity and defines its derivation.
  • Declare a field as a reference to another Convergent Billing entity.
  • Determine whether an attribute's value can be updated as an input, or read as an output.

Note:

The fields enabled on the Display, Properties, Security, SQL, and Expression tabs depend on the type of attribute being created (determined by the option selected in the Attribute type is based on group on the Properties tab).

Associated Actions

Viewing an Attribute Type

To view an attribute type:

  1. Select the Maintenance Data Definition Attribute Types option. The Attribute Type Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of attribute types matching the specified criteria is displayed.
  4. Select the appropriate attribute type and click the Detail… button, or double-click the appropriate attribute type. The Attribute Type Definition form is displayed in View Only mode.

Need more information on attribute types?

Defining an Attribute Type

To define an attribute type:

  1. Select the Maintenance Data Definition Attribute Types option. The Attribute Type Search form is displayed.
  2. Click the New… button. The Attribute Type Definition form is displayed in Insert Mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Need more information on attribute types?

Updating an Attribute Type

Caution:

Modifying an attribute type may result in corruption of data and/or forms that use the attribute type.

To update an attribute type:

  1. View the appropriate attribute type.
  2. Click the Update button on the toolbar. The Attribute Type Definition form is displayed in Update mode.
  3. Update the attribute type as required.
  4. Click the Save button on the toolbar.
  5. If the attribute type is used by another element, a Confirm window is displayed. Click the Yes button.

Need more information on attribute types?

Deleting an Attribute Type

Caution:

Deleting an attribute type may result in corruption of data and/or forms that use the attribute type.

To delete an attribute type:

  1. View the appropriate attribute type.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the attribute type is to be deleted.
