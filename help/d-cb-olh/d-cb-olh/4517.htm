Server Licence Checks

The Authentication server checks the licence when the BKR or BGP are started.

The Encryption Services Server (tresec) checks the licence when the server starts, to determine if encryption is enabled.

The licence status is checked at the start of every bill run by a call to biLicenceStatus&(). There is a minimum of 24 hours between licence entity checks by the BGP. If a bill run is started within 24 hours of the last licence entity check, the licence entity is not checked. If a bill run is started more than 24 hours after the last licence entity check, the licence entity is checked again. If the licence entity check fails for any reason, biInvoiceGenerate&() does not run.

The BKR requests a licence check every 24 hours from its first successful start. If the licence has expired, or if interference is detected by the Authentication server, an error is logged but the BKR does not shut down.

It is also recommended that the licence recalculation task be run every 24 hours.

If licence violation errors occur, perform the following steps:

  1. Check that the LICENCE configuration item attributes are correct; if necessary, restore them to the correct values.
  2. If the BKR still does not start, escalate the problem as required by company procedures. Intervention by CSG International may be required to generate a new licence.
