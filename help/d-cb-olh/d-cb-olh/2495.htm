Online Help for Singleview Convergent Billing 9.1

Data Records

Convergent Billing stores three types of record for each database entity:

  • Current Record (the record with a start date before or equal to today and an end date equal to or after today).
  • One or more historical records (records with an end date before today).
  • One or more future records (records with a start date in the future).

Records of each type can be updated provided sufficient update privileges are available.

Update privileges required for types of update:

Type of Update

Privilege Required

Used To

Normal

Update

Update current and future records.

Historical Update

Update an historical record, or the current record and backdate the changes.

Date range

Date Range Update

Modify the date range of an historical record.

 

Convergent Billing supports two main types of update: normal and date range updates.

 

Type of Update

Used To

Normal

Update details of a specified record, but restricts movement of the start and end dates to within the date range specified for that record.

Date range

Modify the date range specified for a record (that is, start date and end date of a record can be moved. Date range updates can only be performed using a history form.

 

Normal and date range updates to all record types (historical, current, and future) can be done, provided sufficient update privileges are available.

During day-to-day use of Convergent Billing, usually only current records need to be updated, and therefore only Update or Historical Update privileges are granted.
