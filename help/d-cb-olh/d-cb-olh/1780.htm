Customer Nodes in a Hierarchical Structure

Figure 1 displays customer nodes in a simple hierarchical structure.

 

Figure 1 Customer Nodes in a Hierarchical Structure

 

A relationship exists between Node A and the two nodes directly below it, namely Nodes B and C. In terms of the hierarchy, Node A is said to be the parent of Nodes B and C. Conversely, Nodes B and C are said to be children of Node A.

A similar parent-child relationship exists between Node B and Nodes D and E, and between Node C and Nodes F and G. Notice that Node B is both a child (of Node A) and a parent (of Nodes D and E). Similarly, Node C is both a child (of Node A) and a parent (of Nodes F and G).
