Schedule Definition Form

The Schedule Definition form is used to view, define, and maintain a schedule.

Defines when an activity (or task) will be executed and the frequency at which that task will be executed.

Context: Convergent Billing

See also Pre-schedule Count and Task.

The upper part of the form is used to enter general information about the schedule, such as its name and description. The upper part of the form also displays the name of the schedule from which this schedule was replicated (if applicable) and the instance on which it is run.

The lower part of the form contains the following tabs for entering more specific details about the schedule:

Settings

The fields on this tab are used to:
  • Specify schedule type, instance name, email address, pre-schedule count, repeat units, repeat type, schedule status, and schedule dates.
  • Replicate the schedule to a specified set of instances.

Note:

If email notification is not required, leave the Email Address field blank or set it to one of the following:

  • /dev/null
  • nobody.

Task Generation

The Task Generation tab contains information regarding the tasks being created by this schedule.

Prerequisites

The Prerequisites tab is used to manage prerequisite schedules for this schedule.

The tab contains a table of prerequisite conditions defined on this schedule, with each row in the table containing a schedule whose tasks form a prerequisite condition for running the tasks associated with this schedule.

The Search… and Assign buttons are used to add prerequisite schedules.

The Details… button displays further information about the selected prerequisite schedule.

The Start Task immediately when all prerequisites satisfied? check box is used to start the schedule as soon as all prerequisites are True rather than waiting for the scheduled start date, and can be selected even if even if no task prerequisites are specified.

Parameters

The fields on the Parameters tab are used to specify additional parameters for the schedule. The schedule type determines which fields are displayed on the tab. Some schedule types do not display any fields.

The form can be displayed in one of three form modes, with the current mode being indicated in the form’s title.

A form can be displayed in one of the following modes:

Form Mode

Description

View Only mode

Allows information held in Convergent Billing to be viewed, but not updated (modified).

Insert mode

Allows new information to be recorded (entered) in Convergent Billing.

Update mode

Allows information held in Convergent Billing to be viewed and updated (modified).
