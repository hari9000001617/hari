Account Type Definition Form

The Account Type Definition form is used to view, define, and update an account type.

Specifies any defaults or entity validation for accounts specified with that type. Account types are used in the definition of charge categories, payment types, adjustment types, and customer types to specify the type of account to which charges are applied.

Context: Convergent Billing, Customer Management, Configuration Studio

See also Account and Charge Category.

The upper part of the form can be used to specify:

  • Account type account class and account category

    Category (or subclass) of one of the available account classes a Convergent Billing account belongs to. Although account categories are extensible, Convergent Billing does not validate account categories against account classes.

    Specifies whether a Convergent Billing account is an asset or liability account. Account classes are not extensible.

    Context: Singleview

    See also Account Category, Account Type, and General Ledger Account Class.

  • Headings used in the Account History form to display charges and payments
  • Default currency

    Currency in which tariffs and accounts are calculated. The currency can be monetary, or points such as Frequent Flyer, Fly-Buys, or Barter points.

    See also Currency Conversion.

  • Default type of invoice
  • Expression to generate new account numbers

    Combinations of operators (arithmetic and relational) and variables that return a single value when evaluated. Variables used in an expression depend on the variable type, and the context and application environment in which the expression is being used. Use of direct variables in expressions also depends on the defined database view. Expressions are used in client and server processes.

    In Process Engine, expressions can be used in searches, rule validations, parameters, and derivations; they can also be used to conditionally trigger the execution of EPM scripts.

    Context: Convergent Billing, Process Engine

    See also Expression Block, Expression Parser Module (EPM), Expression Type, and Variable.

  • Entity validation to control the appearance of the Reservations tab of the Account History form
  • Function called when any of the balance-related fields of an account of this type are updated
  • Entity validation to control the appearance of customisable elements of the Account History form
  • Multiple accounts of this account type to be added to the same customer node.
  • Balance management for the reservation of funds for real-time tariffs.
    Tariffs used with entitlements to consume allocations in real-time. Real-time tariffs can be used to generate activation, cancellation, or recurring charges.

    Context: Convergent Billing, Commerce Engine

    See also Reservation, Balance Management, and Tariff.

The lower part of the form consists of the following tabs for specific details to be entered and viewed:

Aging Classification Criteria

The Aging Classification Criteria tab is used to:

  • Display or enter details to classify groups of transactions by arbitrary age brackets.
  • Set up display of groups.
  • Enter an expression to classify transactions.

Dispute

The Dispute tab is used to define how disputed amounts are calculated for accounts of the specified account type.

Need more information about disputes?

Details are displayed when the Account History form for an account associated with the account type is opened.

Associated Actions

Viewing an Account Type

To view an account type:

  1. Select the Maintenance Accounts Account Types option. The Account Type Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of account types that match the specified criteria is displayed.
  4. Select the appropriate account type and click the Detail... button, or double-click the appropriate account type. The Account Type Definition form is displayed in View Only mode.

Need more information on account types?

Defining an Account Type

To define an account type:

  1. Select the Maintenance Accounts Account Types option. The Account Type Search form is displayed.
  2. Click the New... button. The Account Type Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Need more information on account types?

Updating an Account Type

To update an account type:

  1. View the appropriate account type.
  2. Click the Update button on the toolbar. The Account Type Definition form is displayed in Update mode.
  3. Update the account type details as required.
  4. Click the Save button on the toolbar.

Need more information on account types?

Deleting an Account Type

To delete an account type:

  1. View the appropriate account type.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the account type is to be deleted.
