Variable Definition Search Form

The Variable Definition Search form is used to search for one or more variables, using a specific field, criteria, method, and date.

Named entity containing a value that is referenced in Convergent Billing. A variable is a field in a database record or a more complex expression based entity (containing other variables, calculated values, and constants).

Convergent Billing supports the following types of variable:

  • Local variables
  • Direct variables
  • Entity validation variables
  • Functions
  • Derived attributes
  • Subtotals
  • Tariffs.

Variables have specialised properties such as data type, context, and application environment, and are generally used in expressions.

Context: Convergent Billing

See also Application Environment, Context, Data Type, Derived Attribute, Direct Variable, Entity Validation, Expression, Function, Lists, Subtotal, and Tariff.

The lower part of the form is used to list each of the variables returned by the search that matches the specified criteria.

Multiple instances of the form can be displayed by clearing the Reuse form? check box.

Caution:

If multiple instances of the form are required for display (for example, to access details for several account types), do not select the Reuse form? check box.

If selected, the Reuse form? check box overrides display of information in a new form, and details are displayed in the existing form.
