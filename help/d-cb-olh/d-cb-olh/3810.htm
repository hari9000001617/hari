Statement Detail Form

The Statement Detail form is used to:

  • View details of a specified statement

    Document itemising charges (and possibly credits) accrued against a customer's account for services used by, or supplied to, that customer over a specified period. Unlike an invoice that itemises charges, a statement provides a record of the charges.

    Context: Convergent Billing

    See also Invoice and Reporting Level.

  • View an electronic image of the statement sent to the customer
  • Reprint the statement.

The upper part of the form summarises the statement details such as the statement number, associated customer account number, amount, date issued, and account to which the charges associated with the services listed on the statement are billed.

The upper part displays the effective date of the bill run that generated the invoice.

The lower part of the form consists of the following tabs, for more specific details to be viewed:

Transactions

The Transactions tab specifies details about the payments and adjustments displayed on the statement image (for example, date the transaction was completed, whether the transaction was an adjustment or payment, amount, and receipt number of the transaction).

Queries

The Queries tab specifies details about any queries raised against the statement, such as query number, type of query, when the query was opened and closed, and how the query was resolved.

Images

The Images tab is used to specify details about the statement image, such as the format of the statement, the name and address of the person who received the statement, and whether or not the statement has been printed.

General

The General tab is used to enter information relevant to a specific statement type.

Note:

The fields and number of fields on this tab depends on the statement type selected.

The form can only be displayed in View Only mode.

Associated Actions

Viewing a Statement

Two ways to search for a statement are to:

  • Perform a general statement search.
  • View the statement using the customer's account history.

To perform a general statement search:

  1. Select the Accounts Statements option. The Statement Search form is displayed.
  2. Use any one of the search criteria to restrict the number of records retrieved by specifying a wildcard search.
  3. Click the Search button. A list of the records that match the selection criteria is displayed on the lower part of the form. The list includes records for statement nodes and non-reporting nodes.

    The list of records retrieved when a non-specific search is performed includes a record for each statement node (a customer node that has a report level of Statement) and each non-reporting node (a customer node that has a report level of None) that match the selection criteria.

    For statement nodes, the amount displayed in the Statement Amount column is the 'rolled-up' amount displayed on the customer's statement. The 'rolled-up' amount includes the charges incurred at the statement node and the charges for any non-reporting nodes that are children of the specified statement node. For non-reporting nodes, the amount displayed in the Statement Amount column indicates only the charges incurred by that node.

  4. Double-click the required entry in the list or select the entry and click the Detail... button. The Statement Detail form is displayed.

To view a statement using the customer's account history:

  1. View the customer’s account history.
  2. Double-click the appropriate statement transaction in the list of transactions. The Statement Detail form is displayed with details of the specified statement.

Tip:

To locate the correct statement within the list of transactions, ask the enquirer for the date of the statement; the date is usually printed prominently on the statement in a predefined location.

Viewing Service Details

To view details for a generated statement:

  1. View the appropriate statement.
  2. Click the Statement Details... button. The Statement Item Details form is displayed.
  3. Enter the appropriate search criteria in the Invoice Hierarchy Search Criteria group and click the Search button. A list of entities matching the specified criteria is displayed in the Invoice Hierarchy section.
  4. Select the appropriate entity. The statement charges for the entity are displayed in the Item Details grid.

Viewing a Statement Image

Image Viewer Requirements

An image can be generated in a number of different formats; the appropriate image viewer must therefore be available. Consult the System Administrator about the relevant viewer used.

To view the image:

  1. View the appropriate statement.
  2. Select the Images tab and click the Statement Image... button.

Depending on whether the statement image is generated, Convergent Billing may ask whether the user wants to:

  • View the image

    Click the Yes button to display the image using the appropriate viewer, or the No button to regenerate the image.

  • Schedule a task to generate the image

    Click the OK button to schedule the appropriate task. A message indicating that the schedule was submitted to, and has reached, the server is displayed. When the scheduled task is complete, the appropriate viewer is run automatically to display the generated image.

    To cancel the request to schedule a task to generate the image, click the Cancel button.

Viewing the Associated Invoice

To view details of the invoice associated with the current statement:

  1. View the appropriate statement.
  2. Click the Display Invoice... button. The Invoice Detail form is displayed with details of the invoice for the account specified.
