Adjustment Batches

Entering adjustments in batches is simply an alternative method of entering and processing adjustments of the same adjustment type.

Once created, individual adjustments can be added to and removed from the adjustment batch. Each adjustment that is added to the adjustment batch is assigned an adjustment number when that adjustment is saved. The adjustment batch is assigned an adjustment batch number when the first adjustment within the adjustment batch is saved. While the adjustment batch is being created, the batch status is set to ‘Entered’.

Adjustments within an adjustment batch are posted to the relevant customer accounts only when the adjustment batch is applied. Once an adjustment batch is applied, the batch status is set to ‘Applied’. Details of individual adjustments within a batch can now be viewed.

Individual adjustments within an adjustment batch can be modified either before or after applying the batch.

If errors are discovered in an adjustment when that adjustment is posted (for example, the adjustment was posted to the wrong account), the adjustment batch can be un-applied (restrictions apply). Un-applying an adjustment batch results in all adjustments included in the batch being un-posted. Individual adjustments can then be modified and the adjustment batch re-applied. Facilities are provided to:

An adjustment batch can only be un-applied if an adjustment within that batch was not:

  • Included on (appeared on) an invoice or statement.
  • Posted to the Sales Journal.
  • Change the allocation of the adjustment to an invoice.
  • Reverse the adjustment.
  • Delete the adjustment.
