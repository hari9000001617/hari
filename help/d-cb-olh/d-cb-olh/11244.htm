GL Code Pairs Rules and Validation

Convergent Billing uses a general ledger (GL) guidance entity to map relevant charges to GL codes.

ID that each account in a general ledger is identified by, and sometimes known as an account code. GL codes are used to map transactions in Singleview to the appropriate account in the external general ledger system.

Context: Convergent Billing

See also Adjustment Type and Payment Type.

The type of the GL guidance entity is not explicitly displayed in the Convergent Billing client. Each type is distinguished by the combination of Earned But Unbilled (EBUB), Billed and Earned (BE), Billed but Unearned (BUE) GL, and Earned But Unbilled (EBUB) Pending Consolidation code pairs. For more information about GL guidance entity types, refer to the Finance Guide for Singleview.

A GL guidance entity has either one or two pairs of GL codes, depending on the type. Each GL code pair has a debit and a credit GL code. Refer to the following table for the valid combination of GL code pairs for each type of GL guidance entity.

In addition to the valid combinations of GL code pairs, the following rules apply to GL guidance entities:

  • All GL codes associated with a GL guidance entity must have the same reporting entity over the GL guidance entity's date range.
  • The Earned But Unbilled (EBUB) GL code pair cannot contain accounts receivable GL codes (an account category code of Receivable), and the debit GL code must be different from the credit GL code.

  • Only one of the Billed and Earned (BE) GL codes can be an accounts receivable GL code; however, an accounts receivable GL code does not need to be specified.
  • Only one of the Billed but Unearned (BUE) GL codes can be an accounts receivable GL code; however, an accounts receivable GL code does not need to be specified.
  • If both a Billed and Earned (BE) GL code pair and a Billed but Unearned (BUE) GL code pair are defined, and the debit GL code for one pair represents an accounts receivable GL code, then:
    • The debit GL code of the other pair must be the same GL code.
    • Neither of the credit GL codes can represent an accounts receivable GL code.

    If the credit GL code for one pair represents an accounts receivable GL code, then:

    • The credit GL code of the other pair must be the same GL code.
    • Neither of the debit GL codes can represent an accounts receivable GL code.
  • If the Billed and Earned (BE) GL codes are specified and the Earned but Unbilled (EBUB) GL codes are not specified, the Earned but Unbilled (EBUB) Pending Consolidation GL codes are enabled. This cannot be an accounts receivable code.
