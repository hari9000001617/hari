General Ledger Guidance

GL guidance is the term used in Convergent Billing to describe the process of determining the GL codes (and those accounts in the external general ledger system) affected by a transaction to allow revenue recognition over time. GL guidance allows the GL codes affected by a transaction to be determined when the Sales Journal posting process is run.

Note:

GL guidance is only used for normalised event and invoice transactions and not for payment and adjustment transactions.

GL guidance entities for a transaction are determined by the configurable GL guidance function. The GL guidance entities are associated with transactions, which define the GL codes affected by a transaction. GL guidance records are used to maintain a running history of the transactions that affect GL codes. The history is later used during the Sales Journal posting process.

For more information about GL guidance, refer to the Finance Guide for Singleview.
