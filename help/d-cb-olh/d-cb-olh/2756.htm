Data Interface Language Definition History Form

The Data Interface Language Definition History form is used to view a:

  • List of records held in the Convergent Billing database for the data interface language definition.

    Defines the composition of the event records that may be processed by the Event Normalisation (ENM) process and an implicit algorithm that allows the event record to be decoded by the interpreter.

    Context: Convergent Billing

    See also Alpha Event and Event Normalisation (ENM) Process.

    The list of records is displayed in chronological order and includes the current record, one or more historical records (if available), and one or more future records (if defined).

    Record having a start date in the future.

    Context: Convergent Billing, Customer Management

    See also Current Record and Historical Record.

    Record having an end date before today.

    Context: Convergent Billing

    See also Current Record and Future Record.

    Record having a start date before or equal to today, and an end date equal to or after today.

    Context: Convergent Billing, Customer Management, Process Engine

    See also Future Record and Historical Record.

  • Specific record and, if privileges are available, to update a specific record.

The form can be displayed in one of two form modes (View Only or Update mode), with the current mode indicated in the form’s title.

A form can be displayed in one of the following modes:

Associated Actions

Creating a DIL Definition

To define a DIL definition:

  1. Select the Maintenance Server DIL Specifications option. The Data Interface Language Search form is displayed.
  2. Click the New... button. The Data Interface Language Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields on the form.
  4. Click the Save button on the toolbar.

Note:

When a new or modified DIL definition is saved, the server validates the definition by performing an automatically-created one-off task. A message states that validation is in progress, and offers the opportunity of cancelling it. If cancelled, the DIL definition is not saved. When the validation is completed, the two possible outcomes are success or failure.

If the validation fails, the Task Detail form is displayed with the reasons for failure. The Data Interface Language Definition form remains open in Update or Insert mode.

If the validation succeeds, the changes are committed and a message appears on the status bar to this effect.

Viewing a DIL Definition

To view a DIL definition:

  1. Select the Maintenance Server DIL Specifications option. The Data Interface Language Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of the defined DIL definitions matching the specified criteria is displayed.
  4. Select the appropriate DIL definition and click the Detail... button, or double-click the appropriate DIL definition. The Data Interface Language Definition form is displayed in View Only mode.

Need more information on DIL Definitions?

Updating a DIL Definition

To update a DIL definition:

  1. View the appropriate DIL definition.
  2. Click the Update button on the toolbar. The DIL Definition form is displayed in Update mode.
  3. Update the DIL definition details as required.
  4. Click the Save button on the toolbar.

Need more information on DIL Definitions?

Deleting a DIL Definition

To delete a DIL definition:

  1. View the appropriate DIL definition.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the DIL definition is to be deleted.

Need more information on DIL Definitions?

Activating a DIL Definition

To activate a DIL definition, and attach the definition to a Normalised Event File Type:

  1. Select the Maintenance Server Normalised Event File Type. The Normalised Event File Type Search form is displayed.
  2. Enter any selection criteria to locate the appropriate Normalised Event File Type and click the Search button.
  3. Double-click the list entry of the Normalised Event File Type to be modified. The Normalised Event File Type Definition form is displayed in View Only mode.
  4. Click the Update button on the toolbar.
  5. Click the DIL Decoding Method option, and select the appropriate DIL specification from the drop-down list.
  6. Click the Save button on the toolbar.
