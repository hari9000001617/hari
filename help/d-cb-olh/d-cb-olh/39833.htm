Protected Data

Data in Convergent Billing can be protected in the following ways:

  • Encryption

    Data is encrypted in the database using a configurable algorithm.

  • Masking

    Data is fully or partially hidden using a configurable mask when viewed by a user.

Protected data is hidden in all outputs, including debug, print, and trace files, and appears as a default mask of six asterisks ('******').

Access to protected data is controlled by user security group privileges.

Data protection is applied to a field on a form by configuring attribute type settings that are implemented using entity validation.

When configuring an attribute type for encryption or masking, the following restrictions apply:

  • Data protection can only be implemented on string-based fields, and therefore can only be implemented on attribute types based on:
    • Data type
    • Original field.
  • If encryption is configured on an attribute type with a data type of string, the Max Characters field must be set.

When implementing an attribute type configured for encryption or masking using entity validation, the following restrictions apply:

  • The attribute type can only be implemented on string fields.
  • In a derived attribute table, only results columns should be associated with an encryption attribute type. Index columns are not supported for encryption.
  • If encryption is configured, the length of the encrypted value must not be greater than the size of the field.

For more information about data protection, refer to the Security and Auditing Configuration Guide for Singleview.
