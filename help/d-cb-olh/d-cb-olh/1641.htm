Composite Data Types

Composite data types can be of two types:

  • An array of scalar type
  • A string hash of scalar type.

The EPM enforces strict typing, so the appropriate type declaration character must suffix variable names. The following table lists examples.

 

Data Type

Description

Type
Declaration
Character

Examples

ARRAY

Order of sequence of elements of the declared type. Elements are stored and accessed using an integer 'index' value. The first element in an array has an index of 0.

[ ]

{ var a&[], b?[];
a&[0] :=10;
b&[0] [0] :=10;

HASH

Unordered set of elements of the declared type. Elements are stored and accessed using a string 'key' value. Also known as an associative array.

{ }

{ var a&{}. b?{};
a&{'First'} :=10;
b&{'First'} [0] :=10;
