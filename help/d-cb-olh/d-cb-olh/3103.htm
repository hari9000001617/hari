Payment Rejection Details Form

The Payment Rejection Details form is used to reject a payment recorded against a customer account, by specifying the date (defaults to today’s date), a code indicating the reason, and the affected credit and debit GL codes.

Amount paid by a customer. The payment is recorded against a specific customer's account and optionally against one or more specified invoices.

Context: Convergent Billing, Customer Management

See also Payment Batch and Payment Type.

When saved, a rejected payment is displayed as a debit in the account from which it is rejected. A rejected payment is given the same receipt number as the original payment.

Associated Action

Rejecting a Payment

To reject a payment the appropriate security privileges must be available.

Why Reject a Payment?

Payments may need to be rejected. For example, a payment must be rejected if the customer's cheque is returned by the bank having been dishonoured because funds are not available in the customer's account, or the cheque is missing a signature. A credit card transaction also might not be approved because the payment would exceed the customer's credit limit, so the associated payment needs to be rejected.

To resolve these situations so that the transactions in the customer's account history reflect the account status, the payment needs to be rejected.

Note:

Rejecting a payment does not remove that transaction from a customer's account history. It inserts an additional transaction, a debit transaction equal to the amount of the original payment. Rejecting a payment allows reversal of the effect of posting a payment to a customer's account.

Tip:

A payment can only be rejected when it has been posted to a customer's account. Therefore, a payment made as part of a payment batch can only be rejected if that batch is applied. Payments included in a payment batch that is not yet applied can be removed from the batch.

To reject a payment:

  1. View the appropriate payment.
  2. Click the Reject Payment... button. The Payment Rejection Details form is displayed.

    Note:

    If rejecting a payment retrospectively, it may be necessary to change the rejection date.

  3. Select a rejection code appropriate to the reason for rejecting the payment.
  4. Unless specifically told to do so, do not change the default values in the Credit GL Code and Debit GL Code fields.
  5. Click the Save button to reject the payment. Control is returned to the Individual Payment Detail form, where the Payment Status field is now set to 'Rejected'.

Rejected payments are displayed as debits on the Account History form. The rejected payment is given the same receipt number as the original payment.
