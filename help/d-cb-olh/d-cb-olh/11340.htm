Real-time Monitor (RTM)

The Real-time Monitor (RTM) allows the real-time collection of data and the execution of alarms in a Convergent Billing environment. Convergent Billing is deployed in environments where rapid detection and response to variation in critical metrics is required. For example, a Commerce Engine deployment, where Singleview interfaces directly with the network, might require real-time statistics analysis.

If enabled, the RTM polls at a configurable frequency (for example, every ten seconds) for event data. Event data can be selected from a number of sources (for example, UNIX machine statistics, Tuxedo performance statistics, Commerce Engine metrics). The event data is passed through a set of alarm conditions, which can be defined to trigger an action such as a UNIX script or EPM function (for example, to forward an email message). Event data is then written back to the tremon.

RTM event data held in the TRE monitor can be retrieved using GetStats.

RTM capability can be extended to provide a real-time graphical view of the statistics monitored by the RTM using a web browser. For more information, contact the CSG International Contact Centre.
