Online Help for Singleview Convergent Billing 9.1

Configuration Explorer

Convergent Billing is a highly configurable system designed to increase competitive advantage by making it easier to design, deliver, and bill the products and services customers require. Convergent Billing offers support for rapidly configuring new products and tariffs to meet customers’ changing requirements.

Configuring products involves a combination of service types, equipment types, tariffs, subtotals, facility groups, derived attributes, and product offers. The combination of these elements provides the flexibility required to deliver a product offering tailored to specific customer requirements.

Convergent Billing is also designed to meet the growing demands for a more personalised service. The customer care environment allows users to respond to new and existing customers promptly, consistently, and professionally. However, the system also provides the capability of customising the customer care forms to perform functions to manage specific business requirements. Administrators of the system therefore have control over the information captured by users.

Configuring the customer care environment involves a combination of reference types, attribute types, help entry definitions, entity validation, address formats, and customer care types.

In addition, comprehensive security features enable the control of access to information and the use of the system. Configuring security settings involves a combination of security groups, form roles, function roles, work groups, profiles, and user definitions.

To manage the extensive configuration options available, the system provides the capability of viewing the entire configuration in an environment similar to Windows Explorer or FrontPage Explorer. The Configuration Explorer (see the Configuration Fundamentals Guide for Singleview Convergent Billing) enables the viewing of the contents of a configuration object, all the references to the object, and the properties of the object. Viewing the configuration of the system from a central area enables administrators to:

  • Investigate the effects of changes to the configuration.
  • Quickly examine the entire configuration of a configuration object.
  • Customise the configuration from the Configuration Explorer.
  • Copy and print diagrams of the configuration.
  • Search for specific configuration information.
