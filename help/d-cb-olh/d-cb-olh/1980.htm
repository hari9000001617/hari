Adjustment Types

An adjustment type is used to define common parameters for adjustment records. Adjustment type examples are listed in the following table.

 

Adjustment Type

Description

Cancellation Charges

Apply cancellation charges.

Call Disconnected

Adjustment for a charge for a disconnected call.

Calls not recognised

Adjustment for calls listed on the invoice but not recognised by the customer.

 

The adjustment type relies on a number of items being defined and available in the system; these include the appropriate currency, general ledger codes, and account types.

If customising any of the fields on the Individual Adjustment Detail form for a specific adjustment type, the appropriate entity validation definition, attribute types, reference types, and context-sensitive help must be defined before the adjustment type.

For more information about adjustment types, refer to the Finance Guide for Singleview.
