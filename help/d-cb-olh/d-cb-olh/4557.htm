Negative Acknowledgement Message

Format

<length>|NACK|<seq nr>|0|0|

where:

  • <length> is total number of bytes in the message.
  • <seq nr> is the sequence number of the message not correctly received.

Example

00000023|NACK|7254|0|0|

Remarks

  • The rating engine waits for a user definable length of time to receive a response after transmitting a charge message. If no response is received during this period, the rating engine closes the named pipe or TCP/IP socket connection.

    Note:

    Reading events from a TCP/IP socket is deprecated and provided only for backward compatibility.

  • The value of the <seq nr> field should be identical to the value of the <seq nr> field in the charge message, which was transmitted. If these values are not identical, the rating engine ignores this negative acknowledgment.
  • If the values of the <seq nr> fields are identical, the rating engine re-transmits its most recent message. If another negative acknowledgment is then received, the rating engine again re-transmits the message. The rating engine continues in this manner until:
    1. An acknowledgment is received.
    2. The number of transmission attempts reaches a user-definable limit (in which case the rating engine closes the named pipe or TCP/IP socket connection).
    3. The named pipe or TCP/IP socket connection is closed by the network or by the remote host.
