Service Detail Form

The Service Detail form is used to enter, view, and update details of a service.

Provides a specific component of the overall product for which the customer is charged. Each service has an associated service type, and unique service number against which usage is charged. One or more facilities (service options) or items of equipment, or both, can be associated with each service.

Context: Convergent Billing

See also Equipment, Facility, and Service Type.

The upper part of the form displays basic details about the service, such as the service number, service type, service status, and dates when the service is required.

The lower part of the form consists of the following tabs for entering and viewing more specific details about the service:

Details

The Details tab is used to specify more explicit details about the service being created. The fields enabled on this tab are determined by the type of service being created.

Service Addresses

The Service Addresses tab contains the A-Address and B-Address tabs for entering address(es) to which the service is connected.

Contact

The Contact tab displays the details of the primary contact of the customer to whom this service is being assigned. Facilities are provided on this tab to search for and assign a different contact for this service.

Industry

The Industry tab is used for industry-specific fields associated with the specified service type. If fields are present on this tab, each field should be completed.

Contract

If the query relates directly to one of the customer's contracts, the tab can be used to specify that contract. Facilities are provided on this tab to search for and assign the appropriate contract to the query.

Billing

The Billing tab displays billing details associated with the service. The Priority field indicates the priority used to process the service's events during billing. Within the same customer node, services with a higher priority (lower numerical value) are processed before services with a lower priority (higher numerical value). The Complexity field indicates the time expected to perform a bill run on the service.

Lists

The Lists tab displays the lists that have been configured for the specified service type. The left-hand panel displays the type of list that can be selected and the right-hand panel displays the information associated with the selected list.

General

The General tab displays only fields associated with the specified service type.

Note:

Fields present on this tab should be completed.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

A form can be displayed in one of the following modes:

Associated Actions

Viewing a Service

To view summary information about each service:

  1. Display the Customer Hierarchy form for the appropriate customer.
  2. Select the Products and Services options in the Display for customer group. Summary information about each product instance and its associated services for the specified customer is displayed in the right-hand panel.

Creating a Service

To create a service:

  1. View the relevant product instance.
  2. Highlight one of the service types listed.
  3. Click the Create… button. A message is displayed prompting confirmation that the start and end dates for the product instance are correct.
  4. Click the Yes button to continue creating the service with the specified dates. The Service Detail form is displayed.
  5. If the dates are not correct, click the No or Cancel button to return to the Product Instance Detail form to change the dates.
  6. Complete the form for the selected service.
  7. Click the Save button on the toolbar. Control is returned to the Product Instance Detail form.
  8. To allocate an optional service to the product instance, click the Add Service… button. The Optional Product Component form is displayed.
  9. Select the optional component and click the Add button. The additional component is added to the Product Instance Detail form, the Optional Product Component form is closed, and control is returned to the Product Instance Detail form.
  10. Complete steps 2 to 6 for each additional component to be allocated to the product instance.
  11. Click the Save button on the toolbar.

Updating a Service

To update a service:

  1. View the appropriate service.
  2. Click the Update button on the toolbar. The Service Detail form is displayed in Update mode.
  3. Update the service details as required.
  4. Click Save on the toolbar.

Cancelling a Service

Caution:

  • Cancelling a service cancels the whole product instance.
  • Restrictions apply when cancelling a service.

To cancel a specific service:

  1. View the service using one of the available methods (summary or detailed).
  2. Click the Delete button on the toolbar. A Confirm window is displayed.
  3. Change the date, if required, and click the Yes button.

To cancel all services associated with a specific product instance, cancel the product instance.

The following applies when cancelling services:

  • Delete button is enabled only if the service can be cancelled within the service's record date range.
  • Clicking the Delete button for services that do not have a status of Product Creation, cancels the service.
  • Minimum cancel date is set to the earliest date at which the service becomes optional, and remains optional until the end of time. The earliest possible cancel date is the service record's effective start date.
  • Default cancel date is set as follows:
    • If the maximum number of permitted services for the service's item exceeds the number allowed by the product instance or the companion instance settings and remains at that level until the end of time, the earliest date at which this occurs is used.
    • If the service becomes optional at some future date and remains optional until the end of time, that date is used.
    • If the service is optional as of today's date and remains optional until the end of time, today's date is used.
  • If a service becomes optional, then mandatory, then optional again, only the last optional period can be cancelled.

Deleting a Service

Note:

Restrictions apply when deleting a service.

To delete a service:

  1. View the service using one of the available methods (summary or detailed).
  2. Click the Delete button on the toolbar. A Confirm window is displayed.
  3. Change the date, if required, and click the Yes button. The service status changes to Cancelled.

Note:

Using the Delete button is the preferred method for deletions. The following applies:

  • Clicking the Delete button for services that have a status of Product Creation deletes the service.
  • The date from which to delete the service must be equal to the effective start date of the service.

Creating a List

To create a list:

  1. Display either of the following:
    • Customer Detail form for the appropriate customer
    • Service Detail form for the appropriate service
  2. Click the Lists tab.
  3. Click the Update button on the toolbar.
  4. Select the appropriate list from the left-hand panel of the tab.
  5. Complete the fields on the right-hand side of the tab.
  6. Click the Save button on the toolbar.
