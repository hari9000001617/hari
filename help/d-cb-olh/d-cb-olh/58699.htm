Service Type Definition Form

The Service Type Definition form is used to enter, view, and update a service type.

Specifies any defaults or entity validation for services specified with that type. If required, an icon can be specified to identify each service type.

Context: Convergent Billing, Configuration Studio

See also Icon and Service.

The upper part is used to enter general information about the service type, such as its name, description, identifying icon, and service form to be used.

The lower part containing the following tabs is used for entering more specific details about the service type:

Settings

The Settings tab is used to specify the GL code, address format for A and B parties, cancellation status for services of this type if an associated product offer is cancelled, and entity validation details that can be assigned for the service type, allowing the service details displayed for this service type to be customised.

The use of the GL code specified on this tab is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance.

Charge Category

The Charge Category tab is used to override the behaviour of the default charge category to be used to create a charge category instance when a service of a specified type is created. For example, the tab can be used to override the GL code to which charges are posted so that the GL code selected is based on a specific service type.

Note:

The field on this tab cannot be used to define a new charge category.

The use of the GL code specified on this tab is deprecated and provided for backward compatibility only. GL codes are assigned to charges using GL guidance.

Equipment

The Equipment tab is used to define the default types and quantities of equipment required for the specified service type.

The types and quantities specfied on this tab are displayed in the equipment section in the lower half of the Services tab of the Product Definition form when the specified service type is selected for inclusion in a product definition. The default types and quantities can be overridden on the Services tab of the Product Definition form.

Derived Attribute Tables (Lists)

The Derived Attribute Tables (Lists) tab is used to specify the service context derived attributes required for the specified service type.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

A form can be displayed in one of the following modes:

Associated Actions

Viewing a Service Type

To view a service type:

  1. Select the Maintenance Products Service Types option. The Service Type Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of service types that match the specified criteria is displayed.
  4. Select the appropriate service type and click the Detail... button, or double-click the appropriate service type. The Service Type Definition form is displayed in View Only mode.

Need more information on service types?

Defining a Service Type

To define a service type:

  1. Select the Maintenance Products Service Types option. The Service Type Search form is displayed.
  2. Click the New... button. The Service Type Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields. If required, override the default charge category.
  4. Click the Save button on the toolbar.

Need more information on service types?

Updating a Service Type

To update a service type:

  1. View the appropriate service type.
  2. Click the Update button on the toolbar. The Service Type Definition form is displayed in Update mode.
  3. Update the service type details as required.
  4. Click the Save button on the toolbar.

Need more information on service types?

Deleting a Service Type

To delete a service type:

  1. View the appropriate service type.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the service type is to be deleted.
