Individual Payment Form

The Individual Payment form is used to enter, view, and update payment details.

Amount paid by a customer. The payment is recorded against a specific customer's account and optionally against one or more specified invoices.

Context: Convergent Billing, Customer Management

See also Payment Batch and Payment Type.

The upper part of the form contains a set of read-only fields in which information about the payment, such as the date it is recorded and its current status, is displayed.

The lower part of the form consists of the following tabs for entering or viewing details about the current payment:

Details

The Details tab is used to specify payment type, payment location, payment amount, date the payment was received, and the account to which the payment is to be posted. The payment type specified determines the currency in which the payment is recorded and determines whether additional information needs to be entered for the payment using fields on the General tab.

Set the credit account explicitly by entering a value in the Credit Account Number field or implicitly by entering a value in the Invoice Number or Adjustment Number field. Tab out of the field to implicitly set the Credit Account Name field and, if not set, the Currency field.

A default is set for the Amount field equal to the:

  • Balance of the account, if the Credit Account Number field is entered but not the Invoice Number field.
  • Total outstanding amount for the invoice, if the Credit Account Number field was set by specifying the Invoice Number field.
  • Total outstanding amount for the adjustment, if the Credit Account Number field was set by specifying the Adjustment Number field.

For each of these default values, if the account is a liability account, the sign of the value is reversed before the value is displayed.

The Details tab is also used to enter the name of the person making the payment. By default, the name of the primary customer contact for the specified customer account is used.

When a payment is recorded, this tab provides access to facilities that allow the payment receipt to be previewed and printed.

General

The fields displayed on the General tab depend on the value set in the Payment Type field on the Details tab. Fields are often not displayed on this tab when the payment type is set for cash payments.

Default values for the Debit Account Details group may be set, depending on the payment type selected. If the payment type does not specify a debit account type, the Account Number field cannot be entered; otherwise, it can be edited.

Note:

Details should be entered in all fields displayed on this tab.

Allocations

The Allocations tab is used to:

  • Display invoices or adjustments for the account in the Credit Account Number field on the Details tab.
  • Allocate payment to one or more of the displayed invoices and adjustments.

Note:

The Allocations tab is displayed when all the following conditions are met:

  • The user's form role includes the View Invoice Allocations access level for the Individual Payment form.
  • The Credit Account Number field on the Details tab specifies a prime account.
  • Either the Currency or Amount field on the Details tab contains a value.

A Receivable Type column is displayed only if an invoice on the tab is divided into receivable types or a credit amount is assigned to receivable types.

The tab displays items according to these rules:

  • An invoice with a zero total amount outstanding is displayed if the invoice has receivable types with non-zero outstanding amounts.
  • If an invoice is divided into multiple receivable types, each division is displayed on a separate row in the grid with the Receivable Type column. Otherwise, the invoice is displayed in a single row, with an empty Receivable Type column.
  • If an invoice has a receivable type, the Total Amount and Amount Due columns display the values for the receivable type. Otherwise, the invoice values are displayed.
  • If a credit amount is divided into multiple receivable types, each receivable type is shown on a separate row with empty invoice-related columns empty. Any credit amount not assigned to a receivable type is not displayed in a row on the tab.

Inserting or Updating a Payment

When inserting or updating a payment, the tab displays:

  • Invoices
    • Outstanding invoices, including invoices with an issue date after the value entered in the payment's Date Received field.
    • Invoices paid as part of this payment.
  • Adjustments
    • Outstanding adjustments with a status of Applied.
    • Adjustments that have been paid as part of this payment.

Note:

All outstanding positive invoices or negative adjustments must be paid before any payment amount is left unallocated as credit.

The payment can be allocated in the following ways:

  • To a single invoice, adjustment, or payment
  • Split between multiple invoices and adjustments
  • Credited to a specified account.

Note:

A payment amount can be entered against each invoice but the value must be between zero and the value of the Amount Due field inclusive.

The Amount column on the Allocations tab can be edited if the Update Invoice Allocations access level is selected for the Individual Payment form in a user's form role.

Debit Account Allocations

The Debit Account Allocations tab displays invoices or adjustments associated with the debit account on the General tab, and can allocate the payment to them.

Note:

The Debit Account Allocations tab is displayed when all the following conditions are met:

  • The user's form role includes the View Invoice Allocations access level for the Individual Payment form.
  • The Debit Account Number field on the Details tab specifies a prime account.

The tab operates like the Allocations tab, with the following differences:

  • The total of the invoice and adjustment allocations and credit amount equals the negative of the payment amount.
  • All outstanding invoices or adjustments do not have to be paid before any payment amount can be left unallocated as credit.

Note:

The Amount column on the Debit Account Allocations tab can be edited if the Update Invoice Allocations access level is selected for the Individual Payment form in a user's form role.

Up-Front Charges

The Up-Front Charges tab lists any up-front charges associated with the specified account, the amount of each up-front charge, and the total of all up-front charges. New up-front charges can be entered and existing up-front charges can be modified or deleted.

Note:

The Up-Front Charges tab is displayed only if a Credit Account Number is specified.

Currency Conversion

The Currency Conversion tab is used to display conversion details of a payment.

The Conversion Date field defaults to the Date Received field on the Details tab and is updated when the Date Received field is modified. If the Conversion Date field is updated, it is not updated when the Date Received field is modified.

The tab is displayed only if the currency of the payment is different from the currency of the account.

Note:

A receipt number is assigned automatically by Convergent Billing when the payment is saved.

Associated Actions

Viewing a Payment

Search for and view details of a receipted payment using one of the following:

  • Receipt number
  • Account number into which the payment was made.
  • Name of the customer's account.

To search using a receipt number:

  1. Select the Accounts Payments Manage Individual Payments option. The Payment Detail Search form is displayed.
  2. Enter the receipt number in the Receipt Number field.
  3. Click the Search… button.

If the search returns:

  • A single record, the Individual Payment Detail form is displayed (if the Always Display List? option was not selected).
  • Multiple records, or the Always Display List? option is selected, the Payment Detail List form is displayed.

    To access the Individual Payment Detail form from this form, double-click the appropriate payment in the list of retrieved payments, or select the appropriate payment and click the Details… button. The Individual Payment Detail form is displayed detailing the specified payment.

To search using an account number into which the payment was made:

  1. Select the Accounts Accounts History option. The Account History form is displayed.
  2. Enter the customer's account number in the Account Number field.
  3. Click the Refresh History button. Details of all transactions associated with the specified account are listed.
  4. To view the details of a specific payment, double-click the appropriate payment, or highlight the payment and click the Detail... button. The Individual Payment Detail form is displayed detailing the specified payment.

To search for the customer account into which the payment was made:

  1. View the appropriate customer’s details.
  2. Click the Account History… button. The Account History form for the specified customer is displayed.
  3. To view the details of a specific payment, double-click the appropriate payment, or highlight the payment and click the Details... button. The Individual Payment Detail form is displayed detailing the specified payment.

Entering a Payment

To enter a payment:

  1. Select the Accounts Payment New Individual Payment option. The Individual Payment Detail form is displayed in Insert mode.
  2. Enter the payment details on the Details tab.

    The payment type selected determines the currency in which the payment is recorded and whether additional information needs to be entered in the fields on the General tab. The payment type's GL codes are displayed on the General tab.

    The Credit Account Number and Invoice Number fields are used for the account or invoice number to which the payment is being posted. Enter the account or invoice number and press the Tab key. The account or invoice number is verified against accounts held in the database and, if valid, the corresponding account name, account number, outstanding invoice number, and amount are displayed. If invalid, a warning message is displayed in the status bar. To rectify this, press the Delete key and enter a valid account or invoice number.

    If the payment is being made against a specific account (by entering the account number in the Credit Account Number field) the value that is automatically displayed in the Amount field does not include any unapplied payments or adjustments (for example, future-dated payments).

    If the payment is being made against a specific invoice (by entering the invoice number in the Invoice Number field) the value that is automatically displayed in the Amount field includes unapplied payments and adjustments.

    The date-time in the Date Received field defaults to the current date-time. The date is overridden by entering the required date either directly or using the drop-down calendar. The time is overridden by entering the required time; this may need to be done if the payment is entered retrospectively. If a future date is entered, the payment is not applied until the specified date.

    The Paid by group specifies the person who made the payment. If required, search for and assign a different contact as the payee (use the Search... and Assign buttons located in the Paid by group).

    For payments, the Invoice Number field reflects the current status. The word 'Credit' is displayed if there are no outstanding invoices, 'Split' is displayed if the payment was applied to a number of outstanding invoices or if the payment is allocated to one or more invoices and an amount is left over in credit.

  3. Select the General tab, and complete the appropriate data entry fields.
  4. Select the Invoices tab, and allocate the amount paid to one or more of the listed invoices.
  5. If appropriate, select the Debit Account Invoices tab and enter the appropriate amounts.
  6. If required, select the Up-Front Charges tab and enter the appropriate item and amount.
  7. Click the Save button on the toolbar.

    Clicking Save generates a receipt, assigns a receipt number to the payment, saves the payment details, and posts the payment amount to the specified account. Control is returned to the Individual Payment Detail form.

  8. If required, print the receipt.

Tip:

Rather than entering a number of payments individually, consider creating a batch for each type of payment and adding payments as they are received to the relevant payment batch. The payment batch can then be applied at the end of a predefined period (for example, at the end of the day, or at the end of a week). An organisation's business practices and billing cycles determine whether this is viable, and if so, the frequency at which payment batches need to be applied.

Rejecting a Payment

To reject a payment the appropriate security privileges must be available.

Why Reject a Payment?

Payments may need to be rejected. For example, a payment must be rejected if the customer's cheque is returned by the bank having been dishonoured because funds are not available in the customer's account, or the cheque is missing a signature. A credit card transaction also might not be approved because the payment would exceed the customer's credit limit, so the associated payment needs to be rejected.

To resolve these situations so that the transactions in the customer's account history reflect the account status, the payment needs to be rejected.

Note:

Rejecting a payment does not remove that transaction from a customer's account history. It inserts an additional transaction, a debit transaction equal to the amount of the original payment. Rejecting a payment allows reversal of the effect of posting a payment to a customer's account.

Tip:

A payment can only be rejected when it has been posted to a customer's account. Therefore, a payment made as part of a payment batch can only be rejected if that batch is applied. Payments included in a payment batch that is not yet applied can be removed from the batch.

To reject a payment:

  1. View the appropriate payment.
  2. Click the Reject Payment... button. The Payment Rejection Details form is displayed.

    Note:

    If rejecting a payment retrospectively, it may be necessary to change the rejection date.

  3. Select a rejection code appropriate to the reason for rejecting the payment.
  4. Unless specifically told to do so, do not change the default values in the Credit GL Code and Debit GL Code fields.
  5. Click the Save button to reject the payment. Control is returned to the Individual Payment Detail form, where the Payment Status field is now set to 'Rejected'.

Rejected payments are displayed as debits on the Account History form. The rejected payment is given the same receipt number as the original payment.

Deleting a Payment

To delete a payment not yet posted to a customer's account:

  1. View the appropriate payment.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the payment is to be deleted.

A payment posted to a customer's account can also be deleted using the same procedure if:

  • The payment is not included on (appeared on) an invoice or statement.
  • The payment has not been posted to the sales journal.
  • Security privileges to do so are available.

Viewing a Payment’s History of Changes

When working with a payment batch, it is possible to view information about a selected payment's history (for example, that a payment was rejected and later accepted).

To view the history of any changes to a payment:

  1. View the appropriate payment batch.
  2. Select the appropriate payment from the lower part of the form.
  3. Click the Changes… button.
  4. The Summary of Changes form is displayed in View Only mode.
