Online Help for Singleview Convergent Billing 9.1

Tariff Definition Form

The Tariff Definition form is used to enter, view, and update details of a tariff. The upper part of the form contains basic details about the tariff, such as the name, description, and period for which the tariff is applicable.

Rental (recurring, activation, and cancellation) events are configured using the Recurring, Activation, and Cancellation tabs, which are only displayed when the appropriate class (on the Settings tab) is selected.

The lower part of the form consists of the following tabs for entering and reviewing more specific configuration details:

Settings

The Settings tab is used to:

  • Define settings for the tariff being configured (for example, class, type, and application environment in which the tariff applies)
  • Select check boxes (whether this is a global tariff and whether it is billable)
  • Define the invoice section number and associated text.
  • Currency to be used for all charges generated by this tariff.

    By entering a currency expression, the components of the tariff can be evaluated in different currencies.

The Recurring, Activation, and Cancellation options in the Class group can only be selected for an application environment of Rating.

Click the Eligibility button to display the Expression List form to enter expressions that control the application of the tariff.

Specifies expressions that control the application of the tariff. If the list is empty or all expressions evaluate to TRUE, the tariff is applied.

Which variables can be used in eligibility expressions depends on the application environment and context of the tariff as follows:

  • Application environment of Billing:

    All variables with an application environment of Billing or Rating are available, except those with a context of Charge if the tariff's context is not Charge. If a keyed charge is defined, the key variable name (defined in the Variable Name field on the Charges tab) is available.

  • Application environment of Rating:

    Variables must have the same application environment and if the tariff is:

    • Global (Global Event? is selected), only variables with the same context are available.
    • Not global (that is, a product tariff), variables with a context of Service or Normalised Event are available, irrespective of the tariff's context.

Conditions

The Conditions tab is used to define the conditions on which the tariff table is searched. Each condition entered defines a column in the resulting tariff table.

To add an entry (a row of data defining the condition) to the table, click the Insert button and enter details for the condition. To rearrange the order of the conditions, use the Up and Down buttons.

To view the tariff table, click the Table... button.

Charges

The Charges tab is used to specify the charge and charge expressions to be used to calculate the value of a tariff if the conditions (specified on the Conditions tab) are met.

To add an entry (a row of data defining the charge) to the table, click the Insert button and enter details for the charge. To rearrange the order of the charges, use the Up and Down buttons.

To view the tariff table, click the Table... button.

Accounting

The Accounting tab is used to specify:
  • Accounting expressions used to return GL codes and account numbers for the tariff.
  • Whether the account is included in charges calculated in the rating process for the specified account.
  • Account check expressions for actions to be taken following rating.

The expression entered in the selected field can be used to dynamically assign the account numbers and GL codes to the costed records they generate. Any expressions entered override the charge category specified in the Settings tab, and the charge categories specified in the Tariff tab of the Product Definition form for non-global tariffs.

The Real-time Account Aggregation group is used to configure Convergent Billing to maintain a record of unbilled amounts (current amounts outstanding) for an account. The unbilled amounts are used to provide real-time account balance management for Commerce Engine and real-time tariffs, and are displayed on the Account History form in the Unbilled Amount (or Current Amount) field. An expression can be specified to check the account balance and perform a specific action if it exceeds a pre-set amount.

Note:

The From GL Code and To GL Code fields on the Accounting tab are deprecated and provided for backward compatibility only.

When the GL subsystem is enabled (the GL_ENABLED reference type has a single enabled reference code), the fields are only enabled for billable tariffs. The GL codes are assigned when a charge is created. However, the fields do not affect GL guidance or the sales posting process, but can be accessed by the GL guidance function. The GL codes used during the sales posting and GL upload process are determined using General Ledger Guidance.

The From GL Code and To GL Code fields are enabled if the GL subsystem is enabled.

Recurring

The Recurring tab is displayed only when the Recurring option is selected in the Class group on the Settings tab.

The tab is used to enter parameters used by the RGP to generate an event that recurs on a periodic basis for an entity (equipment, service, facility group, or product instance). For example, parameters can be defined for a line or appliance rental that is billed quarterly. A generated event that spans multiple charge periods can optionally be split into multiple events, according to charge periods and a prorate period. Parameters can be defined to prorate a recurring event by a fixed date.

The charges generated for a recurring event can be defined on the Charges tab of the tariff, or of any tariff that applies to the event.

The tab also provides the option of generating the recurring event for inclusion in an interim invoice (generated by running the Interim Bill Run schedule type), without affecting the regular invoice cycle of the customer. A recurring event can also be generated for periods where a product has a status other than Active.

The generation of an event is controlled by the following expression lists:

  • Recurring/Activation/Cancellation Event Eligibility Expression List

    Specifies conditions that control the generation (by the RGP) of a rental event for an entity (product, facility group, service, or equipment). If the list is empty, or all conditions are TRUE, an event is generated. Otherwise, if any expression in the list is FALSE, an event is not generated.

    Depending on the entity the tariff is associated with, expressions can include direct variables from one of the following sets:

    • Direct variables based on RGP_PRODUCT_INSTANCE_V
    • Direct variables based on RGP_FAC_GROUP_INSTANCE_V
    • Direct variables based on RGP_SERVICE_HISTORY_V
    • Direct variables based on RGP_EQUIPMENT_HISTORY_V.

    Expressions can also include functions with an application environment of Any, Rating, or Data Entry.

    Caution:

    A function with an application environment of Rating or Data Entry can be used within the expression list; however, rather than all direct variables available for the relevant application environment, only the direct variables referenced above are valid.

    The eligibility expression lists do not run in a rating application environment, hence they do not have access to standard direct variables, tariffs and derived attributes. Derived attribute variables must not be used in this context. To access derived attribute details in the expression list, use the appropriate functional interface:

    • DerivedTableLookupByDatex&() for access to global derived attributes.
    • ServiceTableLookup&() for access to service context derived attributes.
    • CustomerNodeTableLookup&() or CustomerNodeTableLookup?[]() for access to customer node or customer derived attributes.

    For more information about these functions, refer to the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2).

  • Recurring/Activation/Cancellation Event Assignment Expression List

    Specifies the expressions that the RGP uses to assign values to the generated event for the entity (product, facility group, service, or equipment). Fields of the event are assigned values by referring to direct variables based on the RGP_NORMALISED_EVENT_V view. For example, the C_PARTY_NAME field is set by assigning a value to the CPartyName$ direct variable.

    Depending on the entity the tariff is associated with, expressions can include direct variables from one of the following sets:

    • Direct variables based on RGP_PRODUCT_INSTANCE_V
    • Direct variables based on RGP_FAC_GROUP_INSTANCE_V
    • Direct variables based on RGP_SERVICE_HISTORY_V
    • Direct variables based on RGP_EQUIPMENT_HISTORY_V.

    Expressions can also include functions with an application environment of Any, Rating, or Data Entry.

    Caution:

    A function with an application environment of Rating or Data Entry can be used within the expression list; however, rather than all direct variables available for the relevant application environment, only the direct variables referenced above are valid.

    The assignment expression lists do not run in a rating application environment, hence they do not have access to standard direct variables, tariffs and derived attributes. Derived attribute variables must not be used in this context. To access derived attribute details in the expression list, use the appropriate functional interface:

    • DerivedTableLookupByDatex&() for access to global derived attributes.
    • ServiceTableLookup&() for access to service context derived attributes.
    • CustomerNodeTableLookup&() or CustomerNodeTableLookup?[]() for access to customer node or customer derived attributes.

    For more information about these functions, refer to the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2).

    Assignment expressions are evaluated only for rental events. Adjustment events are created by copying the values from the rental event being adjusted.

    Tip:

    It is recommended that all data that affects the eligibility of the rating tariff or the content of the generated charge be included in the generated event (by the assignment expressions). Including this data enables a tariff's eligibility criteria and charge expressions to avoid accessing RGP_SERVICE_HISTORY_V. Depending on the cause of the adjustment, rental tariffs that access details in RGP_SERVICE_HISTORY_V may see the current details, rather than the details that caused the adjustment event to be generated.

  • Recurring/Activation/Cancellation Event Adjustment Expression List

    Specifies the conditions that control the generation (by the RGP in adjustment mode) of an adjustment event for a previous rental event for an entity (product, facility group, service, or equipment). If the list is empty, or all conditions are TRUE, an adjustment event is not generated. Otherwise, if any expression in the list is FALSE, an adjustment event is generated.

    Depending on the entity the tariff is associated with, expressions can include direct variables from one of the following sets:

    • Direct variables based on RGP_PRODUCT_INSTANCE_V
    • Direct variables based on RGP_FAC_GROUP_INSTANCE_V
    • Direct variables based on RGP_SERVICE_HISTORY_V
    • Direct variables based on RGP_EQUIPMENT_HISTORY_V.

    An expression can include direct variables based on RGP_NORMALISED_EVENT_V.

    Expressions can also include functions with an application environment of Any, Rating, or Data Entry.

    Caution:

    A function with an application environment of Rating or Data Entry can be used within the expression list; however, rather than all direct variables available for the relevant application environment, only the direct variables referenced above are valid.

    The adjustment expression lists do not run in a rating application environment, hence they do not have access to standard direct variables, tariffs and derived attributes. Derived attribute variables must not be used in the context. To access derived attribute details in the expression list, use the appropriate functional interface:

    • DerivedTableLookupByDatex&() for access to global derived attributes.
    • ServiceTableLookup&() for access to service context derived attributes.
    • CustomerNodeTableLookup&() or CustomerNodeTableLookup?[]() for access to customer node or customer derived attributes.

    For more information about these functions, refer to the Configurer's Reference Guide for Singleview Convergent Billing (Vol. 2).

Activation

The Activation tab is displayed only when the Activation option is selected in the Class group on the Settings tab.

The tab is used to:

  • Enter parameters to be used by the RGP to generate a one-off event on activation of an entity (equipment, service, facility group, or product instance).

    The charges generated for an activation event can be defined on the Charges tab of the tariff, or of any tariff that applies to the event.

  • Provide an option to generate the activation event for inclusion in an interim invoice (generated by running the Interim Bill Run schedule type), without affecting the regular invoice cycle of the customer.

The generation of an event is controlled by the following expression lists:

  • Recurring/Activation/Cancellation Event Eligibility Expression List
  • Recurring/Activation/Cancellation Event Assignment Expression List
  • Recurring/Activation/Cancellation Event Adjustment Expression List

Cancellation

The Cancellation tab is displayed only when the Cancellation option is selected in the Class group on the Settings tab.

The Cancellation tab is used to enter parameters to be used by the RGP to generate a one-off event on cancellation of an entity (equipment, service, facility group, or product instance). The charges generated for a cancellation event can be defined on the Charges tab of the tariff, or of any tariff that applies to the event.

The tab also provides the option of generating the cancellation event for inclusion in an interim invoice (generated by running the Interim Bill Run schedule type), without affecting the regular invoice cycle of the customer.

The generation of an event is controlled by the following expression lists:

  • Recurring/Activation/Cancellation Event Eligibility Expression List
  • Recurring/Activation/Cancellation Event Assignment Expression List
  • Recurring/Activation/Cancellation Event Adjustment Expression List

Associated Actions

Viewing a Tariff Definition

To view a tariff definition:

  1. Select the Maintenance Tariffs Tariffs Definition option. The Tariff Definition Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of tariff definitions matching the specified criteria is displayed.
  4. Select the appropriate tariff definition and click the Detail... button, or double-click the appropriate tariff definition.
  5. The Tariff Definition form is displayed in View Only mode.

Need more information on tariffs?

Defining a Tariff Definition

To define a tariff definition:

  1. Select the Maintenance Tariffs Tariffs, Definition option. The Tariff Definition Search form is displayed.
  2. Click the New... button. The Tariff Definition form is displayed in Insert mode.
  3. Complete the appropriate data entry fields.
  4. Click the Save button on the toolbar.

Need more information on tariffs?

Updating a Tariff Definition

To update a tariff definition:

  1. View the appropriate tariff definition.
  2. Click the Update button on the toolbar. The Tariff Definition form is displayed in Update mode.
  3. Update the tariff details as required.
  4. Click the Save button on the toolbar.

Need more information on tariffs?

Deleting a Tariff Definition

Caution:

Deleting a tariff definition also deletes the associated tariff table.

To delete a tariff definition:

  1. View the appropriate tariff definition.
  2. Click the Delete button on the toolbar.
  3. When prompted, click the Yes button to confirm that the specified tariff definition is to be deleted.
