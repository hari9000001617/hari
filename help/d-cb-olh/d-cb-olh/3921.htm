Task Details Form

The Task Details form is used to enter, view, and update details about a task.

Action or operation scheduled to be performed on behalf of a client by the Scheduled Task Dispatcher (STD) process. The STD process generates the required tasks based on the schedule definition. A task is executed as a server process by the Task Launch and Monitor (TLM) process.

Context: Convergent Billing

See also Schedule, Scheduled Task Dispatcher (STD) Process, and Task Launch and Monitor (TLM) Process.

The upper part of the form displays the task ID, task status, status reason, and the schedule name.

The lower part of the form consists of the following tabs, for viewing more specific details:

Details

The Details tab displays a summary of standard information about the task. The following fields are displayed: Email Address, Instance Name, Effective Date, Scheduled Start Date, Baseline group with Start Date, End Date, Duration (secs), Maximum Duration, Purge Period (days), Expected/Actual group with Start Date, End Date, and Duration (secs).

From the Details tab, it is possible to resubmit or stop a task and view the details of resubmitted, original, or dependent tasks associated with the task.

Parameters

The Parameters tab displays parameters passed to the server process (if any). Any parameters specified are derived from the schedule that created the task, but they can be modified before the task is run, or when the task is resubmitted.

Clicking the Bill Run… button displays the Bill Run Detail form with the bill run for this task.

Note:

The Bill Run… button is enabled only if:

  • The form is in view mode.
  • The task created a bill run, or the task is associated with a schedule whose schedule type has a task type of Bill Run Operation, Bill Run Customer Operation, or Bill Run Operation (Prerequisite).
  • The task status is Success, Failure, or Running.

Prerequisites

The Prerequisites tab displays any prerequisite tasks assigned to the current task. From the Prerequisites tab, it is possible to search for and assign tasks as prerequisite to the current task and configure the conditions that must be satisfied on the prerequisite tasks before the current task can progress.

The Start task immediately when all prerequisites satisfied? check box allows the task to start as soon as all prerequisites are true rather than waiting for the Scheduled Start Date. This can be selected even if no task prerequisites are specified.

Resources

The Resources tab displays the details of resources that have been or will be consumed while the current task is running. The Dependent Resource group within the Resources tab displays details of the resource that resulted in the current task’s status.

Results

For bill runs, customers suppressed from billing, using either the Customer Detail form or Billing Suppress schedule type, are identified as Skipped in the Results tab. Customers excluded from the bill run for configuration reasons (for example, invoice due amount does not reach a predetermined limit) are designated as Suppressed in the Results tab.

Clicking the Report… button displays the report generated by the task.

The form can be displayed in one of three form modes, with the current mode indicated in the form’s title.

Associated Actions

Viewing a Task

To view a schedule:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of schedules matching the specified criteria is displayed.
  4. Select the appropriate schedule and click the Details... button, or double-click the appropriate schedule. The Schedule Definition form is displayed in View Only mode.

Need more information on schedules?

Schedules are required to run tasks at specific times and frequencies in Convergent Billing. Schedules are used to manage all billing tasks such as the rating of events, and the preparation and printing of invoices. Schedules are also used to generate reports. Tasks can be set up to run automatically at predefined intervals or at a specified time and date. The schedule definition is determined in part by the associated schedule type.

When setting up a schedule, an instance on Convergent Billing can be specified to run the schedule. If an instance is not specified, the schedule is run on the default instance defined for the scheduler. (The default instance is defined in the STD configuration item.)

Preconfigured functions can be automatically executed on commencement or completion of a scheduled task, and notification messages provided to assist with system operations.

A schedule specifies:

  • When a task is to be run for the first time.
  • The Convergent Billing instance the schedule is run on.
  • The effective date (used as a base to compute the effective date for tasks that are created).
  • How often the task should be repeated.
  • What program the task needs to perform its function, and what parameters should be passed to it.
  • Selection parameters such as whether a report should include data on all customer nodes or only include data on a specific customer node.
  • Display parameters, such as whether the data included in a report is to be sorted, the sort order, and the sort sequence.

Creating a Schedule

To define a schedule:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Select the Create a new schedule option and click the New... button. The Schedule Definition form is displayed in Insert mode.
  3. Complete the data entry fields in the upper part of the form.
  4. Complete the data entry fields on the Settings tab.
  5. Complete the data entry fields on the Parameters tab. Some of fields on this tab may be optional, depending on the associated schedule type.
  6. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  7. Submit the schedule.

Scheduling an Immediate Task

To schedule an immediate one-off task:

  1. Select the Server Tasks One-off Task option. The One-off Task Definition form is displayed.
  2. Select the schedule type from the drop-down list. The lower part of the form displays the parameters associated with the schedule type.
  3. Enter any parameters that may be required by the schedule.
  4. Click the Save button on the toolbar. The schedule is saved and a one-off task is submitted.
  5. Click the Display Task… button to open the Task Details form and monitor the progress of the task created.

When the task is complete, a notification is sent by email.

Scheduling a Future One-off Task

To schedule a one-off task:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Select the Create a new schedule option and click the New... button. The Schedule Definition form is displayed in Insert mode.
  3. Complete the data entry fields in the upper part of the form.
  4. Select the schedule type from the drop-down list. The Parameters tab is populated with the parameters associated with the schedule type.
  5. Complete the fields on the Settings tab.
  6. Complete the fields on the Parameters tab.
  7. Click the Save button on the toolbar. The Submit Schedule form is displayed.
