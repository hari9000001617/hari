Online Help for Singleview Convergent Billing 9.1

Submit Schedule Form

The Submit Schedule form is used to:

  • Submit one or more schedules.

    Defines when an activity (or task) will be executed and the frequency at which that task will be executed.

    Context: Convergent Billing

    See also Pre-schedule Count and Task.

  • Submit a specific schedule (using the Submit button) and to cancel the submission of the schedule (using the Cancel button).
  • Regenerate all tasks associated with the schedule.
  • Select the notification options on schedule receipt and completion.

The three Submit options are used to:

 

Option

Used To

Regenerate tasks?

Cancel any pending jobs associated with the original schedule. Regenerate tasks? is only used when modifying a schedule.

Be notified on receipt?

Direct the scheduler to open a window on the workstation to notify the user when the submitted schedule is received at the server.

Be notified on completion?

Direct the scheduler to open a window on the workstation to notify when the first scheduled task is completed. User must be logged in to Convergent Billing at the time the notification is sent for it to be displayed. A notification of the completion of any remaining tasks is performed using email (to the schedule's email address).

 

  • Cancel associated pending jobs.
  • Request notification of the received schedule.
  • Request notification of the first completed, scheduled task.

Associated Actions

Viewing a Schedule

To view a schedule:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Enter or select the appropriate search criteria and search method.
  3. Click the Search button. A list of schedules matching the specified criteria is displayed.
  4. Select the appropriate schedule and click the Details... button, or double-click the appropriate schedule. The Schedule Definition form is displayed in View Only mode.

Need more information on schedules?

Schedules are required to run tasks at specific times and frequencies in Convergent Billing. Schedules are used to manage all billing tasks such as the rating of events, and the preparation and printing of invoices. Schedules are also used to generate reports. Tasks can be set up to run automatically at predefined intervals or at a specified time and date. The schedule definition is determined in part by the associated schedule type.

When setting up a schedule, an instance on Convergent Billing can be specified to run the schedule. If an instance is not specified, the schedule is run on the default instance defined for the scheduler. (The default instance is defined in the STD configuration item.)

Preconfigured functions can be automatically executed on commencement or completion of a scheduled task, and notification messages provided to assist with system operations.

A schedule specifies:

  • When a task is to be run for the first time.
  • The Convergent Billing instance the schedule is run on.
  • The effective date (used as a base to compute the effective date for tasks that are created).
  • How often the task should be repeated.
  • What program the task needs to perform its function, and what parameters should be passed to it.
  • Selection parameters such as whether a report should include data on all customer nodes or only include data on a specific customer node.
  • Display parameters, such as whether the data included in a report is to be sorted, the sort order, and the sort sequence.

Creating a New Schedule

To define a schedule:

  1. Select the Server Tasks Schedules option. The Schedule Search form is displayed.
  2. Select the Create a new schedule option and click the New... button. The Schedule Definition form is displayed in Insert mode.
  3. Complete the data entry fields in the upper part of the form.
  4. Complete the data entry fields on the Settings tab.
  5. Complete the data entry fields on the Parameters tab. Some of fields on this tab may be optional, depending on the associated schedule type.
  6. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  7. Submit the schedule.

Scheduling an Immediate One-off Task

To schedule an immediate one-off task:

  1. Select the Server Tasks One-off Task option. The One-off Task Definition form is displayed.
  2. Select the schedule type from the drop-down list. The lower part of the form displays the parameters associated with the schedule type.
  3. Enter any parameters that may be required by the schedule.
  4. Click the Save button on the toolbar. The schedule is saved and a one-off task is submitted.
  5. Click the Display Task… button to open the Task Details form and monitor the progress of the task created.

When the task is complete, a notification is sent by email.

Monitoring a Schedule

To monitor a schedule:

  1. View the appropriate schedule.
  2. Click the Tasks… button. The Task List form is displayed.
  3. Select the appropriate task and click the Details... button, or double-click the appropriate task. Details of the selected task are displayed in the Task Details form.

Stopping a Schedule

To stop a schedule:

  1. View the appropriate schedule.
  2. Click the Update button on the toolbar. The Schedule Definition form is displayed in Update mode.
  3. Set the Schedule Status field to 'Complete'.
  4. Click the Save button on the toolbar. The Submit Schedule form is displayed.
  5. Submit the schedule.

Deleting a Schedule

To delete a schedule:

  1. View the appropriate schedule.
  2. Click the Delete button on the toolbar.
  3. Confirm that the schedule is to be deleted.
