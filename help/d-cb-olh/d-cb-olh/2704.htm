Customer Search Form

The Customer Search form is used to search for one or more customers, based on specified search criteria on either the Details or Address tab.

The Address tab is used to enter the customer’s address (either in full or in part) so that it can be used as one of the search criteria.

The fields in which to enter the customer’s address are displayed only when an address format is selected (in the Format field). The address format determines the layout (number and type of fields) of the address.

If a customer type is selected in the Details tab, the format defaults to the address format specified in the customer type.

The Details tab is used to:

  • Refine a search by specifying search criteria such as customer name, account number, customer type, service number, network number, and one or more primary IDs.

    Only records matching all criteria specified are retrieved by the search and displayed on the Customer Hierarchy form.

  • Enter information relevant to the customer to be created.

    Any information entered is included on the Customer Detail form when it is displayed in Insert mode.

By default, all records active on the date the search is performed are included in the search. The Search Date field can be used to search for historical records active at a specified time, or future records.

The display format for the retrieved record(s) can be chosen by selecting or clearing the Always Display Hierarchy check box.

If a single customer record is returned from a search, that customer’s details are displayed on the Customer Detail form. If more than one customer record is returned, the records are listed in the left-hand panel of the Customer Hierarchy form. The required record can then be selected and its details displayed on the Customer Detail form.

If selected, the Always Display Hierarchy? option overrides the automatic display of the Customer Detail form when only a single record is retrieved; in such cases the Customer Hierarchy form is displayed.

A Soundex option is also included to allow a name similar to the one being searching for to be entered, if exact details are unavailable.

Note:

Searches based on customer name are case-insensitive.

Selecting the Search for customers option enables the Search… button on the form, to search for customers matching the specified criteria. Selecting the Create a new customer option enables the New… button on the form, for creating a customer.

The display format for the retrieved records can be determined by selecting or clearing the Always Display Hierarchy? and Reuse Customer Form? check boxes.

If multiple instances of the Customer Detail form are required for display (for example, to access details for several customers) the Reuse customer form? option should not be selected.

Caution:

If selected, the Reuse customer form? option overrides the display of the customer's details in a new form; in such cases, the customer's details are displayed in the existing form.

Multiple instances of the Customer Hierarchy form can be displayed by clearing the Reuse Last Hierarchy? check box.

If multiple instances of the Customer Hierarchy form are required for display (for example, to focus a search for a specific customer) the Reuse last hierarchy? option should not be selected.

Caution:

If selected, the Reuse last Hierarchy? option overrides the display of the customer hierarchy in a new form; the customer hierarchy is displayed in the existing form.

Associated Actions

Viewing Customer Details

To view a customer's details:

  1. Select the Customer Care Customers Manage Customer option. The Customer Search form is displayed.
  2. Enter the name of the customer in the Customer Name field.
  3. Enter additional search criteria in the other fields, if required.
  4. If part or all of the customer's address is available, select an address format from the drop-down list, then enter the details.
  5. Select one of the following options:
    • Always Display Hierarchy?
    • Reuse Last Hierarchy? option to list customers on single or multiple instances of the Customer Hierarchy form.
  6. Select the Search for customers option and click the Search… button.

    If the search returns:

    • A single customer record, the Customer Detail form is displayed (if the Always Display Hierarchy? option was not selected).
    • More than one customer record, or the Always Display Hierarchy? option was selected, the Customer Hierarchy form is displayed listing each of the customer records that matched the search criteria in a new form. If the Reuse Last Hierarchy? option was selected, the customer records are displayed on the last Customer Hierarchy form.
  7. To view:
    • Items associated with a specific customer record, click the customer name in the left-hand panel; the items (contacts, products, services, equipment, queries, and contracts) associated with that customer are listed in the right-hand panel (if all available display options are selected).

       

      Example 1

       

      To list all queries associated with the current customer node, select the Open Queries and Closed Queries display options.

      Example 2

       

      To list all components associated with the current customer node, select all thirteen display options.

       

    • Details of a specific customer record, double-click the appropriate customer record, or select the record and click the Details… button. The Customer Detail form is displayed in View Only mode.

Creating a New Customer

Customer details are recorded on the Customer Detail form, which can be accessed by:

  • Using the Customer Search form
  • Dragging and dropping the customer icon into the left-hand panel of the Customer Hierarchy form.

Although both methods can be used to create customers, the second method is primarily used to create customer nodes within a customer hierarchy.

To create a customer using the Customer Search form:

  1. Select the Customer Care Customers New Customer option. The Customer Search form is displayed.
  2. Enter the appropriate search criteria on the Details tab. If part or all of the customer's address is available, click the Address tab and select an address format from the drop-down list, then enter the address details.
  3. Ensure that the Create a new customer option is selected and click the New… button. The system performs a search of the database and then displays either the Customer Detail form or the Customer List form.

    When the New… button on the Customer Search form is clicked, a search of the database is performed based on the information specified on the search form.

    • If no records are found that match the criteria, the Customer Detail form is displayed in Insert mode.
    • If one or more records that match the information about the search form are found, those records are displayed on the Customer List form. Details of each record (using the Details… button) can then be displayed, and if an appropriate match does not exist, a new record can be created by clicking the New… button on the Customer List form; the Customer Detail form is then displayed in Insert mode.

    Any information entered on the Customer Search form is inserted into the appropriate fields on the Customer Detail form irrespective of the mechanism by which the Customer Detail form is displayed.

    If the Customer List form is displayed, view each of the records displayed. If an appropriate match does not exist, click the New… button on the Customer List form. The Customer Detail form is displayed in Insert mode.

  4. Complete the basic customer details in the upper part of the form.
  5. Select and then complete each tab relevant to the customer type specified.
  6. Click the Invoice Formats… button on the Invoicing tab. The Customer Invoice Format form is displayed.
  7. Select an invoice format.
  8. Click the Save button on the toolbar.

To create a customer using the technique of dragging and dropping:

  1. Display the Customer Hierarchy form.
  2. To create a:
    • root customer node, drag and drop the Customer icon onto the Hierarchy node (located at the top of the left-hand panel).
    • child customer node, drag and drop the Customer icon onto the appropriate customer node in the left-hand panel.

      Node that is a direct subordinate (or child) of a node (referred to as a parent customer node) located at the next highest level of the customer hierarchy.

      Context: Convergent Billing, Customer Management

      Note:

      A child customer node can also be the parent customer node of a node (or nodes) located at a lower level in a customer hierarchy.

      See also Customer Hierarchy, Parent Customer Node, and Root Customer Node.

    In both cases, the Customer Detail form is opened in Insert mode.

    As the icon is dragged into the left-hand panel of the Customer Hierarchy form, the cursor changes from depicting a stop symbol to depicting an arrow. If the icon is dropped when the stop symbol is displayed, no action is taken by the system.

     

    Symbol

    Description

    This cursor symbol is used to indicate that the customer icon can be dropped onto one of the customer nodes (listed in the left-hand panel of the Customer Hierarchy form) or the hierarchy node.

    This cursor symbol is used to indicate that dropping the customer icon will be ignored by Convergent Billing; that is, no action will be taken.

     

  3. Complete the basic customer details in the upper part of the form.
  4. Select and then complete each tab relevant to the customer type specified.
  5. Click the Save button on the toolbar.

Deleting a Customer

Note:

If the customer account is being finalised, the customer record can be deactivated (cancelled) rather than being deleted.

To delete a customer's record:

  1. Cancel each product instance associated with the customer to be deleted.
  2. View the appropriate customer's details.
  3. Click the History button on the toolbar. The Customer History form is displayed.
  4. Click the Delete button on the toolbar.
  5. Click the Save button on the toolbar.
